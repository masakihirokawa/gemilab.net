## マルチターンチャットとは

通常の `generateContent()` 呼び出しは1回のやりとりで完結しますが、チャットアプリではユーザーとAIが複数回のやりとりを続ける必要があります。Gemini API の `startChat()` メソッドを使えば、会話の文脈を自動的に維持しながら対話を続けられます。

この記事では、マルチターンチャットの基本から、履歴管理、System Instructions との組み合わせ、実用的なチャットボット構築まで段階的に解説します。

---

## 基本的なマルチターンチャット

### シンプルな対話の実装

```python
# basic_chat.py
# Gemini API で基本的なマルチターンチャットを実装

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

# チャットセッションを開始
chat = model.start_chat()

# 1回目のメッセージ
response1 = chat.send_message("Pythonでリストの重複を削除する方法を教えて")
print("AI:", response1.text)

# 期待出力:
# AI: リストの重複を削除するにはいくつかの方法があります:
# 1. set() を使う: list(set(my_list))
# 2. dict.fromkeys() を使う: list(dict.fromkeys(my_list))
# ...

# 2回目のメッセージ（前の文脈が維持される）
response2 = chat.send_message("それぞれの方法の計算量は？")
print("AI:", response2.text)

# 期待出力:
# AI: 先ほど紹介した各手法の計算量を比較します:
# - set(): O(n) — ハッシュテーブルを利用
# - dict.fromkeys(): O(n) — 順序を保持しつつ重複除去
# ...

# 会話履歴を確認
for message in chat.history:
    print(f"{message.role}: {message.parts[0].text[:50]}...")
```

`start_chat()` で作成されたチャットセッションは、送受信したすべてのメッセージを `history` プロパティに自動で蓄積します。2回目の「それぞれの方法」という質問が正しく解釈されるのは、1回目の質問と回答が文脈として維持されているからです。

## 会話履歴を指定してチャットを開始する

過去の会話を復元したい場合、`start_chat()` に `history` パラメータを渡せます。

```python
# resume_chat.py
# 保存済みの会話履歴からチャットを再開

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

# 過去の会話履歴を用意
history = [
    {"role": "user", "parts": ["私はWebデザイナーです。最近AIツールに興味があります。"]},
    {"role": "model", "parts": ["Webデザイナーの方に便利なAIツールはたくさんあります。デザイン生成、コーディング補助、画像編集など、どの分野に興味がありますか？"]},
    {"role": "user", "parts": ["コーディング補助に興味があります。"]},
    {"role": "model", "parts": ["コーディング補助のAIツールでは、GitHub Copilotが有名です。HTML/CSSの補完やJavaScriptのロジック提案をリアルタイムで行えます。"]},
]

# 履歴付きでチャットを再開
chat = model.start_chat(history=history)

# 文脈を踏まえた質問ができる
response = chat.send_message("それ以外のおすすめは？")
print(response.text)

# 期待出力:
# GitHub Copilot 以外にも、Webデザイナー向けのコーディング補助ツールがあります:
# - Cursor: VS Code ベースのAIエディタ...
# - Claude Code: コマンドラインからコード生成...
# ...
```

この機能を使えば、データベースに保存した会話履歴を読み込んでセッションを再開できます。

## System Instructions との組み合わせ

`System Instructions` でAIの性格や応答ルールを設定し、マルチターンチャットと組み合わせると、一貫した振る舞いのチャットボットが作れます。

```python
# chatbot_with_system.py
# System Instructions でカスタマーサポートボットを構築

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

# System Instructions でボットの役割を定義
model = genai.GenerativeModel(
    model_name="gemini-2.0-flash",
    system_instruction="""あなたは「TechShop」のカスタマーサポート担当です。
以下のルールに従ってください：

1. 常に丁寧語で対応する
2. 製品に関する質問は正確に回答する
3. 価格に関する不正確な情報は言わない（「確認します」と回答）
4. クレームには共感を示した上で解決策を提案する
5. 回答は200文字以内に簡潔にまとめる
6. 対応範囲外の質問には「担当部署をご案内します」と回答する

取り扱い製品:
- スマートウォッチ SW-100（29,800円）
- ワイヤレスイヤホン WE-200（12,800円）
- ポータブルスピーカー PS-300（8,980円）"""
)

chat = model.start_chat()

# 対話例
queries = [
    "SW-100の防水性能を教えてください",
    "先日買ったWE-200の左側から音が出ないんですが",
    "修理にはどれくらいかかりますか？",
]

for query in queries:
    response = chat.send_message(query)
    print(f"お客様: {query}")
    print(f"サポート: {response.text}")
    print("---")
```

System Instructions は会話全体に適用されるため、何ターン目であっても「TechShop のサポート担当」としての振る舞いが維持されます。詳しくは [プロンプトエンジニアリング入門](/articles/gemini-basics/gemini-prompt-engineering-guide) を参照してください。

## ストリーミングでリアルタイム応答する

チャット UI では、レスポンス全体の生成を待たずにテキストをリアルタイムで表示する「ストリーミング」が一般的です。

```python
# streaming_chat.py
# ストリーミングでリアルタイムにチャット応答を表示

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

chat = model.start_chat()

def stream_chat(message: str) -> str:
    """メッセージを送信し、ストリーミングで応答を表示する"""
    print(f"You: {message}")
    print("AI: ", end="", flush=True)

    full_response = ""
    response = chat.send_message(message, stream=True)

    for chunk in response:
        text = chunk.text
        print(text, end="", flush=True)
        full_response += text

    print()  # 改行
    return full_response

# 対話型チャットループ
stream_chat("量子コンピュータを5歳児に説明してください")
stream_chat("もう少し詳しく、小学生向けに説明して")
stream_chat("実際にどんなことに使えるの？")
```

`send_message()` に `stream=True` を渡すと、レスポンスがチャンク単位で返されます。各チャンクを即座に出力することで、ユーザーはAIが「考えながら答えている」ような自然な体験を得られます。

## 会話の保存と読み込み

実用的なチャットアプリでは、会話を保存して後から再開できる仕組みが必要です。

```python
# save_load_chat.py
# 会話履歴を JSON ファイルに保存・読み込み

import json
from pathlib import Path
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

def save_history(chat, filepath: str) -> None:
    """会話履歴を JSON ファイルに保存"""
    history_data = []
    for message in chat.history:
        history_data.append({
            "role": message.role,
            "text": message.parts[0].text,
        })

    Path(filepath).write_text(
        json.dumps(history_data, ensure_ascii=False, indent=2),
        encoding="utf-8"
    )
    print(f"保存完了: {len(history_data)} メッセージ → {filepath}")

def load_and_resume(filepath: str) -> genai.ChatSession:
    """保存済み履歴を読み込んでチャットを再開"""
    data = json.loads(Path(filepath).read_text(encoding="utf-8"))

    history = [
        {"role": item["role"], "parts": [item["text"]]}
        for item in data
    ]

    model = genai.GenerativeModel("gemini-2.0-flash")
    return model.start_chat(history=history)

# 使用例
model = genai.GenerativeModel("gemini-2.0-flash")
chat = model.start_chat()

chat.send_message("Reactの状態管理ライブラリを比較して")
chat.send_message("Zustandを詳しく教えて")

# 保存
save_history(chat, "chat_history.json")

# 別のタイミングで再開
resumed_chat = load_and_resume("chat_history.json")
response = resumed_chat.send_message("Zustandのサンプルコードを書いて")
print(response.text)
```

## トークン使用量の管理

マルチターンチャットでは会話が長くなるほどトークン消費が増えます。`count_tokens()` メソッドで事前に確認できます。

```python
# token_management.py
# トークン使用量をモニタリングしながらチャット

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")
chat = model.start_chat()

MAX_TOKENS = 50000  # トークン上限

def send_with_token_check(message: str) -> str:
    """トークン数をチェックしてからメッセージを送信"""
    # 現在の履歴 + 新しいメッセージのトークン数を計算
    token_count = model.count_tokens(
        chat.history + [{"role": "user", "parts": [message]}]
    )

    print(f"現在のトークン数: {token_count.total_tokens} / {MAX_TOKENS}")

    if token_count.total_tokens > MAX_TOKENS:
        print("⚠ トークン上限に近づいています。会話を要約します。")
        # 履歴を要約して圧縮する処理を実装可能
        return "トークン上限に達しました。新しい会話を開始してください。"

    response = chat.send_message(message)
    return response.text

# 使用例
print(send_with_token_check("TypeScript の型推論の仕組みを教えて"))
print(send_with_token_check("ジェネリクスとの関係は？"))
```

## FAQ

### Q1. マルチターンチャットと単発の generateContent() の違いは何ですか？

A1. `generateContent()` は各呼び出しが独立しており、前の質問の文脈は引き継がれません。`startChat()` を使うマルチターンチャットでは、会話の全履歴が自動的に各リクエストに含まれるため、文脈を踏まえた応答が可能です。

### Q2. 会話履歴が非常に長くなった場合、どう対処すべきですか？

A2. 3つのアプローチがあります。（1）直近N件のメッセージだけを保持する。（2）古い会話を Gemini で要約してトークンを圧縮する。（3）`count_tokens()` でトークン数を監視し、閾値を超えたら新しいセッションを開始する。用途に応じて組み合わせてください。

### Q3. マルチターンチャットでストリーミングは使えますか？

A3. はい、`chat.send_message(message, stream=True)` でストリーミングが有効になります。通常の `generateContent` のストリーミングと同じように、チャンク単位でレスポンスを受け取れます。詳しくは [ストリーミング＆チャット](/articles/gemini-api/streaming-and-chat) を参照してください。

### Q4. 画像や動画をチャットの中で送ることはできますか？

A4. はい、Gemini はマルチモーダル入力に対応しているため、テキストと画像を組み合わせたメッセージをチャット内で送信できます。`parts` に画像データを含めることで、チャットの文脈を維持しながら画像に関する質問も可能です。詳しくは [マルチモーダル API ガイド](/articles/gemini-api/multimodal-api-guide) を参照してください。

## まとめ

Gemini API のマルチターンチャットは、`startChat()` で会話セッションを作成し、`send_message()` で対話を続けるシンプルな API です。会話履歴の自動管理、System Instructions との組み合わせ、ストリーミング対応により、実用的なチャットボットを短いコードで構築できます。

まずは基本的なチャットを実装し、用途に応じて会話の保存・復元やトークン管理を追加していくのがおすすめです。API の基本については [Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart)、プロンプト技法については [プロンプトエンジニアリング入門](/articles/gemini-basics/gemini-prompt-engineering-guide) も参考にしてください。
