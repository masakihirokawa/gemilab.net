## はじめに

AI を活用したバックエンドサービスの需要が急速に高まっています。特に Gemini API は、テキスト生成・画像認識・Function Calling などの多彩な機能を備えており、Node.js と組み合わせることで軽量かつスケーラブルな AI マイクロサービスを素早く立ち上げることができます。

この記事では、Google の Gemini API と Node.js（Express.js）を使い、実際にプロダクションで運用できるレベルの AI マイクロサービスを構築する手順をステップバイステップで解説します。ストリーミング応答、レート制限、エラーハンドリングといった本番運用に欠かせないテクニックも網羅しています。

対象読者は、Node.js の基本的な知識があり、Gemini API を使ったサーバーサイド開発に興味のある方です。

## 環境構築と API キーの取得

まず、プロジェクトの初期セットアップを行います。

### 前提条件

- Node.js 20 以上
- npm または yarn
- Google AI Studio のアカウント（[Google AI Studio](https://aistudio.google.com/) で無料作成）

### プロジェクトの初期化

```bash
mkdir gemini-microservice && cd gemini-microservice
npm init -y
npm install express @google/generative-ai dotenv cors helmet express-rate-limit
npm install -D typescript @types/node @types/express tsx
```

### API キーの設定

[Google AI Studio](https://aistudio.google.com/) にアクセスし、API キーを発行します。取得したキーを `.env` ファイルに保存しましょう。

```bash
# .env
GEMINI_API_KEY=YOUR_API_KEY
PORT=3000
NODE_ENV=production
```

セキュリティのため、API キーは環境変数で管理し、ソースコードにハードコードしないことが鉄則です。

## Express.js × Gemini API の基本構成

まず、Gemini API クライアントを初期化し、Express.js サーバーと接続する基本構成を作ります。

```javascript
// src/server.js
import express from 'express';
import cors from 'cors';
import helmet from 'helmet';
import rateLimit from 'express-rate-limit';
import { GoogleGenerativeAI } from '@google/generative-ai';
import dotenv from 'dotenv';

dotenv.config();

const app = express();
const PORT = process.env.PORT || 3000;

// Gemini API クライアントの初期化（アプリ起動時に1回のみ）
const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
const model = genAI.getGenerativeModel({ model: 'gemini-2.5-flash' });

// ミドルウェア設定
app.use(helmet());
app.use(cors());
app.use(express.json({ limit: '10mb' }));

// レート制限（1分あたり30リクエスト）
const limiter = rateLimit({
  windowMs: 60 * 1000,
  max: 30,
  message: { error: 'Too many requests. Please try again later.' }
});
app.use('/api/', limiter);

// ヘルスチェック
app.get('/health', (req, res) => {
  res.json({ status: 'ok', timestamp: new Date().toISOString() });
});

app.listen(PORT, () => {
  console.log(`AI Microservice running on port ${PORT}`);
});
```

ポイントは、`GoogleGenerativeAI` のインスタンスをアプリ起動時に1回だけ作成していることです。リクエストごとに再生成するとオーバーヘッドが発生します。

## テキスト生成エンドポイントの実装

最も基本的なテキスト生成エンドポイントを実装します。

```javascript
// src/routes/generate.js
import { Router } from 'express';
import { GoogleGenerativeAI } from '@google/generative-ai';

const router = Router();
const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

// POST /api/generate — テキスト生成
router.post('/generate', async (req, res) => {
  try {
    const { prompt, maxTokens = 1024, temperature = 0.7 } = req.body;

    if (!prompt || typeof prompt !== 'string') {
      return res.status(400).json({ error: 'prompt is required' });
    }

    const model = genAI.getGenerativeModel({
      model: 'gemini-2.5-flash',
      generationConfig: {
        maxOutputTokens: maxTokens,
        temperature: temperature,
      },
    });

    const result = await model.generateContent(prompt);
    const response = result.response;
    const text = response.text();

    // 使用トークン数の取得
    const usage = response.usageMetadata;

    res.json({
      text,
      usage: {
        promptTokens: usage?.promptTokenCount || 0,
        completionTokens: usage?.candidatesTokenCount || 0,
        totalTokens: usage?.totalTokenCount || 0,
      },
    });
  } catch (error) {
    console.error('Generate error:', error.message);
    res.status(500).json({ error: 'Generation failed', details: error.message });
  }
});

export default router;
```

このエンドポイントは、POST リクエストで `prompt` を受け取り、Gemini API でテキストを生成して返します。`maxTokens` と `temperature` もリクエストパラメータで制御できるようにしています。

## ストリーミング応答の実装

チャットアプリケーションやリアルタイム処理では、ストリーミング応答が不可欠です。Server-Sent Events（SSE）を使った実装を見てみましょう。

```javascript
// POST /api/stream — ストリーミング生成
router.post('/stream', async (req, res) => {
  const { prompt, systemInstruction } = req.body;

  if (!prompt) {
    return res.status(400).json({ error: 'prompt is required' });
  }

  // SSE ヘッダー設定
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  try {
    const model = genAI.getGenerativeModel({
      model: 'gemini-2.5-flash',
      systemInstruction: systemInstruction || undefined,
    });

    const result = await model.generateContentStream(prompt);

    for await (const chunk of result.stream) {
      const text = chunk.text();
      if (text) {
        // SSE フォーマットでチャンクを送信
        res.write(`data: ${JSON.stringify({ text })}\n\n`);
      }
    }

    // ストリーム完了を通知
    res.write(`data: ${JSON.stringify({ done: true })}\n\n`);
    res.end();
  } catch (error) {
    console.error('Stream error:', error.message);
    res.write(`data: ${JSON.stringify({ error: error.message })}\n\n`);
    res.end();
  }
});
```

クライアント側からは `EventSource` や `fetch` で SSE を受信できます。

```javascript
// クライアント側の実装例
const response = await fetch('/api/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ prompt: 'Node.js の非同期処理について解説してください' }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const lines = decoder.decode(value).split('\n');
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      if (data.done) break;
      // data.text を UI にリアルタイム表示
      process.stdout.write(data.text);
    }
  }
}
```

ストリーミングを利用すると、最初のトークンが届くまでの体感待ち時間を大幅に短縮でき、ユーザー体験が向上します。

## エラーハンドリングとリトライ戦略

本番環境では、API のレート制限超過やネットワーク障害に備えたリトライ機構が必須です。

```javascript
// src/utils/retry.js
export async function withRetry(fn, maxRetries = 3, baseDelay = 1000) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      const isRetryable =
        error.status === 429 || // Rate limit
        error.status === 503 || // Service unavailable
        error.message?.includes('RESOURCE_EXHAUSTED');

      if (!isRetryable || attempt === maxRetries) {
        throw error;
      }

      // 指数バックオフ + ジッター
      const delay = baseDelay * Math.pow(2, attempt - 1) + Math.random() * 500;
      console.warn(`Retry ${attempt}/${maxRetries} after ${Math.round(delay)}ms`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
```

この関数を使って、API 呼び出しをラップします。

```javascript
// リトライ付きの生成
const result = await withRetry(async () => {
  return await model.generateContent(prompt);
}, 3, 1000);
```

指数バックオフとジッター（ランダム遅延）を組み合わせることで、複数クライアントからの同時リトライが集中するのを防ぎます。エラーハンドリングの詳しいパターンについては、[Gemini API エラーハンドリング＆リトライパターン実践ガイド](/articles/gemini-api/gemini-api-error-handling-retry-patterns) も参考にしてください。

## マルチモーダル対応エンドポイント

Gemini API の強みのひとつは、テキストだけでなく画像や PDF もシームレスに処理できることです。画像解析エンドポイントも追加しましょう。

```javascript
import multer from 'multer';

const upload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: 20 * 1024 * 1024 }, // 20MB
});

// POST /api/analyze-image — 画像解析
router.post('/analyze-image', upload.single('image'), async (req, res) => {
  try {
    if (!req.file) {
      return res.status(400).json({ error: 'Image file is required' });
    }

    const { prompt = 'この画像の内容を詳しく説明してください' } = req.body;

    const model = genAI.getGenerativeModel({ model: 'gemini-2.5-flash' });

    const imagePart = {
      inlineData: {
        data: req.file.buffer.toString('base64'),
        mimeType: req.file.mimetype,
      },
    };

    const result = await model.generateContent([prompt, imagePart]);
    const text = result.response.text();

    res.json({ analysis: text });
  } catch (error) {
    console.error('Image analysis error:', error.message);
    res.status(500).json({ error: 'Image analysis failed' });
  }
});
```

このエンドポイントにより、アップロードされた画像を Gemini API で解析し、その内容をテキストで返すマイクロサービスが完成します。Gemini API のマルチモーダル活用についてさらに深く学びたい方は、[Gemini API マルチモーダル活用テクニック集](/articles/gemini-api/gemini-api-multimodal-techniques) も合わせてご覧ください。

## 本番デプロイに向けた設計パターン

### ログ構造化

```javascript
// src/middleware/logger.js
export function requestLogger(req, res, next) {
  const start = Date.now();

  res.on('finish', () => {
    const duration = Date.now() - start;
    const log = {
      method: req.method,
      path: req.path,
      status: res.statusCode,
      duration: `${duration}ms`,
      timestamp: new Date().toISOString(),
    };
    console.log(JSON.stringify(log));
  });

  next();
}
```

### Graceful Shutdown

```javascript
// サーバー終了時の安全なクリーンアップ
const server = app.listen(PORT);

process.on('SIGTERM', () => {
  console.log('SIGTERM received. Shutting down gracefully...');
  server.close(() => {
    console.log('Server closed');
    process.exit(0);
  });

  // 強制終了のタイムアウト（30秒）
  setTimeout(() => {
    console.error('Forced shutdown after timeout');
    process.exit(1);
  }, 30000);
});
```

### Docker コンテナ化

```dockerfile
FROM node:20-alpine

WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY src/ ./src/

EXPOSE 3000
USER node

CMD ["node", "src/server.js"]
```

Docker イメージを軽量に保つため、`node:20-alpine` をベースに、開発依存パッケージを含めない `npm ci --omit=dev` を使用しています。

## よくある質問（FAQ）

### Q1: Gemini API の無料枠でマイクロサービスを運用できますか？

はい、開発やプロトタイプ段階であれば十分に可能です。2026年3月現在、Gemini 2.5 Flash は1分あたり30リクエスト、1日あたり1,500リクエストの無料枠があります。本番で大量のリクエストを処理する場合は、従量課金プランへの移行を検討してください。

### Q2: ストリーミングと通常のリクエスト、どちらを使うべきですか？

ユーザーが応答をリアルタイムで確認する UI（チャット画面など）ではストリーミングが最適です。一方、バッチ処理やバックグラウンドジョブでは通常リクエストの方がシンプルで扱いやすいです。レスポンスが長くなる場合はストリーミングによる体感速度の改善効果が特に大きくなります。

### Q3: 複数のモデルを使い分けるにはどうすればよいですか？

`getGenerativeModel()` の `model` パラメータで切り替えられます。たとえば、軽量な処理には `gemini-2.5-flash`、高精度が求められる分析には `gemini-2.5-pro` を使い分けるといった戦略が有効です。リクエストパラメータにモデル名を含め、ルーティング時に動的に切り替える設計がおすすめです。

### Q4: Node.js でメモリリークを防ぐには？

Gemini API クライアントのインスタンスはグローバルで1つだけ保持し、リクエストごとに再生成しないことが重要です。また、ストリーミング応答ではクライアント切断時に適切にストリームを終了する処理を入れてください。`req.on('close', ...)` イベントでクリーンアップを実装するのが定石です。
