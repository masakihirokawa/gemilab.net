## Gemini トークンカウント API ガイド

Gemini APIの利用コストはトークン数に基づいて計算されます。**トークンカウント API** を活用することで、リクエスト前に正確な費用を見積もり、コスト最適化を実現できます。本ガイドでは、トークンの仕組みからコスト管理まで、実践的な内容を解説します。

## トークンとは

**トークン**は、言語モデルが処理する最小単位です。1トークンは大まかに4文字に相当しますが、言語や単語によって異なります。

### トークンの基本概念

- **入力トークン**: ユーザーが送信したプロンプトやコンテキストを構成するトークン
- **出力トークン**: モデルが生成したレスポンスを構成するトークン
- **トークンの効率**: より少ないトークンで同じ情報を表現することで、コスト削減が可能

### トークン数の概算

```
英語テキスト: 約1単語 = 1.3トークン
日本語テキスト: 約1文字 = 1トークン
コード: 約1行 = 10-20トークン
```

## トークンカウント API の使用方法

### 基本的な使用法

```python
import anthropic

client = anthropic.Anthropic(api_key="YOUR_API_KEY")

def count_tokens(text: str, model: str = "claude-3-5-sonnet-20241022") -> dict:
    """テキストのトークン数をカウント"""
    response = client.messages.count_tokens(
        model=model,
        messages=[
            {"role": "user", "content": text}
        ]
    )

    return {
        "input_tokens": response.input_tokens,
        "output_tokens": response.output_tokens if hasattr(response, 'output_tokens') else 0,
        "total_tokens": response.input_tokens
    }

# 使用例
text = "Gemini APIを使用してAIアプリケーションを構築する方法について説明してください。"
result = count_tokens(text)

print(f"入力トークン数: {result['input_tokens']}")
print(f"合計トークン数: {result['total_tokens']}")

# 予期される出力:
# 入力トークン数: 45
# 合計トークン数: 45
```

## 複数の入力形式でのトークンカウント

Gemini APIは、テキストだけでなく、マルチモーダルコンテンツ（画像や動画）もサポートしています。各形式のトークン消費量を理解することが重要です。

### テキスト＋画像でのカウント

```python
import base64
import httpx

def count_tokens_with_image(text: str, image_path: str) -> dict:
    """テキストと画像を含むコンテンツのトークンをカウント"""

    # 画像をBase64エンコード
    with open(image_path, "rb") as image_file:
        image_data = base64.standard_b64encode(image_file.read()).decode("utf-8")

    # 画像の形式を特定
    media_type = "image/jpeg"
    if image_path.lower().endswith(".png"):
        media_type = "image/png"
    elif image_path.lower().endswith(".webp"):
        media_type = "image/webp"

    response = client.messages.count_tokens(
        model="claude-3-5-sonnet-20241022",
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "image",
                        "source": {
                            "type": "base64",
                            "media_type": media_type,
                            "data": image_data,
                        },
                    },
                    {
                        "type": "text",
                        "text": text
                    }
                ],
            }
        ],
    )

    return {
        "input_tokens": response.input_tokens,
        "image_tokens": response.input_tokens - len(text.split()),
        "total_tokens": response.input_tokens
    }

# 使用例
image_tokens = count_tokens_with_image(
    "この画像に何が映っていますか？",
    "sample_image.jpg"
)

print(f"入力トークン数: {image_tokens['input_tokens']}")
print(f"画像のトークン数: {image_tokens['image_tokens']}")
```

## コスト見積もりと最適化

### コスト計算機

```python
class TokenCostCalculator:
    """Gemini API のコスト計算"""

    # 2024年現在の料金（1M トークンあたり）
    PRICING = {
        "gemini-1.5-pro": {
            "input": 1.25,      # $1.25 per 1M input tokens
            "output": 5.00      # $5.00 per 1M output tokens
        },
        "gemini-1.5-flash": {
            "input": 0.075,     # $0.075 per 1M input tokens
            "output": 0.30      # $0.30 per 1M output tokens
        }
    }

    @staticmethod
    def calculate_cost(
        input_tokens: int,
        output_tokens: int,
        model: str = "gemini-1.5-pro"
    ) -> dict:
        """入出力トークン数からコストを計算"""
        if model not in TokenCostCalculator.PRICING:
            raise ValueError(f"Unknown model: {model}")

        pricing = TokenCostCalculator.PRICING[model]

        input_cost = (input_tokens / 1_000_000) * pricing["input"]
        output_cost = (output_tokens / 1_000_000) * pricing["output"]
        total_cost = input_cost + output_cost

        return {
            "input_cost": round(input_cost, 6),
            "output_cost": round(output_cost, 6),
            "total_cost": round(total_cost, 6),
            "currency": "USD"
        }

    @staticmethod
    def estimate_monthly_cost(
        daily_requests: int,
        avg_input_tokens: int,
        avg_output_tokens: int,
        model: str
    ) -> dict:
        """月間コストを見積もる"""
        monthly_input = daily_requests * 30 * avg_input_tokens
        monthly_output = daily_requests * 30 * avg_output_tokens

        cost = TokenCostCalculator.calculate_cost(
            monthly_input,
            monthly_output,
            model
        )

        return {
            "monthly_input_tokens": monthly_input,
            "monthly_output_tokens": monthly_output,
            **cost
        }

# 使用例
cost = TokenCostCalculator.calculate_cost(
    input_tokens=5000,
    output_tokens=2000,
    model="gemini-1.5-pro"
)

print(f"入力コスト: ${cost['input_cost']}")
print(f"出力コスト: ${cost['output_cost']}")
print(f"合計コスト: ${cost['total_cost']}")

# 月間コスト見積もり
monthly_estimate = TokenCostCalculator.estimate_monthly_cost(
    daily_requests=1000,
    avg_input_tokens=500,
    avg_output_tokens=300,
    model="gemini-1.5-flash"
)

print(f"\n月間推定コスト: ${monthly_estimate['total_cost']}")
```

```python
def optimize_prompt(full_text: str, max_tokens: int = 500) -> str:
    """プロンプトを最適化してトークン数を削減"""

    # 長いテキストを要約
    if len(full_text) > max_tokens * 4:  # 概算
        summary_prompt = f"次のテキストを簡潔に要約してください：\n{full_text[:1000]}"
        summary = client.messages.create(
            model="gemini-1.5-flash",
            max_tokens=200,
            messages=[{"role": "user", "content": summary_prompt}]
        )
        return summary.content[0].text

    return full_text

# 使用例
original_text = "...長いテキスト..."
optimized = optimize_prompt(original_text)
```

### 2. キャッシング戦略

同じコンテキストを複数回使用する場合、プロンプトキャッシング を活用：

```python
def generate_with_caching(
    system_context: str,
    user_query: str
) -> tuple[str, dict]:
    """キャッシング対応のコンテンツ生成"""

    response = client.messages.create(
        model="gemini-1.5-pro",
        max_tokens=1024,
        system=[
            {
                "type": "text",
                "text": system_context,
                # キャッシング有効化
                "cache_control": {"type": "ephemeral"}
            }
        ],
        messages=[{"role": "user", "content": user_query}]
    )

    usage = {
        "input_tokens": response.usage.input_tokens,
        "cache_creation_input_tokens": response.usage.cache_creation_input_tokens,
        "cache_read_input_tokens": response.usage.cache_read_input_tokens,
        "output_tokens": response.usage.output_tokens
    }

    return response.content[0].text, usage

# キャッシングの効果を確認
system_context = "あなたは専門的なテクニカルライターです..."

# 1回目（キャッシュ作成）
result1, usage1 = generate_with_caching(system_context, "Gemini APIについて説明してください")
print(f"1回目 - キャッシュ作成: {usage1['cache_creation_input_tokens']} tokens")

# 2回目（キャッシュ読み込み）
result2, usage2 = generate_with_caching(system_context, "Gemini APIの料金について説明してください")
print(f"2回目 - キャッシュ読み込み: {usage2['cache_read_input_tokens']} tokens")
```

### 3. バッチ処理による効率化

```python
def batch_count_tokens(texts: list[str], model: str) -> dict:
    """複数テキストのトークンを一括カウント"""

    total_tokens = 0
    token_per_text = []

    for text in texts:
        response = client.messages.count_tokens(
            model=model,
            messages=[{"role": "user", "content": text}]
        )
        tokens = response.input_tokens
        token_per_text.append(tokens)
        total_tokens += tokens

    return {
        "total_tokens": total_tokens,
        "per_text": token_per_text,
        "average_tokens": total_tokens / len(texts)
    }

# 使用例
texts = [
    "最初のテキスト",
    "2番目のテキスト",
    "3番目のテキスト"
]

result = batch_count_tokens(texts, "gemini-1.5-flash")
print(f"合計トークン数: {result['total_tokens']}")
```

## [ストリーミングとチャット](/articles/gemini-api/streaming-and-chat) を活用したコスト削減

ストリーミングを使用することで、不必要な再試行や重複リクエストを削減できます：

```python
def streaming_with_token_tracking(prompt: str) -> dict:
    """ストリーミングでトークンを追跡"""

    token_count = client.messages.count_tokens(
        model="gemini-1.5-flash",
        messages=[{"role": "user", "content": prompt}]
    )

    input_tokens = token_count.input_tokens
    output_tokens = 0

    with client.messages.stream(
        model="gemini-1.5-flash",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    ) as stream:
        for text in stream.text_stream:
            output_tokens += len(text.split())
            print(text, end="", flush=True)

    return {
        "input_tokens": input_tokens,
        "estimated_output_tokens": output_tokens
    }
```

## よくある質問

### Q1: トークンカウント API 自体に費用はかかりますか？

A: いいえ。トークンカウント API の呼び出しは無料です。これにより、実際のリクエスト前にコストを見積もることができます。

### Q2: 画像のトークン数はどのように計算されますか？

A: 画像のトークン数は、画像の解像度によって異なります。一般的に、低解像度画像は約 100-300トークン、高解像度画像は最大 1,600-2,000トークン消費します。

### Q3: キャッシングでどのくらいコストを削減できますか？

A: キャッシュ読み込みは、通常の入力トークン料金の約10%で計算されるため、キャッシングが有効に機能すれば、90%のコスト削減が可能です。
