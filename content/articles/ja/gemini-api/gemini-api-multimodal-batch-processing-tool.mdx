Gemini APIのマルチモーダル機能を使えば、画像・PDF・音声といった異なる形式のファイルを1つのAPIで統一的に処理できます。この記事では、Pythonで**マルチモーダルバッチ処理ツール**を構築する方法をステップバイステップで解説します。

この記事で学べること:

- Gemini APIで画像・PDF・音声を統一的に処理するアーキテクチャ設計

- エラーハンドリングとリトライ機構の実装
- 処理結果をJSON/CSVで出力する実用的なツール設計

## アーキテクチャ設計

バッチ処理ツール全体の構成は次のようになります:

```
入力フォルダ（画像/PDF/音声が混在）
    ↓
ファイル分類（拡張子で判定）
    ↓
┌──────────┬──────────┬──────────┐
│  画像処理  │  PDF処理  │  音声処理  │
│ キャプション │   要約    │  文字起こし │
└──────────┴──────────┴──────────┘
    ↓
結果を統合して JSON/CSV 出力
```

設計上のポイントは3つです:

1. **ファイル種別の自動判定** — 拡張子ベースでルーティング
2. **非同期並列処理** — `asyncio.Semaphore` でAPI同時リクエスト数を制御
3. **リトライ付きエラーハンドリング** — 一時的な障害で全体が止まらない設計

## ファイル種別ごとの処理関数

### 画像ファイルの解析

画像ファイルを読み込み、キャプション生成や内容説明を行う関数です:

```python
import asyncio
from pathlib import Path

async def process_image(file_path: Path) -> dict:
    """画像ファイルを解析してキャプションを生成する"""
    image_bytes = file_path.read_bytes()
    mime_type = _get_mime_type(file_path)

    response = client.models.generate_content(
        model=MODEL_ID,
        contents=[
            {
                "role": "user",
                "parts": [
                    {"text": "この画像の内容を200文字以内で説明してください。主要な被写体、構図、色調、雰囲気を含めてください。"},
                    {"inline_data": {"mime_type": mime_type, "data": image_bytes}}
                ]
            }
        ]
    )

    return {
        "file": file_path.name,
        "type": "image",
        "caption": response.text.strip(),
        "status": "success"
    }
```

### MIME タイプの判定

ファイル拡張子からMIMEタイプを自動判定するヘルパー関数です:

```python
def _get_mime_type(file_path: Path) -> str:
    """拡張子からMIMEタイプを判定"""
    ext = file_path.suffix.lower()
    mime_map = {
        # 画像
        ".jpg": "image/jpeg", ".jpeg": "image/jpeg",
        ".png": "image/png", ".gif": "image/gif",
        ".webp": "image/webp", ".bmp": "image/bmp",
        # PDF
        ".pdf": "application/pdf",
        # 音声
        ".mp3": "audio/mpeg", ".wav": "audio/wav",
        ".m4a": "audio/mp4", ".ogg": "audio/ogg",
        ".flac": "audio/flac",
    }
    return mime_map.get(ext, "application/octet-stream")
```

## 並列バッチ処理エンジン

ここが本記事の核心部分です。`asyncio.Semaphore` を使って同時リクエスト数を制御しながら、全ファイルを効率的に処理します。

```python
import json
import csv
from datetime import datetime
from tqdm.asyncio import tqdm_asyncio

# ファイル種別とハンドラーの対応表
FILE_HANDLERS = {
    ".jpg": process_image, ".jpeg": process_image,
    ".png": process_image, ".gif": process_image,
    ".webp": process_image, ".bmp": process_image,
    ".pdf": process_pdf,
    ".mp3": process_audio, ".wav": process_audio,
    ".m4a": process_audio, ".ogg": process_audio,
    ".flac": process_audio,
}

# 同時リクエスト数の上限（API レートリミット対策）
MAX_CONCURRENCY = 5


async def process_single_file(
    file_path: Path,
    semaphore: asyncio.Semaphore,
    max_retries: int = 3
) -> dict:
    """1ファイルを処理する（セマフォ＋リトライ付き）"""
    ext = file_path.suffix.lower()
    handler = FILE_HANDLERS.get(ext)

    if handler is None:
        return {
            "file": file_path.name,
            "type": "unknown",
            "status": "skipped",
            "error": f"未対応の拡張子: {ext}"
        }

    for attempt in range(1, max_retries + 1):
        try:
            async with semaphore:
                result = await asyncio.to_thread(lambda: asyncio.run(handler(file_path)))
                # 同期呼び出しをラップ（SDK が同期の場合）
                return result
        except Exception as e:
            if attempt == max_retries:
                return {
                    "file": file_path.name,
                    "type": ext,
                    "status": "error",
                    "error": f"{type(e).__name__}: {str(e)}"
                }
            # 指数バックオフで待機
            wait_time = 2 ** attempt
            print(f"  ⚠ {file_path.name} リトライ {attempt}/{max_retries}（{wait_time}秒後）")
            await asyncio.sleep(wait_time)


async def batch_process(input_dir: str, output_path: str = "results.json"):
    """フォルダ内の全ファイルをバッチ処理する"""
    input_path = Path(input_dir)
    if not input_path.is_dir():
        raise FileNotFoundError(f"入力ディレクトリが見つかりません: {input_dir}")

    # 対応ファイルを収集
    files = [
        f for f in input_path.iterdir()
        if f.is_file() and f.suffix.lower() in FILE_HANDLERS
    ]

    if not files:
        print("処理対象のファイルが見つかりませんでした。")
        return

    print(f"📁 {len(files)} ファイルを検出しました")
    print(f"  画像: {sum(1 for f in files if f.suffix.lower() in {'.jpg','.jpeg','.png','.gif','.webp','.bmp'})} 件")
    print(f"  PDF:  {sum(1 for f in files if f.suffix.lower() == '.pdf')} 件")
    print(f"  音声: {sum(1 for f in files if f.suffix.lower() in {'.mp3','.wav','.m4a','.ogg','.flac'})} 件")

    semaphore = asyncio.Semaphore(MAX_CONCURRENCY)

    # 並列実行（プログレスバー付き）
    tasks = [process_single_file(f, semaphore) for f in files]
    results = await tqdm_asyncio.gather(*tasks, desc="処理中")

    # 統計の計算
    success = sum(1 for r in results if r["status"] == "success")
    errors = sum(1 for r in results if r["status"] == "error")
    skipped = sum(1 for r in results if r["status"] == "skipped")

    # JSON 出力
    output = {
        "processed_at": datetime.now().isoformat(),
        "total_files": len(files),
        "success": success,
        "errors": errors,
        "skipped": skipped,
        "results": results
    }

    with open(output_path, "w", encoding="utf-8") as f:
        json.dump(output, f, ensure_ascii=False, indent=2)

    print(f"\n✅ 処理完了: 成功 {success} / エラー {errors} / スキップ {skipped}")
    print(f"📄 結果を {output_path} に保存しました")

    return output
```

### 実行方法

```python

    input_dir = sys.argv[1] if len(sys.argv) > 1 else "./input_files"
```

## CSV出力への対応

JSONだけでなく、スプレッドシートで確認しやすいCSV形式でも出力できるようにします:

```python
def export_to_csv(results: list[dict], output_path: str = "results.csv"):
    """処理結果をCSV形式で出力する"""
    fieldnames = ["file", "type", "status", "caption", "summary",
                  "transcription_summary", "error"]

    with open(output_path, "w", newline="", encoding="utf-8-sig") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames, extrasaction="ignore")
        writer.writeheader()
        for row in results:
            writer.writerow(row)

    print(f"📊 CSV を {output_path} に保存しました")
```

`utf-8-sig` エンコーディングを使うことで、Excelで開いた際に文字化けを防止できます。

### 大きなファイルへの対応

Gemini APIのインラインデータには20MBの上限があります。大きなファイルは File API を使ってアップロードしてから処理します:

```python
async def process_large_file(file_path: Path, prompt: str) -> dict:
    """20MB超のファイルを File API 経由で処理する"""
    # File API でアップロード
    uploaded = client.files.upload(file=file_path)

    # アップロード完了を待機
    import time
    while uploaded.state.name == "PROCESSING":
        time.sleep(2)
        uploaded = client.files.get(name=uploaded.name)

    if uploaded.state.name == "FAILED":
        raise RuntimeError(f"ファイルアップロード失敗: {uploaded.name}")

    # アップロード済みファイルを参照して生成
    response = client.models.generate_content(
        model=MODEL_ID,
        contents=[
            {"role": "user", "parts": [
                {"text": prompt},
                {"file_data": {"file_uri": uploaded.uri, "mime_type": uploaded.mime_type}}
            ]}
        ]
    )

    # 不要になったファイルを削除
    client.files.delete(name=uploaded.name)

    return {
        "file": file_path.name,
        "result": response.text.strip(),
        "status": "success"
    }
```

## よくあるエラーと対処法

バッチ処理で発生しがちなエラーとその対処法をまとめます。

| エラー | 原因 | 対処法 |
|--------|------|--------|
| `429 Resource Exhausted` | APIレートリミット超過 | `MAX_CONCURRENCY` を下げる（3〜5推奨） |
| `400 Invalid argument` | 未対応のファイル形式 | MIME タイプ判定を確認。破損ファイルも原因になる |
| `413 Request too large` | ファイルサイズ超過（20MB以上） | File API 経由のアップロードに切り替える |
| `500 Internal error` | Gemini側の一時障害 | リトライ機構で自動復旧。3回失敗したらスキップ |

## FAQ

### Q: 無料プランでもバッチ処理は可能ですか？

はい、可能です。ただし無料プランではAPIリクエスト数に制限があります（Gemini 2.5 Flash の場合、1分あたり15リクエスト）。`MAX_CONCURRENCY` を `2` に設定し、リクエスト間に待機時間を入れることで安定して処理できます。大量のファイルを処理する場合は、[Gemini Batch Processing API](/articles/gemini-api/gemini-batch-processing-api-guide) の利用も検討してください。50%のコスト削減が可能です。

### Q: 動画ファイルもバッチ処理できますか？

はい、Gemini APIは動画の入力にも対応しています。`.mp4`、`.mov`、`.avi` 等の拡張子を `FILE_HANDLERS` に追加し、動画用の処理関数を作成すれば対応可能です。ただし動画はファイルサイズが大きくなるため、File API 経由でアップロードする方式を推奨します。

### Q: 処理結果の精度を上げるコツはありますか？

いくつかのテクニックがあります。まず、プロンプトに具体的な指示を含めることが重要です。「要約してください」だけでなく「300文字以内で、重要なポイントを3つ含めて要約してください」のように制約を明示すると品質が上がります。また、画像の場合は解像度が高いほど認識精度が向上します。PDFは構造化されたドキュメント（見出し、段落が明確なもの）の方が精度が高くなります。

### Q: APIキーのセキュリティはどう管理すべきですか？

APIキーは必ず環境変数や `.env` ファイルで管理し、ソースコードにハードコードしないでください。チームで共有する場合は、Google Cloud の Secret Manager や AWS Secrets Manager 等の専用サービスを利用しましょう。
