## Gemini API エラーハンドリングの重要性

Gemini API を本番環境で運用する際、**エラーは必ず発生する** ものです。重要なのは、エラーの原因を素早く特定し、適切に対処することです。

本記事では、Gemini API で最も頻出する 5 つのエラーコード、その原因、そして実装べき対処法を詳しく解説します。

## Gemini API で最も一般的な 5 つのエラーコード

### HTTP 429：Rate Limit Exceeded（レート制限超過）

**頻出度**：★★★★★

```
HTTP/1.1 429 Too Many Requests
Retry-After: 30
X-RateLimit-Limit-Requests: 1000
X-RateLimit-Limit-Tokens: 100000
X-RateLimit-Remaining-Requests: 0
X-RateLimit-Remaining-Tokens: 5432
```

#### 原因

- **リクエスト数超過**：1分間に 1,000 回以上のリクエスト
- **トークン数超過**：1分間に 100,000 トークン以上の処理
- **同時接続数超過**：Starter プラン（無料）では同時接続が5～10に制限

#### 対処法

```python
import time
import random
from google.api_core.exceptions import ResourceExhausted
from google.generativeai import genai

def call_gemini_with_retry(prompt, max_retries=5):
    """
    指数バックオフを用いた Gemini API 呼び出し

    429 エラー時は自動的にリトライ。最大5回まで試行。
    """
    base_wait = 1  # 初回待機時間：1秒

    for attempt in range(max_retries):
        try:
            model = genai.GenerativeModel("gemini-2.0-flash")
            response = model.generate_content(prompt)
            return response.text

        except ResourceExhausted as e:
            if attempt == max_retries - 1:
                # 最大リトライ回数に達した場合は例外を再発生
                raise Exception(f"Max retries exceeded: {str(e)}")

            # 指数バックオフ：1秒 → 2秒 → 4秒 → 8秒 → 16秒
            wait_time = base_wait * (2 ** attempt)

            # ジッター追加：ランダムな遅延を加えて、複数クライアントの同時リトライを避ける
            jitter = random.uniform(0, wait_time * 0.1)
            total_wait = wait_time + jitter

            print(f"Rate limited. Retrying in {total_wait:.1f}s (attempt {attempt + 1}/{max_retries})")
            time.sleep(total_wait)

        except Exception as e:
            print(f"Unexpected error: {str(e)}")
            raise

# 使用例
prompt = "日本の GDP について詳しく教えてください"
response = call_gemini_with_retry(prompt)
print(response)
```

#### ベストプラクティス

- **リクエスト頻度の制限**：最初のテスト段階では 10 秒に 1 リクエストに抑える
- **キューイング機構の実装**：大量のリクエストは task queue（Celery、RQ）で分散処理
- **Pro プランへの移行検討**：本番運用では Starter プランの制限では不十分な場合が多い

### HTTP 400：Bad Request（不正なリクエスト形式）

**頻出度**：★★★★

```
HTTP/1.1 400 Bad Request

{
  "error": {
    "code": 400,
    "message": "Invalid value for: body.contents[0].parts[0]",
    "status": "INVALID_ARGUMENT"
  }
}
```

#### 原因（最も多い順）

1. **フロントマターの記述誤り**
   - 無効な JSON Schema
   - required フィールドの欠落

2. **テキスト長の超過**
   - 単一メッセージが 30,000 文字を超える
   - 会話履歴を含むトータルが 100,000 トークンを超える

3. **サポートされていないモデルの指定**
   - `gemini-1.5` という存在しないモデル名
   - 廃止されたモデル（`gemini-pro` は 2025年3月廃止）

4. **Content-Type の誤指定**
   - `application/json` を指定しているが、実際は XML や YAML を送信

#### 対処法

```python
import json
from google.generativeai import genai
from google.api_core.exceptions import InvalidArgument

def validate_and_call_gemini(prompt, system_instruction=None, **kwargs):
    """
    バリデーション付き Gemini API 呼び出し
    """

    # Step 1: モデル名の検証
    valid_models = [
        "gemini-2.0-flash",
        "gemini-2.0-flash-exp",
        "gemini-1.5-pro",
        "gemini-1.5-flash"
    ]
    model_name = kwargs.get("model", "gemini-2.0-flash")

    if model_name not in valid_models:
        raise ValueError(f"Invalid model: {model_name}. Valid options: {valid_models}")

    # Step 2: プロンプト長の検証
    if len(prompt) > 30000:
        raise ValueError(f"Prompt too long: {len(prompt)} chars. Max: 30000")

    # Step 3: generation_config の検証
    config = kwargs.get("generation_config", {})
    if not isinstance(config, dict):
        raise TypeError("generation_config must be a dictionary")

    valid_keys = ["temperature", "top_p", "top_k", "max_output_tokens", "stop_sequences"]
    for key in config.keys():
        if key not in valid_keys:
            raise ValueError(f"Invalid generation_config key: {key}")

    # Step 4: API 呼び出し（実際のエラー処理付き）
    try:
        model = genai.GenerativeModel(
            model_name,
            system_instruction=system_instruction
        )
        response = model.generate_content(
            prompt,
            generation_config=config
        )
        return response.text

    except InvalidArgument as e:
        error_msg = str(e)

        # エラーメッセージから問題を推測
        if "Invalid value for" in error_msg:
            print("❌ Request format error. Check:")
            print("   - Model name spelling")
            print("   - Prompt content encoding (UTF-8)")
            print("   - generation_config keys")

        raise

# 使用例
try:
    response = validate_and_call_gemini(
        "Python での並列処理について説明してください",
        model="gemini-2.0-flash",
        generation_config={"temperature": 0.7, "max_output_tokens": 1000}
    )
    print(response)
except Exception as e:
    print(f"Error: {e}")
```

### HTTP 403：Permission Denied（権限不足）

**頻出度**：★★★

```
HTTP/1.1 403 Forbidden

{
  "error": {
    "code": 403,
    "message": "The caller does not have permission",
    "status": "PERMISSION_DENIED"
  }
}
```

#### 原因

1. **API キーが無効または期限切れ**
2. **Billing アカウントが接続されていない**
3. **API キーに対応する Google Cloud Project で Gemini API が有効化されていない**
4. **API キーがコピー時に改行・スペースを含んでいる**

#### 対処法

```python
import os
import re
from google.generativeai import genai
from google.api_core.exceptions import PermissionDenied

def setup_gemini_api():
    """
    API キーの検証と安全なセットアップ
    """

    # Step 1: 環境変数から API キーを読み込み
    api_key = os.getenv("GOOGLE_API_KEY", "").strip()

    if not api_key:
        raise EnvironmentError(
            "GOOGLE_API_KEY environment variable not set. "
            "Get it from https://aistudio.google.com/app/apikey"
        )

    # Step 2: API キーの形式検証（通常は 39 文字の英数字）
    if not re.match(r"^[A-Za-z0-9\-_]{30,}$", api_key):
        raise ValueError("API key format appears invalid. Regenerate from AI Studio.")

    # Step 3: Gemini の初期化
    genai.configure(api_key=api_key)

    # Step 4: 接続テスト
    try:
        model = genai.GenerativeModel("gemini-2.0-flash")
        test_response = model.generate_content("Hello")
        print("✅ Gemini API setup successful")
        return True

    except PermissionDenied:
        print("❌ API key is invalid or billing not configured.")
        print("   1. Go to https://aistudio.google.com/app/apikey")
        print("   2. Click 'Create API key'")
        print("   3. Ensure billing is enabled in Google Cloud Console")
        raise

# 使用例
setup_gemini_api()
```

### HTTP 500：Internal Server Error（サーバーエラー）

**頻出度**：★★

```
HTTP/1.1 500 Internal Server Error

{
  "error": {
    "code": 500,
    "message": "Internal error occurred.",
    "status": "INTERNAL"
  }
}
```

#### 原因

1. **Google のサーバー側の一時的な障害**（稀）
2. **モデルが処理できない特殊な入力データ**
3. **大量のメディア（画像・動画）を含むリクエスト**

#### 対処法

```python
import time
from google.api_core.exceptions import InternalServerError

def call_gemini_with_500_recovery(prompt, max_retries=3):
    """
    500 エラーの自動リカバリ

    Google サーバーの一時的な障害時は自動リトライ
    """
    base_wait = 5  # 初期待機：5秒

    for attempt in range(max_retries):
        try:
            model = genai.GenerativeModel("gemini-2.0-flash")
            response = model.generate_content(prompt)
            return response.text

        except InternalServerError as e:
            if attempt == max_retries - 1:
                # 最終リトライでも失敗した場合
                print("Server is unavailable. Try again in a few minutes.")
                raise

            wait_time = base_wait * (2 ** attempt)
            print(f"Server error. Retrying in {wait_time}s (attempt {attempt + 1}/{max_retries})")
            time.sleep(wait_time)
```

### HTTP 404：Not Found（リソース未検出）

**頻出度**：★

```
HTTP/1.1 404 Not Found

{
  "error": {
    "code": 404,
    "message": "Requested model does not exist"
  }
}
```

#### 原因

- **モデル名のスペル誤り**
- **非公開または廃止済みモデルへのアクセス**

#### 対処法

```python
def list_available_models():
    """
    利用可能なモデル一覧を表示
    """
    for model in genai.list_models():
        if "generateContent" in model.supported_generation_methods:
            print(f"✅ {model.name}")
```

## 本番環境向け：包括的なエラーハンドリング実装例

```python
import logging
import time
import random
from typing import Optional
from google.generativeai import genai
from google.api_core.exceptions import (
    GoogleAPICallError,
    InvalidArgument,
    PermissionDenied,
    ResourceExhausted,
    InternalServerError,
    NotFound
)

# ロギング設定
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class GeminiAPIClient:
    def __init__(self, api_key: str, model: str = "gemini-2.0-flash"):
        self.api_key = api_key
        self.model_name = model
        genai.configure(api_key=api_key)

    def generate_with_error_handling(
        self,
        prompt: str,
        max_retries: int = 5,
        timeout_seconds: int = 30
    ) -> Optional[str]:
        """
        包括的なエラーハンドリング付き生成
        """

        for attempt in range(max_retries):
            try:
                model = genai.GenerativeModel(self.model_name)
                response = model.generate_content(
                    prompt,
                    request_options={"timeout": timeout_seconds}
                )

                logger.info("✅ Request successful")
                return response.text

            except ResourceExhausted:
                # 429: レート制限
                if attempt < max_retries - 1:
                    wait = (2 ** attempt) + random.uniform(0, 1)
                    logger.warning(f"Rate limited. Retrying in {wait:.1f}s")
                    time.sleep(wait)
                else:
                    logger.error("Rate limit exceeded. Upgrade to Pro plan.")
                    raise

            except InvalidArgument as e:
                # 400: 不正リクエスト
                logger.error(f"Invalid request format: {e}")
                raise

            except PermissionDenied:
                # 403: 認証エラー
                logger.error("API key invalid or billing not configured")
                raise

            except InternalServerError:
                # 500: サーバーエラー
                if attempt < max_retries - 1:
                    wait = 5 * (2 ** attempt)
                    logger.warning(f"Server error. Retrying in {wait}s")
                    time.sleep(wait)
                else:
                    logger.error("Server unavailable after retries")
                    raise

            except NotFound:
                # 404: モデル未検出
                logger.error(f"Model '{self.model_name}' not found")
                raise

            except GoogleAPICallError as e:
                logger.error(f"Unexpected API error: {e}")
                raise

        return None

# 使用例
client = GeminiAPIClient(api_key="your-api-key-here")
response = client.generate_with_error_handling(
    "Python の `asyncio` モジュールについて説明してください"
)
print(response)
```

## よくある質問（FAQ）

**Q1: リトライロジックはどの程度の間隔で実装すべきか？**

A: 指数バックオフが標準です。初期 1 秒から始め、毎回倍にしていき、最大 64 秒まで。ジッターを 10% 加えることで、複数クライアントの同時リトライを避けられます。

**Q2: Starter プランと Pro プランでエラー頻度は異なるか？**

A: 大きく異なります。Starter は 1 分当たり 1,000 リクエスト・100,000 トークン制限。Pro は 1 分当たり 1,500 リクエスト・400,000 トークン制限。本番運用なら Pro が事実上必須です。

**Q3: エラーログはどの粒度で記録すべきか？**

A: WARNING 以上は必ず記録。リトライのたびにすべてをログすると、ファイルサイズが肥大化します。最初と最後のみ、あるいは失敗時のみ記録が推奨です。

**Q4: タイムアウトはどの程度に設定すべきか？**

A: `request_options={"timeout": 30}` で 30 秒がバランス良好。長すぎるとリソース浪費、短すぎると誤検出が増えます。
