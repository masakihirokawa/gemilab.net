Google が提供する **Gemini 2.5 Flash** は、速度・コスト効率・推論能力を高次元で両立した次世代 AI モデルです。2025年に登場して以来、開発者から高い評価を受けており、特に「思考（Thinking）」機能のオン・オフを切り替えられるハイブリッド設計が注目されています。

## Gemini 2.5 Flash とは

Gemini 2.5 Flash は Google DeepMind が開発したモデルで、シリーズ内でもっとも速く・もっとも低コストなクラスに属します。単なる「軽量版」ではなく、Gemini 2.5 Pro に迫る推論能力を持ちつつ、レイテンシとコストを大幅に削減した設計が特徴です。

### 主な特徴

- **ハイブリッド推論モデル**: 思考モード（Thinking）を `thinking_budget` パラメータでオン・オフできる世界初の本格的なハイブリッドモデル
- **1M トークンのコンテキストウィンドウ**: 長い文書・コードベース・会話履歴を丸ごと処理可能
- **マルチモーダル対応**: テキスト・画像・動画・音声・PDF を横断的に処理
- **低コスト**: Gemini 2.5 Pro と比較して大幅に安価

### スペック一覧

| 項目 | 値 |
|------|-----|
| コンテキストウィンドウ | 1,048,576 トークン |
| 最大出力トークン | 65,536 トークン |
| 入力料金 | $0.15 / 1M トークン（思考なし） |
| 出力料金 | $0.60 / 1M トークン（思考なし） |
| 思考トークン料金 | $0.35 / 1M トークン |
| 対応モダリティ | テキスト / 画像 / 動画 / 音声 / PDF |

## 基本的な使い方

### テキスト生成（思考なし）

最もシンプルな使い方は、`gemini-2.5-flash` モデルを指定してテキスト生成を行うことです。

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

model = genai.GenerativeModel("gemini-2.5-flash")

response = model.generate_content("Pythonで素数を判定する関数を書いてください。")
print(response.text)
```

### ストリーミングレスポンス

長い応答をリアルタイムで受信したい場合はストリーミングが便利です。

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

model = genai.GenerativeModel("gemini-2.5-flash")

for chunk in model.generate_content("量子コンピュータについて詳しく説明してください。", stream=True):
    print(chunk.text, end="", flush=True)

print()  # 改行
```

## Thinking（思考）機能の使い方

Gemini 2.5 Flash 最大の特徴が `thinking_budget` パラメータです。内部推論に使用するトークン数の上限を指定することで、精度とコスト・速度のバランスを細かく制御できます。

### thinking_budget の仕様

| 値 | 挙動 |
|----|------|
| `0` | 思考なし（最速・最安） |
| `1〜1024` | 軽量思考 |
| `1025〜8192` | 中程度の思考 |
| `8193〜24576` | 深い思考（最高精度） |

### thinking_budget を指定する

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

model = genai.GenerativeModel("gemini-2.5-flash")

# 深い思考を有効化（数学・推論タスク向け）
response = model.generate_content(
    "次の方程式を解いてください: x^3 - 6x^2 + 11x - 6 = 0",
    generation_config=genai.GenerationConfig(
        thinking_config=genai.types.ThinkingConfig(
            thinking_budget=8192
        )
    )
)

print("思考トークン数:", response.usage_metadata.thoughts_token_count)
print("回答:", response.text)
```

### 思考内容を確認する

`include_thoughts=True` を設定すると、モデルの思考プロセスも取得できます（デバッグや学習目的に有効）。

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

model = genai.GenerativeModel("gemini-2.5-flash")

response = model.generate_content(
    "東京からニューヨークまでの最短飛行ルートを考えてください。",
    generation_config=genai.GenerationConfig(
        thinking_config=genai.types.ThinkingConfig(
            thinking_budget=4096,
            include_thoughts=True
        )
    )
)

# 思考プロセスを表示
for part in response.candidates[0].content.parts:
    if part.thought:
        print("=== 思考プロセス ===")
        print(part.text)
    else:
        print("=== 最終回答 ===")
        print(part.text)
```

## マルチモーダル入力

Gemini 2.5 Flash はテキスト以外にも多様なデータを処理できます。

### 画像の解析

```python
import google.generativeai as genai
import PIL.Image
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

model = genai.GenerativeModel("gemini-2.5-flash")

# ローカル画像を読み込む
image = PIL.Image.open("chart.png")

response = model.generate_content([
    "この図表を分析して、主要なトレンドを日本語で説明してください。",
    image
])

print(response.text)
```

### PDF の解析

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

# PDF ファイルをアップロード
pdf_file = genai.upload_file("report.pdf", mime_type="application/pdf")

model = genai.GenerativeModel("gemini-2.5-flash")

response = model.generate_content([
    "このレポートの要点を3つ挙げ、それぞれ日本語で説明してください。",
    pdf_file
])

print(response.text)
```

## コスト最適化のベストプラクティス

### ユースケース別の thinking_budget 推奨設定

| ユースケース | 推奨 budget | 理由 |
|------------|------------|------|
| チャット・Q&A | 0〜1024 | 即時応答が重要 |
| コード生成 | 2048〜8192 | 論理的整合性が必要 |
| 数学・パズル | 8192〜24576 | 深い推論が有効 |
| 文書要約 | 0〜512 | 単純なタスク |
| エージェント | 4096〜16384 | 計画立案が必要 |

### システムキャッシュの活用

同じシステムプロンプトを繰り返し使う場合は、コンテキストキャッシュを利用するとコストを削減できます（キャッシュされた入力トークンは通常料金の 25% で処理されます）。

## FAQ

**Q: Gemini 2.5 Flash と Gemini 2.5 Pro はどちらを使うべきですか？**

A: コスト・速度を重視するなら Flash、最高精度が必要なタスク（高度なコーディング・複雑な推論）では Pro を選択してください。Flash の thinking_budget を高めに設定すれば、多くのケースで Pro に近い品質を実現できます。

**Q: thinking_budget=0 と通常の Flash の違いは何ですか？**

A: `thinking_budget=0` はモデルの思考プロセスを無効化し、最速・最安で動作します。一般的な Q&A やテキスト変換など、推論が不要なタスクに適しています。

**Q: 1M トークンのコンテキストウィンドウは実際に使えますか？**

A: 使用可能です。ただし、トークン数が増えるほどレイテンシとコストが増加します。長文書処理には有効ですが、必要以上に長いコンテキストを含めないよう設計することをおすすめします。

**Q: REST API から直接呼び出せますか？**

A: はい。SDK を使わなくても cURL や任意の HTTP クライアントから利用できます。エンドポイントは `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent` です。
