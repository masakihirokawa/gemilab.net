# Gemini 3 Flash Agentic Vision 完全ガイド — 画像を「見て・考えて・操作する」AIの実践活用

## はじめに — Agentic Vision とは何か

従来のAIによる画像認識は「受動的な観察」にとどまっていました。画像を入力すると、モデルがその内容を記述する——それだけです。しかし、2026年1月にGoogleが発表した **Agentic Vision** は、画像理解のパラダイムを根本から変えました。

Agentic Visionは、Gemini 3 Flashに搭載された新機能で、モデルが画像に対して**能動的に調査を行い**、必要に応じて**Pythonコードを生成・実行**し、画像を**加工・分析**した上で回答を返します。つまり、AIが「見る」だけでなく「考えて」「操作する」ことで、視覚的な根拠に基づいた正確な回答を提供するのです。

この記事では、Agentic Visionの仕組みから実装方法まで、コード例付きで解説します。対象読者は、Gemini APIの基本的な使い方を理解しており、画像解析の精度を向上させたい開発者の方です。

## Agentic Vision の仕組み — Think-Act-Observe ループ

Agentic Visionの核心は、**Think-Act-Observe（考える・行動する・観察する）** という反復ループにあります。

### Think（思考）

モデルはユーザーのクエリと入力画像を分析し、回答に必要なステップを計画します。たとえば「この写真に写っている建物の数を数えてください」というプロンプトに対して、モデルは「画像をセグメンテーションして建物の輪郭を検出し、個別にカウントする」という戦略を立てます。

### Act（行動）

計画に基づいて、Gemini 3 FlashがPythonコードを生成し実行します。画像のクロップ、回転、アノテーション（注釈付け）、オブジェクトのカウント、数値計算など、さまざまな画像操作を行います。

### Observe（観察）

操作後の画像がモデルのコンテキストウィンドウに追加され、モデルは変更後の視覚データを再度確認します。この結果に基づいて、さらなる分析が必要であれば再びActフェーズに戻り、十分な情報が得られていればユーザーへの回答を生成します。

この反復プロセスにより、単純な画像記述では得られない**高精度な視覚分析**が可能になります。

## 基本実装 — Agentic Vision を有効化する

Agentic Visionを有効にするには、`code_execution` ツールと `thinking_config` を組み合わせて指定します。以下は最もシンプルな実装例です。

```python
from google import genai
from google.genai import types
import os

# クライアントの初期化
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 画像ファイルをアップロード
image = client.files.upload(file="building_photo.jpg")

# Agentic Vision を有効化してリクエスト
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=[image, "この写真に写っている建物の数を正確に数えてください"],
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)],
        thinking_config=types.ThinkingConfig(
            thinking_level="HIGH",
        ),
    ),
)

# 結果を表示
for part in response.candidates[0].content.parts:
    if part.text:
        print(part.text)
    elif part.executable_code:
        print(f"[実行されたコード]\n{part.executable_code.code}")
    elif part.code_execution_result:
        print(f"[実行結果]\n{part.code_execution_result.output}")
```

**期待される出力例:**

```
[思考] 画像を分析して建物の輪郭を検出します...

[実行されたコード]
from PIL import Image
import numpy as np
# 画像のエッジ検出と領域分割を実行
...

[実行結果]
検出された建物の数: 7

この写真には7棟の建物が確認できます。左側に3棟の低層建築、
中央に2棟の中層ビル、右側に2棟の高層タワーがあります。
```

ポイントは2つです。`code_execution` ツールを有効にすることでモデルがPythonコードを実行可能になり、`thinking_level` を `"HIGH"` に設定することでより深い推論が行われます。

## 実践例1 — 商品画像の自動分析

ECサイトの商品画像を自動で分析し、構造化データとして抽出する例を見てみましょう。

```python
from google import genai
from google.genai import types
import json
import os

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

def analyze_product_image(image_path: str) -> dict:
    """商品画像を分析し、構造化データを返す"""
    image = client.files.upload(file=image_path)

    prompt = """
    この商品画像を詳細に分析してください。
    以下の情報をJSON形式で出力してください:
    - product_name: 商品名の推定
    - colors: 使われている色のリスト
    - dimensions_ratio: 縦横比（幅:高さ）
    - text_on_image: 画像内に書かれているテキスト
    - quality_score: 商品写真としての品質スコア（1-10）
    - suggestions: 改善提案のリスト

    画像を操作して正確な情報を取得してください。
    テキストが小さい場合はクロップして拡大してから読み取ってください。
    """

    response = client.models.generate_content(
        model="gemini-3-flash-preview",
        contents=[image, prompt],
        config=types.GenerateContentConfig(
            tools=[types.Tool(code_execution=types.ToolCodeExecution)],
            thinking_config=types.ThinkingConfig(thinking_level="HIGH"),
        ),
    )

    # レスポンスからテキスト部分を抽出
    result_text = ""
    for part in response.candidates[0].content.parts:
        if part.text:
            result_text += part.text

    return result_text

# 使用例
result = analyze_product_image("product_sneaker.jpg")
print(result)
```

Agentic Visionが有効な場合、モデルは画像内の小さなテキストを読み取るために**自動的に該当部分をクロップ・拡大**します。従来の画像認識では読み取れなかった細かい情報も、この能動的なアプローチにより高精度で取得できるようになります。

    必ずコードを使って、ピクセル分析による正確な値を算出してください。
    目視での概算ではなく、計算に基づいた値を求めてください。

        model="gemini-3-flash-preview",
        contents=[image, prompt],
        config=types.GenerateContentConfig(
            tools=[types.Tool(code_execution=types.ToolCodeExecution)],
            thinking_config=types.ThinkingConfig(thinking_level="HIGH"),

従来のモデルではグラフの値を「推定」するしかありませんでしたが、Agentic Visionはピクセル位置を実際にコードで計算するため、**数値的に正確な読み取り**が可能です。

## パフォーマンスとコストの最適化

Agentic Visionは強力ですが、コード実行を伴うため通常のリクエストよりもトークン消費が増加します。以下のポイントを押さえてコストを最適化しましょう。

### thinking_level の使い分け

| thinking_level | 用途 | トークン消費 |
|---|---|---|
| `LOW` | 簡単な画像分類・キャプション | 少ない |
| `MEDIUM` | 一般的な画像分析 | 中程度 |
| `HIGH` | 複雑な計測・数値抽出 | 多い |

単純なタスクには `LOW` を、精密な分析が必要な場合のみ `HIGH` を使用するのが効率的です。

### media_resolution との組み合わせ

Gemini 3で導入された [media_resolution パラメータ](/articles/gemini-api/gemini-3-media-resolution-api-guide) と組み合わせることで、さらに精度とコストのバランスを調整できます。

```python
# 高解像度での詳細分析
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=[
        types.Content(
            parts=[
                types.Part.from_uri(
                    file_uri=image.uri,
                    mime_type="image/jpeg",
                    video_metadata=types.VideoMetadata(
                        media_resolution="HIGH"
                    ),
                ),
                types.Part.from_text("この画像の細部を分析してください"),
            ]
        )
    ],
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)],
        thinking_config=types.ThinkingConfig(thinking_level="HIGH"),
    ),
)
```

### バッチ処理でコストを削減

大量の画像を処理する場合は、[Batch API](/articles/gemini-api/gemini-batch-processing-api-guide) の利用も検討してください。非同期処理で50%のコスト削減が見込めます。

## Agentic Vision が効果的なユースケース

Agentic Visionは以下のようなシーンで特に効果を発揮します。

**高精度が必要な場面**: グラフの数値読み取り、建築図面の計測、医療画像の領域分析など、「だいたい合っている」では不十分なタスクに最適です。

**小さなテキストの読み取り**: 画像内の注釈、ラベル、型番などの細かいテキストを、クロップ・拡大して正確に読み取ります。

**カウンティングタスク**: 画像内のオブジェクトを正確にカウントする必要がある場面。セグメンテーションとマスク処理をコードで行うことで、目視よりも正確にカウントできます。

**比較分析**: 2枚の画像の差分検出、変化の定量化など、複数画像にまたがる分析にも対応します。

## よくある質問（FAQ）

### Q1: Agentic Vision は Gemini 3 Pro でも使えますか？

Agentic Visionは現在 **Gemini 3 Flash** で利用可能です。Gemini 3 Proでもコード実行自体は対応していますが、Agentic Visionとして最適化されているのはFlashモデルです。Flashモデルはコストパフォーマンスに優れているため、画像分析タスクには [Gemini 3 Flash](/articles/gemini-basics/gemini-3-flash-guide) の使用をおすすめします。

### Q2: Agentic Vision で実行できるPythonコードに制限はありますか？

コード実行はサンドボックス環境で行われるため、外部ネットワークへのアクセスやファイルシステムへの永続的な書き込みはできません。利用可能なライブラリは `numpy`、`PIL（Pillow）`、`matplotlib` などの主要な科学計算・画像処理ライブラリに限られます。

### Q3: 通常の画像認識とAgentic Vision、どちらを使うべきですか？

画像のキャプション生成や大まかな分類であれば、通常の画像認識（コード実行なし）で十分です。数値の正確な読み取り、細部の分析、画像内テキストの抽出など**精密な結果が求められるタスク**では、Agentic Visionの使用を推奨します。コード実行を伴う分、レイテンシとコストは増加しますので、タスクの要件に応じて使い分けましょう。

### Q4: Agentic Vision のレスポンスはどのくらい時間がかかりますか？

Think-Act-Observeループの反復回数に依存しますが、一般的に通常のリクエストの2〜5倍程度の時間がかかります。単純なタスクで3〜5秒、複雑な分析で10〜20秒程度が目安です。リアルタイム性が求められるアプリケーションでは、非同期処理との併用を検討してください。
