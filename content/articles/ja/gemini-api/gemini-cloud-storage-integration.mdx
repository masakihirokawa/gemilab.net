## はじめに：Gemini APIとCloud Storageの連携が大幅強化

2026年3月、GoogleはGemini APIのファイル入力機能を大幅にアップデートしました。従来の **File API**（20MBまで）に加え、**Google Cloud Storage（GCS）バケット**および**プリサインドURL（署名付きURL）**を直接データソースとして指定できるようになりました。さらに、ファイルサイズの上限が **100MB** に引き上げられました。

この変更により、以下のようなユースケースが現実的になります：

- 大容量の動画ファイルをGCSに保存しGeminiで分析
- 社内のS3互換ストレージやAzure Blob StorageのプリサインドURLを活用
- Vertex AIパイプラインとの連携でエンタープライズ規模の処理を実現

本記事では、この新しいCloud Storage連携の設定方法から実践的なコード例まで、徹底的に解説します。

## Cloud Storage連携の仕組み

### 従来のFile APIとの違い

| 項目 | 旧来のFile API | 新・Cloud Storage連携 |
|------|---------------|----------------------|
| ファイルサイズ上限 | 20MB | **100MB** |
| ストレージ | Googleサーバー（一時保存） | **GCSバケット / 任意のプリサインドURL** |
| URLサポート | なし | **GCS URI・HTTP署名付きURL** |
| 永続性 | 48時間で自動削除 | バケット側で管理 |
| コスト | APIへの転送コスト | GCSのストレージ + 転送コスト |

### サポートされる入力形式

新機能では、以下の形式でファイルを指定できます：

1. **GCS URI**（例：`gs://my-bucket/my-video.mp4`）
2. **公開URL**（HTTPSで直接アクセス可能なファイル）
3. **プリサインドURL**（署名付きの一時的なURL）

## 実装：PythonでGCS連携

### 方法1：GCS URIを直接指定する

GCSバケットに保存したファイルをGeminiに直接渡す最もシンプルな方法です。

```python
import google.generativeai as genai
import os

# APIキーの設定
genai.configure(api_key=os.environ["GEMINI_API_KEY"])

# Gemini 3 Proモデルを使用
model = genai.GenerativeModel("gemini-3-pro-preview")

# GCS URIを直接指定（gs:// 形式）
gcs_uri = "gs://my-project-bucket/documents/annual_report_2025.pdf"

response = model.generate_content([
    {
        "file_data": {
            "mime_type": "application/pdf",
            "file_uri": gcs_uri  # GCS URIをそのまま指定
        }
    },
    "このPDFレポートの主要な財務指標を箇条書きでまとめてください。"
])

print(response.text)

# 期待する出力例:
# - 売上高: 前年比15%増（1,200億円）
# - 営業利益率: 18.5%（前年比2.1ポイント改善）
# - 海外売上比率: 42%（前年比5ポイント増）
# - 設備投資額: 80億円（成長領域への重点配分）
```

### 方法2：プリサインドURLを使用する

S3やAzure Blob Storageなど、Google Cloud以外のサービスのファイルも活用できます。

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-3-flash-preview")

# AWS S3のプリサインドURL（一時的なアクセス用URL）
presigned_url = "https://my-s3-bucket.s3.amazonaws.com/videos/product_demo.mp4?X-Amz-Signature=..."

response = model.generate_content([
    {
        "file_data": {
            "mime_type": "video/mp4",
            "file_uri": presigned_url  # プリサインドURLも同じ形式で指定
        }
    },
    "この製品デモ動画の主な特徴と改善点を分析してください。"
])

print(response.text)
```

### 方法3：大容量ファイル（最大100MB）の処理

新しい上限100MBを活用した、大容量動画ファイルの処理例です。

```python
import google.generativeai as genai
import os
from google.cloud import storage

def analyze_large_video_from_gcs(
    bucket_name: str,
    blob_name: str,
    prompt: str
) -> str:
    """
    GCSの大容量動画ファイル（最大100MB）をGeminiで分析する

    Args:
        bucket_name: GCSバケット名
        blob_name: ファイルパス（バケット内）
        prompt: 分析指示プロンプト

    Returns:
        Geminiの分析結果テキスト
    """
    genai.configure(api_key=os.environ["GEMINI_API_KEY"])
    model = genai.GenerativeModel("gemini-3-pro-preview")

    # GCS URIを構築
    gcs_uri = f"gs://{bucket_name}/{blob_name}"

    # ファイルサイズを確認（オプション）
    storage_client = storage.Client()
    bucket = storage_client.bucket(bucket_name)
    blob = bucket.blob(blob_name)
    blob.reload()
    file_size_mb = blob.size / (1024 * 1024)

    if file_size_mb > 100:
        raise ValueError(f"ファイルサイズ {file_size_mb:.1f}MB は上限100MBを超えています")

    print(f"📁 ファイルサイズ: {file_size_mb:.1f}MB")
    print(f"🚀 Geminiで分析中...")

    response = model.generate_content([
        {
            "file_data": {
                "mime_type": "video/mp4",
                "file_uri": gcs_uri
            }
        },
        prompt
    ])

    return response.text


# 使用例
result = analyze_large_video_from_gcs(
    bucket_name="my-media-bucket",
    blob_name="recordings/webinar_2026_q1.mp4",
    prompt="このウェビナー動画のハイライトを5点にまとめ、タイムスタンプ付きで教えてください。"
)
print(result)

# 期待する出力例:
# [00:02:15] イントロダクション：2026年のAIトレンド概観
# [00:08:30] Gemini 3の新機能デモ（思考モード・コンピュータ使用ツール）
# [00:22:45] エンタープライズ向けVertex AI活用事例
# [00:38:10] Q&A：開発者からの技術的な質問への回答
# [00:52:00] まとめと次回ウェビナーの告知
```

## 実装：JavaScriptでCloud Storage連携

Node.jsでGCSファイルをGemini APIに渡す方法です。

```javascript
import { GoogleGenerativeAI } from "@google/genai";

const genai = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

/**
 * GCSのファイルをGeminiで分析する
 * @param {string} gcsUri - gs://バケット名/ファイルパス
 * @param {string} mimeType - MIMEタイプ
 * @param {string} prompt - 分析プロンプト
 * @returns {Promise<string>} 分析結果
 */
async function analyzeFileFromGCS(gcsUri, mimeType, prompt) {
  const model = genai.getGenerativeModel({ model: "gemini-3-flash-preview" });

  const result = await model.generateContent([
    {
      fileData: {
        mimeType: mimeType,
        fileUri: gcsUri, // gs:// URI または HTTPS プリサインドURL
      },
    },
    { text: prompt },
  ]);

  return result.response.text();
}

// 使用例：GCSの画像ファイルを分析
const response = await analyzeFileFromGCS(
  "gs://product-images-bucket/catalog/item_001.jpg",
  "image/jpeg",
  "この商品画像の特徴を説明し、ECサイト向けの商品説明文を200文字で作成してください。"
);

console.log(response);
// 期待する出力例:
// 【商品説明】
// シンプルで洗練されたデザインのノートパソコンスタンド。
// アルミニウム合金製で高い耐久性を誇り、6段階の角度調整が可能。
// 放熱効果に優れ、長時間作業でも快適な姿勢を維持できます。
// MacBookをはじめ15インチまでのノートPCに対応。

// バッチ処理の例：複数ファイルを並列処理
async function batchAnalyzeFromGCS(files) {
  const promises = files.map(({ uri, mimeType, prompt }) =>
    analyzeFileFromGCS(uri, mimeType, prompt)
  );

  return Promise.all(promises);
}

const files = [
  {
    uri: "gs://reports-bucket/q1_2026.pdf",
    mimeType: "application/pdf",
    prompt: "この四半期レポートのKPIを表形式でまとめてください。"
  },
  {
    uri: "gs://reports-bucket/q2_2026.pdf",
    mimeType: "application/pdf",
    prompt: "この四半期レポートのKPIを表形式でまとめてください。"
  }
];

const results = await batchAnalyzeFromGCS(files);
results.forEach((result, i) => {
  console.log(`\n=== Q${i + 1} レポート ===`);
  console.log(result);
});
```

## GCSバケットの権限設定

Gemini APIがGCSバケットにアクセスするには、適切なIAM権限が必要です。

### サービスアカウントの設定

```bash
# サービスアカウントの作成
gcloud iam service-accounts create gemini-gcs-reader \
  --display-name="Gemini GCS Reader"

# バケットへの読み取り権限を付与
gcloud storage buckets add-iam-policy-binding gs://my-bucket \
  --member="serviceAccount:gemini-gcs-reader@PROJECT_ID.iam.gserviceaccount.com" \
  --role="roles/storage.objectViewer"

# キーファイルを生成（ローカル開発用）
gcloud iam service-accounts keys create key.json \
  --iam-account=gemini-gcs-reader@PROJECT_ID.iam.gserviceaccount.com
```

### 環境変数で認証を設定

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/key.json"
```

> **Vertex AI利用の場合**: Vertex AIを通じてGeminiにアクセスすると、GCSの認証が自動的に統合されます。詳細は [Vertex AI Gemini入門ガイド](/articles/gemini-api/vertex-ai-quickstart) を参照してください。

## よくあるエラーと対処法

### エラー1：アクセス権限エラー

```
PermissionDenied: 403 GET https://storage.googleapis.com/.../file.pdf:
ACCESS_DENIED
```

**対処法**: サービスアカウントに `storage.objectViewer` 権限を付与しているか確認してください。GCSバケットのIAMポリシーを確認します。

### エラー2：ファイルサイズオーバー

```
InvalidArgument: 400 File size exceeds the limit of 100MB
```

**対処法**: ファイルが100MBを超えています。動画の場合は解像度を下げるか、分割して処理してください。PDFの場合は、必要なページのみを抽出することも有効です。

### エラー3：サポートされていないMIMEタイプ

```
InvalidArgument: Unsupported MIME type
```

**対処法**: `mime_type` パラメータが正しいことを確認してください。主要なMIMEタイプは以下の通りです：

| ファイル形式 | MIMEタイプ |
|------------|-----------|
| PDF | `application/pdf` |
| MP4動画 | `video/mp4` |
| JPEG画像 | `image/jpeg` |
| PNG画像 | `image/png` |
| MP3音声 | `audio/mp3` |
| WAV音声 | `audio/wav` |
| テキスト | `text/plain` |

### エラー4：プリサインドURLの期限切れ

```
InvalidArgument: URL has expired
```

**対処法**: プリサインドURLの有効期限を確認してください。Geminiがファイルを処理する時間（通常1〜3分）を考慮して、十分な有効期限を設定してください（推奨: 最低30分）。

## 応用：エンタープライズ規模のパイプライン

Cloud Storage連携を活用した、スケーラブルなドキュメント処理パイプラインの例です。

```python
import google.generativeai as genai
from google.cloud import storage, pubsub_v1
import json
import os

def process_document_pipeline(event, context):
    """
    Cloud Functionsトリガーで動作するドキュメント処理パイプライン
    GCSにファイルがアップロードされると自動的にGeminiで分析する

    Cloud FunctionsのGCSトリガーイベントを受け取る
    """
    genai.configure(api_key=os.environ["GEMINI_API_KEY"])
    model = genai.GenerativeModel("gemini-3-flash-preview")

    # イベントからGCSファイル情報を取得
    bucket_name = event["bucket"]
    file_name = event["name"]
    mime_type = event.get("contentType", "application/octet-stream")

    gcs_uri = f"gs://{bucket_name}/{file_name}"
    print(f"📄 処理中: {gcs_uri}")

    # ファイルタイプに応じてプロンプトを切り替え
    prompts = {
        "application/pdf": "このPDFの要約と重要なキーワードをJSON形式で返してください。",
        "image/jpeg": "この画像の内容を説明し、タグをJSON配列で返してください。",
        "image/png": "この画像の内容を説明し、タグをJSON配列で返してください。",
        "video/mp4": "この動画のシーン一覧とタイムスタンプをJSON形式で返してください。"
    }

    prompt = prompts.get(mime_type, "このファイルの内容を要約してください。")

    # Geminiで分析
    response = model.generate_content(
        [{"file_data": {"mime_type": mime_type, "file_uri": gcs_uri}}, prompt],
        generation_config={"response_mime_type": "application/json"}
    )

    result = {
        "file": gcs_uri,
        "analysis": json.loads(response.text)
    }

    # 結果をPub/Subに送信して後続処理へ
    publisher = pubsub_v1.PublisherClient()
    topic_path = publisher.topic_path(
        os.environ["GOOGLE_CLOUD_PROJECT"],
        "gemini-analysis-results"
    )
    publisher.publish(topic_path, json.dumps(result).encode())

    print(f"✅ 分析完了: {file_name}")
    return result
```

このパターンは以下のような用途に活用できます：

- 契約書・請求書の自動データ抽出
- 製品画像の自動タグ付けとカタログ管理
- 会議録音の自動文字起こしと要約
- セキュリティ監視カメラ映像の異常検知

詳しいVertex AIでの本番環境構築については、[Vertex AI Gemini本番環境ガイド](/articles/gemini-api/vertex-ai-gemini-production) をご参照ください。

## File APIとCloud Storage連携の使い分け

| ユースケース | 推奨する方法 |
|-------------|------------|
| 20MB未満の一時ファイル | File API |
| 20〜100MBの大容量ファイル | **Cloud Storage連携（新機能）** |
| GCPインフラと統合されたシステム | **GCS URI直接指定** |
| AWS S3やAzure Blobのファイル | **プリサインドURL** |
| 永続的に再利用するファイル | GCS + URI指定 |
| プロトタイピング・テスト | File API |

## FAQ

### Q1. Cloud Storage連携にはVertex AIが必要ですか？

**A.** いいえ、必須ではありません。GCS URIはGemini Developer API（Gemini APIキー）からも使用できます。ただし、Vertex AIを使用すると、Google Cloudの認証・IAMが自動的に統合されるため、大規模なエンタープライズ用途ではVertex AIの利用を推奨します。

### Q2. プリサインドURLを使う際のセキュリティは？

**A.** プリサインドURLはGeminiのAPIサーバーからアクセスされます。有効期限を短く設定し（30分〜1時間推奨）、使用後は無効化することをお勧めします。また、APIキーは必ず環境変数や Secret Manager で管理し、コードに埋め込まないようにしてください。

### Q3. 100MBを超えるファイルはどう処理すればよいですか？

**A.** 100MB以上のファイルは、現時点では直接処理できません。動画であればFFmpegで分割するか解像度を下げる、PDFであればページ単位に分割するなどの前処理が必要です。非常に大きなファイルの場合は、Vertex AIのLong Context機能や、動画の場合はシーンごとにサムネイルを抽出してマルチモーダル処理するアプローチも有効です。

### Q4. 非公開のGCSファイルにアクセスできますか？

**A.** はい、可能です。Gemini APIがGCSにアクセスするためのサービスアカウントに `storage.objectViewer` 権限を付与することで、非公開バケットのファイルも処理できます。Vertex AI経由の場合はWorkload Identityが使用できるため、よりセキュアな構成が可能です。

### Q5. 対応しているファイル形式は？

**A.** PDF、画像（JPEG、PNG、WebP、HEIC）、動画（MP4、MOV、AVI等）、音声（MP3、WAV、FLAC等）、テキスト（TXT、HTML、Markdown、CSV等）が対応しています。ただし、Geminiのモデルによってサポートするモダリティが異なるため、動画や音声はGemini 3 Pro/Flashなど対応モデルを使用してください。

Gemini APIのCloud Storage連携は、エンタープライズ規模のAIワークフローを大幅に簡素化する重要なアップデートです。

今回のポイントをまとめます：

- **ファイルサイズ上限が100MBに拡大**（従来は20MB）
- **GCS URIを直接指定**できるため、アップロード不要で処理可能

まずは [Google AI Studio](https://aistudio.google.com/) でAPIキーを取得し、GCSのファイルをGeminiに渡してみましょう。既存のクラウドインフラとシームレスに統合できる強力な機能です。
