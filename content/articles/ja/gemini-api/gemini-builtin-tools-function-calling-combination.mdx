# Gemini API 組み込みツールとFunction Callingの組み合わせ完全ガイド

## はじめに

2026年3月18日、GoogleはGemini APIに**組み込みツール（Built-in Tools）とFunction Callingの組み合わせ**機能を正式にリリースしました。この機能を使うと、Google SearchやGoogle Maps、コード実行といったサーバーサイドの組み込みツールと、開発者が独自に定義したカスタム関数を**1回のAPIリクエスト**で同時に利用できるようになります。

この記事では以下を学べます。

- ツールコンテキスト循環（Tool Context Circulation）の仕組み
- `include_server_side_tool_invocations` フラグの使い方
- PythonとJavaScriptによる完全なコード例
- マルチターン会話でのコンテキスト保持の方法
- 実際のユースケースと活用パターン

対象読者は、Gemini APIのFunction Callingを理解している中〜上級の開発者です。Function Callingの基礎から学びたい方は、まず[Function Calling完全ガイド](/articles/gemini-advanced/function-calling-guide)をご覧ください。

## ツールコンテキスト循環とは

従来のGemini APIでは、組み込みツール（Google Searchなど）とカスタム関数を同時に使うことができませんでした。今回リリースされた**ツールコンテキスト循環（Tool Context Circulation）**により、以下のワークフローが単一の会話ターン内で実現できます。

1. ユーザーのクエリを受信
2. Geminiが**組み込みツール**（例：Google Search）をサーバーサイドで実行
3. その結果（toolCall / toolResponse）をモデルのコンテキストに保持
4. 同じコンテキストで**カスタム関数**（functionCall）を呼び出し
5. クライアントが関数を実行してレスポンスを返却
6. 最終的な統合応答を生成

この仕組みにより、「リアルタイムデータを検索→業務ロジックを実行→結果を統合」というエージェント的なワークフローが大幅に簡素化されます。

### サポートされる組み込みツール

| ツール | 実行場所 | 用途 |
|--------|----------|------|
| Google Search | サーバーサイド | リアルタイム情報検索 |
| Google Maps | サーバーサイド | 地図・店舗情報検索 |
| URL Context | サーバーサイド | Webページ内容取得 |
| File Search | サーバーサイド | ドキュメント検索 |
| Code Execution | サーバーサイド | コード実行・計算 |
| カスタム関数 | クライアントサイド | 任意のビジネスロジック |

> **注意**: この機能は現在プレビュー版で、**Gemini 3シリーズのモデルのみ**で利用可能です（`gemini-3-flash-preview`、`gemini-3.1-pro-preview`など）。

## Pythonによる実装例

以下は「最も北にある米国の都市を調べ、その天気を取得する」という2段階のワークフローの例です。Google Searchで地名を調べ、カスタム関数で天気APIを呼び出します。

```python
from google import genai
from google.genai import types
import os

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# カスタム関数の定義（天気取得）
get_weather_tool = {
    "name": "getWeather",
    "description": "指定された都市の現在の天気情報を取得します。",
    "parameters": {
        "type": "object",
        "properties": {
            "city": {
                "type": "string",
                "description": "都市名（例：東京、ニューヨーク）",
            },
        },
        "required": ["city"],
    },
}

# ターン1：組み込みツール（Google Search）＋カスタム関数を同時指定
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents="アメリカで最も北にある都市はどこですか？そこの今日の天気も教えてください。",
    config=types.GenerateContentConfig(
        tools=[
            types.Tool(
                google_search=types.ToolGoogleSearch(),
                function_declarations=[get_weather_tool],
            ),
        ],
        # ⚠️ 必須: このフラグでツールコンテキスト循環を有効化
        include_server_side_tool_invocations=True,
    ),
)

# レスポンスの確認
for part in response.candidates[0].content.parts:
    if hasattr(part, "function_call"):
        print(f"🔧 カスタム関数呼び出し: {part.function_call.name}")
        print(f"   引数: {part.function_call.args}")
    elif hasattr(part, "tool_code"):
        print(f"🔍 組み込みツール実行結果あり")

# ターン2：カスタム関数の実行結果を返す
# （実際のシステムでは天気APIを呼び出してデータを取得）
weather_data = {
    "city": "Utqiaġvik, Alaska",
    "temperature": -25,
    "condition": "快晴",
    "wind": "北西 15km/h",
}

# 会話履歴にターン1のレスポンスを追加し、functionResponseを送信
history = [
    types.Content(
        role="user",
        parts=[types.Part(text="アメリカで最も北にある都市はどこですか？そこの今日の天気も教えてください。")]
    ),
    response.candidates[0].content,  # ターン1のモデルレスポンス（thought_signatureを含む）
    types.Content(
        role="user",
        parts=[
            types.Part(
                function_response=types.FunctionResponse(
                    name="getWeather",
                    response=weather_data,
                )
            )
        ],
    ),
]

# ターン2の送信（ツールを再指定することが必要）
final_response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=history,
    config=types.GenerateContentConfig(
        tools=[
            types.Tool(
                google_search=types.ToolGoogleSearch(),
                function_declarations=[get_weather_tool],
            ),
        ],
        include_server_side_tool_invocations=True,
    ),
)

print("\n📝 最終回答:")
print(final_response.text)
# 期待する出力例:
# アメリカ最北端の都市はアラスカ州のUtqiaġvik（ウトキアービック）です。
# 今日の天気は快晴で気温は-25℃、北西からの風が15km/hで吹いています。
```

## JavaScriptによる実装例

```javascript
import { GoogleGenAI } from '@google/genai';

const client = new GoogleGenAI({ apiKey: process.env.GOOGLE_API_KEY });

// カスタム関数の定義
const getWeatherTool = {
    name: "getWeather",
    description: "指定された都市の天気情報を取得します",
    parameters: {
        type: "OBJECT",
        properties: {
            city: {
                type: "STRING",
                description: "都市名（例：東京）"
            }
        },
        required: ["city"]
    }
};

// ツール設定（組み込み＋カスタムを同時指定）
const tools = [
    { googleSearch: {} },
    { functionDeclarations: [getWeatherTool] }
];

async function runCombinedTools() {
    // ターン1：初回クエリ
    const result = await client.models.generateContent({
        model: "gemini-3-flash-preview",
        contents: [{
            role: "user",
            parts: [{ text: "アメリカ最北端の都市を調べて、そこの天気も取得してください。" }]
        }],
        generationConfig: {
            // ツールコンテキスト循環を有効化
            includeServerSideToolInvocations: true,
        },
        tools: tools,
    });

    const modelContent = result.candidates[0].content;

    // functionCallパートを見つける
    let functionCallPart = null;
    for (const part of modelContent.parts) {
        if (part.functionCall) {
            console.log(`🔧 関数呼び出し: ${part.functionCall.name}`);
            console.log(`   引数: ${JSON.stringify(part.functionCall.args)}`);
            functionCallPart = part;
        }
    }

    if (!functionCallPart) {
        console.log("カスタム関数の呼び出しなし");
        return;
    }

    // 天気データの取得（実際はAPIを呼び出す）
    const weatherData = {
        city: functionCallPart.functionCall.args.city,
        temperature: -25,
        condition: "快晴",
        wind: "北西 15km/h"
    };

    // ターン2：functionResponseを含めて送信
    const finalResult = await client.models.generateContent({
        model: "gemini-3-flash-preview",
        contents: [
            { role: "user", parts: [{ text: "アメリカ最北端の都市を調べて、そこの天気も取得してください。" }] },
            modelContent, // ⚠️ thought_signatureを含むターン1のレスポンスを必ずそのまま返す
            {
                role: "user",
                parts: [{
                    functionResponse: {
                        name: "getWeather",
                        response: weatherData
                    }
                }]
            }
        ],
        generationConfig: {
            includeServerSideToolInvocations: true,
        },
        tools: tools,
    });

    console.log("\n📝 最終回答:");
    console.log(finalResult.candidates[0].content.parts[0].text);
}

runCombinedTools().catch(console.error);
```

## 重要なポイント：thought_signature の保持

この機能を正しく動作させるうえで最も重要なのが、**thought_signature（思考シグネチャ）の保持**です。

各パートに含まれる `thought_signature` フィールドは暗号化されたコンテキスト情報であり、これをモデルに返さないと次のターンでエラーが発生します。

```python
# ❌ NG: パートを手動で再構築するとthought_signatureが失われる
history_wrong = [
    types.Content(role="model", parts=[
        types.Part(text=response.text)  # thought_signatureが含まれない！
    ])
]

# ✅ OK: ターン1のレスポンスをそのまま使用する
history_correct = [
    response.candidates[0].content  # thought_signatureを含む完全なコンテンツ
]
```

thought_signatureの詳細については[Gemini Thought Signatures完全ガイド](/articles/gemini-api/gemini-thought-signatures-guide)を参照してください。

## よくあるエラーと対処法

### エラー1：`include_server_side_tool_invocations` を指定し忘れ

**症状**: 組み込みツールの結果がコンテキストに含まれず、カスタム関数との連携が機能しない。

**対処法**: 全ターンで `include_server_side_tool_invocations=True`（Python）または `includeServerSideToolInvocations: true`（JS）を指定する。

### エラー2：thought_signature の欠落

**症状**: 2ターン目で `InvalidArgument` または `FAILED_PRECONDITION` エラーが発生する。

**対処法**: ターン1のレスポンスオブジェクトを加工せずそのまま履歴に追加する。

### エラー3：AUTO モードの使用

**症状**: `FUNCTION_CALLING_MODE.AUTO` を指定するとエラーになる。

**対処法**: `VALIDATED` モードを使用する（デフォルト）。この機能はAUTOモード非対応。

# 組み込みツール: Google Search（株価検索）

## FAQ

**Q1. この機能はGemini 2.5シリーズでも使えますか？**

A1. いいえ。現時点では**Gemini 3シリーズのモデルのみ**でサポートされています（`gemini-3-flash-preview`、`gemini-3.1-pro-preview`）。Gemini 2.5では、組み込みツールとFunction Callingを別々のAPIコールで管理する従来の方法を使用してください。

**Q2. 複数の組み込みツールを同時に使えますか？**

A2. はい。ただし、1つの `types.Tool` オブジェクトに複数の組み込みツールとカスタム関数を含めることができます。たとえば、Google SearchとCode Executionを同時に指定することも可能です。どの組み込みツールを実行するかはモデルが自律的に判断します。

**Q3. 1ターンに何回まで関数が呼び出されますか？**

A3. 1ターンのレスポンスで複数の関数呼び出しが発生する場合があります（並列Function Calling）。`response.candidates[0].content.parts` をループして、すべての `function_call` パートを処理するようにコードを設計してください。すべてのレスポンスを返した後、再度 `generate_content` を呼び出します。

**Q4. コストへの影響はありますか？**

A4. 組み込みツールの実行ごとにトークンが消費されます。また、Google Searchグラウンディングは使用回数ベースの課金が発生する場合があります。コスト管理には[Gemini API コスト最適化ガイド](/articles/gemini-api/gemini-api-cost-optimization)を参照し、プロジェクトの支出上限設定も活用してください。

## まとめ

Gemini APIの組み込みツールとFunction Callingの組み合わせ機能（ツールコンテキスト循環）は、エージェントアーキテクチャを大幅に簡素化する強力な新機能です。

- `include_server_side_tool_invocations=True` を必ず指定する
- ターン1のレスポンスオブジェクトを加工せず履歴に含める
- thought_signature は絶対に省略しない
- 現在はGemini 3モデルのみ対応（プレビュー版）

この機能を活用することで、リアルタイムデータ検索から業務ロジック実行まで、複雑なエージェントワークフローをよりシンプルなコードで実現できます。
