## 構造化出力とは何か

Gemini API の構造化出力機能を使うと、モデルのレスポンスを JSON スキーマに準拠した形式で受け取れます。自由形式のテキスト出力をパースする手間がなくなり、アプリケーションとの統合が格段にシンプルになります。

従来のプロンプトエンジニアリングでは「JSON で返して」と指示しても、余計なマークダウン記法が混入したり、フィールドが欠落したりする問題がありました。構造化出力はこれをプロトコルレベルで解決します。

---

## 基本的な JSON Mode の使い方

最もシンプルな構造化出力は、`responseMimeType` を `application/json` に設定する方法です。

```typescript
// basic-json-mode.ts
// Gemini API の JSON Mode 基本例

import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

const model = genAI.getGenerativeModel({
  model: "gemini-2.0-flash",
  generationConfig: {
    responseMimeType: "application/json",
  },
});

const result = await model.generateContent(
  "東京の天気予報を3日分、JSON形式で返してください。各日に date, weather, high_temp, low_temp を含めてください。"
);

const forecast = JSON.parse(result.response.text());
console.log(forecast);

// 期待出力:
// [
//   { "date": "2026-03-14", "weather": "晴れ", "high_temp": 18, "low_temp": 8 },
//   { "date": "2026-03-15", "weather": "曇り", "high_temp": 15, "low_temp": 7 },
//   { "date": "2026-03-16", "weather": "雨", "high_temp": 12, "low_temp": 6 }
// ]
```

ただし、この方法ではレスポンスのフィールド名や型が保証されません。より厳密な制御には `responseSchema` を使います。

## スキーマ定義による型安全な出力

`responseSchema` を指定すると、Gemini はスキーマに完全に準拠した JSON を返します。フィールドの欠落や型の不一致が起きません。

```typescript
// schema-defined-output.ts
// responseSchema で出力の型を厳密に制御

import { GoogleGenerativeAI, SchemaType } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

const model = genAI.getGenerativeModel({
  model: "gemini-2.0-flash",
  generationConfig: {
    responseMimeType: "application/json",
    responseSchema: {
      type: SchemaType.OBJECT,
      properties: {
        products: {
          type: SchemaType.ARRAY,
          items: {
            type: SchemaType.OBJECT,
            properties: {
              name: { type: SchemaType.STRING, description: "商品名" },
              category: {
                type: SchemaType.STRING,
                enum: ["electronics", "clothing", "food", "other"],
                description: "商品カテゴリ",
              },
              price: { type: SchemaType.NUMBER, description: "税込価格（円）" },
              inStock: { type: SchemaType.BOOLEAN, description: "在庫あり" },
            },
            required: ["name", "category", "price", "inStock"],
          },
        },
        totalCount: { type: SchemaType.INTEGER, description: "商品の総数" },
      },
      required: ["products", "totalCount"],
    },
  },
});

const result = await model.generateContent(
  "オンラインストアの人気商品トップ3を返してください。"
);

const data = JSON.parse(result.response.text());
console.log(data);

// 期待出力:
// {
//   "products": [
//     { "name": "ワイヤレスイヤホン Pro", "category": "electronics", "price": 12800, "inStock": true },
//     { "name": "オーガニックコットンTシャツ", "category": "clothing", "price": 3980, "inStock": true },
//     { "name": "プレミアム抹茶パウダー", "category": "food", "price": 2480, "inStock": false }
//   ],
//   "totalCount": 3
// }
```

`enum` を使うことで、カテゴリ値を事前定義した選択肢に限定できます。`required` フィールドの指定により、必須項目の欠落も防げます。

## TypeScript との型安全な統合

Gemini のスキーマ定義と TypeScript の型システムを組み合わせることで、エンドツーエンドの型安全を実現できます。

```typescript
// typed-gemini-client.ts
// TypeScript の型システムと Gemini の構造化出力を統合

import { GoogleGenerativeAI, SchemaType } from "@google/generative-ai";

// TypeScript の型定義（アプリケーション側で使用）
interface BookReview {
  title: string;
  author: string;
  rating: number;
  summary: string;
  tags: string[];
}

interface ReviewResponse {
  reviews: BookReview[];
  generatedAt: string;
}

// Gemini スキーマ定義（API リクエスト側で使用）
const reviewSchema = {
  type: SchemaType.OBJECT,
  properties: {
    reviews: {
      type: SchemaType.ARRAY,
      items: {
        type: SchemaType.OBJECT,
        properties: {
          title: { type: SchemaType.STRING },
          author: { type: SchemaType.STRING },
          rating: { type: SchemaType.NUMBER },
          summary: { type: SchemaType.STRING },
          tags: {
            type: SchemaType.ARRAY,
            items: { type: SchemaType.STRING },
          },
        },
        required: ["title", "author", "rating", "summary", "tags"],
      },
    },
    generatedAt: { type: SchemaType.STRING },
  },
  required: ["reviews", "generatedAt"],
} as const;

// 型安全なヘルパー関数
async function generateBookReviews(genre: string): Promise<ReviewResponse> {
  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);
  const model = genAI.getGenerativeModel({
    model: "gemini-2.0-flash",
    generationConfig: {
      responseMimeType: "application/json",
      responseSchema: reviewSchema,
    },
  });

  const result = await model.generateContent(
    `${genre}ジャンルのおすすめ書籍を3冊、レビュー付きで返してください。ratingは1-5の小数点1桁。`
  );

  // JSON.parse の結果を型アサーション
  const parsed: ReviewResponse = JSON.parse(result.response.text());
  return parsed;
}

// 使用例
const reviews = await generateBookReviews("SF小説");
console.log(`${reviews.reviews.length} reviews generated at ${reviews.generatedAt}`);
reviews.reviews.forEach((r) => {
  console.log(`  ${r.title} by ${r.author} — ${r.rating}/5`);
});

// 期待出力:
// 3 reviews generated at 2026-03-14T18:00:00Z
//   三体 by 劉慈欣 — 4.8/5
//   プロジェクト・ヘイル・メアリー by アンディ・ウィアー — 4.6/5
//   ニューロマンサー by ウィリアム・ギブスン — 4.3/5
```

## バリデーション層の追加

スキーマでの制約に加え、アプリケーション層で Zod を使ったバリデーションを追加すると、より堅牢になります。

```typescript
// validation-layer.ts
// Zod によるランタイムバリデーション

import { z } from "zod";

const BookReviewSchema = z.object({
  reviews: z.array(
    z.object({
      title: z.string().min(1),
      author: z.string().min(1),
      rating: z.number().min(1).max(5),
      summary: z.string().min(10).max(500),
      tags: z.array(z.string()).min(1).max(5),
    })
  ),
  generatedAt: z.string().datetime(),
});

type ValidatedReviewResponse = z.infer<typeof BookReviewSchema>;

function validateResponse(raw: unknown): ValidatedReviewResponse {
  const result = BookReviewSchema.safeParse(raw);
  if (!result.success) {
    console.error("Validation errors:", result.error.flatten());
    throw new Error(`Invalid response: ${result.error.message}`);
  }
  return result.data;
}

// 使用例
const rawData = JSON.parse(geminiResponse);
const validated = validateResponse(rawData);
console.log(`Validated ${validated.reviews.length} reviews`);

// 期待出力: Validated 3 reviews
// (バリデーションエラーがあれば詳細なエラーメッセージが出力される)
```

## エラーハンドリングとリトライ

構造化出力でも、レート制限やネットワークエラーは発生します。堅牢なリトライロジックを組み込みましょう。

```typescript
// retry-handler.ts
// 指数バックオフによるリトライ

async function withRetry<T>(
  fn: () => Promise<T>,
  maxRetries = 3,
  baseDelay = 1000
): Promise<T> {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;

      const delay = baseDelay * Math.pow(2, attempt);
      const jitter = Math.random() * delay * 0.1;
      console.log(`Retry ${attempt + 1}/${maxRetries} in ${Math.round(delay + jitter)}ms`);
      await new Promise((r) => setTimeout(r, delay + jitter));
    }
  }
  throw new Error("Unreachable");
}

// 使用例
const reviews = await withRetry(() => generateBookReviews("ミステリー"));
console.log(reviews);

// 期待出力（リトライ発生時）:
// Retry 1/3 in 1050ms
// 3 reviews generated at ...
```

## FAQ

### Q1. responseSchema と responseMimeType だけの違いは何ですか？

A1. `responseMimeType: "application/json"` だけだと JSON 形式で返すことは保証されますが、フィールド名や型は保証されません。`responseSchema` を追加することで、指定したスキーマに完全に準拠したレスポンスが返ります。

### Q2. enum で定義した値以外がレスポンスに含まれることはありますか？

A2. 基本的にはありません。Gemini は enum で定義した値のいずれかを選択します。ただし、安全のためアプリケーション側でも Zod などによるバリデーションを追加することを推奨します。

### Q3. ネストが深い複雑なスキーマにも対応していますか？

A3. はい、配列の中にオブジェクト、オブジェクトの中に配列といったネスト構造に対応しています。ただし、スキーマが複雑すぎるとトークン消費が増え、レスポンス速度が低下するため、必要最小限の構造にすることを推奨します。

### Q4. 構造化出力はストリーミングでも使えますか？

A4. `responseMimeType: "application/json"` を設定してもストリーミングは可能ですが、JSON の完全性は最後のチャンクが到着するまで保証されません。ストリーミングで部分的な結果を使いたい場合は、各チャンクの累積結果を `JSON.parse` でバリデーションする必要があります。

### Q5. Function Calling と構造化出力の使い分けは？

A5. 外部ツールの呼び出しが必要な場合は Function Calling、モデルの回答自体を構造化したい場合は構造化出力を使います。レストラン検索APIを呼ぶなら Function Calling、レビュー分析の結果をJSON で返すなら構造化出力が適切です。
