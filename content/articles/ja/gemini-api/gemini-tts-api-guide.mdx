## はじめに：Gemini 2.5 TTSとは

Googleは2025年12月、Gemini APIに搭載されたテキスト読み上げ（TTS）機能を大幅にアップグレードしました。そして2026年3月のアップデートでは**表現力の強化、精密なペーシング制御、シームレスな対話生成**が追加され、AIによる音声合成は新たなステージに突入しています。

Gemini 2.5 TTSは以下の2モデルが提供されています。

- **`gemini-2.5-flash-preview-tts`** — 低遅延重視。リアルタイムアシスタントや大量処理に最適
- **`gemini-2.5-pro-preview-tts`** — 品質重視。ポッドキャスト・ナレーション・感情表現が豊かな音声制作に最適

この記事では、これらのモデルをPython SDKで活用する方法を、実践的なコード例とともに解説します。

## 基本的な使い方：最初のTTS生成

最もシンプルなTTS生成の例です。テキストを渡すと音声バイナリが返ってきます。

```python
import os
import wave
from google import genai
from google.genai import types

# クライアント初期化
client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# テキストを音声に変換
response = client.models.generate_content(
    model="gemini-2.5-flash-preview-tts",
    contents="こんにちは！Gemini TTSのテストです。",
    config=types.GenerateContentConfig(
        response_modalities=["AUDIO"],
        speech_config=types.SpeechConfig(
            voice_config=types.VoiceConfig(
                prebuilt_voice_config=types.PrebuiltVoiceConfig(
                    voice_name="Kore"  # 日本語に対応した音声
                )
            )
        )
    )
)

# 音声データを取得して保存
audio_data = response.candidates[0].content.parts[0].inline_data.data

# WAVファイルとして保存
with wave.open("output.wav", "wb") as wav_file:
    wav_file.setnchannels(1)   # モノラル
    wav_file.setsampwidth(2)   # 16-bit PCM
    wav_file.setframerate(24000)  # 24kHz
    wav_file.writeframes(audio_data)

print("✅ output.wav に保存しました")
# 期待する出力: ✅ output.wav に保存しました
```

## スタイルプロンプトで音声表現を制御する

Gemini TTSの最大の強みは、**自然言語のスタイルプロンプト**で音声の雰囲気・トーン・ペースを細かく制御できる点です。プロンプトは3つの要素で構成します。

1. **Audio Profile（音声プロフィール）** — キャラクターの基本設定・アーキタイプ
2. **Scene（シーン）** — 物理的な環境・感情的な雰囲気
3. **Director's Notes（演出メモ）** — スタイル・アクセント・ペース

```python
import os
import wave
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# スタイルプロンプトを組み込んだTTS生成
style_prompt = """
Audio Profile: プロフェッショナルなポッドキャストホスト。落ち着きがあり、知性的な印象を与える声。
Scene: スタジオ収録環境。静かで集中した雰囲気。
Director's Notes: テンポはゆっくりと明瞭に。重要なキーワードは少し強調してください。
"""

text_to_speak = "今日は、Gemini APIの最新機能についてお話しします。特に注目すべきは、テキスト読み上げ機能の大幅な進化です。"

response = client.models.generate_content(
    model="gemini-2.5-pro-preview-tts",  # 品質重視モデルを使用
    contents=f"{style_prompt}\n\n{text_to_speak}",
    config=types.GenerateContentConfig(
        response_modalities=["AUDIO"],
        speech_config=types.SpeechConfig(
            voice_config=types.VoiceConfig(
                prebuilt_voice_config=types.PrebuiltVoiceConfig(
                    voice_name="Aoede"
                )
            )
        )
    )
)

audio_data = response.candidates[0].content.parts[0].inline_data.data

with wave.open("podcast_style.wav", "wb") as wav_file:
    wav_file.setnchannels(1)
    wav_file.setsampwidth(2)
    wav_file.setframerate(24000)
    wav_file.writeframes(audio_data)

print("✅ podcast_style.wav に保存しました")
# 期待する出力: ✅ podcast_style.wav に保存しました
```

### スタイル例：感情・トーンのバリエーション

| スタイル指示 | 用途 |
|-------------|------|
| `明るく元気に、テンポ速め` | 広告・紹介動画 |
| `落ち着いた低音、権威ある口調` | ニュース読み上げ |
| `温かく親しみやすく` | サポートボット・アシスタント |

| `ゆっくりと、間を大切に` | 瞑想・リラクゼーション音声 |

## マルチスピーカー対応：対話音声の生成

Gemini 2.5 TTSはマルチスピーカー（複数話者）の対話音声生成にも対応しています。インタビューや会話形式のコンテンツを一度の APIコールで生成できます。

```python
import os
import wave
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# マルチスピーカー対話音声の生成
response = client.models.generate_content(
    model="gemini-2.5-pro-preview-tts",
    contents="""
    TTS_SPEAKERS:
    - Speaker1: Aoede (女性、明るいトーン)
    - Speaker2: Charon (男性、落ち着いたトーン)

    DIALOGUE:
    [Speaker1]: Gemini APIの新しいTTS機能、使ってみましたか？
    [Speaker2]: はい、表現力が格段に上がっていて驚きました。
    [Speaker1]: 特にペーシング制御が素晴らしいですよね。感情に合わせて自動調整してくれます。
    [Speaker2]: 開発者としては、スタイルプロンプトが使いやすくなったのが一番嬉しいです。
    """,
    config=types.GenerateContentConfig(
        response_modalities=["AUDIO"],
        speech_config=types.SpeechConfig(
            multi_speaker_voice_config=types.MultiSpeakerVoiceConfig(
                speaker_voice_configs=[
                    types.SpeakerVoiceConfig(
                        speaker="Speaker1",
                        voice_config=types.VoiceConfig(
                            prebuilt_voice_config=types.PrebuiltVoiceConfig(
                                voice_name="Aoede"
                            )
                        )
                    ),
                    types.SpeakerVoiceConfig(
                        speaker="Speaker2",
                        voice_config=types.VoiceConfig(
                            prebuilt_voice_config=types.PrebuiltVoiceConfig(
                                voice_name="Charon"
                            )
                        )
                    )
                ]
            )
        )
    )
)

audio_data = response.candidates[0].content.parts[0].inline_data.data

with wave.open("dialogue.wav", "wb") as wav_file:
    wav_file.setnchannels(1)
    wav_file.setsampwidth(2)
    wav_file.setframerate(24000)
    wav_file.writeframes(audio_data)

print("✅ dialogue.wav にマルチスピーカー音声を保存しました")
# 期待する出力: ✅ dialogue.wav にマルチスピーカー音声を保存しました
```

## ストリーミング対応：リアルタイム音声生成

リアルタイムアシスタントやライブ応答が必要なアプリには、ストリーミングモードが最適です。

```python
import os
import io
import wave
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# ストリーミングで音声を受信
audio_chunks = []

for chunk in client.models.generate_content_stream(
    model="gemini-2.5-flash-preview-tts",  # 低遅延モデルを使用
    contents="ストリーミングTTSのテストです。リアルタイムで音声が生成されます。",
    config=types.GenerateContentConfig(
        response_modalities=["AUDIO"],
        speech_config=types.SpeechConfig(
            voice_config=types.VoiceConfig(
                prebuilt_voice_config=types.PrebuiltVoiceConfig(
                    voice_name="Kore"
                )
            )
        )
    )
):
    if (chunk.candidates and
        chunk.candidates[0].content.parts and
        chunk.candidates[0].content.parts[0].inline_data):
        audio_chunks.append(
            chunk.candidates[0].content.parts[0].inline_data.data
        )
        print(".", end="", flush=True)  # 進行状況を表示

print("\n✅ ストリーミング完了")

# 全チャンクを結合して保存
full_audio = b"".join(audio_chunks)
with wave.open("streaming_output.wav", "wb") as wav_file:
    wav_file.setnchannels(1)
    wav_file.setsampwidth(2)
    wav_file.setframerate(24000)
    wav_file.writeframes(full_audio)

print("✅ streaming_output.wav に保存しました")
# 期待する出力:
# ......
# ✅ ストリーミング完了
# ✅ streaming_output.wav に保存しました
```

## よくあるエラーと対処法

### エラー1：`INVALID_ARGUMENT: voice_name not supported`

```
google.api_core.exceptions.InvalidArgument: 400 INVALID_ARGUMENT: voice_name not supported
```

**原因**: 指定した音声名が対象モデルでサポートされていない。
**対処**: [公式ドキュメント](https://ai.google.dev/gemini-api/docs/speech-generation)で現在使用可能な音声一覧を確認してください。

---

### エラー2：`response_modalities` の設定漏れ

```
AttributeError: 'NoneType' object has no attribute 'data'
```

**原因**: `response_modalities=["AUDIO"]` を忘れると音声データが返されない。
**対処**: `GenerateContentConfig` に必ず `response_modalities=["AUDIO"]` を指定してください。

---

### エラー3：モデル名の混在

**原因**: `gemini-2.5-flash`（通常モデル）と `gemini-2.5-flash-preview-tts`（TTS専用モデル）を混同している。
**対処**: TTS生成には必ず `-tts` サフィックスのついたモデルを使用してください。

## Flash vs Pro：どちらを選ぶべきか

| 比較項目 | Flash TTS | Pro TTS |
|---------|-----------|---------|
| **レイテンシ** | 低い（リアルタイム向き） | 高め |
| **音声品質** | 高品質 | 最高品質 |
| **表現力** | 豊か | 非常に豊か |
| **推奨ユースケース** | チャットボット、リアルタイム応答 | ポッドキャスト、プレミアムコンテンツ |
| **コスト** | 低め | 高め |

**結論**: プロトタイプやリアルタイム系には Flash、最終的な高品質コンテンツには Pro を選びましょう。

## FAQ

**Q1: Gemini TTSは日本語に対応していますか？**

A: はい、対応しています。音声モデルによって日本語の品質に差があるため、`Kore` や `Aoede` など、日本語サポートが確認されている音声を選択することを推奨します。また、スタイルプロンプトに日本語で指示を書くことで、自然な日本語音声を生成できます。

**Q2: 生成できる音声の長さに制限はありますか？**

A: Gemini TTS APIには入力テキストのトークン上限があります（モデルによって異なりますが通常数千トークン）。長文のナレーションが必要な場合は、テキストを適切なセクションに分割して複数回APIを呼び出し、音声ファイルを結合する方法を取りましょう。

**Q3: 生成された音声ファイルは何形式で出力されますか？**

A: Gemini TTSは **PCM（生音声データ）** として返します。サンプルレートは24kHz、16-bit、モノラルが標準です。上記のコード例のように、Pythonの `wave` ライブラリを使ってWAVファイルとして保存するか、`pydub` 等でMP3に変換することができます。
