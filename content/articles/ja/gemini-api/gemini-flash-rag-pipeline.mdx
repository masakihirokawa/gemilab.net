## はじめに

大規模言語モデルを活用したアプリケーション開発において、「検索拡張生成（RAG: Retrieval-Augmented Generation）」は必要不可欠なテクニックです。しかし、従来のRAGパイプラインには課題がありました：検索結果の品質に依存、複数回のAPI呼び出しによるレイテンシ増加、そしてコンテキストサイズの制限です。

**Gemini 3 Flash** の登場により、これらの課題は大きく改善されました。100万トークン（約75万単語）のコンテキストウィンドウを備えたこのモデルは、従来の手法を根本的に変えます。

本記事では、Gemini 3 Flash の圧倒的なコンテキストウィンドウを活用して、シンプルかつ高速なRAGパイプラインを構築する方法を、実践的なコード例とともに解説します。

## なぜ100万トークンのコンテキストが革新的なのか

従来のRAGアプローチでは、以下のようなフローが一般的でした：

1. ユーザーの質問を受け取る
2. ベクトルデータベースで関連ドキュメントを検索
3. 検索結果をプロンプトに組み込む
4. LLMに送信して回答を生成

このアプローチには複数のボトルネックがあります：
- **検索精度への依存** ：最適なドキュメントが上位に表示されなければ、生成品質が低下
- **レイテンシ** ：複数のAPI呼び出しと検索処理が必要
- **文脈ロス** ：数千トークンに限定されるため、重要な背景情報を失うことがある

Gemini 3 Flash の100万トークンコンテキストなら、**大規模なドキュメント群をすべてモデルに渡すことが可能**です。これにより：

- ✅ 検索精度に依存しない包括的な理解
- ✅ 単一のAPI呼び出しで完結
- ✅ ドキュメント間の関連性を直接学習
- ✅ 複雑なクエリに対する正確な回答

### ステップ2：ドキュメント処理と分割

大規模なドキュメント群を効率的に処理するには、適切な分割戦略が重要です。

```python
def chunk_documents(documents: list[str], max_chunk_size: int = 8000) -> list[dict]:
    """
    ドキュメントをチャンクに分割

    Args:
        documents: 処理対象のドキュメント一覧
        max_chunk_size: 1チャンクの最大トークン数（概算）

    Returns:
        チャンク化されたドキュメント
    """
    chunks = []

    for doc_idx, doc in enumerate(documents):
        # 簡易的なトークン数推定：1単語≈1.3トークン
        estimated_tokens = len(doc.split()) * 1.3

        if estimated_tokens <= max_chunk_size:
            chunks.append({
                "content": doc,
                "source": f"doc_{doc_idx}",
                "chunk_index": 0
            })
        else:
            # 大規模ドキュメントは段落ごとに分割
            paragraphs = doc.split("\n\n")
            current_chunk = ""
            chunk_count = 0

            for paragraph in paragraphs:
                if len((current_chunk + paragraph).split()) * 1.3 > max_chunk_size:
                    if current_chunk:
                        chunks.append({
                            "content": current_chunk.strip(),
                            "source": f"doc_{doc_idx}",
                            "chunk_index": chunk_count
                        })
                        chunk_count += 1
                    current_chunk = paragraph
                else:
                    current_chunk += "\n\n" + paragraph

            if current_chunk:
                chunks.append({
                    "content": current_chunk.strip(),
                    "source": f"doc_{doc_idx}",
                    "chunk_index": chunk_count
                })

    return chunks
```

### ステップ3：コンテキストウィンドウの最適化

100万トークンを活用する際の核となるステップです。

```python
def build_optimized_context(
    chunks: list[dict],
    max_context_tokens: int = 900000
) -> tuple[str, list[dict]]:
    """
    チャンクからコンテキストを構築（900Kトークンまで利用可能）

    Args:
        chunks: チャンク化されたドキュメント
        max_context_tokens: 利用可能な最大トークン数

    Returns:
        （構築されたコンテキスト、含まれたメタデータ）
    """
    context_parts = []
    used_chunks = []
    estimated_tokens = 0

    # チャンクを優先度順に追加
    for chunk in chunks:
        chunk_tokens = len(chunk["content"].split()) * 1.3

        if estimated_tokens + chunk_tokens <= max_context_tokens:
            source_info = f"[{chunk['source']} - Chunk {chunk['chunk_index']}]"
            context_parts.append(f"{source_info}\n{chunk['content']}\n")
            estimated_tokens += chunk_tokens
            used_chunks.append(chunk)
        else:
            break

    # コンテキストをフォーマット
    formatted_context = f"""
# リファレンスドキュメント

以下は、質問に答えるために提供されるドキュメント群です。
これらのドキュメントはすべて同じコンテキストウィンドウに含まれています。

---

{"".join(context_parts)}

---
"""

    return formatted_context, used_chunks
```

### ステップ4：RAGクエリの実行

```python
def execute_rag_query(
    model,
    chunks: list[dict],
    query: str,
    include_reasoning: bool = True
) -> dict:
    """
    RAGクエリを実行

    Args:
        model: 初期化されたGenerativeModel
        chunks: チャンク化されたドキュメント
        query: ユーザーの質問
        include_reasoning: 推論プロセスを含めるか

    Returns:
        回答とメタデータ
    """

    # コンテキストを構築
    context, used_chunks = build_optimized_context(chunks)

    # プロンプトを構築
    system_prompt = """
あなたは専門的で正確な回答を提供するAIアシスタントです。
提供されたドキュメントに基づいて、ユーザーの質問に詳細かつ正確に答えてください。
ドキュメントに情報がない場合は、その旨を明確に述べてください。
"""

    if include_reasoning:
        system_prompt += "\n回答の前に、短い思考プロセスを共有してください。"

    full_prompt = f"""{system_prompt}

{context}

## ユーザーの質問

{query}

## 回答

"""

    # リクエストを計測
    start_time = time.time()

    try:
        response = model.generate_content(full_prompt)
        elapsed_time = time.time() - start_time

        return {
            "answer": response.text,
            "status": "success",
            "elapsed_time": elapsed_time,
            "chunks_used": len(used_chunks),
            "metadata": {
                "source_documents": list(set(c["source"] for c in used_chunks))
            }
        }

    except Exception as e:
        return {
            "answer": None,
            "status": "error",
            "error": str(e),
            "elapsed_time": time.time() - start_time
        }
```

### ステップ5：完全なパイプラインの統合

```python
class GeminiRAGPipeline:
    """Gemini 3 Flashを使用したRAGパイプライン"""

    def __init__(self, api_key: Optional[str] = None):
        if api_key:
            genai.configure(api_key=api_key)
        self.model = initialize_model()
        self.chunks = []

    def load_documents(self, documents: list[str]):
        """ドキュメントをロードして処理"""
        self.chunks = chunk_documents(documents)
        print(f"✓ {len(self.chunks)}個のチャンクにドキュメントを分割")

    def query(self, question: str) -> dict:
        """質問を実行"""
        if not self.chunks:
            return {
                "answer": None,
                "status": "error",
                "error": "ドキュメントがロードされていません"
            }

        return execute_rag_query(self.model, self.chunks, question)

    def batch_query(self, questions: list[str]) -> list[dict]:
        """複数の質問を実行"""
        results = []
        for i, question in enumerate(questions, 1):
            print(f"質問 {i}/{len(questions)} を処理中...")
            result = self.query(question)
            results.append({
                "question": question,
                "result": result
            })
        return results

# 使用例
pipeline = GeminiRAGPipeline()

documents = [
    "Gemini 3 Flashは、Googleが開発した最新世代の言語モデルです。...",
    "RAGパイプラインは、外部ナレッジベースとLLMを組み合わせた技術です...",
    # 大規模なドキュメント群...
]

pipeline.load_documents(documents)

# 単一クエリ
result = pipeline.query("Gemini 3 Flashの主な特徴は何ですか？")
print(f"回答: {result['answer']}")
print(f"処理時間: {result['elapsed_time']:.2f}秒")
```

```python
def estimate_tokens(text: str) -> int:
    """テキストのトークン数を推定"""
    # 平均的な日本語テキストの場合
    # 1文字≈0.5トークン、1単語≈1.3トークン
    if any(ord(c) > 0x3000 for c in text):  # 日本語判定
        return len(text) // 2
    return int(len(text.split()) * 1.3)
```

### コンテキストオーバーフロー

100万トークンを超える場合は、チャンク選択を改善してください：

```python
def smart_chunk_selection(
    chunks: list[dict],
    query: str,
    max_tokens: int = 900000
) -> list[dict]:
    """クエリに基づいて最適なチャンクを選択"""
    # スコアリングロジックを実装
    scored_chunks = []
    for chunk in chunks:
        # キーワードマッチングスコア
        score = sum(1 for word in query.split()
                   if word.lower() in chunk["content"].lower())
        scored_chunks.append((score, chunk))

    # スコア降順でソート
    scored_chunks.sort(reverse=True, key=lambda x: x[0])

    # トークンが閾値以内のチャンクを選択
    selected = []
    total_tokens = 0
    for score, chunk in scored_chunks:
        chunk_tokens = estimate_tokens(chunk["content"])
        if total_tokens + chunk_tokens <= max_tokens:
            selected.append(chunk)
            total_tokens += chunk_tokens

    return selected
```

```python
import asyncio

async def async_query(pipeline, questions: list[str]):
    """非同期でクエリを実行"""
    tasks = [
        asyncio.to_thread(pipeline.query, question)
        for question in questions
    ]
    return await asyncio.gather(*tasks)
```
