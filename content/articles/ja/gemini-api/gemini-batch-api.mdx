## Gemini Batch APIとは

Gemini Batch APIは、大量のテキスト処理リクエストを効率的かつ低コストで処理するためのAPI です。リアルタイム処理が不要な場合、バッチ処理を使用することで、API呼び出し費用を通常の50%まで削減できます。

企業向けのデータ分析、コンテンツ生成、機械学習のパイプラインなど、大規模な処理が必要なユースケースで特に活躍します。本ガイドでは、Batch APIの基本から実装方法までを詳しく解説します。

## Batch APIの基本概念

### バッチ処理のメリット

- **コスト削減**: 通常料金の約50%で利用可能
- **スケーラビリティ**: 数千〜数百万リクエストまで対応
- **スケジューリング**: 処理時間を柔軟に設定可能
- **統計情報**: リクエストごとの処理結果を追跡可能

### リアルタイムAPIとの違い

| 特性 | Batch API | リアルタイムAPI |
|------|----------|------------|
| レスポンス時間 | 数分〜数時間 | 数秒以内 |
| 費用 | 50%割引 | 標準料金 |
| 用途 | 大量バッチ処理 | リアルタイム応答 |
| スケーラビリティ | 非常に高い | 高い |

## Batch APIの実装

### リクエストの準備

Batch APIを使用する場合、リクエストはJSONL形式（JSON Lines）で準備します。各行が1つのリクエストオブジェクトです。

```python
import json
from typing import List

def create_batch_requests(texts: List[str]) -> str:
    """テキストのリストからBatch APIリクエストを生成"""
    requests = []
    for i, text in enumerate(texts):
        request = {
            "custom_id": f"request_{i}",
            "params": {
                "model": "gemini-2.0-flash",
                "contents": [
                    {
                        "role": "user",
                        "parts": [
                            {
                                "text": f"次のテキストを要約してください:\n\n{text}"
                            }
                        ]
                    }
                ]
            }
        }
        requests.append(json.dumps(request))

    return "\n".join(requests)

# 使用例
texts = [
    "人工知能は現代のテクノロジー業界を大きく変えています...",
    "機械学習モデルのトレーニングには膨大なデータが必要です...",
    "クラウドコンピューティングはビジネスのスケーリングを加速させます..."
]

batch_content = create_batch_requests(texts)
print("生成されたBatchリクエスト:")
print(batch_content[:200] + "...")

# 出力例:
# 生成されたBatchリクエスト:
# {"custom_id": "request_0", "params": {"model": "gemini-2.0-flash", ...}}
# {"custom_id": "request_1", "params": {"model": "gemini-2.0-flash", ...}}
```

### バッチの送信

準備したリクエストをBatch APIエンドポイントに送信します。

```python
import google.generativeai as genai

def submit_batch(batch_file_path: str) -> str:
    """Batch APIにリクエストを送信"""
    with open(batch_file_path, 'r') as f:
        batch_content = f.read()

    # Batch APIに送信
    response = genai.batch.create(
        model="gemini-2.0-flash",
        requests=batch_content
    )

    return response.name

# リクエストをファイルに保存
with open("batch_requests.jsonl", "w") as f:
    f.write(batch_content)

# バッチを送信
batch_name = submit_batch("batch_requests.jsonl")
print(f"バッチが送信されました: {batch_name}")

# 出力例:
# バッチが送信されました: projects/my-project/batches/batch_12345
```

### 処理結果の取得

バッチの処理状況を確認し、完了後に結果を取得します。

```python
import time

def wait_for_batch_completion(batch_name: str, max_wait_seconds: int = 3600) -> dict:
    """バッチの完了を待機して結果を取得"""
    start_time = time.time()

    while time.time() - start_time < max_wait_seconds:
        batch = genai.batch.get(batch_name)

        if batch.state == "COMPLETED":
            print(f"バッチが完了しました: {batch_name}")
            return batch
        elif batch.state == "FAILED":
            raise Exception(f"バッチが失敗しました: {batch.error}")

        print(f"状態: {batch.state} - 処理済み: {batch.processed_count}")
        time.sleep(30)  # 30秒待機

    raise TimeoutError("バッチの完了がタイムアウトしました")

# バッチの完了を待機
completed_batch = wait_for_batch_completion(batch_name)
print(f"処理済みリクエスト: {completed_batch.processed_count}")
print(f"失敗したリクエスト: {completed_batch.failed_count}")

# 出力例:
# 状態: PROCESSING - 処理済み: 50
# 状態: PROCESSING - 処理済み: 100
# バッチが完了しました: projects/my-project/batches/batch_12345
# 処理済みリクエスト: 150
# 失敗したリクエスト: 0
```

## コスト最適化のベストプラクティス

### バッチサイズの最適化

- **最小バッチ**: 最低10リクエストからの送信推奨
- **最適バッチ**: 数百〜数千リクエストで効率最大化
- **大規模バッチ**: 10万以上のリクエストも処理可能

### エラーハンドリング

```python
def process_batch_results(batch_name: str) -> dict:
    """バッチ結果の処理とエラーハンドリング"""
    batch = genai.batch.get(batch_name)

    results = {
        "successful": [],
        "failed": []
    }

    for response in batch.responses:
        if response.error:
            results["failed"].append({
                "custom_id": response.custom_id,
                "error": response.error.message
            })
        else:
            results["successful"].append({
                "custom_id": response.custom_id,
                "content": response.result.candidates[0].content
            })

    return results

# 結果の処理
batch_results = process_batch_results(batch_name)
print(f"成功: {len(batch_results['successful'])}")
print(f"失敗: {len(batch_results['failed'])}")
```

## よくある質問

### Q1: Batch APIとStream APIの違いは何ですか？

**A:** Batch APIは複数のリクエストを非同期で一括処理し、コストが50%割引される点が特徴です。Stream APIはリアルタイムでレスポンスをストリーミング受信します。大量処理はBatch API、リアルタイム性が必要な場合はStream APIを選択してください。

### Q2: バッチの処理にはどのくらい時間がかかりますか？

**A:** 通常、数分〜数時間で完了します。バッチサイズとGeminiの負荷によって異なります。最大24時間以内に完了することが保証されています。

### Q3: 失敗したリクエストを再試行できますか？

**A:** はい。失敗したリクエストのカスタムIDを確認し、新しいバッチで再送信できます。エラーメッセージから原因を特定して修正してください。
