## はじめに

LINEは日本国内で月間アクティブユーザー数9,700万人以上を誇るメッセージングプラットフォームです。このLINE上にGemini APIを活用したAIチャットボットを構築できれば、顧客対応の自動化、社内ヘルプデスク、パーソナルアシスタントなど、多彩なユースケースに対応できます。

この記事では、Google の Gemini API と LINE Messaging API を Python（Flask）で組み合わせ、メッセージの受信からAI応答の返信までを一気通貫で実装する方法を解説します。

**この記事で学べること:**

- LINE Developers でチャネルを作成し Messaging API を有効化する手順
- Gemini API（Google AI Python SDK）のセットアップ
- Flask で Webhook サーバーを構築し、ユーザーメッセージに Gemini が応答する仕組み
- 会話履歴を保持するマルチターン対応
- Cloud Run へのデプロイ方法

Gemini API の基本的な使い方については [Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart) も合わせてご覧ください。

## 前提知識・環境準備

### 必要なもの

- **Python 3.11 以上**
- **Google AI Studio の API キー**（[Google AI Studio](https://aistudio.google.com/) で無料取得）
- **LINE Developers アカウント**（[LINE Developers](https://developers.line.biz/) で無料登録）
- **ngrok**（ローカル開発時の HTTPS トンネル用）または **Google Cloud Run**（本番デプロイ用）

### LINE チャネルの作成

1. [LINE Developers Console](https://developers.line.biz/console/) にログイン
2. 「プロバイダー」を作成（例: `My AI Bot`）
3. 「Messaging API」チャネルを作成
4. チャネル設定画面で以下を取得:
   - **チャネルシークレット**（Basic settings タブ）
   - **チャネルアクセストークン**（Messaging API タブ → 「発行」ボタン）
5. Webhook URL は後のステップで設定します

### Python パッケージのインストール

```bash
# プロジェクトディレクトリを作成
mkdir gemini-line-bot && cd gemini-line-bot

# 仮想環境を作成・有効化
python -m venv venv

# 必要パッケージをインストール
pip install google-genai flask line-bot-sdk gunicorn
```

まず、Gemini API クライアントを初期化し、テキスト生成が正しく動作することを確認しましょう:

```python
# test_gemini.py
from google import genai

client = genai.Client(api_key="YOUR_GEMINI_API_KEY")

response = client.models.generate_content(
    model="gemini-3-flash",
    contents="こんにちは！自己紹介してください。"

print(response.text)
```

## LINE Bot サーバーの構築

### メインアプリケーション

Flask で Webhook サーバーを構築し、LINE からのメッセージを受信して Gemini API で応答を生成します:

```python
# app.py
import os
import hashlib
import hmac
import base64
from flask import Flask, request, abort
from google import genai
from linebot.v3 import WebhookParser
from linebot.v3.messaging import (
    Configuration,
    ApiClient,
    MessagingApi,
    ReplyMessageRequest,
    TextMessage,
)
from linebot.v3.webhooks import MessageEvent, TextMessageContent

app = Flask(__name__)

# --- 設定 ---
GEMINI_API_KEY = os.environ["GEMINI_API_KEY"]
LINE_CHANNEL_SECRET = os.environ["LINE_CHANNEL_SECRET"]
LINE_CHANNEL_ACCESS_TOKEN = os.environ["LINE_CHANNEL_ACCESS_TOKEN"]

# Gemini クライアント初期化
gemini_client = genai.Client(api_key=GEMINI_API_KEY)

# LINE API 初期化
configuration = Configuration(access_token=LINE_CHANNEL_ACCESS_TOKEN)
parser = WebhookParser(LINE_CHANNEL_SECRET)

# ユーザーごとの会話履歴（メモリ内保持）
conversation_history: dict[str, list[dict]] = {}
MAX_HISTORY = 20  # 直近20メッセージまで保持


def get_gemini_response(user_id: str, user_message: str) -> str:
    """Gemini APIでマルチターン応答を生成する"""
    # 会話履歴を取得（なければ初期化）
    if user_id not in conversation_history:
        conversation_history[user_id] = []

    history = conversation_history[user_id]

    # ユーザーメッセージを履歴に追加
    history.append({"role": "user", "parts": [{"text": user_message}]})

    # Gemini APIを呼び出し
    response = gemini_client.models.generate_content(
        model="gemini-3-flash",
        contents=history,
        config={
            "system_instruction": (
                "あなたはLINE上で動作する親切なAIアシスタントです。"
                "日本語で簡潔に、フレンドリーな口調で回答してください。"
                "回答は300文字以内に収めてください。"
            ),
            "temperature": 0.7,
            "max_output_tokens": 500,
        },
    )

    assistant_message = response.text

    # アシスタント応答を履歴に追加
    history.append({"role": "model", "parts": [{"text": assistant_message}]})

    # 履歴が長くなりすぎたら古い部分を削除
    if len(history) > MAX_HISTORY:
        conversation_history[user_id] = history[-MAX_HISTORY:]

    return assistant_message


@app.route("/callback", methods=["POST"])
def callback():
    """LINE Webhookエンドポイント"""
    signature = request.headers.get("X-Line-Signature", "")
    body = request.get_data(as_text=True)

    try:
        events = parser.parse(body, signature)
    except Exception:
        abort(400)

    for event in events:
        if isinstance(event, MessageEvent) and isinstance(
            event.message, TextMessageContent
        ):
            user_id = event.source.user_id
            user_message = event.message.text

            # Gemini で応答を生成
            reply_text = get_gemini_response(user_id, user_message)

            # LINE に返信
            with ApiClient(configuration) as api_client:
                line_bot_api = MessagingApi(api_client)
                line_bot_api.reply_message(
                    ReplyMessageRequest(
                        reply_token=event.reply_token,
                        messages=[TextMessage(text=reply_text)],
                    )
                )

    return "OK"


@app.route("/health", methods=["GET"])
def health():
    """ヘルスチェック用エンドポイント"""
    return "OK"


if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8080, debug=True)
```

### コードのポイント

- **`conversation_history`**: ユーザーIDをキーとした辞書で、会話の文脈を保持します。これにより「さっきの続きで…」のような自然な会話が実現します
- **`system_instruction`**: Gemini に LINE ボットとしての振る舞いを指示。回答の長さやトーンをここで制御します
- **`MAX_HISTORY`**: メモリ効率のため直近20メッセージに制限。本番環境では Redis や Firestore での永続化を検討しましょう

## ローカルでの動作確認

### ngrok でトンネルを作成

LINE の Webhook は HTTPS が必須です。ローカル開発では ngrok を使います:

```bash
# ターミナル1: Flask サーバーを起動
export GEMINI_API_KEY="your_key"
export LINE_CHANNEL_SECRET="your_secret"
export LINE_CHANNEL_ACCESS_TOKEN="your_token"
python app.py

# ターミナル2: ngrok で HTTPS トンネルを開通
ngrok http 8080
# 出力例: https://xxxx-xxx-xxx.ngrok-free.app
```

### LINE Developers で Webhook を設定

1. LINE Developers Console → 自分のチャネル → Messaging API タブ
2. 「Webhook URL」に ngrok の URL + `/callback` を入力
   - 例: `https://xxxx-xxx-xxx.ngrok-free.app/callback`
3. 「Webhook の利用」を ON に設定
4. 「検証」ボタンで接続テスト → **成功** と表示されれば完了

LINE アプリでボットを友だち追加し、メッセージを送ってみましょう。Gemini が応答を返せば成功です。

## 画像メッセージへの対応（マルチモーダル）

Gemini の強みの一つはマルチモーダル対応です。画像を送信した場合にも AI が解析・応答できるようにしましょう:

```python
# app.py に追加
import requests
from linebot.v3.webhooks import ImageMessageContent


def download_line_image(message_id: str) -> bytes:
    """LINE サーバーから画像をダウンロード"""
    url = f"https://api-data.line.me/v2/bot/message/{message_id}/content"
    headers = {"Authorization": f"Bearer {LINE_CHANNEL_ACCESS_TOKEN}"}
    resp = requests.get(url, headers=headers)
    return resp.content


def get_gemini_image_response(user_id: str, image_bytes: bytes) -> str:
    """画像を Gemini に送信して解析結果を取得"""
    import base64

    image_b64 = base64.b64encode(image_bytes).decode("utf-8")

    contents = [
        {
            "role": "user",
            "parts": [
                {"text": "この画像について説明してください。"},
                {
                    "inline_data": {
                        "mime_type": "image/jpeg",
                        "data": image_b64,
                    }
                },
            ],
        }
    ]

    response = gemini_client.models.generate_content(
        model="gemini-3-flash",
        contents=contents,
        config={
            "system_instruction": (
                "画像の内容を日本語で簡潔に説明してください。"
                "300文字以内で回答してください。"
            ),
            "max_output_tokens": 500,
        },
    )
    return response.text
```

## Google Cloud Run へのデプロイ

本番運用では Cloud Run にデプロイします。サーバーレスで自動スケーリングが可能なため、LINE Bot のバックエンドに最適です。

### Dockerfile の作成

```dockerfile
FROM python:3.11-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY app.py .

CMD ["gunicorn", "--bind", "0.0.0.0:8080", "--workers", "2", "app:app"]
```

### requirements.txt

```text
google-genai>=1.0.0
flask>=3.0
line-bot-sdk>=3.0
gunicorn>=22.0
requests>=2.31
```

### デプロイコマンド

```bash
# Google Cloud プロジェクトを設定
gcloud config set project YOUR_PROJECT_ID

# Cloud Run にデプロイ（環境変数を設定）
gcloud run deploy gemini-line-bot \
  --source . \
  --region asia-northeast1 \
  --allow-unauthenticated \
  --set-env-vars "GEMINI_API_KEY=your_key,LINE_CHANNEL_SECRET=your_secret,LINE_CHANNEL_ACCESS_TOKEN=your_token" \
  --memory 512Mi \
  --min-instances 0 \
  --max-instances 10

# 出力例:
# Service URL: https://gemini-line-bot-xxxxx-an.a.run.app
```

デプロイ後、LINE Developers Console の Webhook URL を Cloud Run の URL（`https://gemini-line-bot-xxxxx-an.a.run.app/callback`）に更新すれば完了です。

## よくある質問（FAQ）

### Q1: LINE Bot の応答が遅いのですが、どうすれば改善できますか？

Gemini API の応答時間は通常1〜3秒です。体感速度を上げるには以下の方法があります:

- **`gemini-3-flash`** を使用する（Pro よりも高速）
- **`max_output_tokens`** を小さめに設定し、回答の長さを制限する
- Cloud Run の **最小インスタンス数を 1** に設定してコールドスタートを回避する
- LINE の **プッシュメッセージ**で「考え中...」と先に返し、非同期で本回答を送る

### Q2: 会話履歴を永続化するにはどうすればよいですか？

上記のサンプルではメモリ内に保持していますが、Cloud Run ではインスタンスが再起動すると失われます。本番環境では以下の選択肢があります:

- **Firebase Firestore**: サーバーレスで相性が良い。ユーザーIDをドキュメントIDにして会話を保存
- **Redis（Memorystore）**: 高速な読み書きが必要な場合に最適
- **Cloud SQL（PostgreSQL）**: 検索や分析も行いたい場合

### Q3: ボットに独自のキャラクター性を持たせるにはどうすればよいですか？

`system_instruction` を工夫することでボットの人格をカスタマイズできます:

```python
config={
    "system_instruction": (
        "あなたは「めぐみ」という名前の料理アシスタントです。"
        "関西弁で明るく返答してください。"
        "料理に関する質問に詳しく、レシピ提案が得意です。"
        "料理以外の話題でも気さくに対応してください。"
    ),
}
```

### Q4: Gemini のレート制限に引っかかった場合はどうすればよいですか？

Gemini API の無料枠には毎分リクエスト数の制限があります。制限に達した場合は:

- **指数バックオフ**でリトライする
- **API キーを有料プランにアップグレード**する（Google AI Studio で設定）
- **Context Caching** を活用してトークン消費を削減する

レート制限の詳細な対処法は [Gemini API レート制限・クォータ管理ガイド](/articles/gemini-api/gemini-api-rate-limiting-quota-management) で解説しています。

### Q5: セキュリティ面で気をつけることはありますか？

LINE Bot を本番運用する際のセキュリティ対策:

- **署名検証**: LINE Webhook の `X-Line-Signature` ヘッダーを必ず検証する（上記のサンプルでは `WebhookParser` が自動処理）
- **環境変数**: API キーやトークンをコードにハードコードしない。Cloud Run の環境変数または Secret Manager を使用する
- **入力バリデーション**: ユーザー入力が極端に長い場合はトリミングする
- **HTTPS 必須**: Cloud Run はデフォルトで HTTPS が有効
