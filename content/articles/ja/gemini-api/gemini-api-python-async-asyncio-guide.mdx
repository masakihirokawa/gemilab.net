## はじめに：なぜ非同期処理が重要なのか

Gemini API を使ったAIアプリケーションを本番運用すると、すぐに「同期処理の限界」にぶつかります。たとえばユーザーが一度に複数の質問を送信したとき、同期処理では1件ずつ順番に処理するため、後続のリクエストが待ち状態になります。

Python の `asyncio` を活用した非同期処理に切り替えると、次のメリットが得られます。

- **スループット向上**: 複数のAPI呼び出しを並列実行し、待機時間を有効活用できる
- **レスポンス改善**: ユーザーへの応答時間を大幅に短縮できる
- **リソース効率化**: CPUを無駄にブロックせず、I/O待機中に他の処理を進められる
- **スケーラビリティ**: 同じサーバーでより多くの同時接続を捌ける

この記事では Google AI Python SDK（`google-generativeai`）を使い、asyncio で Gemini API を扱う実践的な手法を段階的に解説します。基本的な非同期リクエストから、並列バッチ処理・ストリーミング・レート制限対応まで網羅します。

なお、Gemini API の基本的な使い方については [Gemini API クイックスタートガイド](/articles/gemini-api/gemini-api-quickstart) を先にご確認ください。

## 基本：同期処理から非同期処理へ

### 従来の同期処理（問題点）

```python
import google.generativeai as genai
import time

model = genai.GenerativeModel("gemini-2.5-flash")

def process_questions_sync(questions: list[str]) -> list[str]:
    """同期処理：1件ずつ順番に実行（遅い）"""
    results = []
    for q in questions:
        response = model.generate_content(q)
        results.append(response.text)
    return results

questions = ["Pythonとは？", "AIとは？", "asyncioとは？"]
start = time.time()
answers = process_questions_sync(questions)
print(f"同期処理 所要時間: {time.time() - start:.2f}秒")
# 例: 同期処理 所要時間: 6.84秒
```

3件のリクエストが直列実行されるため、処理時間がそのまま累積します。

### 非同期処理への移行

Google AI Python SDK は `generate_content_async()` メソッドを提供しています。これを `asyncio` と組み合わせます。

```python
import google.generativeai as genai
import asyncio
import time

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

async def process_single(question: str) -> str:
    """単一の非同期リクエスト"""
    response = await model.generate_content_async(question)
    return response.text

async def process_questions_async(questions: list[str]) -> list[str]:
    """非同期処理：全リクエストを並列実行（速い）"""
    tasks = [process_single(q) for q in questions]
    results = await asyncio.gather(*tasks)
    return list(results)

async def main():
    questions = ["Pythonとは？", "AIとは？", "asyncioとは？"]
    start = time.time()
    answers = await process_questions_async(questions)
    print(f"非同期処理 所要時間: {time.time() - start:.2f}秒")
    for q, a in zip(questions, answers):
        print(f"Q: {q}\nA: {a[:80]}...\n")

asyncio.run(main())
# 例: 非同期処理 所要時間: 2.31秒（約3倍高速化）
```

`asyncio.gather()` を使うことで、複数のコルーチンが並列実行され、API のI/O待機時間を重ね合わせることができます。

## 並列バッチ処理の実装

大量のテキストをまとめて処理するバッチ処理では、並列数を制限しながら効率よく処理することが重要です。無制限に並列化するとレート制限に引っかかります。

### セマフォで同時実行数を制限する

```python
import asyncio
import google.generativeai as genai
from typing import Optional

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

# 同時実行数を制限するセマフォ（Gemini API の無料枠: 15 RPM を考慮）
CONCURRENCY_LIMIT = 5
semaphore = asyncio.Semaphore(CONCURRENCY_LIMIT)

async def safe_generate(
    text: str,
    index: int,
    retry_count: int = 3
) -> tuple[int, Optional[str]]:
    """セマフォ付き安全な非同期リクエスト（リトライ対応）"""
    async with semaphore:
        for attempt in range(retry_count):
            try:
                response = await model.generate_content_async(text)
                return index, response.text
            except Exception as e:
                if attempt < retry_count - 1:
                    wait_time = 2 ** attempt  # 指数バックオフ
                    print(f"[{index}] エラー（{e}）: {wait_time}秒後にリトライ")
                    await asyncio.sleep(wait_time)
                else:
                    print(f"[{index}] リトライ上限到達: {e}")
                    return index, None

async def batch_process(texts: list[str]) -> list[Optional[str]]:
    """大量テキストの並列バッチ処理"""
    tasks = [safe_generate(text, i) for i, text in enumerate(texts)]
    results_with_index = await asyncio.gather(*tasks)

    # インデックス順に並べ直す
    sorted_results = sorted(results_with_index, key=lambda x: x[0])
    return [result for _, result in sorted_results]

async def main():
    # 20件のテキストを並列処理
    sample_texts = [
        f"以下の文章を要約してください：テキスト{i}の内容です。"
        for i in range(20)
    ]

    results = await batch_process(sample_texts)
    success_count = sum(1 for r in results if r is not None)
    print(f"処理完了: {success_count}/{len(sample_texts)} 件成功")

asyncio.run(main())
```

セマフォを使うことで、API のレート制限（RPM: Requests Per Minute）を守りながら最大限の並列処理が実現できます。詳しいレート制限対策は [Gemini API レート制限・クォータ管理ガイド](/articles/gemini-api/gemini-api-rate-limiting-quota-management) も参考にしてください。

## 非同期ストリーミングの実装

Gemini API はストリーミングレスポンスをサポートしています。長い回答をリアルタイムに表示したい場合、非同期ストリーミングが有効です。

### 基本的な非同期ストリーミング

```python
import asyncio
import google.generativeai as genai

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

async def stream_response(prompt: str) -> str:
    """非同期ストリーミングで逐次出力"""
    full_text = ""

    # stream=True でストリーミングを有効化
    async for chunk in await model.generate_content_async(
        prompt,
        stream=True
    ):
        if chunk.text:
            print(chunk.text, end="", flush=True)
            full_text += chunk.text

    print()  # 改行
    return full_text

async def main():
    prompt = "Pythonの非同期処理について、初心者にわかりやすく解説してください。"
    result = await stream_response(prompt)
    print(f"\n--- 合計 {len(result)} 文字を受信 ---")

asyncio.run(main())
```

### FastAPI との統合（SSE: Server-Sent Events）

Webアプリケーションでは、Server-Sent Events（SSE）を使いブラウザにリアルタイム配信できます。

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import google.generativeai as genai
import asyncio
import json

app = FastAPI()
genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

async def generate_sse_stream(prompt: str):
    """SSEフォーマットでGemini APIストリームを配信"""
    try:
        async for chunk in await model.generate_content_async(
            prompt,
            stream=True
        ):
            if chunk.text:
                # SSEフォーマット: "data: {...}\n\n"
                data = json.dumps({"text": chunk.text}, ensure_ascii=False)
                yield f"data: {data}\n\n"

        # 完了シグナル
        yield f"data: {json.dumps({'done': True})}\n\n"

    except Exception as e:
        error_data = json.dumps({"error": str(e)})
        yield f"data: {error_data}\n\n"

@app.get("/api/stream")
async def stream_endpoint(prompt: str):
    """ストリーミングエンドポイント"""
    return StreamingResponse(
        generate_sse_stream(prompt),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",  # Nginx バッファリング無効化
        }
    )
```

## マルチターン会話の非同期実装

チャットアプリでは、会話履歴を保持しながら非同期で応答を返す必要があります。

```python
import asyncio
import google.generativeai as genai
from dataclasses import dataclass, field
from typing import AsyncGenerator

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])

@dataclass
class AsyncChatSession:
    """非同期マルチターンチャットセッション"""
    model_name: str = "gemini-2.5-flash"
    system_instruction: str = "あなたは親切なAIアシスタントです。"
    history: list = field(default_factory=list)

    def __post_init__(self):
        self.model = genai.GenerativeModel(
            self.model_name,
            system_instruction=self.system_instruction
        )
        self.chat = self.model.start_chat(history=self.history)

    async def send_message(self, message: str) -> str:
        """非同期でメッセージを送信し、レスポンスを返す"""
        response = await self.chat.send_message_async(message)
        return response.text

    async def stream_message(self, message: str) -> AsyncGenerator[str, None]:
        """非同期ストリーミングでメッセージを送信"""
        async for chunk in await self.chat.send_message_async(
            message, stream=True
        ):
            if chunk.text:
                yield chunk.text

async def demo_chat():
    """マルチターン会話のデモ"""
    session = AsyncChatSession(
        system_instruction="あなたはPythonの専門家です。簡潔に答えてください。"
    )

    conversation = [
        "Pythonのリスト内包表記とは何ですか？",
        "それをasyncioと組み合わせることはできますか？",
        "具体的なコード例を見せてください。"
    ]

    for user_msg in conversation:
        print(f"\nユーザー: {user_msg}")
        print("Gemini: ", end="")

        # ストリーミングで逐次表示
        async for chunk in session.stream_message(user_msg):
            print(chunk, end="", flush=True)
        print()

asyncio.run(demo_chat())
```

## エラーハンドリングのベストプラクティス

非同期処理では、エラーが発生したときの適切な処理が同期処理以上に重要になります。

```python
import asyncio
import google.generativeai as genai
from google.api_core import exceptions as google_exceptions
import logging

logger = logging.getLogger(__name__)

class AsyncGeminiClient:
    """エラーハンドリング付き非同期Geminiクライアント"""

    def __init__(self, api_key: str, model_name: str = "gemini-2.5-flash"):
        genai.configure(api_key=api_key)
        self.model = genai.GenerativeModel(model_name)
        self.semaphore = asyncio.Semaphore(10)  # 最大10並列

    async def generate(
        self,
        prompt: str,
        timeout: float = 30.0,
        max_retries: int = 3
    ) -> str | None:
        """タイムアウト・リトライ付きの安全な生成"""
        async with self.semaphore:
            for attempt in range(max_retries):
                try:
                    # タイムアウト付きで実行
                    response = await asyncio.wait_for(
                        self.model.generate_content_async(prompt),
                        timeout=timeout
                    )
                    return response.text

                except asyncio.TimeoutError:
                    logger.warning(f"タイムアウト（{timeout}秒）: attempt {attempt+1}")
                    if attempt == max_retries - 1:
                        return None

                except google_exceptions.ResourceExhausted as e:
                    # レート制限エラー: 指数バックオフで待機
                    wait_time = 60 * (2 ** attempt)
                    logger.warning(f"レート制限: {wait_time}秒待機")
                    await asyncio.sleep(wait_time)

                except google_exceptions.InvalidArgument as e:
                    # 入力エラー: リトライ不可
                    logger.error(f"無効なリクエスト: {e}")
                    return None

                except Exception as e:
                    logger.error(f"予期しないエラー: {e}")
                    if attempt == max_retries - 1:
                        return None
                    await asyncio.sleep(2 ** attempt)

        return None

async def main():
    import os
    client = AsyncGeminiClient(api_key=os.environ["YOUR_GEMINI_API_KEY"])

    # 並列処理でも安全にエラー処理
    prompts = [
        "機械学習とは？",
        "ディープラーニングとは？",
        "強化学習とは？",
    ]

    tasks = [client.generate(p) for p in prompts]
    results = await asyncio.gather(*tasks, return_exceptions=False)

    for prompt, result in zip(prompts, results):
        status = "✅" if result else "❌"
        print(f"{status} {prompt[:20]}: {(result or 'エラー')[:50]}")

asyncio.run(main())
```

## パフォーマンス最適化のヒント

### asyncio.TaskGroup を活用する（Python 3.11+）

```python
import asyncio

async def process_with_taskgroup(prompts: list[str]) -> list[str]:
    """Python 3.11+ の TaskGroup で構造化された並列処理"""
    results = {}

    async with asyncio.TaskGroup() as tg:
        for i, prompt in enumerate(prompts):
            # タスクに名前をつけて管理
            task = tg.create_task(
                model.generate_content_async(prompt),
                name=f"task_{i}"
            )
            results[i] = task

    # TaskGroup 終了後、全タスク完了が保証される
    return [results[i].result().text for i in range(len(prompts))]
```

### キャッシュで重複リクエストを削減

```python
import asyncio
from functools import lru_cache
import hashlib

# 非同期対応のシンプルなインメモリキャッシュ
_cache: dict[str, str] = {}

async def cached_generate(prompt: str) -> str:
    """同一プロンプトへのキャッシュ対応リクエスト"""
    # プロンプトのハッシュをキャッシュキーに
    cache_key = hashlib.md5(prompt.encode()).hexdigest()

    if cache_key in _cache:
        print(f"キャッシュヒット: {prompt[:30]}")
        return _cache[cache_key]

    response = await model.generate_content_async(prompt)
    _cache[cache_key] = response.text
    return response.text
```

## FAQ

**Q1. `generate_content_async()` と `generate_content()` の違いは何ですか？**

`generate_content()` は同期（ブロッキング）関数で、呼び出したスレッドをレスポンスが返るまでブロックします。`generate_content_async()` はコルーチンを返す非同期関数で、`await` を使って呼び出すことでイベントループをブロックしません。FastAPI や async フレームワークを使っている場合は必ず非同期版を使用してください。

**Q2. 非同期処理でもレート制限（RPM・TPM）は守る必要がありますか？**

はい、非同期で並列実行しても、Gemini API のレート制限は同じく適用されます。むしろ並列化することでレート制限に達しやすくなるため、セマフォや指数バックオフによる制御が重要です。無料枠では 1 分あたり 15 リクエスト（RPM）、有料プランでは 1,000 RPM が上限目安です（モデルにより異なります）。

**Q3. `asyncio.run()` を複数回呼び出せますか？**

1つのプロセスで `asyncio.run()` を複数回呼び出すことは可能ですが、すでに実行中のイベントループ内から `asyncio.run()` を呼ぶとエラーになります。Jupyter Notebook など既にイベントループが走っている環境では `await coroutine` を直接使うか、`nest_asyncio` ライブラリを使ってください。

**Q4. 非同期ストリーミング中に接続が切れたらどうなりますか？**

非同期ジェネレータ内でネットワーク例外（`aiohttp.ClientConnectionError` など）が発生します。`try/except` で捕捉し、適切なエラーレスポンスを返すようにしてください。FastAPI の `StreamingResponse` では、クライアントが切断すると `asyncio.CancelledError` が発生します。
