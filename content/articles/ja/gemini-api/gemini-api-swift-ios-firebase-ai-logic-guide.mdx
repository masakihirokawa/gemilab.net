## はじめに — なぜ iOS アプリに Gemini を組み込むのか

モバイルアプリに AI 機能を搭載する需要は急速に高まっています。Gemini API は Google が提供する最新の生成 AI モデルであり、テキスト生成、画像認識、マルチモーダル入力など幅広い機能を備えています。

iOS アプリから Gemini API を利用するための公式推奨手段が **Firebase AI Logic SDK** です。以前は `google-generative-ai-swift` という独立したパッケージが提供されていましたが、現在は非推奨となり、Firebase SDK に統合されています。Firebase AI Logic SDK を使えば、サーバーを経由せずにアプリから直接 Gemini API を呼び出すことが可能です。

この記事では、Firebase AI Logic SDK のセットアップから、テキスト生成、画像認識、ストリーミングレスポンスの実装まで、SwiftUI を使った実践的なコード例とともに解説します。

**この記事の対象読者:**

- Swift / SwiftUI の基本的な開発経験がある方
- iOS アプリに AI 機能を組み込みたい個人開発者
- Gemini API をモバイルアプリで活用したい方

## 環境準備とセットアップ

### 必要な環境

Firebase AI Logic SDK を利用するには、以下の環境が必要です。

- **Xcode 16.0 以上**
- **iOS 16.0 以上** をターゲットとするプロジェクト
- **Firebase SDK v12.5.0 以上**
- **Firebase プロジェクト**（[Firebase Console](https://console.firebase.google.com/) で作成）
- **Gemini API キー**（Firebase Console の AI Logic セクションで有効化）

### Step 1: Firebase プロジェクトのセットアップ

まず Firebase Console でプロジェクトを作成し、iOS アプリを登録します。`GoogleService-Info.plist` をダウンロードして Xcode プロジェクトに追加してください。

### Step 2: Swift Package Manager で SDK を追加

Xcode でプロジェクトを開き、**File > Add Package Dependencies** から Firebase SDK を追加します。

```
https://github.com/firebase/firebase-ios-sdk
```

バージョンは **12.5.0 以上** を指定し、ライブラリ選択画面で **FirebaseAILogic** にチェックを入れます。

### Step 3: AppDelegate で Firebase を初期化

```swift
// AppDelegate.swift または App.swift
import SwiftUI
import FirebaseCore

@main
struct MyAIApp: App {
    init() {
        FirebaseApp.configure()
    }

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

### Step 4: Gemini モデルのインスタンスを取得

```swift
import FirebaseAILogic

// Gemini 2.5 Flash モデルを使用（軽量・高速）
let ai = FirebaseAI.firebaseAI(backend: .googleAI())
let model = ai.generativeModel(modelName: "gemini-2.5-flash")
```

`modelName` には利用したいモデルを指定します。コストと速度のバランスを考慮して、一般的な用途には `gemini-2.5-flash` が適しています。より高度な推論が必要な場合は `gemini-2.5-pro` を検討してください。

## テキスト生成の基本実装

最もシンプルなユースケースとして、プロンプトに対するテキスト生成を実装してみましょう。

```swift
import SwiftUI
import FirebaseAILogic

struct TextGenerationView: View {
    @State private var prompt = ""
    @State private var response = ""
    @State private var isLoading = false

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack(spacing: 16) {
            TextField("プロンプトを入力", text: $prompt)
                .textFieldStyle(.roundedBorder)
                .padding(.horizontal)

            Button("生成") {
                Task { await generateText() }
            }
            .disabled(prompt.isEmpty || isLoading)

            if isLoading {
                ProgressView()
            }

            ScrollView {
                Text(response)
                    .padding()
            }
        }
        .padding()
    }

    func generateText() async {
        isLoading = true
        defer { isLoading = false }

        do {
            let result = try await model.generateContent(prompt)
            if let text = result.text {
                response = text
            }
        } catch {
            response = "エラーが発生しました: \(error.localizedDescription)"
        }
    }
}

// 期待する動作:
// プロンプト「日本の四季の魅力を3行で説明して」を入力すると、
// Gemini が自然な日本語で四季の魅力を説明するテキストを返す
```

このコードでは `model.generateContent()` に文字列を渡すだけで、Gemini API からのレスポンスを取得できます。非同期処理は Swift の `async/await` で自然に記述できるため、コールバック地獄に陥ることはありません。

## ストリーミングレスポンスの実装

長いテキストを生成する場合、レスポンス全体を待つのではなく、生成されたチャンクを逐次表示するストリーミングが有効です。ユーザーに即座にフィードバックを返せるため、UX が大幅に向上します。

```swift
import SwiftUI
import FirebaseAILogic

struct StreamingView: View {
    @State private var prompt = ""
    @State private var streamedText = ""
    @State private var isStreaming = false

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack(spacing: 16) {
            TextField("プロンプトを入力", text: $prompt)
                .textFieldStyle(.roundedBorder)
                .padding(.horizontal)

            Button(isStreaming ? "生成中..." : "ストリーミング生成") {
                Task { await streamGenerate() }
            }
            .disabled(prompt.isEmpty || isStreaming)

            ScrollView {
                Text(streamedText)
                    .padding()
                    .frame(maxWidth: .infinity, alignment: .leading)
            }
        }
        .padding()
    }

    func streamGenerate() async {
        isStreaming = true
        streamedText = ""
        defer { isStreaming = false }

        do {
            let stream = try model.generateContentStream(prompt)
            for try await chunk in stream {
                if let text = chunk.text {
                    streamedText += text
                }
            }
        } catch {
            streamedText += "\nエラー: \(error.localizedDescription)"
        }
    }
}

// 期待する動作:
// プロンプト送信後、テキストがリアルタイムで1チャンクずつ画面に追加表示される
// ChatGPT や Gemini アプリのようなタイピングエフェクトに近い体験を実現
```

`generateContentStream()` は `AsyncThrowingStream` を返すため、`for try await` ループで各チャンクを受信できます。UI の更新は `@State` プロパティへの代入で自動的に反映されます。

## 画像認識（マルチモーダル入力）

Gemini の強力な機能の一つがマルチモーダル入力です。テキストと画像を同時に送信し、画像の内容を分析させることができます。

```swift
import SwiftUI
import PhotosUI
import FirebaseAILogic

struct ImageAnalysisView: View {
    @State private var selectedImage: UIImage?
    @State private var photoPickerItem: PhotosPickerItem?
    @State private var analysisResult = ""
    @State private var isAnalyzing = false

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack(spacing: 16) {
            PhotosPicker(
                selection: $photoPickerItem,
                matching: .images
            ) {
                if let image = selectedImage {
                    Image(uiImage: image)
                        .resizable()
                        .scaledToFit()
                        .frame(maxHeight: 200)
                } else {
                    Label("画像を選択", systemImage: "photo")
                }
            }
            .onChange(of: photoPickerItem) { _, newValue in
                Task {
                    if let data = try? await newValue?.loadTransferable(
                        type: Data.self
                    ) {
                        selectedImage = UIImage(data: data)
                    }
                }
            }

            Button("画像を分析") {
                Task { await analyzeImage() }
            }
            .disabled(selectedImage == nil || isAnalyzing)

            if isAnalyzing { ProgressView() }

            ScrollView {
                Text(analysisResult)
                    .padding()
            }
        }
        .padding()
    }

    func analyzeImage() async {
        guard let image = selectedImage else { return }
        isAnalyzing = true
        defer { isAnalyzing = false }

        do {
            let result = try await model.generateContent(
                "この画像の内容を詳しく説明してください。",
                image
            )
            if let text = result.text {
                analysisResult = text
            }
        } catch {
            analysisResult = "エラー: \(error.localizedDescription)"
        }
    }
}

// 期待する動作:
// ユーザーが写真ライブラリから画像を選択し「画像を分析」をタップすると、
// Gemini が画像の内容（被写体、色、構図など）を日本語で詳細に説明する
```

`generateContent()` にはテキストプロンプトと `UIImage` を同時に渡すことができます。Firebase AI Logic SDK が内部で画像を適切なフォーマットに変換して API に送信してくれるため、Base64 エンコードなどを手動で行う必要はありません。

Gemini API のマルチモーダル機能についてさらに詳しく知りたい方は、[Gemini マルチモーダル API 完全ガイド](/articles/gemini-api/multimodal-api-guide) も参考にしてください。

## チャット機能の実装

連続した会話のコンテキストを保持するチャット機能も、Firebase AI Logic SDK で簡単に実装できます。

```swift
import SwiftUI
import FirebaseAILogic

struct ChatMessage: Identifiable {
    let id = UUID()
    let role: String   // "user" or "model"
    let text: String
}

struct ChatView: View {
    @State private var messages: [ChatMessage] = []
    @State private var inputText = ""
    @State private var isResponding = false
    @State private var chat: Chat?

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack {
            ScrollView {
                LazyVStack(alignment: .leading, spacing: 12) {
                    ForEach(messages) { message in
                        HStack {
                            if message.role == "user" { Spacer() }
                            Text(message.text)
                                .padding(12)
                                .background(
                                    message.role == "user"
                                        ? Color.blue.opacity(0.2)
                                        : Color.gray.opacity(0.2)
                                )
                                .cornerRadius(12)
                            if message.role == "model" { Spacer() }
                        }
                    }
                }
                .padding()
            }

            HStack {
                TextField("メッセージ", text: $inputText)
                    .textFieldStyle(.roundedBorder)
                Button("送信") {
                    Task { await sendMessage() }
                }
                .disabled(inputText.isEmpty || isResponding)
            }
            .padding()
        }
        .onAppear {
            chat = model.startChat()
        }
    }

    func sendMessage() async {
        let userText = inputText
        inputText = ""
        messages.append(ChatMessage(role: "user", text: userText))
        isResponding = true
        defer { isResponding = false }

        do {
            let response = try await chat?.sendMessage(userText)
            if let text = response?.text {
                messages.append(ChatMessage(role: "model", text: text))
            }
        } catch {
            messages.append(
                ChatMessage(role: "model", text: "エラー: \(error.localizedDescription)")
            )
        }
    }
}

// 期待する動作:
// ユーザーとAIの会話がチャットUIで表示され、
// 過去のメッセージのコンテキストを保持した応答が返される
```

`model.startChat()` で `Chat` インスタンスを作成し、`chat.sendMessage()` でメッセージを送信します。SDK が内部で会話履歴を管理してくれるため、コンテキストを手動で組み立てる必要はありません。

## System Instructions でモデルの振る舞いを制御する

アプリの用途に合わせて、Gemini の応答スタイルを制御したい場面は多いでしょう。System Instructions を設定することで、モデルの振る舞いをカスタマイズできます。

```swift
import FirebaseAILogic

// System Instructions 付きでモデルを初期化
let model = FirebaseAI.firebaseAI(backend: .googleAI())
    .generativeModel(
        modelName: "gemini-2.5-flash",
        systemInstruction: ModelContent(
            role: "system",
            parts: [
                .text("""
                あなたは親切な料理アシスタントです。
                以下のルールに従ってください:
                - 質問には簡潔に、3文以内で回答する
                - 食材のアレルギー情報を必ず含める
                - 料理初心者にも分かりやすい言葉を使う
                """)
            ]
        )
    )

// 生成パラメータのカスタマイズ
let config = GenerationConfig(
    temperature: 0.7,      // 創造性（0.0〜2.0）
    topP: 0.9,             // 核サンプリング
    topK: 40,              // 上位K個の候補から選択
    maxOutputTokens: 1024  // 最大出力トークン数
)

let modelWithConfig = FirebaseAI.firebaseAI(backend: .googleAI())
    .generativeModel(
        modelName: "gemini-2.5-flash",
        generationConfig: config,
        systemInstruction: ModelContent(
            role: "system",
            parts: [.text("あなたは料理アシスタントです。")]
        )
    )
```

`temperature` を低くすると決定論的な応答に、高くするとより創造的な応答になります。アプリの要件に合わせて調整してください。

## エラーハンドリングのベストプラクティス

本番アプリではエラーハンドリングが不可欠です。Firebase AI Logic SDK で発生しうるエラーとその対処法を確認しておきましょう。

```swift
import FirebaseAILogic

func generateWithErrorHandling(prompt: String) async -> String {
    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    do {
        let result = try await model.generateContent(prompt)

        // Safety フィルターの確認
        if let candidate = result.candidates.first {
            if let finishReason = candidate.finishReason,
               finishReason == .safety {
                return "安全フィルターにより応答がブロックされました。"
            }
        }

        return result.text ?? "応答が空でした。"

    } catch let error as GenerateContentError {
        switch error {
        case .promptBlocked(let response):
            // プロンプトがブロックされた場合
            return "プロンプトが安全基準に抵触しました。"
        case .responseStoppedEarly(let reason, _):
            // 応答が途中で停止した場合
            return "応答が途中で停止しました: \(reason)"
        default:
            return "生成エラー: \(error.localizedDescription)"
        }
    } catch {
        // ネットワークエラーなどの一般的なエラー
        return "通信エラー: \(error.localizedDescription)"
    }
}
```

特にモバイルアプリでは通信環境が不安定になることが多いため、リトライ処理やオフライン時のフォールバックも検討しましょう。Gemini API のエラー対処について網羅的に学びたい方は、[Gemini API エラーハンドリング完全ガイド](/articles/gemini-api/gemini-api-error-handling-troubleshooting) が参考になります。

## 実践的なアプリ構成例 — AI レシピアシスタント

ここまでの技術を組み合わせて、実用的なアプリの構成例を示します。カメラで撮影した食材の画像を Gemini に分析させ、レシピを提案する AI レシピアシスタントです。

```swift
import SwiftUI
import FirebaseAILogic

class RecipeAssistantViewModel: ObservableObject {
    @Published var suggestedRecipe = ""
    @Published var isLoading = false

    private let model: GenerativeModel

    init() {
        model = FirebaseAI.firebaseAI(backend: .googleAI())
            .generativeModel(
                modelName: "gemini-2.5-flash",
                systemInstruction: ModelContent(
                    role: "system",
                    parts: [
                        .text("""
                        あなたはプロの料理アシスタントです。
                        画像に写っている食材を特定し、
                        それらを使ったレシピを1つ提案してください。
                        レシピは以下の形式で出力してください:
                        - 料理名
                        - 所要時間
                        - 材料リスト
                        - 手順（番号付き）
                        - ワンポイントアドバイス
                        """)
                    ]
                )
            )
    }

    func analyzeAndSuggest(image: UIImage) async {
        await MainActor.run { isLoading = true }
        defer { Task { @MainActor in isLoading = false } }

        do {
            let stream = try model.generateContentStream(
                "この画像の食材を分析して、レシピを提案してください。",
                image
            )
            var fullText = ""
            for try await chunk in stream {
                if let text = chunk.text {
                    fullText += text
                    await MainActor.run {
                        suggestedRecipe = fullText
                    }
                }
            }
        } catch {
            await MainActor.run {
                suggestedRecipe = "エラーが発生しました。再度お試しください。"
            }
        }
    }
}
```

この構成では、System Instructions でモデルの出力形式を制御し、ストリーミングで UX を最適化しています。マルチモーダル入力とストリーミングを組み合わせることで、ユーザーが食材の写真を撮るだけでリアルタイムにレシピが表示される体験を実現できます。

Gemini API を活用した開発環境のセットアップについては、[Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart) でも基本的な手順を紹介しています。

## よくある質問（FAQ）

### Q1: 旧 `google-generative-ai-swift` パッケージから移行するにはどうすればよいですか？

旧パッケージは 2026 年 6 月に完全非推奨となります。移行手順としては、まず `GoogleGenerativeAI` パッケージを削除し、Firebase SDK の `FirebaseAILogic` ライブラリを追加します。`import GoogleGenerativeAI` を `import FirebaseAILogic` に変更し、モデルの初期化コードを `FirebaseAI.firebaseAI(backend: .googleAI()).generativeModel(...)` の形式に書き換えてください。API の使い方（`generateContent` や `generateContentStream`）はほぼ同じです。

### Q2: Firebase AI Logic SDK は無料で使えますか？

Firebase AI Logic SDK 自体は無料です。ただし、Gemini API の利用には API の使用量に応じた料金が発生します。開発・テスト段階では無料枠（1 分あたり 15 リクエスト）で十分対応できます。本番環境でのコスト管理については、[Google AI Studio](https://aistudio.google.com/) でクォータと使用量を確認できます。

### Q3: オフライン環境での動作は可能ですか？

Firebase AI Logic SDK はクラウドベースの API を呼び出すため、ネットワーク接続が必須です。オフライン対応が必要な場合は、よく使うレスポンスをローカルにキャッシュする設計を検討してください。Gemini Nano のオンデバイス推論が将来的に SDK に統合される可能性はありますが、現時点では Pixel デバイス向けの限定提供です。

### Q4: App Store の審査で AI 機能が問題になることはありますか？

Apple は AI 機能を持つアプリの審査において、ユーザーに AI が生成したコンテンツであることを明示すること、不適切なコンテンツのフィルタリング機能を実装していることを求めています。Firebase AI Logic SDK にはセーフティフィルターが組み込まれているため、基本的な対策は SDK 側でカバーされています。ただし、アプリ固有の安全対策も追加で実装することを推奨します。

### Q5: Gemini 2.5 Pro と Flash のどちらを使うべきですか？

一般的な用途（チャット、テキスト生成、簡単な画像分析）には `gemini-2.5-flash` が最適です。高速で料金も抑えられます。複雑な推論、長文の生成、高精度な分析が必要な場面では `gemini-2.5-pro` を検討してください。アプリ内でユースケースに応じてモデルを切り替える設計にすると、コストと品質のバランスが取れます。

## まとめ

Firebase AI Logic SDK を使うことで、iOS アプリへの Gemini API 統合は驚くほどシンプルになります。テキスト生成、ストリーミング、マルチモーダル入力、チャット機能と、モバイル AI アプリに必要な機能が一通り揃っています。

開発の第一歩として、この記事のコード例をベースに小さなプロトタイプを作ってみることをおすすめします。Firebase の認証やデータベースと組み合わせれば、パーソナライズされた AI 体験を持つアプリを効率よく構築できるでしょう。
