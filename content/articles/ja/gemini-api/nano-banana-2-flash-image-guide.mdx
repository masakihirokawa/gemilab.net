Googleが2026年2月にリリースした**Nano Banana 2（Gemini 3.1 Flash Image Preview）**は、Flash アーキテクチャにPro級の4K画像生成能力を統合した高効率モデルです。高速かつ低コストで高品質な画像を生成できるため、大量の画像処理が必要な開発者やクリエイターにとって注目の選択肢となっています。

この記事では、Nano Banana 2 の主要機能、API を使った実装方法、料金体系、そして実践的なコード例を詳しく解説します。

## Nano Banana 2 とは？主な特徴を解説

Nano Banana 2（モデルID: `gemini-3.1-flash-image-preview`）は、Gemini 3 Pro Image の高効率版として位置づけられています。速度とコストを重視するユースケース向けに最適化されており、以下の特徴を持っています。

**高解像度出力**: 512px（0.5K）から最大4Kまで、用途に応じた解像度を選択できます。デフォルトは1Kで、必要に応じて2Kや4Kへ引き上げることが可能です。

**豊富なアスペクト比**: 1:1、3:2、16:9、9:16、21:9 など14種類のアスペクト比に対応しています。SNS投稿用の正方形から超ワイドなバナーまで、幅広いフォーマットをカバーします。

**高精度テキスト描画**: マーケティングモックアップやグリーティングカードなど、テキストを含む画像生成において正確で読みやすい文字を描画します。

**キャラクター一貫性**: 最大5キャラクターの一貫性維持と、1ワークフロー内で最大14オブジェクトの忠実な再現が可能です。ストーリーテリングやシリーズものの画像制作に活用できます。

**Web検索によるグラウンディング**: Google検索と画像検索の結果をリアルタイムで参照し、特定のテーマや被写体をより正確に描写します。

## Python で画像を生成する

基本的なテキストから画像への生成コードは以下のとおりです。

```python
from google import genai
from google.genai import types
import base64

# クライアントの初期化
client = genai.Client(api_key="YOUR_API_KEY")

# テキストから画像を生成
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents="夕日に照らされた富士山と桜の風景、写実的なスタイル",
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        image_config=types.ImageConfig(
            aspect_ratio="16:9",
            image_size="2K"
        )
    )
)

# レスポンスから画像を保存
for part in response.candidates[0].content.parts:
    if part.inline_data is not None:
        image_data = base64.b64decode(part.inline_data.data)
        with open("generated_image.png", "wb") as f:
            f.write(image_data)
        print("画像を保存しました: generated_image.png")
    elif part.text is not None:
        print(f"テキスト: {part.text}")

# 期待される出力:
# テキスト: Here is the generated image of Mt. Fuji at sunset with cherry blossoms.
# 画像を保存しました: generated_image.png
```

このコードでは、16:9 のアスペクト比で 2K 解像度の画像を生成しています。`response_modalities` に `"IMAGE"` を含めることがポイントです。

## JavaScript で画像を生成する

Node.js 環境での実装例です。

```javascript
import { GoogleGenAI } from "@google/genai";
import fs from "fs";

const ai = new GoogleGenAI({ apiKey: "YOUR_API_KEY" });

async function generateImage() {
  const response = await ai.models.generateContent({
    model: "gemini-3.1-flash-image-preview",
    contents: "A futuristic Tokyo cityscape at night with neon lights, photorealistic",
    config: {
      responseModalities: ["TEXT", "IMAGE"],
      imageConfig: {
        aspectRatio: "21:9",
        imageSize: "4K"
      }
    }
  });

  // レスポンスから画像を抽出して保存
  for (const part of response.candidates[0].content.parts) {
    if (part.inlineData) {
      const buffer = Buffer.from(part.inlineData.data, "base64");
      fs.writeFileSync("output.png", buffer);
      console.log("画像を保存しました: output.png");
    } else if (part.text) {
      console.log("テキスト:", part.text);
    }
  }
}

generateImage();

// 期待される出力:
// テキスト: Here is a futuristic Tokyo cityscape at night with neon lights.
// 画像を保存しました: output.png
```

21:9 の超ワイドアスペクト比と 4K 解像度を指定しています。バナーやヒーロー画像に最適な設定です。

## 画像編集とマルチターン会話

Nano Banana 2 は単発の画像生成だけでなく、生成した画像を対話的に編集することもできます。

```python
from google import genai
from google.genai import types
from PIL import Image
import base64
import io

client = genai.Client(api_key="YOUR_API_KEY")

# 既存の画像を読み込む
with open("original.png", "rb") as f:
    image_bytes = f.read()

# 画像を編集する指示を送信
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[
        types.Content(
            parts=[
                types.Part(
                    inline_data=types.Blob(
                        mime_type="image/png",
                        data=base64.b64encode(image_bytes).decode()
                    )
                ),
                types.Part(text="この画像の背景を雪景色に変更してください")
            ]
        )
    ],
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        image_config=types.ImageConfig(
            image_size="2K"
        )
    )
)

# 編集された画像を保存
for part in response.candidates[0].content.parts:
    if part.inline_data is not None:
        edited = base64.b64decode(part.inline_data.data)
        with open("edited_image.png", "wb") as f:
            f.write(edited)
        print("編集画像を保存しました")

# 期待される出力:
# 編集画像を保存しました
```

参照画像を最大10枚まで入力できるため、スタイルの参考画像を複数渡して一貫したテイストの画像を生成することも可能です。

## Thinking モードの活用

Nano Banana 2 は Thinking モードをサポートしており、画像生成の品質を制御できます。

```python
# Thinking レベルを指定して画像生成
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents="精密な建築図面スタイルの東京タワーのイラスト",
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        thinking_config=types.ThinkingConfig(
            thinking_level="HIGH",       # "MINIMAL" or "HIGH"
            include_thoughts=True        # 思考プロセスを表示
        ),
        image_config=types.ImageConfig(
            image_size="2K"
        )
    )
)

# Thinking の内容を確認
for part in response.candidates[0].content.parts:
    if hasattr(part, "thought") and part.thought:
        print(f"思考: {part.text}")
    elif part.text:
        print(f"回答: {part.text}")

# 期待される出力:
# 思考: ユーザーは建築図面スタイルの東京タワーを求めている...
# 回答: Here is an architectural blueprint-style illustration of Tokyo Tower.
```

`HIGH` を指定すると、より複雑な構図やディテールの要求に対して品質が向上します。ただし、Thinking トークンは表示設定にかかわらず課金対象となる点に注意してください。

## 料金と性能の比較

Nano Banana 2 は、Pro モデルと比較して大幅にコストを抑えられます。

| 項目 | Nano Banana 2 | Gemini 3 Pro Image |
|------|--------------|-------------------|
| 標準画像（1K） | 約$0.05 | 約$0.08 |
| 4K画像 | 約$0.15 | 約$0.40 |
| 生成速度 | 4〜6秒 | 8〜12秒 |
| 最大解像度 | 4K | 4K |
| キャラクター一貫性 | 最大5キャラ | 最大5キャラ |

大量の画像生成が必要なプロジェクトでは、Batch API を利用することでさらにコストを最適化できます。Batch API は非同期処理で、通常より長いターンアラウンドタイムと引き換えに割引料金が適用されます。

## よくあるエラーと対処法

**エラー: `SAFETY_BLOCKED`**
安全性フィルターにより画像生成がブロックされた場合に発生します。プロンプトを調整し、暴力的・性的な内容を含まないようにしてください。

**エラー: `INVALID_ARGUMENT` (image_size)**
サポートされていない解像度を指定した場合に発生します。`"0.5K"`, `"1K"`, `"2K"`, `"4K"` のいずれかを使用してください。

**エラー: `RATE_LIMIT_EXCEEDED`**
レート制限に達した場合は、リクエスト間にウェイトを入れるか、Batch API の利用を検討してください。

## FAQ

### Q: Nano Banana 2 と Gemini 3 Pro Image の違いは？

Nano Banana 2 は Flash アーキテクチャをベースにしており、Pro Image と比較して生成速度が約2倍速く、コストは37〜63%安くなっています。画質面ではProに迫る品質を実現していますが、極めて高度な構図や光の表現ではProがやや優位です。大量処理や高速レスポンスが求められるユースケースに適しています。

### Q: 生成された画像のライセンスは？

Gemini API で生成された画像には SynthID の電子透かしが埋め込まれます。商用利用については Google の利用規約を確認してください。個人開発やプロトタイピングでの利用は一般的に許可されています。

### Q: Function Calling やライブAPIとの組み合わせは可能？

現時点のプレビュー版では、Function Calling、Live API、構造化出力（Structured Outputs）はサポートされていません。テキスト＋画像の入出力に特化したモデルとなっています。将来のアップデートで機能が追加される可能性があります。

### Q: 無料枠で使えますか？

Google AI Studio の無料枠でも Nano Banana 2 を試すことができます。ただし、レート制限が設けられており、本番環境での大量利用には有料プランへのアップグレードが必要です。
