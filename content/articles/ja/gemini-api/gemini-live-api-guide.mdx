## はじめに

2026年初頭、GoogleはGemini Live APIをVertex AI上で**一般提供（GA）**として正式リリースしました。このAPIを使うことで、音声・映像・テキストを組み合わせたリアルタイムの会話AIを構築できるようになりました。コールセンターや教育アプリ、インタラクティブなアシスタントなど、幅広い用途で活用が期待されています。

本記事では、Gemini Live APIの概要から実際の実装方法まで、ステップバイステップで解説します。

## Gemini Live APIとは

Gemini Live APIは、低レイテンシのマルチモーダル会話を実現するためのAPIです。従来のチャットAPIとは異なり、以下の特徴があります。

- **リアルタイムの双方向通信**：WebSocketベースの接続で音声・映像データをストリーミング
- **音声と映像の同時処理**：マイクとカメラからの入力を同時に処理できる
- **低レイテンシ**：会話の自然なテンポを維持するための高速応答
- **Gemini 2.5 Flash Native Audioモデルを採用**：音声認識と音声合成を統合したネイティブ対応

Vertex AI上でGAとなったことで、エンタープライズグレードのSLAや、GCP（Google Cloud Platform）の豊富なセキュリティ機能と組み合わせて利用できるようになりました。

## 基本的な実装

### テキストベースのリアルタイム会話

まずは最も基本的なテキストの双方向会話から始めましょう。

```python
import asyncio
from google import genai
from google.genai.types import LiveConnectConfig, Content, Part

# クライアントの初期化（Vertex AIを使用）
client = genai.Client(
    vertexai=True,
    project="your-project-id",
    location="us-central1"
)

MODEL = "gemini-2.5-flash-live-preview"

async def text_chat():
    config = LiveConnectConfig(
        response_modalities=["TEXT"],
    )

    async with client.aio.live.connect(model=MODEL, config=config) as session:
        # ユーザーのメッセージを送信
        await session.send(
            input="Gemini Live APIについて教えてください",
            end_of_turn=True
        )

        # レスポンスを受信
        async for response in session.receive():
            if response.text:
                print(response.text, end="", flush=True)

asyncio.run(text_chat())
```

### 音声入出力の実装

次に、マイクからの入力とスピーカーへの出力を実装します。

```python
import asyncio
import pyaudio
from google import genai
from google.genai.types import (
    LiveConnectConfig,
    SpeechConfig,
    VoiceConfig,
    PrebuiltVoiceConfig
)

# オーディオの設定
FORMAT = pyaudio.paInt16
CHANNELS = 1
SEND_SAMPLE_RATE = 16000   # マイク入力サンプルレート
RECEIVE_SAMPLE_RATE = 24000  # スピーカー出力サンプルレート
CHUNK_SIZE = 1024

client = genai.Client(
    vertexai=True,
    project="your-project-id",
    location="us-central1"
)

MODEL = "gemini-2.5-flash-live-preview"

async def voice_chat():
    pya = pyaudio.PyAudio()

    # スピーカー出力ストリームを開く
    output_stream = pya.open(
        format=FORMAT,
        channels=CHANNELS,
        rate=RECEIVE_SAMPLE_RATE,
        output=True
    )

    config = LiveConnectConfig(
        response_modalities=["AUDIO"],
        speech_config=SpeechConfig(
            voice_config=VoiceConfig(
                prebuilt_voice_config=PrebuiltVoiceConfig(
                    voice_name="Aoede"  # 利用可能なボイス: Aoede, Charon, Fenrir, Kore, Puck など
                )
            )
        )
    )

    async with client.aio.live.connect(model=MODEL, config=config) as session:
        # マイク入力ストリームを開く
        input_stream = pya.open(
            format=FORMAT,
            channels=CHANNELS,
            rate=SEND_SAMPLE_RATE,
            input=True,
            frames_per_buffer=CHUNK_SIZE
        )

        async def send_audio():
            """マイクから音声を送信"""
            while True:
                data = await asyncio.get_event_loop().run_in_executor(
                    None, input_stream.read, CHUNK_SIZE
                )
                await session.send(input={"data": data, "mime_type": "audio/pcm"})

        async def receive_audio():
            """音声レスポンスを再生"""
            async for response in session.receive():
                if response.data:
                    output_stream.write(response.data)

        # 送受信を並行実行
        await asyncio.gather(send_audio(), receive_audio())

asyncio.run(voice_chat())
```

## 高度な活用：映像入力の追加

カメラからの映像もリアルタイムで処理できます。

```python
import asyncio
import cv2
import base64
from google import genai
from google.genai.types import LiveConnectConfig

client = genai.Client(
    vertexai=True,
    project="your-project-id",
    location="us-central1"
)

MODEL = "gemini-2.5-flash-live-preview"

async def video_chat():
    config = LiveConnectConfig(
        response_modalities=["TEXT"],
        system_instruction="あなたは映像を分析するAIアシスタントです。見えているものを日本語で説明してください。"
    )

    cap = cv2.VideoCapture(0)  # カメラを開く

    async with client.aio.live.connect(model=MODEL, config=config) as session:
        async def send_video_frames():
            """カメラのフレームを定期的に送信"""
            while True:
                ret, frame = cap.read()
                if not ret:
                    break

                # フレームをJPEGにエンコード
                _, buffer = cv2.imencode(".jpg", frame)
                image_data = base64.b64encode(buffer).decode("utf-8")

                # 映像データを送信
                await session.send(
                    input={
                        "mime_type": "image/jpeg",
                        "data": image_data
                    }
                )
                await asyncio.sleep(0.5)  # 0.5秒ごとにフレーム送信

        async def receive_responses():
            """テキストレスポンスを受信"""
            async for response in session.receive():
                if response.text:
                    print(f"AI: {response.text}")

        await asyncio.gather(send_video_frames(), receive_responses())

    cap.release()

asyncio.run(video_chat())
```

## System Instructionでカスタマイズ

Gemini Live APIでは、`system_instruction`を使ってAIの振る舞いをカスタマイズできます。

```python
config = LiveConnectConfig(
    response_modalities=["AUDIO"],
    system_instruction="""
    あなたは親切な日本語カスタマーサポートAIです。
    - 常に丁寧な敬語を使用してください
    - 技術的な質問には具体的な手順を説明してください
    - わからない場合は正直に伝え、人間のオペレーターへの転送を提案してください
    """,
    speech_config=SpeechConfig(
        voice_config=VoiceConfig(
            prebuilt_voice_config=PrebuiltVoiceConfig(voice_name="Kore")
        )
    )
)
```

## FAQ

**Q: Gemini Live APIの料金はどのくらいですか？**

A: Vertex AI上でのGemini Live APIは、音声入出力のトークン数に基づいて課金されます。2026年1月28日から正式課金が開始されました。具体的な料金は[Google CloudのVertex AI料金ページ](https://cloud.google.com/vertex-ai/generative-ai/pricing)をご確認ください。

**Q: 対応言語は何ですか？**

A: 日本語を含む多くの言語に対応しています。音声認識・音声合成ともに日本語のサポートが充実しています。

**Q: 接続の最大時間はありますか？**

A: 1セッションあたりの最大接続時間は15分です。長時間の会話が必要な場合は、セッションを再接続する実装を行ってください。

**Q: Vertex AI SDKは使えますか？**

A: 2026年6月以降のVertex AI SDKのリリースではGeminiのサポートが終了します。新機能を利用するには**Google Gen AI SDK**への移行が必要です。移行は比較的シンプルで、`google-cloud-aiplatform`から`google-genai`パッケージへの切り替えが主な変更点です。

**Q: ストリーミングの遅延はどのくらいですか？**

A: 一般的に100ms〜300ms程度の遅延で動作します。ネットワーク環境やリージョンの設定によって変わります。
