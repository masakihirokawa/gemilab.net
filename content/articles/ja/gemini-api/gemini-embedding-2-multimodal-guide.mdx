## Gemini Embedding 2 とは

2026年3月10日、Googleは**Gemini Embedding 2**（モデルID: `gemini-embedding-2-preview`）を公開プレビューとして発表しました。これはGoogleが提供する**初の完全マルチモーダル埋め込みモデル**です。

従来の埋め込みモデルはテキストのみ、あるいは画像のみを扱うことが多く、異なるモダリティ（テキスト・画像・動画・音声）を横断した検索を実現するには、複数のモデルを組み合わせる必要がありました。Gemini Embedding 2 はこの問題を根本から解決し、**テキスト・画像・動画・音声・PDFをすべて単一の統一ベクトル空間に変換**します。

この記事では、Gemini Embedding 2 の主な特徴、API の使い方、そして実際のユースケースへの応用方法をステップバイステップで解説します。

---

## 主な特徴と性能

Gemini Embedding 2 の主な仕様は以下の通りです。

| 項目 | 詳細 |
|------|------|
| モデルID | `gemini-embedding-2-preview` |
| 対応モダリティ | テキスト・画像・動画・音声・PDF |
| 出力次元数 | 128〜3,072次元（推奨: 768, 1536, 3072） |
| テキスト入力上限 | 8,192トークン |
| 画像入力上限 | 1リクエストあたり最大6枚（PNG/JPEG） |
| 音声入力上限 | 最大80秒（MP3/WAV） |
| 動画入力上限 | 最大128秒（MP4/MOV、H264/H265/AV1/VP9） |
| PDF入力上限 | 最大6ページ |

**パフォーマンス面での優位性**: 早期採用者のレポートによると、従来のマルチモデルパイプライン（テキスト用モデル＋画像用モデルを別々に運用）と比較して、**レイテンシが約70%削減、リコール率が約20%向上**したとのことです。

既存の埋め込みモデル `gemini-embedding-001` とはベクトル空間が**互換性なし**（incompatible）のため、既存のベクトルストアを持っている場合は全件の再埋め込みが必要になります。

## 基本的な使い方

### テキストの埋め込み

まず、最もシンプルなテキスト埋め込みから始めましょう。

```python
import os
from google import genai

# クライアントを初期化
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# テキストを埋め込む
result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=["Gemini APIはGoogleが提供するAIモデルのAPIです。"]
)

embedding = result.embeddings[0].values
print(f"埋め込み次元数: {len(embedding)}")
# 出力例: 埋め込み次元数: 3072

# 複数のテキストを一度に埋め込む（バッチ処理）
texts = [
    "Gemini APIの使い方を学ぼう",
    "マルチモーダルAIとは何か",
    "RAGシステムの構築方法"
]
batch_result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=texts
)
print(f"バッチ処理: {len(batch_result.embeddings)} 件の埋め込み生成完了")
```

### 出力次元数のカスタマイズ

用途に応じて出力次元数を指定できます。次元数を減らすと、ストレージコストとレイテンシを削減できます。

```python
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 768次元で出力（軽量・高速）
result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=["テスト文章"],
    config=types.EmbedContentConfig(output_dimensionality=768)
)
print(f"次元数: {len(result.embeddings[0].values)}")  # 768
```

## マルチモーダル埋め込みの実装

Gemini Embedding 2 の最大の特徴は、画像・音声・動画を**テキストと同一のベクトル空間**に変換できる点です。

### 画像の埋め込み

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# ローカル画像ファイルを読み込む
with open("sample.jpg", "rb") as f:
    image_bytes = f.read()

# 画像を埋め込む
result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=[
        types.Part.from_bytes(data=image_bytes, mime_type="image/jpeg")
    ]
)

image_embedding = result.embeddings[0].values
print(f"画像の埋め込み次元数: {len(image_embedding)}")
# 出力例: 画像の埋め込み次元数: 3072
```

### テキストと画像のクロスモーダル検索

最も強力なユースケースは、**テキストクエリで画像を検索する**クロスモーダル検索です。テキストと画像が同一の埋め込み空間に存在するため、コサイン類似度で直接比較できます。

```python
import os
import numpy as np
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

def get_embedding(content):
    """テキストまたは画像バイトから埋め込みを取得する"""
    if isinstance(content, str):
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[content]
        )
    else:
        # バイト列の場合は画像として扱う
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[types.Part.from_bytes(data=content, mime_type="image/jpeg")]
        )
    return np.array(result.embeddings[0].values)

def cosine_similarity(a, b):
    """コサイン類似度を計算する"""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 画像データベースを構築（実際はファイルパスのリストなど）
image_files = ["cat.jpg", "dog.jpg", "landscape.jpg"]
image_embeddings = []

for img_path in image_files:
    with open(img_path, "rb") as f:
        emb = get_embedding(f.read())
        image_embeddings.append((img_path, emb))

# テキストクエリで画像を検索
query = "かわいい猫の写真"
query_embedding = get_embedding(query)

# 類似度スコアで並べ替え
results = [
    (path, cosine_similarity(query_embedding, emb))
    for path, emb in image_embeddings
]
results.sort(key=lambda x: x[1], reverse=True)

print("検索結果（テキスト → 画像）:")
for path, score in results:
    print(f"  {path}: 類似度 {score:.4f}")
# 出力例:
# 検索結果（テキスト → 画像）:
#   cat.jpg: 類似度 0.8912
#   dog.jpg: 類似度 0.6234
#   landscape.jpg: 類似度 0.3421
```

### 音声・動画の埋め込み

音声ファイルや動画ファイルも同様に埋め込めます。

## 実践: マルチモーダルRAGシステムの構築

Gemini Embedding 2 を使えば、テキスト・画像・PDF を横断した統合RAGシステムを構築できます。以下は、PDFドキュメントと画像を組み合わせた検索システムの例です。

```python
import os
from google import genai
from google.genai import types
import numpy as np

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

class MultimodalRAG:
    """テキスト・画像・PDFを横断した検索システム"""

    def __init__(self):
        self.index = []  # (content_id, type, embedding) のリスト

    def add_text(self, text_id, text):
        """テキストをインデックスに追加"""
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[text]
        )
        emb = np.array(result.embeddings[0].values)
        self.index.append((text_id, "text", emb))
        print(f"✅ テキスト追加: {text_id}")

    def add_image(self, image_id, image_bytes):
        """画像をインデックスに追加"""
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[
                types.Part.from_bytes(data=image_bytes, mime_type="image/jpeg")
            ]
        )
        emb = np.array(result.embeddings[0].values)
        self.index.append((image_id, "image", emb))
        print(f"✅ 画像追加: {image_id}")

    def search(self, query_text, top_k=3):
        """テキストクエリでマルチモーダル検索"""
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[query_text]
        )
        query_emb = np.array(result.embeddings[0].values)

        scores = []
        for content_id, content_type, emb in self.index:
            score = np.dot(query_emb, emb) / (
                np.linalg.norm(query_emb) * np.linalg.norm(emb)
            )
            scores.append((content_id, content_type, float(score)))

        scores.sort(key=lambda x: x[2], reverse=True)
        return scores[:top_k]

# 使用例
rag = MultimodalRAG()
rag.add_text("doc1", "Gemini APIはGoogleのAIモデルへのアクセスを提供します。")
rag.add_text("doc2", "マルチモーダルとは複数の情報形式（テキスト・画像・音声）を扱うことです。")

with open("diagram.jpg", "rb") as f:
    rag.add_image("img1", f.read())

results = rag.search("AIのマルチモーダル処理とは", top_k=3)
print("\n検索結果:")
for content_id, content_type, score in results:
    print(f"  [{content_type}] {content_id}: {score:.4f}")
```

## gemini-embedding-001 からの移行

`gemini-embedding-001` からの移行を検討している場合、**重要な注意事項**があります。

両モデルのベクトル空間は**完全に非互換**です。`gemini-embedding-001` で生成したベクトルと `gemini-embedding-2-preview` で生成したベクトルを直接比較することはできません。移行手順は以下の通りです。

1. **全データの再埋め込みが必須**: 既存のベクトルストア（Pinecone、Weaviate、pgvector 等）に保存されたベクトルを、新モデルで全件再生成する
2. **段階的移行を推奨**: 新旧インデックスを並行稼働させ、新インデックスが完成してから切り替える
3. **コスト試算**: データ量が多い場合は事前にAPI使用量とコストを計算する

```python
# 移行スクリプトの例（既存テキストリストから再埋め込み）
old_texts = ["テキスト1", "テキスト2", "テキスト3"]  # 既存データ
new_embeddings = []

# バッチ処理で効率よく再埋め込み
batch_size = 10
for i in range(0, len(old_texts), batch_size):
    batch = old_texts[i:i+batch_size]
    result = client.models.embed_content(
        model="gemini-embedding-2-preview",
        contents=batch
    )
    for j, emb in enumerate(result.embeddings):
        new_embeddings.append({
            "text": batch[j],
            "embedding": emb.values
        })
    print(f"処理済み: {min(i+batch_size, len(old_texts))}/{len(old_texts)}")

print(f"✅ 再埋め込み完了: {len(new_embeddings)} 件")
# 出力例: ✅ 再埋め込み完了: 3 件
```

## よくある質問（FAQ）

**Q1. Gemini Embedding 2 は本番環境で使えますか？**

現時点（2026年3月）では、`gemini-embedding-2-preview` はパブリックプレビュー段階です。APIの仕様や料金は変更される可能性があります。重要な本番システムへの採用前に、最新の[公式ドキュメント](https://ai.google.dev/gemini-api/docs/models/gemini-embedding-2-preview)で安定版リリースの状況を確認してください。

**Q2. テキスト専用の用途なら、gemini-embedding-001 と比べてどちらが良いですか？**

テキストのみを扱う場合、`gemini-embedding-001` と `gemini-embedding-2-preview` はどちらも高品質な埋め込みを生成します。ただし、将来的にマルチモーダル検索への拡張を検討しているなら、今から `gemini-embedding-2-preview` に乗り換えておくことで、後の全件再埋め込みコストを回避できます。

**Q3. 1リクエストでテキストと画像を混在させて埋め込めますか？**

はい、可能です。テキストと画像が混在したコンテンツ（例：商品説明テキスト＋商品画像）を1リクエストで埋め込めます。これにより、マルチモーダルなコンテキストを一つのベクトルで表現できます。

**Q4. ベクトルの保存には何を使えばいいですか？**

オープンソースでは pgvector（PostgreSQL 拡張）や Chroma、クラウドサービスでは Pinecone や Weaviate、Google Cloud の Vertex AI Vector Search が人気です。3,072次元のベクトルを大量に扱う場合は、ANN（近似最近傍探索）をサポートするソリューションを選ぶと検索速度が向上します。

**Q5. 料金はどのくらいかかりますか？**

料金は[Google AI Studio](https://aistudio.google.com/)または[Vertex AI](https://cloud.google.com/vertex-ai)の公式料金ページを確認してください。プレビュー期間中は変更される可能性があります。

## まとめ

Gemini Embedding 2 は、テキスト・画像・動画・音声・PDFを統一的に扱える画期的な埋め込みモデルです。このモデルにより、従来は複数のモデルを組み合わせなければ実現できなかったクロスモーダル検索が、シンプルなAPIコールだけで実装可能になりました。

主なポイントをまとめると、Gemini Embedding 2 は初のマルチモーダル埋め込みモデルで、出力次元数は128〜3,072次元まで柔軟に設定できます。従来モデルとの埋め込み空間は非互換なので全件再埋め込みが必要です。クロスモーダル検索、マルチモーダルRAG、レコメンドシステムなど幅広い用途に活用できます。

Gemini Embeddings APIの基礎については [Embeddings API 基本ガイド](/articles/gemini-api/embeddings-api-guide) を、マルチモーダルRAGの詳細設計については [Multimodal RAG システム構築ガイド](/articles/gemini-advanced/gemini-multimodal-rag) を参照してください。また、Gemini APIの基本的な使い方から始めたい方は [Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart) もご覧ください。
