Gemini APIの**URLコンテキストツール**は、ウェブページのURL・PDF・画像などのリンクをそのままモデルに渡し、リアルタイムでコンテンツを参照させる機能です。2026年2月に一般提供（GA）となり、PDFや画像フォーマットのサポートが追加されました。手動でコンテンツをダウンロードしてアップロードする手間なく、URLを指定するだけで最新情報を元にした応答が得られます。

この記事では、URLコンテキストツールの基本から応用までを、動作するコード例を交えて解説します。

## URLコンテキストツールとは

URLコンテキストツールは、プロンプトと一緒にURLを渡すと、GeminiがそのURLのコンテンツを取得・解析した上で回答を生成するツールです。従来の[File API](/articles/gemini-api/file-api-guide)を使ったファイルアップロードと異なり、URLを文字列として指定するだけで済みます。

主なユースケースは次のとおりです。

- ニュース記事や技術ブログの要約・翻訳
- 公式ドキュメントやPDFレポートの内容に基づいたQ&A
- 複数のWebページを横断した比較・分析
- 画像URLを指定したビジュアル情報の解析

### サポートされるコンテンツタイプ

URLコンテキストツールがサポートするコンテンツは以下の通りです。

| フォーマット | 対応状況 |
|---|---|
| Webページ（HTML） | ✅ |
| PDF | ✅ （表・構造も解析可能） |
| 画像（PNG・JPEG・BMP・WebP） | ✅ |
| JSON / XML / CSV | ✅ |
| プレーンテキスト / RTF | ✅ |
| JavaScript / CSS | ✅ |

## 基本的な使い方

### Python — ウェブページを要約する

```python
import os
from google import genai
from google.genai import types

# クライアントの初期化
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# URLコンテキストツールを有効にしてコンテンツを取得・要約
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        "次のURLの記事を日本語で500字以内に要約してください: "
        "https://blog.google/technology/ai/"
    ),
    config=types.GenerateContentConfig(
        tools=[types.Tool(url_context=types.UrlContext())]
    ),
)

print(response.text)
# 期待する出力例:
# Googleが発表した最新のAI技術動向について、Gemini 3シリーズの詳細や…
```

### JavaScript — ウェブページを要約する

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

async function summarizeUrl(url) {
  const response = await ai.models.generateContent({
    model: "gemini-2.5-pro",
    contents: `次のURLの内容を日本語で要約してください: ${url}`,
    config: {
      tools: [{ urlContext: {} }],
    },
  });
  return response.text;
}

// 使用例
const summary = await summarizeUrl("https://blog.google/technology/ai/");
console.log(summary);
// 期待する出力例:
// Googleが発表した最新AI情報: Gemini 3シリーズが登場し…
```

## PDFの解析

PDFのURLを直接渡すことで、テキストだけでなく表や構造情報も解析できます。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 公開されているPDFレポートを解析
pdf_url = "https://example.com/annual-report-2025.pdf"

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        f"次のPDFレポートから売上データを抽出し、"
        f"主要な数値をリストアップしてください: {pdf_url}"
    ),
    config=types.GenerateContentConfig(
        tools=[types.Tool(url_context=types.UrlContext())]
    ),
)

print(response.text)
# 期待する出力例:
# 2025年度売上データ:
# - 総売上高: $X billion（前年比 +XX%）
# - 営業利益: $X billion…
```

## 複数URLの同時参照

1回のリクエストで複数のURLを参照し、比較・分析することも可能です。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 複数のURLを含むプロンプト
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        "以下の2つの技術記事を比較し、共通点と相違点を整理してください。\n"
        "記事1: https://example.com/article-a\n"
        "記事2: https://example.com/article-b"
    ),
    config=types.GenerateContentConfig(
        tools=[types.Tool(url_context=types.UrlContext())]
    ),
)

print(response.text)
# 期待する出力例:
# 【共通点】
# - 両記事ともLLMの応用事例を扱っている…
# 【相違点】
# - 記事1はAPIコスト最適化に焦点を当てており…
```

## Googleサーチとの組み合わせ

URLコンテキストツールは[Grounding with Google Search](/articles/gemini-advanced/grounding-and-search)と組み合わせることで、特定URLの情報と最新のウェブ検索結果を同時に参照できます。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        "次のドキュメントに書かれているGemini 2.5 Proの機能について、"
        "最新の更新情報も加味して解説してください: "
        "https://example.com/gemini-overview.pdf"
    ),
    config=types.GenerateContentConfig(
        tools=[
            types.Tool(url_context=types.UrlContext()),
            types.Tool(google_search=types.GoogleSearch()),
        ]
    ),
)

print(response.text)
```

> **注意**: Googleサーチツールとの組み合わせ時はコスト（トークン数）が増加します。用途に応じて使い分けてください。

## よくあるエラーと対処法

### エラー: URLにアクセスできない

プライベートなURLや認証が必要なページ、アクセス制限のかかったドメインはGeminiが取得できません。

```
Error: Failed to fetch URL content. Ensure the URL is publicly accessible.
```

**対処法**: 公開されているURLのみ使用してください。認証が必要なコンテンツは[File API](/articles/gemini-api/file-api-guide)でアップロードしてください。

### エラー: コンテンツサイズ超過

1リクエストあたりのコンテキストウィンドウ制限を超えた場合に発生します。

**対処法**: URLの数を減らす、または[Context Caching](/articles/gemini-api/context-caching-guide)を活用して大規模ドキュメントを分割処理してください。

### エラー: 動的コンテンツが取得できない

JavaScriptで動的に生成されるコンテンツは取得できないことがあります。

**対処法**: 静的なHTMLが返されるエンドポイントのURLや、APIレスポンスのURLを使用してください。

## 応用例 — 技術ドキュメントQ&Aボット

URLコンテキストツールを使えば、公式ドキュメントのURLをコンテキストとして与えるだけで、精度の高いQ&Aボットを構築できます。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# ドキュメントURLとユーザーの質問
doc_urls = [
    "https://cloud.google.com/vertex-ai/docs",
    "https://ai.google.dev/gemini-api/docs",
]

def ask_about_docs(question: str, urls: list[str]) -> str:
    url_list = "\n".join(f"- {u}" for u in urls)
    prompt = (
        f"以下の公式ドキュメントを参照して、質問に答えてください。\n"
        f"ドキュメント:\n{url_list}\n\n"
        f"質問: {question}"
    )
    response = client.models.generate_content(
        model="gemini-2.5-pro",
        contents=prompt,
        config=types.GenerateContentConfig(
            tools=[types.Tool(url_context=types.UrlContext())]
        ),
    )
    return response.text

# 使用例
answer = ask_about_docs(
    "Gemini APIのストリーミングを有効にする方法を教えてください",
    doc_urls,
)
print(answer)
# 期待する出力例:
# Gemini APIのストリーミングを有効にするには、generate_content_stream()メソッドを使用します。
# 以下のようにコードを記述します:…
```

## FAQ

**Q. URLコンテキストツールは無料で使えますか？**

A. 取得したURLのコンテンツはトークンとしてカウントされます。通常のGemini API料金体系（入力トークン単価）が適用されます。Google AI Studioの無料枠内でも使用可能ですが、大量のコンテンツを参照する場合はトークン消費量にご注意ください。

**Q. URLコンテキストツールとFile APIの違いは何ですか？**

A. URLコンテキストツールはGeminiが自動的にURLのコンテンツを取得するのに対し、[File API](/articles/gemini-api/file-api-guide)は手動でファイルをアップロードする方式です。URLが公開されている場合はURLコンテキストの方が簡単ですが、認証が必要なファイルやローカルファイルにはFile APIを使用してください。

**Q. 一度に指定できるURLの数に制限はありますか？**

A. 公式に明示された上限はありませんが、参照するURLが多いほどコンテキストウィンドウを消費します。Gemini 2.5 Proの最大コンテキストは100万トークンですが、実用上は5〜10URL程度での使用が安定しています。

**Q. Gemini 2.5 Flash でも使えますか？**

A. はい。`gemini-2.5-flash`を含む主要なGeminiモデルでURLコンテキストツールを使用できます。コストを抑えたい場合はFlashモデルの利用も選択肢の一つです。
