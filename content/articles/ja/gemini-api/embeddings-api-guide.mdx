## Embeddings（埋め込み）とは

**Embeddings** はテキストを数値ベクトルに変換する技術です。意味的に近いテキストは近いベクトル空間に配置されるため、単純なキーワード検索では見つからない「意味的な類似性」を扱えます。

Gemini Embeddings API は、テキストを高品質なベクトルに変換するための API です。

## 利用場面

- **セマンティック検索**: 意味で検索するナレッジベース
- **文書類似度**: 重複コンテンツの検出、関連記事のレコメンド
- **RAG（検索拡張生成）**: ドキュメントを検索してからLLMに渡す手法
- **テキスト分類**: 教師データが少ない分類タスク
- **クラスタリング**: 大量のフィードバックやコメントのグルーピング

## 基本的な使い方

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

# テキストをEmbeddingに変換
result = genai.embed_content(
    model="models/text-embedding-004",
    content="機械学習とは何ですか？",
    task_type="retrieval_query"
)

embedding = result["embedding"]
print(f"Vector dimension: {len(embedding)}")  # 768次元
```

## task_type の種類

`task_type` を適切に指定するとEmbeddingの品質が向上します。

| task_type | 用途 |
|-----------|------|
| `retrieval_query` | 検索クエリ（質問） |
| `retrieval_document` | 検索対象ドキュメント |
| `semantic_similarity` | 文章間の類似度計算 |
| `classification` | テキスト分類 |
| `clustering` | テキストのクラスタリング |

## コサイン類似度で文書検索

```python
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# ドキュメントを事前にEmbeddingに変換して保存
documents = [
    "Gemini 3 Flash は Google の最新 AI モデルです",
    "Python は人気のプログラミング言語です",
    "機械学習では大量のデータでモデルを訓練します",
    "Google AI Studio で Gemini API を無料で試せます",
]

doc_embeddings = []
for doc in documents:
    result = genai.embed_content(
        model="models/text-embedding-004",
        content=doc,
        task_type="retrieval_document"
    )
    doc_embeddings.append(result["embedding"])

# クエリで検索
query = "Google の新しい AI はどれですか？"
query_result = genai.embed_content(
    model="models/text-embedding-004",
    content=query,
    task_type="retrieval_query"
)
query_emb = query_result["embedding"]

# 類似度スコアを計算して上位を返す
scores = [(cosine_similarity(query_emb, doc_emb), doc)
          for doc_emb, doc in zip(doc_embeddings, documents)]
scores.sort(reverse=True)

for score, doc in scores[:3]:
    print(f"{score:.3f}: {doc}")
```

## シンプルな RAG の実装

```python
def rag_search(query, docs, doc_embeddings, model_name="gemini-3-flash-preview"):
    # 1. クエリをEmbeddingに変換
    query_emb = genai.embed_content(
        model="models/text-embedding-004",
        content=query,
        task_type="retrieval_query"
    )["embedding"]

    # 2. 最も関連するドキュメントを取得
    scores = [(cosine_similarity(query_emb, emb), doc)
              for emb, doc in zip(doc_embeddings, docs)]
    scores.sort(reverse=True)
    top_docs = [doc for _, doc in scores[:3]]

    # 3. 取得したドキュメントをコンテキストとしてLLMに渡す
    context = "\n\n".join(top_docs)
    prompt = f"""以下のコンテキスト情報を参考にして、質問に答えてください。

コンテキスト:
{context}

質問: {query}"""

    model = genai.GenerativeModel(model_name)
    response = model.generate_content(prompt)
    return response.text

# 使用例
answer = rag_search("Gemini の最新モデルについて教えて", documents, doc_embeddings)
print(answer)
```

## バッチ処理でのコスト最適化

大量のドキュメントを処理する場合は、バッチで処理して API コールを最小化します。

```python
batch_size = 100
all_embeddings = []

for i in range(0, len(documents), batch_size):
    batch = documents[i:i+batch_size]
    # text-embedding-004 は複数テキストの同時処理に対応
    results = [
        genai.embed_content(
            model="models/text-embedding-004",
            content=doc,
            task_type="retrieval_document"
        )["embedding"]
        for doc in batch
    ]
    all_embeddings.extend(results)
```

## まとめ

Gemini Embeddings API を使うと、テキストの意味的な類似性を扱うアプリケーションを手軽に構築できます。

- `text-embedding-004` モデルで高品質な 768 次元のベクトルを取得
- `task_type` を適切に設定することで精度が向上
- コサイン類似度で類似ドキュメントを高速に検索可能
- RAG パターンで LLM の知識を最新情報で拡張できる

ベクトルデータベース（Pinecone、Weaviate、pgvector など）と組み合わせることで、本格的なスケールの検索システムも構築できます。
