## Gemini API × Next.js で本格的な AI チャットを作ろう

「Gemini API を使って何か作りたいけど、フロントエンドからバックエンドまで全部自分でやるのは大変…」そう思っていませんか？

Next.js を使えば、フロントエンドとバックエンドを 1 つのプロジェクトで完結でき、Gemini API との連携もシンプルに実装できます。このチュートリアルでは、ストリーミング対応の AI チャットアプリを、実際に動くコードとともにゼロから構築します。

完成すると、ユーザーがメッセージを送信すると Gemini がリアルタイムで応答を生成し、文字が次々と画面に表示される、ChatGPT のような体験を実現できます。

<Callout type="info">
このチュートリアルでは Gemini 3 Flash を使用します。高速かつ低コストで、チャットアプリに最適なモデルです。
</Callout>

## 前提知識と環境準備

### 必要なもの

- **Node.js** 18 以上（推奨: 20 LTS）
- **npm** または **pnpm**
- **Google AI Studio の API キー**（[aistudio.google.com](https://aistudio.google.com/) で無料取得可能）
- TypeScript と React の基本知識

### API キーの取得

Google AI Studio にアクセスし、左メニューの「API keys」から新しいキーを作成します。無料枠では 1 分あたり 15 リクエスト、1 日あたり 1,500 リクエストが利用可能です。

### プロジェクトの作成

ターミナルで以下のコマンドを実行します。

```bash
npx create-next-app@latest gemini-chat-app --typescript --tailwind --app --src-dir
cd gemini-chat-app
```

次に、Google AI SDK をインストールします。

```bash
npm install @google/genai
```

環境変数ファイルを作成し、API キーを設定します。

```bash
# .env.local
GEMINI_API_KEY=your_api_key_here
```

<Callout type="warning">
API キーはクライアントサイドに絶対に露出させないでください。必ずサーバーサイド（Route Handlers や Server Actions）から Gemini API を呼び出します。
</Callout>

## アーキテクチャの設計

このアプリのアーキテクチャは以下のようになります。

```
ブラウザ (React)
  ↓ POST /api/chat （メッセージ送信）
Next.js Route Handler (サーバーサイド)
  ↓ generateContentStream()
Gemini API（ストリーミングレスポンス）
  ↓ ReadableStream
ブラウザ（リアルタイム表示）
```

ポイントは 3 つあります。

1. **API キーの保護**: Route Handler でサーバーサイドのみ API キーを使用
2. **ストリーミング**: `ReadableStream` でレスポンスをリアルタイム配信
3. **会話履歴の管理**: クライアントで状態管理し、毎回のリクエストに含める

## Route Handler の実装

まず、Gemini API と通信するバックエンドを作ります。

```typescript
// src/app/api/chat/route.ts
import { GoogleGenAI } from "@google/genai";
import { NextRequest } from "next/server";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

export async function POST(request: NextRequest) {
  const { messages } = await request.json();

  // Gemini API 用にメッセージ形式を変換
  const contents = messages.map(
    (msg: { role: string; content: string }) => ({
      role: msg.role === "assistant" ? "model" : "user",
      parts: [{ text: msg.content }],
    })
  );

  // ストリーミングでレスポンスを生成
  const response = await ai.models.generateContentStream({
    model: "gemini-3-flash",
    contents,
    config: {
      systemInstruction:
        "あなたは親切で知識豊富なAIアシスタントです。" +
        "日本語で簡潔かつ正確に回答してください。",
      temperature: 0.7,
      maxOutputTokens: 2048,
    },
  });

  // ReadableStream に変換してクライアントに返す
  const stream = new ReadableStream({
    async start(controller) {
      const encoder = new TextEncoder();
      try {
        for await (const chunk of response) {
          const text = chunk.text ?? "";
          if (text) {
            controller.enqueue(encoder.encode(text));
          }
        }
      } catch (error) {
        console.error("Streaming error:", error);
        controller.error(error);
      } finally {
        controller.close();
      }
    },
  });

  return new Response(stream, {
    headers: {
      "Content-Type": "text/plain; charset=utf-8",
      "Transfer-Encoding": "chunked",
    },
  });
}
```

このコードのポイントを解説します。

- `generateContentStream` を使うことで、Gemini がトークンを生成するたびにチャンクが返ってきます
- `ReadableStream` でそのチャンクをそのままクライアントに流します
- `role` の変換に注意してください。Gemini API では AI の応答は `"model"` ですが、フロントエンドでは `"assistant"` と表現するのが一般的です

## チャット UI コンポーネントの実装

次に、フロントエンドのチャット画面を作ります。

```typescript
// src/app/components/ChatWindow.tsx
"use client";

import { useState, useRef, useEffect, FormEvent } from "react";

type Message = {
  role: "user" | "assistant";
  content: string;
};

export default function ChatWindow() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState("");
  const [isLoading, setIsLoading] = useState(false);
  const messagesEndRef = useRef<HTMLDivElement>(null);

  // 新しいメッセージが追加されたら自動スクロール
  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [messages]);

  async function handleSubmit(e: FormEvent) {
    e.preventDefault();
    if (!input.trim() || isLoading) return;

    const userMessage: Message = { role: "user", content: input.trim() };
    const updatedMessages = [...messages, userMessage];
    setMessages(updatedMessages);
    setInput("");
    setIsLoading(true);

    try {
      const response = await fetch("/api/chat", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ messages: updatedMessages }),
      });

      if (!response.ok) throw new Error("API request failed");
      if (!response.body) throw new Error("No response body");

      // ストリーミングレスポンスを読み取る
      const reader = response.body.getReader();
      const decoder = new TextDecoder();
      let assistantContent = "";

      // アシスタントのメッセージを追加（空の状態で）
      setMessages((prev) => [...prev, { role: "assistant", content: "" }]);

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        const text = decoder.decode(value, { stream: true });
        assistantContent += text;

        // メッセージをリアルタイム更新
        setMessages((prev) => {
          const newMessages = [...prev];
          newMessages[newMessages.length - 1] = {
            role: "assistant",
            content: assistantContent,
          };
          return newMessages;
        });
      }
    } catch (error) {
      console.error("Chat error:", error);
      setMessages((prev) => [
        ...prev,
        {
          role: "assistant",
          content: "エラーが発生しました。もう一度お試しください。",
        },
      ]);
    } finally {
      setIsLoading(false);
    }
  }

  return (
    <div className="flex flex-col h-screen max-w-3xl mx-auto">
      {/* メッセージ一覧 */}
      <div className="flex-1 overflow-y-auto p-4 space-y-4">
        {messages.length === 0 && (
          <div className="text-center text-gray-400 mt-20">
            <p className="text-2xl mb-2">✨</p>
            <p>メッセージを入力して会話を始めましょう</p>
          </div>
        )}
        {messages.map((msg, i) => (
          <div
            key={i}
            className={`flex ${
              msg.role === "user" ? "justify-end" : "justify-start"
            }`}
          >
            <div
              className={`max-w-[80%] rounded-2xl px-4 py-3 ${
                msg.role === "user"
                  ? "bg-blue-600 text-white"
                  : "bg-gray-100 text-gray-800"
              }`}
            >
              <p className="whitespace-pre-wrap">{msg.content}</p>
            </div>
          </div>
        ))}
        <div ref={messagesEndRef} />
      </div>

      {/* 入力フォーム */}
      <form
        onSubmit={handleSubmit}
        className="border-t p-4 flex gap-2"
      >
        <input
          type="text"
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="メッセージを入力..."
          className="flex-1 border rounded-xl px-4 py-3 focus:outline-none focus:ring-2 focus:ring-blue-500"
          disabled={isLoading}
        />
        <button
          type="submit"
          disabled={isLoading || !input.trim()}
          className="bg-blue-600 text-white px-6 py-3 rounded-xl hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed transition-colors"
        >
          {isLoading ? "生成中..." : "送信"}
        </button>
      </form>
    </div>
  );
}
```

## マルチターン会話の仕組み

このアプリでは、`messages` 配列をそのまま Gemini API に送信しているため、会話の文脈が維持されます。たとえば以下のような会話が可能です。

```
ユーザー: TypeScript の interface と type の違いは？
Gemini: interface は拡張が可能で、type は...（詳しい説明）

ユーザー: 具体的なコード例を見せて
Gemini: 先ほどの説明を踏まえて、以下のコード例を紹介します...
```

Gemini は前のやり取りを踏まえて応答するため、自然な会話体験を実現できます。

## 本番環境に向けた改善ポイント

基本形ができたところで、本番環境で使うための改善点を紹介します。

### レート制限の実装

API キーの不正利用を防ぐため、レート制限を追加します。

```typescript
// src/lib/rate-limit.ts
const rateLimitMap = new Map<string, { count: number; resetTime: number }>();

export function checkRateLimit(
  ip: string,
  maxRequests = 20,
  windowMs = 60000
): boolean {
  const now = Date.now();
  const entry = rateLimitMap.get(ip);

  if (!entry || now > entry.resetTime) {
    rateLimitMap.set(ip, { count: 1, resetTime: now + windowMs });
    return true;
  }

  if (entry.count >= maxRequests) {
    return false;
  }

  entry.count++;
  return true;
}
```

Route Handler でこの関数を使用します。

```typescript
// src/app/api/chat/route.ts（追加部分）
import { checkRateLimit } from "@/lib/rate-limit";

export async function POST(request: NextRequest) {
  const ip = request.headers.get("x-forwarded-for") ?? "unknown";

  if (!checkRateLimit(ip)) {
    return new Response("Too many requests", { status: 429 });
  }

  // ...既存のコード
}
```

### エラーハンドリングの強化

Gemini API はモデルの安全フィルターによってレスポンスがブロックされることがあります。

```typescript
// レスポンスのフィニッシュ理由を確認する例
for await (const chunk of response) {
  if (chunk.candidates?.[0]?.finishReason === "SAFETY") {
    controller.enqueue(
      encoder.encode("安全上の理由により、この質問には回答できません。")
    );
    break;
  }
  const text = chunk.text ?? "";
  if (text) {
    controller.enqueue(encoder.encode(text));
  }
}
```

## よくある質問（FAQ）

### Q1: Gemini 3 Flash と Gemini 3 Pro、どちらを使うべきですか？

チャットアプリには **Gemini 3 Flash** がおすすめです。応答速度が速く、コストも低いため、リアルタイムのチャット体験に最適です。複雑な推論や分析が必要な場合は Gemini 3 Pro を検討してください。モデル選びについて詳しくは[モデル選定ガイド](/articles/gemini-basics/gemini-model-selection-guide)を参照してください。

### Q2: API キーの無料枠でどのくらい使えますか？

無料枠では 1 分あたり 15 リクエスト、1 日あたり 1,500 リクエストが利用可能です。個人開発やプロトタイプには十分ですが、本番運用では従量課金プランへの移行をおすすめします。コスト管理については[API コスト最適化ガイド](/articles/gemini-api/gemini-api-cost-optimization)が参考になります。

### Q3: 画像やファイルのアップロードにも対応できますか？

はい。Gemini API はマルチモーダル入力に対応しているため、画像やPDFをアップロードして分析することも可能です。`parts` 配列に `inlineData` フィールドを追加するだけで実装できます。詳しくは[マルチモーダル API ガイド](/articles/gemini-api/multimodal-api-guide)を参照してください。

### Q4: 会話履歴をデータベースに保存するにはどうすればよいですか？

Firebase Firestore や Supabase などのデータベースを使えば簡単に保存できます。ユーザー認証と組み合わせることで、過去の会話を呼び出す機能も実装できます。Firebase との連携方法は[Firebase 統合ガイド](/articles/gemini-dev/gemini-firebase-integration-guide)が参考になります。

### Q5: Server Actions を使う方法もありますか？

はい。Next.js の Server Actions を使えば、Route Handler を別途作成せずにサーバーサイドの処理を実装できます。ただし、ストリーミングの実装はやや複雑になるため、チャットアプリには Route Handler の方がシンプルです。

このチュートリアルでは、Gemini API と Next.js を使って以下の内容を実装しました。
