## Function Callingが変える収益自動化の仕組み

Gemini APIのFunction Calling機能は、AIモデルが外部ツールやAPIを自律的に呼び出せる仕組みです。これを収益化の文脈で使うと、「ユーザーの行動データを分析し、最適なタイミングでアップセル提案を行い、課金処理まで実行する」という一連のフローをGeminiが自律的に判断・実行できるようになります。

従来のルールベースの自動化（「7日後にクーポンメールを送る」など）と異なり、Function Callingを活用したAIエージェントは、ユーザーごとの行動パターンを動的に判断し、最適なアクションを選択します。この記事では、この仕組みを一から構築する方法を紹介します。

## 収益自動化システムのアーキテクチャ

システム全体は3つの自動化ループで構成されます。

### ループ1: コンテンツ → トラフィック → 集客

Geminiがキーワードトレンドを分析し、SEOに最適化されたコンテンツを生成。公開後のPV・滞在時間・直帰率をモニタリングし、次のコンテンツ戦略にフィードバックします。

### ループ2: 集客 → 課金 → 収益

無料コンテンツから有料プランへの転換を最適化。ユーザーのエンゲージメントスコアに基づき、Function Callingで適切なタイミングにStripeのクーポン発行やプラン提案を実行します。

### ループ3: 収益 → 分析 → 改善

日次で収益データを分析し、解約リスクの高いユーザーへの施策やプライシングの最適化を提案。人間のレビューを経て改善を適用します。

### エージェントのメインループ

```python
# revenue_agent.py
# Function Callingベースの収益自動化エージェント
import google.generativeai as genai
from tools import revenue_tools
import json

genai.configure(api_key="YOUR_GEMINI_API_KEY")
model = genai.GenerativeModel(
    "gemini-2.5-pro",
    tools=revenue_tools
)

# ツールの実行ハンドラ
def execute_tool(function_call):
    """Function Callingで返されたツール呼び出しを実行"""
    name = function_call.name
    args = dict(function_call.args)

    if name == "check_user_engagement":
        # 実際のDBからユーザーデータを取得
        return {
            "user_id": args["user_id"],
            "engagement_score": 72,
            "articles_read_30d": 15,
            "current_plan": "free",
            "days_since_signup": 14,
            "premium_articles_attempted": 3
        }

    elif name == "create_stripe_coupon":
        # Stripe APIでクーポンを作成
        import stripe
        coupon = stripe.Coupon.create(
            percent_off=args["discount_percent"],
            duration=args.get("duration", "once"),
            max_redemptions=1,
            metadata={"user_id": args["user_id"]}
        )
        return {"coupon_id": coupon.id, "status": "created"}

    elif name == "send_personalized_email":
        # メール送信（実際にはSendGrid等を使用）
        print(f"📧 Sending {args['template']} email to {args['user_id']}")
        return {"status": "sent", "template": args["template"]}

    elif name == "get_revenue_metrics":
        return {
            "period": args["period"],
            "total_revenue": 45800,
            "new_subscribers": 12,
            "churned": 2,
            "mrr": 38400,
            "conversion_rate": 0.034
        }


async def run_revenue_agent():
    """収益エージェントのメインループ"""
    chat = model.start_chat()

    # エージェントへの初期指示
    response = chat.send_message(
        """あなたは収益最適化エージェントです。
以下のタスクを順番に実行してください:

1. 今月の収益メトリクスを確認
2. 無料ユーザーの中からアップセル候補を特定
3. 候補ユーザーのエンゲージメントを確認
4. 適切な施策を実行（クーポン発行やメール送信）

各ステップで必要なツールを呼び出してください。"""
    )

    # Function Callingのループ
    while response.candidates[0].content.parts:
        part = response.candidates[0].content.parts[0]

        if hasattr(part, "function_call") and part.function_call:
            # ツールの実行
            result = execute_tool(part.function_call)
            print(f"🔧 Tool: {part.function_call.name}")
            print(f"   Result: {json.dumps(result, ensure_ascii=False)}")

            # 結果をGeminiに返してループを続行
            response = chat.send_message(
                genai.protos.Content(
                    parts=[genai.protos.Part(
                        function_response=genai.protos.FunctionResponse(
                            name=part.function_call.name,
                            response={"result": result}
                        )
                    )]
                )
            )
        else:
            # テキスト応答（最終結果）
            print(f"\n📊 Agent Report:\n{part.text}")
            break

# 実行
import asyncio
asyncio.run(run_revenue_agent())

# 期待する出力例:
# 🔧 Tool: get_revenue_metrics
#    Result: {"period": "month", "total_revenue": 45800, ...}
# 🔧 Tool: check_user_engagement
#    Result: {"user_id": "user_123", "engagement_score": 72, ...}
# 🔧 Tool: create_stripe_coupon
#    Result: {"coupon_id": "cpn_xxx", "status": "created"}
# 🔧 Tool: send_personalized_email
#    Result: {"status": "sent", "template": "upsell_pro"}
#
# 📊 Agent Report:
# 今月の収益は¥45,800、MRR ¥38,400です。
# user_123はエンゲージメントスコア72で、プレミアム記事を3回
# 閲覧試行しています。20%割引クーポンを発行し、
# Proプランへのアップセルメールを送信しました。
```

## セキュリティとガバナンスの考慮事項

課金処理を自動化する場合、セキュリティは特に重要です。

NemoClawのOpenShellを併用することで、Function CallingエージェントがアクセスできるAPIエンドポイントを厳密に制限できます。たとえば「Stripeのクーポン作成と顧客参照は許可するが、プランの削除や払い戻しは禁止する」といった細かなポリシー設定が可能です。

また、全てのFunction Call呼び出しは監査ログに記録されるため、「いつ、どのツールが、どのパラメータで呼び出されたか」を事後に追跡できます。金銭が関わる自動化では、この透明性が不可欠です。

## よくある質問（FAQ）

### Q: Function Callingのレスポンス時間は収益システムに影響しますか？

A: Gemini 2.5 ProのFunction Callingレスポンスは通常1〜3秒です。リアルタイム決済には向きませんが、バッチ処理（日次分析、週次レポート、定期的なアップセル判定）には十分な速度です。即座の決済処理自体はStripeのWebhookで直接ハンドリングし、AIエージェントは戦略的な判断に集中させる設計が効果的です。

### Q: Function Callingで誤った課金操作が実行されるリスクはありますか？

A: リスクはゼロではありません。そのため、本番環境では「AIが判断 → 人間が承認 → 実行」という承認フローを組み込むことを推奨します。高額なクーポンの発行やプラン変更には承認ステップを必須とし、低リスクな操作（メール送信やレポート生成）のみを完全自動化する段階的アプローチが現実的です。

### Q: Gemini APIの料金とFunction Callingのコストはどれくらいですか？

A: Function CallingはGemini APIの通常の入出力トークンとして課金されます。追加の課金はありません。ツール定義のトークンは入力に含まれるため、多数のツールを定義する場合はトークン効率を考慮してください。1日1回の分析＋週数回のアップセル判定で、月間$20〜40程度が目安です。

### Q: 複数のサービス（Stripe, SendGrid, Analytics）を同時にFunction Callingで制御できますか？

A: はい、Geminiは1回のセッションで複数のツールを順序立てて呼び出せます。先にAnalyticsでデータを取得し、その結果に基づいてStripeでクーポンを作成し、最後にSendGridでメールを送る——という複数ステップのワークフローを自律的に実行できます。

## まとめ — Function Callingは「考えて動く」収益システムの核

Gemini APIのFunction Callingは、単なるAPI呼び出しの自動化ではなく、「状況を判断し、最適なツールを選び、実行する」知的な自動化を実現する仕組みです。収益化の文脈では、ルールベースの自動化では対応しきれない、ユーザーごとの最適な施策判断をAIに委ねることができます。

大切なのは、完全な自律化を目指すのではなく、人間の判断が必要なポイントを明確にした上で、AIが最も価値を発揮する領域を見極めることです。まずは分析レポートの自動生成から始めて、徐々に範囲を広げていくのがおすすめです。
