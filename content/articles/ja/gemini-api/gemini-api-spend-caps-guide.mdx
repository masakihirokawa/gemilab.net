2026年3月16日、Googleは**Gemini APIにスペンドキャップ（支出上限）機能**を正式に導入しました。これにより、開発者はプロジェクトごとに月間API使用料の上限を設定できるようになり、予期せぬ高額請求のリスクを大幅に軽減できます。

個人開発者や副業でAIアプリを開発している方にとって、APIコストの予算管理は非常に重要な課題です。この記事では、スペンドキャップの設定方法から実践的な活用術まで、詳しく解説します。

## スペンドキャップとは何か

スペンドキャップとは、Gemini APIのプロジェクトに対して設定できる**月間最大支出額の上限**です。設定した上限に達すると、そのプロジェクトからのAPIリクエストがブロックされ、それ以上の課金が発生しなくなります。

### なぜスペンドキャップが必要だったのか

2025年夏、一部の開発者がGemini APIの課金バグにより予期せず高額な請求を受けるという問題が発生しました。この出来事は、AI API利用における費用管理の重要性を多くの開発者に再認識させました。

今回導入されたスペンドキャップは、その教訓をもとに実装された**セーフティネット機能**です。個人開発から小規模チームまで、誰でも安心してGemini APIを活用できる環境が整いました。

### スペンドキャップの仕組み

スペンドキャップは以下の2つのレベルで機能します：

- **プロジェクトレベル**：個別プロジェクトごとに月額上限を設定
- **請求アカウントレベル**：Tierシステムに基づく上限（2026年4月1日より強制適用開始）

重要な注意点として、スペンドキャップには**約10分の適用遅延**があります。上限に達した後も、この遅延期間中に発生した超過分は請求されるため、余裕を持った上限設定が推奨されます。

## Google AI Studioでのスペンドキャップ設定方法

スペンドキャップの設定はGoogle AI Studioから簡単に行えます。以下の手順に従ってください。

### 前提条件

- Google AI Studioのアカウント
- 対象プロジェクトの**プロジェクト編集者、オーナー、または管理者**ロール
- Gemini APIの有料プラン（Tier 1以上）

### 設定手順

**1. Google AI Studioにアクセス**

[Google AI Studio](https://aistudio.google.com/)にアクセスしてサインインします。

**2. プロジェクトを選択**

画面上部のプロジェクト選択ドロップダウンから、スペンドキャップを設定したいプロジェクトを選択します。

**3. Spendページに移動**

左サイドバーの「Spend」メニューをクリックします。現在の使用量と設定可能な上限が表示されます。

**4. スペンドキャップを設定**

「Monthly spend cap」セクションの「Edit spend cap」ボタンをクリックし、月額上限（ドル単位）を入力して保存します。

```
設定例：
- 個人開発の実験用: $10/月
- 副業のプロトタイプ: $50/月
- 小規模本番アプリ: $200/月
```

設定は保存後すぐに有効になり、変更・無効化するまで継続して適用されます。

## Pythonで現在の使用量をモニタリングする

スペンドキャップと組み合わせて、コードレベルでAPIの使用量を監視することが推奨されます。

```python
import google.generativeai as genai
import os

# APIキーの設定
genai.configure(api_key=os.environ["GEMINI_API_KEY"])

# モデルの初期化
model = genai.GenerativeModel("gemini-3-flash-preview")

def generate_with_usage_tracking(prompt: str) -> dict:
    """
    プロンプトを送信し、トークン使用量を記録する関数

    Args:
        prompt: 送信するテキストプロンプト

    Returns:
        dict: レスポンスとトークン使用量の情報
    """
    response = model.generate_content(
        prompt,
        generation_config=genai.GenerationConfig(
            max_output_tokens=1024,  # 出力トークンを制限してコスト管理
        )
    )

    # 使用量の取得
    usage = response.usage_metadata

    result = {
        "text": response.text,
        "prompt_tokens": usage.prompt_token_count,
        "output_tokens": usage.candidates_token_count,
        "total_tokens": usage.total_token_count,
    }

    # コスト推算（gemini-3-flash-previewの概算価格）
    # 入力: $0.075/1M tokens, 出力: $0.30/1M tokens
    input_cost = (result["prompt_tokens"] / 1_000_000) * 0.075
    output_cost = (result["output_tokens"] / 1_000_000) * 0.30
    result["estimated_cost_usd"] = input_cost + output_cost

    return result

# 使用例
result = generate_with_usage_tracking("Gemini APIの主な特徴を3つ教えてください。")

print(f"レスポンス: {result['text'][:100]}...")
print(f"使用トークン: {result['total_tokens']:,}")
print(f"推算コスト: ${result['estimated_cost_usd']:.6f}")
# 出力例:
# レスポンス: Gemini APIの主な特徴は以下の3つです：1. マルチモーダル対応...
# 使用トークン: 312
# 推算コスト: $0.000094
```

このように、各APIコールの使用量をトラッキングすることで、スペンドキャップに到達する前に使用量を把握できます。

## 予算管理のベストプラクティス

スペンドキャップを最大限に活用するための実践的なアドバイスを紹介します。

### プロジェクト分離戦略

複数のプロジェクトを管理している場合は、**用途ごとにGCPプロジェクトを分離**することを強くお勧めします。

```
プロジェクト構成例:
- gemilab-dev (開発・実験用): $20/月の上限
- gemilab-staging (ステージング): $50/月の上限
- gemilab-prod (本番): $500/月の上限
```

これにより、開発中の実験コードが誤って本番APIキーを使い過ぎることを防げます。

### 段階的な上限設定

新しいプロジェクトを始める際は、低い上限から始めて徐々に増やす**段階的アプローチ**が有効です。

1. **フェーズ1（プロトタイプ）**: $10/月 — 基本機能の検証
2. **フェーズ2（ベータ版）**: $50/月 — 実際のユーザーテスト
3. **フェーズ3（本番移行）**: 実績データに基づいて適切な上限を設定

### アラートとモニタリングの組み合わせ

GCPの**Cloud Monitoringと予算アラート**を合わせて使うことで、より細かい管理が可能です。

```python
# Cloud Monitoring APIを使った使用量チェックの実装例
from google.cloud import monitoring_v3
from datetime import datetime, timedelta, timezone

def check_api_usage_percentage(project_id: str, spend_cap_usd: float) -> float:
    """
    現在の月間使用量がスペンドキャップの何%かを返す

    Args:
        project_id: GCPプロジェクトID
        spend_cap_usd: 設定済みスペンドキャップ（ドル）

    Returns:
        float: スペンドキャップに対する使用率（0.0〜1.0）
    """
    client = monitoring_v3.MetricServiceClient()

    now = datetime.now(timezone.utc)
    start_of_month = now.replace(day=1, hour=0, minute=0, second=0)

    interval = monitoring_v3.TimeInterval(
        end_time=now,
        start_time=start_of_month,
    )

    # 月初からの累計コストを取得
    results = client.list_time_series(
        request={
            "name": f"projects/{project_id}",
            "filter": 'metric.type="serviceruntime.googleapis.com/billing/consumed_billable_resource_count"',
            "interval": interval,
            "view": monitoring_v3.ListTimeSeriesRequest.TimeSeriesView.FULL,
        }
    )

    # 使用率を計算（簡略化した例）
    total_usage = sum(point.value.double_value for series in results for point in series.points)
    usage_ratio = total_usage / spend_cap_usd if spend_cap_usd > 0 else 0.0

    return min(usage_ratio, 1.0)  # 最大1.0（100%）を返す

# 使用例
ratio = check_api_usage_percentage("my-project-id", 50.0)
print(f"スペンドキャップ使用率: {ratio * 100:.1f}%")
# 出力例: スペンドキャップ使用率: 34.2%

if ratio > 0.8:
    print("⚠️ 警告: スペンドキャップの80%を超えました！")
```

## Tier制度との関係

2026年4月1日から、**Tier制度によるアカウントレベルの支出上限**も強制適用が開始されます。スペンドキャップとTier制度は独立して機能します。

| Tier | 月額上限（目安） | 対象 |
|------|----------------|------|
| Tier 1 | 低コスト | 個人・小規模開発 |
| Tier 2 | 中コスト | 成長フェーズのスタートアップ |
| Tier 3 | 高コスト | 本番規模の商用サービス |

Tierのアップグレードは、実績に応じてGoogleに申請することで可能です。

## よくある質問（FAQ）

### Q1. スペンドキャップに達したらどうなりますか？

スペンドキャップに達すると、そのプロジェクトからのGemini APIリクエストが一時的にブロックされます。約10分の遅延があるため、その間の超過分は課金される場合があります。アプリが止まるのを防ぐために、Google Cloud Monitoringのアラート設定を合わせて行うことをお勧めします。

### Q2. スペンドキャップはいつでも変更できますか？

はい、プロジェクト編集者以上の権限があれば、Google AI StudioのSpendページからいつでも上限を変更または無効化できます。変更は即時反映されます。ただし、上限を上げる際は慎重に判断してください。

### Q3. 無料枠（Free Tier）にもスペンドキャップは適用されますか？

無料枠の範囲内での利用には課金が発生しないため、スペンドキャップの設定は不要です。スペンドキャップは**有料利用（Tier 1以上）**に対して機能します。無料枠の詳細については、[Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart)を参照してください。

### Q4. 複数のプロジェクトで合計の上限を設定できますか？

現在のスペンドキャップはプロジェクト単位での設定です。複数プロジェクトの合計を管理するには、GCPの**請求アカウントレベルの予算アラート**を組み合わせることをお勧めします。

### Q5. APIキーが漏洩した場合、スペンドキャップは保護になりますか？

スペンドキャップはコスト上限として機能するため、APIキーが漏洩した場合の被害を限定する効果があります。ただし、根本的な対策としてAPIキーの即時ローテーションと、可能であればIPアドレス制限の設定も合わせて行ってください。
