## コンテキストキャッシュとは

Gemini API のコンテキストキャッシュ（Context Caching）は、大きなプロンプトやドキュメントを一度アップロードしてキャッシュし、**後続のリクエストでそのキャッシュを再利用**する機能です。

同じ資料に対して複数の質問をする場合や、長大なコードベースを何度も参照するエージェントに特に有効です。

## なぜコストが削減されるのか

通常の Gemini API リクエストでは、毎回プロンプト全体のトークンに課金されます。

例えば 100,000 トークンのドキュメントに 10 回質問すると:
- **キャッシュなし**: 100,000 × 10 = 100万トークン分の入力コスト
- **キャッシュあり**: 100,000（初回）+ 10 × (小さな質問のみ) = 大幅削減

キャッシュされたトークンは通常の入力トークンより低いレートで課金されるため、実質的なコスト削減効果は最大 90% 以上になることもあります。

## セットアップ手順

### 1. ドキュメントをキャッシュとして作成

```python
import google.generativeai as genai
from google.generativeai.types import content_types

genai.configure(api_key="YOUR_API_KEY")

# キャッシュするコンテンツを準備
large_document = open("technical_manual.txt").read()

# キャッシュを作成（TTL: 30分）
cache = genai.caching.CachedContent.create(
    model="gemini-3-flash-preview",
    contents=[{"role": "user", "parts": [{"text": large_document}]}],
    system_instruction="あなたは技術マニュアルの専門家です。",
    ttl="1800s"  # 30分間キャッシュ
)

print(f"Cache created: {cache.name}")
```

### 2. キャッシュを使ってモデルに質問

```python
model = genai.GenerativeModel.from_cached_content(cache)

# キャッシュを再利用した質問 — 大きなドキュメント分のトークンは課金されない
response1 = model.generate_content("第3章の要点を日本語で要約してください")
response2 = model.generate_content("インストール手順のステップを箇条書きで教えてください")
response3 = model.generate_content("トラブルシューティングセクションの内容は？")
```

### 3. キャッシュの一覧・削除

```python
# キャッシュ一覧
for c in genai.caching.CachedContent.list():
    print(f"{c.name}: expires {c.expire_time}")

# 手動削除
cache.delete()
```

## 適切なユースケース

コンテキストキャッシュが特に効果的なシーンは以下の通りです。

**ドキュメントQA**: 長大なPDF・仕様書に対して繰り返し質問するチャットボット

**コードベース分析**: 大規模なリポジトリを読み込んで複数の分析を実行するエージェント

**法律・医療文書**: 長い契約書や論文を一度キャッシュし、複数の観点から分析

**データ処理パイプライン**: 同じプロンプトテンプレートで大量データを逐次処理

## コスト試算の例

Gemini 3 Flash で 500,000 トークンのドキュメントを 50 回参照する場合:

| 方法 | 入力トークン合計 | 概算コスト |
|------|-----------------|-----------|
| キャッシュなし | 25,000,000 | $12.50 |
| キャッシュあり | 500,000 + 少量 | ~$1.00 |

実際には質問部分のトークンも加算されますが、大きなドキュメントが支配的な場合は劇的な削減効果があります。

## TTL（有効期限）の設定

キャッシュには TTL（Time-To-Live）が必要です。最小 1 分、最大 1 時間で設定できます。

- **短期タスク**: 5〜10分で十分
- **長時間処理**: 最大 1 時間
- **定期的な再利用**: キャッシュを再作成するスクリプトを用意

TTL 切れ後はキャッシュが自動削除されます。有効期限の延長も API で可能です。

## まとめ

コンテキストキャッシュは Gemini API でコスト効率を高めるための強力な機能です。

- 同じドキュメントへの繰り返しアクセスでコストを大幅削減
- 実装はシンプルで既存コードへの組み込みも容易
- 長大なプロンプト・資料を扱う用途で特に効果的

大量処理や長文分析のワークロードには、ぜひコンテキストキャッシュの導入を検討してみてください。
