## なぜ JSON Mode が必要か

LLM からのテキスト出力をパースしようとして、フォーマットが崩れてエラーになった経験はありませんか？

Gemini API の **JSON Mode** と **構造化出力（Structured Output）** を使えば、モデルが必ず指定したスキーマに従った JSON を返すよう強制できます。バックエンド連携やデータ処理パイプラインでの信頼性が大幅に向上します。

## JSON Mode の基本

最もシンプルな方法は、`response_mime_type` に `application/json` を指定する方法です。

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-flash-preview")

response = model.generate_content(
    "次の商品情報をJSONで整理してください: iPhone 15 Pro、価格199,800円、色ブラック、ストレージ256GB",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json"
    )
)

import json
data = json.loads(response.text)
print(data)
# {"name": "iPhone 15 Pro", "price": 199800, "color": "ブラック", "storage": "256GB"}
```

## スキーマを指定した構造化出力

より厳密な出力制御には、`response_schema` でスキーマを指定します。

```python
import typing_extensions as typing

class Product(typing.TypedDict):
    name: str
    price: int
    color: str
    storage: str
    available: bool

response = model.generate_content(
    "次の商品情報をJSONで整理してください: iPhone 15 Pro、価格199,800円、色ブラック、ストレージ256GB、在庫あり",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=Product
    )
)

data = json.loads(response.text)
# 必ず Product の型に従った JSON が返る
```

## 配列の出力

複数アイテムのリストを返す場合は、配列スキーマを使います。

```python
class Review(typing.TypedDict):
    reviewer: str
    rating: int
    comment: str
    date: str

response = model.generate_content(
    """次のレビューテキストから各レビューを抽出してください:
    山田さん（5点）: 非常に使いやすい、2026/3/10
    田中さん（3点）: 普通です、2026/3/12""",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=list[Review]
    )
)

reviews = json.loads(response.text)
for r in reviews:
    print(f"{r['reviewer']}: {r['rating']}点 - {r['comment']}")
```

## ネストされた構造

複雑なスキーマにも対応しています。

```python
class Address(typing.TypedDict):
    street: str
    city: str
    postal_code: str

class Customer(typing.TypedDict):
    id: int
    name: str
    email: str
    address: Address
    is_premium: bool

response = model.generate_content(
    "次の顧客情報を構造化してください: ID 1001, 鈴木一郎, suzuki@example.com, 東京都渋谷区1-1-1, 〒150-0001, プレミアム会員",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=Customer
    )
)
```

## Enum の活用

カテゴリの分類など、特定の値のみを許容したい場合は `enum` を使います。

```python
import enum

class Sentiment(enum.Enum):
    POSITIVE = "positive"
    NEGATIVE = "negative"
    NEUTRAL = "neutral"

class SentimentResult(typing.TypedDict):
    text: str
    sentiment: Sentiment
    confidence: float

response = model.generate_content(
    "次のレビューのセンチメントを分析してください: 「とても満足しています！また使いたい」",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=SentimentResult
    )
)
```

## よくある使いどころ

- **情報抽出**: メール・文書からの構造化データ抽出
- **分類タスク**: テキストのカテゴリ分類
- **データ変換**: 非構造化テキスト → データベース挿入用JSON
- **API レスポンス生成**: バックエンドAPIとのシームレスな連携
- **フォームデータ解析**: 自由記述の回答を構造化

## まとめ

Gemini API の JSON Mode と構造化出力を使えば、LLM 出力の予測可能性を大幅に高められます。

- `response_mime_type: "application/json"` だけで基本的なJSON出力が可能
- `response_schema` でより厳密なスキーマ制御が可能
- Python の TypedDict・Enum と組み合わせて型安全な出力を実現

バックエンドやデータパイプラインとの連携では、積極的に構造化出力を活用しましょう。
