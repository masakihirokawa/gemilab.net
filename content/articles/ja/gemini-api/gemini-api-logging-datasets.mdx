## APIの使い方を可視化する新ツール

2026年3月、GoogleはGemini API向けの新しいログ＆データセットツールをリリースしました。このツールにより、APIリクエストの記録・分析・データセット管理がGoogle AI Studioのダッシュボードから直接行えるようになります。

従来は開発者が自前でログ基盤を構築する必要がありましたが、この機能によりAPIの利用状況を標準機能として把握できるようになりました。

## 主な機能

### リクエストログの自動記録

API呼び出しのリクエストとレスポンスを自動的に記録します。記録される情報は以下の通りです。

- 入力プロンプト（テキスト・画像・動画）
- 出力レスポンス
- 使用トークン数
- レイテンシ
- モデル名・バージョン
- エラー情報

### データセットの作成と管理

ログからデータセットを作成し、モデル評価やファインチューニングに活用できます。

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

# ログからデータセットを作成
dataset = genai.datasets.create(
    name="customer-support-logs",
    description="カスタマーサポートのAPI呼び出しログ",
    source="logs",
    filter={
        "date_range": {"start": "2026-03-01", "end": "2026-03-12"},
        "model": "gemini-2.5-pro"
    }
)
```

### フィルタリングと検索

大量のログから特定のリクエストを素早く見つけるための検索機能が用意されています。

- 日付範囲でのフィルタ
- モデルバージョンでのフィルタ
- レスポンス品質スコアでのフィルタ
- テキスト検索（プロンプト内容での検索）

## 活用シーン

### プロンプト品質の改善

ログを分析することで、どのプロンプトが良い結果を出しているか、どのプロンプトが期待通りの結果を出せていないかを定量的に把握できます。

### コスト最適化

トークン使用量をリクエスト単位で確認できるため、コストの高いリクエストを特定し、プロンプトの短縮やモデルの使い分け（Pro vs Flash）による最適化が行えます。

### モデル評価

同じプロンプトセットを異なるモデルバージョンで実行し、出力品質を比較するA/Bテストが簡単に行えます。

```python
# データセットを使ったモデル比較
evaluation = genai.evaluations.run(
    dataset="customer-support-logs",
    models=["gemini-2.5-pro", "gemini-2.5-flash"],
    metrics=["quality", "latency", "cost"]
)

print(evaluation.summary())
```

### 異常検知

レスポンス品質の急激な変化やエラー率の上昇を検知し、問題の早期発見に役立てることができます。

## Google AI Studio での確認方法

1. [Google AI Studio](https://aistudio.google.com/) にアクセス
2. 左メニューから「Logs & Datasets」を選択
3. ダッシュボードでログの概要を確認
4. 必要に応じてフィルタを適用して詳細を確認

## プライバシーとセキュリティ

- ログデータはプロジェクトに紐づき、他のユーザーからはアクセスできません
- 機密データを含むログは暗号化されて保存されます
- ログの保持期間はデフォルトで30日（カスタマイズ可能）
- GDPRやSOC2準拠の環境で運用されます

## 注意点

- ログ機能は現在ベータ版として提供されています
- 大量のログが蓄積されると、ストレージコストが発生する場合があります
- リアルタイムのログストリーミングは今後のアップデートで対応予定
