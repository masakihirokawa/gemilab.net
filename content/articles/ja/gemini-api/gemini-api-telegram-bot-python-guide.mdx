## はじめに

Telegram は月間アクティブユーザー数が9億人を超える世界的なメッセージングプラットフォームです。その強力な Bot API と Gemini API の高度な AI 機能を組み合わせることで、テキスト応答だけでなく画像認識やドキュメント分析にも対応した、本格的な AI アシスタントボットを構築できます。

この記事では、Python の `python-telegram-bot` ライブラリと Google の Gemini API を使い、マルチモーダル対応の Telegram ボットを一から構築する方法を解説します。Discord や LINE、Slack 向けのボット構築経験がある方にとっても、Telegram ならではの機能（インラインモード、マークダウン対応、大容量ファイル送信）を活かした実装パターンは新たな発見があるはずです。

**この記事で学べること:**

- Telegram Bot API の基本設定と BotFather でのボット作成
- Gemini API を使ったテキスト会話・マルチターン対応の実装
- 画像送信に対する AI 分析機能の組み込み
- エラーハンドリングとレート制限への対処

## 前提知識・環境準備

### 必要なもの

- Python 3.10 以上
- Google AI Studio の API キー（[取得方法はこちら](/articles/gemini-api/gemini-api-quickstart)）
- Telegram アカウント

### Telegram Bot の作成

まず、Telegram で [@BotFather](https://t.me/BotFather) にメッセージを送り、ボットを作成します。

1. BotFather に `/newbot` と送信
2. ボットの表示名を入力（例: `My Gemini Assistant`）
3. ボットのユーザー名を入力（例: `my_gemini_assistant_bot`、末尾は `bot` で終わる必要あり）
4. 発行された **API トークン** を安全に保存

### パッケージのインストール

```bash
pip install python-telegram-bot google-genai python-dotenv
```

### 環境変数の設定

プロジェクトルートに `.env` ファイルを作成します。

```bash
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
GEMINI_API_KEY=your_gemini_api_key_here
```

## 基本的なテキスト応答ボットの構築

まずは、ユーザーのメッセージに Gemini API で応答するシンプルなボットを作りましょう。

```python
import os
import logging
from dotenv import load_dotenv
from google import genai
from telegram import Update
from telegram.ext import (
    ApplicationBuilder,
    CommandHandler,
    MessageHandler,
    ContextTypes,
    filters,
)

# 環境変数の読み込み
load_dotenv()

# ログ設定
logging.basicConfig(
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s",
    level=logging.INFO,
)
logger = logging.getLogger(__name__)

# Gemini クライアントの初期化
client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))

async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    /start コマンドのハンドラー
    """
    await update.message.reply_text(
        "こんにちは！Gemini AI アシスタントです。\n"
        "テキストや画像を送ってください。AI が分析・回答します。\n\n"
        "コマンド一覧:\n"
        "/start - ボットの説明\n"
        "/reset - 会話履歴をリセット"
    )

async def reset(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    /reset コマンドで会話履歴をクリア
    """
    context.user_data.pop("history", None)
    await update.message.reply_text("会話履歴をリセットしました。")

async def handle_message(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    テキストメッセージに Gemini API で応答
    """
    user_message = update.message.text

    # 会話履歴の管理（最大20ターン）
    history = context.user_data.get("history", [])
    history.append({"role": "user", "parts": [{"text": user_message}]})

    # 履歴が長くなりすぎないよう制限
    if len(history) > 40:
        history = history[-40:]

    try:
        # Gemini API 呼び出し
        response = client.models.generate_content(
            model="gemini-3-flash",
            contents=history,
            config={
                "system_instruction": (
                    "あなたは親切で知識豊富なAIアシスタントです。"
                    "日本語で質問された場合は日本語で、"
                    "英語で質問された場合は英語で回答してください。"
                    "回答は簡潔かつ正確に行ってください。"
                ),
                "max_output_tokens": 2048,
                "temperature": 0.7,
            },
        )

        reply = response.text

        # アシスタントの応答を履歴に追加
        history.append({"role": "model", "parts": [{"text": reply}]})
        context.user_data["history"] = history

        # Telegram のメッセージ長制限（4096文字）に対応
        if len(reply) > 4000:
            for i in range(0, len(reply), 4000):
                await update.message.reply_text(reply[i : i + 4000])
        else:
            await update.message.reply_text(reply)

    except Exception as e:
        logger.error(f"Gemini API error: {e}")
        await update.message.reply_text(
            "申し訳ありません。エラーが発生しました。しばらくしてからもう一度お試しください。"
        )

def main():
    """
    ボットを起動
    """
    app = (
        ApplicationBuilder()
        .token(os.getenv("TELEGRAM_BOT_TOKEN"))
        .build()
    )

    app.add_handler(CommandHandler("start", start))
    app.add_handler(CommandHandler("reset", reset))
    app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))

    logger.info("Bot is running...")
    app.run_polling()

if __name__ == "__main__":
    main()
```

実行すると、Telegram でボットにテキストを送るだけで Gemini API を通じた AI 応答が返ってきます。会話履歴も保持されるため、文脈を踏まえたマルチターン会話が可能です。

## マルチモーダル対応 — 画像分析機能の追加

Gemini API の強みであるマルチモーダル機能を活かし、ユーザーが送信した画像を AI で分析する機能を追加します。

```python
import io
import httpx

async def handle_photo(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    画像メッセージを受信して Gemini で分析
    """
    # 最も高解像度の画像を取得
    photo = update.message.photo[-1]
    file = await context.bot.get_file(photo.file_id)

    # 画像データをダウンロード
    async with httpx.AsyncClient() as http_client:
        response = await http_client.get(file.file_path)
        image_bytes = response.content

    # キャプション（ユーザーの質問）があれば使用
    caption = update.message.caption or "この画像について詳しく説明してください。"

    try:
        await update.message.reply_text("画像を分析中です...")

        # Gemini API にマルチモーダルリクエスト
        response = client.models.generate_content(
            model="gemini-3-flash",
            contents=[
                {
                    "role": "user",
                    "parts": [
                        {"text": caption},
                        {
                            "inline_data": {
                                "mime_type": "image/jpeg",
                                "data": __import__("base64").b64encode(
                                    image_bytes
                                ).decode(),
                            }
                        },
                    ],
                }
            ],
            config={
                "max_output_tokens": 2048,
                "temperature": 0.5,
            },
        )

        await update.message.reply_text(response.text)

    except Exception as e:
        logger.error(f"Image analysis error: {e}")
        await update.message.reply_text(
            "画像の分析中にエラーが発生しました。別の画像でお試しください。"
        )
```

`main()` 関数に以下のハンドラーを追加します。

```python
app.add_handler(MessageHandler(filters.PHOTO, handle_photo))
```

これで、ユーザーが画像を送信すると Gemini がその内容を分析して回答します。画像にキャプションを付けて送れば、「この料理のカロリーは？」「このエラーメッセージの解決方法は？」といった具体的な質問にも対応できます。

## ドキュメント分析機能の追加

PDF やテキストファイルを送信して AI に分析してもらう機能も実装できます。

```python
async def handle_document(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    ドキュメント（PDF・テキスト）を受信して Gemini で分析
    """
    document = update.message.document

    # サポートする MIME タイプ
    supported_types = {
        "application/pdf": "application/pdf",
        "text/plain": "text/plain",
        "text/csv": "text/csv",
    }

    mime_type = document.mime_type
    if mime_type not in supported_types:
        await update.message.reply_text(
            "対応していないファイル形式です。PDF、テキスト、CSV ファイルを送信してください。"
        )
        return

    # ファイルサイズチェック（20MB制限）
    if document.file_size > 20 * 1024 * 1024:
        await update.message.reply_text(
            "ファイルサイズが大きすぎます（上限: 20MB）。"
        )
        return

    file = await context.bot.get_file(document.file_id)

    async with httpx.AsyncClient() as http_client:
        response = await http_client.get(file.file_path)
        file_bytes = response.content

    caption = update.message.caption or "このドキュメントの内容を要約してください。"

    try:
        await update.message.reply_text("ドキュメントを分析中です...")

        import base64
        response = client.models.generate_content(
            model="gemini-3-flash",
            contents=[
                {
                    "role": "user",
                    "parts": [
                        {"text": caption},
                        {
                            "inline_data": {
                                "mime_type": mime_type,
                                "data": base64.b64encode(file_bytes).decode(),
                            }
                        },
                    ],
                }
            ],
            config={
                "max_output_tokens": 4096,
                "temperature": 0.3,
            },
        )

        reply = response.text
        if len(reply) > 4000:
            for i in range(0, len(reply), 4000):
                await update.message.reply_text(reply[i : i + 4000])
        else:
            await update.message.reply_text(reply)

    except Exception as e:
        logger.error(f"Document analysis error: {e}")
        await update.message.reply_text(
            "ドキュメントの分析中にエラーが発生しました。"
        )
```

`main()` 関数にハンドラーを追加します。

```python
app.add_handler(MessageHandler(filters.Document.ALL, handle_document))
```

## エラーハンドリングとレート制限対策

本番運用では、API のレート制限やネットワークエラーへの適切な対処が不可欠です。リトライロジックを組み込んだラッパー関数を作成しましょう。

```python
import asyncio
from google.api_core.exceptions import ResourceExhausted, ServiceUnavailable

async def call_gemini_with_retry(contents, config, max_retries=3):
    """
    リトライ付き Gemini API 呼び出し
    """
    for attempt in range(max_retries):
        try:
            response = client.models.generate_content(
                model="gemini-3-flash",
                contents=contents,
                config=config,
            )
            return response

        except ResourceExhausted:
            # レート制限 — 指数バックオフで待機
            wait_time = 2 ** attempt
            logger.warning(
                f"Rate limited. Retrying in {wait_time}s "
                f"(attempt {attempt + 1}/{max_retries})"
            )
            await asyncio.sleep(wait_time)

        except ServiceUnavailable:
            # サービス一時停止 — リトライ
            wait_time = 3 ** attempt
            logger.warning(
                f"Service unavailable. Retrying in {wait_time}s "
                f"(attempt {attempt + 1}/{max_retries})"
            )
            await asyncio.sleep(wait_time)

    raise Exception("Maximum retries exceeded")

# 使用例: handle_message 内で
# response = await call_gemini_with_retry(
#     contents=history,
#     config={"max_output_tokens": 2048, "temperature": 0.7},
# )
```

エラーハンドリングのより詳しいパターンについては、「[Gemini API のエラーハンドリングとリトライパターン完全ガイド](/articles/gemini-api/gemini-api-error-handling-retry-patterns)」も参考にしてください。

## よくある質問（FAQ）

### Q: Telegram Bot の無料枠に制限はありますか？

Telegram Bot API 自体は完全無料で利用できます。メッセージ送信数にも基本的な制限はありません（ただし、1秒あたり30メッセージ、1チャットあたり1秒1メッセージの送信レート制限があります）。コストが発生するのは Gemini API 側のトークン使用量のみです。

### Q: ボットをグループチャットで使うにはどうすればよいですか？

BotFather で `/setprivacy` コマンドを実行し、プライバシーモードを `Disable` に設定します。これにより、グループ内の全メッセージをボットが受信できるようになります。ただし、グループでは `@ボット名` でメンションされたメッセージのみに応答する設計が推奨されます。

### Q: Discord や LINE ボットとの大きな違いは何ですか？

Telegram Bot API はメッセージ長の上限が4,096文字と比較的大きく、マークダウン記法やインラインキーボードなどの UI 機能が充実しています。また、Webhook と Polling の両方をサポートしており、デプロイの自由度が高い点も特徴です。他のプラットフォームとの比較は「[Gemini API × Discord Bot 構築ガイド](/articles/gemini-api/gemini-api-discord-bot-python-guide)」や「[LINE Bot 構築ガイド](/articles/gemini-api/gemini-api-line-bot-python-guide)」もあわせてご覧ください。

### Q: Webhook モードに切り替えるメリットはありますか？

Polling モードは開発時に手軽ですが、本番環境では Webhook モードが推奨されます。Webhook はメッセージ受信時にのみサーバーが処理を行うため、リソース消費が少なく、応答速度も向上します。HTTPS エンドポイントが必要ですが、Cloudflare Tunnel や ngrok を使えば簡単に設定できます。
