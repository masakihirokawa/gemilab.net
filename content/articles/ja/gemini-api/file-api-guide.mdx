## はじめに

Gemini APIには、プロンプトとは別にファイルを事前にアップロードできる **File API** が用意されています。大きなファイルや複数のリクエストで再利用したいメディアを扱う際に非常に便利な機能です。

本記事では、Gemini File APIの基本から実践的な活用方法まで、Pythonコードを交えて詳しく解説します。

---

## 前提知識

- Python 3.9以上の環境
- Gemini APIキー（[Google AI Studio](https://aistudio.google.com/) で取得可能）
- `google-genai` パッケージのインストール

```bash
pip install google-genai
```

---

## File API とは

File API は、Gemini API の一部として提供される**メディアファイル管理サービス**です。主な特徴は以下の通りです。

- **無料で利用可能**（File APIの使用自体に追加コストなし）
- **48時間保存**：アップロードしたファイルは48時間保持される
- **最大ファイルサイズ**：1ファイルあたり 2 GB（リクエスト全体では 100 MB 以上の場合はFile APIを推奨）
- **対応フォーマット**：画像・PDF・動画・音声・テキストなど幅広いファイル形式

### File API が必要なケース

リクエスト全体（ファイル＋テキストプロンプト＋システム指示）が **100 MB を超える場合** は File API の使用が必須です。それ以下であればインラインでBase64エンコードして渡すこともできますが、File APIを使うことで再利用性が高まります。

## 基本的な使い方

### 1. クライアントの初期化

```python
import os
from google import genai

client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
```

### 2. ファイルのアップロード

```python
# 画像ファイルをアップロード
myfile = client.files.upload(file="./photo.jpg")
print(f"アップロード完了: {myfile.uri}")
```

アップロードが成功すると、`myfile.uri` にファイルの一意なURIが返されます。

### 3. アップロードしたファイルでコンテンツ生成

```python
# アップロードしたファイルを使ってコンテンツ生成
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[myfile, "この画像に写っているものを詳しく説明してください。"]
)
print(response.text)
```

## PDFファイルの活用

PDFは特によく使われるユースケースです。最大 **50 MB** のPDFを処理できます。

```python
# PDFをアップロード
pdf_file = client.files.upload(
    file="./report.pdf",
    config={"mime_type": "application/pdf"}
)

# PDFの内容について質問
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        pdf_file,
        "このPDFの主要なポイントを3点にまとめてください。"
    ]
)
print(response.text)
```

## 動画ファイルの活用

動画ファイルの処理には少し時間がかかるため、**アクティブ状態になるまで待機**する必要があります。

```python
import time

# 動画ファイルをアップロード
video_file = client.files.upload(file="./presentation.mp4")

# 処理が完了するまで待機
while video_file.state.name == "PROCESSING":
    print("動画を処理中...")
    time.sleep(5)
    video_file = client.files.get(name=video_file.name)

if video_file.state.name == "FAILED":
    raise ValueError("動画の処理に失敗しました")

print(f"処理完了: {video_file.uri}")

# 動画の内容について分析
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        video_file,
        "この動画の内容を日本語で要約してください。"
    ]
)
print(response.text)
```

## 複数ファイルの組み合わせ

複数のファイルとテキストを組み合わせることも可能です。

```python
# 複数の画像をアップロード
image1 = client.files.upload(file="./before.jpg")
image2 = client.files.upload(file="./after.jpg")

# 2枚の画像を比較
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        "2枚の画像を比較して、違いを列挙してください。\n\n画像1（before）:",
        image1,
        "\n\n画像2（after）:",
        image2
    ]
)
print(response.text)
```

## ファイルの管理

### アップロード済みファイルの一覧取得

```python
# すべてのファイルを一覧表示
for file in client.files.list():
    print(f"名前: {file.name}, URI: {file.uri}, 状態: {file.state}")
```

### ファイルの詳細取得

```python
# ファイル名を使って詳細を取得
file_info = client.files.get(name="files/xxxxxx")
print(f"作成日時: {file_info.create_time}")
print(f"有効期限: {file_info.expiration_time}")
```

### ファイルの削除

```python
# ファイルを手動で削除
client.files.delete(name=myfile.name)
print("ファイルを削除しました")
```

## よくある質問（FAQ）

**Q: File APIは無料で使えますか？**
A: はい、File APIのファイルアップロード・管理自体は無料です。ただし、アップロードしたファイルを使ってコンテンツを生成する際は、通常のGemini APIの料金が発生します。

**Q: ファイルはいつまで保存されますか？**
A: アップロードから **48時間後** に自動削除されます。それ以前に削除したい場合は `client.files.delete()` を使用してください。

**Q: 対応しているファイル形式は何ですか？**
A: 画像（JPEG, PNG, WebP, HEIC, HEIF）、動画（MP4, MOV, AVI等）、音声（MP3, WAV, FLAC等）、PDF、テキスト（TXT, HTML, CSS, Markdown等）に対応しています。

**Q: ファイルのURIは他のプロジェクトでも使えますか？**
A: いいえ。ファイルのURIはアップロードしたAPIキーのプロジェクトに紐付けられています。APIキーのセキュリティ管理には十分ご注意ください。

Gemini File API を活用することで、大きなファイルや複数リクエストでの再利用が効率的になります。主なポイントをまとめます。

- File API は無料で利用でき、48時間ファイルを保持する
- 100 MB を超えるリクエストでは File API の使用が必須
- 画像・PDF・動画・音声など幅広いファイル形式に対応
- 動画は処理完了を待ってから使用する必要がある
- ファイルはAPIキーのプロジェクトに紐付けられるため、セキュリティに注意
