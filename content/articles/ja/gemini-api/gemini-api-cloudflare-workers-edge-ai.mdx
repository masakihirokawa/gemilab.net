## はじめに

AI アプリケーションのレスポンス速度は、ユーザー体験を大きく左右します。Gemini API は強力な言語モデルを提供しますが、ユーザーとサーバーの物理的な距離によってはレイテンシーが課題になることがあります。

ここで活躍するのが **Cloudflare Workers** です。世界300以上のエッジロケーションでコードを実行でき、ユーザーに最も近いサーバーからレスポンスを返せます。Gemini API と組み合わせることで、グローバルに低レイテンシーな AI API を構築できます。

この記事では、Cloudflare Workers 上で Gemini API を活用したエッジ AI API の構築方法を、環境構築からデプロイまで実践的に解説します。

**この記事で学べること:**

- Cloudflare Workers で Gemini API を呼び出す基本パターン
- KV Storage を使った AI レスポンスのキャッシュ戦略
- ストリーミングレスポンスの実装方法
- エラーハンドリングとレート制限への対処
- 本番環境を想定したセキュリティ設定

**対象読者:** Gemini API の基本を理解しており、Cloudflare Workers でのデプロイ経験がある方（または興味がある方）

### 基本的な API エンドポイント

Cloudflare Workers から Gemini API を呼び出す最もシンプルな実装です。

```typescript
// src/index.ts
export interface Env {
  GEMINI_API_KEY: string;
  GEMINI_MODEL: string;
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    // CORS ヘッダーの設定
    if (request.method === "OPTIONS") {
      return new Response(null, {
        headers: {
          "Access-Control-Allow-Origin": "*",
          "Access-Control-Allow-Methods": "POST, OPTIONS",
          "Access-Control-Allow-Headers": "Content-Type",
        },
      });
    }

    if (request.method !== "POST") {
      return new Response(
        JSON.stringify({ error: "Method not allowed" }),
        { status: 405, headers: { "Content-Type": "application/json" } }
      );
    }

    try {
      const { prompt } = await request.json() as { prompt: string };

      if (!prompt || typeof prompt !== "string") {
        return new Response(
          JSON.stringify({ error: "Invalid prompt" }),
          { status: 400, headers: { "Content-Type": "application/json" } }
        );
      }

      // Gemini API を呼び出し
      const geminiResponse = await fetch(
        `https://generativelanguage.googleapis.com/v1beta/models/${env.GEMINI_MODEL}:generateContent?key=${env.GEMINI_API_KEY}`,
        {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({
            contents: [
              { role: "user", parts: [{ text: prompt }] }
            ],
            generationConfig: {
              temperature: 0.7,
              maxOutputTokens: 1024,
            },
          }),
        }
      );

      const data = await geminiResponse.json();

      // 期待される出力:
      // {
      //   "response": "Gemini からの応答テキスト...",
      //   "model": "gemini-3-flash",
      //   "edge_location": "NRT" (東京の場合)
      // }
      return new Response(
        JSON.stringify({
          response: data.candidates?.[0]?.content?.parts?.[0]?.text ?? "",
          model: env.GEMINI_MODEL,
          edge_location: request.cf?.colo ?? "unknown",
        }),
        {
          headers: {
            "Content-Type": "application/json",
            "Access-Control-Allow-Origin": "*",
          },
        }
      );
    } catch (error) {
      return new Response(
        JSON.stringify({ error: "Internal server error" }),
        { status: 500, headers: { "Content-Type": "application/json" } }
      );
    }
  },
};
```

`request.cf?.colo` でリクエストを処理したエッジロケーションを確認でき、実際にユーザーに近い場所で処理されていることを確認できます。

## KV Storage による AI レスポンスのキャッシュ戦略

同じプロンプトに対する Gemini API の呼び出しを毎回行うのは非効率です。Cloudflare KV を使ってレスポンスをキャッシュし、コスト削減とレスポンス高速化を実現しましょう。

### KV Namespace の作成と設定

```bash
# KV Namespace を作成
wrangler kv namespace create CACHE

# 出力された ID を wrangler.toml に追記
```

```toml
# wrangler.toml に追記
[[kv_namespaces]]
binding = "CACHE"
id = "your-namespace-id-here"
```

### キャッシュ付き API の実装

```typescript
// src/index.ts（キャッシュ対応版）
import { createHash } from "node:crypto";

export interface Env {
  GEMINI_API_KEY: string;
  GEMINI_MODEL: string;
  CACHE: KVNamespace;
}

// プロンプトからキャッシュキーを生成
function generateCacheKey(prompt: string, model: string): string {
  const hash = createHash("sha256")
    .update(`${model}:${prompt}`)
    .digest("hex")
    .slice(0, 16);
  return `gemini:${hash}`;
}

async function callGeminiAPI(
  prompt: string,
  env: Env
): Promise<string> {
  const response = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/${env.GEMINI_MODEL}:generateContent?key=${env.GEMINI_API_KEY}`,
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        contents: [{ role: "user", parts: [{ text: prompt }] }],
        generationConfig: {
          temperature: 0.7,
          maxOutputTokens: 1024,
        },
      }),
    }
  );

  const data = await response.json();
  return data.candidates?.[0]?.content?.parts?.[0]?.text ?? "";
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    if (request.method !== "POST") {
      return new Response("Method not allowed", { status: 405 });
    }

    const { prompt, skipCache } = await request.json() as {
      prompt: string;
      skipCache?: boolean;
    };

    const cacheKey = generateCacheKey(prompt, env.GEMINI_MODEL);

    // キャッシュを確認
    if (!skipCache) {
      const cached = await env.CACHE.get(cacheKey);
      if (cached) {
        return new Response(
          JSON.stringify({
            response: cached,
            cached: true,
            edge_location: request.cf?.colo,
          }),
          { headers: { "Content-Type": "application/json" } }
        );
      }
    }

    // Gemini API を呼び出し
    const result = await callGeminiAPI(prompt, env);

    // KV にキャッシュ（TTL: 1時間）
    await env.CACHE.put(cacheKey, result, { expirationTtl: 3600 });

    // 期待される出力:
    // { "response": "...", "cached": false, "edge_location": "NRT" }
    return new Response(
      JSON.stringify({
        response: result,
        cached: false,
        edge_location: request.cf?.colo,
      }),
      { headers: { "Content-Type": "application/json" } }
    );
  },
};
```

キャッシュヒット時は Gemini API を呼び出さずに即座にレスポンスを返すため、レスポンスタイムが大幅に短縮されます。

## ストリーミングレスポンスの実装

長文生成時にユーザーを待たせないために、ストリーミングレスポンスを実装しましょう。Cloudflare Workers は `ReadableStream` をネイティブサポートしているため、Gemini API のストリーミングとの相性が抜群です。

```typescript
// src/streaming.ts
async function streamGeminiResponse(
  prompt: string,
  env: Env
): Promise<ReadableStream> {
  const response = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/${env.GEMINI_MODEL}:streamGenerateContent?alt=sse&key=${env.GEMINI_API_KEY}`,
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        contents: [{ role: "user", parts: [{ text: prompt }] }],
        generationConfig: {
          temperature: 0.7,
          maxOutputTokens: 2048,
        },
      }),
    }
  );

  if (!response.ok || !response.body) {
    throw new Error(`Gemini API error: ${response.status}`);
  }

  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  return new ReadableStream({
    async pull(controller) {
      const { done, value } = await reader.read();

      if (done) {
        controller.enqueue(
          new TextEncoder().encode("data: [DONE]\n\n")
        );
        controller.close();
        return;
      }

      const chunk = decoder.decode(value, { stream: true });
      // SSE 形式のチャンクをそのまま転送
      controller.enqueue(new TextEncoder().encode(chunk));
    },
    cancel() {
      reader.cancel();
    },
  });
}

// ストリーミング対応エンドポイント
// GET /stream?prompt=...
export async function handleStream(
  request: Request,
  env: Env
): Promise<Response> {
  const url = new URL(request.url);
  const prompt = url.searchParams.get("prompt");

  if (!prompt) {
    return new Response("Missing prompt", { status: 400 });
  }

  const stream = await streamGeminiResponse(prompt, env);

  return new Response(stream, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      "Connection": "keep-alive",
      "Access-Control-Allow-Origin": "*",
    },
  });
}
```

ストリーミングの詳しい仕組みについては、[Gemini API ストリーミング＆チャットガイド](/articles/gemini-api/streaming-and-chat) も参考にしてください。

## エラーハンドリングとレート制限への対処

本番環境では、Gemini API のエラーやレート制限に適切に対処する必要があります。

### エクスポネンシャルバックオフ付きリトライ

```typescript
// src/utils/retry.ts
async function fetchWithRetry(
  url: string,
  options: RequestInit,
  maxRetries: number = 3
): Promise<Response> {
  let lastError: Error | null = null;

  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      const response = await fetch(url, options);

      // 429 (Rate Limit) または 503 (Service Unavailable) はリトライ
      if (response.status === 429 || response.status === 503) {
        const retryAfter = response.headers.get("Retry-After");
        const delay = retryAfter
          ? parseInt(retryAfter) * 1000
          : Math.min(1000 * Math.pow(2, attempt), 10000);

        console.log(
          `Rate limited. Retrying in ${delay}ms (attempt ${attempt + 1})`
        );
        await new Promise((r) => setTimeout(r, delay));
        continue;
      }

      return response;
    } catch (error) {
      lastError = error as Error;
      const delay = Math.min(1000 * Math.pow(2, attempt), 10000);
      await new Promise((r) => setTimeout(r, delay));
    }
  }

  throw lastError ?? new Error("Max retries exceeded");
}

// 使用例:
// const response = await fetchWithRetry(geminiUrl, requestOptions);
// 期待される動作:
// - 1回目失敗 → 1秒待機 → 2回目失敗 → 2秒待機 → 3回目で成功
```

### レート制限の分散管理

Cloudflare Workers の Durable Objects を使えば、グローバルなレート制限カウンターを実装できます。ただし、より簡易的なアプローチとして KV ベースのレート制限も有効です。

```typescript
// src/utils/rate-limit.ts
async function checkRateLimit(
  clientIP: string,
  kv: KVNamespace,
  limit: number = 60,
  window: number = 60
): Promise<{ allowed: boolean; remaining: number }> {
  const key = `ratelimit:${clientIP}`;
  const current = parseInt((await kv.get(key)) ?? "0");

  if (current >= limit) {
    return { allowed: false, remaining: 0 };
  }

  await kv.put(key, String(current + 1), {
    expirationTtl: window,
  });

  return { allowed: true, remaining: limit - current - 1 };
}
```

Gemini API のレート制限について詳しくは、[Gemini API レート制限とクォータ管理ガイド](/articles/gemini-api/gemini-api-rate-limiting-quota-management) をご覧ください。

## よくある質問（FAQ）

### Q1: Cloudflare Workers の無料プランで Gemini API を使えますか？

はい、Cloudflare Workers の無料プランでも Gemini API を呼び出せます。無料プランでは1日あたり10万リクエスト、1リクエストあたり10msのCPU時間が上限です。Gemini API の呼び出しは外部リクエストのため CPU 時間にはほぼ影響しませんが、KV の読み書きには若干の CPU 時間がかかります。個人プロジェクトやプロトタイプには十分な枠です。

### Q2: Gemini API のレスポンスをキャッシュする際の注意点は？

キャッシュする際は、プロンプトの内容に応じて TTL（有効期限）を調整しましょう。時事ネタや最新情報を返すプロンプトは短めの TTL（数分〜数十分）、普遍的な知識に関するプロンプトは長めの TTL（数時間〜1日）が適切です。また、ユーザーごとにパーソナライズされた応答はキャッシュを避けるか、ユーザー ID をキャッシュキーに含めましょう。

### Q3: ストリーミングとキャッシュを両立させるにはどうすればよいですか？

ストリーミングとキャッシュの併用には工夫が必要です。おすすめのパターンは、まずキャッシュを確認し、ヒットすればキャッシュ済みの完全なレスポンスを返す方法です。キャッシュミスの場合はストリーミングで返しつつ、レスポンス全体をバッファに蓄積し、完了後に KV へ書き込むアプローチが効果的です。

### Q4: Cloudflare Workers から Gemini API を呼ぶ際のレイテンシーはどの程度ですか？

Cloudflare Workers 自体のコールドスタートはほぼゼロ（0ms に近い）ですが、Gemini API へのリクエスト時間はモデルやプロンプトの長さに依存します。Gemini 3 Flash で短いプロンプトの場合、通常 500ms〜1.5秒程度です。エッジからの呼び出しにより、従来のオリジンサーバー経由と比べて 50〜200ms のレイテンシー削減が見込めます。

## まとめ

Gemini API と Cloudflare Workers の組み合わせは、グローバルに展開する AI アプリケーションにとって強力な選択肢です。

この記事で紹介した主なポイントを振り返ります。

- **基本的な API 呼び出し**: Workers の `fetch` ハンドラから Gemini API を直接呼び出すシンプルなパターン
- **KV キャッシュ**: 同一プロンプトへの重複呼び出しを避け、コスト削減とレスポンス高速化を実現
- **ストリーミング**: `ReadableStream` によるリアルタイムなレスポンス配信
- **エラーハンドリング**: エクスポネンシャルバックオフとレート制限で本番環境の安定性を確保
- **セキュリティ**: API キー認証と入力サニタイズで安全な公開 API を構築
