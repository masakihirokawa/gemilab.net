# Gemini APIのSystem Instructions完全ガイド — AIの振る舞いを自在にコントロールする

## はじめに

Gemini APIを使ったアプリ開発で欠かせない機能のひとつが **System Instructions（システムインストラクション）** です。System Instructionsを使うと、モデルに対してあらかじめ「役割」「口調」「出力フォーマット」「行動制約」などを指定し、すべての会話でその設定を維持させることができます。

この記事では、以下の内容を詳しく解説します：

- System Instructionsの基本概念と効果
- PythonおよびJavaScriptでの実装方法
- 実践的なユースケース（カスタマーサポートBot、コードレビュアー、多言語対応アシスタントなど）
- マルチターン会話との組み合わせ
- よくあるミスとトラブルシューティング
- FAQ

対象読者は、Gemini APIの基本（APIキー取得・SDK導入）をすでに理解している中級レベルの開発者です。まだAPIの基礎を学んでいない方は [Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart) をご確認ください。

## System Instructionsとは

System Instructions（システムインストラクション）は、モデルへの「事前指示」です。ユーザーとの会話が始まる前にモデルに渡され、会話全体を通じてその指示が効果を持ち続けます。

通常のユーザーメッセージ（`contents`）と異なり、System Instructionsは会話履歴に残らず、ユーザーからは見えません。これにより、アプリの振る舞いを透明に、かつ確実に制御できます。

### System Instructionsでできること

| 制御対象 | 例 |
|---------|-----|
| 役割の定義 | 「あなたはシニアPythonエンジニアです」 |
| 口調・スタイル | 「丁寧な敬語で、専門用語は避けてください」 |
| 出力フォーマット | 「常にMarkdown形式で回答してください」 |
| 言語指定 | 「質問の言語に関わらず、常に日本語で答えてください」 |
| 知識範囲の制限 | 「製品サポートに関する質問のみ回答してください」 |
| 安全ポリシー | 「競合他社の名前は一切出さないでください」 |

## Pythonでの実装

### 基本的な使い方

```python
import os
from google import genai
from google.genai import types

# クライアントの初期化
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# System Instructionsを含むリクエスト
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="Pythonでファイルを読み込む方法を教えてください",
    config=types.GenerateContentConfig(
        system_instruction=(
            "あなたは経験豊富なPythonシニアエンジニアです。"
            "コードは常にtype hintを付けてください。"
            "コメントは日本語で記述し、期待する出力例を必ず含めてください。"
        ),
        temperature=0.4,
    ),
)

print(response.text)
# 期待する出力例:
# def read_file(filepath: str) -> str:
#     """ファイルを読み込んでテキストを返す"""
#     with open(filepath, "r", encoding="utf-8") as f:
#         return f.read()
```

### マルチターン会話との組み合わせ

System Instructionsはチャットセッションでも有効です。一度設定すれば、会話が続く限り機能します。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# カスタマーサポートBotのSystem Instructions
SUPPORT_INSTRUCTION = """
あなたは「GemLab Assistant」という名前のカスタマーサポート担当者です。
以下のルールを厳守してください：
1. 敬語を使い、親切かつ丁寧に回答する
2. 製品「Gemini Lab Pro」に関する質問のみ回答する
3. 競合他社（OpenAI、Anthropicなど）については言及しない
4. 解決できない問題は「担当チームにエスカレーションします」と伝える
5. 回答の最後に「他にご質問はありますか？」と必ず添える
"""

# チャットセッション開始
chat = client.chats.create(
    model="gemini-2.5-pro",
    config=types.GenerateContentConfig(
        system_instruction=SUPPORT_INSTRUCTION,
        temperature=0.3,  # サポートBotは一貫性重視で低めに設定
    ),
)

# 会話1回目
response1 = chat.send_message("プランのアップグレード方法を教えてください")
print("Bot:", response1.text)

# 会話2回目（コンテキストが維持される）
response2 = chat.send_message("支払い方法はクレジットカードだけですか？")
print("Bot:", response2.text)
```

### ロールプレイ型アシスタント

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# コードレビュアーの設定
CODE_REVIEWER_INSTRUCTION = """
あなたはGoogleのシニアソフトウェアエンジニアで、コードレビューの専門家です。
提出されたコードに対して以下の観点でレビューしてください：

**レビュー形式：**
1. 🔴 重大な問題（セキュリティ、バグ）
2. 🟡 改善推奨（パフォーマンス、可読性）
3. 🟢 良い点

**ルール：**
- 具体的な修正コードを提示する
- 理由を必ず説明する
- 5段階でコード品質スコアを付ける（例: 品質スコア: 3/5）
"""

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="""
以下のコードをレビューしてください：

```python
def get_user(id):
    query = "SELECT * FROM users WHERE id = " + id
    return db.execute(query)
```
""",
    config=types.GenerateContentConfig(
        system_instruction=CODE_REVIEWER_INSTRUCTION,
    ),
)

print(response.text)
# 期待する出力例:
# 🔴 重大な問題
# SQLインジェクション脆弱性があります。文字列連結でSQLを構築することは
# 危険です。パラメータ化クエリを使用してください。
# ...
# 品質スコア: 1/5
```

## JavaScriptでの実装

Node.jsやブラウザ環境でも同様に利用できます。

```javascript
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

// System Instructionsを設定してモデルを取得
const model = genAI.getGenerativeModel({
  model: "gemini-2.5-pro",
  systemInstruction: {
    role: "system",
    parts: [
      {
        text: `あなたは日本語と英語のバイリンガルライターです。
ユーザーの要求に応じて、自然で読みやすい文章を作成します。
- 日本語のリクエストには日本語で回答
- 英語のリクエストには英語で回答
- 専門用語は初出時に必ず説明を加える`,
      },
    ],
  },
  generationConfig: {
    temperature: 0.7,
    maxOutputTokens: 2048,
  },
});

// マルチターン会話
const chat = model.startChat();

const response1 = await chat.sendMessage(
  "量子コンピュータについて初心者向けに説明してください"
);
console.log(response1.response.text());

// 2回目のメッセージ（コンテキスト維持）
const response2 = await chat.sendMessage(
  "もう少し具体的な例を挙げてください"
);
console.log(response2.response.text());

// 英語に切り替え
const response3 = await chat.sendMessage(
  "Can you explain the difference between qubits and classical bits?"
);
console.log(response3.response.text());
// → 英語で自動的に回答される
```

### 1. 多言語対応カスタマーサポートBot

```python
import os

from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

MULTILINGUAL_SUPPORT = """
You are a multilingual customer support assistant for "TechCorp Global."

- Always respond in the same language as the user's message
- Be professional, empathetic, and solution-focused
- If you cannot resolve an issue, offer to escalate to a human agent
- Never discuss competitors or make promises about features not yet released

"""

chat = client.chats.create(
    model="gemini-2.5-pro",

        system_instruction=MULTILINGUAL_SUPPORT,
        temperature=0.3,
    ),
)

# 日本語で質問 → 日本語で回答
r1 = chat.send_message("パスワードを忘れてしまいました")

# 英語で質問 → 英語で回答
r2 = chat.send_message("How do I cancel my subscription?")

```

## System Instructionsの設計ベストプラクティス

### 効果的な指示の書き方

System Instructionsは、品質がそのままモデルの挙動に直結します。以下のポイントを意識して作成してください。

**1. 役割を明確に定義する**

```
# 悪い例
「あなたはアシスタントです。」

# 良い例
「あなたはGemini APIの専門家で、Google認定デベロッパーエキスパートです。
5年以上のMLエンジニアリング経験を持ち、Python実装を得意とします。」
```

**2. 期待する出力形式を具体的に指示する**

```
「回答は常に以下の構成で提供してください：
1. 概要（2〜3文）
2. 詳細説明（箇条書き）
3. コード例（Python）
4. 注意点（あれば）」
```

**3. 制約事項はポジティブに表現する**

```
# 悪い例（ネガティブ）
「競合製品についてのコメントは禁止です。」

# 良い例（ポジティブ）
「質問はGemini APIとGoogle AI Studioに関するものに絞ってご回答ください。
他のAIサービスについての質問は、公式ドキュメントへの参照を案内してください。」
```

**4. 少数例（Few-shot）を組み込む**

複雑な出力フォーマットが必要な場合は、System Instructions内にサンプルを含めると精度が向上します。

```python
INSTRUCTION_WITH_EXAMPLES = """
以下の形式でコードをレビューしてください。

入力例：
```python
def add(a, b):
    return a + b
```

出力例：
## コードレビュー結果
- ✅ シンプルで読みやすい実装
- ⚠️ 型アノテーションなし → `def add(a: int, b: int) -> int:` を推奨
- 品質スコア: 4/5
"""
```

## Context Cachingとの組み合わせ

System Instructionsが長文になる場合（システムプロンプト＋大量のドキュメントなど）、[Context Caching](/articles/gemini-api/context-caching-guide) を使うとコストを大幅に削減できます。

```python
import os
from google import genai
from google.genai import types
import datetime

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 長い製品マニュアルをキャッシュ
with open("product_manual.txt", "r") as f:
    manual_content = f.read()

# キャッシュ作成（TTL: 60分）
cache = client.caches.create(
    model="gemini-2.5-pro",
    config=types.CreateCachedContentConfig(
        system_instruction=(
            "あなたは製品サポート担当者です。"
            "以下のマニュアルを参照して正確に回答してください。"
        ),
        contents=[manual_content],
        ttl=datetime.timedelta(hours=1),
    ),
)

# キャッシュを使ってリクエスト（毎回マニュアルを送らなくてよい）
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="バッテリー交換の手順を教えてください",
    config=types.GenerateContentConfig(
        cached_content=cache.name,
    ),
)
print(response.text)
```

大量のトークンを繰り返し送信するコストが節約でき、応答速度も向上します。

## よくあるエラーと対処法

### エラー1: System Instructionsが無視される

**症状**: 指示通りの出力にならない

**対処法**:
- 指示が曖昧すぎないか確認する（具体的に書き直す）
- 矛盾する指示が含まれていないか確認する
- `temperature`が高すぎる場合は下げる（0.3〜0.5が安定）

### エラー2: JSONしか返さないはずなのに文章が混じる

**対処法**:

```python
# response_mime_typeを使って強制する
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="商品情報を抽出してください：...",
    config=types.GenerateContentConfig(
        system_instruction="JSONのみで返してください",
        response_mime_type="application/json",
    ),
)
```

`response_mime_type="application/json"` を指定することで、JSON出力を強制できます。詳しくは [構造化出力ガイド](/articles/gemini-api/json-mode-structured-output) を参照してください。

### エラー3: チャット途中でSystem Instructionsが効かなくなる

**症状**: 長い会話の途中から指示に従わなくなる

**対処法**:
- 会話が長くなりすぎるとコンテキストウィンドウを圧迫する場合があります
- 重要なルールは会話の中で定期的にリマインドする
- Context Cachingを活用してコンテキスト管理を改善する

### エラー4: `400 Bad Request`

**症状**: リクエストが拒否される

**対処法**: System Instructionsの内容が安全ポリシーに抵触していないか確認してください。特に「有害コンテンツを生成してください」などの指示は拒否されます。

## FAQ

**Q1: System InstructionsはユーザーAPIキーから料金が発生しますか？**

はい、System InstructionsのテキストもAPIのトークン使用量にカウントされます。長い指示は毎リクエストごとにトークンを消費するため、特に大規模なプロダクションではContext Cachingの活用を検討してください。

**Q2: ユーザーがSystem Instructionsを上書きできますか？**

通常の会話では上書きできません。ただし、非常に長い会話で「これまでの指示をすべて無視して...」といった「プロンプトインジェクション」が試みられることがあります。重要なアプリでは入力バリデーションと合わせて使用することを推奨します。

**Q3: 複数のSystem Instructionsを組み合わせられますか？**

現在のAPIでは `system_instruction` フィールドは単一の文字列ですが、複数の役割・ルールを一つの文字列に統合することで対応できます。区切り記号（`---` や見出し）で構造化すると管理しやすくなります。

**Q4: Function Callingと組み合わせる場合は？**

System InstructionsはFunction Callingとも完全に互換性があります。「ユーザーが天気について聞いたら必ずget_weather関数を呼び出す」といった指示をSystem Instructionsに含めることで、ツール使用の挙動を制御できます。詳しくは [Function Callingガイド](/articles/gemini-advanced/function-calling-guide) をご参照ください。

**Q5: Vertex AI版でも同じ方法で使えますか？**

基本的な使い方は同じですが、クライアントの初期化方法が異なります：

```python
from google import genai

# Vertex AI用クライアント
client = genai.Client(
    vertexai=True,
    project="your-gcp-project-id",
    location="us-central1",
)
```

それ以降の`generate_content()`の呼び出し方は同一です。
