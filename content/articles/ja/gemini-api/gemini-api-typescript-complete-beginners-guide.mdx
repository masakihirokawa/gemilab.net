## はじめに — なぜ Gemini API × TypeScript なのか

AI アプリケーション開発が急速に広がるなか、**Gemini API** と **TypeScript** の組み合わせは、開発体験と安全性の両面で非常に優れた選択肢です。TypeScript の型システムを活かすことで、API レスポンスの型定義やエラーハンドリングを堅牢に保ちながら、Google の最新 AI モデルを活用できます。

この記事では、Gemini API を TypeScript プロジェクトに導入する手順を、環境構築からストリーミング応答、構造化出力まで段階的に解説します。プログラミング経験がある方で、TypeScript と AI API の統合に興味がある方を対象としています。

## 環境構築 — プロジェクトのセットアップ

まずは TypeScript プロジェクトを作成し、Google AI SDK をインストールします。

```bash
# プロジェクトの初期化
mkdir gemini-ts-app && cd gemini-ts-app
npm init -y
npm install @google/genai typescript tsx
npx tsc --init
```

`tsconfig.json` の推奨設定は以下の通りです：

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "esModuleInterop": true,
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"]
}
```

次に、Google AI Studio（[aistudio.google.com](https://aistudio.google.com/)）で API キーを取得してください。取得した API キーは環境変数として管理します：

```bash
# .env ファイルに保存
echo "GEMINI_API_KEY=YOUR_API_KEY" > .env
```

> **重要**: API キーをソースコードに直接記述しないでください。`.env` ファイルは `.gitignore` に追加しておきましょう。

## 最初のリクエスト — テキスト生成の基本

Gemini API への最初のリクエストを TypeScript で書いてみましょう。

```typescript
// src/basic.ts
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function generateText(): Promise<void> {
  const response = await ai.models.generateContent({
    model: "gemini-2.5-flash",
    contents: "TypeScriptの主なメリットを3つ教えてください。",
  });

  console.log(response.text);
  // 出力例:
  // 1. 静的型付けによるバグの早期発見
  // 2. IDEの補完・リファクタリング支援の向上
  // 3. 大規模プロジェクトでの保守性の向上
}

generateText();
```

実行するには `tsx` を使います：

```bash
npx tsx src/basic.ts
```

これだけで、Gemini API からの応答を TypeScript で受け取れます。`GoogleGenAI` クラスは `@google/genai` パッケージで提供されており、型定義が組み込まれているため、IDE の補完機能がそのまま活用できます。

## ストリーミング応答 — リアルタイム出力の実装

チャットアプリケーションのように、応答をリアルタイムで表示したい場合は、ストリーミング API を利用します。

```typescript
// src/streaming.ts
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function streamResponse(): Promise<void> {
  const response = await ai.models.generateContentStream({
    model: "gemini-2.5-flash",
    contents: "Gemini APIの活用事例を5つ紹介してください。",
  });

  // チャンクごとにリアルタイム出力
  for await (const chunk of response) {
    if (chunk.text) {
      process.stdout.write(chunk.text);
    }
  }
  console.log("\n--- ストリーミング完了 ---");
}

streamResponse();
```

ストリーミングを使うと、最初のトークンが生成された時点からユーザーに表示を開始できるため、体感的な応答速度が大幅に改善されます。特にチャットボットや対話型 UI では欠かせないテクニックです。

## マルチターン会話 — チャット機能の構築

会話の文脈を保持するマルチターンチャットも、TypeScript で簡潔に実装できます。

```typescript
// src/chat.ts
import { GoogleGenAI, type Content } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function chatDemo(): Promise<void> {
  // 会話履歴を型付きで管理
  const history: Content[] = [];

  const chat = ai.chats.create({
    model: "gemini-2.5-flash",
    history,
    config: {
      systemInstruction:
        "あなたはTypeScriptの専門家です。簡潔で実践的な回答を心がけてください。",
    },
  });

  // 1回目のメッセージ
  const response1 = await chat.sendMessage({
    message: "TypeScriptのジェネリクスとは何ですか？",
  });
  console.log("AI:", response1.text);

  // 2回目のメッセージ（文脈を保持）
  const response2 = await chat.sendMessage({
    message: "具体的なコード例を見せてください。",
  });
  console.log("AI:", response2.text);
}

chatDemo();
```

`Content` 型をインポートすることで、会話履歴の型安全性が確保されます。TypeScript ならではの恩恵として、`role` フィールドの誤入力や不正な構造がコンパイル時に検出されます。

## 構造化出力 — JSON スキーマで型安全なレスポンスを取得する

Gemini API の強力な機能の一つが、**構造化出力（Structured Output）**です。JSON スキーマを指定することで、API からの応答を常に決められた形式で受け取れます。

```typescript
// src/structured.ts
import { GoogleGenAI, Type } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

// TypeScript の型定義
interface BookRecommendation {
  title: string;
  author: string;
  reason: string;
  difficulty: "beginner" | "intermediate" | "advanced";
}

async function getStructuredOutput(): Promise<void> {
  const response = await ai.models.generateContent({
    model: "gemini-2.5-flash",
    contents: "TypeScript学習におすすめの書籍を3冊紹介してください。",
    config: {
      responseMimeType: "application/json",
      responseSchema: {
        type: Type.ARRAY,
        items: {
          type: Type.OBJECT,
          properties: {
            title: { type: Type.STRING, description: "書籍タイトル" },
            author: { type: Type.STRING, description: "著者名" },
            reason: { type: Type.STRING, description: "おすすめの理由" },
            difficulty: {
              type: Type.STRING,
              enum: ["beginner", "intermediate", "advanced"],
              description: "難易度",
            },
          },
          required: ["title", "author", "reason", "difficulty"],
        },
      },
    },
  });

  const books: BookRecommendation[] = JSON.parse(response.text ?? "[]");
  books.forEach((book) => {
    console.log(`📚 ${book.title}（${book.author}）`);
    console.log(`   難易度: ${book.difficulty}`);
    console.log(`   理由: ${book.reason}\n`);
  });
}

getStructuredOutput();
// 出力例:
// 📚 プログラミングTypeScript（Boris Cherny）
//    難易度: intermediate
//    理由: 型システムの深い理解に最適な一冊
```

この構造化出力と TypeScript の型定義を組み合わせることで、API レスポンスを安全にパースし、下流の処理に渡すことができます。さらに詳しい実装パターンを知りたい方は、[Gemini API 構造化出力の完全ガイド](/articles/gemini-api/gemini-structured-output-advanced)で高度なスキーマ設計のテクニックを詳しく解説しています。

## エラーハンドリング — 本番を見据えた堅牢な実装

API 呼び出しには必ずエラー処理を組み込みましょう。TypeScript のカスタムエラー型を活用すると、エラーの種別に応じた適切な処理が書けます。

```typescript
// src/error-handling.ts
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function safeGenerate(prompt: string): Promise<string | null> {
  const MAX_RETRIES = 3;

  for (let attempt = 1; attempt <= MAX_RETRIES; attempt++) {
    try {
      const response = await ai.models.generateContent({
        model: "gemini-2.5-flash",
        contents: prompt,
      });
      return response.text ?? null;
    } catch (error: unknown) {
      if (error instanceof Error) {
        // レート制限エラーの場合はリトライ
        if (error.message.includes("429")) {
          const waitMs = Math.min(1000 * Math.pow(2, attempt), 30000);
          console.warn(
            `レート制限 — ${waitMs}ms 後にリトライ（${attempt}/${MAX_RETRIES}）`
          );
          await new Promise((r) => setTimeout(r, waitMs));
          continue;
        }
        // API キーエラー
        if (error.message.includes("401") || error.message.includes("403")) {
          console.error("認証エラー: APIキーを確認してください。");
          return null;
        }
      }
      console.error(`予期しないエラー（試行 ${attempt}）:`, error);
    }
  }
  console.error("最大リトライ回数に到達しました。");
  return null;
}

// 使用例
(async () => {
  const result = await safeGenerate("Gemini APIの最大の強みは何ですか？");
  if (result) {
    console.log(result);
  }
})();
```

指数バックオフ付きのリトライパターンは、レート制限や一時的なネットワークエラーに対する基本的な対策です。本番環境では、このパターンをベースに、ログ収集やアラート通知を追加していくとよいでしょう。

## よくある質問（FAQ）

### Q1. Gemini API の TypeScript SDK はどこで入手できますか？

`@google/genai` パッケージを npm からインストールします。このパッケージには TypeScript の型定義が同梱されており、追加で `@types` パッケージをインストールする必要はありません。`npm install @google/genai` でセットアップは完了です。

### Q2. Gemini API は無料で使えますか？

Google AI Studio 経由で取得する API キーには無料枠が用意されています。Gemini 2.5 Flash は毎分15リクエスト、1日1,500リクエストまで無料で利用可能です（2026年3月時点）。本格的な開発には有料プランへの移行が推奨されますが、学習やプロトタイピングには十分な無料枠が提供されています。

### Q3. TypeScript と JavaScript のどちらで開発すべきですか？

TypeScript を強くおすすめします。Gemini API のレスポンスは複雑なネスト構造を持つことがあり、型定義があることでプロパティの参照ミスやフィールドの欠落をコンパイル時に検出できます。特に構造化出力を利用する場合、TypeScript の型システムが開発効率と安全性を大きく向上させます。

### Q4. Node.js 以外の環境（Deno / Bun）でも使えますか？

`@google/genai` パッケージは Deno と Bun でも動作します。Deno の場合は `npm:@google/genai` として import し、Bun の場合は `bun add @google/genai` でインストールできます。ただし、公式にサポートされているのは Node.js 環境です。
