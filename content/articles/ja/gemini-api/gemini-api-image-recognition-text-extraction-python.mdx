## はじめに

写真に映った看板の文字を読み取りたい、名刺の情報をデータベースに自動登録したい、レシートの金額を集計したい——こうした「画像からテキストを抽出する」ニーズは、ビジネスでも個人開発でも非常に多いものです。

従来のOCR（光学文字認識）は専用ライブラリの導入や前処理に手間がかかりましたが、Gemini APIのマルチモーダル機能を使えば、Python数行のコードで高精度な画像認識とテキスト抽出を実現できます。Gemini は画像の「文脈」を理解するため、単純な文字読み取りだけでなく、レイアウトの把握や要約まで一度に処理できるのが大きな強みです。

この記事では、Google AI Python SDK（`google-genai`）を使って、画像認識・テキスト抽出を実装する方法をゼロからわかりやすく解説します。初めて Gemini API を触る方でも、この記事を読み終える頃には実用的な画像解析ツールを作れるようになります。

## 環境準備と API キーの取得

まず、開発に必要な環境を整えましょう。

**必要なもの:**

- Python 3.10 以上
- Google AI Studio のアカウント（無料）
- Gemini API キー

**API キーの取得手順:**

1. [Google AI Studio](https://aistudio.google.com/) にアクセスしてログイン
2. 左メニューの「Get API key」をクリック
3. 「Create API key」で新しいキーを発行

**SDK のインストール:**

```bash
pip install google-genai pillow
```

`google-genai` が Gemini API の公式 Python SDK、`pillow` は画像の読み込みと前処理に使います。

**API キーの設定:**

```python
import os
os.environ["GOOGLE_API_KEY"] = "YOUR_API_KEY"
```

セキュリティ上、APIキーはコードにハードコードせず、環境変数や `.env` ファイルで管理するのが望ましい方法です。

## 画像からテキストを抽出する基本実装

最もシンプルな実装から始めましょう。1枚の画像を Gemini に送って、含まれるテキストを抽出します。

```python
from google import genai
from google.genai import types
import pathlib

# クライアントの初期化
client = genai.Client()

# 画像ファイルを読み込む
image_path = pathlib.Path("receipt.jpg")
image_data = image_path.read_bytes()

# Gemini API に画像を送信してテキストを抽出
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        types.Part.from_bytes(data=image_data, mime_type="image/jpeg"),
        "この画像に含まれるテキストをすべて正確に抽出してください。"
    ],
)

print(response.text)
```

**期待される出力（レシート画像の場合）:**

```
〇〇スーパーマーケット
東京都渋谷区〇〇 1-2-3
TEL: 03-XXXX-XXXX

2026/03/28 14:32
レジ#3 担当: 田中

牛乳 1L          ¥198
食パン 6枚切      ¥158
たまご 10個       ¥248
---
小計             ¥604
消費税(8%)        ¥48
合計             ¥652
```

このコードのポイントは、画像データとプロンプトを `contents` リストにまとめて送信しているだけということです。Gemini が画像の中身を認識し、テキスト部分を読み取って返してくれます。

## URL から直接画像を解析する

ローカルファイルだけでなく、Web上の画像URLから直接テキストを抽出することもできます。

```python
from google import genai
from google.genai import types

client = genai.Client()

# URLから画像を取得して解析
image_url = "https://example.com/business-card.png"

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        types.Part.from_uri(file_uri=image_url, mime_type="image/png"),
        "この名刺に書かれている情報を、名前・会社名・役職・電話番号・メールアドレスの項目に分けて抽出してください。",
    ],
)

print(response.text)
```

**期待される出力:**

```
- 名前: 山田太郎
- 会社名: 株式会社テクノロジーズ
- 役職: エンジニアリングマネージャー
- 電話番号: 090-XXXX-XXXX
- メールアドレス: taro.yamada@example.com
```

プロンプトで「項目に分けて」と指示するだけで、Gemini が構造化されたデータとして出力してくれます。これは従来のOCRライブラリにはない、LLMならではの強みです。

## 構造化データとして抽出する（JSON出力）

実務では、抽出したテキストをそのまま使うのではなく、JSONなどの構造化データとして取得したいケースが多いでしょう。Gemini API の `response_mime_type` を使えば、確実にJSON形式で受け取れます。

```python
from google import genai
from google.genai import types
import json

client = genai.Client()

image_path = "invoice.png"
with open(image_path, "rb") as f:
    image_data = f.read()

# JSON形式で構造化抽出
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        types.Part.from_bytes(data=image_data, mime_type="image/png"),
        """この請求書画像から以下の情報をJSON形式で抽出してください:
        - company_name: 請求元の会社名
        - invoice_number: 請求書番号
        - date: 発行日
        - items: 明細（品名、数量、単価、金額の配列）
        - subtotal: 小計
        - tax: 消費税
        - total: 合計金額""",
    ],
    config=types.GenerateContentConfig(
        response_mime_type="application/json",
    ),
)

# JSONをパースして利用
data = json.loads(response.text)
print(json.dumps(data, ensure_ascii=False, indent=2))
```

**期待される出力:**

```json
{
  "company_name": "株式会社サンプルテック",
  "invoice_number": "INV-2026-0328",
  "date": "2026-03-28",
  "items": [
    {
      "name": "Webサイト制作",
      "quantity": 1,
      "unit_price": 500000,
      "amount": 500000
    },
    {
      "name": "サーバー保守（月額）",
      "quantity": 3,
      "unit_price": 30000,
      "amount": 90000
    }
  ],
  "subtotal": 590000,
  "tax": 59000,
  "total": 649000
}
```

この方法を使えば、請求書や領収書の情報を自動的にデータベースに取り込むパイプラインを構築できます。

## 複数画像の一括処理

大量の画像を処理したい場合は、バッチ処理の仕組みを作りましょう。以下は、フォルダ内の画像をまとめて処理する実用的なスクリプトです。

```python
from google import genai
from google.genai import types
import pathlib
import json
import time

client = genai.Client()

def extract_text_from_image(image_path: pathlib.Path) -> dict:
    """画像からテキストを抽出して構造化データを返す"""
    image_data = image_path.read_bytes()

    # 拡張子からMIMEタイプを判定
    mime_map = {".jpg": "image/jpeg", ".jpeg": "image/jpeg",
                ".png": "image/png", ".webp": "image/webp"}
    mime_type = mime_map.get(image_path.suffix.lower(), "image/jpeg")

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=[
            types.Part.from_bytes(data=image_data, mime_type=mime_type),
            "この画像に含まれるすべてのテキストを抽出し、JSON形式で返してください。"
            "キーは 'extracted_text'（全文）と 'summary'（内容の要約）にしてください。",
        ],
        config=types.GenerateContentConfig(
            response_mime_type="application/json",
        ),
    )

    result = json.loads(response.text)
    result["source_file"] = str(image_path.name)
    return result

# フォルダ内の画像を一括処理
image_dir = pathlib.Path("./images")
results = []

for img_file in sorted(image_dir.glob("*.{jpg,png,jpeg,webp}")):
    print(f"処理中: {img_file.name}")
    try:
        result = extract_text_from_image(img_file)
        results.append(result)
        # APIレート制限を考慮して少し待機
        time.sleep(1)
    except Exception as e:
        print(f"  エラー: {e}")
        results.append({"source_file": img_file.name, "error": str(e)})

# 結果をJSONファイルに保存
output_path = pathlib.Path("extraction_results.json")
output_path.write_text(json.dumps(results, ensure_ascii=False, indent=2))
print(f"\n完了: {len(results)} 件の結果を {output_path} に保存しました")
```

処理対象が数百枚を超える場合は、[Gemini Batch Processing API 完全ガイド](/articles/gemini-api/gemini-batch-processing-api-guide) で解説している非同期バッチ処理の利用を検討してみてください。大量リクエストを効率的に処理でき、コストも50%削減できます。

## 画像認識の精度を上げるプロンプトテクニック

Gemini の画像認識精度は、プロンプトの書き方で大きく変わります。以下のテクニックを押さえておきましょう。

**言語を明示する:**

```python
# 日本語と英語が混在する画像の場合
prompt = "この画像に含まれるテキストを、日本語と英語を区別してすべて抽出してください。"
```

**レイアウト情報を保持する:**

```python
# 表形式のデータを正確に読み取りたい場合
prompt = """この画像はスプレッドシートのスクリーンショットです。
表のレイアウト（行と列の関係）を正確に保ったまま、
すべてのセルの内容をCSV形式で出力してください。
ヘッダー行も含めてください。"""
```

**手書き文字への対応:**

```python
# 手書きメモや手紙の場合
prompt = """この画像には手書きの文字が含まれています。
できるだけ正確に読み取ってください。
読み取りに自信がない部分は [?] で示してください。"""
```

**画像の種類を伝える:**

プロンプトで「この画像はレシートです」「名刺の写真です」「ホワイトボードの写真です」のように種類を伝えると、Gemini がコンテキストを理解して認識精度が上がります。

より高度なマルチモーダル活用テクニック（動画解析・音声入力との組み合わせなど）については、[Gemini API マルチモーダル実践テクニック集](/articles/gemini-api/gemini-api-multimodal-techniques) で体系的に解説しています。

## よくある質問（FAQ）

### Q: Gemini API で処理できる画像の最大サイズは？

A: Gemini API は 1 リクエストあたり最大 20MB の画像を処理できます。対応フォーマットは JPEG、PNG、WebP、HEIC、HEIF です。大きな画像は送信前に `pillow` でリサイズすると、APIの応答速度も向上します。

### Q: 従来のOCRライブラリ（Tesseract等）と比べた利点は？

A: Gemini の最大の利点は「文脈理解」です。従来のOCRは文字を1つずつ認識するだけですが、Gemini はドキュメント全体のレイアウトを理解し、テーブルの構造や段落の関係を把握した上でテキストを抽出します。また、手書き文字や低品質の画像でも高い認識率を発揮します。さらに、プロンプト次第で「要約して」「JSONにして」「翻訳して」といった後処理まで一度に実行できるため、後段のパイプラインがシンプルになります。

### Q: 無料枠でどのくらい処理できますか？

A: Gemini 2.5 Flash の無料枠では、1分あたり最大15リクエストを処理できます。画像1枚につき1リクエストなので、1日あたり数百枚の処理が十分に可能です。大量処理が必要な場合は、有料プランに切り替えることでレートリミットを緩和できます。料金の詳細は [Google AI Studio](https://aistudio.google.com/) で確認できます。

### Q: 日本語の縦書きテキストも認識できますか？

A: はい、Gemini は日本語の縦書きテキストにも対応しています。プロンプトで「縦書きの日本語テキストを読み取ってください」と明示すると、より正確な結果が得られます。古い書体や特殊なフォントの場合は、認識精度が下がることがありますが、プロンプトに「旧字体が含まれる可能性があります」などの補足情報を加えると改善します。
