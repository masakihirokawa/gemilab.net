## はじめに — なぜVercel AI SDK × Gemini APIなのか

AIアプリケーション開発において、フロントエンドとバックエンドの統合は複雑になりがちです。ストリーミングレスポンスの処理、会話履歴の管理、エラーハンドリング——これらを一から実装するのは大きな負担です。

**Vercel AI SDK**は、このような課題を解決するオープンソースライブラリです。統一されたAPIを通じて複数のAIプロバイダーに接続でき、Gemini APIとの組み合わせでは特に強力な開発体験を提供します。

この記事では、Next.js 15（App Router）とVercel AI SDKを使って、Gemini APIを活用したストリーミング対応AIチャットアプリを構築する方法をステップバイステップで解説します。環境構築からデプロイまで、すぐに動くコードとともにお伝えします。

### この記事で学べること

- Vercel AI SDKの基本概念とアーキテクチャ
- `@ai-sdk/google`プロバイダーの設定方法
- `streamText`と`useChat`を使ったリアルタイムストリーミング
- Function Calling（ツール呼び出し）の統合
- Structured Output（構造化出力）の活用
- 本番運用のためのエラーハンドリングとベストプラクティス

## 前提知識と環境準備

### 必要なもの

- **Node.js 18以上**（推奨: 20 LTS）
- **Gemini APIキー**（[Google AI Studio](https://aistudio.google.com/)で無料取得可能）
- **Next.js**の基本知識（App Router）
- **TypeScript**の基本知識

### プロジェクトのセットアップ

まず、Next.jsプロジェクトを作成し、必要なパッケージをインストールします。

```bash
# Next.jsプロジェクトの作成
npx create-next-app@latest my-gemini-chat --typescript --tailwind --app
cd my-gemini-chat

# Vercel AI SDKとGeminiプロバイダーのインストール
npm install ai @ai-sdk/google
```

次に、環境変数ファイル`.env.local`を作成します。

```bash
# .env.local
GOOGLE_GENERATIVE_AI_API_KEY=your_api_key_here
```

> **ポイント**: Vercel AI SDKのGoogleプロバイダーは、`GOOGLE_GENERATIVE_AI_API_KEY`環境変数を自動的に読み込みます。コード内でAPIキーを直接指定する必要はありません。

## Vercel AI SDKの基本アーキテクチャ

Vercel AI SDKは3つのレイヤーで構成されています。

### AI SDK Core

サーバーサイドで動作する中核ライブラリです。`generateText`（テキスト生成）、`streamText`（ストリーミング生成）、`generateObject`（構造化出力）などの関数を提供します。

### AI SDK UI

フロントエンド向けのReact Hooksを提供します。`useChat`（チャットUI）、`useCompletion`（テキスト補完）など、状態管理を含むUIコンポーネントが揃っています。

### AI SDK Providers

各AIプロバイダーへの接続を抽象化するレイヤーです。Gemini APIには`@ai-sdk/google`パッケージを使用します。

```
┌─────────────────────────────────┐
│  フロントエンド (React)          │
│  useChat / useCompletion         │
│  ← AI SDK UI                    │
├─────────────────────────────────┤
│  APIルート (Next.js)             │
│  streamText / generateText       │
│  ← AI SDK Core                  │
├─────────────────────────────────┤
│  @ai-sdk/google                  │
│  ← AI SDK Provider              │
│  → Gemini API                    │
└─────────────────────────────────┘
```

## ステップ1 — シンプルなテキスト生成

まず、Gemini APIで基本的なテキスト生成を行うAPIルートを作成してみましょう。

```typescript
// app/api/generate/route.ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

export async function POST(req: Request) {
  const { prompt } = await req.json();

  // generateTextでGemini 3 Proにリクエスト
  const { text } = await generateText({
    model: google('gemini-3-pro'),
    prompt,
  });

  return Response.json({ text });
}
```

たったこれだけのコードで、Gemini APIとの連携が完了します。`google()`関数にモデル名を渡すだけで、認証やリクエスト形式の変換はすべてSDKが処理します。

## ステップ2 — ストリーミングチャットの実装

次に、リアルタイムストリーミングに対応したチャットアプリを構築します。これがVercel AI SDKの真価が発揮される部分です。

### サーバーサイド（APIルート）

```typescript
// app/api/chat/route.ts
import { google } from '@ai-sdk/google';
import { streamText } from 'ai';

export const maxDuration = 30; // Vercelのタイムアウト設定

export async function POST(req: Request) {
  const { messages } = await req.json();

  // streamTextでストリーミングレスポンスを生成
  const result = streamText({
    model: google('gemini-3-pro'),
    system: 'あなたは親切で知識豊富なAIアシスタントです。日本語で回答してください。',
    messages,
  });

  // toDataStreamResponseでAI SDK互換のストリームを返す
  return result.toDataStreamResponse();
}
```

### クライアントサイド（チャットUI）

```tsx
// app/chat/page.tsx
'use client';

import { useChat } from '@ai-sdk/react';

export default function ChatPage() {
  const { messages, input, handleInputChange, handleSubmit, isLoading } =
    useChat({
      api: '/api/chat',
    });

  return (
    <div className="max-w-2xl mx-auto p-4">
      <h1 className="text-2xl font-bold mb-4">Gemini AI チャット</h1>

      {/* メッセージ一覧 */}
      <div className="space-y-4 mb-4">
        {messages.map((m) => (
          <div
            key={m.id}
            className={`p-3 rounded-lg ${
              m.role === 'user'
                ? 'bg-blue-100 ml-12'
                : 'bg-gray-100 mr-12'
            }`}
          >
            <p className="text-sm font-semibold mb-1">
              {m.role === 'user' ? 'あなた' : 'Gemini'}
            </p>
            <p className="whitespace-pre-wrap">{m.content}</p>
          </div>
        ))}
      </div>

      {/* 入力フォーム */}
      <form onSubmit={handleSubmit} className="flex gap-2">
        <input
          value={input}
          onChange={handleInputChange}
          placeholder="メッセージを入力..."
          className="flex-1 p-2 border rounded-lg"
          disabled={isLoading}
        />
        <button
          type="submit"
          disabled={isLoading}
          className="px-4 py-2 bg-blue-500 text-white rounded-lg disabled:opacity-50"
        >
          {isLoading ? '生成中...' : '送信'}
        </button>
      </form>
    </div>
  );
}
```

`useChat`フックが、ストリーミングレスポンスの受信・パース・状態更新をすべて自動処理します。会話履歴の管理も組み込まれているため、マルチターンの対話がそのまま動作します。

## ステップ3 — Function Calling（ツール呼び出し）の統合

Gemini APIの強力な機能であるFunction Callingも、Vercel AI SDKで簡潔に実装できます。

```typescript
// app/api/chat-with-tools/route.ts
import { google } from '@ai-sdk/google';
import { streamText, tool } from 'ai';
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: google('gemini-3-pro'),
    system: 'あなたは天気情報を提供できるAIアシスタントです。',
    messages,
    tools: {
      // 天気情報を取得するツール
      getWeather: tool({
        description: '指定された都市の現在の天気情報を取得します',
        parameters: z.object({
          city: z.string().describe('都市名（例: 東京、大阪）'),
          unit: z.enum(['celsius', 'fahrenheit']).default('celsius'),
        }),
        execute: async ({ city, unit }) => {
          // 実際のAPIコールに置き換えてください
          const weatherData = {
            city,
            temperature: unit === 'celsius' ? 22 : 72,
            unit,
            condition: '晴れ',
            humidity: 45,
          };
          return weatherData;
        },
      }),
      // 為替レートを取得するツール
      getExchangeRate: tool({
        description: '通貨ペアの為替レートを取得します',
        parameters: z.object({
          from: z.string().describe('変換元通貨コード（例: USD）'),
          to: z.string().describe('変換先通貨コード（例: JPY）'),
        }),
        execute: async ({ from, to }) => {
          // 実際のAPIコールに置き換えてください
          return { from, to, rate: 148.5, timestamp: new Date().toISOString() };
        },
      }),
    },
    maxSteps: 5, // ツール呼び出しの最大回数
  });

  return result.toDataStreamResponse();
}
```

`tool()`関数でツールを定義し、`z`（Zod）でパラメータのスキーマを指定します。`maxSteps`を設定することで、Geminiがツールの結果を受け取り、さらに推論を続ける「マルチステップ」の処理が可能になります。

## ステップ4 — Structured Output（構造化出力）

APIレスポンスを型安全なオブジェクトとして受け取りたい場合、`generateObject`を使います。

```typescript
// app/api/analyze/route.ts
import { google } from '@ai-sdk/google';
import { generateObject } from 'ai';
import { z } from 'zod';

// レスポンスのスキーマを定義
const SentimentSchema = z.object({
  sentiment: z.enum(['positive', 'negative', 'neutral']),
  confidence: z.number().min(0).max(1),
  keywords: z.array(z.string()).describe('テキストのキーワード'),
  summary: z.string().describe('テキストの要約（50文字以内）'),
});

export async function POST(req: Request) {
  const { text } = await req.json();

  // generateObjectで構造化された結果を取得
  const { object } = await generateObject({
    model: google('gemini-3-pro'),
    schema: SentimentSchema,
    prompt: `以下のテキストの感情分析を行ってください:\n\n${text}`,
  });

  // objectはSentimentSchema型として型安全にアクセス可能
  return Response.json(object);
}
```

`generateObject`はGeminiの構造化出力機能を活用し、指定したZodスキーマに完全に準拠したJSONを返します。TypeScriptの型推論も効くため、フロントエンドでのデータ処理が格段に楽になります。

## ステップ5 — マルチモーダル入力の処理

Geminiの強みであるマルチモーダル機能も、Vercel AI SDKで簡単に扱えます。

```typescript
// app/api/vision/route.ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

export async function POST(req: Request) {
  const formData = await req.formData();
  const image = formData.get('image') as File;
  const question = formData.get('question') as string;

  // 画像をBase64に変換
  const bytes = await image.arrayBuffer();
  const base64 = Buffer.from(bytes).toString('base64');

  const { text } = await generateText({
    model: google('gemini-3-pro'),
    messages: [
      {
        role: 'user',
        content: [
          {
            type: 'image',
            image: `data:${image.type};base64,${base64}`,
          },
          {
            type: 'text',
            text: question || 'この画像について説明してください。',
          },
        ],
      },
    ],
  });

  return Response.json({ text });
}
```

メッセージの`content`配列にテキストと画像を混在させるだけで、Geminiのマルチモーダル推論が利用できます。

## 本番運用のためのベストプラクティス

### エラーハンドリング

```typescript
// lib/error-handler.ts
import { APICallError } from 'ai';

export function handleAIError(error: unknown): Response {
  if (error instanceof APICallError) {
    // レート制限エラー
    if (error.statusCode === 429) {
      return Response.json(
        { error: 'リクエストが多すぎます。しばらく待ってから再試行してください。' },
        { status: 429 }
      );
    }
    // 認証エラー
    if (error.statusCode === 401) {
      return Response.json(
        { error: 'APIキーが無効です。' },
        { status: 401 }
      );
    }
  }
  // その他のエラー
  console.error('AI API Error:', error);
  return Response.json(
    { error: '予期しないエラーが発生しました。' },
    { status: 500 }
  );
}
```

### モデルの動的切り替え

```typescript
// lib/model-config.ts
import { google } from '@ai-sdk/google';

export function getModel(useCase: 'chat' | 'analysis' | 'fast') {
  switch (useCase) {
    case 'chat':
      return google('gemini-3-pro');       // 高品質な対話
    case 'analysis':
      return google('gemini-3-pro', {
        structuredOutputs: true,            // 構造化出力の最適化
      });
    case 'fast':
      return google('gemini-3-flash');      // 高速レスポンス
    default:
      return google('gemini-3-flash');
  }
}
```

## よくある質問（FAQ）

### Q1: Vercel AI SDKは無料で使えますか？

はい、Vercel AI SDK自体は完全にオープンソース（Apache 2.0ライセンス）で無料です。ただし、Gemini APIの利用には、無料枠を超えた場合に課金が発生します。[Google AI Studioの料金ページ](https://ai.google.dev/pricing)で最新の料金を確認してください。

### Q2: Vercel以外のプラットフォームにデプロイできますか？

はい、可能です。Vercel AI SDKは「Vercel」の名前がついていますが、Next.jsが動作する環境であればどこでもデプロイできます。Cloudflare Workers、AWS Lambda、セルフホスティングなど、お好みのプラットフォームを選択できます。

### Q3: Gemini 3 ProとGemini 3 Flashのどちらを使うべきですか？

用途によって異なります。対話品質や複雑な推論が求められる場合はGemini 3 Pro、低レイテンシやコスト効率を重視する場合はGemini 3 Flashがおすすめです。Vercel AI SDKでは`google('gemini-3-flash')`のようにモデル名を変えるだけで切り替えられるため、ユースケースに応じた使い分けが容易です。詳しくは「[Gemini 3 Pro完全ガイド](/articles/gemini-api/gemini-3-pro-guide)」をご参照ください。

### Q4: Function CallingとStructured Outputの違いは何ですか？

Function Callingは、モデルが外部のツールやAPIを呼び出すための仕組みです。一方、Structured Outputは、モデルの応答を指定したJSONスキーマに準拠させる機能です。前者は「モデルにアクションを実行させたい」場合、後者は「型安全なデータを取得したい」場合に使います。両方の活用法については「[Gemini APIツール呼び出し上級ガイド](/articles/gemini-api/gemini-api-tool-use-advanced)」もご覧ください。

## まとめ

Vercel AI SDKとGemini APIの組み合わせは、TypeScript/Next.js開発者にとって非常に強力な選択肢です。統一されたAPI設計により、ストリーミング、Function Calling、構造化出力、マルチモーダル入力といったGeminiの高度な機能を、最小限のコードで活用できます。

特に`useChat`フックによるフロントエンド統合は、従来のWebSocket実装やSSE処理を大幅に簡素化します。プロトタイプから本番アプリケーションまで、一貫した開発体験で効率的にAIアプリを構築できるでしょう。

Gemini APIの基本をさらに学びたい方は「[Gemini APIクイックスタート](/articles/gemini-api/gemini-api-quickstart)」から始めてみてください。また、Python環境での開発に興味がある方は「[Gemini API × Python自動化レシピ10選](/articles/gemini-api/gemini-api-python-automation-recipes)」も参考になるはずです。
