## はじめに — なぜDiscordでGemini AIボットを作るのか

Discordは世界中で2億人以上のアクティブユーザーを抱えるコミュニケーションプラットフォームです。ゲームコミュニティだけでなく、開発者コミュニティや学習グループ、ビジネスチームでも広く使われています。

このDiscordにGemini APIを搭載したAIチャットボットを導入すれば、メンバーの質問に24時間対応したり、画像を分析して回答したり、コードレビューを自動化したりと、コミュニティの価値を大幅に高めることができます。

この記事では、PythonのDiscord.pyライブラリとGoogle Gemini APIを組み合わせて、以下の機能を持つボットを構築する方法をステップバイステップで解説します。

- テキストメッセージへのAI応答
- 画像添付ファイルのマルチモーダル分析
- 会話履歴を保持したコンテキスト対話
- スラッシュコマンドによる機能拡張

Gemini APIの基本的な使い方については「[Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart)」も参考にしてください。

## 前提知識と環境準備

### 必要なもの

- **Python 3.10以上**がインストールされた環境
- **Googleアカウント**（Gemini API キー取得用）
- **Discordアカウント**（Bot作成用）
- テキストエディタまたはIDE（VS Code推奨）

### Gemini APIキーの取得

[Google AI Studio](https://aistudio.google.com/) にアクセスし、左メニューから「Get API key」をクリックして新しいAPIキーを作成します。無料枠でも十分にボット開発が可能です。

### Discord Botの作成

1. [Discord Developer Portal](https://discord.com/developers/applications) にアクセス
2. 「New Application」をクリックし、ボット名を入力
3. 左メニューの「Bot」セクションで「Add Bot」をクリック
4. 「MESSAGE CONTENT INTENT」を有効化（メッセージ内容を読み取るために必須）
5. ボットトークンをコピーして安全な場所に保存

### Botの招待URL生成

「OAuth2」→「URL Generator」で以下のスコープと権限を設定します。

- **Scopes**: `bot`, `applications.commands`
- **Bot Permissions**: `Send Messages`, `Read Message History`, `Attach Files`, `Use Slash Commands`

生成されたURLからボットをサーバーに招待します。

## 基本的なボットの構築

まずはシンプルなテキスト応答ボットから始めましょう。メンションされたときにGemini APIを使って回答するボットを作ります。

```python
# bot.py
import os
import discord
from discord.ext import commands
from google import genai
from dotenv import load_dotenv

load_dotenv()

# Gemini クライアントの初期化
client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))

# Discord Bot の設定
intents = discord.Intents.default()
intents.message_content = True  # メッセージ内容の読み取りを許可
bot = commands.Bot(command_prefix="!", intents=intents)

# 使用するモデル
MODEL_NAME = "gemini-2.5-flash"

@bot.event
async def on_ready():
    print(f"✅ {bot.user} としてログインしました")
    # スラッシュコマンドを同期
    await bot.tree.sync()

@bot.event
async def on_message(message):
    # ボット自身のメッセージは無視
    if message.author == bot.user:
        return

    # メンションされた場合にのみ応答
    if bot.user.mentioned_in(message):
        # メンション部分を除去してプロンプトを取得
        prompt = message.content.replace(f"<@{bot.user.id}>", "").strip()

        if not prompt:
            await message.reply("何かご質問はありますか？お気軽にどうぞ。")
            return

        async with message.channel.typing():
            try:
                response = client.models.generate_content(
                    model=MODEL_NAME,
                    contents=prompt,
                )
                reply_text = response.text

                # Discord のメッセージ上限は2000文字
                if len(reply_text) > 1900:
                    # 長い回答は分割して送信
                    chunks = [reply_text[i:i+1900] for i in range(0, len(reply_text), 1900)]
                    for chunk in chunks:
                        await message.reply(chunk)
                else:
                    await message.reply(reply_text)

            except Exception as e:
                await message.reply(f"エラーが発生しました: {str(e)[:200]}")

    # コマンド処理を継続
    await bot.process_commands(message)

# ボットを起動
bot.run(os.getenv("DISCORD_TOKEN"))
```

ターミナルで `python bot.py` を実行すると、ボットがオンラインになります。Discordサーバーでボットをメンションして質問を送ると、Gemini APIが回答を生成します。

## マルチモーダル対応 — 画像認識機能の追加

Gemini APIの大きな強みはマルチモーダル対応です。ユーザーが画像を添付してメッセージを送った場合に、画像の内容を認識して回答する機能を追加しましょう。

```python
import io
import httpx

async def analyze_image(image_url: str, prompt: str) -> str:
    """画像URLからデータを取得し、Geminiで分析する"""
    async with httpx.AsyncClient() as http_client:
        img_response = await http_client.get(image_url)
        image_data = img_response.content
        content_type = img_response.headers.get("content-type", "image/png")

    # Gemini APIに画像とプロンプトを送信
    response = client.models.generate_content(
        model=MODEL_NAME,
        contents=[
            {
                "parts": [
                    {"text": prompt if prompt else "この画像の内容を説明してください。"},
                    {
                        "inline_data": {
                            "mime_type": content_type,
                            "data": image_data,
                        }
                    },
                ]
            }
        ],
    )
    return response.text

@bot.event
async def on_message(message):
    if message.author == bot.user:
        return

    if bot.user.mentioned_in(message):
        prompt = message.content.replace(f"<@{bot.user.id}>", "").strip()

        async with message.channel.typing():
            try:
                # 画像添付がある場合はマルチモーダル処理
                if message.attachments:
                    attachment = message.attachments[0]
                    if attachment.content_type and attachment.content_type.startswith("image/"):
                        result = await analyze_image(attachment.url, prompt)
                        await message.reply(result)
                        await bot.process_commands(message)
                        return

                # テキストのみの場合
                if not prompt:
                    await message.reply("何かご質問はありますか？お気軽にどうぞ。")
                    await bot.process_commands(message)
                    return

                response = client.models.generate_content(
                    model=MODEL_NAME,
                    contents=prompt,
                )
                await message.reply(response.text[:1900])

            except Exception as e:
                await message.reply(f"エラーが発生しました: {str(e)[:200]}")

    await bot.process_commands(message)
```

これで、画像を添付してメンションすると、Geminiが画像の内容を分析して回答します。コードのスクリーンショットを送ってレビューを依頼する、料理の写真からレシピを推測させるなど、さまざまな活用が可能です。

## 会話履歴の保持 — コンテキストを理解する対話

単発の質問応答だけでなく、前の会話を踏まえた対話ができるとボットの実用性が大幅に向上します。チャンネルごとに会話履歴を保持する仕組みを実装しましょう。

```python
from collections import defaultdict, deque

# チャンネルごとの会話履歴（最大10ターン保持）
conversation_history = defaultdict(lambda: deque(maxlen=10))

def build_chat_contents(channel_id: str, new_message: str) -> list:
    """会話履歴を含むcontentsリストを構築する"""
    history = conversation_history[channel_id]
    contents = []

    for role, text in history:
        contents.append({"role": role, "parts": [{"text": text}]})

    contents.append({"role": "user", "parts": [{"text": new_message}]})
    return contents

async def chat_with_history(channel_id: str, user_message: str) -> str:
    """会話履歴を考慮してGeminiに問い合わせる"""
    contents = build_chat_contents(channel_id, user_message)

    response = client.models.generate_content(
        model=MODEL_NAME,
        contents=contents,
        config={
            "system_instruction": (
                "あなたはDiscordサーバーのフレンドリーなAIアシスタントです。"
                "簡潔でわかりやすい回答を心がけてください。"
                "コードを含む回答の場合は、Markdownのコードブロックを使ってください。"
            ),
        },
    )

    reply = response.text

    # 会話履歴に追加
    conversation_history[channel_id].append(("user", user_message))
    conversation_history[channel_id].append(("model", reply))

    return reply
```

`conversation_history` は `deque(maxlen=10)` を使って最新10ターン分のみ保持します。これにより、メモリ消費を抑えつつ、文脈のある対話が実現できます。

## スラッシュコマンドの実装

Discord.pyのスラッシュコマンドを使うと、ユーザーが `/` で始まるコマンドから直感的にボット機能を呼び出せます。

```python
@bot.tree.command(name="ask", description="Gemini AIに質問する")
async def ask_command(interaction: discord.Interaction, question: str):
    """スラッシュコマンドで質問を受け付ける"""
    await interaction.response.defer()  # 処理中であることを通知

    try:
        reply = await chat_with_history(
            str(interaction.channel_id), question
        )
        if len(reply) > 1900:
            await interaction.followup.send(reply[:1900] + "...")
        else:
            await interaction.followup.send(reply)
    except Exception as e:
        await interaction.followup.send(f"エラーが発生しました: {str(e)[:200]}")

@bot.tree.command(name="clear", description="会話履歴をリセットする")
async def clear_command(interaction: discord.Interaction):
    """チャンネルの会話履歴をクリアする"""
    channel_id = str(interaction.channel_id)
    conversation_history[channel_id].clear()
    await interaction.response.send_message("会話履歴をリセットしました。")

@bot.tree.command(name="summarize", description="直近の会話を要約する")
async def summarize_command(interaction: discord.Interaction, count: int = 20):
    """チャンネルの直近メッセージを要約する"""
    await interaction.response.defer()

    # 直近のメッセージを取得
    messages = []
    async for msg in interaction.channel.history(limit=count):
        if not msg.author.bot:
            messages.append(f"{msg.author.display_name}: {msg.content}")

    messages.reverse()
    conversation_text = "\n".join(messages)

    prompt = f"以下のDiscordの会話を簡潔に要約してください:\n\n{conversation_text}"

    try:
        response = client.models.generate_content(
            model=MODEL_NAME,
            contents=prompt,
        )
        await interaction.followup.send(f"📝 **会話の要約**\n\n{response.text[:1800]}")
    except Exception as e:
        await interaction.followup.send(f"エラーが発生しました: {str(e)[:200]}")
```

`/ask` コマンドで質問、`/clear` で履歴リセット、`/summarize` で直近の会話要約ができます。スラッシュコマンドはDiscordの自動補完が効くため、ユーザーにとって使いやすいインターフェースになります。

## エラーハンドリングとレート制限対策

本番運用では、APIのレート制限やネットワークエラーへの対策が欠かせません。リトライロジックとエラーハンドリングを実装します。

```python
import asyncio
from google.api_core import exceptions as google_exceptions

async def safe_generate(prompt: str, max_retries: int = 3) -> str:
    """リトライ付きのGemini API呼び出し"""
    for attempt in range(max_retries):
        try:
            response = client.models.generate_content(
                model=MODEL_NAME,
                contents=prompt,
            )
            return response.text

        except google_exceptions.ResourceExhausted:
            # レート制限エラー — 指数バックオフで待機
            wait_time = 2 ** attempt
            print(f"⚠️ レート制限。{wait_time}秒後にリトライします...")
            await asyncio.sleep(wait_time)

        except google_exceptions.InvalidArgument as e:
            # 入力に問題がある場合はリトライしない
            return f"入力エラー: リクエスト内容を確認してください。"

        except Exception as e:
            if attempt < max_retries - 1:
                await asyncio.sleep(1)
            else:
                return f"申し訳ありません。現在応答できません。しばらくしてからもう一度お試しください。"

    return "リクエストの処理に失敗しました。時間をおいて再度お試しください。"
```

特に `ResourceExhausted`（429エラー）は無料枠では頻繁に発生する可能性があるため、指数バックオフによるリトライが重要です。Gemini APIのレート制限について詳しくは「[Gemini API レート制限・クォータ管理ガイド](/articles/gemini-api/gemini-api-rate-limiting-quota-management)」をご覧ください。

## 完成版のコード全体

ここまでの機能をまとめた完成版コードを示します。

```python
# bot.py — Gemini API × Discord.py 完成版
import os
import io
import asyncio
from collections import defaultdict, deque

import discord
import httpx
from discord.ext import commands
from google import genai
from google.api_core import exceptions as google_exceptions
from dotenv import load_dotenv

load_dotenv()

# --- 初期化 ---
client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))
MODEL_NAME = "gemini-2.5-flash"

intents = discord.Intents.default()
intents.message_content = True
bot = commands.Bot(command_prefix="!", intents=intents)

# チャンネルごとの会話履歴
conversation_history = defaultdict(lambda: deque(maxlen=10))

# --- ユーティリティ関数 ---
async def safe_generate(contents, system_instruction=None, retries=3):
    config = {}
    if system_instruction:
        config["system_instruction"] = system_instruction

    for attempt in range(retries):
        try:
            response = client.models.generate_content(
                model=MODEL_NAME,
                contents=contents,
                config=config if config else None,
            )
            return response.text
        except google_exceptions.ResourceExhausted:
            await asyncio.sleep(2 ** attempt)
        except Exception:
            if attempt == retries - 1:
                return "申し訳ありません。現在応答を生成できません。"
            await asyncio.sleep(1)
    return "リクエストの処理に失敗しました。"

# --- イベントハンドラ ---
@bot.event
async def on_ready():
    print(f"✅ {bot.user} としてログインしました")
    await bot.tree.sync()

@bot.event
async def on_message(message):
    if message.author == bot.user:
        return

    if bot.user.mentioned_in(message):
        prompt = message.content.replace(f"<@{bot.user.id}>", "").strip()
        channel_id = str(message.channel.id)

        async with message.channel.typing():
            # 画像添付の処理
            if message.attachments:
                att = message.attachments[0]
                if att.content_type and att.content_type.startswith("image/"):
                    async with httpx.AsyncClient() as http:
                        img = await http.get(att.url)
                    contents = [{
                        "parts": [
                            {"text": prompt or "この画像について説明してください。"},
                            {"inline_data": {
                                "mime_type": att.content_type,
                                "data": img.content,
                            }},
                        ]
                    }]
                    reply = await safe_generate(contents)
                    await message.reply(reply[:1900])
                    await bot.process_commands(message)
                    return

            if not prompt:
                await message.reply("何かご質問はありますか？")
                await bot.process_commands(message)
                return

            # 会話履歴を使った応答
            history = conversation_history[channel_id]
            contents = [{"role": r, "parts": [{"text": t}]} for r, t in history]
            contents.append({"role": "user", "parts": [{"text": prompt}]})

            reply = await safe_generate(
                contents,
                system_instruction="フレンドリーなAIアシスタントとして簡潔に回答してください。",
            )

            history.append(("user", prompt))
            history.append(("model", reply))

            if len(reply) > 1900:
                for i in range(0, len(reply), 1900):
                    await message.reply(reply[i:i+1900])
            else:
                await message.reply(reply)

    await bot.process_commands(message)

# --- スラッシュコマンド ---
@bot.tree.command(name="ask", description="Gemini AIに質問する")
async def ask_command(interaction: discord.Interaction, question: str):
    await interaction.response.defer()
    reply = await safe_generate(question)
    await interaction.followup.send(reply[:1900])

@bot.tree.command(name="clear", description="会話履歴をリセット")
async def clear_command(interaction: discord.Interaction):
    conversation_history[str(interaction.channel_id)].clear()
    await interaction.response.send_message("会話履歴をリセットしました。")

@bot.tree.command(name="summarize", description="直近の会話を要約")
async def summarize_command(interaction: discord.Interaction, count: int = 20):
    await interaction.response.defer()
    messages = []
    async for msg in interaction.channel.history(limit=count):
        if not msg.author.bot:
            messages.append(f"{msg.author.display_name}: {msg.content}")
    messages.reverse()
    prompt = f"以下のDiscordの会話を簡潔に要約してください:\n\n" + "\n".join(messages)
    reply = await safe_generate(prompt)
    await interaction.followup.send(f"📝 **会話の要約**\n\n{reply[:1800]}")

bot.run(os.getenv("DISCORD_TOKEN"))
```

期待される出力（ボット起動時）:

```

```

## よくある質問（FAQ）

### Q1. 無料のGemini APIでDiscordボットを運用できますか？

はい、Gemini APIの無料枠で十分に運用可能です。`gemini-2.5-flash` モデルは無料枠で1分あたり15リクエストまで利用でき、小〜中規模のDiscordサーバーであれば問題なく対応できます。ユーザー数が増えてきた場合は、従量課金プランへの移行を検討してください。

### Q2. ボットが応答しない場合のチェックポイントは？

まず確認すべきは以下の3点です。（1）Discord Developer Portalで「MESSAGE CONTENT INTENT」が有効になっているか、（2）`.env` ファイルのトークンとAPIキーが正しいか、（3）ボットに必要な権限（Send Messages、Read Message History）が付与されているか。それでも解決しない場合は、コンソールのエラーログを確認してください。

### Q3. 会話履歴はサーバー再起動後も保持されますか？

上記の実装ではメモリ上に保持しているため、再起動するとリセットされます。永続化が必要な場合は、SQLiteやRedisなどのデータストアに履歴を保存する実装に変更してください。`deque` の内容をJSON形式でファイルに書き出す簡易的な方法でも対応可能です。

### Q4. Function Callingを使ってボットの機能を拡張できますか？

はい、Gemini APIのFunction Calling機能を組み合わせれば、天気予報の取得、データベース検索、外部API連携などが可能です。詳しくは「[Function Calling ガイド](/articles/gemini-advanced/function-calling-guide)」をご覧ください。
