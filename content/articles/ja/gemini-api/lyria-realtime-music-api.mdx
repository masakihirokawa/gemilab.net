## はじめに：Lyria RealTimeとは

Google DeepMindが開発した**Lyria RealTime**は、Gemini APIエコシステムで利用できるリアルタイム音楽生成モデルです。テキストプロンプトを受け取り、エンドツーエンドでリアルタイムに音楽を連続ストリーミング生成する革新的な技術で、2026年現在も実験的APIとして無料で利用できます。

従来のAI音楽生成ツールが「数秒〜数十秒の音楽ファイルを一括生成」するのに対し、Lyria RealTimeは**理論上無限に続く音楽ストリーム**をリアルタイムで生成できます。ゲームのBGM、インタラクティブな音楽体験、ライブパフォーマンスのバッキングトラック生成など、幅広い用途で活用が期待されています。

この記事では、Lyria RealTime APIの基本から実践的なPython実装まで、ステップバイステップで解説します。

---

## Lyria RealTimeの主な特徴

Lyria RealTimeには他の音楽生成AIと差別化する特徴があります。

- **連続ストリーミング生成**: 最大10分間のセッションで途切れない音楽を生成
- **リアルタイム制御**: BPM・密度・音楽スタイルをリアルタイムで変更可能（変更から最大2秒で反映）
- **高品質オーディオ**: 48kHzステレオストリーム
- **SynthID透かし**: 生成された音楽には聴覚では検知できないSynthIDの電子透かしが自動埋め込まれる
- **無料利用**: 現在は実験的APIとして無料で提供中

## Step 1: 基本的な音楽生成（シンプルな実装）

まずはシンプルな実装から始めましょう。Google AI Python SDKを使って音楽ストリームを受信し、WAVファイルとして保存する例です。

```python
import asyncio
import wave
import os
from google import genai
from google.genai import types

# クライアント初期化
client = genai.Client(
    api_key=os.environ.get("GOOGLE_API_KEY"),
    http_options={"api_version": "v1alpha"}
)

async def generate_music_to_file(
    prompt: str,
    output_file: str = "output_music.wav",
    duration_seconds: int = 30
):
    """
    Lyria RealTimeでプロンプトから音楽を生成してWAVファイルに保存する

    Args:
        prompt: 音楽スタイルを指定するテキストプロンプト（英語推奨）
        output_file: 保存先のWAVファイルパス
        duration_seconds: 生成する秒数（最大600秒 = 10分）

    Returns:
        出力ファイルパス

    Expected output:
        "🎵 音楽生成中: upbeat jazz piano with smooth bass..."
        "✅ 30秒の音楽を output_music.wav に保存しました"
    """
    config = types.LiveMusicGenerationConfig(
        bpm=120,           # テンポ（60〜200 BPM）
        density=0.7,       # 音符の密度（0.0〜1.0）
        guidance=4.0,      # プロンプト追従度（0.0〜6.0）
    )

    # 48kHz ステレオ設定
    sample_rate = 48000
    channels = 2
    audio_data = bytearray()

    print(f"🎵 音楽生成中: {prompt[:50]}...")

    async with client.aio.live.music.connect(
        model="models/lyria-realtime-exp"
    ) as session:
        # 設定とプロンプトを送信
        await session.set_weighted_prompts(
            prompts=[
                types.WeightedPrompt(text=prompt, weight=1.0)
            ]
        )
        await session.set_music_generation_config(config=config)
        await session.play()

        # 指定秒数分のオーディオを収集
        target_bytes = sample_rate * channels * 2 * duration_seconds  # 16bit PCM

        async for message in session.receive():
            if message.server_content and message.server_content.audio_chunks:
                for chunk in message.server_content.audio_chunks:
                    audio_data.extend(chunk.data)
                    if len(audio_data) >= target_bytes:
                        break
            if len(audio_data) >= target_bytes:
                break

    # WAVファイルとして保存
    with wave.open(output_file, 'wb') as wf:
        wf.setnchannels(channels)
        wf.setsampwidth(2)  # 16bit
        wf.setframerate(sample_rate)
        wf.writeframes(bytes(audio_data[:target_bytes]))

    print(f"✅ {duration_seconds}秒の音楽を {output_file} に保存しました")
    return output_file

# 実行
asyncio.run(generate_music_to_file(
    prompt="upbeat jazz piano with smooth bass and light drums, lounge style",
    output_file="jazz_lounge.wav",
    duration_seconds=30
))
```

## Step 2: リアルタイムパラメータ制御

Lyria RealTimeの真骨頂は、生成中にパラメータをリアルタイムで変更できることです。たとえば、BPMを上げてテンポを加速させたり、音楽スタイルをシームレスに切り替えたりできます。

```python
import asyncio
import os
from google import genai
from google.genai import types

client = genai.Client(
    api_key=os.environ.get("GOOGLE_API_KEY"),
    http_options={"api_version": "v1alpha"}
)

async def dynamic_music_session():
    """
    セッション中にBPMとプロンプトをリアルタイム変更するデモ

    動作: 10秒ごとにテンポと音楽スタイルを段階的に変化させる

    Expected output:
        "🎸 フェーズ1: ゆったりとしたアコースティック (BPM: 80)"
        "⚡ フェーズ2: テンポアップ! (BPM: 120)"
        "🔥 フェーズ3: エネルギッシュなロック (BPM: 160)"
    """
    async with client.aio.live.music.connect(
        model="models/lyria-realtime-exp"
    ) as session:

        # フェーズ1: ゆったりとしたアコースティックギター
        print("🎸 フェーズ1: ゆったりとしたアコースティック (BPM: 80)")
        await session.set_weighted_prompts(prompts=[
            types.WeightedPrompt(
                text="slow acoustic guitar fingerpicking, gentle and calm",
                weight=1.0
            )
        ])
        await session.set_music_generation_config(
            config=types.LiveMusicGenerationConfig(bpm=80, density=0.4)
        )
        await session.play()
        await asyncio.sleep(10)  # 10秒間生成

        # フェーズ2: テンポアップ
        print("⚡ フェーズ2: テンポアップ! (BPM: 120)")
        await session.set_music_generation_config(
            config=types.LiveMusicGenerationConfig(bpm=120, density=0.6)
        )
        # ⚠️ BPM変更はコンテキストリセットが必要
        await session.reset_context()
        await session.play()
        await asyncio.sleep(10)

        # フェーズ3: エネルギッシュなロック
        print("🔥 フェーズ3: エネルギッシュなロック (BPM: 160)")
        await session.set_weighted_prompts(prompts=[
            types.WeightedPrompt(
                text="energetic electric guitar rock, heavy drums, powerful riffs",
                weight=1.0
            )
        ])
        await session.set_music_generation_config(
            config=types.LiveMusicGenerationConfig(bpm=160, density=0.9)
        )
        await session.reset_context()
        await session.play()
        await asyncio.sleep(10)

        print("✅ ダイナミック音楽セッション完了")

asyncio.run(dynamic_music_session())
```

## Step 3: 設定パラメータ詳細

Lyria RealTimeで制御できるパラメータを整理します。

### `LiveMusicGenerationConfig` のパラメータ

| パラメータ | 型 | 範囲 | デフォルト | 説明 |
|-----------|-----|------|----------|------|
| `bpm` | int | 60〜200 | - | テンポ（Beats Per Minute） |
| `density` | float | 0.0〜1.0 | - | 音符の密度（低=シンプル、高=複雑） |
| `guidance` | float | 0.0〜6.0 | 4.0 | プロンプトへの追従度 |

### `WeightedPrompt` のパラメータ

複数のプロンプトに重みを付けてブレンドできます。

```python
# 2つのスタイルをブレンドする例
await session.set_weighted_prompts(prompts=[
    types.WeightedPrompt(
        text="smooth jazz piano with saxophone",
        weight=0.7  # 70% ジャズ
    ),
    types.WeightedPrompt(
        text="cinematic orchestral strings, epic and emotional",
        weight=0.3  # 30% オーケストラ
    )
])
```

### コンテキストリセットが必要なケース

以下の変更時には `session.reset_context()` の呼び出しが必要です：

- **BPMの変更**（リズムの基本パターンが変わるため）
- **音楽スケール（調性）の変更**

プロンプトの変更や`density`の変更だけならリセット不要です。

## よくあるエラーと対処法

### エラー1: `WebSocketConnectionError`

```
ConnectionRefusedError: WebSocket connection to generativelanguage.googleapis.com failed
```

**対処法**: APIキーが正しいか確認。`GOOGLE_API_KEY`環境変数が設定されているか確認してください。また `http_options={"api_version": "v1alpha"}` が指定されているか確認してください。

### エラー2: セッションタイムアウト

10分（600秒）のセッション上限に達すると接続が切断されます。

```python
# 長時間の音楽生成には再接続ロジックを実装

    generated_chunks = []

    for session_num in range(total_minutes // 10 + 1):
        print(f"セッション {session_num + 1} 開始")
        async with client.aio.live.music.connect(
            model="models/lyria-realtime-exp"
        ) as session:
            await session.set_weighted_prompts(
                prompts=[types.WeightedPrompt(text=prompt, weight=1.0)]
            )
            await session.play()
            # 各セッション最大9分50秒
            # ... 音楽データ収集ロジック
```

### エラー3: `density`/`guidance`が反映されない

`set_music_generation_config`の後に必ず`play()`を呼び出してください。

## 応用：ゲームBGM自動生成システム

Lyria RealTimeの強みを活かした実践的な応用例として、ゲームの状況に応じてBGMをリアルタイム変化させるシステムを紹介します。

```python
import asyncio
import os
from enum import Enum
from google import genai
from google.genai import types

client = genai.Client(
    api_key=os.environ.get("GOOGLE_API_KEY"),
    http_options={"api_version": "v1alpha"}
)

class GameState(Enum):
    EXPLORATION = "exploration"
    COMBAT = "combat"
    BOSS = "boss"
    VICTORY = "victory"

# ゲーム状態ごとの音楽設定
GAME_MUSIC_CONFIGS = {
    GameState.EXPLORATION: {
        "prompt": "ambient exploration music, mysterious forest, gentle flute and strings",
        "bpm": 80,
        "density": 0.4
    },
    GameState.COMBAT: {
        "prompt": "intense battle music, fast percussion, electric guitar, urgent and action-packed",
        "bpm": 150,
        "density": 0.8
    },
    GameState.BOSS: {
        "prompt": "epic boss battle, massive orchestral, dramatic choir, thunderous drums",
        "bpm": 140,
        "density": 1.0
    },
    GameState.VICTORY: {
        "prompt": "triumphant victory fanfare, uplifting brass and strings, celebratory",
        "bpm": 120,
        "density": 0.6
    }
}

async def adaptive_game_bgm(state_sequence: list):
    """
    ゲーム状態に応じてBGMをリアルタイム切り替えするデモ

    Args:
        state_sequence: [(GameState, 秒数), ...] のリスト

    Expected output:
        "🌲 [EXPLORATION] 探索BGM開始 (BPM: 80)"
        "⚔️  [COMBAT] 戦闘BGMへ切り替え (BPM: 150)"
        "👹 [BOSS] ボスBGMへ切り替え (BPM: 140)"
        "🏆 [VICTORY] 勝利ファンファーレ (BPM: 120)"
    """
    emoji_map = {
        GameState.EXPLORATION: "🌲",
        GameState.COMBAT: "⚔️ ",
        GameState.BOSS: "👹",
        GameState.VICTORY: "🏆"
    }

    async with client.aio.live.music.connect(
        model="models/lyria-realtime-exp"
    ) as session:
        for state, duration in state_sequence:
            config = GAME_MUSIC_CONFIGS[state]
            print(f"{emoji_map[state]} [{state.value.upper()}] BGM (BPM: {config['bpm']})")

            await session.set_weighted_prompts(prompts=[
                types.WeightedPrompt(text=config["prompt"], weight=1.0)
            ])
            await session.set_music_generation_config(
                config=types.LiveMusicGenerationConfig(
                    bpm=config["bpm"],
                    density=config["density"]
                )
            )
            await session.reset_context()
            await session.play()
            await asyncio.sleep(duration)

# デモ実行
asyncio.run(adaptive_game_bgm([
    (GameState.EXPLORATION, 8),
    (GameState.COMBAT, 8),
    (GameState.BOSS, 8),
    (GameState.VICTORY, 6),
]))
```

## FAQ

**Q1. Lyria RealTimeは商用利用できますか？**

現在は実験的APIとして提供されており、商用利用ポリシーはGoogleの利用規約に準じます。生成された音楽にはSynthIDの透かしが埋め込まれています。商用プロジェクトへの組み込みを検討する場合は、最新の利用規約を必ず確認してください。

**Q2. Lyria 3とLyria RealTimeの違いは何ですか？**

Lyria 3はGeminiアプリで利用できる消費者向けモデルで、30秒の完成した楽曲（ボーカル付き、MP3/MP4形式）を生成します。Lyria RealTimeは開発者向けのAPIで、WebSocketを通じて連続するオーディオストリームをリアルタイム生成する技術です。Lyria 3の公開APIは2026年3月時点では未提供です。

**Q3. 日本語のプロンプトは使えますか？**

Lyria RealTimeは音楽生成モデルのため、プロンプトは**英語で指定することを強く推奨**します。「ジャズピアノ」のような単語より、「smooth jazz piano with saxophone, upbeat swing rhythm」のように具体的に英語で記述するほうが、意図通りの音楽が生成されやすくなります。

**Q4. モバイルアプリからLyria RealTimeを使えますか？**

WebSocketを通じたネットワーク接続が必要なため、サーバーサイドでの実行が基本です。モバイルアプリに組み込む場合は、バックエンドサーバーでLyria RealTimeに接続し、生成した音楽データをアプリに転送するアーキテクチャが推奨されます。

**Q5. セッション中にボーカルを追加できますか？**

現在のLyria RealTime APIはインストゥルメンタル音楽の生成に特化しており、ボーカル生成はサポートしていません。ボーカルを含む楽曲を生成したい場合はLyria 3（Geminiアプリ経由）を利用してください。
