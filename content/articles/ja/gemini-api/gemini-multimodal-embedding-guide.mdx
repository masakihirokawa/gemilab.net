## Gemini マルチモーダルエンベディング — 新時代の到来

2026年3月、Google は **gemini-embedding-2-preview** を発表しました。このモデルは、テキスト、画像、動画、音声、PDF を同一ベクトル空間にマッピングする、業界初のマルチモーダルエンベディングモデルです。従来は異なるモダリティ（データ種別）ごとに異なるモデルが必要でしたが、gemini-embedding-2 はこの制限を取り払いました。

## マルチモーダルエンベディングとは

### 基本概念

エンベディングとは、テキストや画像などのデータを、数百～数千次元の数値ベクトルに変換するプロセスです。マルチモーダルエンベディングは、複数の異なるデータ形式（テキスト、画像、動画など）を **同一のベクトル空間** に変換します。

```
テキスト: "日本の富士山の風景"
  ↓
[0.23, -0.54, 0.81, ..., 0.12]  ← 768次元ベクトル

画像: 富士山の写真
  ↓
[0.24, -0.53, 0.79, ..., 0.11]  ← 768次元ベクトル（ほぼ同じ！）

動画: 富士山の朝焼け映像
  ↓
[0.25, -0.52, 0.82, ..., 0.10]  ← 768次元ベクトル（近い！）
```

同一空間に変換されたため、テキストと画像、画像と動画の **セマンティック（意味的）な類似度** を計算できます。

### 従来との違い

| 特性 | 従来型エンベディング | gemini-embedding-2 |
|------|------------------|-----------------|
| 対応モダリティ | テキストのみ | テキスト・画像・動画・音声・PDF |
| ベクトル空間 | モダリティごとに異なる | 統一 |
| クロスモーダル検索 | 不可 | 可能 |
| セマンティック一致性 | モダリティ内のみ | 全モダリティ間で検証 |
| API 数 | 5+ 個必要 | 1 個で統一 |

<Callout type="info">
gemini-embedding-2 が実現する統一ベクトル空間は、異なるデータ形式間で意味的な関連性を検出する能力を持ちます。
</Callout>

## gemini-embedding-2 の仕様

### モデルパラメータ

- **モデル名**: `gemini-embedding-2-preview`（正式版は 2026年 6月予定）

## 実践的な応用例

### 1. RAG（Retrieval-Augmented Generation）の進化

従来の RAG は、テキスト文書のみを対象としていました。gemini-embedding-2 を使用すれば、テキスト・画像・動画すべてを検索対象とできます。

```python
import numpy as np
from google.generativeai import embed_content, generate_content

class MultimodalRAG:
    def __init__(self, documents, model="models/gemini-embedding-2-preview"):
        self.model = model
        self.documents = documents
        self.embeddings = self._embed_all_documents()

    def _embed_all_documents(self):
        """全ドキュメント（テキスト・画像・動画）をエンベディング化"""
        embeddings = []
        for doc in self.documents:
            if doc["type"] == "text":
                response = embed_content(
                    model=self.model,
                    content={"parts": [{"text": doc["content"]}]}
                )
            elif doc["type"] == "image":
                response = embed_content(
                    model=self.model,
                    content={
                        "parts": [{
                            "inline_data": {
                                "mime_type": "image/jpeg",
                                "data": doc["content"]
                            }
                        }]
                    }
                )
            elif doc["type"] == "video":
                response = embed_content(
                    model=self.model,
                    content={
                        "parts": [{
                            "file_data": {
                                "mime_type": "video/mp4",
                                "file_uri": doc["content"]
                            }
                        }]
                    }
                )
            embeddings.append(response["embedding"])
        return embeddings

    def retrieve(self, query, top_k=3):
        """クエリに最も関連する k 個のドキュメントを検索"""
        query_embedding = embed_content(
            model=self.model,
            content={"parts": [{"text": query}]}
        )["embedding"]

        # コサイン類似度で検索
        similarities = []
        for i, doc_embedding in enumerate(self.embeddings):
            similarity = np.dot(query_embedding, doc_embedding) / (
                np.linalg.norm(query_embedding) * np.linalg.norm(doc_embedding)
            )
            similarities.append((i, similarity))

        # 上位 k 個を返す
        top_indices = sorted(similarities, key=lambda x: x[1], reverse=True)[:top_k]
        return [self.documents[idx] for idx, _ in top_indices]

    def generate_with_rag(self, query):
        """RAG を用いて LLM で回答生成"""
        relevant_docs = self.retrieve(query)

        context = "参考資料:\n"
        for doc in relevant_docs:
            context += f"- [{doc['type']}] {doc['title']}\n"

        prompt = f"{context}\n質問: {query}"

        response = generate_content(
            model="gemini-pro",
            contents=[{"parts": [{"text": prompt}]}]
        )
        return response.text

# 使用例
documents = [
    {
        "type": "text",
        "title": "日本の春の風習",
        "content": "花見は日本の春を代表する伝統的な行事である..."
    },
    {
        "type": "image",
        "title": "京都の桜",
        "content": base64_cherry_blossom_kyoto
    },
    {
        "type": "video",
        "title": "東京の春の風景",
        "content": "gs://bucket/tokyo_spring.mp4"
    }
]

rag = MultimodalRAG(documents)
result = rag.generate_with_rag("日本の春について教えてください")
print(result)
```

<Callout type="success">
マルチモーダル RAG により、テキスト検索では見つからない関連画像・動画を同時に取得でき、より豊かな回答が生成できます。
</Callout>

### 3. マルチモーダルレコメンデーションシステム

ユーザーの閲覧履歴（テキスト記事、画像、動画）から類似コンテンツを推奨します。

```python
class MultimodalRecommender:
    def __init__(self, user_history, all_content):
        self.user_history = user_history  # ユーザーが見たコンテンツリスト
        self.all_content = all_content    # 全利用可能コンテンツ
        self.user_profile = self._build_user_profile()

    def _build_user_profile(self):
        """ユーザーの閲覧履歴から総合プロファイルベクトルを構築"""
        embeddings = []
        for item in self.user_history:
            emb = embed_content(
                model="models/gemini-embedding-2-preview",
                content=_prepare_doc_content(item)
            )["embedding"]
            embeddings.append(emb)

        # 平均ベクトルをプロファイルとする
        return np.mean(embeddings, axis=0)

    def recommend(self, top_k=10):
        """ユーザープロファイルに基づいて top_k 件のコンテンツを推奨"""
        recommendations = []

        for content in self.all_content:
            # 既に閲覧済みならスキップ
            if content["id"] in [h["id"] for h in self.user_history]:
                continue

            # コンテンツをエンベディング化
            content_embedding = embed_content(
                model="models/gemini-embedding-2-preview",
                content=_prepare_doc_content(content)
            )["embedding"]

            # ユーザープロファイルとの類似度
            similarity = np.dot(self.user_profile, content_embedding) / (
                np.linalg.norm(self.user_profile) * np.linalg.norm(content_embedding)
            )

            recommendations.append((content, similarity))

        return sorted(recommendations, key=lambda x: x[1], reverse=True)[:top_k]
```

### 4. 動画の自動チャプタリング

長時間の動画から、セマンティックに異なるシーンを自動検出し、チャプタ分割します。

```python
def auto_chapter_video(video_uri, chapter_duration_sec=300):
    """動画を自動的にチャプターに分割"""

    chapters = []
    current_chapter = {
        "start": 0,
        "frames": [],
        "embeddings": []
    }

    # 動画をフレーム抽出（省略：実装は動画処理ライブラリを使用）
    frames = extract_frames_from_video(video_uri, interval_sec=10)

    for frame in frames:
        # フレームをエンベディング化
        frame_emb = embed_content(
            model="models/gemini-embedding-2-preview",
            content={
                "parts": [{
                    "inline_data": {
                        "mime_type": "image/jpeg",
                        "data": frame["data"]
                    }
                }]
            }
        )["embedding"]

        current_chapter["frames"].append(frame)
        current_chapter["embeddings"].append(frame_emb)

        # チャプター長チェック
        if len(current_chapter["frames"]) >= chapter_duration_sec // 10:
            # チャプターの特性を計算（最初と最後のフレームの距離）
            first_emb = np.array(current_chapter["embeddings"][0])
            last_emb = np.array(current_chapter["embeddings"][-1])
            scene_change = 1 - np.dot(first_emb, last_emb) / (
                np.linalg.norm(first_emb) * np.linalg.norm(last_emb)
            )

            chapters.append({
                "start": current_chapter["start"],
                "end": frame["timestamp"],
                "scene_change_score": scene_change
            })

            current_chapter = {
                "start": frame["timestamp"],
                "frames": [],
                "embeddings": []
            }

    return chapters
```

## ベクトル検索データベースとの統合

gemini-embedding-2 で生成されたベクトルは、各種ベクトル検索データベースに直接統合できます。

### Pinecone との統合例

```python
import pinecone
from google.generativeai import embed_content

# Pinecone インデックスの初期化
pinecone.init(api_key="your-api-key")
index = pinecone.Index("gemini-multimodal")

# ドキュメントをエンベディング化してアップロード
def index_documents(documents):
    for i, doc in enumerate(documents):
        embedding = embed_content(
            model="models/gemini-embedding-2-preview",
            content=_prepare_doc_content(doc)
        )["embedding"]

        index.upsert(
            vectors=[
                (str(i), embedding, {
                    "type": doc["type"],
                    "title": doc["title"],
                    "source": doc["source"]
                })
            ]
        )

# クエリ実行
def search(query):
    query_embedding = embed_content(
        model="models/gemini-embedding-2-preview",
        content={"parts": [{"text": query}]}
    )["embedding"]

    results = index.query(
        vector=query_embedding,
        top_k=10,
        include_metadata=True
    )
    return results
```

<Callout type="warning">
ベクトル検索は近似検索であり、完全な正確性は保証されません。本番環境では、結果をフィルタリング・再ランキングすることをお勧めします。
</Callout>

## パフォーマンスと最適化

### レイテンシ最適化

```python
# バッチ処理で複数ドキュメントを同時エンベディング化
def batch_embed(documents, batch_size=32):
    """バッチ処理により API コストとレイテンシを最適化"""
    embeddings = []

    for i in range(0, len(documents), batch_size):
        batch = documents[i:i+batch_size]

        # 複数パートを単一リクエストで処理
        parts = []
        for doc in batch:
            if doc["type"] == "text":
                parts.append({"text": doc["content"]})
            elif doc["type"] == "image":
                parts.append({
                    "inline_data": {
                        "mime_type": "image/jpeg",
                        "data": doc["content"]
                    }
                })

        response = embed_content(
            model="models/gemini-embedding-2-preview",
            content={"parts": parts}
        )

        embeddings.extend(response["embeddings"])

    return embeddings
```

### コスト削減

- テキストの類似度計算のみが必要な場合は、引き続き専用テキストエンベディングモデルを使用
- 画像・動画は必要時のみエンベディング化（キャッシング推奨）
- バッチ処理で API コールを削減
