## media_resolutionとは？ — マルチモーダルAIの新しい制御パラメータ

Gemini 3シリーズでは、画像や動画を処理する際の**解像度レベル**を開発者が細かく制御できる `media_resolution` パラメータが導入されました。このパラメータを活用することで、トークン消費量・レイテンシ・認識精度のバランスを用途に応じて最適化できます。

従来のGemini 2.5シリーズでは、画像を送信するとモデルが自動的に解像度を決定していました。しかし、Gemini 3では開発者自身が `media_resolution_low`、`media_resolution_medium`、`media_resolution_high`、`media_resolution_ultra_high`（Flashのみ）の4段階から選択できるようになっています。

これにより、たとえば「簡単なサムネイル分類には低解像度で高速処理」「OCRや細かい図表の読み取りには高解像度で精密処理」といった使い分けが可能になりました。

## 解像度レベルとトークン消費量の関係

各解像度レベルが1枚の画像に割り当てるトークン数は以下のとおりです。

| 解像度レベル | トークン数（画像1枚） | 対応モデル |
|---|---|---|
| `media_resolution_low` | 約280トークン | Pro / Flash |
| `media_resolution_medium` | 約560トークン | Pro / Flash |
| `media_resolution_high` | 約1,120トークン | Pro / Flash |
| `media_resolution_ultra_high` | 約2,240トークン | Flashのみ |

画像のトークン消費は解像度レベルに対してほぼ線形にスケールします。一方、**動画フレームはより積極的に圧縮される**ため、動画処理の場合はさらにコスト効率が良くなります。

### コスト計算の具体例

たとえば、100枚の商品画像を分類するバッチ処理を考えてみましょう。

- **低解像度**: 100 × 280 = 28,000トークン
- **高解像度**: 100 × 1,120 = 112,000トークン

低解像度を選ぶだけで、**トークン消費を75%削減**できます。画像の内容が明確で細部の読み取りが不要な場合、低解像度でも十分な精度が得られることが多いです。

## 基本的な使い方 — GenerationConfigでデフォルト設定

最もシンプルな方法は、`GenerationConfig` でリクエスト全体のデフォルト解像度を設定する方法です。

```python
import google.generativeai as genai
from google.generativeai.types import GenerationConfig

# APIキーの設定
genai.configure(api_key="YOUR_API_KEY")

# モデルの初期化（デフォルト解像度を medium に設定）
model = genai.GenerativeModel(
    model_name="gemini-3-pro",
    generation_config=GenerationConfig(
        media_resolution="media_resolution_medium"
    )
)

# 画像を使った推論
image = genai.upload_file("product_photo.jpg")
response = model.generate_content([
    "この商品の特徴を3つ挙げてください。",
    image
])

print(response.text)
# 期待される出力例:
# 1. コンパクトなデザインで持ち運びに便利です
# 2. マットブラックの仕上げで高級感があります
# 3. USB-Cポートを搭載しており、急速充電に対応しています
```

この設定により、リクエスト内のすべての画像・動画が `media_resolution_medium`（約560トークン/枚）で処理されます。

## パーツ単位の解像度指定 — 1リクエストで複数の解像度を混在

Gemini 3の大きな特徴は、**1つのリクエスト内でパーツごとに異なる解像度を設定できる**点です。これにより、重要な画像には高解像度を、補助的な画像には低解像度を割り当てるといった柔軟な最適化が可能です。

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro")

# 詳細な図面は高解像度で処理
diagram = genai.upload_file("architecture_diagram.png")
diagram_part = genai.types.Part(
    file_data=diagram,
    media_resolution="media_resolution_high"
)

# 参考用のスクリーンショットは低解像度で処理
screenshot = genai.upload_file("app_screenshot.png")
screenshot_part = genai.types.Part(
    file_data=screenshot,
    media_resolution="media_resolution_low"
)

response = model.generate_content([
    "以下の2つの画像を比較してください。1枚目はシステム構成図、2枚目はアプリの画面です。",
    diagram_part,
    screenshot_part
])

print(response.text)
# 期待される出力例:
# システム構成図では、フロントエンドからAPIゲートウェイを経由して
# マイクロサービス群に接続するアーキテクチャが示されています。
# 一方、アプリ画面ではユーザーダッシュボードが表示されており...
```

この例では、構成図（1,120トークン）+ スクリーンショット（280トークン）= **合計1,400トークン**で処理されます。両方を高解像度にした場合の2,240トークンと比較して、約37%のトークン削減が実現できます。

## 動画処理での活用

動画を扱う場合も `media_resolution` パラメータが有効です。動画フレームは画像よりも積極的に圧縮されるため、コスト効率がさらに高くなります。

```python
import google.generativeai as genai
from google.generativeai.types import GenerationConfig

genai.configure(api_key="YOUR_API_KEY")

model = genai.GenerativeModel(
    model_name="gemini-3-flash",
    generation_config=GenerationConfig(
        media_resolution="media_resolution_low"
    )
)

# 動画ファイルのアップロード
video = genai.upload_file("meeting_recording.mp4")

# 動画の内容を要約
response = model.generate_content([
    "この会議動画の主要な議題と結論を箇条書きでまとめてください。",
    video
])

print(response.text)
# 期待される出力例:
# ■ 議題1: Q2の売上目標について
#   → 前年比15%増の目標を全員で合意
# ■ 議題2: 新製品のリリーススケジュール
#   → 6月末のリリースに向けてマイルストーンを確認
# ■ 議題3: カスタマーサポート体制の強化
#   → チャットボット導入を来月から試験運用
```

会議の要約など、映像の細部が重要でないケースでは `media_resolution_low` で十分です。

## ユースケース別の推奨設定

実際のプロジェクトで `media_resolution` をどう設定すべきか、代表的なユースケースごとの推奨値をまとめました。

### 低解像度（`media_resolution_low`）が適するケース

- 画像の大まかな分類（風景 / 人物 / 食べ物）
- サムネイル画像のフィルタリング
- 動画のシーン分割・要約
- SNS投稿画像のセンチメント分析

### 中解像度（`media_resolution_medium`）が適するケース

- 商品画像の説明文生成
- UI/UXデザインのレビュー
- プレゼンテーション資料の内容把握
- 一般的なドキュメント画像の読み取り

### 高解像度（`media_resolution_high`）が適するケース

- OCR（文書や名刺の文字認識）
- 技術図面・設計図の分析
- 医療画像の初期スクリーニング
- 細かい表やグラフのデータ抽出

### 超高解像度（`media_resolution_ultra_high` / Flashのみ）

- 高解像度の地図・衛星画像分析
- 微細な欠陥検出（製造ラインの品質管理）
- 極めて小さいフォントのOCR

## Pro と Flash の違い

`media_resolution` パラメータの対応状況はモデルによって異なります。

| 機能 | Gemini 3 Pro | Gemini 3 Flash |
|---|---|---|
| low / medium / high | ✅ | ✅ |
| ultra_high | ❌ | ✅ |
| パーツ単位の指定 | ✅ | ✅ |
| GenerationConfig指定 | ✅ | ✅ |

Flashは `ultra_high` をサポートしているため、最高精度が必要な画像処理にはFlashが適しています。一方、Proは言語理解や推論能力が高いため、画像から得た情報を基にした複雑な分析にはProが最適です。

用途に応じてモデルと解像度の組み合わせを選択しましょう。詳しいモデル選定については「[Geminiモデル選択ガイド](/articles/gemini-api/gemini-model-selection-guide)」も参考にしてください。

## トークンカウントAPIで事前にコストを確認する

本番環境でコストを管理するために、`count_tokens` APIを活用して事前にトークン数を確認できます。

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro")

image = genai.upload_file("document_scan.jpg")

# 各解像度でのトークン数を比較
for resolution in ["media_resolution_low", "media_resolution_medium", "media_resolution_high"]:
    part = genai.types.Part(
        file_data=image,
        media_resolution=resolution
    )
    token_count = model.count_tokens([
        "この書類の内容を要約してください。",
        part
    ])
    print(f"{resolution}: {token_count.total_tokens} tokens")

# 期待される出力例:
# media_resolution_low: 295 tokens
# media_resolution_medium: 575 tokens
# media_resolution_high: 1135 tokens
```

この方法を使えば、リクエストを送信する前にコストを正確に見積もることができます。トークンカウントの詳細については「[Gemini トークンカウントガイド](/articles/gemini-api/gemini-token-counting)」をご覧ください。

## よくある質問（FAQ）

### Q1: media_resolutionを指定しない場合、デフォルトの解像度はどうなりますか？

`media_resolution` を明示的に指定しない場合、Gemini 3はモデルのデフォルト設定で画像を処理します。一般的には `media_resolution_medium` 相当の処理が行われますが、入力画像のサイズやコンテンツによって自動調整される場合があります。コストを厳密に管理したい場合は、必ず明示的に解像度を指定することをおすすめします。

### Q2: 動画の場合、フレームごとに異なる解像度を設定できますか？

動画ファイル全体に対して1つの `media_resolution` が適用されます。フレーム単位での解像度指定はできません。ただし、重要なシーンだけを画像として切り出し、個別に高解像度で処理するという方法で対応できます。

### Q3: media_resolution_ultra_highはProモデルで使えないのですか？

現時点（2026年3月）では、`media_resolution_ultra_high` はGemini 3 Flashモデルでのみサポートされています。Proモデルで使用しようとするとエラーが返されます。超高解像度が必要な場合はFlashモデルを使用し、その結果を基にProモデルで高度な分析を行うパイプラインを構築するのが効果的です。

### Q4: Gemini 2.5シリーズでもmedia_resolutionは使えますか？

`GenerationConfig` を通じたデフォルト解像度の設定はGemini 2.5シリーズを含むすべてのマルチモーダルモデルで利用可能です。ただし、パーツ単位の細かい解像度指定はGemini 3シリーズの新機能です。

## まとめ — media_resolutionでコストと品質を両立する

Gemini 3の `media_resolution` パラメータは、マルチモーダルAPIを本番運用する上で非常に重要な最適化ツールです。用途に応じた適切な解像度レベルの選択、パーツ単位での柔軟な指定、そしてトークンカウントAPIによる事前のコスト確認を組み合わせることで、品質を維持しながらコストを大幅に削減できます。

まずは `count_tokens` APIで自分のユースケースに最適な解像度を検証し、段階的に本番環境へ導入していくことをおすすめします。Gemini APIのコスト管理についてさらに詳しく知りたい方は「[Gemini APIコスト最適化ガイド](/articles/gemini-api/gemini-api-cost-optimization)」もあわせてご覧ください。
