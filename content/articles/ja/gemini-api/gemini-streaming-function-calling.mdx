Gemini 3 シリーズのリリースとともに、**Function Calling（関数呼び出し）** 機能が大幅に強化されました。特に注目すべきは次の2つの新機能です。

1. **ストリーミング Function Calling** — 関数の引数をリアルタイムにストリーム受信できる
2. **マルチモーダル Function レスポンス** — 関数レスポンスに画像・PDFなどを含められる

この記事では、これらの新機能の仕組みと実際の使い方をコード例付きで解説します。従来の Function Calling の基礎知識がある方を対象としているため、まず基本を学びたい方は [Function Calling 基礎ガイド](/articles/gemini-api/function-calling-guide) を先に読むことをお勧めします。

---

## ストリーミング Function Calling とは

従来の Gemini API では、Function Calling を使う際、モデルが関数名と引数を完全に生成してから一括でレスポンスが返ってきました。引数が多い場合や、モデルの思考時間が長い場合、ユーザーには何も表示されない待ち時間が発生していました。

**ストリーミング Function Calling** では、`streamFunctionCallArguments: true` を設定することで、モデルが引数を生成しながらリアルタイムにストリーム配信されます。これにより：

- 引数が部分的に届いた時点でプログレスバーなどのUIを更新できる
- ユーザーへのフィードバックが早くなる
- 長い引数（大きなJSONなど）でも待ち時間を体感しにくくなる

この機能は **Gemini 3 Pro** および **Gemini 3 Flash** 以降のモデルで利用できます。

---

## マルチモーダル Function レスポンスとは

従来の Function Calling では、関数の実行結果（functionResponse）はテキスト文字列のみでした。Gemini 3 では、functionResponse に **画像・PDF・ファイルデータ** を含められるようになりました。

たとえば次のようなユースケースが可能です：

- ユーザーが「この画像を分析して」と依頼 → モデルが画像取得関数を呼び出す → 関数が画像データを返す → モデルが画像を視覚的に解析して回答
- PDF形式のレポートを取得する関数を呼び出し → PDFの内容をモデルが読み込んでサマリーを生成
- スクリーンショットを撮る関数 → 画像を返す → モデルがUI状態を理解して次のアクションを判断

これにより、**コンピュータ操作エージェント** や **視覚的なデバッグツール** など、より高度なアプリケーションが構築できるようになりました。

## ストリーミング Function Calling の実装

### Python での実装例

以下の例では、天気情報を取得する関数をストリーミングで呼び出します。

```python
import os
import json
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 関数定義
get_weather_tool = types.Tool(
    function_declarations=[
        types.FunctionDeclaration(
            name="get_weather",
            description="指定した都市の現在の天気情報を取得する",
            parameters=types.Schema(
                type=types.Type.OBJECT,
                properties={
                    "city": types.Schema(
                        type=types.Type.STRING,
                        description="都市名（例: Tokyo, Osaka）"
                    ),
                    "units": types.Schema(
                        type=types.Type.STRING,
                        enum=["celsius", "fahrenheit"],
                        description="温度の単位"
                    )
                },
                required=["city"]
            )
        )
    ]
)

# ストリーミング Function Calling の設定
config = types.GenerateContentConfig(
    tools=[get_weather_tool],
    function_calling_config=types.FunctionCallingConfig(
        mode=types.FunctionCallingMode.AUTO,
        stream_function_call_arguments=True  # ← ストリーミングを有効化
    )
)

# ストリーミングリクエスト
print("🌤 天気情報を取得中...")
for chunk in client.models.generate_content_stream(
    model="gemini-3-flash-preview",
    contents="東京の今日の天気を摂氏で教えて",
    config=config
):
    # 関数呼び出しの引数をリアルタイムに受信
    if chunk.candidates[0].content.parts:
        for part in chunk.candidates[0].content.parts:
            if hasattr(part, "function_call") and part.function_call:
                fc = part.function_call
                print(f"📡 関数呼び出し検出: {fc.name}")
                print(f"  引数（ストリーミング中）: {fc.args}")

# 実際の関数を実行してレスポンスを返す（ダミーデータ）
function_response = {
    "city": "Tokyo",
    "temperature": 18,
    "units": "celsius",
    "condition": "晴れ",
    "humidity": 55
}

# 期待する出力例:
# 🌤 天気情報を取得中...
# 📡 関数呼び出し検出: get_weather
#   引数（ストリーミング中）: {'city': 'Tokyo', 'units': 'celsius'}
```

### JavaScript (Node.js) での実装例

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

// 関数定義
const getWeatherTool = {
  functionDeclarations: [
    {
      name: "get_weather",
      description: "指定した都市の現在の天気情報を取得する",
      parameters: {
        type: "object",
        properties: {
          city: {
            type: "string",
            description: "都市名（例: Tokyo, Osaka）",
          },
          units: {
            type: "string",
            enum: ["celsius", "fahrenheit"],
            description: "温度の単位",
          },
        },
        required: ["city"],
      },
    },
  ],
};

async function streamingFunctionCall() {
  const stream = await ai.models.generateContentStream({
    model: "gemini-3-flash-preview",
    contents: [{ role: "user", parts: [{ text: "大阪の天気を摂氏で教えて" }] }],
    config: {
      tools: [getWeatherTool],
      functionCallingConfig: {
        mode: "AUTO",
        streamFunctionCallArguments: true, // ← ストリーミングを有効化
      },
    },
  });

  for await (const chunk of stream) {
    const parts = chunk.candidates?.[0]?.content?.parts ?? [];
    for (const part of parts) {
      if (part.functionCall) {
        console.log(`📡 関数呼び出し: ${part.functionCall.name}`);
        console.log(`  引数:`, part.functionCall.args);
      }
      if (part.text) {
        process.stdout.write(part.text);
      }
    }
  }
}

streamingFunctionCall();
```

## マルチモーダル Function レスポンスの実装

### 画像を返す関数の例

以下は、スクリーンショットを撮影して返す関数のシミュレーション例です。

```python
import base64
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# スクリーンショット取得ツールの定義
screenshot_tool = types.Tool(
    function_declarations=[
        types.FunctionDeclaration(
            name="take_screenshot",
            description="現在の画面のスクリーンショットを撮影して返す",
            parameters=types.Schema(
                type=types.Type.OBJECT,
                properties={
                    "region": types.Schema(
                        type=types.Type.STRING,
                        enum=["full", "active_window"],
                        description="スクリーンショットの範囲"
                    )
                },
                required=["region"]
            )
        )
    ]
)

# Step 1: モデルに関数呼び出しを要求
response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="画面全体のスクリーンショットを撮って、今どんなアプリが開いているか教えて",
    config=types.GenerateContentConfig(tools=[screenshot_tool])
)

# Step 2: 関数呼び出しを取得
function_call = response.candidates[0].content.parts[0].function_call
print(f"関数呼び出し: {function_call.name}, 引数: {function_call.args}")

# Step 3: 実際にスクリーンショットを撮影（ここではダミー画像を使用）
# 実際のアプリではここで pyautogui や Pillow を使ってスクリーンショットを取得
with open("screenshot.png", "rb") as f:
    image_bytes = f.read()
image_b64 = base64.b64encode(image_bytes).decode()

# Step 4: マルチモーダルデータを含む関数レスポンスを送信
multimodal_response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents=[
        # ユーザーの最初のメッセージ
        types.Content(
            role="user",
            parts=[types.Part(text="画面全体のスクリーンショットを撮って、今どんなアプリが開いているか教えて")]
        ),
        # モデルの関数呼び出し
        response.candidates[0].content,
        # 関数のレスポンス（画像データを含む）
        types.Content(
            role="user",
            parts=[
                types.Part(
                    function_response=types.FunctionResponse(
                        name="take_screenshot",
                        response={
                            "result": "screenshot_captured",
                            "region": "full"
                        }
                    )
                ),
                # ← ここがポイント: 画像データをfunctionResponseと一緒に送信
                types.Part(
                    inline_data=types.Blob(
                        mime_type="image/png",
                        data=image_b64
                    )
                )
            ]
        )
    ]
)

print("📸 モデルの分析結果:")
print(multimodal_response.text)
# 期待する出力:
# 📸 モデルの分析結果:
# スクリーンショットを確認しました。現在以下のアプリが開いています:
# - ターミナル（Python スクリプト実行中）
# - Visual Studio Code（main.py を編集中）
# - Chrome ブラウザ（Google AI Studio を表示）
```

### PDFレポートを処理する例

```python
import base64
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# PDFレポート取得ツール
report_tool = types.Tool(
    function_declarations=[
        types.FunctionDeclaration(
            name="get_quarterly_report",
            description="指定した四半期のビジネスレポートをPDF形式で取得する",
            parameters=types.Schema(
                type=types.Type.OBJECT,
                properties={
                    "quarter": types.Schema(
                        type=types.Type.STRING,
                        description="四半期（例: 2026Q1, 2025Q4）"
                    )
                },
                required=["quarter"]
            )
        )
    ]
)

# 関数呼び出しを取得
response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="2026年第1四半期のレポートを要約してください",
    config=types.GenerateContentConfig(tools=[report_tool])
)

function_call = response.candidates[0].content.parts[0].function_call

# 実際のPDFファイルを読み込む
with open("2026Q1_report.pdf", "rb") as f:
    pdf_bytes = f.read()
pdf_b64 = base64.b64encode(pdf_bytes).decode()

# PDFを含むマルチモーダルレスポンスを送信
final_response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents=[
        types.Content(
            role="user",
            parts=[types.Part(text="2026年第1四半期のレポートを要約してください")]
        ),
        response.candidates[0].content,
        types.Content(
            role="user",
            parts=[
                types.Part(
                    function_response=types.FunctionResponse(
                        name="get_quarterly_report",
                        response={"quarter": "2026Q1", "status": "found"}
                    )
                ),
                # PDFデータを含む
                types.Part(
                    inline_data=types.Blob(
                        mime_type="application/pdf",
                        data=pdf_b64
                    )
                )
            ]
        )
    ]
)

print(final_response.text)
```

## よくあるエラーと対処法

### エラー1: `streamFunctionCallArguments` が効かない

**原因**: Gemini 2.x 系のモデルではサポートされていない。

**対処**: `gemini-3-flash-preview` または `gemini-3-pro-preview` を使用してください。

```python
# ❌ Gemini 2.x では動作しない
model="gemini-2.5-flash"

# ✅ Gemini 3 系で動作する
model="gemini-3-flash-preview"
```

### エラー2: マルチモーダルレスポンスで `Invalid function response` エラー

**原因**: `functionResponse` と画像データを別々の `parts` として送っていない。

**対処**: `functionResponse` のパートと `inline_data` のパートを同じ `Content` に並べて含める必要があります。

```python
# ✅ 正しい書き方
parts=[
    types.Part(function_response=types.FunctionResponse(...)),
    types.Part(inline_data=types.Blob(mime_type="image/png", data=...))
]
```

### エラー3: ストリーミング中に引数が `None` になる

**原因**: 初期のチャンクでは関数引数がまだ生成されていない場合がある。

**対処**: `None` チェックを必ず入れてください。

```python
for chunk in stream:
    for part in chunk.candidates[0].content.parts:
        if hasattr(part, "function_call") and part.function_call:
            args = part.function_call.args
            if args:  # ← None チェックを忘れずに
                print(f"引数: {args}")
```

## 実際のユースケース

### ユースケース1: リアルタイム進捗表示

ストリーミング Function Calling を使うと、引数が届き次第プログレスバーを更新できます。これにより、複雑な検索クエリを構築しているときでも「処理中」の状態をユーザーに伝えられます。

### ユースケース2: ビジュアルエージェント

PDFや画像レポートを動的に取得してモデルに渡すことで、複数のデータソースを横断して分析するパイプラインが構築できます。詳細は [Gemini 高度ドキュメント処理](/articles/gemini-api/gemini-document-processing-advanced) も参照してください。

## FAQ

**Q: ストリーミング Function Calling はどのモデルで使えますか？**

A: 現時点では Gemini 3 Pro および Gemini 3 Flash（プレビュー版含む）で利用可能です。Gemini 2.x 系モデルではサポートされていません。Vertex AI 経由でも同じ設定で利用できます。

**Q: マルチモーダル Function レスポンスで使える MIME タイプは何ですか？**

A: `image/png`、`image/jpeg`、`image/webp`、`image/gif`、`application/pdf` が利用可能です。動画ファイルや音声ファイルは現時点では functionResponse 内での送信はサポートされていません。File API 経由でアップロードした `fileData` を使用することで、より大きなファイルを扱えます。

**Q: ストリーミング中に引数が途中で変わることはありますか？**

A: ストリーミングは引数を段階的に追記していくため、最終的な引数は最後のチャンクで確定します。途中のチャンクは部分的なデータを含むことがあります。確実な引数を使いたい場合は、ストリームが完了してから関数を実行してください。

## まとめ

Gemini 3 で強化された Function Calling の新機能をまとめます。

- **ストリーミング Function Calling**: `streamFunctionCallArguments: true` を設定するだけでリアルタイムに引数を受信でき、UX が向上する
- **マルチモーダル Function レスポンス**: 関数の結果に画像・PDFを含めることで、視覚的なエージェントや高度なドキュメント処理が可能になる
- いずれも **Gemini 3 Pro / Flash** 系モデルが必要

これらの機能を組み合わせることで、これまでにない高度なエージェントアプリケーションを構築できます。次のステップとして、[Gemini エージェント本番システム](/articles/gemini-advanced/gemini-agent-production-system) や [Function Calling 本番運用ガイド](/articles/gemini-api/gemini-function-calling-production) も確認してみてください。
