# Gemini API で自然言語からSQLを生成する — Text-to-SQL 実践ガイド

## はじめに

「先月の売上トップ10の商品を教えて」——こんな自然言語の質問をSQLクエリに変換し、データベースから直接回答を得られたら便利だと思いませんか？Gemini API の強力な言語理解能力を活用すれば、Text-to-SQL システムを比較的簡単に構築できます。

このガイドでは、Gemini API を使った Text-to-SQL システムの構築方法を、スキーマ設計からクエリ実行まで段階的に解説します。

## Text-to-SQL とは

Text-to-SQL とは、ユーザーが自然言語で入力した質問を、データベースに対して実行可能な SQL クエリに自動変換する技術です。従来はルールベースや専用モデルで実現していましたが、LLM の登場によりはるかに柔軟で正確な変換が可能になりました。

Gemini API が Text-to-SQL に適している理由は次の通りです。

- 長いコンテキストウィンドウでスキーマ情報を大量に渡せる
- 構造化出力（JSON Mode）で確実に SQL のみを返せる
- マルチターン会話で曖昧なクエリを対話的に絞り込める

## 前提条件

- Google AI Studio の API キー
- Python 3.10 以上
- `google-genai` SDK（`pip install google-genai`）
- SQLite（Python 標準ライブラリに含まれる）

## ステップ2：スキーマ情報の抽出

Gemini に正確な SQL を生成してもらうには、データベースのスキーマ情報を正確に伝えることが不可欠です。

```python
def extract_schema(db_path: str) -> str:
    conn = sqlite3.connect(db_path)
    cursor = conn.cursor()

    cursor.execute(
        "SELECT name FROM sqlite_master WHERE type='table'"
    )
    tables = cursor.fetchall()

    schema_parts = []
    for (table_name,) in tables:
        cursor.execute(f"PRAGMA table_info({table_name})")
        columns = cursor.fetchall()
        col_defs = []
        for col in columns:
            col_defs.append(
                f"  {col[1]} {col[2]}"
                f"{'  PRIMARY KEY' if col[5] else ''}"
                f"{'  NOT NULL' if col[3] else ''}"
            )
        schema_parts.append(
            f"TABLE {table_name} (\n"
            + ",\n".join(col_defs)
            + "\n)"
        )

    conn.close()
    return "\n\n".join(schema_parts)
```

## ステップ3：Gemini API による SQL 生成

ここからが本題です。Gemini API を使って自然言語から SQL を生成するコアロジックを実装します。

```python
from google import genai

client = genai.Client(api_key="YOUR_API_KEY")

SYSTEM_PROMPT = """あなたはSQLエキスパートです。
ユーザーの自然言語の質問を、与えられたスキーマに基づいて
SQLite互換のSQLクエリに変換してください。

ルール：
1. SELECT文のみ生成すること（INSERT/UPDATE/DELETEは禁止）
2. SQLクエリのみを返し、説明文は含めないこと
3. テーブル名・カラム名はスキーマに存在するもののみ使用すること
4. 曖昧な質問には最も合理的な解釈でクエリを作成すること
"""

def generate_sql(question: str, schema: str) -> str:
    prompt = f"""## データベーススキーマ
{schema}

## ユーザーの質問
{question}

## 生成するSQLクエリ
"""
    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=prompt,
        config={
            "system_instruction": SYSTEM_PROMPT,
            "temperature": 0.0,
        },
    )
    sql = response.text.strip()
    # コードブロックの装飾を除去
    if sql.startswith("```"):
        sql = sql.split("\n", 1)[1]
    if sql.endswith("```"):
        sql = sql.rsplit("```", 1)[0]
    return sql.strip()
```

ポイントは `temperature: 0.0` に設定することです。SQL 生成では創造性よりも正確性が重要なため、決定的な出力を得るのがベストプラクティスです。

## ステップ4：安全なクエリ実行

生成された SQL をそのまま実行するのは危険です。安全性を担保するバリデーション層を追加しましょう。

```python
import re

BLOCKED_KEYWORDS = [
    "DROP", "DELETE", "UPDATE", "INSERT",
    "ALTER", "CREATE", "TRUNCATE", "EXEC",
]

def validate_sql(sql: str) -> bool:
    upper_sql = sql.upper().strip()
    if not upper_sql.startswith("SELECT"):
        return False
    for keyword in BLOCKED_KEYWORDS:
        if re.search(rf"\b{keyword}\b", upper_sql):
            return False
    return True

def execute_query(db_path: str, sql: str) -> list:
    if not validate_sql(sql):
        raise ValueError(
            "安全でないクエリが検出されました"
        )
    conn = sqlite3.connect(db_path)
    conn.execute("PRAGMA query_only = ON")
    cursor = conn.cursor()
    cursor.execute(sql)
    columns = [desc[0] for desc in cursor.description]
    results = cursor.fetchall()
    conn.close()
    return {"columns": columns, "rows": results}
```

`PRAGMA query_only = ON` により、データベースレベルでも書き込みを禁止しています。

## ステップ5：エンドツーエンドのパイプライン

すべてのコンポーネントを組み合わせて、完全な Text-to-SQL パイプラインを構築します。

```python
def ask_database(question: str, db_path: str = "ecommerce.db"):
    schema = extract_schema(db_path)
    sql = generate_sql(question, schema)
    print(f"生成されたSQL: {sql}")

    result = execute_query(db_path, sql)
    print(f"カラム: {result['columns']}")
    for row in result["rows"]:
        print(row)
    return result

# 使用例
ask_database("在庫が100個以上ある商品を価格が高い順に教えて")
# → SELECT name, price, stock FROM products
#   WHERE stock >= 100 ORDER BY price DESC

ask_database("カテゴリごとの商品数を教えて")
# → SELECT category, COUNT(*) as count
#   FROM products GROUP BY category
```

## 精度を向上させるテクニック

### Few-Shot プロンプティング

システムプロンプトに質問と回答のペアを数例含めることで、生成精度が大幅に向上します。

```python
FEW_SHOT_EXAMPLES = """
質問: 最も高い商品は？
SQL: SELECT name, price FROM products ORDER BY price DESC LIMIT 1

質問: 2026年3月の注文数は？
SQL: SELECT COUNT(*) FROM orders WHERE order_date BETWEEN '2026-03-01' AND '2026-03-31'

質問: 注文金額の合計が最も多い顧客は？
SQL: SELECT customer_name, SUM(total_amount) as total FROM orders GROUP BY customer_name ORDER BY total DESC LIMIT 1
"""
```

### 構造化出力で確実に SQL だけを受け取る

`response_mime_type` を指定して JSON 形式で SQL を返すようにすると、パース処理が安定します。

```python
from google.genai import types

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=prompt,
    config=types.GenerateContentConfig(
        system_instruction=SYSTEM_PROMPT,
        temperature=0.0,
        response_mime_type="application/json",
        response_schema={
            "type": "object",
            "properties": {
                "sql": {"type": "string"},
                "explanation": {"type": "string"},
            },
            "required": ["sql"],
        },
    ),
)
```

## 本番環境での考慮事項

### レートリミットとキャッシュ

同じ質問に対して毎回 API を呼び出すのはコストの無駄です。クエリキャッシュの導入を検討してください。

```python
from functools import lru_cache

@lru_cache(maxsize=256)
def cached_generate_sql(question: str, schema_hash: str):
    schema = get_schema_by_hash(schema_hash)
    return generate_sql(question, schema)
```

### エラーハンドリング

SQL の実行に失敗した場合、エラーメッセージを Gemini にフィードバックして再生成させるリトライ機構が有効です。

```python
def ask_with_retry(question, db_path, max_retries=2):
    schema = extract_schema(db_path)
    sql = generate_sql(question, schema)

    for attempt in range(max_retries + 1):
        try:
            return execute_query(db_path, sql)
        except Exception as e:
            if attempt == max_retries:
                raise
            fix_prompt = (
                f"以下のSQLがエラーになりました。\n"
                f"SQL: {sql}\n"
                f"エラー: {e}\n"
                f"修正したSQLを生成してください。"
            )
            sql = generate_sql(fix_prompt, schema)
```

## まとめ

Gemini API を使った Text-to-SQL システムの構築方法を解説しました。重要なポイントをまとめると次の通りです。

- スキーマ情報を正確に渡すことが精度向上の鍵
- `temperature: 0.0` で決定的な SQL 生成を行う
- SELECT 文のみ許可するバリデーション層を必ず実装する
- Few-Shot やスキーマコメントで精度をさらに高める
- 本番環境ではキャッシュとリトライ機構を導入する

この技術をベースに、社内データの自然言語検索ツールやBIダッシュボードのチャットインターフェースなど、様々な応用が可能です。
