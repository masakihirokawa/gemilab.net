## はじめに

Gemini API を使ってPDFの要約や音声の文字起こし、動画の分析をしたいと思ったことはありませんか？テキストプロンプトだけでなく、さまざまなメディアファイルを直接 Gemini に渡して処理できるのが **Files API** です。

Files API を使えば、最大 **2GB** のファイルをアップロードし、48時間のキャッシュ期間内に何度でも参照できます。これにより、毎回のリクエストで巨大なファイルを送信する必要がなくなり、コストと処理時間を大幅に削減できます。

この記事では、Files API の仕組みから Python での実装方法、対応ファイル形式、そして実践的なユースケースまでを網羅的に解説します。

## Files API とは何か

Files API は、Gemini API が提供するファイル管理用のエンドポイントです。主に以下の機能を提供します。

- **ファイルのアップロード**: ローカルのファイルを Google のサーバーにアップロード
- **ファイルの一覧取得**: アップロード済みファイルの確認
- **ファイルの削除**: 不要になったファイルの手動削除
- **ファイルの参照**: アップロードしたファイルを `generateContent` リクエストで使用

アップロードされたファイルは **48時間** 保持され、その間は URI を指定するだけで何度でもプロンプトに含めることができます。API キーに紐づくプロジェクト単位で管理され、他のユーザーからはアクセスできません。

### 対応ファイル形式

| カテゴリ | 形式 | MIME タイプ例 |
|---------|------|-------------|
| 画像 | PNG, JPEG, WebP, HEIC, HEIF | `image/png`, `image/jpeg` |
| 音声 | MP3, WAV, FLAC, AAC, OGG | `audio/mp3`, `audio/wav` |
| 動画 | MP4, MPEG, MOV, AVI, WebM | `video/mp4`, `video/quicktime` |
| ドキュメント | PDF, TXT, HTML, CSS, JS | `application/pdf`, `text/plain` |
| データ | CSV, XML, JSON | `text/csv`, `application/json` |

## ファイルのアップロードと処理

### 基本的なアップロード

以下は PDF ファイルをアップロードして要約を生成する例です。

```python
from google import genai
import os

# クライアントの初期化
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# PDFファイルをアップロード
uploaded_file = client.files.upload(
    file="report.pdf",
    config={"display_name": "quarterly-report"}
)

print(f"アップロード完了: {uploaded_file.name}")
print(f"URI: {uploaded_file.uri}")
print(f"状態: {uploaded_file.state}")
# 期待する出力:
# アップロード完了: files/xxxxxxxxxxxx
# URI: https://generativelanguage.googleapis.com/v1beta/files/xxxxxxxxxxxx
# 状態: ACTIVE

# アップロードしたPDFの内容を要約
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        "このPDFの内容を日本語で3つの要点にまとめてください。",
        uploaded_file
    ]
)

print(response.text)
```

### 動画ファイルの処理

動画は処理に時間がかかるため、アップロード後にステータスを確認する必要があります。

```python
import time
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 動画をアップロード
video_file = client.files.upload(file="presentation.mp4")

# 処理完了を待機（動画は PROCESSING 状態から始まる）
while video_file.state == "PROCESSING":
    print("動画を処理中...")
    time.sleep(5)
    video_file = client.files.get(name=video_file.name)

if video_file.state == "ACTIVE":
    print("処理完了！分析を開始します。")

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=[
            "この動画の内容を分析し、主要なトピックとタイムスタンプを列挙してください。",
            video_file
        ]
    )
    print(response.text)
else:
    print(f"エラー: ファイルの状態が {video_file.state} です")
# 期待する出力:
# 動画を処理中...
# 動画を処理中...
# 処理完了！分析を開始します。
# 00:00 - 導入部分: プロジェクトの概要説明
# 02:30 - 技術アーキテクチャの解説
# ...
```

## アップロード済みファイルの管理

### ファイル一覧の取得

```python
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# アップロード済みファイルの一覧を取得
for f in client.files.list():
    print(f"{f.name} | {f.display_name} | {f.mime_type} | {f.state}")
# 期待する出力:
# files/abc123 | quarterly-report | application/pdf | ACTIVE
# files/def456 | presentation.mp4 | video/mp4 | ACTIVE
```

### ファイルの削除

48時間の自動削除を待たずに、手動で削除することもできます。

```python
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# ファイルを削除
client.files.delete(name="files/abc123")
print("ファイルを削除しました")
```

## 実践的なユースケース

### ユースケース1: 複数PDFの一括分析

複数のPDFをアップロードして横断的に分析する例です。

```python
from google import genai
import glob

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# 複数のPDFをアップロード
pdf_files = []
for pdf_path in glob.glob("reports/*.pdf"):
    uploaded = client.files.upload(file=pdf_path)
    pdf_files.append(uploaded)
    print(f"アップロード: {pdf_path}")

# 全PDFを横断して分析
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        "以下の複数のレポートを横断的に分析し、"
        "共通するトレンドと各レポート固有の洞察をまとめてください。",
        *pdf_files
    ]
)

print(response.text)
```

### ユースケース2: 動画からの議事録自動生成

```python
from google import genai
import time

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

video = client.files.upload(file="team_meeting.mp4")

while video.state == "PROCESSING":
    time.sleep(5)
    video = client.files.get(name=video.name)

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        "この会議動画から以下の形式で議事録を作成してください:\n"
        "1. 日時と参加者\n"
        "2. アジェンダ\n"
        "3. 各議題の議論内容と決定事項\n"
        "4. 次回までのアクションアイテム（担当者付き）",
        video
    ]
)

print(response.text)
```

## よくあるエラーと対処法

| エラー | 原因 | 対処法 |
|--------|------|--------|
| `FAILED_PRECONDITION` | ファイルがまだ処理中 | `state` が `ACTIVE` になるまで待機 |
| `INVALID_ARGUMENT` | 非対応のファイル形式 | 対応形式一覧を確認 |
| `RESOURCE_EXHAUSTED` | アップロード上限超過 | 不要ファイルを削除して再試行 |
| `NOT_FOUND` | ファイルが期限切れ（48時間経過） | 再アップロードが必要 |

## FAQ

### Q1: Files API のファイルサイズ上限はどれくらいですか？

1ファイルあたり最大 **2GB** までアップロード可能です。ただし、`generateContent` に含められるファイルの合計サイズにもモデルごとの上限があります。大容量ファイルを扱う場合は、`gemini-2.5-pro`（100万トークン対応）の利用を検討してください。

### Q2: アップロードしたファイルは安全ですか？

アップロードされたファイルは API キーに紐づくプロジェクト内でのみアクセス可能です。他のユーザーやプロジェクトからはアクセスできません。また、48時間後に自動削除されるため、長期保存されることはありません。機密データを扱う場合は、処理後に `files.delete()` で即座に削除することを推奨します。

### Q3: Files API と inline data（Base64）の違いは何ですか？

小さなファイル（数MB以下）であれば、Base64エンコードして `inline_data` として直接リクエストに含める方法もあります。Files API は主に以下のケースで有利です。
- ファイルサイズが大きい（20MB以上）場合
- 同じファイルを複数回のリクエストで再利用する場合
- 動画など処理に時間がかかるファイルを事前にアップロードしておく場合

### Q4: アップロードしたファイルをコンテキストキャッシュと併用できますか？

はい、可能です。Files API でアップロードしたファイルを[コンテキストキャッシュ](/articles/gemini-api/context-caching-guide)と組み合わせることで、大容量ファイルの処理コストをさらに削減できます。長時間にわたって同じファイルを参照する場合に特に効果的です。
