## はじめに：Gemini Interactions APIとは

Googleは2026年3月、**Gemini Interactions API**をベータ版として公開しました。これはGeminiモデルおよびエージェントと対話するための**統合インターフェース**であり、従来の`generate_content`ベースのAPIを大きく拡張するものです。

従来のGemini APIはステートレスなリクエスト/レスポンス型でしたが、Interactions APIは**マルチターンの会話状態を自動的に管理**し、エージェント的なタスク実行（Deep Researchなど）を開発者のアプリに直接組み込めるようにします。

### Interactions APIの主なメリット

- **会話状態の自動管理** — セッションIDを使ってマルチターン対話を簡単に維持
- **エージェント機能の統合** — Deep Researchエージェントなど高度なAI機能を1つのAPIで呼び出し
- **Thought Signaturesのサポート** — Gemini 3の推論チェーンを会話をまたいで保持
- **統一されたツール管理** — 検索・コード実行・File SearchなどをシームレスにAPI経由で制御

## 基本的な使い方：最初のInteractionを送信する

Interactions APIの中心となるのは `client.interactions` オブジェクトです。`start()` メソッドで新しいインタラクションセッションを開始します。

```python
import os
from google import genai

# クライアント初期化
client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# 新しいInteractionセッションを開始
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    system_instruction="あなたはPythonの専門家です。コードの質問に丁寧に答えてください。"
)

# 最初のメッセージを送信
response = interaction.send_message(
    "Pythonでフィボナッチ数列を生成する効率的な方法を教えてください。"
)

print(response.text)
# 期待する出力例:
# フィボナッチ数列を効率的に生成する方法はいくつかあります。
# 最もメモリ効率が高いのはジェネレータを使った方法です...
```

## マルチターン会話：会話状態の自動管理

Interactions APIの最大の強みは、会話履歴を手動で管理しなくてよい点です。`interaction` オブジェクトが内部で状態を保持します。

```python
import os
from google import genai

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# セッションを開始
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    system_instruction="あなたはコードレビューの専門家です。"
)

# 1ターン目：コードを送信
response1 = interaction.send_message(
    """
    以下のコードをレビューしてください。

    ```python
    def get_user(id):
        conn = sqlite3.connect('users.db')
        cursor = conn.cursor()
        cursor.execute(f"SELECT * FROM users WHERE id = {id}")
        return cursor.fetchone()
    ```
    """
)
print("【レビュー1回目】")
print(response1.text)

# 2ターン目：前のコンテキストを踏まえた追加質問（履歴は自動管理）
response2 = interaction.send_message(
    "セキュリティの観点から最も重大な問題点を1つ挙げて、修正版のコードも書いてください。"
)
print("\n【レビュー2回目】")
print(response2.text)

# セッション情報を確認
print(f"\nセッションID: {interaction.session_id}")
print(f"ターン数: {interaction.turn_count}")
# 期待する出力例:
# セッションID: int_a1b2c3d4e5f6
# ターン数: 2
```

## Thought Signatures：推論チェーンを会話をまたいで保持する

Gemini 3以降、**Thought Signatures**という暗号化された推論表現が導入されました。これをInteractions APIと組み合わせることで、複雑なマルチステップの推論チェーンを効率的に維持できます。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Thought Signaturesを有効化してセッション開始
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    config=types.InteractionConfig(
        thinking_config=types.ThinkingConfig(
            include_thoughts=True,        # 思考プロセスを含める
            thinking_level="high",        # 深い推論レベル（low/medium/high）
        )
    )
)

# 複雑な問題を投げる
response = interaction.send_message(
    "アルゴリズムの時間計算量をO(n²)からO(n log n)に改善するための戦略を、マージソートを例に説明してください。"
)

# Thought Signaturesが含まれているか確認
if response.thoughts:
    print("=== 推論プロセス（Thought Signature）===")
    print(response.thoughts[:200] + "...\n")  # 最初の200文字を表示

print("=== 最終回答 ===")
print(response.text)
# 期待する出力例:
# === 推論プロセス（Thought Signature）===
# [encrypted thought signature data]...
#
# === 最終回答 ===
# マージソートはO(n log n)の時間計算量を実現するアルゴリズムです...
```

## Deep Researchエージェントの呼び出し

Interactions APIが真価を発揮するのが、**Deep Researchエージェント**との連携です。自律的なマルチステップリサーチをアプリに組み込めます。

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Deep Researchエージェントを有効化してセッション開始
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    tools=[
        types.Tool(google_search=types.GoogleSearch()),  # Webリサーチを許可
        types.Tool(code_execution=types.CodeExecution()),  # コード実行を許可
    ],
    config=types.InteractionConfig(
        agent_config=types.AgentConfig(
            deep_research=True  # Deep Researchモードを有効化
        )
    )
)

print("リサーチを開始します...")

# リサーチタスクを指示
response = interaction.send_message(
    """
    「2026年のAIエージェントフレームワーク」について調査してください。
    以下の観点でまとめてください：
    1. 主要なフレームワーク（LangChain、LlamaIndex、Google ADK等）の最新状況
    2. 各フレームワークの特徴と適用シーン
    3. 開発者コミュニティでのトレンド
    """
)

# リサーチ中のステップを確認
if response.research_steps:
    print(f"\n実行したリサーチステップ: {len(response.research_steps)}件")
    for i, step in enumerate(response.research_steps, 1):
        print(f"  ステップ{i}: {step.description}")

print("\n=== リサーチ結果 ===")
print(response.text)
# 期待する出力例:
# 実行したリサーチステップ: 5件
#   ステップ1: Google Searchで最新フレームワーク情報を収集
#   ステップ2: LangChain公式ドキュメントを参照
#   ...
# === リサーチ結果 ===
# 2026年のAIエージェントフレームワーク調査結果...
```

## セッションの保存と復元

Interactions APIではセッションIDを使って会話を後から再開できます。

```python
import os
import json
from google import genai

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# --- セッションの開始と保存 ---
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview"
)

response = interaction.send_message("機械学習の基礎を教えてください。")
print(response.text[:100] + "...")

# セッションIDを保存（データベースやファイルに保存する想定）
saved_session_id = interaction.session_id
print(f"\nセッションID保存: {saved_session_id}")

# --- 後でセッションを再開 ---
resumed_interaction = client.interactions.resume(
    session_id=saved_session_id,
    model="gemini-3.1-pro-preview"
)

# 前の会話の続きから再開
response2 = resumed_interaction.send_message(
    "その中でも、ニューラルネットワークについてもう少し詳しく説明してください。"
)
print("\n【再開後の返答】")
print(response2.text)
# 期待する出力例:
# 【再開後の返答】
# もちろんです。先ほどの機械学習の概要を踏まえて、
# ニューラルネットワークについて詳しく説明します...
```

## ストリーミング対応：リアルタイムでレスポンスを受信する

長文の回答やリサーチ結果をリアルタイムで表示したい場合は、ストリーミングモードを使います。

```python
import os
from google import genai

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

interaction = client.interactions.start(
    model="gemini-3.1-pro-preview"
)

print("=== ストリーミング回答 ===")
# send_message_stream でリアルタイム受信
for chunk in interaction.send_message_stream(
    "Pythonの非同期プログラミング（asyncio）の仕組みを詳しく説明してください。"
):
    if chunk.text:
        print(chunk.text, end="", flush=True)

print("\n\n=== 完了 ===")
# 期待する出力例:
# === ストリーミング回答 ===
# Pythonの非同期プログラミング（asyncio）は...
# （リアルタイムでテキストが表示される）
# === 完了 ===
```

## 従来のgenerate_contentとの比較

| 比較項目 | `generate_content` | `Interactions API` |
|---------|-------------------|-------------------|
| **会話状態** | 手動管理が必要 | 自動管理 |
| **マルチターン** | `contents`配列に手動追加 | `send_message()`のみ |
| **エージェント機能** | 限定的 | Deep Research等に対応 |
| **Thought Signatures** | 非対応 | 対応（Gemini 3以降） |
| **セッション再開** | 不可 | セッションIDで可能 |
| **ユースケース** | シンプルな1回のリクエスト | 複雑な対話・エージェント |

**結論**: シンプルな単発リクエストには`generate_content`、複雑なマルチターン対話やエージェントワークフローには**Interactions API**を選びましょう。

## FAQ

**Q1: Interactions APIはすべてのGeminiモデルで使えますか？**

A: 現時点（2026年3月）では、**Gemini 3.1 Pro Preview** および **Gemini 3 Flash Preview** でのサポートが確認されています。Gemini 2.5系モデルや旧モデルでの対応は順次拡大予定です。最新の対応モデル一覧は[公式ドキュメント](https://ai.google.dev/gemini-api/docs/models)でご確認ください。

**Q2: Interactions APIのセッションはどのくらい保持されますか？**

A: ベータ版現在、セッションはサーバー側で最大**24時間**保持されます。長期的な会話の保持が必要な場合は、会話履歴をローカルに保存し、新セッションで`history`パラメータとして渡す方法も利用できます。

**Q3: Deep Researchエージェントを使うと料金はどうなりますか？**

A: Deep Researchモードを有効にすると、内部でGoogle Searchが複数回呼び出されます。2026年3月のアップデートで、検索料金は**1,000クエリあたり$14**の従量課金制に変更されました。コスト管理のため、`max_search_queries`パラメータで上限を設定することを推奨します。
