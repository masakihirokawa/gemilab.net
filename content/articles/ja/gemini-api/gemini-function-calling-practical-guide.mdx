## はじめに — Function Calling で Gemini の可能性を拡張する

Gemini は優れたテキスト生成能力を持っていますが、リアルタイムの天気情報や最新のデータベース内容など、モデルが持っていない情報には直接アクセスできません。この制限を突破するのが Function Calling（関数呼び出し）機能です。

Function Calling を使うと、Gemini が「この場面では外部ツールを呼び出すべきだ」と判断し、必要なパラメータを自動生成します。開発者はそのパラメータを使って実際のAPI呼び出しを行い、結果を Gemini に返すことで、リアルタイムの情報を含んだ応答を生成できます。

本記事では、基本的な使い方から複数ツールの同時連携まで、Python コードを使って段階的に解説します。

## Function Calling の仕組み

Function Calling は以下の3ステップで動作します。

1. **関数の定義**: 利用可能な関数（ツール）の名前、説明、パラメータを Gemini に伝える
2. **関数の呼び出し判断**: Gemini がユーザーの質問を分析し、適切な関数とパラメータを選択する
3. **結果の統合**: 関数の実行結果を Gemini に渡し、自然な文章で応答を生成する

```python
# Function Calling の全体フロー（概念図）
# ユーザー: 「東京の天気を教えて」
#   ↓
# Gemini: 関数 get_weather(city="Tokyo") を呼び出すべきと判断
#   ↓
# 開発者のコード: 天気APIを実際に呼び出す
#   ↓
# Gemini: 「東京の現在の天気は晴れ、気温は22℃です」と応答
```

## 基本実装: 天気情報の取得

最もシンプルな例として、天気情報を取得する Function Calling を実装します。

```python
import google.generativeai as genai

# APIキーの設定
genai.configure(api_key="YOUR_API_KEY")

# Step 1: 関数の定義
get_weather = genai.protos.FunctionDeclaration(
    name="get_weather",
    description="指定された都市の現在の天気情報を取得する",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "city": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="天気を取得したい都市名（例: Tokyo, New York）",
            ),
            "unit": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                enum=["celsius", "fahrenheit"],
                description="温度の単位（デフォルト: celsius）",
            ),
        },
        required=["city"],
    ),
)

# ツール定義をモデルに渡す
tool = genai.protos.Tool(function_declarations=[get_weather])
model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    tools=[tool],
)

# Step 2: ユーザーの質問を送信
chat = model.start_chat()
response = chat.send_message("東京の天気を教えてください")

# Gemini が関数呼び出しを要求した場合
function_call = response.candidates[0].content.parts[0].function_call
print(f"呼び出し関数: {function_call.name}")
print(f"パラメータ: {dict(function_call.args)}")

# 期待される出力:
# 呼び出し関数: get_weather
# パラメータ: {'city': 'Tokyo', 'unit': 'celsius'}
```

### 実際のAPI呼び出しと結果の返却

```python
import requests

def call_weather_api(city: str, unit: str = "celsius") -> dict:
    """実際の天気APIを呼び出す関数"""
    # OpenWeatherMap API の例
    api_key = "YOUR_WEATHER_API_KEY"
    url = f"https://api.openweathermap.org/data/2.5/weather"
    params = {
        "q": city,
        "appid": api_key,
        "units": "metric" if unit == "celsius" else "imperial",
    }
    resp = requests.get(url, params=params)
    data = resp.json()
    return {
        "city": city,
        "temperature": data["main"]["temp"],
        "condition": data["weather"][0]["description"],
        "humidity": data["main"]["humidity"],
    }

# Step 3: 関数を実行して結果を Gemini に返す
weather_data = call_weather_api(
    city=function_call.args["city"],
    unit=function_call.args.get("unit", "celsius"),
)

# 結果をGeminiに送信
response = chat.send_message(
    genai.protos.Content(
        parts=[
            genai.protos.Part(
                function_response=genai.protos.FunctionResponse(
                    name="get_weather",
                    response={"result": weather_data},
                )
            )
        ]
    )
)

print(response.text)
# 期待される出力:
# 東京の現在の天気は晴れで、気温は22℃、湿度は45%です。
# 過ごしやすい気候ですね。
```

## 複数ツールの同時連携

実用的なアプリケーションでは、複数のツールを同時に定義し、Gemini に状況に応じて使い分けさせます。

```python
import google.generativeai as genai

# 天気取得ツール
get_weather = genai.protos.FunctionDeclaration(
    name="get_weather",
    description="都市の現在の天気を取得する",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "city": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="都市名",
            ),
        },
        required=["city"],
    ),
)

# データベース検索ツール
search_database = genai.protos.FunctionDeclaration(
    name="search_database",
    description="商品データベースからキーワードで検索する",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "query": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="検索キーワード",
            ),
            "max_results": genai.protos.Schema(
                type=genai.protos.Type.INTEGER,
                description="最大取得件数（デフォルト: 5）",
            ),
        },
        required=["query"],
    ),
)

# メール送信ツール
send_email = genai.protos.FunctionDeclaration(
    name="send_email",
    description="指定されたアドレスにメールを送信する",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "to": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="送信先メールアドレス",
            ),
            "subject": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="メールの件名",
            ),
            "body": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="メールの本文",
            ),
        },
        required=["to", "subject", "body"],
    ),
)

# 全ツールをモデルに登録
tools = genai.protos.Tool(
    function_declarations=[get_weather, search_database, send_email]
)

model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    tools=[tools],
)

# Gemini が質問の内容に応じて適切なツールを自動選択する
chat = model.start_chat()

# 例1: 天気に関する質問 → get_weather が選択される
response = chat.send_message("大阪の天気は？")

# 例2: 商品に関する質問 → search_database が選択される
response = chat.send_message("ワイヤレスイヤホンを探して")

# 例3: メール送信の依頼 → send_email が選択される
response = chat.send_message("田中さんに会議の日程変更をメールで連絡して")
```

## エラーハンドリングとベストプラクティス

### 関数呼び出しのバリデーション

```python
def handle_function_call(response) -> dict | None:
    """Function Call のレスポンスを安全に処理する"""
    try:
        candidate = response.candidates[0]
        part = candidate.content.parts[0]

        # Function Call が含まれているか確認
        if not hasattr(part, "function_call") or not part.function_call.name:
            # 通常のテキスト応答の場合
            return None

        fc = part.function_call
        return {
            "name": fc.name,
            "args": dict(fc.args),
        }
    except (IndexError, AttributeError) as e:
        print(f"Function Call の解析に失敗: {e}")
        return None

# 使用例
result = handle_function_call(response)
if result:
    print(f"関数 {result['name']} を {result['args']} で実行")
else:
    print(f"テキスト応答: {response.text}")
```

### ベストプラクティス

Function Calling を効果的に使うためのポイントをまとめます。

**関数の説明は具体的に書く**: 「データを取得する」ではなく「商品データベースからキーワードで商品名・価格・在庫数を検索する」のように、何を・どこから・どんな形式で取得するかを明示してください。説明が曖昧だと、Gemini が適切なタイミングで関数を選択できなくなります。

**パラメータには型とデフォルト値を明記する**: `required` に含めるパラメータは最小限にし、オプションパラメータにはデフォルト値を説明に含めると、Gemini がより柔軟に関数を呼び出せます。

**エラー時は意味のあるメッセージを返す**: 関数の実行が失敗した場合、空の結果ではなくエラーの理由を Gemini に返してください。Gemini はエラー内容を理解し、ユーザーに適切な説明を行えます。

## よくある質問（FAQ）

### Q1: Function Calling は全ての Gemini モデルで使えますか？

Function Calling は Gemini 2.5 Pro および Gemini 2.5 Flash で利用可能です。Gemini 1.5 シリーズでもサポートされていますが、2.5 シリーズの方がより正確な関数選択とパラメータ生成を行います。Google AI Studio のプレイグラウンドでも Function Calling をテストできます。

### Q2: 1回のリクエストで複数の関数を同時に呼び出すことは可能ですか？

はい、Gemini は Parallel Function Calling をサポートしており、1つのレスポンスで複数の関数呼び出しを返すことがあります。たとえば「東京と大阪の天気を比較して」という質問に対して、`get_weather(city="Tokyo")` と `get_weather(city="Osaka")` の2回の呼び出しが同時に返されます。

### Q3: Function Calling の利用に追加料金はかかりますか？

Function Calling 自体に追加料金はかかりません。通常の API 利用と同じく、入出力トークン数に基づいて課金されます。ただし、関数定義のスキーマもトークンとしてカウントされるため、多数の関数を定義すると入力トークンが増加する点に注意してください。

### Q4: セキュリティ上の注意点はありますか？

重要な点として、Function Calling は「Gemini が関数のパラメータを決定する」だけであり、関数の実行自体は開発者のコード内で行われます。そのため、受け取ったパラメータを必ずバリデーションし、SQLインジェクションやパストラバーサルなどの攻撃を防ぐ実装が必要です。

Gemini API の Function Calling は、AIモデルと外部ツールを橋渡しするための強力な機能です。天気情報の取得からデータベース検索、メール送信まで、適切な関数定義を与えるだけで Gemini が状況に応じたツール選択を自動的に行います。
