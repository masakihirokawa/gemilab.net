Gemini Live API は双方向音声ストリーミングと低レイテンシ推論を組み合わせた、会話 AI 構築のための最先端インターフェースです。2026 年に GA を迎えた Google の **Agent Development Kit（ADK）**と統合することで、カレンダー操作・天気取得・データベース検索といったツールを自然な音声コマンドで呼び出せる自律 AI アシスタントを構築できます。

本記事では WebSocket ベースの音声ストリーミング実装から Cloud Run への本番デプロイまでを、動作する Python コードとともに段階的に解説します。

## Gemini Live API と ADK の役割分担

| コンポーネント | 役割 |
|--------------|------|
| **Gemini Live API** | リアルタイム双方向音声ストリーミング、VAD（音声活動検出）、テキスト↔音声変換 |
| **Agent Development Kit (ADK)** | エージェントライフサイクル管理、ツール定義・実行、マルチターン会話状態 |
| **Cloud Run** | スケーラブルな WebSocket サーバーのデプロイ基盤 |

### アーキテクチャ概要

```
[クライアント（ブラウザ/モバイル）]
         │ WebSocket (音声バイナリ + JSON コマンド)
         ↓
[Cloud Run: FastAPI + WebSocket サーバー]
         │ google-genai Live API セッション
         ↓
[Gemini Live API（Gemini 3.1 Pro）]
         │ ADK ツール呼び出し
         ├── [カレンダー API]
         ├── [天気 API]
         └── [社内 DB 検索]
```

## 基本的な Live API セッション実装

```python
# voice_assistant/session.py
import asyncio
import json
import logging
from typing import AsyncGenerator

from google import genai
from google.genai import types

logger = logging.getLogger(__name__)

# Live API で使用できるモデル（2026年3月時点）
LIVE_MODEL = "gemini-3.0-flash-preview"

SYSTEM_INSTRUCTION = """
あなたは高性能な音声 AI アシスタントです。
ユーザーの質問に簡潔かつ正確に答えてください。
ツールが利用可能な場合は積極的に使用し、リアルタイムな情報を提供してください。
"""

class LiveVoiceSession:
    """Gemini Live API との双方向音声セッションを管理するクラス"""

    def __init__(self, api_key: str):
        self.client = genai.Client(api_key=api_key, http_options={"api_version": "v1alpha"})
        self.session = None
        self._audio_queue: asyncio.Queue[bytes] = asyncio.Queue()

    async def connect(self, tools: list | None = None) -> None:
        """Live API セッションを開始する"""
        config = types.LiveConnectConfig(
            response_modalities=["AUDIO", "TEXT"],  # 音声＋テキスト両方で出力
            speech_config=types.SpeechConfig(
                voice_config=types.VoiceConfig(
                    prebuilt_voice_config=types.PrebuiltVoiceConfig(voice_name="Kore")
                )
            ),
            system_instruction=types.Content(
                parts=[types.Part(text=SYSTEM_INSTRUCTION)]
            ),
            tools=tools or [],
        )

        self.session = await self.client.aio.live.connect(
            model=LIVE_MODEL,
            config=config,
        )
        logger.info("🎤 Live API セッション開始")

    async def stream_audio_input(self, audio_chunk: bytes) -> None:
        """クライアントから受け取った音声チャンクを Live API に送信する"""
        if not self.session:
            raise RuntimeError("セッションが未接続です。connect() を先に呼び出してください。")

        await self.session.send(
            input=types.LiveClientRealtimeInput(
                media_chunks=[
                    types.Blob(data=audio_chunk, mime_type="audio/pcm;rate=16000")
                ]
            )
        )

    async def receive_responses(self) -> AsyncGenerator[dict, None]:
        """
        Live API からのレスポンスを非同期ジェネレーターとして受信する

        Yields:
            {"type": "audio", "data": bytes}    — 音声データ
            {"type": "text", "data": str}       — テキスト
            {"type": "tool_call", "data": dict} — ツール呼び出しリクエスト
            {"type": "turn_complete"}            — ターン完了通知
        """
        async for response in self.session.receive():
            if response.server_content:
                content = response.server_content

                if content.model_turn:
                    for part in content.model_turn.parts:
                        if part.inline_data:
                            # 音声データ（PCM 形式）
                            yield {"type": "audio", "data": part.inline_data.data}
                        elif part.text:
                            # テキスト出力（ログ・字幕用）
                            yield {"type": "text", "data": part.text}

                if content.turn_complete:
                    yield {"type": "turn_complete"}

            elif response.tool_call:
                # ADK ツール呼び出しリクエスト
                yield {"type": "tool_call", "data": {
                    "id": response.tool_call.function_calls[0].id,
                    "name": response.tool_call.function_calls[0].name,
                    "args": dict(response.tool_call.function_calls[0].args),
                }}

    async def send_tool_result(self, call_id: str, result: str) -> None:
        """ツール実行結果を Live API に返送する"""
        await self.session.send(
            input=types.LiveClientToolResponse(
                function_responses=[
                    types.FunctionResponse(id=call_id, response={"result": result})
                ]
            )
        )

    async def close(self) -> None:
        if self.session:
            await self.session.close()
            logger.info("🔇 Live API セッション終了")
```

## ADK ツールの定義と統合

ADK では Python デコレータでツールを定義し、Live API に自動的に公開できます。

```python
# voice_assistant/tools.py
import httpx
from google.adk.tools import FunctionTool
from datetime import datetime, timedelta
import os

async def get_current_weather(location: str, unit: str = "celsius") -> str:
    """
    指定した場所の現在の天気を取得する

    Args:
        location: 場所（例: "東京", "Tokyo, JP"）
        unit: 温度単位（"celsius" または "fahrenheit"）

    Returns:
        天気情報の文字列
    """
    # 実際の実装では OpenWeatherMap API などを使用
    async with httpx.AsyncClient(timeout=5.0) as client:
        try:
            resp = await client.get(
                "https://api.openweathermap.org/data/2.5/weather",
                params={
                    "q": location,
                    "appid": os.getenv("OPENWEATHER_API_KEY"),
                    "units": "metric" if unit == "celsius" else "imperial",
                    "lang": "ja",
                }
            )
            resp.raise_for_status()
            data = resp.json()
            temp = data["main"]["temp"]
            desc = data["weather"][0]["description"]
            return f"{location}の現在の天気: {desc}、気温 {temp:.1f}°{'C' if unit == 'celsius' else 'F'}"
        except Exception as e:
            return f"天気情報の取得に失敗しました: {str(e)}"

async def get_calendar_events(date_str: str = "today") -> str:
    """
    指定した日付のカレンダー予定を取得する

    Args:
        date_str: 日付文字列（"today", "tomorrow", "YYYY-MM-DD" 形式）

    Returns:
        予定一覧の文字列
    """
    if date_str == "today":
        target_date = datetime.now()
    elif date_str == "tomorrow":
        target_date = datetime.now() + timedelta(days=1)
    else:
        target_date = datetime.strptime(date_str, "%Y-%m-%d")

    # 実際の実装では Google Calendar API を呼び出す
    # ここではモックデータを返す
    mock_events = [
        {"time": "10:00", "title": "チームミーティング"},
        {"time": "14:00", "title": "プロジェクトレビュー"},
        {"time": "16:30", "title": "1on1"},
    ]

    date_label = target_date.strftime("%Y年%m月%d日")
    events_str = "\n".join([f"  {e['time']} {e['title']}" for e in mock_events])
    return f"{date_label}の予定:\n{events_str}"

# ─── ADK FunctionTool として登録 ───
weather_tool = FunctionTool(func=get_current_weather)
calendar_tool = FunctionTool(func=get_calendar_events)

# Live API に渡すツール定義を生成
AVAILABLE_TOOLS = [weather_tool, calendar_tool]

# 期待する動作例:
# ユーザー「今日の東京の天気は？」
# → get_current_weather(location="東京") が呼び出される
# → 「東京の現在の天気: 晴れ、気温 18.5°C」が返される
# → AI が自然な文章に変換して音声で回答
```

## FastAPI WebSocket サーバーの実装

```python
# server.py
import asyncio
import json
import os
import logging
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.middleware.cors import CORSMiddleware
from voice_assistant.session import LiveVoiceSession
from voice_assistant.tools import AVAILABLE_TOOLS

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

app = FastAPI(title="Gemini Voice Assistant API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 本番では適切なオリジンに制限すること
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.websocket("/ws/voice")
async def voice_endpoint(websocket: WebSocket):
    """
    音声 AI アシスタントの WebSocket エンドポイント

    クライアントからのメッセージ形式:
    - バイナリ: 音声 PCM データ（16kHz、16bit、モノラル）
    - JSON テキスト: {"type": "end_of_speech"} — 発話終了通知

    サーバーからのレスポンス形式:
    - JSON: {"type": "audio", "data": "<base64>"}
    - JSON: {"type": "text", "data": "..."}
    - JSON: {"type": "turn_complete"}
    - JSON: {"type": "error", "message": "..."}
    """
    await websocket.accept()
    session = LiveVoiceSession(api_key=os.getenv("GOOGLE_API_KEY"))

    try:
        # ADK ツール定義を Live API 形式に変換して渡す
        live_tools = [{"function_declarations": [t.get_declaration() for t in AVAILABLE_TOOLS]}]
        await session.connect(tools=live_tools)

        # 音声受信タスクとレスポンス送信タスクを並行実行
        receive_task = asyncio.create_task(_receive_audio(websocket, session))
        send_task = asyncio.create_task(_send_responses(websocket, session))

        await asyncio.gather(receive_task, send_task)

    except WebSocketDisconnect:
        logger.info("クライアントが切断しました")
    except Exception as e:
        logger.error(f"エラー: {e}")
        await websocket.send_json({"type": "error", "message": str(e)})
    finally:
        await session.close()

async def _receive_audio(websocket: WebSocket, session: LiveVoiceSession):
    """クライアントからの音声データを受信して Live API に送信する"""
    async for message in websocket.iter_bytes():
        await session.stream_audio_input(message)

async def _send_responses(websocket: WebSocket, session: LiveVoiceSession):
    """Live API からのレスポンスをクライアントに転送する"""
    import base64

    async for response in session.receive_responses():
        if response["type"] == "audio":
            # 音声データは base64 エンコードして JSON で送信
            await websocket.send_json({
                "type": "audio",
                "data": base64.b64encode(response["data"]).decode(),
            })
        elif response["type"] == "tool_call":
            # ツール呼び出しを実行して結果を返送
            tool_data = response["data"]
            result = await _execute_tool(tool_data["name"], tool_data["args"])
            await session.send_tool_result(tool_data["id"], result)
        else:
            await websocket.send_json(response)

async def _execute_tool(name: str, args: dict) -> str:
    """ツール名と引数からツールを実行して結果を返す"""
    tool_map = {t.func.__name__: t.func for t in AVAILABLE_TOOLS}
    if name in tool_map:
        return await tool_map[name](**args)
    return f"ツール '{name}' が見つかりません"

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8080)
```

## よくある質問（FAQ）

**Q1. Live API のレイテンシはどのくらいですか？**

Gemini 3.0 Flash を使用した場合、音声入力から最初の音声出力チャンクまでのレイテンシは東京リージョンで平均 300〜500ms です。ツール呼び出しが発生する場合は追加で 200〜800ms かかります。レイテンシを最小化するには①Cloud Run のリージョンをユーザーに近い場所（日本なら `asia-northeast1`）に設定する、②ウォームインスタンスを最低 1 台確保する（`--min-instances 1`）、③ツール実装にタイムアウト（5 秒以内）を設定する、の 3 点が有効です。

**Q2. 複数ユーザーの同時接続はどうスケールしますか？**

Cloud Run は WebSocket 接続を含むステートフルなセッションをサポートしています。デフォルトでは 1 インスタンスに最大 80 の同時接続を処理でき、`--max-instances` で上限を設定したオートスケールが自動的に動作します。100 同時ユーザーを想定する場合、`--min-instances 2 --max-instances 20` が安全な設定です。

**Q3. Live API はどの言語の音声に対応していますか？**

2026 年 3 月時点で、Live API は日本語・英語・スペイン語・フランス語・ドイツ語・韓国語・中国語（簡体字）を含む 24 言語の音声入力をサポートしています。出力音声（TTS）は英語のプリビルトボイスのみ提供されています。日本語で話しかけて日本語テキストで返答するパターンは完全にサポートされていますが、日本語音声出力は現時点ではベータ機能として提供されています。

## まとめ

Gemini Live API × ADK の組み合わせで実現できる音声 AI アシスタントのポイントをまとめます。

- **双方向リアルタイムストリーミング**：WebSocket でゼロレイテンシに近い音声入出力を実現
- **ツール連携**：ADK の `FunctionTool` を使い、カレンダー・天気・DB など外部サービスを音声コマンドで操作
- **マルチターン会話**：ADK がセッション状態を管理し、文脈を保った長い会話を実現
- **Cloud Run デプロイ**：スケーラブルな WebSocket サーバーを最小限のインフラ管理で本番運用
