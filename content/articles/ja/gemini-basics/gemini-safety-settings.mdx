## Gemini セーフティ設定ガイド

**セーフティ設定**は、Gemini APIが有害なコンテンツの生成や処理を防ぐための重要な機能です。このガイドでは、セーフティ設定の概念から実装方法まで、初心者でも理解できるように解説します。

## セーフティ設定とは

Gemini APIは、不適切なコンテンツの生成を防ぐために、複数のセーフティ機構を備えています。これらの機構は、以下の4つの有害カテゴリを対象としています：

### 4つの有害カテゴリ

1. **HARM_CATEGORY_HATE_SPEECH** - 差別や偏見に関するコンテンツ
2. **HARM_CATEGORY_SEXUALLY_EXPLICIT** - 性的に露骨なコンテンツ
3. **HARM_CATEGORY_VIOLENCE** - 暴力的なコンテンツ
4. **HARM_CATEGORY_DANGEROUS_ILLEGAL_ACTIVITY** - 違法行為や危険な活動

## セーフティレベル（Threshold）の理解

各有害カテゴリに対して、フィルタリング閾値を設定することで、ブロック対象の厳しさを調整できます。

### 4つのレベル

| レベル | 説明 | 使用例 |
|--------|------|--------|
| **BLOCK_NONE** | フィルタリングなし | 開発・テスト環境、自動化されたコンテンツ監視 |
| **BLOCK_ONLY_HIGH** | 高リスク | 最も高い危険度のみブロック |
| **BLOCK_MEDIUM_AND_ABOVE** | 中-高リスク | 一般的なアプリケーション（推奨） |
| **BLOCK_LOW_AND_ABOVE** | すべてをブロック | 教育機関やキッズコンテンツなど |

## セーフティ設定の実装

### 基本的な設定方法

```python
import anthropic

client = anthropic.Anthropic(api_key="YOUR_API_KEY")

# シンプルなセーフティ設定
message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    safety_settings=[
        {
            "category": "HARM_CATEGORY_HATE_SPEECH",
            "threshold": "BLOCK_MEDIUM_AND_ABOVE"
        },
        {
            "category": "HARM_CATEGORY_VIOLENCE",
            "threshold": "BLOCK_MEDIUM_AND_ABOVE"
        }
    ],
    messages=[
        {
            "role": "user",
            "content": "Geminiのセーフティ設定について説明してください"
        }
    ]
)

print(message.content[0].text)

# 予期される出力:
# Gemini APIのセーフティ設定は、有害なコンテンツの生成を防ぐための機能です。
# ...（安全な応答）
```

### カスタマイズされたセーフティ設定

各使用ケースに応じて、異なるセーフティレベルを適用できます。例えば、コンテンツモデレーション用途では厳しいフィルタリングを、創作支援では比較的緩いフィルタリングを設定することが一般的です。

```python
# 用途別のセーフティ設定例

# 1. 教育用途（厳しいフィルタリング）
educational_settings = [
    {
        "category": "HARM_CATEGORY_HATE_SPEECH",
        "threshold": "BLOCK_LOW_AND_ABOVE"
    },
    {
        "category": "HARM_CATEGORY_SEXUALLY_EXPLICIT",
        "threshold": "BLOCK_LOW_AND_ABOVE"
    },
    {
        "category": "HARM_CATEGORY_VIOLENCE",
        "threshold": "BLOCK_LOW_AND_ABOVE"
    },
    {
        "category": "HARM_CATEGORY_DANGEROUS_ILLEGAL_ACTIVITY",
        "threshold": "BLOCK_LOW_AND_ABOVE"
    }
]

# 2. 創作支援（緩いフィルタリング）
creative_settings = [
    {
        "category": "HARM_CATEGORY_HATE_SPEECH",
        "threshold": "BLOCK_ONLY_HIGH"
    },
    {
        "category": "HARM_CATEGORY_VIOLENCE",
        "threshold": "BLOCK_ONLY_HIGH"
    }
]

# 3. テスト・開発用（フィルタリングなし）
testing_settings = [
    {
        "category": "HARM_CATEGORY_HATE_SPEECH",
        "threshold": "BLOCK_NONE"
    },
    {
        "category": "HARM_CATEGORY_SEXUALLY_EXPLICIT",
        "threshold": "BLOCK_NONE"
    },
    {
        "category": "HARM_CATEGORY_VIOLENCE",
        "threshold": "BLOCK_NONE"
    },
    {
        "category": "HARM_CATEGORY_DANGEROUS_ILLEGAL_ACTIVITY",
        "threshold": "BLOCK_NONE"
    }
]

# セーフティ設定を適用
def generate_with_safety_level(prompt, safety_settings):
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=1024,
        safety_settings=safety_settings,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

# 教育用途で実行
educational_response = generate_with_safety_level(
    "子どもたちに教える物語を作成してください",
    educational_settings
)
```

## セーフティ設定からのレスポンス処理

Gemini APIがセーフティ理由によってブロックした場合、レスポンスには stop_reason が "end_turn" ではなく"safety" が設定されます。

```python
def safe_generate_content(prompt: str, safety_settings: list) -> dict:
    """セーフティ設定を考慮してコンテンツを生成"""
    try:
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=1024,
            safety_settings=safety_settings,
            messages=[{"role": "user", "content": prompt}]
        )

        # stop_reason を確認
        if response.stop_reason == "safety":
            return {
                "success": False,
                "error": "セーフティフィルタによってブロックされました",
                "blocked": True
            }

        return {
            "success": True,
            "content": response.content[0].text,
            "blocked": False
        }

    except Exception as e:
        return {
            "success": False,
            "error": str(e),
            "blocked": False
        }

# 使用例
result = safe_generate_content(
    "プロダクト評価について説明してください",
    creative_settings
)

if result["success"]:
    print("生成結果:", result["content"])
else:
    if result["blocked"]:
        print("有害なコンテンツと判定されました")
    else:
        print("エラーが発生しました:", result["error"])
```

## よくある質問

### Q1: セーフティ設定を無効化できますか？

A: はい、すべてのカテゴリを "BLOCK_NONE" に設定することで、セーフティフィルタリングを無効化できます。ただし、本番環境での使用は推奨されません。開発やテスト用途に限定してください。

### Q2: セーフティ設定がプライバシーに影響を与えることはありますか？

A: いいえ。セーフティ設定はコンテンツのフィルタリングロジックに影響するだけで、入力データの保存や収集には影響しません。プライバシーポリシーを確認して、データの取り扱いについて理解しておくことが重要です。

### Q3: セーフティ設定でブロックされた場合、何をすればいいですか？

A: ブロックされた場合は、プロンプトの表現方法を変更するか、セーフティレベルを調整してください。重要な機能の場合は、適切なセーフティレベルを選択し、ユーザーに説明を提供することが大切です。

### Q4: セーフティ設定はどの程度信頼できますか？

A: セーフティ設定は有害なコンテンツの大部分をフィルタリングできますが、100%完全ではありません。重要なアプリケーションでは、セーフティ設定に加えて、人間によるレビューやその他のモデレーション手法を組み合わせることをお勧めします。
