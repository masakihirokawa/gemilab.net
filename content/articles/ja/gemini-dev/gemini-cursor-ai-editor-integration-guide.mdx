## Cursor と Gemini の組み合わせが注目される理由

AI コーディングエディタとして急速に普及している **Cursor** は、OpenAI の GPT シリーズだけでなく、Google の **Gemini** モデルとも連携できることをご存知でしょうか。Gemini 3.1 Pro の長大なコンテキストウィンドウや、高速な Gemini 3.1 Flash の推論能力を Cursor の中で直接活用できれば、開発効率は飛躍的に向上します。

本記事では、Cursor エディタに Gemini モデルを設定する手順から、日常のコーディングで効果的に使いこなすテクニックまでを、初めての方にもわかりやすく解説します。Gemini API キーの取得方法、Cursor 側の設定画面での操作、そしてコード補完・チャット・Composer 機能でのベストプラクティスをステップバイステップでお伝えします。

## Gemini API キーの取得と準備

Cursor で Gemini を使うには、まず **Google AI Studio** から API キーを取得する必要があります。

### API キー取得の手順

1. [Google AI Studio](https://aistudio.google.com/) にアクセスし、Google アカウントでログインします
2. 左メニューの「Get API Key」をクリックします
3. 「Create API key」を選択し、プロジェクトを指定してキーを生成します
4. 生成されたキーを安全な場所にコピーして保管します

```python
# API キーの動作確認（Python）
import google.generativeai as genai

genai.configure(api_key="YOUR_GEMINI_API_KEY")

model = genai.GenerativeModel("gemini-3.1-pro")
response = model.generate_content("Hello, Gemini!")
print(response.text)
# 出力例: Hello! How can I help you today?
```

**注意**: API キーは環境変数や `.env` ファイルで管理し、コードに直接書き込まないようにしましょう。無料枠でも十分な量のリクエストが利用できるため、個人開発にもおすすめです。

## Cursor で Gemini モデルを設定する方法

Cursor は **OpenAI 互換 API** をサポートしているため、Gemini API を OpenAI 互換エンドポイント経由で利用できます。以下の手順で設定を行います。

### 設定手順（OpenAI 互換モード）

1. Cursor を起動し、`Cmd + ,`（Mac）または `Ctrl + ,`（Windows/Linux）で設定画面を開きます
2. 左メニューから「Models」セクションに移動します
3. 「Add model」をクリックして、以下の情報を入力します

設定に必要な情報は以下の通りです。

- **Model name**: `gemini-3.1-pro`（または `gemini-3.1-flash`）
- **API Base URL**: `https://generativelanguage.googleapis.com/v1beta/openai/`
- **API Key**: Google AI Studio で取得したキー

```javascript
// Cursor の settings.json に直接記述する場合の例
// （Cursor の設定画面から操作する方が安全です）
{
  "cursor.ai.models": [
    {
      "name": "gemini-3.1-pro",
      "apiBase": "https://generativelanguage.googleapis.com/v1beta/openai/",
      "apiKey": "YOUR_GEMINI_API_KEY",
      "provider": "openai-compatible"
    },
    {
      "name": "gemini-3.1-flash",
      "apiBase": "https://generativelanguage.googleapis.com/v1beta/openai/",
      "apiKey": "YOUR_GEMINI_API_KEY",
      "provider": "openai-compatible"
    }
  ]
}
```

### モデルの使い分け

Gemini のモデルはそれぞれ特徴が異なるため、用途に応じて使い分けることで、コストパフォーマンスを最大化できます。

- **Gemini 3.1 Pro**: 複雑なリファクタリング、アーキテクチャ設計、大規模コードベースの理解に最適。100万トークンを超えるコンテキストウィンドウが強み
- **Gemini 3.1 Flash**: 日常的なコード補完、簡単な質問、スニペット生成に最適。応答速度が速くコストも低い
- **Gemini 3.1 Flash-Lite**: 最もコストが低いモデル。シンプルな補完やフォーマット整形に向いている

Cursor のチャット機能では、メッセージ入力時にモデルを切り替えられるため、質問の複雑さに応じてリアルタイムに選択できます。

## Cursor チャットで Gemini を活用するテクニック

Cursor の右サイドパネルにある **Chat** 機能は、Gemini モデルとの対話でコードに関する質問をするのに最適です。

### 効果的なプロンプトの書き方

Gemini はコンテキストの理解力が高いため、現在開いているファイルの内容を踏まえた質問が可能です。以下のようなプロンプトパターンが効果的です。

```
# パターン1: コードの説明を求める
この関数の処理内容をステップバイステップで説明してください。
特にエラーハンドリングの部分で改善できる点があれば指摘してください。

# パターン2: リファクタリングの提案
このコンポーネントを React のベストプラクティスに従ってリファクタリングしてください。
パフォーマンス最適化とアクセシビリティの改善も含めてください。

# パターン3: テストコードの生成
この関数に対する単体テストを Jest で書いてください。
正常系・異常系・境界値のテストケースを含めてください。
```

### `@` メンションによるコンテキスト指定

Cursor のチャットでは `@` 記号を使って特定のファイルやシンボルを参照できます。Gemini の長大なコンテキストウィンドウと組み合わせると、プロジェクト全体を俯瞰した回答が得られます。

- `@filename.ts` — 特定のファイルをコンテキストに含める
- `@folder` — フォルダ全体を参照する
- `@codebase` — プロジェクト全体から関連コードを検索する

Gemini 3.1 Pro の 100 万トークン超のコンテキストウィンドウは、大規模プロジェクトの横断的な理解に特に威力を発揮します。複数ファイルにまたがるリファクタリングや、依存関係の分析といった作業で、他のモデルでは扱いきれない量のコードを一度に渡せるのは大きな利点です。

## Composer 機能で Gemini を使ったコード生成

Cursor の **Composer**（`Cmd + I` / `Ctrl + I`）は、自然言語の指示からコードを生成・編集する強力な機能です。Gemini モデルを選択して使うことで、独自の強みを活かしたコード生成が可能になります。

### 実践例: API エンドポイントの作成

以下は、Composer で Gemini に指示を出して REST API エンドポイントを生成する例です。

```typescript
// Composer への指示:
// 「ユーザー認証付きのブログ記事 CRUD API を Express で作成してください。
//   バリデーション、エラーハンドリング、型定義を含めてください。」

// Gemini が生成するコード例:
import express, { Request, Response, NextFunction } from 'express';
import { z } from 'zod';

// バリデーションスキーマの定義
const createArticleSchema = z.object({
  title: z.string().min(1).max(200),
  content: z.string().min(10),
  tags: z.array(z.string()).optional(),
});

type CreateArticleInput = z.infer<typeof createArticleSchema>;

const router = express.Router();

// 記事の作成
router.post('/articles', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const validated: CreateArticleInput = createArticleSchema.parse(req.body);

    // データベースへの保存処理
    const article = await createArticle({
      ...validated,
      authorId: req.user.id,
      createdAt: new Date(),
    });

    res.status(201).json({ success: true, data: article });
  } catch (error) {
    if (error instanceof z.ZodError) {
      res.status(400).json({
        success: false,
        errors: error.errors,
      });
      return;
    }
    next(error);
  }
});

// 期待する出力: Express ルーターが正常に登録され、
// POST /articles でバリデーション付きの記事作成が可能になる
```

Gemini は特に **型定義の正確さ** と **エラーハンドリングの網羅性** に優れています。TypeScript プロジェクトでの利用では、型推論が的確で余計な `any` 型が生成されにくいという声も多く聞かれます。

## Gemini CLI との併用ワークフロー

Cursor での Gemini 活用に加えて、ターミナルから **Gemini CLI** を併用するワークフローも非常に効率的です。Cursor のターミナルパネルで Gemini CLI を起動すれば、エディタを離れることなくコマンドライン操作と AI アシストを切り替えられます。

```bash
# Cursor のターミナルで Gemini CLI を使う例
# プロジェクト全体のコード品質をチェック
gemini "このプロジェクトの src/ ディレクトリを確認して、
  パフォーマンス上の問題点を指摘してください"

# テストカバレッジの改善提案
gemini "src/utils/ 内のユーティリティ関数で
  テストが不足しているものをリストアップしてください"
```

Gemini CLI の詳しい使い方については [Gemini CLI ターミナル AI 開発ガイド](/articles/gemini-dev/gemini-cli-terminal-ai-development-guide) も参考になります。また、より高度なエージェント機能を統合した開発環境に興味がある方は、[Gemini CLI × AGENTS.md で構築する AI エージェント統合開発環境](/articles/gemini-advanced/gemini-cli-agents-md-integrated-dev) もご覧ください。

## トラブルシューティングと注意点

Cursor で Gemini を使う際に起こりやすい問題と、その対処法をまとめます。

### よくある問題と解決策

**接続エラーが発生する場合**: API Base URL が正しいか確認してください。末尾のスラッシュ（`/`）の有無で接続に失敗するケースがあります。`https://generativelanguage.googleapis.com/v1beta/openai/` が正しい形式です。

**応答が遅い場合**: Gemini 3.1 Pro は高品質な回答を生成するため、やや応答に時間がかかることがあります。速度を優先する場合は Gemini 3.1 Flash に切り替えてみてください。

**トークン制限エラーが表示される場合**: 無料プランには 1 分あたりのリクエスト数と 1 日あたりのトークン数に上限があります。頻繁にエラーが出る場合は、[Google AI Studio](https://aistudio.google.com/) でクォータを確認するか、従量課金プランへの移行を検討してください。

### セキュリティ上の注意

- API キーは Cursor の設定画面から入力し、`settings.json` に平文で残さないようにしましょう
- 機密性の高いコードを扱う場合は、Gemini API のデータポリシーを事前に確認してください
- チーム開発では、API キーを個人ごとに発行し、共有しないようにしましょう

## FAQ

### Q: Cursor で Gemini を使うのに費用はかかりますか？

Gemini API には無料枠があり、個人開発レベルであれば追加費用なしで利用できることが多いです。ただし、Cursor 自体の Pro プランは別途サブスクリプション費用がかかります。Gemini API の無料枠は、1 分あたり 15 リクエスト（3.1 Flash）、1 日あたり 1,500 リクエストが目安です。

### Q: Gemini と Cursor に標準搭載されている GPT-4o、どちらを使うべきですか？

用途によって使い分けるのがベストです。大量のコードを一度に理解させたい場合は Gemini 3.1 Pro のコンテキストウィンドウが有利です。一般的なコーディング支援では GPT-4o も優秀なので、両方を試して自分のワークフローに合う方を選ぶのがおすすめです。

### Q: Cursor 以外の AI エディタでも Gemini は使えますか？

はい。VS Code + Roo Code 拡張や、Gemini Code Assist（Google 公式の VS Code 拡張）でも Gemini モデルを利用できます。詳しくは [VS Code × Roo Code × Gemini 開発環境構築ガイド](/articles/gemini-dev/vscode-roo-code-gemini-dev-environment) をご覧ください。

### Q: Cursor の Composer で Gemini を使うときのコツはありますか？

Composer では、指示を具体的かつ構造化することが重要です。「○○を作って」だけでなく、「TypeScript で、Zod バリデーション付きで、エラーハンドリングを含めて○○を作って」のように、技術スタックや品質要件を明示すると、Gemini はより正確なコードを生成します。
