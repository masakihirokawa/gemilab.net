## はじめに

チーム開発において、コードレビューは品質を担保する重要なプロセスです。しかし、レビュー待ちの時間がボトルネックになることも少なくありません。Gemini API の優れたコード理解力を活用すれば、プルリクエスト（PR）が作成された瞬間に自動でレビューコメントを付けるBotを構築できます。

この記事では、Gemini 2.5 Pro の長大なコンテキストウィンドウとコード解析能力を活かし、GitHub Actions 上で動作する AI コードレビューBotをゼロから作り上げます。人間のレビュアーが見落としがちなバグパターンやセキュリティリスクを自動検出し、開発チームの生産性を大幅に向上させましょう。

**この記事で学べること:**

- Gemini API を使ったコード差分（diff）の解析手法
- GitHub Actions でPR作成時に自動実行されるワークフローの構築
- レビューコメントをPRに直接投稿するPythonスクリプトの実装

## 前提知識・環境準備

### 必要なもの

- **Google AI Studio のAPIキー**: [Google AI Studio](https://aistudio.google.com/) から取得
- **GitHub リポジトリ**: レビュー対象のリポジトリ（パブリック・プライベートどちらでも可）
- **Python 3.10以上**: ローカル開発・テスト用
- **GitHub Actions**: CI/CDランナー（無料枠で十分）

### APIキーの取得

まだAPIキーをお持ちでない方は、[Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart)を参照してください。Google AI Studio にアクセスし、「Get API key」からキーを発行します。

### 依存パッケージのインストール

```bash
pip install google-genai PyGithub
```

`google-genai` は Gemini API の公式Pythonクライアント、`PyGithub` は GitHub API を操作するためのライブラリです。

## アーキテクチャの全体像

AI コードレビューBotは、以下の流れで動作します。

1. **PRが作成・更新される** → GitHub Actions ワークフローがトリガー
2. **差分を取得** → GitHub API でPRのファイル変更一覧とdiffを取得
3. **Gemini API で解析** → コード差分をプロンプトに含めてレビューを依頼
4. **コメント投稿** → レビュー結果をPRのコメントとして投稿

```
┌─────────┐     ┌──────────────┐     ┌─────────────┐     ┌──────────┐
│  PR作成  │────▶│ GitHub Actions│────▶│  Gemini API  │────▶│ PRコメント │
│  更新    │     │  ワークフロー  │     │  コード解析   │     │  投稿     │
└─────────┘     └──────────────┘     └─────────────┘     └──────────┘
```

## ステップ1: コアとなるレビュースクリプトの実装

まず、Gemini API にコード差分を送信し、レビューコメントを生成するPythonスクリプトを作成します。

```python
# review_bot.py
import os
import json
from google import genai
from github import Github

# 環境変数から認証情報を取得
GEMINI_API_KEY = os.environ["GEMINI_API_KEY"]
GITHUB_TOKEN = os.environ["GITHUB_TOKEN"]
REPO_NAME = os.environ["GITHUB_REPOSITORY"]
PR_NUMBER = int(os.environ["PR_NUMBER"])

# Gemini クライアント初期化
client = genai.Client(api_key=GEMINI_API_KEY)

# GitHub クライアント初期化
gh = Github(GITHUB_TOKEN)
repo = gh.get_repo(REPO_NAME)
pr = repo.get_pull(PR_NUMBER)


def get_pr_diff() -> str:
    """PRの変更ファイルとdiffを取得"""
    files = pr.get_files()
    diff_text = ""
    for f in files:
        # バイナリファイルはスキップ
        if f.patch is None:
            continue
        diff_text += f"### {f.filename}\n"
        diff_text += f"Status: {f.status}\n"
        diff_text += f"```diff\n{f.patch}\n```\n\n"
    return diff_text


def review_with_gemini(diff: str) -> str:
    """Gemini API でコードレビューを実行"""
    prompt = f"""あなたは経験豊富なシニアソフトウェアエンジニアです。
以下のプルリクエストの差分をレビューしてください。

## レビュー観点
1. **バグ・ロジックエラー**: 潜在的なバグやエッジケースの見落とし
2. **セキュリティ**: SQLインジェクション、XSS、認証漏れなど
3. **パフォーマンス**: N+1クエリ、不要なループ、メモリリークの可能性
4. **可読性**: 変数名、関数分割、コメントの適切さ
5. **ベストプラクティス**: 言語固有のイディオム、設計パターン

## 出力形式
以下のJSON形式で出力してください:
{{
  "summary": "全体的な評価（1〜2文）",
  "issues": [
    {{
      "severity": "critical|warning|suggestion",
      "file": "ファイル名",
      "description": "指摘内容の説明"
    }}
  ],
  "positive": ["良い点を1〜2個"]
}}

issuesが無い場合は空配列にしてください。

## PRの差分
{diff}
"""

    response = client.models.generate_content(
        model="gemini-2.5-pro",
        contents=prompt,
        config={
            "response_mime_type": "application/json",
            "temperature": 0.2,  # 安定したレビューのため低めに設定
        },
    )
    return response.text


def format_review_comment(review_json: str) -> str:
    """レビュー結果をMarkdown形式に整形"""
    review = json.loads(review_json)

    comment = "## 🤖 AI Code Review by Gemini\n\n"
    comment += f"**総評**: {review['summary']}\n\n"

    # 良い点
    if review.get("positive"):
        comment += "### ✅ 良い点\n"
        for point in review["positive"]:
            comment += f"- {point}\n"
        comment += "\n"

    # 指摘事項
    if review.get("issues"):
        severity_icons = {
            "critical": "🔴",
            "warning": "🟡",
            "suggestion": "🔵",
        }
        comment += "### 📝 指摘事項\n\n"
        for issue in review["issues"]:
            icon = severity_icons.get(issue["severity"], "⚪")
            comment += f"{icon} **[{issue['severity'].upper()}]** "
            comment += f"`{issue['file']}`\n"
            comment += f"> {issue['description']}\n\n"
    else:
        comment += "### 📝 指摘事項\nなし — コードは良好です！ 🎉\n\n"

    comment += "---\n"
    comment += "*Powered by Gemini 2.5 Pro — [Gemini Lab](https://gemilab.net)*"
    return comment


def main():
    print(f"Reviewing PR #{PR_NUMBER} in {REPO_NAME}...")

    # 差分を取得
    diff = get_pr_diff()
    if not diff.strip():
        print("No reviewable changes found.")
        return

    # 差分が大きすぎる場合は先頭部分のみ
    max_chars = 100_000  # Gemini 2.5 Pro は100万トークン対応
    if len(diff) > max_chars:
        diff = diff[:max_chars] + "\n\n(... truncated due to size)"

    # Gemini でレビュー実行
    review_json = review_with_gemini(diff)
    print(f"Review result: {review_json[:200]}...")

    # コメント整形・投稿
    comment = format_review_comment(review_json)
    pr.create_issue_comment(comment)
    print("Review comment posted successfully!")


if __name__ == "__main__":
    main()
```

### コードのポイント

- **Structured Output**: `response_mime_type: "application/json"` を指定することで、Gemini がJSON形式で安定した出力を返します
- **低い temperature**: コードレビューでは創造性より正確性が重要なため、`temperature: 0.2` に設定
- **差分サイズ制限**: 万一巨大なPRでもエラーにならないよう、最大文字数でトランケート

## ステップ2: GitHub Actions ワークフローの設定

リポジトリの `.github/workflows/ai-review.yml` に以下のワークフローを作成します。

```yaml
# .github/workflows/ai-review.yml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai PyGithub

      - name: Run AI Review
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GITHUB_REPOSITORY: ${{ github.repository }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
        run: python review_bot.py
```

### シークレットの設定

リポジトリの **Settings → Secrets and variables → Actions** から以下を登録します。

| シークレット名 | 値 |
|---|---|
| `GEMINI_API_KEY` | Google AI Studio で取得したAPIキー |

`GITHUB_TOKEN` は GitHub Actions が自動で提供するため、手動設定は不要です。

## ステップ3: レビュー精度を向上させるプロンプトチューニング

基本的なBotが動いたら、プロンプトを改善してレビューの質を高めましょう。

### プロジェクト固有のルールを追加する

```python
# プロジェクト固有のレビュールール（.ai-review-rules.md から読み込み）
def load_custom_rules() -> str:
    rules_path = ".ai-review-rules.md"
    if os.path.exists(rules_path):
        with open(rules_path, "r") as f:
            return f.read()
    return ""

# プロンプトに追加
custom_rules = load_custom_rules()
if custom_rules:
    prompt += f"\n## プロジェクト固有のルール\n{custom_rules}\n"
```

`.ai-review-rules.md` の例:

```markdown
# レビュールール
- TypeScript では `any` 型の使用を禁止する
- API エンドポイントには必ずバリデーションを実装する
- console.log はデバッグ用のため、PRに含めない
- テストファイルの変更には対応するテストケースの追加を確認する
```

### ファイルタイプ別にレビュー基準を変える

```python
def get_file_context(filename: str) -> str:
    """ファイル種別に応じた追加コンテキストを返す"""
    if filename.endswith((".ts", ".tsx")):
        return "TypeScript: 型安全性、null チェック、async/await の正しい使用を重点確認"
    elif filename.endswith(".py"):
        return "Python: 型ヒント、例外処理、PEP 8準拠を重点確認"
    elif filename.endswith((".yml", ".yaml")):
        return "YAML設定: インデント、環境変数の漏洩、セキュリティ設定を確認"
    elif filename.endswith("Dockerfile"):
        return "Docker: マルチステージビルド、root実行の回避、レイヤーキャッシュ最適化を確認"
    return ""
```

## ステップ4: レート制限とエラーハンドリング

本番環境で安定稼働させるために、適切なエラーハンドリングを実装します。

```python
import time
from google.api_core import exceptions as google_exceptions


def review_with_retry(diff: str, max_retries: int = 3) -> str:
    """リトライ付きでGemini APIを呼び出す"""
    for attempt in range(max_retries):
        try:
            return review_with_gemini(diff)
        except google_exceptions.ResourceExhausted:
            # レート制限に達した場合
            wait_time = 2 ** attempt * 10  # 10, 20, 40秒
            print(f"Rate limited. Retrying in {wait_time}s... ({attempt + 1}/{max_retries})")
            time.sleep(wait_time)
        except google_exceptions.InvalidArgument as e:
            # プロンプトが長すぎる場合
            print(f"Input too large: {e}")
            if len(diff) > 10000:
                diff = diff[:len(diff) // 2] + "\n\n(... truncated)"
                continue
            raise
        except Exception as e:
            print(f"Unexpected error: {e}")
            if attempt < max_retries - 1:
                time.sleep(5)
                continue
            raise
    raise RuntimeError("Max retries exceeded for Gemini API call")
```

## よくある質問（FAQ）

### Q1: レビューにかかる時間はどのくらいですか？

Gemini 2.5 Pro の場合、一般的なPR（数百行の変更）で10〜30秒程度です。GitHub Actions の起動時間を含めても1〜2分以内にコメントが投稿されます。大規模なPR（数千行）でも、Gemini の100万トークンのコンテキストウィンドウのおかげで問題なく処理できます。

### Q2: APIの利用料金はどのくらいかかりますか？

Gemini 2.5 Pro の料金は入力トークンあたり約$1.25/100万トークン、出力$10/100万トークンです（2026年3月時点）。一般的なPRレビュー1回あたりの入力は約2,000〜10,000トークン、出力は約500〜1,000トークンなので、1レビューあたり約$0.01〜$0.05 程度です。月に100回レビューしても数ドル程度で収まります。

### Q3: 機密コードをGemini APIに送信しても安全ですか？

Google の利用規約では、API経由で送信されたデータはモデルの学習に使用されません。ただし、機密性の高いプロジェクトでは、Vertex AI のVPC Service Controls やプライベートエンドポイントの利用を検討してください。詳しくは [Vertex AI でGemini をプロダクション利用する方法](/articles/gemini-dev/vertex-ai-gemini-production)もご覧ください。

### Q4: 人間のレビューは不要になりますか？

AI レビューは人間のレビューを**補完**するものです。アーキテクチャの妥当性やビジネスロジックの正しさは、依然として人間のレビュアーの判断が必要です。AIが定型的なチェック（コーディング規約、セキュリティパターン、潜在バグ）を担当することで、人間はより高レベルな判断に集中できるようになります。

## まとめ

Gemini API と GitHub Actions を組み合わせることで、PR作成と同時に自動でコードレビューを実行するBotを構築できます。セットアップにかかる時間はわずか30分程度ですが、チーム全体の開発速度とコード品質に大きなインパクトをもたらします。

- **カスタムルールの蓄積**: プロジェクト固有の`.ai-review-rules.md` を育てていく

GitHub Actions × Gemini API の連携についてさらに詳しくは、[Gemini API × GitHub Actions CI/CD パイプラインガイド](/articles/gemini-dev/gemini-github-actions-ci-cd-guide)をお読みください。
