## はじめに

Google AI Studio（`aistudio.google.com`）は Gemini API を最も簡単に試せるブラウザツールですが、多くの初心者がセットアップで詰まります。

このガイドでは、**実際に起こりやすい 6 つのエラーと、それぞれの確実な解決手順**を紹介します。

## 1. API キーが認識されないエラー

### 問題：「API key not valid」と表示される

API キーを貼り付けたのに、何度も同じエラーが出る場合、多くの場合は以下のいずれかです：

1. **リージョン制限** — 日本など一部の地域では API キー生成が制限される
2. **課金設定がない** — プロジェクトに支払い方法が登録されていない
3. **キーをコピペする際に余分な空白が含まれている**
4. **複数のプロジェクトがあり、違うプロジェクトから API キーを取得している**

### 解決方法：正しいプロジェクトと課金を確認

**ステップ 1: Google Cloud Console で課金を有効化**

```
1. https://console.cloud.google.com を開く
2. 画面左上の「プロジェクト選択」ドロップダウンをクリック
3. 利用中のプロジェクトを選択
4. 左メニューの「お支払い」をクリック
5. 「お支払いアカウントをリンク」を確認
6. リンクがなければ「お支払いアカウントをリンク」から新規作成
```

**ステップ 2: Gemini API を有効化**

```
1. Google Cloud Console で「API とサービス」→「有効な API とサービス」
2. 「+ API とサービスを有効にする」をクリック
3. 「Generative Language API」を検索して有効化
```

**ステップ 3: API キーを新規作成**

```bash
# 正しく作成されているか確認するコマンド
curl -H "x-goog-api-key: YOUR_API_KEY" \
  "https://generativelanguage.googleapis.com/v1beta/models?key=YOUR_API_KEY"

# 正常な場合は JSON が返る
# エラーの場合は "Invalid API key" または "Permission denied"
```

**⚠️ API キー管理のベストプラクティス:**

```python
import os
from dotenv import load_dotenv

# .env ファイルに保存
# GOOGLE_API_KEY=your_actual_key_here

load_dotenv()
api_key = os.getenv("GOOGLE_API_KEY")

if not api_key:
    raise ValueError(
        "GOOGLE_API_KEY not found in environment. "
        "Create a .env file with GOOGLE_API_KEY=your_key"
    )

import google.generativeai as genai
genai.configure(api_key=api_key)
```

## 2. 「Model not available」エラー

### 問題：モデル名が古い、またはサポートされていない

Google は頻繁にモデルを更新・廃止します。古い名前を使うとエラーが出ます。

**廃止予定のモデル:**
- ❌ `gemini-1.5-pro-latest` （2026 年 1 月廃止）
- ❌ `gemini-pro` （非推奨）
- ❌ `gemini-pro-vision` （非推奨）

**現在推奨のモデル:**
- ✅ `gemini-2.5-pro` （最新の高性能）
- ✅ `gemini-2.5-flash` （高速・低コスト）
- ✅ `gemini-2.0-flash-exp` （画像生成対応）

### 解決方法：利用可能なモデル一覧を確認

```python
import google.generativeai as genai

def list_available_models():
    """現在利用可能なモデルを表示"""
    for model in genai.list_models():
        # 重要な情報を表示
        print(f"\nModel: {model.name}")
        print(f"  Display Name: {model.display_name}")
        print(f"  Input tokens: {model.input_token_limit}")
        print(f"  Output tokens: {model.output_token_limit}")
        print(f"  Supported methods: {model.supported_generation_methods}")

list_available_models()

# 期待される出力:
# Model: models/gemini-2.5-pro
#   Display Name: Gemini 2.5 Pro
#   Input tokens: 1000000
#   ...
```

**期待される出力:**
```
Model: models/gemini-2.5-pro
  Display Name: Gemini 2.5 Pro
  Input tokens: 1000000
  Output tokens: 4096
  Supported methods: ['generateContent', 'countTokens']

Model: models/gemini-2.5-flash
  Display Name: Gemini 2.5 Flash
  ...
```

### Google AI Studio のモデル選択

```
Google AI Studio（https://aistudio.google.com）で:

1. 左上の「＋」ボタン → 「新しいチャット」
2. モデルドロップダウンをクリック
3. 推奨モデルは：
   • 一般的な用途: gemini-2.5-pro
   • 高速応答重視: gemini-2.5-flash
   • 画像生成: gemini-2.0-flash-exp
```

## 3. プロンプトテストと API で異なる結果が出る

### 問題：Google AI Studio では動作するが、API では違う回答が来る

これは以下の 3 つが異なるためです：

1. **Temperature（創造性）の設定**
2. **System instruction（システムプロンプト）の有無**
3. **安全フィルター設定**

### 解決方法：パラメータを同期させる

```python
import google.generativeai as genai

def test_with_consistent_params(prompt):
    """Google AI Studio と同じパラメータで生成"""
    model = genai.GenerativeModel(
        model_name="gemini-2.5-pro",
        # ⚠️ これらの値を Google AI Studio と合わせる
        generation_config=genai.GenerationConfig(
            temperature=1.0,      # Google AI Studioのデフォルト
            top_p=0.95,
            top_k=40,
            max_output_tokens=8192,
        ),
        safety_settings=[
            genai.types.SafetySetting(
                category=genai.types.HarmCategory.HARM_CATEGORY_HARASSMENT,
                threshold=genai.types.HarmBlockThreshold.BLOCK_ONLY_HIGH,
            ),
            genai.types.SafetySetting(
                category=genai.types.HarmCategory.HARM_CATEGORY_HATE_SPEECH,
                threshold=genai.types.HarmBlockThreshold.BLOCK_ONLY_HIGH,
            ),
        ]
    )

    response = model.generate_content(prompt)
    return response.text

# 使用例
result = test_with_consistent_params(
    "日本の首都を説明してください。"
)
print(result)
```

**期待される出力:**
```
東京は日本の首都です。...
```

### パラメータの比較表

| パラメータ | Google AI Studio デフォルト | 説明 |
|----------|--------------------------|------|
| Temperature | 1.0 | 1.0 = 創造的、0 = 決定的 |
| Top P | 0.95 | 高いほど多様な回答 |
| Top K | 40 | 候補の数を制限 |
| Max tokens | 8192 | 最大出力文字数 |

## 4. ファイルアップロード失敗

### 問題：「File too large」または形式エラー

Google AI Studio でファイルをアップロードする際の制限：

- **無料プラン**: 最大 20 MB
- **有料プラン**: 最大 2 GB
- **サポート形式**: PDF、画像（PNG、JPG、WEBP）、テキスト、動画

### 解決方法：ファイルの分割とメタデータ確認

```python
import google.generativeai as genai
import os

def upload_file_safely(file_path, max_size_mb=20):
    """ファイルサイズと形式をチェックしてからアップロード"""

    # ファイルサイズチェック
    file_size_mb = os.path.getsize(file_path) / (1024 * 1024)
    if file_size_mb > max_size_mb:
        print(f"Error: {file_size_mb:.1f} MB exceeds limit of {max_size_mb} MB")
        print("Solutions:")
        print("1. Split the file into smaller parts")
        print("2. Compress the file")
        print("3. Upgrade to paid plan (2 GB limit)")
        return None

    # 形式チェック
    supported_extensions = {
        '.pdf': 'application/pdf',
        '.txt': 'text/plain',
        '.png': 'image/png',
        '.jpg': 'image/jpeg',
        '.jpeg': 'image/jpeg',
        '.webp': 'image/webp',
        '.mp4': 'video/mp4',
    }

    _, ext = os.path.splitext(file_path)
    if ext.lower() not in supported_extensions:
        print(f"Error: Format {ext} not supported")
        print(f"Supported: {list(supported_extensions.keys())}")
        return None

    # アップロード実行
    try:
        file = genai.upload_file(file_path)
        print(f"✓ Uploaded: {file.name}")
        print(f"  MIME type: {file.mime_type}")
        return file
    except Exception as e:
        print(f"Upload failed: {e}")
        return None

# 使用例
file = upload_file_safely("document.pdf")
if file:
    model = genai.GenerativeModel("gemini-2.5-pro")
    response = model.generate_content([
        "このPDFを要約してください",
        file
    ])
    print(response.text)
```

**期待される出力:**
```
✓ Uploaded: files/abc123def456
  MIME type: application/pdf
```

### ファイル分割スクリプト

```python
def split_large_file(file_path, chunk_size_mb=15):
    """大きなテキストファイルを分割"""
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()

    total_size_mb = len(content.encode('utf-8')) / (1024 * 1024)
    if total_size_mb <= chunk_size_mb:
        print("File is small enough, no split needed")
        return [file_path]

    chunk_size = chunk_size_mb * 1024 * 1024
    parts = []

    for i in range(0, len(content), chunk_size):
        part_content = content[i:i+chunk_size]
        part_filename = f"{file_path}.part{len(parts)+1}.txt"
        with open(part_filename, 'w', encoding='utf-8') as f:
            f.write(part_content)
        parts.append(part_filename)

    print(f"Split into {len(parts)} parts")
    return parts

# 使用例
parts = split_large_file("large_document.txt")
for part in parts:
    upload_file_safely(part)
```

## 5. ファインチューンされたモデルでエラー

### 問題：「Model not found for tuning」または訓練失敗

Gemini のファインチューニング（モデルカスタマイズ）は API 経由のみで、Google AI Studio からはできません。

### 解決方法：API で正しくファインチューニング

```python
import google.generativeai as genai
import time

def create_tuning_job(training_data_file, display_name):
    """モデルをファインチューニング"""
    base_model = "models/gemini-1.5-flash-001"

    operation = genai.create_tuned_model(
        source_model=base_model,
        training_data=training_data_file,
        id=f"custom-model-{int(time.time())}",
        display_name=display_name,
        epoch_count=3,
        batch_size=4,
        learning_rate=0.001,
    )

    # 訓練進行状況を監視
    result = operation.result()
    print(f"Tuning complete: {result.name}")
    return result.name

def use_tuned_model(tuned_model_name, prompt):
    """ファインチューンされたモデルで生成"""
    model = genai.GenerativeModel(model_name=tuned_model_name)
    response = model.generate_content(prompt)
    return response.text

# 訓練データの形式（JSON Lines）:
# {"text_input": "...", "output": "..."}
# {"text_input": "...", "output": "..."}

# 使用例
# tuned_name = create_tuning_job("training_data.jsonl", "Custom Q&A")
# result = use_tuned_model(tuned_name, "質問")
```

**訓練データの例:**
```json
{"text_input": "What is machine learning?", "output": "Machine learning is..."}
{"text_input": "How does AI work?", "output": "AI works by..."}
{"text_input": "What is deep learning?", "output": "Deep learning is..."}
```

## 6. 「Export to Code」が動作しない

### 問題：生成されたコードが実行できない

Google AI Studio の「Export to Code」機能で生成したコードが以下のエラーを出す：

- ❌ `ModuleNotFoundError: No module named 'google'`
- ❌ 古い API 構文を使っている
- ❌ 依存パッケージが記載されていない

### 解決方法：最新の SDK をインストール

```bash
# 最新の Python SDK をインストール
pip install google-generativeai --upgrade

# バージョン確認
python -c "import google.generativeai; print(google.generativeai.__version__)"
```

**期待される出力:**
```
0.7.0  # または最新版
```

### エクスポート済みコードのテンプレート

```python
import google.generativeai as genai

# Step 1: API キーを設定
genai.configure(api_key="YOUR_API_KEY")

# Step 2: モデルを初期化
model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    system_instruction="You are a helpful assistant."
)

# Step 3: コンテンツを生成
response = model.generate_content(
    [
        "Your prompt here",
        # ファイルを含める場合：

```

## FAQ

**Q. API キーをどこに保存すればいい？**

A. 絶対に `.env` ファイルに保存し、**リポジトリに push しない**でください。Git に追加しないよう `.gitignore` に `*.env` を記載します。本番環境では環境変数やシークレット管理サービスを使用してください。

**Q. オフラインで Google AI Studio は使えますか？**

A. いいえ。Google AI Studio は完全なクラウドサービスなので、常にインターネット接続が必要です。オフライン環境では[Claude](/articles/claude-ai)などの他の選択肢を検討してください。

**Q. ファイルをアップロードすると、どのくらい保持されますか？**

A. アップロードされたファイルは **2 日間**保持されます。それ以降は自動削除されます。長期保存が必要な場合は、Cloud Storage などの外部ストレージを使用してください。

**Q. 複数のチャットセッションを保存できますか？**

A. Google AI Studio のチャット履歴は自動的にブラウザに保存されます。左メニューから以前のセッションを再開できます。ただし、削除すると復元できないので注意してください。

## ベストプラクティス

1. **API キーは絶対に公開しない** — GitHub 上に誤って push していないか確認
2. **環境変数で API キーを管理** — コードには `os.getenv()` で参照
3. **モデル名は常に最新を確認** — 定期的に `list_models()` を実行
4. **ファイルは必要な期間だけ保持** — 2 日で自動削除されることを念頭に
5. **Temperature を明記する** — テスト結果を再現するため
