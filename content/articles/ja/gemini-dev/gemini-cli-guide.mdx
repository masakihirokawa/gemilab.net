## Gemini CLIでターミナルからAIパワーを活用

Gemini CLIは、ターミナルから直接Gemini AIの強力な機能にアクセスできる開発ツールです。コードレビュー、ドキュメント生成、ファイル分析、そしてシェルワークフローの自動化まで、開発プロセスを大幅に効率化できます。

## インストールと初期設定

### インストール

Gemini CLIは`npm`経由で簡単にインストールできます：

```bash
# グローバルインストール（推奨）
npm install -g @anthropic-ai/gemini-cli

# インストールの確認
gemini --version
```

### APIキーの設定

Gemini CLIを使用するには、Gemini APIキーが必要です：

```bash
# 環境変数で設定（推奨）
export GEMINI_API_KEY="your-api-key-here"

# または、設定ファイルで管理
gemini config set api-key "your-api-key-here"

# 設定の確認
gemini config show
```

より安全な方法として、`.env`ファイルに保存し、`.gitignore`に追加します：

```bash
# .env ファイルを作成
echo "GEMINI_API_KEY=your-api-key-here" > .env

# .gitignoreに追加（絶対に忘れずに）
echo ".env" >> .gitignore
```

## 基本コマンド

### テキスト生成

最も基本的な使用方法は、プロンプトを直接渡すことです：

```bash
# シンプルなテキスト生成
gemini prompt "Pythonでの非同期プログラミングについて説明してください"

# 複数行のプロンプト
gemini prompt "
次の関数を最適化してください：
def slow_calculation(n):
    result = 0
    for i in range(n):
        result += i
    return result
"
```

### ファイルの分析

```bash
# ファイルのコード分析
gemini analyze app.py

# エラーの診断と修正提案
gemini analyze error.log --analyze-errors

# 複数ファイルの一括分析
gemini analyze src/*.py --summary
```

### コード生成

```bash
# 特定の言語でコード生成
gemini generate "FastAPIを使用したRESTful APIエンドポイントの実装" --lang python

# 複数のファイルを生成
gemini generate "ユーザー認証システムの実装" --lang python --output-dir generated/

# テンプレートから生成
gemini generate-from-template --template node-express-api
```

## 実践的な活用シーン

### シーン1：コードレビューとリファクタリング

開発中のコードをGemini CLIで自動レビュー：

```bash
# コードレビュー機能
gemini review my-script.py

# 特定の観点でレビュー（パフォーマンス、セキュリティなど）
gemini review my-script.py --focus security

# 詳細なレポート生成
gemini review src/ --detailed --output review-report.md
```

使用例：

```bash
# 既存のスクリプトを最適化
$ gemini review data_processor.py --focus performance
Analyzing: data_processor.py

Performance Issues Found:
1. Line 45: Nested loop can be optimized with list comprehension
2. Line 78: Multiple list iterations - consolidate into single pass
3. Line 92: Regular expression compiled in loop - move to module level

Suggestions:
- Use numpy for bulk operations (estimated 10x speedup)
- Implement caching for repeated calculations
- Consider vectorization for data processing

Cost estimate: 0.8 tokens/execution → 80K tokens/month at current usage
```

### シーン2：ドキュメント生成

```bash
# 関数やモジュールのドキュメント自動生成
gemini doc generate src/utils.py

# READMEの自動生成
gemini doc create-readme --project-description "Python製のデータパイプライン" --features feature1 feature2

# APIドキュメント生成
gemini doc api-docs src/api/ --format openapi
```

実例：

```bash
# FastAPI アプリケーションのドキュメント自動生成
$ gemini doc api-docs app/routes/ --format openapi

- PUT /api/users/{id}
- DELETE /api/users/{id}

Documentation includes:
```

### シーン3：シェルワークフロー統合

Gemini CLIをシェルスクリプトに統合して、強力な自動化パイプラインを構築：

```bash
#!/bin/bash
# build-and-document.sh

echo "🔨 Building project..."
npm run build

echo "🔍 Running tests..."
npm test

echo "📝 Generating documentation..."
gemini doc generate-for-build --output docs/

echo "🤖 Analyzing code quality..."
gemini review src/ --focus quality --output reports/code-review.md

echo "✅ Build complete!"
```

実行：

```bash
chmod +x build-and-document.sh
./build-and-document.sh
```

### シーン4：ファイル処理とバッチ操作

```bash
# ディレクトリ内の全Pythonファイルを分析
for file in src/**/*.py; do
  echo "Analyzing $file..."
  gemini analyze "$file" --output "reports/$(basename $file .py).analysis"
done

# 複数のクエリを順次処理
gemini prompt "$(cat prompts/analyze-data.txt)" < input-file.txt

# パイプでテキストを渡す
cat messy-logs.txt | gemini prompt "このログから重大なエラーを抽出してください"
```

### ビルドスクリプト統合

```bash
#!/bin/bash
# ci-pipeline.sh

# テストを実行
npm test || exit 1

# コードの品質チェック
gemini review src/ --fail-on-critical --output quality-check.md

# カバレッジレポートの生成
gemini analyze coverage/coverage.json --format html --output coverage-report.html

# デプロイ前のセキュリティ確認
gemini security-scan src/ --strict

echo "✅ All checks passed. Ready for deployment."
```

### カスタムプロンプトとテンプレート

```bash
# プロンプトテンプレートを保存
mkdir -p ~/.gemini/prompts

# カスタムテンプレート作成
cat > ~/.gemini/prompts/code-review.txt << 'EOF'
次のコードをレビューしてください：

パフォーマンス観点での改善点：
セキュリティ上の懸念事項：
可読性と保守性の向上案：
EOF

# テンプレートを使用
gemini prompt "$(cat ~/.gemini/prompts/code-review.txt)" < mycode.py
```

## 実践的なワークフロー例

### 完全な開発フロー

```bash
#!/bin/bash
# dev-workflow.sh

PROJECT_NAME="my-project"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# 1. 新規機能の実装
echo "📝 Generating implementation outline..."
gemini prompt "以下の機能の実装計画を作成してください：ユーザー認証システム" \
  > docs/implementation-plan-$TIMESTAMP.md

# 2. コード生成
echo "🤖 Generating boilerplate code..."
gemini generate "Express.js + JWT認証の実装" --lang javascript --output src/auth/

# 3. テストコードの生成
echo "✅ Generating tests..."
gemini generate "前述の認証システムのユニットテスト" --lang javascript --output test/

# 4. コードレビュー
echo "👁️  Reviewing generated code..."
gemini review src/auth/ --focus security --output reports/auth-review-$TIMESTAMP.md

# 5. ドキュメント生成
echo "📖 Generating documentation..."
gemini doc generate src/auth/ --output docs/auth-api-$TIMESTAMP.md

# 6. 統計情報の収集
echo "📊 Generating statistics..."
gemini analyze src/auth/ --stats --output reports/auth-stats-$TIMESTAMP.json

echo "✨ Development workflow complete!"
echo "Report saved in: reports/"
```

実行：

```bash
chmod +x dev-workflow.sh
./dev-workflow.sh
```

## パフォーマンス最適化

### キャッシング戦略

```bash
# 頻繁に使用するプロンプトをキャッシュ
gemini config cache-prompts on

# キャッシュの確認
gemini cache status

# キャッシュクリア
gemini cache clear
```

### バッチ処理の最適化

```bash
# バッチ分析で複数ファイルを効率的に処理
gemini batch-analyze src/ \
  --pattern "*.js" \
  --output-dir reports/ \
  --parallel 4  # 4つの並列処理
```

## トラブルシューティング

### よくある問題と解決方法

```bash
# APIキーエラー
# 解決：APIキーの設定を確認
gemini config show | grep api-key

# ネットワーク接続エラー
# 解決：リトライロジック付きで実行
gemini prompt "..." --retry 3 --retry-delay 2000

# レート制限エラー
# 解決：並列数を減らす、待機時間を追加
gemini batch-analyze src/ --parallel 1 --delay-between-requests 1000

# トークン不足エラー
# 解決：より短いプロンプトを使用、入力サイズを制限
gemini analyze large-file.py --max-tokens 100000
```

## ベストプラクティス

1. **環境変数の活用**: APIキーは環境変数で管理し、決してコミットしない
2. **出力のリダイレクト**: 結果をファイルに保存して後で参照
3. **エラーハンドリング**: スクリプト内では `||` で失敗時の処理を定義
4. **段階的な統合**: 最初は小規模な タスクから、段階的に複雑な使用法へ
5. **キャッシング活用**: 同じプロンプトは再利用しない方針で設計
