## Gemini Code Assistとは？2026年のAIコーディングアシスタント

Gemini Code Assistは、Googleが提供するAI搭載コーディングアシスタントです。VS CodeやIntelliJ IDEAなどの主要IDEに統合され、コード補完、チャット、そして最新の**エージェントモード**によるマルチステップタスクの自動化まで幅広くサポートします。

2026年に入り、Gemini Code Assistは大幅なアップデートを受け、従来の単純なコード補完ツールから、**本格的なAIペアプログラマー**へと進化しました。特に注目すべき新機能は以下の3つです：

- **エージェントモード（Agent Mode）**：複数ファイルにわたる複雑なタスクを自律実行
- **Finish Changes**：記述途中のコードをAIが意図を読み取って完成
- **Outlines**：コードの概要をエディタ内に直接表示

本記事では、これらの最新機能を中心に、Gemini Code Assistの導入から実践的な活用法まで詳しく解説します。

## インストールと初期設定

### VS Codeへのインストール

VS Codeへのインストールは拡張機能マーケットプレイスから行います：

1. VS Codeを開き、拡張機能パネル（`Ctrl+Shift+X` / `Cmd+Shift+X`）を開く
2. 「Gemini Code Assist」を検索してインストール
3. インストール後、Googleアカウントでサインイン

または、コマンドラインからも可能です：

```bash
# VS Code CLIでインストール
code --install-extension google.geminicodeassist

# インストールされた拡張機能を確認
code --list-extensions | grep gemini
```

### IntelliJ IDEAへのインストール

IntelliJ系IDEへは以下の手順でインストールします：

1. `File > Settings > Plugins`を開く（macOSは`IntelliJ IDEA > Preferences > Plugins`）
2. Marketplace タブで「Gemini Code Assist」を検索
3. `Install`をクリックし、IDEを再起動
4. ステータスバーのGeminiアイコンからGoogleアカウントでサインイン

### エディションの選択

Gemini Code Assistには3つのエディションがあります：

| エディション | 対象 | 主な特徴 |
|------------|------|---------|
| **無料版** | 個人開発者 | 基本的なコード補完・チャット |
| **Standard** | チーム・企業 | エンタープライズセキュリティ、高度な補完 |
| **Enterprise** | 大企業 | プライベートコードカスタマイズ、Cloud統合 |

## エージェントモードの使い方

### エージェントモードとは

エージェントモードは、Gemini Code Assistの最も革新的な機能です。単純な質問応答を超え、**複数ファイルの編集、プロジェクト全体のリファクタリング、テストの自動生成**など、複雑なマルチステップタスクを自律的に実行できます。

### エージェントモードの起動

VS Codeでの起動方法：

```
# チャットパネルを開く（Ctrl+Shift+P または Cmd+Shift+P）
> Gemini: Open Chat

# チャット入力欄の「Agent」トグルをONにする
# または、/agent コマンドを使用
/agent このプロジェクトに認証機能を追加してください
```

### エージェントモードの実践例

例えば、Expressアプリに認証機能を追加したい場合：

```javascript
// エージェントへの指示例
// 「このExpressアプリにJWT認証を追加してください。
//  - POST /auth/login エンドポイントを作成
//  - ミドルウェアで保護されたルートを実装
//  - エラーハンドリングも含めてください」

// Gemini Code Assistが自動生成するコード例（auth/middleware.js）
const jwt = require('jsonwebtoken');

/**
 * JWT認証ミドルウェア
 * リクエストヘッダーのAuthorizationトークンを検証する
 */
const authenticateToken = (req, res, next) => {
  const authHeader = req.headers['authorization'];
  // "Bearer TOKEN" 形式からトークンを抽出
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: '認証トークンが必要です' });
  }

  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) {
      // トークンの有効期限切れや改ざんを検出
      return res.status(403).json({ error: '無効なトークンです' });
    }
    req.user = user;
    next();
  });
};

module.exports = { authenticateToken };

// 期待する動作:
// - 有効なトークン → next()を呼び出して次のミドルウェアへ
// - トークンなし → 401 Unauthorized
// - 無効なトークン → 403 Forbidden
```

エージェントは変更を加える前に**実行計画を提示**します。計画を確認し、承認・修正・拒否を選択できます。これにより、AIに完全な自律性を与えつつも、開発者がコントロールを保持できます。

### Auto Approveモード

頻繁な確認が不要な場合は、Auto Approveモードを有効にできます：

```
# VS Codeの設定（settings.json）
{
  "geminicodeassist.agent.autoApprove": true,
  "geminicodeassist.agent.autoApproveScope": "project"
}
```

**注意**: Auto Approveは信頼できるプロジェクトのみで使用し、変更後は必ずdiff確認とロールバックの準備をしておきましょう。

## Finish Changes — AIペアプログラマー

### Finish Changesとは

**Finish Changes**は、Gemini Code Assistの全く新しいアプローチのコード補完機能です。従来の補完ツールと異なり、**プロンプトを書かずに**、あなたの編集途中のコードからAIが意図を読み取り、残りの変更を自動完成させます。

Gemini 3.0を搭載したこの機能は、以下のワークフローをサポートします：

### パターン適用（Applying Patterns）

同じパターンの変更を1回示すだけで、ファイル全体に適用します：

```python
# Before: クラスメソッドにロギングを追加したい
class UserService:
    def get_user(self, user_id: int):
        return self.db.query(User).filter_by(id=user_id).first()

    def create_user(self, name: str, email: str):
        user = User(name=name, email=email)
        self.db.add(user)
        return user

# 1つ目のメソッドにロギングを追加（Finish Changesがパターンを学習）
class UserService:
    def get_user(self, user_id: int):
        logger.info(f"Getting user: {user_id}")  # ← ここまで手動入力
        # Finish Changesが残りを自動補完 ↓
        return self.db.query(User).filter_by(id=user_id).first()

    def create_user(self, name: str, email: str):
        logger.info(f"Creating user: {name}")  # ← 自動追加
        user = User(name=name, email=email)
        self.db.add(user)
        return user
```

### TODOコメントによる指示

コード内に直接指示を書くことも可能です：

```typescript
interface UserProfile {
  id: string;
  name: string;
  email: string;
  // TODO: バリデーション関数を追加してemailの形式を検証する
}

// Finish Changesが自動生成:
function validateEmail(email: string): boolean {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return emailRegex.test(email);
  // 期待する出力: "test@example.com" → true, "invalid-email" → false
}
```

## Outlines — コード理解を加速

### Outlinesとは

**Outlines**は、コードの概要をエディタ内に直接表示する機能です。サイドパネルでの説明ではなく、コードと並んで高レベルの英語サマリーが表示されるため、慣れていないコードベースでも素早く理解できます。

### Outlinesの活用例

大規模なReactコンポーネントを理解する場合：

```javascript
// Gemini Code Assistが自動生成するOutline（グレーで表示）
// ▸ Component: ProductCatalog
// ▸ Purpose: Displays paginated product list with filtering and sorting

const ProductCatalog = ({ initialFilters }) => {
  // ▸ State: products array, loading status, current page, active filters
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);
  const [currentPage, setCurrentPage] = useState(1);
  const [filters, setFilters] = useState(initialFilters || {});

  // ▸ Effect: Fetches products when page or filters change
  useEffect(() => {
    fetchProducts(currentPage, filters);
  }, [currentPage, filters]);

  // ▸ Handler: Updates filter state and resets pagination to page 1
  const handleFilterChange = (newFilter) => {
    setFilters(prev => ({ ...prev, ...newFilter }));
    setCurrentPage(1);
  };

  // ▸ Renders: Loading spinner OR product grid with pagination controls
  return (
    // ...
  );
};
```

新しいプロジェクトへの参画時や、他の開発者が書いたコードをレビューする際に特に効果的です。

## Context Drawer — コンテキスト管理

### 精密なコンテキスト制御

**Context Drawer**を使うと、Geminiが参照するファイルとフォルダを精密に制御できます：

```
# Context Drawerの開き方
# チャットパネル右上の「Context」アイコンをクリック

# よく使う操作:
# - 特定ファイルを追加: ファイルをドラッグ&ドロップ
# - フォルダを除外: src/legacy/ フォルダをコンテキストから削除
# - 一時的な追加: 1会話のみ有効なコンテキストを設定
```

大規模なモノレポで作業する際、関係ないパッケージをコンテキストから除外することで、より精度の高い回答が得られます。

## カスタムスラッシュコマンド

繰り返し行うタスクを自動化するカスタムコマンドを定義できます：

```json
// .gemini/commands.json
{
  "commands": [
    {
      "name": "test",
      "description": "現在のファイルのユニットテストを生成",
      "prompt": "このファイルのユニットテストをJestで作成してください。境界値テストとエラーケースも含めてください。"
    },
    {
      "name": "review",
      "description": "コードレビューを実行",
      "prompt": "このコードをレビューして、パフォーマンス、セキュリティ、可読性の観点から改善点を指摘してください。"
    },
    {
      "name": "doc",
      "description": "JSDocコメントを生成",
      "prompt": "このファイルの全関数とクラスにJSDocコメントを追加してください。パラメータ、戻り値、使用例を含めてください。"
    }
  ]
}
```

定義後は`/test`、`/review`、`/doc`でそれぞれのコマンドを実行できます。

## MCP統合でエコシステムを拡張

Gemini Code AssistはMCP（Model Context Protocol）をサポートしており、外部ツールとの統合が可能です：

```json
// .gemini/mcp.json
{
  "servers": {
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${env:GITHUB_TOKEN}"
      }
    },
    "database": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_CONNECTION_STRING": "${env:DATABASE_URL}"
      }
    }
  }
}
```

MCPを使うと、GitHubのIssue情報やデータベーススキーマを参照しながらコードを生成できます。詳しくは[Gemini MCPインテグレーションガイド](/articles/gemini-dev/gemini-mcp-integration)をご覧ください。

## よくある質問（FAQ）

### Q1. Gemini Code AssistとGemini CLIはどう違いますか？

Gemini Code AssistはVS CodeやIntelliJなどのIDEに統合されたツールで、GUI環境でのコーディングを支援します。一方、[Gemini CLI](/articles/gemini-dev/gemini-cli-guide)はターミナルから使用するコマンドラインツールです。IDEでの開発にはCode Assist、CI/CDパイプラインやスクリプト自動化にはCLIが適しています。2026年現在、両ツールはGemini 2.5を搭載しており、用途に合わせて使い分けましょう。

### Q2. エージェントモードで意図しない変更が行われる心配はありませんか？

エージェントモードはデフォルトで変更前に実行計画を提示します。計画を確認し、承認・修正・拒否を選択できるため、意図しない変更のリスクは低いです。より安全に使うには、変更前後でgit diffを確認し、コミット前にテストを実行する習慣をつけることをお勧めします。Auto Approveモードは、十分に信頼できるプロジェクトにのみ使用してください。

### Q3. 無料版と有料版の主な違いは何ですか？

無料版でも基本的なコード補完、チャット、エージェントモードの基本機能が使用できます。Standard版以上では、エンタープライズセキュリティ（VPC Service Controls対応）、より大きなコンテキストウィンドウ、チームでの利用管理機能が追加されます。Enterprise版ではさらに、自社のプライベートコードリポジトリを使ったモデルカスタマイズが可能です。個人開発者には無料版で十分な場合が多いでしょう。

### Q4. Finish Changesを使うための特別な設定が必要ですか？

特別な設定は不要です。Gemini Code Assistをインストールしてサインインすれば、自動的にFinish Changes機能が有効になります。VS CodeとIntelliJ系IDEの両方でサポートされています。コード編集中にGeminiアイコンが表示されたら、クリックするか`Tab`キーで候補を受け入れられます。
