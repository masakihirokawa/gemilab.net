## はじめに

CI/CD（継続的インテグレーション/継続的デリバリー）パイプラインは、現代のソフトウェア開発において欠かせない仕組みです。GitHub Actionsはその中でも最も広く使われているCI/CDプラットフォームの一つですが、ここにGemini APIを組み込むことで、コードレビューの自動化、PR要約の生成、テストケースの提案など、開発ワークフローを大幅に効率化できます。

この記事では、Gemini APIとGitHub Actionsを連携させる具体的な方法を、実際に動作するワークフロー定義とPythonスクリプトを交えて解説します。対象読者は、GitHub Actionsの基本を理解しており、AIを開発プロセスに取り入れたいと考えているエンジニアの方々です。

## GitHub ActionsでGemini APIを使う準備

### 前提条件

まず、以下の準備が必要です。

1. **Google AI Studioのアカウント**と**APIキー**（[Google AI Studio](https://aistudio.google.com/)で取得）
2. **GitHubリポジトリ**のAdmin権限（Secretsを設定するため）
3. **Python 3.11以上**の基本知識

### APIキーをGitHub Secretsに登録

Gemini APIキーをワークフローで安全に使うため、リポジトリのSecretsに登録します。

1. GitHubリポジトリの **Settings → Secrets and variables → Actions** を開く
2. **New repository secret** をクリック
3. Name: `GEMINI_API_KEY`、Value: APIキーを入力して保存

これにより、ワークフロー内で `${{ secrets.GEMINI_API_KEY }}` としてAPIキーを参照できます。コード内にAPIキーを直接書くことは絶対に避けてください。

## プルリクエストの自動レビュー

最も実用的なユースケースの一つが、PRの差分をGemini APIに送信して自動レビューコメントを生成する仕組みです。

### ワークフロー定義

```yaml
# .github/workflows/gemini-review.yml
name: Gemini Code Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai requests

      - name: Get PR diff
        id: diff
        run: |
          git diff origin/${{ github.base_ref }}...HEAD > pr_diff.txt
          echo "diff_size=$(wc -c < pr_diff.txt)" >> $GITHUB_OUTPUT

      - name: Run Gemini Review
        if: steps.diff.outputs.diff_size > 0
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
        run: python scripts/gemini_review.py
```

### レビュースクリプト

```python
# scripts/gemini_review.py
import os
from google import genai

def main():
    # Gemini APIクライアントの初期化
    client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

    # PR差分の読み込み
    with open("pr_diff.txt", "r") as f:
        diff = f.read()

    # 差分が大きすぎる場合はトリミング
    max_chars = 30000
    if len(diff) > max_chars:
        diff = diff[:max_chars] + "\n... (差分が長いため省略)"

    # Gemini APIにレビューを依頼
    prompt = f"""あなたは経験豊富なソフトウェアエンジニアです。
以下のコード差分をレビューし、改善点を指摘してください。

## レビュー観点
- バグやロジックの問題
- セキュリティ上の懸念
- パフォーマンスの改善余地
- コードの可読性

## 出力形式
各指摘を以下の形式で記述してください:
- **[重要度: 高/中/低]** ファイル名:行数付近 - 指摘内容

---
{diff}
"""

    response = client.models.generate_content(
        model="gemini-3-flash",
        contents=prompt,
    )

    review_comment = response.text
    print("Review generated successfully")
    print(review_comment)

    # GitHub APIでPRにコメントを投稿
    import requests
    repo = os.environ.get("GITHUB_REPOSITORY", "")
    pr_number = os.environ.get("PR_NUMBER", "")
    token = os.environ.get("GITHUB_TOKEN", "")

    if repo and pr_number and token:
        url = f"https://api.github.com/repos/{repo}/issues/{pr_number}/comments"
        headers = {
            "Authorization": f"token {token}",
            "Accept": "application/vnd.github.v3+json",
        }
        body = f"## Gemini AI Code Review\n\n{review_comment}"
        requests.post(url, json={"body": body}, headers=headers)
        # 期待する出力: PRにAIレビューコメントが投稿される

if __name__ == "__main__":
    main()
```

## PR要約の自動生成

大きなPRの内容を簡潔に要約するワークフローも非常に便利です。レビュアーがPRの全体像をすばやく把握でき、レビュー効率が向上します。

### ワークフロー定義

```yaml
# .github/workflows/gemini-summarize.yml
name: Gemini PR Summary

on:
  pull_request:
    types: [opened]

permissions:
  pull-requests: write

jobs:
  summarize:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai requests

      - name: Generate Summary
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
        run: |
          git diff origin/${{ github.base_ref }}...HEAD > pr_diff.txt
          python scripts/gemini_summarize.py
```

### 要約スクリプト

```python
# scripts/gemini_summarize.py
import os
import requests
from google import genai

def main():
    client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

    with open("pr_diff.txt", "r") as f:
        diff = f.read()[:30000]

    prompt = f"""以下のコード差分を分析し、PRの要約を生成してください。

## 出力形式
### 変更の概要
（1〜2文で全体像を説明）

### 主な変更点
- 変更点1
- 変更点2
- 変更点3

### 影響範囲
（この変更が影響するコンポーネントや機能）

### レビュー時の注意点
（レビュアーが特に注意すべきポイント）

---
{diff}
"""

    response = client.models.generate_content(
        model="gemini-3-flash",
        contents=prompt,
    )

    summary = response.text

    # PRの説明を更新
    repo = os.environ.get("GITHUB_REPOSITORY", "")
    pr_number = os.environ.get("PR_NUMBER", "")
    token = os.environ.get("GITHUB_TOKEN", "")

    if repo and pr_number and token:
        url = f"https://api.github.com/repos/{repo}/issues/{pr_number}/comments"
        headers = {
            "Authorization": f"token {token}",
            "Accept": "application/vnd.github.v3+json",
        }
        body = f"## AI-Generated PR Summary\n\n{summary}"
        requests.post(url, json={"body": body}, headers=headers)
        print("PR summary posted successfully")
        # 期待する出力: PRにAI要約コメントが投稿される

if __name__ == "__main__":
    main()
```

## コミットメッセージの品質チェック

コミットメッセージの品質を一定に保つために、Gemini APIでコミットメッセージを評価するワークフローも構築できます。

```yaml
# .github/workflows/gemini-commit-check.yml
name: Commit Message Quality Check

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  check-commits:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai

      - name: Check commit messages
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
        run: |
          # コミットメッセージの取得
          git log origin/${{ github.base_ref }}..HEAD \
            --pretty=format:"%s" > commits.txt

          python -c "
          import os
          from google import genai

          client = genai.Client(api_key=os.environ['GEMINI_API_KEY'])

          with open('commits.txt') as f:
              commits = f.read()

          prompt = f'''以下のコミットメッセージを評価してください。
          Conventional Commitsの形式（feat:, fix:, docs: 等）に
          沿っているか、内容が明確かを判定してください。

          コミットメッセージ:
          {commits}

          各メッセージに対して OK / 改善推奨 を判定し、
          改善推奨の場合は修正案を提示してください。'''

          response = client.models.generate_content(
              model='gemini-3-flash',
              contents=prompt,
          )
          print(response.text)
          # 期待する出力:
          # feat: add user auth → OK
          # fix stuff → 改善推奨: 'fix: resolve login validation error'
          "
```

## テストケースの自動提案

新しいコードが追加されたとき、Gemini APIを使って不足しているテストケースを提案させることもできます。

```python
# scripts/gemini_test_suggest.py
import os
from google import genai

def suggest_tests():
    client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

    with open("pr_diff.txt", "r") as f:
        diff = f.read()[:30000]

    prompt = f"""以下のコード差分を分析し、追加すべきテストケースを提案してください。

## 要件
- 新しく追加・変更された関数やメソッドに対するユニットテスト
- エッジケース（null値、空配列、境界値など）のテスト
- エラーハンドリングのテスト
- テストコードはpytestまたはJest形式で記述

## 出力形式
各テストケースを以下の形式で出力:
1. テスト対象の関数/メソッド名
2. テストの目的
3. テストコード（コピー&ペーストで使える形式）

---
{diff}
"""

    response = client.models.generate_content(
        model="gemini-3-flash",
        contents=prompt,
    )
    print(response.text)
    # 期待する出力:
    # 1. test_calculate_total - 合計計算の正常系テスト
    #    def test_calculate_total():
    #        assert calculate_total([100, 200, 300]) == 600

if __name__ == "__main__":
    suggest_tests()
```

## 実践的なTips：トークン制限とコスト管理

CI/CDパイプラインでGemini APIを使う際は、コストとレート制限を意識することが重要です。

### 差分サイズの制御

```python
import tiktoken

def trim_diff(diff: str, max_tokens: int = 8000) -> str:
    """差分をトークン数で制限する"""
    if len(diff) > max_tokens * 4:  # 概算: 1トークン ≈ 4文字
        # ファイルごとに分割して重要なものを優先
        files = diff.split("diff --git")
        priority_files = []
        other_files = []

        for f in files:
            if any(ext in f for ext in [".py", ".ts", ".js", ".go"]):
                priority_files.append(f)
            else:
                other_files.append(f)

        # ソースコードファイルを優先的に含める
        result = "diff --git".join(priority_files)
        if len(result) < max_tokens * 4:
            remaining = max_tokens * 4 - len(result)
            result += "diff --git".join(other_files)[:remaining]

        return result[:max_tokens * 4]
    return diff
```

### APIコールの最適化

ワークフローの実行回数を最小限に抑えるための工夫も大切です。

- **パスフィルター**を使い、ドキュメントのみの変更ではレビューをスキップ
- **差分サイズが小さい場合**（10行未満など）はAIレビューを省略
- **キャッシュ**を活用し、同じ差分に対する再実行を防止

```yaml
on:
  pull_request:
    paths:
      - "src/**"
      - "lib/**"
      - "!**/*.md"
      - "!**/*.txt"
```

Gemini APIのコスト管理について詳しくは、[Gemini API コスト最適化ガイド](/articles/gemini-api/gemini-api-cost-optimization)もご覧ください。

## セキュリティ上の注意点

CI/CDパイプラインにAIを組み込む際は、セキュリティ面での配慮が欠かせません。

1. **APIキーの管理**: 必ずGitHub Secretsを使用し、コードにハードコードしない
2. **差分の内容**: シークレットや個人情報がdiffに含まれないよう`.gitignore`を適切に設定
3. **外部PRへの対応**: Fork元からのPRではSecretsが利用できないため、`pull_request_target`の使用には十分注意する
4. **レート制限**: APIの呼び出しが集中しないよう、`concurrency`設定でワークフローの同時実行を制限する

```yaml
concurrency:
  group: gemini-review-${{ github.event.pull_request.number }}
  cancel-in-progress: true
```

GitHub Actionsのセキュリティベストプラクティスについては、[Gemini CLI × GitHub Actions連携](/articles/gemini-dev/gemini-cli-terminal-ai-development-guide)の記事も参考になります。

## よくある質問（FAQ）

### Q: Gemini APIの無料枠でCI/CDに使えますか？

A: はい、Gemini 3 Flashには無料枠があり、個人プロジェクトや小規模チームであれば無料枠内で十分に活用できます。ただし、レート制限（1分あたりのリクエスト数）があるため、大規模リポジトリでPRが頻繁に作成される場合は有料プランを検討してください。

### Q: GitHub Actions以外のCI/CDツール（GitLab CI、CircleCIなど）でも使えますか？

A: はい、Gemini APIはHTTPベースのREST APIなので、どのCI/CDプラットフォームでも利用できます。この記事のPythonスクリプト部分はそのまま流用でき、ワークフロー定義の部分だけ各プラットフォームの形式に書き換えれば動作します。

### Q: 差分が非常に大きい場合（数千行の変更）はどうすればいいですか？

A: 差分が大きい場合は、ファイル単位で分割してAPIに送信する方法が効果的です。また、Gemini 3 Proの100万トークンのコンテキストウィンドウを活用すれば、大きな差分でも一度に処理できます。ただし、コストが増加するため、重要なファイル（ソースコード）を優先し、設定ファイルやロックファイルは除外するなどの最適化が推奨されます。

## まとめ

Gemini APIとGitHub Actionsの連携により、コードレビューの自動化、PR要約の生成、コミットメッセージのチェック、テストケースの提案など、開発ワークフロー全体をAIで強化できます。

最初は一つのワークフロー（たとえばPR要約）から始めて、チームの反応を見ながら段階的に拡張していくのがおすすめです。Gemini 3 Flashの高速なレスポンスとコストパフォーマンスの良さは、CI/CDのようなバッチ処理に最適です。
