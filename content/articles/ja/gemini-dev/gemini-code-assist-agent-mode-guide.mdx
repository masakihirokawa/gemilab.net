## Gemini Code Assist Agent Mode とは

Google の **Gemini Code Assist** は、VS Code や IntelliJ などの IDE 上で動作するAIコーディングアシスタントです。その中でも **Agent Mode** は、単なるコード補完を超えた「AIペアプログラマー」として機能する最も強力なモードです。

通常のコード補完が1行や1関数の提案に留まるのに対し、Agent Mode は**自律的なマルチステップ推論**を行い、複数のファイルやフレームワークにまたがる複雑なタスクを一気に解決します。設計ドキュメントからコードを生成したり、TODO コメントから実装を自動化したり、MCP サーバーを使って外部ツールと連携したりと、開発ワークフロー全体をAIがサポートします。

この記事では、Gemini Code Assist Agent Mode のセットアップから実践的な活用法まで、詳しく解説します。

## Agent Mode の有効化とセットアップ

### VS Code での設定

Agent Mode は VS Code と IntelliJ の両方で利用できます。ここでは VS Code での手順を紹介します。

```bash
# 前提条件:
# 1. VS Code がインストール済み
# 2. Gemini Code Assist 拡張機能がインストール済み
# 3. Google アカウントでサインイン済み

# セットアップ手順:
# 1. VS Code のアクティビティバーで Gemini Code Assist アイコンをクリック
# 2. チャットパネルが開く
# 3. 「Agent」トグルをクリックして Agent Mode に切り替え
# 4. プロンプトを入力して Enter

# Agent Mode が有効な場合の表示:
# [Agent Mode: ON] と表示され、チャットの背景色が変わる
```

### IntelliJ での設定

IntelliJ 系の IDE（IntelliJ IDEA、WebStorm、PyCharm 等）でも Agent Mode が利用可能です。Gemini Code Assist プラグインをインストールし、チャットパネルで Agent タブに切り替えるだけです。

## Agent Mode の核心機能

### 1. マルチステップ推論

Agent Mode の最大の強みは、複雑なタスクを複数のステップに分解して自律的に実行する能力です。

```python
# Agent Mode へのプロンプト例:
# 「FastAPI でユーザー認証APIを作って。
#  JWT トークン認証、パスワードハッシュ、
#  リフレッシュトークン対応で」

# → Agent Mode の実行フロー:
# Step 1: プロジェクト構造の分析
# Step 2: 依存パッケージの確認（fastapi, python-jose, passlib）
# Step 3: モデル定義（User, Token, TokenData）
# Step 4: セキュリティユーティリティ（ハッシュ、JWT生成/検証）
# Step 5: APIエンドポイント実装（register, login, refresh, me）
# Step 6: ミドルウェア設定

# 生成されるファイル構造:
# app/
# ├── main.py           ← FastAPI アプリケーション
# ├── models.py         ← Pydantic モデル
# ├── auth.py           ← JWT / ハッシュ ユーティリティ
# ├── routes/
# │   └── auth.py       ← 認証エンドポイント
# └── requirements.txt  ← 依存パッケージ

# auth.py の生成例:
from datetime import datetime, timedelta
from jose import JWTError, jwt
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

SECRET_KEY = "your-secret-key"  # 環境変数から取得推奨
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

# 期待される出力:
# → 5つのファイルが自動生成
# → 各ファイル間の import が正しく設定済み
# → requirements.txt に必要なパッケージが記載済み
```

### 2. コンテキスト認識

Agent Mode はプロジェクト全体のコンテキストを理解します。既存のコードベース、フレームワーク、設定ファイルを読み取り、それに一致するコードを生成します。

```bash
# コンテキスト認識の例:
# プロジェクトに Next.js + TypeScript + Prisma が存在する場合
# → Agent Mode は自動的にこれらの技術スタックに合わせたコードを生成

# プロンプト: 「ブログ記事のCRUD APIを追加して」
# → Prisma スキーマの更新
# → Next.js API Routes（App Router形式）の生成
# → TypeScript の型定義
# → すべて既存のプロジェクト構造に適合
```

### 3. MCP サーバー連携

Agent Mode は **MCP（Model Context Protocol）** サーバーを通じて外部ツールと連携できます。これにより、データベースの操作、APIの呼び出し、ファイルシステムへのアクセスなど、IDE の外にある情報源にもアクセス可能です。

```json
// .vscode/settings.json での MCP サーバー設定例
{
  "gemini.mcp.servers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${env:GITHUB_TOKEN}"
      }
    },
    "database": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "${env:DATABASE_URL}"
      }
    }
  }
}

// MCP連携後のプロンプト例:
// 「GitHub Issue #42 の内容を読んで、その修正を実装して」
// → Agent が GitHub API 経由で Issue を取得
// → 問題の内容を理解
// → 修正コードを自動生成
```

### 4. 計画のレビューと承認

Agent Mode では、AIが提示する実行計画を開発者がレビューし、承認・修正できます。

```bash
# Agent Mode の計画レビューフロー:
# 1. プロンプトを入力
# 2. Agent が実行計画を提示
#    ┌─────────────────────────────────┐
#    │ 計画:                            │
#    │ 1. prisma/schema.prisma を更新   │
#    │ 2. prisma migrate dev を実行     │
#    │ 3. src/routes/blog.ts を作成     │
#    │ 4. src/types/blog.ts を作成      │
#    │ 5. テストを実行                   │
#    └─────────────────────────────────┘
# 3. 開発者が計画をレビュー
#    → 承認: そのまま実行
#    → 修正: コメントで「Step 2はスキップして」等の指示
#    → 拒否: 計画を再生成
```

## 通常モードと Agent Mode の使い分け

| 機能 | 通常モード | Agent Mode |
|------|-----------|------------|
| コード補完 | 1行〜数行 | 複数ファイル |
| 推論ステップ | 単一 | マルチステップ |
| ファイル操作 | 閲覧のみ | 生成・編集 |
| MCP連携 | なし | あり |
| 計画レビュー | なし | あり |
| ターミナル操作 | なし | あり |
| 適したタスク | 小規模な補完 | 複雑な実装 |

**使い分けのポイント**: 1ファイル内の小さな変更は通常モードの方が速いです。複数ファイルにまたがる機能追加やリファクタリングでは Agent Mode が威力を発揮します。

## よくある質問（FAQ）

### Q: Agent Mode は無料で使えますか？

Gemini Code Assist の Individual プラン（無料）でも Agent Mode は利用できます。ただし、1日あたりのリクエスト数や使用可能な機能に制限があります。Enterprise プランではフル機能が利用可能です。

### Q: Agent Mode がファイルを勝手に変更してしまうことはありますか？

Agent Mode は変更を実行する前に計画を提示し、開発者の承認を求めます。ただし、IDE外のリソース（データベースの操作など）に対する変更は取り消し（Undo）ができない場合があるため、注意が必要です。重要な操作を行う際は計画を慎重にレビューしてください。

### Q: どのプログラミング言語に対応していますか？

Agent Mode は Python、JavaScript/TypeScript、Java、Go、Kotlin、C++、Rust など、主要なプログラミング言語に対応しています。特に Python と TypeScript のサポートが最も充実しています。

### Q: Gemini Code Assist Agent Mode と GitHub Copilot の違いは？

最大の違いは MCP 連携とマルチステップ推論です。GitHub Copilot は主にインラインのコード補完に強みがありますが、Gemini Code Assist Agent Mode は複数のファイルを横断する複雑なタスクの自律実行に優れています。また、Google Cloud のサービスとの統合が密接です。

## まとめ — Gemini Code Assist Agent Mode で開発を加速する

Gemini Code Assist Agent Mode は、AIコーディングアシスタントの次の段階を示しています。単なるコード補完ではなく、**マルチステップ推論、MCP連携、計画レビュー**を通じて、複雑な開発タスクをAIと協働で進められます。

特に、複数ファイルにまたがる機能追加、設計ドキュメントからの自動実装、外部ツールとの連携が必要な場面で大きな効果を発揮します。VS Code または IntelliJ で今すぐ試してみてください。
