## Gemini AppFunctions とは？ — アプリ内操作を AI に委任する新しい仕組み

2026年2月、Google は Android 向けの新しいフレームワーク「**AppFunctions**」を発表しました。これは、Gemini が画面のタップ操作やスクリーンリーダーを使わずに、**構造化された API を通じてアプリ内の機能を直接実行できる**仕組みです。

従来の AI アシスタントは、ユーザーの代わりに画面を操作する「UI オートメーション」方式が主流でした。しかし AppFunctions では、アプリ開発者が**自己記述型の関数（self-describing functions）**を定義し、AI エージェントがそれを発見・実行できるようになります。これは、クラウド側の [MCP（Model Context Protocol）](/articles/gemini-dev/gemini-mcp-integration) に相当するオンデバイスソリューションと言えます。

### なぜ AppFunctions が重要なのか

AppFunctions が従来のアプローチと大きく異なる点は以下の通りです。

- **構造化された API**: UI操作ではなく、明示的に定義された関数を通じてアプリの機能にアクセスする
- **オンデバイス実行**: クラウドサーバーを介さず、デバイス上でローカルに実行される
- **自動発見**: Gemini がアプリの提供する機能を自動的に検出し、適切なタイミングで呼び出す
- **大規模展開**: 2026年末までに2億台以上のデバイスで利用可能になる見込み

すでに **Uber**、**DoorDash**、**OpenTable** などの大手サービスが AppFunctions を通じた Gemini 連携を実装しており、「レストランを予約して」「タクシーを呼んで」といった自然言語の指示だけで、アプリ横断の操作が可能になっています。

## AppFunctions の仕組みを理解する

### アーキテクチャ概要

AppFunctions のアーキテクチャは、次の3つのレイヤーで構成されています。

1. **関数定義レイヤー**: 開発者が `AppFunctions` Jetpack ライブラリを使って、公開する機能を定義
2. **ディスカバリーレイヤー**: Gemini がデバイス上のアプリから利用可能な関数を検索
3. **実行レイヤー**: ユーザーの許可を得た上で、Gemini が関数を呼び出し結果を取得

```kotlin
// AppFunctions の基本的な動作フロー（概念コード）
// 1. アプリが関数を登録
// 2. Gemini が「近くのレストランを探して」と指示を受ける
// 3. Gemini がレストラン検索機能を持つアプリを発見
// 4. ユーザーの許可後、関数を呼び出して結果を返す

// ユーザー: 「近くのイタリアンレストランを予約して」
// → Gemini: OpenTable の searchRestaurants() を発見
// → Gemini: 実行許可をユーザーに確認
// → Gemini: searchRestaurants(cuisine="italian", location="nearby") を実行
// → Gemini: 結果をユーザーに表示し、予約を提案
```

### Function Calling との違い

Gemini API の [Function Calling](/articles/gemini-advanced/function-calling-guide) は、クラウド側で動作するサーバーサイドの仕組みです。一方、AppFunctions は**デバイス上で動作するクライアントサイドの仕組み**であり、以下の点で異なります。

| 項目 | Function Calling | AppFunctions |
|------|-----------------|--------------|
| 実行場所 | クラウド（サーバーサイド） | デバイス上（オンデバイス） |
| 対象 | Web API / バックエンド | Android アプリの機能 |
| 定義方法 | JSON Schema | Jetpack ライブラリ |
| ユーザー認証 | API キーベース | Android のパーミッション |
| レイテンシ | ネットワーク依存 | ローカル実行で高速 |

## AppFunctions を実装する — Kotlin によるステップバイステップガイド

### 前提条件

AppFunctions を実装するには、以下の環境が必要です。

- Android Studio Narwhal（2025.3）以降
- Kotlin 2.0 以上
- `minSdk` 26 以上
- Jetpack AppFunctions ライブラリ（`androidx.appfunctions`）

### Step 1: 依存関係の追加

まず、`build.gradle.kts` に AppFunctions ライブラリを追加します。

```kotlin
// build.gradle.kts (Module: app)
dependencies {
    implementation("androidx.appfunctions:appfunctions-runtime:1.0.0-alpha01")
    annotationProcessor("androidx.appfunctions:appfunctions-compiler:1.0.0-alpha01")
    // KSP を使用する場合
    // ksp("androidx.appfunctions:appfunctions-compiler:1.0.0-alpha01")
}
```

### Step 2: AppFunction の定義

`@AppFunction` アノテーションを使って、Gemini に公開する関数を定義します。

```kotlin
import androidx.appfunctions.AppFunction
import androidx.appfunctions.AppFunctionContext

class RestaurantFunctions {

    /**
     * 指定条件でレストランを検索する
     * @param cuisine 料理のジャンル（例: "italian", "japanese"）
     * @param location 検索エリア（例: "tokyo", "nearby"）
     * @param partySize 人数（デフォルト: 2）
     * @return 検索結果のレストランリスト
     */
    @AppFunction(
        displayName = "Search Restaurants",
        description = "Search for restaurants by cuisine type and location"
    )
    suspend fun searchRestaurants(
        context: AppFunctionContext,
        cuisine: String,
        location: String,
        partySize: Int = 2
    ): List<Restaurant> {
        // 実際のレストラン検索ロジック
        val repository = RestaurantRepository.getInstance()
        return repository.search(
            cuisine = cuisine,
            location = location,
            partySize = partySize
        )
    }

    /**
     * レストランを予約する
     * @param restaurantId レストランの一意識別子
     * @param date 予約日（ISO 8601形式）
     * @param time 予約時間（HH:mm形式）
     * @param partySize 人数
     * @return 予約確認情報
     */
    @AppFunction(
        displayName = "Make Reservation",
        description = "Make a restaurant reservation"
    )
    suspend fun makeReservation(
        context: AppFunctionContext,
        restaurantId: String,
        date: String,
        time: String,
        partySize: Int
    ): ReservationConfirmation {
        val repository = RestaurantRepository.getInstance()
        return repository.reserve(
            restaurantId = restaurantId,
            date = date,
            time = time,
            partySize = partySize
        )
    }
}

// データクラスの定義
data class Restaurant(
    val id: String,
    val name: String,
    val cuisine: String,
    val rating: Double,
    val address: String,
    val priceRange: String // "$", "$$", "$$$", "$$$$"
)

data class ReservationConfirmation(
    val confirmationId: String,
    val restaurantName: String,
    val date: String,
    val time: String,
    val partySize: Int,
    val status: String // "confirmed", "pending", "waitlist"
)
```

### Step 3: マニフェストへの登録

`AndroidManifest.xml` に AppFunctions サービスを登録します。

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <application>
        <!-- AppFunctions サービスの登録 -->
        <service
            android:name="androidx.appfunctions.AppFunctionService"
            android:exported="true"
            android:permission="android.permission.BIND_APP_FUNCTION_SERVICE">
            <intent-filter>
                <action android:name="androidx.appfunctions.ACTION_APP_FUNCTION" />
            </intent-filter>
            <meta-data
                android:name="androidx.appfunctions.APP_FUNCTION_CLASS"
                android:value="com.example.myapp.RestaurantFunctions" />
        </service>
    </application>
</manifest>
```

### Step 4: ユーザー同意フローの実装

AppFunctions では、Gemini がアプリの機能を呼び出す前に**必ずユーザーの同意が必要**です。

```kotlin
import androidx.appfunctions.AppFunctionConsentManager

class ConsentHandler(private val activity: Activity) {

    private val consentManager = AppFunctionConsentManager.getInstance(activity)

    /**
     * ユーザーに関数実行の許可を求める
     */
    suspend fun requestConsent(
        functionName: String,
        callerPackage: String
    ): Boolean {
        // システムダイアログでユーザーに確認を求める
        val result = consentManager.requestConsent(
            functionName = functionName,
            callerPackage = callerPackage,
            // 一度許可したら次回以降は自動承認するかどうか
            rememberChoice = true
        )
        return result.isGranted
    }
}
```

## 実践的なユースケース — タスク管理アプリへの統合

ここでは、より実用的な例として、タスク管理アプリに AppFunctions を組み込むケースを紹介します。

```kotlin
class TaskManagerFunctions {

    @AppFunction(
        displayName = "Create Task",
        description = "Create a new task with title, due date, and priority"
    )
    suspend fun createTask(
        context: AppFunctionContext,
        title: String,
        dueDate: String? = null,
        priority: String = "medium", // "low", "medium", "high"
        category: String? = null
    ): TaskResult {
        val taskDao = AppDatabase.getInstance().taskDao()
        val task = Task(
            title = title,
            dueDate = dueDate?.let { LocalDate.parse(it) },
            priority = Priority.valueOf(priority.uppercase()),
            category = category,
            createdAt = Instant.now()
        )
        val id = taskDao.insert(task)
        return TaskResult(
            success = true,
            taskId = id.toString(),
            message = "タスク「$title」を作成しました"
        )
    }

    @AppFunction(
        displayName = "List Tasks",
        description = "List tasks filtered by status, priority, or due date"
    )
    suspend fun listTasks(
        context: AppFunctionContext,
        status: String = "pending", // "pending", "completed", "all"
        priority: String? = null,
        dueBefore: String? = null
    ): List<TaskSummary> {
        val taskDao = AppDatabase.getInstance().taskDao()
        return taskDao.queryTasks(
            status = status,
            priority = priority,
            dueBefore = dueBefore
        ).map { task ->
            TaskSummary(
                id = task.id.toString(),
                title = task.title,
                dueDate = task.dueDate?.toString(),
                priority = task.priority.name.lowercase(),
                status = task.status
            )
        }
    }
}

// ユーザーの自然言語操作例:
// 「明日までに提出するレポートのタスクを高優先度で追加して」
// → createTask(title="レポート提出", dueDate="2026-03-24", priority="high")
//
// 「今週の未完了タスクを見せて」
// → listTasks(status="pending", dueBefore="2026-03-29")
```

この実装により、ユーザーは Gemini に「タスクを追加して」「今日の予定を確認して」と話しかけるだけで、アプリの機能をシームレスに利用できるようになります。

## AppFunctions 実装のベストプラクティス

### セキュリティとプライバシー

AppFunctions を安全に実装するために、以下のガイドラインを守りましょう。

1. **最小権限の原則**: 必要な機能だけを公開し、機密データへのアクセスは最小限にする
2. **ユーザー同意の徹底**: 金銭取引や個人情報に関わる操作は、毎回明示的な同意を求める
3. **入力のバリデーション**: AI エージェントからの入力値は必ず検証し、不正な値を排除する
4. **ログの記録**: どの関数がいつ呼び出されたかを記録し、監査に備える

```kotlin
// 入力バリデーションの例
@AppFunction(
    displayName = "Transfer Funds",
    description = "Transfer money to another account"
)
suspend fun transferFunds(
    context: AppFunctionContext,
    recipientId: String,
    amount: Double,
    currency: String = "JPY"
): TransferResult {
    // バリデーション: 金額の上限チェック
    require(amount > 0) { "送金額は0より大きい値を指定してください" }
    require(amount <= 1_000_000) { "1回の送金上限は100万円です" }

    // バリデーション: 通貨コードの確認
    require(currency in listOf("JPY", "USD", "EUR")) {
        "対応通貨: JPY, USD, EUR"
    }

    // 重要な操作は追加の認証を要求
    val authResult = BiometricAuthManager.authenticate(context)
    if (!authResult.isSuccess) {
        return TransferResult(success = false, message = "認証に失敗しました")
    }

    // 送金処理の実行
    return performTransfer(recipientId, amount, currency)
}
```

### パフォーマンスの最適化

AppFunctions はオンデバイスで実行されるため、レスポンス速度が重要です。

- **非同期処理**: `suspend` 関数を活用し、メインスレッドをブロックしない
- **キャッシュの活用**: 頻繁にアクセスするデータはローカルキャッシュに保持
- **軽量なレスポンス**: 返却データは必要最小限にし、詳細は別途取得する設計にする

## よくある質問（FAQ）

### Q1: AppFunctions は全ての Android デバイスで使えますか？

現時点では、Samsung Galaxy S26 シリーズと Google Pixel シリーズを中心に展開されています。`minSdk` 26 以上が必要で、Android 17 でさらに多くのデバイスに拡大される予定です。2026年末までに2億台以上のデバイスで利用可能になる見込みです。

### Q2: AppFunctions と既存の Function Calling を併用できますか？

はい、併用可能です。Function Calling はクラウド側のバックエンド API との連携に、AppFunctions はデバイス上のアプリ機能の連携に使い分けるのが推奨されます。たとえば、天気情報の取得は Function Calling で、カレンダーへの予定追加は AppFunctions で行うといった使い分けが考えられます。

### Q3: AppFunctions で公開した関数のセキュリティはどう確保されますか？

AppFunctions は Android のパーミッションシステムと統合されており、ユーザーの明示的な同意なしに関数が実行されることはありません。また、開発者はバリデーション、レート制限、生体認証連携などを実装して、追加のセキュリティレイヤーを設けることができます。

### Q4: 個人開発者でも AppFunctions を導入するメリットはありますか？

大いにあります。AppFunctions に対応することで、Gemini が自然言語でアプリの機能を呼び出せるようになり、ユーザー体験が大幅に向上します。2億台以上のデバイスで「あなたのアプリの機能を声で使える」ことは、大きな差別化要因になります。

## まとめ — AppFunctions でアプリの可能性を広げよう

Gemini AppFunctions は、Android アプリに AI エージェント機能を組み込むための強力なフレームワークです。UI オートメーションではなく構造化された API を通じて、安全かつ高速にアプリの機能を Gemini に委任できます。

今のうちから AppFunctions を学び、実装を始めておくことで、Android 17 のリリースと AI エージェントの普及に先駆けた準備ができます。まずは既存アプリの中で「ユーザーがよく使う機能」を1つ AppFunctions として公開するところから始めてみてください。
