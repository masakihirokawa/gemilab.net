## はじめに — VS Code × Gemini で変わる開発体験

Visual Studio Code（VS Code）は世界で最も利用されているコードエディタの一つです。そこに Google の Gemini AI を組み合わせることで、コード補完、エラー修正、テスト生成、ドキュメント作成といった開発タスクが飛躍的に効率化されます。

本記事では、VS Code に Gemini Code Assist 拡張機能をインストールし、日々の開発ワークフローに組み込むまでの手順を丁寧に解説します。はじめてAIコーディング支援を試す方にも分かりやすいよう、スクリーンショットの代わりに操作手順を細かく示していますので、手元で実際に操作しながら読み進めてみてください。

## Gemini Code Assist とは

Gemini Code Assist は、Google が提供する AI コーディング支援ツールです。VS Code 上で以下のような支援を提供します。

- **インラインコード補完**: 入力中のコードに対して、次に書くべきコードをリアルタイムで提案
- **チャットベースのコード生成**: サイドパネルのチャットUIから自然言語でコードを生成・修正
- **コード説明・リファクタリング**: 選択したコードブロックの意味を解説したり、より良い書き方を提案
- **エラー診断と修正提案**: ビルドエラーやランタイムエラーに対する修正案を提示
- **テスト生成**: 関数やクラスに対するユニットテストを自動生成

2026年3月現在、Gemini Code Assist は個人利用であれば無料プランで利用を開始でき、Google アカウントがあればすぐにセットアップできます。

## Step 1: 前提条件の確認

Gemini Code Assist を使い始める前に、以下の環境が整っていることを確認しましょう。

- **VS Code**: バージョン 1.95 以上（2026年3月時点の最新安定版を推奨）
- **Google アカウント**: 無料の Google アカウントがあれば利用可能
- **インターネット接続**: Gemini API との通信に必要

VS Code のバージョンは、メニューの「Help → About」（macOS では「Code → About Visual Studio Code」）から確認できます。古いバージョンの場合は、公式サイト（https://code.visualstudio.com/）から最新版をダウンロードしてください。

## Step 2: Gemini Code Assist 拡張機能のインストール

VS Code に Gemini Code Assist を導入する手順は以下の通りです。

**方法1: VS Code マーケットプレイスからインストール**

1. VS Code を起動する
2. 左サイドバーの拡張機能アイコン（四角が4つ並んだアイコン）をクリック
3. 検索バーに「Gemini Code Assist」と入力
4. 「Gemini Code Assist」（発行元: Google）を見つけて「Install」をクリック
5. インストール完了後、VS Code の再読み込みを求められた場合は「Reload」をクリック

**方法2: コマンドラインからインストール**

ターミナルから以下のコマンドを実行する方法もあります。

```bash
# VS Code の拡張機能をコマンドラインからインストール
code --install-extension google.gemini-code-assist

# インストール確認
code --list-extensions | grep gemini
# 期待する出力: google.gemini-code-assist
```

インストールが完了すると、VS Code の左サイドバーに Gemini のアイコン（星形のマーク）が表示されます。

## Step 3: Google アカウントでサインイン

拡張機能をインストールしたら、Google アカウントでサインインして Gemini を有効化します。

1. サイドバーの Gemini アイコンをクリック
2. 「Sign in with Google」ボタンをクリック
3. ブラウザが開き、Google アカウントの認証画面が表示される
4. 使用する Google アカウントを選択し、アクセス権限を許可する
5. 「認証が完了しました」と表示されたら、VS Code に戻る

サインインが完了すると、Gemini のサイドパネルにチャット画面が表示され、すぐに使い始められます。

## Step 4: 基本設定のカスタマイズ

デフォルト設定でも十分に使えますが、以下の設定を調整するとさらに快適になります。VS Code の設定（`Cmd + ,` / `Ctrl + ,`）を開き、検索バーに「Gemini」と入力してください。

```json
{
  // インラインコード補完を有効にする（デフォルト: true）
  "gemini.codeAssist.inlineSuggestions.enabled": true,

  // コード補完の提案を受け入れるキーバインド
  // Tab キーで確定（デフォルト）
  "gemini.codeAssist.inlineSuggestions.acceptKey": "Tab",

  // チャットパネルで使用するモデル
  // "gemini-2.5-pro" が最も高性能（デフォルト）
  "gemini.codeAssist.model": "gemini-2.5-pro",

  // 自動補完のトリガーディレイ（ミリ秒）
  // 短くするとレスポンスは速くなるがAPI呼び出しが増える
  "gemini.codeAssist.inlineSuggestions.delay": 300
}
```

特に `delay` の値は、タイピング速度に合わせて調整すると良いでしょう。入力のたびに提案が変わって煩わしい場合は `500` 程度に増やし、即座に提案が欲しい場合は `200` に減らしてみてください。

## Step 5: 実践的な活用テクニック

セットアップが完了したら、実際のコーディングで Gemini を活用してみましょう。ここでは代表的な使い方を紹介します。

### コード補完の活用

関数のシグネチャを書き始めると、Gemini が本体の実装を自動的に提案します。

```python
# ファイル名: utils.py
# 関数のシグネチャを入力すると、Gemini が実装を提案する

def calculate_moving_average(data: list[float], window_size: int) -> list[float]:
    # ↓ ここから先を Gemini が自動補完してくれる
    if window_size <= 0:
        raise ValueError("window_size must be positive")
    if window_size > len(data):
        raise ValueError("window_size cannot exceed data length")

    result = []
    for i in range(len(data) - window_size + 1):
        window = data[i:i + window_size]
        result.append(sum(window) / window_size)
    return result

# 期待する出力:
# calculate_moving_average([1, 2, 3, 4, 5], 3)
# → [2.0, 3.0, 4.0]
```

### チャットでコード生成

Gemini のサイドパネルを開いて（`Cmd + Shift + P` → 「Gemini: Open Chat」）、自然言語でリクエストするとコードを生成してくれます。

```
# チャットで入力する例:
「FastAPIでユーザー認証付きのREST APIエンドポイントを作って。
JWTトークンベースで、ログインとプロフィール取得の2つのエンドポイントが必要。」
```

Gemini は要件を理解し、FastAPI のコード一式を生成してくれます。生成されたコードは「Insert at Cursor」ボタンでエディタに直接挿入できます。

### 選択コードの説明・リファクタリング

理解しづらいコードがあるときは、該当箇所を選択して右クリックメニューから「Gemini: Explain This Code」を選ぶと、コードの動作を自然言語で解説してくれます。

```javascript
// 例: このコードを選択して「Explain This Code」を実行
const debounce = (fn, ms) => {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), ms);
  };
};
// Gemini の解説:
// "この関数はデバウンス処理を実装しています。
//  指定されたミリ秒(ms)の間に再度呼び出されると
//  タイマーをリセットし、最後の呼び出しから
//  ms ミリ秒後に fn を1回だけ実行します。"
```

同様に「Gemini: Refactor This Code」を選ぶと、より読みやすく保守しやすいコードへの改善案を提案してくれます。

## トラブルシューティング

Gemini Code Assist を使っていて問題が発生した場合の対処法をまとめます。

### Q: インライン補完が表示されない

**A**: まず設定で `gemini.codeAssist.inlineSuggestions.enabled` が `true` になっているか確認してください。他のAIコード補完拡張（GitHub Copilot など）と競合している場合は、一方を無効にすると解決することがあります。また、VS Code の出力パネル（Output → Gemini Code Assist）でエラーメッセージを確認してください。

### Q: 「Authentication failed」と表示される

**A**: Google アカウントの認証トークンが期限切れになっている可能性があります。コマンドパレット（`Cmd + Shift + P`）から「Gemini: Sign Out」を実行した後、再度「Gemini: Sign In」でサインインし直してください。

### Q: 補完の品質を上げるには？

**A**: Gemini はプロジェクト内のファイルコンテキストを参照して補完精度を高めます。プロジェクトのルートに明確なディレクトリ構造と、適切なコメント・型定義があると、より正確な補完が得られます。また、`settings.json` でモデルを `gemini-2.5-pro` に設定すると、より高品質な補完が期待できます。
