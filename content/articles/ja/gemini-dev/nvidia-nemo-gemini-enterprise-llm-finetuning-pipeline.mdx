# NVIDIA NeMo Framework × Gemini API — エンタープライズLLMファインチューニングとハイブリッド推論パイプライン

エンタープライズのLLM運用において、汎用モデルの能力だけでは足りません。ドメイン固有の知識、企業文化、業界規制に対応したカスタムモデルが必要です。NVIDIA NeMo Frameworkは、単一GPUからクラウドスケール（数千ノード）まで、エンドツーエンドのLLM学習パイプラインを提供する業界標準のツールです。

本記事では、NeMo FrameworkでGPU加速ファインチューニングを実行し、Gemini APIとの**ハイブリッド推論**構造を構築する実装パターンを解説します。これにより、コスト効率と品質を両立させたエンタープライズLLMシステムが実現します。

## NeMo Frameworkの核となる3層アーキテクチャ

NVIDIA NeMo Frameworkは、プリトレーニング（Pretraining）、ポストトレーニング（Post-training）、強化学習（RLHF）を統一されたPyTorch APIで提供します。

```python
# NeMo 2.0: PyTorch Native Custom Loopでの柔軟な学習管理
import nemo.lightning as nl
from nemo.collections import llm

# ステップ1: モデルアーキテクチャの定義
config = llm.GPTModel(
    num_layers=32,
    hidden_size=4096,
    num_attention_heads=32,
    ffn_hidden_size=16384,
    max_position_embeddings=8192,
    use_flash_attention=True,  # H100 NVIDIA GPU対応
)

# ステップ2: トークナイザーの準備（既存モデルまたはカスタム）
tokenizer = nl.TokenizerFactory.get(
    name="nemotron",
    vocab_size=256000,
)

# ステップ3: 分散トレーニング設定（単一GPU～マルチノード）
trainer = nl.Trainer(
    devices=8,  # 単一マシンで8GPU使用
    strategy="fsdp",  # Fully Sharded Data Parallel
    max_epochs=3,
    precision="bf16",  # NVIDIA A100/H100推奨
)

# ステップ4: データローダーとオプティマイザー
model = llm.GPTModel(config)
optim = nl.OptimizerFactory.get(
    name="megatron_adam",
    learning_rate=2e-4,
    weight_decay=0.1,
)

# ステップ5: カスタム学習ループ（NeMo 2.0の強力な機能）
trainer.fit(model, train_dataloaders=train_loader)
```

このアプローチにより、社内データセットを使用した**ハイパフォーマンスなドメイン適応**が可能になります。

## ファインチューニングのベストプラクティス

### Step 1: データ準備と前処理

```python
from nemo.collections.common.tokenizers.sentencepiece_tokenizer import SentencePieceTokenizer
from torch.utils.data import DataLoader, IterableDataset

class EnterpriseDataset(IterableDataset):
    """企業固有のテキストデータセット"""

    def __init__(self, data_path: str, tokenizer, max_seq_len: int = 2048):
        self.tokenizer = tokenizer
        self.max_seq_len = max_seq_len

        # 社内ドキュメント、チャットログ、技術仕様書などを読み込み
        with open(data_path, 'r', encoding='utf-8') as f:
            self.texts = f.readlines()

    def __iter__(self):
        for text in self.texts:
            tokens = self.tokenizer.text_to_ids(text.strip())

            # シーケンス長を一定にパディング/トランケート
            if len(tokens) > self.max_seq_len:
                tokens = tokens[:self.max_seq_len]
            else:
                tokens = tokens + [self.tokenizer.pad_id] * (self.max_seq_len - len(tokens))

            yield {
                "input_ids": torch.tensor(tokens),
                "labels": torch.tensor(tokens),  # 自己教師あり学習
            }

# データローダーの構築
train_dataset = EnterpriseDataset(
    data_path="./data/company_corpus.txt",
    tokenizer=tokenizer,
)

train_loader = DataLoader(
    train_dataset,
    batch_size=32,
    num_workers=4,
)
```

### Step 2: LoRA（Low-Rank Adaptation）による軽量ファインチューニング

メモリと計算リソースが制限されている場合、LoRA パラメータだけを更新することで、**モデル全体の0.1～1%のパラメータ数**で高い適応性を実現します。

```python
from nemo.collections.nlp.parts.peft import LoRA

# Nemotron-4 70Bモデルのロード（事前学習済み）
base_model = llm.GPTModel.from_pretrained("nvidia/Nemotron-4-70B-base")

# LoRA設定: クエリ・バリュー層のみにアダプターを挿入
lora_config = {
    "adapter_name": "enterprise_lora",
    "lora_r": 16,  # ランク（高いほど表現力UP、メモリ増加）
    "lora_alpha": 32,
    "target_modules": ["q_proj", "v_proj"],  # 注意層のみ
    "lora_dropout": 0.05,
}

model_with_lora = base_model.add_adapter("enterprise", lora_config)

# LoRAパラメータのみを学習
for param in model_with_lora.parameters():
    param.requires_grad = False  # 全パラメータを凍結

for param in model_with_lora.get_adapter_state_dict("enterprise").values():
    param.requires_grad = True  # LoRAパラメータのみ有効

# 軽量な学習ループ
optimizer = torch.optim.AdamW(
    model_with_lora.parameters(),
    lr=1e-4,
)

for epoch in range(3):
    for batch in train_loader:
        logits = model_with_lora(input_ids=batch["input_ids"])
        loss = torch.nn.functional.cross_entropy(
            logits.view(-1, tokenizer.vocab_size),
            batch["labels"].view(-1)
        )
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
```

## ハイブリッド推論パイプラインの設計

ファインチューニング後のモデルをすべてのタスクで使用するのではなく、**タスク特性に応じて推論エンジンを使い分ける**ハイブリッドアーキテクチャが実務的です。

```python
from google.generativeai import genai
import asyncio

class HybridInferencePipeline:
    """NeMoファインチューニングモデル × Gemini APIのハイブリッド推論"""

    def __init__(self, nemo_model_path: str, gemini_api_key: str):
        # NeMoファインチューニングモデル（オンプレミスまたはプライベートGPU）
        self.nemo_model = self._load_nemo_model(nemo_model_path)

        # Gemini API設定
        genai.configure(api_key=gemini_api_key)
        self.gemini_client = genai.GenerativeModel("gemini-2.0-flash")

    def _load_nemo_model(self, model_path: str):
        """保存されたNeMoファインチューニングモデルのロード"""
        from nemo.collections import llm
        model = llm.GPTModel.restore_from_checkpoint(model_path)
        model.eval()  # 推論モード
        return model

    async def process_query(
        self,
        user_query: str,
        use_nemo: bool = None,
        context: str = None,
    ) -> dict:
        """
        ユースケースに応じて推論エンジンを自動選択

        Args:
            user_query: ユーザーの質問
            use_nemo: Trueなら強制的にNeMo使用、Noneなら自動選択
            context: 企業知識ベースやドキュメント参照

        Returns:
            {"response": str, "engine": str, "confidence": float}
        """

        # ステップ1: クエリ分類（Gemini APIで高速判定）
        classification = await self._classify_query(user_query)

        # ステップ2: 推論エンジンの選択
        if use_nemo is not None:
            use_nemo_flag = use_nemo
        elif classification["requires_domain_knowledge"]:
            use_nemo_flag = True  # 企業固有知識が必要 → NeMo使用
        else:
            use_nemo_flag = False  # 汎用知識で対応 → Gemini使用

        # ステップ3: 各エンジンで推論実行
        if use_nemo_flag:
            response, confidence = await self._inference_nemo(user_query, context)
        else:
            response, confidence = await self._inference_gemini(user_query, context)

        return {
            "response": response,
            "engine": "NeMo" if use_nemo_flag else "Gemini",
            "confidence": confidence,
            "classification": classification,
        }

    async def _classify_query(self, query: str) -> dict:
        """Gemini APIでクエリの特性を分類"""
        prompt = f"""以下のクエリを分析してください:
"{query}"

JSON形式で以下を出力:
{{
  "requires_domain_knowledge": boolean,
  "task_type": "summary|analysis|generation|retrieval",
  "estimated_complexity": "low|medium|high"
}}"""

        response = await self.gemini_client.generate_content_async(prompt)
        import json
        return json.loads(response.text)

    async def _inference_nemo(self, query: str, context: str) -> tuple:
        """NeMoファインチューニングモデルで推論"""
        import torch

        # プロンプト組立て
        prompt = f"""【企業知識ベース】
{context or '（コンテキスト未提供）'}

【ユーザーの質問】
{query}

【回答】"""

        # トークン化
        tokens = self.nemo_model.tokenizer.text_to_ids(prompt)
        input_ids = torch.tensor([tokens]).to(self.nemo_model.device)

        # 推論（ビームサーチで複数候補生成）
        with torch.no_grad():
            output_ids = self.nemo_model.generate(
                input_ids=input_ids,
                max_new_tokens=512,
                temperature=0.7,
                top_p=0.9,
                num_beams=4,  # ビームサーチ
                length_penalty=0.6,
            )

        # デコード
        response_text = self.nemo_model.tokenizer.ids_to_text(output_ids[0].tolist())

        # 信頼度の推定（シンプルな方法）
        confidence = 0.85  # NeMoファインチューニングは高信頼度

        return response_text, confidence

    async def _inference_gemini(self, query: str, context: str) -> tuple:
        """Gemini APIで推論"""
        prompt = f"""【コンテキスト】
{context or '（なし）'}

【質問】
{query}"""

        response = await self.gemini_client.generate_content_async(prompt)

        # Gemini APIの安全性フィルターでブロックされた場合
        if not response.text:
            confidence = 0.0
            return "[回答を生成できません - フィルター適用]", confidence

        confidence = 0.75  # Gemini APIは汎用知識で中程度の信頼度
        return response.text, confidence


# 使用例
async def main():
    pipeline = HybridInferencePipeline(
        nemo_model_path="./checkpoints/nemo_finetuned_latest.nemo",
        gemini_api_key="YOUR_GEMINI_API_KEY",
    )

    result = await pipeline.process_query(
        user_query="当社の新製品ローンチに向けた市場分析レポートを要約してください",
        context="【製品ローンチプロジェクト】Q2商品化予定、市場規模は500億円想定...",
    )

    print(f"回答: {result['response']}")
    print(f"使用エンジン: {result['engine']}")
    print(f"信頼度: {result['confidence']:.2%}")

asyncio.run(main())
```

```python
# NIM推論クライアント（REST API）
import aiohttp

class NIMInferenceClient:
    def __init__(self, nim_endpoint: str = "http://localhost:8000"):
        self.endpoint = nim_endpoint

    async def generate(self, prompt: str, max_tokens: int = 512) -> str:
        """NIM経由でNeMoモデルを推論"""
        payload = {
            "model": "nemotron-4-340b-instruct",
            "messages": [
                {"role": "user", "content": prompt}
            ],
            "temperature": 0.7,
            "max_tokens": max_tokens,
        }

        async with aiohttp.ClientSession() as session:
            async with session.post(
                f"{self.endpoint}/v1/chat/completions",
                json=payload,
            ) as response:
                data = await response.json()
                return data["choices"][0]["message"]["content"]


# Gemini APIとの簡単な切り替え
async def unified_inference(
    prompt: str,
    use_nemo: bool = False,
    nim_endpoint: str = "http://localhost:8000",
):
    """企業知識向けはNIM、汎用はGeminiを使用"""

    if use_nemo:
        client = NIMInferenceClient(nim_endpoint)
        response = await client.generate(prompt)
    else:
        response = genai.GenerativeModel("gemini-2.0-flash").generate_content(prompt).text

    return response
```

## よくある質問（FAQ）

**Q1: NeMo vs. Gemini API - どちらを選ぶべき？**

NeMoは社内データ・専門知識向け、Gemini APIは汎用知識・高速性向き。本記事のハイブリッドアプローチが実務的です。NeMoはVPU/GPU投資が必要、Gemini APIは従量課金モデルで初期コスト低。

**Q2: LoRAランク16と64の違いは？**

ランク16で十分なケースが多く、メモリ効率が良い。ランク64は表現力が高い反面、ファインチューニング時間が2倍以上になる。ドメイン複雑度に応じて実験的に決定。

**Q3: RLHF訓練に必要な人間アノテーション量は？**

通常、1000～5000サンプルの比較アノテーション（応答A vs B）で有効な報酬モデル構築が可能。クラウドソーシングで週単位で収集可能。

**Q4: マルチモーダル（テキスト+画像）ファインチューニングは可能？**

NeMo 2.0はマルチモーダルアーキテクチャをサポート。Gemini APIも画像入力が可能。ハイブリッドパイプラインに画像処理を組み込めます。

**Q5: ファインチューニング後のモデルライセンスは？**

Nemotronは商用利用OKなopen LLM。ファインチューニング版も同様。ただし出力内容（著作権）の取扱いには注意。

**Q6: 推論レイテンシー目標はどのくらい？**

企業チャットボット: 1～3秒/応答、バッチ分析: 10～30秒/ドキュメント。NIM + GPUキャッシングで0.5秒/応答も可能（コスト増）。
