## Google AI エコシステムで開発を完結させる

Stitch と Gemini は、同じ Google が提供する AI ツールです。この2つを組み合わせると、**フロントエンドのデザイン生成（Stitch）** から **バックエンドのロジック実装（Gemini API）** まで、Google AI エコシステムの中で完結した開発フローが実現します。

サードパーティツールへの依存を最小限に抑えながら、AI が主導するフルスタック開発——これが Stitch × Gemini の目指す姿です。

---

## Google Stitch とは

**Google Stitch**（[stitch.withgoogle.com](https://stitch.withgoogle.com)）は、Google が提供する**生成 AI 搭載の UI デザインツール**です。

主な特徴：
- 自然言語プロンプトから Material Design 3 準拠の UI を生成
- React、HTML/CSS 形式でのエクスポート
- コンポーネント単位での生成・カスタマイズ
- Google のデザイン言語と完全に統合

---

## Stitch × Gemini の基本ワークフロー

### フロー全体像

```
[Stitch] → UI コンポーネント生成（React / HTML）
    ↓
[Gemini API] → コードの最適化・バックエンド連携コード生成
    ↓
[Firebase / Vertex AI] → デプロイ・AI 機能の追加
```

---

## MCP 経由での Stitch 連携

Gemini CLI や Claude Code の MCP に Stitch サーバーを追加することで、AI との会話の中でシームレスに UI を生成できます。

### MCP 設定（mcp_config.json）

```json
{
  "mcpServers": {
    "stitch": {
      "command": "npx",
      "args": ["-y", "@google/stitch-mcp-server@latest"],
      "env": {
        "GOOGLE_API_KEY": "YOUR_GOOGLE_API_KEY"
      }
    }
  }
}
```

### 設定後の使用例（Gemini CLI）

```bash
# Gemini CLI でStitch MCPを活用
gemini "Stitchでダッシュボード画面を生成して、
Firebase Firestore と連携する React コンポーネントに変換してください"
```

## Python での Stitch × Gemini 統合

### Stitch API でコンポーネントを生成し、Gemini で拡張する

```python
import google.generativeai as genai
import requests
import json

genai.configure(api_key="YOUR_GEMINI_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

def generate_ui_with_stitch(description: str, components: list[str]) -> dict:
    """
    Google Stitch API を使って UI コンポーネントを生成
    (Stitch API の実装例 - 公式 API エンドポイントに合わせて調整)
    """
    # Stitch API リクエスト
    stitch_payload = {
        "description": description,
        "components": components,
        "style": "material-design-3",
        "platform": "react",
        "dark_mode": True
    }

    # Stitch から生成された React コードを返す
    # (実際の API エンドポイントは公式ドキュメントを参照)
    return stitch_payload  # 簡略化

def enhance_stitch_with_gemini(stitch_output: str, requirements: str) -> str:
    """
    Stitch が生成した UI を Gemini API でさらに拡張
    """
    prompt = f"""
以下の Google Stitch が生成した React コードに、
指定された機能を追加してください：

Stitch 生成コード:
{stitch_output}

追加要件:
{requirements}

出力形式:
1. 拡張された React コンポーネント（TypeScript）
2. 必要な型定義
3. テスト用の mock データ
"""
    response = model.generate_content(prompt)
    return response.text

# 使用例: ダッシュボード UI の生成と拡張
ui_description = """
SaaS アプリのダッシュボード:
- 月次売上グラフ（折れ線）
- ユーザー数カード（前月比）
- 最新アクティビティフィード
- Material Design 3、ダーク/ライトモード
"""

enhancements = """
- Firebase Firestore からリアルタイムデータを取得
- recharts でグラフを描画
- React Query でキャッシュ管理
- ローディングスケルトンアニメーション
"""

stitch_result = generate_ui_with_stitch(ui_description, ["Chart", "Card", "Feed"])
enhanced_code = enhance_stitch_with_gemini(str(stitch_result), enhancements)
print(enhanced_code)
```

**2. Gemini API で問い合わせ応答を自動化**

```python
def create_ai_contact_form(form_data: dict) -> dict:
    """
    問い合わせフォームの送信を受け取り、
    Gemini API で自動返信文を生成
    """

    # 問い合わせ内容に基づいて返信文を生成
    prompt = f"""
以下の問い合わせに対して、丁寧でプロフェッショナルな返信メールを作成してください：

送信者名: {form_data.get('name')}
メール: {form_data.get('email')}
問い合わせ内容: {form_data.get('message')}

返信内容：
- 問い合わせを受け取った感謝の言葉
- 問い合わせ内容への具体的な回答または次のステップ
- 営業担当から 24 時間以内に連絡する旨
- 会社名：Dolice Labs
"""
    response = model.generate_content(prompt)

    return {
        "auto_reply": response.text,
        "original_query": form_data,
        "status": "processed"
    }
```

**3. Stitch コードと組み合わせる**

```python
def integrate_ai_with_stitch_form(stitch_form_code: str) -> str:
    """
    Stitch が生成したフォームコードに Gemini AI 連携を追加
    """
    prompt = f"""
以下の Stitch 生成フォームコードに、
Gemini API との連携を追加してください：

フォームコード:
{stitch_form_code}

追加する機能:
1. フォーム送信時に /api/contact エンドポイントに POST
2. 送信中のローディングアニメーション（Stitch のスタイルに合わせる）
3. 送信成功時のサクセスメッセージ（Gemini の自動返信内容を表示）
4. バリデーション（メール形式、必須フィールド）
5. エラーハンドリング（ネットワークエラー等）
```

## Stitch × Gemini × Vertex AI のトリプル連携

さらに発展させ、Vertex AI を加えた3層構成にすることで、エンタープライズグレードのアプリが構築できます。

```python
from google.cloud import aiplatform
from vertexai.generative_models import GenerativeModel as VertexModel

def stitch_gemini_vertex_pipeline(
    ui_description: str,
    business_logic: str,
    model_type: str = "gemini-2.0-flash"
) -> dict:
    """
    Stitch（UI）× Gemini API（ロジック）× Vertex AI（スケール）の統合パイプライン
    """

    # 1. Stitch で UI を生成
    ui_code = f"# Stitch generated UI for: {ui_description}"

    # 2. Gemini API でビジネスロジックを実装
    gemini_prompt = f"""
UI 仕様: {ui_description}
ビジネスロジック要件: {business_logic}

以下を実装してください：
- React コンポーネント（Stitch スタイル準拠）
- API クライアント（Vertex AI エンドポイント呼び出し）
- エラーハンドリング
- TypeScript 型定義
"""
    gemini_model = genai.GenerativeModel(model_type)
    logic_code = gemini_model.generate_content(gemini_prompt).text

    # 3. Vertex AI でスケーラブルなバックエンドを構成
    vertex_config = {
        "endpoint": "us-central1-aiplatform.googleapis.com",
        "model": "gemini-2.0-flash-001",
        "features": ["streaming", "function_calling", "grounding"]
    }

    return {
        "ui": ui_code,
        "logic": logic_code,
        "vertex_config": vertex_config,
        "deployment_target": "Cloud Run + Firebase Hosting"
    }
```

## Stitch × Gemini が特に効果的なシーン

### 1. プロトタイプ → 本番への移行

スタートアップのよくある課題「プロトタイプは動くが本番品質にできない」を解決します。Stitch でビジュアルを固め、Gemini で本番品質のコードに昇華させます。

### 2. 非エンジニアによる UI 作成

デザイナーや PM が Stitch で UI を生成し、Gemini が技術的な実装詳細を補完——エンジニアがいなくてもプロダクション品質の UI が作れます。

### 3. デザインの一貫性と Material Design 準拠

両ツールとも Google の Material Design を根拠としているため、デザイントークン（カラー、スペーシング、タイポグラフィ）が自然と一致します。

## まとめ

Google Stitch × Gemini API は、Google AI エコシステム内で完結する開発フローを提供します。

- **Stitch**: 美しい UI を言葉で生成（Material Design 3 準拠）
- **Gemini API**: ロジック実装・バックエンド連携・コード拡張
- **Vertex AI**: エンタープライズグレードのスケールアップ

この組み合わせにより、個人開発者でも企業レベルのプロダクトを短期間で構築できる時代が来ています。Google AI エコシステムのシナジーを最大限に活用してください。
