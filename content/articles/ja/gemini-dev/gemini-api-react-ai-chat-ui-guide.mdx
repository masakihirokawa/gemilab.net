## はじめに — なぜ React × Gemini API なのか

フロントエンドの世界で圧倒的なシェアを持つ React と、Google が提供する最新の大規模言語モデル Gemini API を組み合わせれば、高品質な AI チャットアプリケーションを驚くほど短時間で構築できます。

Next.js のようなフルスタックフレームワークを使わず、Vite + React + Express.js というシンプルな構成を選ぶことで、既存のプロジェクトへの組み込みも容易になります。この記事では、プロジェクトの初期設定からストリーミングレスポンス、会話履歴の管理まで、実用的な AI チャット UI を一歩ずつ構築していきます。

この記事で学べること:

- Vite で React + TypeScript プロジェクトをセットアップする方法
- Express.js をプロキシサーバーとして使い、API キーを安全に管理するパターン
- Gemini API のストリーミングレスポンスをリアルタイムに表示するテクニック
- マルチターン会話の状態管理と UI 設計のベストプラクティス

## 環境準備

### 必要なもの

- **Node.js 20 以上**
- **Google AI Studio の API キー**（[Google AI Studio](https://aistudio.google.com/) で無料取得可能）
- **お好みのコードエディタ**（VS Code 推奨）

### プロジェクトの作成

まず Vite を使って React + TypeScript プロジェクトを作成します。

```bash
# プロジェクト作成
npm create vite@latest gemini-chat -- --template react-ts
cd gemini-chat

# 依存パッケージのインストール
npm install

# バックエンド用パッケージ
npm install express cors @google/genai
npm install -D @types/express @types/cors tsx
```

ディレクトリ構成は次のようになります:

- `src/` — React フロントエンド
- `server/` — Express.js バックエンド（API プロキシ）
- `.env` — 環境変数（API キー）

### 環境変数の設定

プロジェクトルートに `.env` ファイルを作成します。

```bash
# .env
GEMINI_API_KEY=YOUR_API_KEY
```

**重要**: `.env` ファイルは必ず `.gitignore` に追加し、API キーをリポジトリにコミットしないでください。

## バックエンド API プロキシの構築

ブラウザから直接 Gemini API を呼び出すと、API キーがクライアントに露出してしまいます。Express.js でプロキシサーバーを構築し、キーをサーバーサイドに隠蔽するのが鉄則です。

```typescript
// server/index.ts
import express from "express";
import cors from "cors";
import { GoogleGenAI } from "@google/genai";

const app = express();
app.use(cors({ origin: "http://localhost:5173" }));
app.use(express.json());

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

// ── 通常レスポンス ──
app.post("/api/chat", async (req, res) => {
  try {
    const { messages } = req.body;

    // Gemini API の contents 形式に変換
    const contents = messages.map(
      (msg: { role: string; text: string }) => ({
        role: msg.role === "user" ? "user" : "model",
        parts: [{ text: msg.text }],
      })
    );

    const response = await ai.models.generateContent({
      model: "gemini-2.5-flash",
      contents,
    });

    res.json({
      text: response.text ?? "",
    });
  } catch (error: any) {
    console.error("Gemini API error:", error.message);
    res.status(500).json({ error: error.message });
  }
});

// ── ストリーミングレスポンス ──
app.post("/api/chat/stream", async (req, res) => {
  try {
    const { messages } = req.body;

    const contents = messages.map(
      (msg: { role: string; text: string }) => ({
        role: msg.role === "user" ? "user" : "model",
        parts: [{ text: msg.text }],
      })
    );

    // SSE ヘッダーの設定
    res.setHeader("Content-Type", "text/event-stream");
    res.setHeader("Cache-Control", "no-cache");
    res.setHeader("Connection", "keep-alive");

    const response = await ai.models.generateContentStream({
      model: "gemini-2.5-flash",
      contents,
    });

    for await (const chunk of response) {
      const text = chunk.text ?? "";
      if (text) {
        res.write(`data: ${JSON.stringify({ text })}\n\n`);
      }
    }

    res.write("data: [DONE]\n\n");
    res.end();
  } catch (error: any) {
    console.error("Stream error:", error.message);
    res.status(500).json({ error: error.message });
  }
});

const PORT = 3001;
app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
```

`package.json` にサーバー起動スクリプトを追加します:

```json
{
  "scripts": {
    "dev": "vite",
    "server": "tsx --env-file=.env server/index.ts"
  }
}
```

これで `npm run server` と `npm run dev` をそれぞれ別ターミナルで起動すれば、フロントエンドとバックエンドが連携します。

## React チャット UI の実装

### メッセージの型定義

```typescript
// src/types.ts
export interface Message {
  id: string;
  role: "user" | "model";
  text: string;
  timestamp: Date;
  isStreaming?: boolean;
}
```

### カスタムフック — useChat

チャットのロジックをカスタムフックに切り出すことで、UI コンポーネントをシンプルに保ちます。

```typescript
// src/hooks/useChat.ts
import { useState, useCallback, useRef } from "react";
import type { Message } from "../types";

const API_BASE = "http://localhost:3001";

export function useChat() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const abortRef = useRef<AbortController | null>(null);

  const sendMessage = useCallback(async (text: string) => {
    const userMessage: Message = {
      id: crypto.randomUUID(),
      role: "user",
      text,
      timestamp: new Date(),
    };

    const assistantMessage: Message = {
      id: crypto.randomUUID(),
      role: "model",
      text: "",
      timestamp: new Date(),
      isStreaming: true,
    };

    setMessages((prev) => [...prev, userMessage, assistantMessage]);
    setIsLoading(true);

    // 送信する会話履歴を構築
    const history = [...messages, userMessage].map((m) => ({
      role: m.role,
      text: m.text,
    }));

    try {
      abortRef.current = new AbortController();

      const response = await fetch(`${API_BASE}/api/chat/stream`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ messages: history }),
        signal: abortRef.current.signal,
      });

      if (!response.ok) throw new Error("API request failed");

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();
      let accumulated = "";

      while (reader) {
        const { done, value } = await reader.read();
        if (done) break;

        const chunk = decoder.decode(value, { stream: true });
        const lines = chunk.split("\n");

        for (const line of lines) {
          if (line.startsWith("data: ")) {
            const data = line.slice(6);
            if (data === "[DONE]") break;

            try {
              const parsed = JSON.parse(data);
              accumulated += parsed.text;

              // ストリーミング中のメッセージを更新
              setMessages((prev) =>
                prev.map((m) =>
                  m.id === assistantMessage.id
                    ? { ...m, text: accumulated }
                    : m
                )
              );
            } catch {
              // JSON パースに失敗した行はスキップ
            }
          }
        }
      }

      // ストリーミング完了
      setMessages((prev) =>
        prev.map((m) =>
          m.id === assistantMessage.id
            ? { ...m, isStreaming: false }
            : m
        )
      );
    } catch (error: any) {
      if (error.name !== "AbortError") {
        setMessages((prev) =>
          prev.map((m) =>
            m.id === assistantMessage.id
              ? {
                  ...m,
                  text: "エラーが発生しました。もう一度お試しください。",
                  isStreaming: false,
                }
              : m
          )
        );
      }
    } finally {
      setIsLoading(false);
      abortRef.current = null;
    }
  }, [messages]);

  const stopGeneration = useCallback(() => {
    abortRef.current?.abort();
  }, []);

  const clearMessages = useCallback(() => {
    setMessages([]);
  }, []);

  return { messages, isLoading, sendMessage, stopGeneration, clearMessages };
}
```

### チャットコンポーネント

```tsx
// src/components/ChatWindow.tsx
import { useRef, useEffect } from "react";
import type { Message } from "../types";

interface Props {
  messages: Message[];
}

export function ChatWindow({ messages }: Props) {
  const bottomRef = useRef<HTMLDivElement>(null);

  // 新しいメッセージが追加されたら自動スクロール
  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [messages]);

  return (
    <div className="chat-window">
      {messages.length === 0 && (
        <div className="empty-state">
          <h2>Gemini AI Chat</h2>
          <p>メッセージを入力して会話を始めましょう</p>
        </div>
      )}

      {messages.map((msg) => (
        <div key={msg.id} className={`message ${msg.role}`}>
          <div className="message-avatar">
            {msg.role === "user" ? "👤" : "✨"}
          </div>
          <div className="message-content">
            <p>{msg.text}</p>
            {msg.isStreaming && <span className="cursor-blink" />}
          </div>
        </div>
      ))}

      <div ref={bottomRef} />
    </div>
  );
}
```

### 入力フォーム

```tsx
// src/components/ChatInput.tsx
import { useState, type FormEvent } from "react";

interface Props {
  onSend: (text: string) => void;
  isLoading: boolean;
  onStop: () => void;
}

export function ChatInput({ onSend, isLoading, onStop }: Props) {
  const [input, setInput] = useState("");

  const handleSubmit = (e: FormEvent) => {
    e.preventDefault();
    if (!input.trim() || isLoading) return;
    onSend(input.trim());
    setInput("");
  };

  return (
    <form className="chat-input" onSubmit={handleSubmit}>
      <textarea
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="メッセージを入力..."
        rows={1}
        onKeyDown={(e) => {
          if (e.key === "Enter" && !e.shiftKey) {
            e.preventDefault();
            handleSubmit(e);
          }
        }}
      />
      {isLoading ? (
        <button type="button" onClick={onStop} className="stop-btn">
          停止
        </button>
      ) : (
        <button type="submit" disabled={!input.trim()}>
          送信
        </button>
      )}
    </form>
  );
}
```

## 会話履歴の永続化

ページをリロードしても会話を保持したい場合は、`sessionStorage` を活用します（ブラウザセッション単位で十分なケースが多いです）。

```typescript
// src/hooks/useChat.ts に追加
import { useEffect } from "react";

// useState の初期値を変更
const [messages, setMessages] = useState<Message[]>(() => {
  try {
    const saved = sessionStorage.getItem("chat-messages");
    return saved ? JSON.parse(saved) : [];
  } catch {
    return [];
  }
});

// messages が変わるたびに保存
useEffect(() => {
  sessionStorage.setItem("chat-messages", JSON.stringify(messages));
}, [messages]);
```

ただし、会話が長くなると `sessionStorage` の容量上限（通常 5 MB）に達する可能性があります。本番環境では IndexedDB や [Gemini API のコンテキストキャッシュ機能](/articles/gemini-api/gemini-api-context-caching-cost-optimization) を活用して、サーバーサイドで履歴を効率的に管理する設計をおすすめします。

## エラーハンドリングのベストプラクティス

本番運用を見据えたエラー処理のポイントをまとめます。

```typescript
// server/index.ts に追加するエラーハンドリング
app.use(
  (
    err: Error,
    req: express.Request,
    res: express.Response,
    next: express.NextFunction
  ) => {
    // レートリミット（429）の場合はリトライ案内
    if (err.message.includes("429")) {
      return res.status(429).json({
        error: "リクエスト数の上限に達しました。しばらく待ってから再度お試しください。",
        retryAfter: 60,
      });
    }

    // モデル過負荷（503）の場合
    if (err.message.includes("503")) {
      return res.status(503).json({
        error: "サーバーが混雑しています。時間をおいて再度お試しください。",
      });
    }

    res.status(500).json({ error: "予期しないエラーが発生しました。" });
  }
);
```

フロントエンド側では、ネットワークエラーとAPIエラーを区別して適切なメッセージを表示しましょう。セキュリティ対策の詳細については [Gemini API 本番環境セキュリティ完全ガイド](/articles/gemini-api/gemini-api-production-security-guide) で体系的に解説しています。

## よくある質問（FAQ）

### Q1: Gemini API の無料枠はどのくらいですか？

Google AI Studio 経由で取得した API キーでは、Gemini 2.5 Flash であれば 1 分あたり 30 リクエスト、1 日あたり 1,500 リクエストまで無料で利用できます（2026 年 3 月時点）。個人開発やプロトタイピングには十分な量です。最新の制限は [Google AI Studio](https://aistudio.google.com/) で確認してください。

### Q2: React 以外のフレームワーク（Vue、Svelte）でも同じアプローチは使えますか？

はい、バックエンドの Express.js プロキシはフレームワークに依存しません。フロントエンド側のカスタムフック部分を各フレームワークの状態管理パターン（Vue の Composition API、Svelte のストアなど）に置き換えるだけで同様に動作します。

### Q3: Next.js を使う場合との違いは何ですか？

Next.js を使う場合は、API Routes や Server Actions を利用してバックエンドを一体化できるため、Express.js サーバーが不要になります。一方、純粋な React + Vite の構成は、既存のバックエンドがある場合や、フロントエンドとバックエンドを別々にデプロイしたい場合に適しています。Next.js での実装に興味がある方は [Gemini API × Next.js AI チャットアプリ構築ガイド](/articles/gemini-api/gemini-api-nextjs-ai-chat-app) も参考になります。

### Q4: ストリーミングを使わない場合と比べてパフォーマンスに差はありますか？

体感速度が大きく異なります。非ストリーミングでは Gemini が全文を生成し終わるまで何も表示されませんが、ストリーミングでは最初のトークンが生成され次第リアルタイムに表示されるため、ユーザーの待ち時間が体感で数秒短縮されます。
