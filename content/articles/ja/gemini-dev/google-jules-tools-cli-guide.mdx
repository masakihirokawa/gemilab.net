## Jules Tools CLIとは？ターミナルからAIコーディングを操作する新しい方法

**Jules Tools**は、GoogleのAIコーディングエージェント「Jules」をターミナルから直接操作するための公式コマンドラインインターフェース（CLI）です。2026年3月にリリースされたこのツールにより、開発者はブラウザを開くことなく、普段の開発ワークフローの中からJulesにタスクを委任できるようになりました。

Jules自体は、Gemini 2.5 Proの高度な思考能力を活用して、バグ修正、テスト作成、リファクタリング、依存関係の更新といったコーディングタスクを非同期で自動実行するエージェントです。Jules Toolsはそのダッシュボードとコマンドサーフェスの役割を果たし、以下のような操作をターミナルから可能にします。

- リモートセッションの作成と管理
- タスクの進捗確認とプル
- 複数タスクの並列実行
- シェルスクリプトやCI/CDパイプラインとの統合

## インストール方法

Jules ToolsはnpmパッケージとしてGoogleから公式に提供されています。Node.js環境があれば、以下のコマンドで即座にインストールできます。

### グローバルインストール

```bash
# npmでグローバルインストール
npm install -g @google/jules

# インストール確認
jules --version
```

### npxで一時実行

インストールせずに試したい場合は、`npx`を使って一時的に実行することもできます。

```bash
# npxで即座に実行（インストール不要）
npx @google/jules remote list --repo
```

### 初回認証

初回実行時にGoogleアカウントでの認証が求められます。認証後、Julesに接続済みのGitHubリポジトリに対してコマンドを実行できるようになります。

```bash
# 認証フローを開始
jules auth login

# 認証状態の確認
jules auth status
```

## 基本コマンド一覧

Jules Tools CLIのコマンド体系は直感的に設計されています。主要なコマンドを見ていきましょう。

### `jules remote list` — リポジトリとセッションの一覧

接続済みのリポジトリや実行中のセッションを確認するコマンドです。

```bash
# 接続済みリポジトリの一覧を表示
jules remote list --repo

# 実行結果の例:
# ┌─────────────────────────────────┐
# │ Connected Repositories          │
# ├─────────────────────────────────┤
# │ masakihirokawa/my-app          │
# │ masakihirokawa/api-server      │
# └─────────────────────────────────┘

# アクティブなセッション一覧を表示
jules remote list --session

# 実行結果の例:
# ┌──────────┬──────────────────────┬──────────┐
# │ ID       │ Task                 │ Status   │
# ├──────────┼──────────────────────┼──────────┤
# │ 123456   │ Fix login bug        │ Running  │
# │ 123457   │ Add unit tests       │ Complete │
# └──────────┴──────────────────────┴──────────┘
```

### `jules new` — 新しいタスクの作成

Julesに新しいコーディングタスクを委任するコマンドです。カレントディレクトリのリポジトリを自動検出するため、多くの場合 `--repo` フラグは省略できます。

```bash
# カレントディレクトリのリポジトリでタスクを作成
jules new "ユーザー認証のユニットテストを追加して"

# 特定のリポジトリを指定してタスクを作成
jules new --repo masakihirokawa/my-app "READMEを英語に翻訳して"

# 実行結果の例:
# ✓ Session created: #128901
# ℹ Jules is working on: "ユーザー認証のユニットテストを追加して"
# ℹ Repository: masakihirokawa/my-app
# ℹ Track progress: jules remote list --session
```

### `jules remote pull` — 完了したタスクの結果を取得

Julesが完了したタスクの成果物（コード変更）をローカルに取り込みます。

```bash
# セッションIDを指定して結果をプル
jules remote pull --session 128901

# 実行結果の例:
# ✓ Changes pulled from session #128901
# ℹ Modified files:
#   - src/auth/__tests__/login.test.ts (+142 lines)
#   - src/auth/__tests__/signup.test.ts (+98 lines)
```

## 並列実行で生産性を最大化する

Jules Toolsの強力な機能の一つが、複数タスクの**並列実行**です。`--parallel` フラグを使うことで、同じタスクを複数のアプローチで同時に試行させたり、異なるタスクを一括で投入することができます。

### 同一タスクの並列実行

```bash
# 3つの並列セッションで同じタスクに取り組む
jules new --parallel 3 "パフォーマンスのボトルネックを特定して最適化案を提示して"

# 実行結果の例:
# ✓ Created 3 parallel sessions: #128902, #128903, #128904
# ℹ Each session will independently work on the task
```

### バッチタスクの一括投入

TODO.mdやissueリストから複数タスクを自動で投入するスクリプトも簡単に作成できます。

```bash
# TODO.mdの各行をタスクとしてJulesに投入
cat TODO.md | while IFS= read -r line; do
  jules new "$line"
  sleep 2  # APIレート制限を考慮
done

# 実行結果の例:
# ✓ Session created: #128905 — "Fix CORS issue in API"
# ✓ Session created: #128906 — "Add pagination to user list"
# ✓ Session created: #128907 — "Update deprecated dependencies"
```

## 実践的なユースケース：CI/CDパイプラインとの統合

Jules Toolsはスクリプタブルに設計されているため、GitHub ActionsなどのCI/CDパイプラインに組み込むことで、開発プロセスをさらに自動化できます。

### GitHub Actionsでの自動テスト生成

```yaml
# .github/workflows/jules-auto-test.yml
name: Jules Auto Test Generation
on:
  pull_request:
    types: [opened]

jobs:
  generate-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Jules Tools
        run: npm install -g @google/jules

      - name: Authenticate with Jules
        run: jules auth login --token ${{ secrets.JULES_API_TOKEN }}

      - name: Generate tests for changed files
        run: |
          CHANGED=$(git diff --name-only origin/main...HEAD | grep -E '\.(ts|js)$')
          if [ -n "$CHANGED" ]; then
            jules new "以下のファイルのユニットテストを作成: $CHANGED"
          fi
```

### TUI（テキストユーザーインターフェース）モード

インタラクティブな操作を好む開発者のために、Jules ToolsにはTUIモードも用意されています。

```bash
# TUIモードでダッシュボードを表示
jules remote

# TUIモードで新しいタスクを対話的に作成
jules /new
```

TUIモードでは、リポジトリの選択やタスクの記述を対話形式で進められるため、CLIに不慣れな場合でも直感的に操作できます。

## Gemini CLIとの違い

Jules ToolsとGemini CLIは似て非なるツールです。どちらもターミナルベースですが、目的と動作が異なります。

| 比較項目 | Jules Tools | Gemini CLI |
|---------|------------|-----------|
| 目的 | タスクの非同期委任 | リアルタイム対話型コーディング |
| 実行場所 | Googleクラウド上のVM | ローカルマシン |
| 処理方式 | 非同期（バックグラウンド） | 同期（リアルタイム） |
| 成果物 | Pull Request / パッチ | 直接のファイル変更 |
| 適したユースケース | バグ修正、テスト生成、リファクタリング | コード探索、デバッグ、対話的開発 |

Gemini CLIについて詳しくは[Gemini CLI完全ガイド](/articles/gemini-dev/gemini-cli-guide)を参照してください。

## よくある質問（FAQ）

### Q: Jules Toolsを使うにはGitHubリポジトリの接続が必須ですか？

はい、JulesはGitHubリポジトリと連携して動作するため、事前にJulesのWebインターフェース（jules.google.com）からリポジトリを接続しておく必要があります。接続後はCLIからすべての操作が可能です。

### Q: セッションが完了したかどうかをどう確認しますか？

`jules remote list --session` で全セッションのステータスを確認できます。ステータスは「Running」「Complete」「Failed」のいずれかで表示されます。また、Julesの設定でメール通知やWebhookを有効にすることもできます。

### Q: ローカルの未コミットな変更はJulesのセッションに反映されますか？

いいえ、JulesはGitHub上のリポジトリ（リモート）を基に作業します。ローカルの変更を反映させたい場合は、先にコミット＆プッシュしてからJulesのタスクを作成してください。

### Q: Julesが生成したコードのレビュー機能はありますか？

はい、Julesには「**Adversarial Review**」と呼ばれる自動レビュー機能が組み込まれています。提案するすべてのコード変更は、Julesの内部批評システムによる査読を経てから開発者に提示されるため、品質の高いアウトプットが期待できます。

### Q: プライベートリポジトリでも利用できますか？

はい、Jules Toolsはプライベートリポジトリでも問題なく利用可能です。GitHubとの接続時に適切な権限を付与するだけで、プライベートリポジトリのコードにアクセスしてタスクを実行できます。
