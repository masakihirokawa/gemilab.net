## Google AI Studioが「開発環境」になった — ビルドモードの衝撃

2026年3月20日、GoogleはGoogle AI Studioに**フルスタックバイブコーディング機能**を追加しました。これまでプロンプトの実験場だったAI Studioが、プロンプト入力からアプリのビルド・プレビュー・デプロイまでをワンストップで行える開発環境に進化しています。

すでに[フルスタックバイブコーディングの概要](/articles/gemini-dev/google-ai-studio-fullstack-vibe-coding)は別記事で解説していますが、本記事では**実際に手を動かして試す**ことにフォーカスします。初めてビルドモードを触る方が、この記事だけでアプリを1つ生成してFirebaseと連携できるようになることを目指します。

## ビルドモードへのアクセスと基本操作

### ビルドモードを開く

Google AI Studio（[aistudio.google.com](https://aistudio.google.com)）にアクセスし、左側のメニューから「**Build**」を選択します。Geminiモデル（2.5 Pro推奨）が自動で選択され、プロンプト入力欄とライブプレビューパネルが表示されます。

### 最初のアプリを生成する

プロンプト欄に自然言語でアプリの要件を入力します。Geminiが要件を解釈し、React（またはAngular / Next.js）ベースのコードを自動生成します。

```
プロンプト例:
「読書管理アプリを作って。本のタイトル・著者・読了日を登録できて、
月ごとの読了数をグラフで表示してほしい。デザインはシンプルで落ち着いた配色で。」
```

数秒〜数十秒で右側のプレビューパネルにアプリが表示されます。この時点では静的なフロントエンドのみですが、すでにUIコンポーネント・状態管理・レスポンシブレイアウトが完成しています。

### コードの確認と手動修正

プレビューの下にある「**Code**」タブを開くと、生成されたソースコードを確認・編集できます。手動で修正を加えた後、再度プロンプトで「ボタンの色を青に変えて」のように対話的に調整することも可能です。

## Firebase自動連携 — データベースと認証をプロンプトで構築

### Antigravityエージェントの自動検出

ビルドモードの最大の特徴は、**Antigravityコーディングエージェント**がアプリの要件を分析し、バックエンドが必要な場合にFirebase連携を自動提案してくれることです。

例えば先ほどの読書管理アプリに対して、以下のようなプロンプトを追加してみます。

```
「データをユーザーごとに保存できるようにして。Googleアカウントでログインさせて。」
```

するとAntigravityエージェントが以下を自動的に検出・提案します。

1. **Cloud Firestore** — 読書データの永続化に必要
2. **Firebase Authentication** — Googleログイン機能に必要

ユーザーが「Firebase連携を承認」ボタンをクリックすると、以下が自動で実行されます。

```javascript
// Antigravityが自動生成するFirebase設定（参考）
import { initializeApp } from "firebase/app";
import { getFirestore, collection, addDoc, getDocs } from "firebase/firestore";
import { getAuth, signInWithPopup, GoogleAuthProvider } from "firebase/auth";

const firebaseConfig = {
  // AI Studioが自動でプロジェクトを作成し、設定値を埋める
  apiKey: "自動設定",
  authDomain: "自動設定",
  projectId: "自動設定",
};

const app = initializeApp(firebaseConfig);
const db = getFirestore(app);
const auth = getAuth(app);

// 本を追加する関数
export async function addBook(userId, book) {
  const docRef = await addDoc(collection(db, `users/${userId}/books`), {
    title: book.title,
    author: book.author,
    finishedDate: book.finishedDate,
    createdAt: new Date(),
  });
  return docRef.id;
}

// 期待する動作:
// addBook("user123", { title: "深層学習", author: "Ian Goodfellow", finishedDate: "2026-03-15" })
// → Firestoreの users/user123/books コレクションにドキュメントが追加される
```

### 自動プロビジョニングの裏側

Firebase連携を承認すると、裏側で以下が自動的に処理されます。

- Firebaseプロジェクトの新規作成（既存プロジェクトの選択も可能）
- Cloud Firestoreの有効化とセキュリティルールの初期設定
- Firebase Authenticationの有効化とGoogleプロバイダーの設定
- アプリコードへのFirebase SDK統合

従来であれば、Firebase Consoleで手動設定する必要があった工程が完全に自動化されています。

## 実践：タスク管理アプリを30分で作る

ここからは、ビルドモードで実用的なアプリを作る手順を具体的に示します。

### ステップ1: 要件をプロンプトで伝える

```
「チーム向けタスク管理アプリを作って。
- タスクのタイトル、担当者、期限、ステータス（未着手/進行中/完了）を管理
- カンバンボード形式で表示
- ドラッグ&ドロップでステータス変更
- リアルタイムで複数ユーザーが同時に使える
- Googleアカウントでログイン」
```

### ステップ2: Firebase連携を承認

Antigravityエージェントが「Firestore（リアルタイムDB）とAuthentication（Googleログイン）が必要です」と提案するので、承認します。

### ステップ3: 対話的に調整

```
「カンバンのカラムにタスク数を表示して」
「期限が今日より前のタスクは赤くハイライトして」
「担当者でフィルタリングできるドロップダウンを追加して」
```

### ステップ4: デプロイ

完成したら「Deploy」ボタンでGoogle Antigravityにデプロイされ、URLが発行されます。チームメンバーにURLを共有するだけで利用開始できます。

## Gemini APIを直接使う場合との違い

ビルドモードは「対話的にアプリを組み立てる」体験です。一方、[Gemini APIをコードから呼び出す](/articles/gemini-api/gemini-api-quickstart)方法は、既存プロジェクトにAI機能を組み込みたい場合に適しています。

| 観点 | ビルドモード | Gemini API直接利用 |
|------|------------|-------------------|
| 対象者 | プロトタイプ作成、ノーコード志向 | 既存プロジェクトにAI統合 |
| 出力 | 完全なアプリケーション | APIレスポンス（テキスト・画像等） |
| カスタマイズ | プロンプト + コード編集 | 完全なコード制御 |
| Firebase連携 | 自動プロビジョニング | 手動設定 |
| デプロイ | ワンクリック（Antigravity） | 自前のインフラ |

## よくある質問（FAQ）

### Q: ビルドモードは無料で使えますか？

Google AI Studioの無料枠内で利用可能です。ただし、Firebase連携時にはFirebaseの無料枠（Sparkプラン）が適用されます。大量のFirestore読み書きやAuthenticationリクエストが発生する場合は、Blazeプラン（従量課金）へのアップグレードが必要になる場合があります。

### Q: 生成されたコードをローカルにダウンロードできますか？

はい。ビルドモードの「Code」タブからソースコード全体をエクスポートできます。ローカル環境で`npm install && npm run dev`すれば、そのまま開発を継続できます。Antigravityにデプロイしたアプリも、Gitリポジトリとしてクローン可能です。

### Q: React以外のフレームワークは使えますか？

現時点でReact、Angular、Next.jsに対応しています。プロンプトで「Angularで作って」と指定すれば、該当フレームワークでコードが生成されます。Vue.jsやSvelteへの対応も今後のアップデートで予定されているとのことです。

### Q: 既存のFirebaseプロジェクトと連携できますか？

はい。Firebase連携の承認画面で「既存プロジェクトを選択」を選べば、すでに作成済みのFirebaseプロジェクトとリンクできます。既存のFirestoreデータやAuthenticationの設定をそのまま活用可能です。

## まとめ — 「アイデア→動くアプリ」の距離がゼロに近づいた

Google AI Studioのビルドモードは、Geminiの言語理解力とAntigravityのコーディングエージェント、Firebaseのバックエンドインフラを組み合わせた、フルスタックアプリ生成の統合環境です。

プロンプトを入力するだけでUIが生成され、「データを保存したい」と言えばFirestoreが自動構築され、「ログイン機能がほしい」と言えばFirebase Authが設定される。この一連の体験は、従来のアプリ開発プロセスを大幅に短縮します。

まずはシンプルなアプリから試してみて、ビルドモードの可能性を体感してみてください。より高度なGemini APIの活用については、[Gemini APIクイックスタート](/articles/gemini-api/gemini-api-quickstart)もあわせてご覧ください。
