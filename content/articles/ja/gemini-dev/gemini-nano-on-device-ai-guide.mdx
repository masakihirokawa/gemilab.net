## Gemini Nano とは

Gemini Nano は、Google が開発した Gemini ファミリーの中で最も軽量なモデルです。クラウドにリクエストを送ることなく、スマートフォンや PC のローカル環境で直接 AI 推論を実行できます。

Gemini Pro や Flash がクラウドベースで高い処理能力を発揮するのに対して、Nano はデバイス上で完結する設計になっており、ネットワーク接続が不要、レイテンシが極めて低い、そしてユーザーのデータがデバイスから外に出ないという特徴を持ちます。

## オンデバイスAI が注目される理由

従来のクラウドベース AI には、いくつかの制約がありました。ネットワーク遅延による応答の遅さ、API コールごとに発生するコスト、そしてセンシティブなデータをクラウドに送信するプライバシーリスクです。

オンデバイス AI はこれらの課題を根本から解決します。推論がローカルで完結するため、飛行機の中やトンネル内などオフライン環境でも利用でき、API 利用料も発生しません。医療情報や個人的なメモなど、外部に送りたくないデータも安全に処理できます。

## Gemini Nano の主な搭載先

### Android デバイス

Google Pixel 8 Pro 以降の Pixel シリーズには Gemini Nano が標準搭載されています。Samsung Galaxy S24 シリーズも Gemini Nano を活用した AI 機能を提供しています。Android の AICore サービスを通じて、サードパーティアプリからも Gemini Nano の機能にアクセスできます。

### Google Chrome

Chrome 126 以降では、ブラウザに Gemini Nano が組み込まれています。Prompt API や Summarization API などの Built-in AI API を通じて、ウェブアプリケーションからオンデバイス AI を利用できます。これにより、ウェブ開発者はサーバー側の AI インフラを用意しなくても、ブラウザ内で AI 機能を提供できるようになりました。

## Chrome Built-in AI API の使い方

Chrome に搭載された Gemini Nano は、JavaScript API を通じて利用できます。現在提供されている主な API を見ていきましょう。

### Prompt API

汎用的なテキスト生成 API です。ユーザーの入力に対して、デバイス上で即座に応答を生成できます。

```javascript
// セッションの作成
const session = await ai.languageModel.create({
  systemPrompt: "あなたは親切なアシスタントです。簡潔に回答してください。"
});

// テキスト生成
const response = await session.prompt("TypeScriptの型推論について教えて");
console.log(response);

// ストリーミング応答
const stream = await session.promptStreaming("Reactのベストプラクティスを3つ挙げて");
for await (const chunk of stream) {
  console.log(chunk);
}
```

### Summarization API

長文テキストの要約をデバイス上で実行します。記事やドキュメントの概要を瞬時に生成でき、ネットワークを経由しないため高速です。

```javascript
const summarizer = await ai.summarizer.create({
  type: "tl;dr",
  length: "short",
  format: "plain-text"
});

const summary = await summarizer.summarize(longArticleText);
console.log(summary);
```

### Writer API と Rewriter API

Writer API は指定したコンテキストに基づいてテキストを生成し、Rewriter API は既存のテキストを別のトーンや形式に書き換えます。

```javascript
// Writer: コンテキストに基づくテキスト生成
const writer = await ai.writer.create({
  tone: "formal",
  length: "medium"
});
const draft = await writer.write("プロジェクトの進捗報告メールを作成");

// Rewriter: テキストの書き換え
const rewriter = await ai.rewriter.create({
  tone: "more-casual"
});
const rewritten = await rewriter.rewrite(formalEmail);
```

## Android での Gemini Nano 活用

Android アプリ開発者は、Google AI Edge SDK を使って Gemini Nano をアプリに組み込めます。

### セットアップ

```kotlin
// build.gradle.kts
dependencies {
    implementation("com.google.ai.edge.litert:litert:1.0.0")
    implementation("com.google.android.gms:play-services-aicore:16.0.0")
}
```

### テキスト生成の実装

```kotlin
import com.google.android.gms.aicore.GenerativeAIClient

val client = GenerativeAIClient.create(context)

// テキスト生成リクエスト
val request = GenerateTextRequest.newBuilder()
    .setPrompt("この文章を要約してください: $inputText")
    .build()

val response = client.generateText(request)
println(response.text)
```

### Smart Reply の実装

Gemini Nano の代表的なユースケースの一つが、メッセージへのスマートリプライ候補の生成です。会話の文脈を理解して、適切な返信候補をリアルタイムで提案します。

```kotlin
val smartReplyClient = SmartReplyClient.create(context)

val conversation = listOf(
    Message("元気？明日の予定ある？", isLocalUser = false),
    Message("元気だよ！明日は空いてるけど", isLocalUser = true),
    Message("じゃあランチ行かない？", isLocalUser = false)
)

val suggestions = smartReplyClient.suggestReplies(conversation)
// → ["いいね！行こう", "何時がいい？", "どこで食べる？"]
```

## Gemini Nano の制約と使い分け

Gemini Nano はすべての用途に適しているわけではありません。モデルサイズが小さい分、複雑な推論や長文生成では Gemini Pro や Flash に劣ります。

オンデバイス AI が適しているケースとしては、リアルタイムの入力補完やスマートリプライ、テキストの短い要約、センシティブデータの分類やフィルタリング、オフライン環境での基本的な AI 機能などが挙げられます。

一方で、クラウド AI を使うべきケースとしては、複雑なコード生成やデバッグ、長文の分析レポート作成、マルチモーダル入力（画像・動画・音声）の高度な処理、最新情報へのアクセスが必要な質問応答などがあります。

実際のアプリケーションでは、まず Gemini Nano でローカル処理を試み、モデルの能力を超えるタスクについてはクラウドの Gemini API にフォールバックするハイブリッドアプローチが推奨されます。

## 今後の展望

Google は Gemini Nano の機能を継続的に拡張しています。マルチモーダル対応（画像理解）の強化、より多くのデバイスへの搭載拡大、Chrome Built-in AI API の正式リリースと API の追加、そしてサードパーティ開発者向けのカスタマイズ機能などが進行中です。

オンデバイス AI は、プライバシーファーストの時代において重要性を増しています。クラウドとエッジの両方を適切に使い分けることで、ユーザー体験を大幅に向上させることができるでしょう。
