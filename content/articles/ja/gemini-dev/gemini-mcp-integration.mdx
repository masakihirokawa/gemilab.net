## Gemini × MCP サーバー連携ガイド

Gemini APIの能力をさらに高める方法として、**Model Context Protocol（MCP）**によるサーバー連携があります。MCPサーバーを活用することで、外部のツール、データベース、APIを直接Geminiから利用できるようになります。本ガイドでは、MCPサーバーの基本から実装方法まで、実践的な内容を解説します。

## Model Context Protocol (MCP) とは

MCPは、AIアシスタントと外部ツールやデータソースの間に標準化されたインターフェースを提供するプロトコルです。このプロトコルを使用することで、Geminiが直接外部リソースにアクセスし、より複雑なタスクを自動化できるようになります。

MCPサーバーの主な特徴：
- **標準化された通信**: JSONベースのプロトコルにより、異なるツール間での通信が統一される
- **動的なツール発見**: サーバーが提供するツールをクライアント側で自動検出可能
- **セキュリティ**: ローカルサーバーを使用することで、APIキーやセンシティブデータの保護が強化される

## MCPサーバーの構造と実装

MCPサーバーを構築するには、以下の基本要素を理解する必要があります。

### MCPサーバーの基本構成

MCPサーバーは通常、以下の3つの主要なメソッドを実装します：

1. **Tool Definition**: サーバーが提供するツールの定義
2. **Tool Execution**: ツールの実際の実行処理
3. **Result Handling**: 結果の適切なフォーマット化と返却

### 実装例：天気データMCPサーバー

以下は、外部天気APIと連携するシンプルなMCPサーバーの実装例です：

```python
# mcp_weather_server.py
import json
import requests
from typing import Any, Dict

class WeatherMCPServer:
    def __init__(self):
        self.tools = {
            "get_weather": {
                "description": "指定した都市の天気情報を取得します",
                "inputSchema": {
                    "type": "object",
                    "properties": {
                        "city": {
                            "type": "string",
                            "description": "都市名"
                        }
                    },
                    "required": ["city"]
                }
            }
        }

    def get_weather(self, city: str) -> Dict[str, Any]:
        """外部APIから天気情報を取得"""
        try:
            # 実際の天気APIを呼び出し（例：OpenWeatherMap）
            response = requests.get(
                f"https://api.openweathermap.org/data/2.5/weather?q={city}&appid=YOUR_API_KEY"
            )
            data = response.json()

            return {
                "success": True,
                "city": data["name"],
                "temperature": data["main"]["temp"],
                "description": data["weather"][0]["main"],
                "humidity": data["main"]["humidity"],
                "wind_speed": data["wind"]["speed"]
            }
        except Exception as e:
            return {
                "success": False,
                "error": str(e)
            }

    def handle_tool_call(self, tool_name: str, parameters: Dict[str, Any]) -> Dict[str, Any]:
        """ツール呼び出しを処理"""
        if tool_name == "get_weather":
            return self.get_weather(parameters["city"])
        else:
            return {"error": f"Unknown tool: {tool_name}"}

# 予期される出力例：
# {
#     "success": true,
#     "city": "Tokyo",
#     "temperature": 18.5,
#     "description": "Cloudy",
#     "humidity": 65,
#     "wind_speed": 3.2
# }
```

## GeminiとMCPサーバーの連携

Gemini APIでMCPサーバーを活用するには、まずサーバーとの接続を確立し、利用可能なツール情報をGeminiに提供します。

### ステップ1：MCPサーバーの起動

MCPサーバーをローカルまたはリモートで起動します：

```bash
# MCPサーバーをスタンドアロンで実行
python -m mcp run weather_server --port 8000
```

### ステップ2：Gemini APIからの連携

[Gemini API](/articles/gemini-api/gemini-api-quickstart) を使用して、MCPサーバーから取得したツール情報をGeminiに渡します：

```python
import anthropic
import requests

client = anthropic.Anthropic()

# MCPサーバーからツール定義を取得
def fetch_tools_from_mcp():
    response = requests.get("http://localhost:8000/tools")
    return response.json()

# MCPサーバーでツール実行
def execute_tool_on_mcp(tool_name: str, parameters: dict) -> str:
    response = requests.post(
        "http://localhost:8000/execute",
        json={"tool": tool_name, "parameters": parameters}
    )
    return json.dumps(response.json())

# Geminiとの対話
mcp_tools = fetch_tools_from_mcp()

messages = [
    {"role": "user", "content": "東京の天気を教えてください"}
]

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    tools=mcp_tools,
    messages=messages
)

# ツール呼び出しがある場合は実行
while response.stop_reason == "tool_use":
    tool_use = next(block for block in response.content if block.type == "tool_use")
    tool_result = execute_tool_on_mcp(tool_use.name, tool_use.input)

    messages.append({"role": "assistant", "content": response.content})
    messages.append({
        "role": "user",
        "content": [
            {
                "type": "tool_result",
                "tool_use_id": tool_use.id,
                "content": tool_result
            }
        ]
    })

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=1024,
        tools=mcp_tools,
        messages=messages
    )

# 最終的な応答を取得
final_text = next(
    (block.text for block in response.content if hasattr(block, "text")),
    ""
)
print(final_text)
```

### ファイルシステムアクセス

特定ディレクトリのファイル操作を制御付きで実行：

```python
class FileSystemMCPServer:
    def __init__(self, allowed_paths: List[str]):
        self.allowed_paths = allowed_paths

    def read_file(self, file_path: str) -> Dict[str, Any]:
        """許可されたパスのみでファイル読み込み"""
        if self.is_allowed_path(file_path):
            with open(file_path, 'r') as f:
                return {"success": True, "content": f.read()}
        return {"success": False, "error": "Access denied"}
```

## セキュリティのベストプラクティス

### 認証と認可

MCPサーバーへのアクセスは、適切な認証メカニズムで保護してください：

- **API キーベース認証**: 本番環境ではJWT等の標準化されたトークンを使用
- **ロールベースアクセス制御 (RBAC)**: ユーザーの役割に応じた権限管理
- **リクエスト検証**: すべての入力値をサーバー側で厳密に検証

### データ保護

- **暗号化**: 機密情報は転送中と保存時に暗号化
- **監査ログ**: すべてのツール実行をログに記録
- **レート制限**: 過度なリクエストを防止するため、レート制限を実装

## よくある質問

### Q1: MCPサーバーの開発に必要なスキルは？

A: MCPサーバーの開発には、対象とする言語（Python、Node.jsなど）の基本的なスキルと、RESTful APIの理解が必要です。Geminiとの連携部分については、公式ドキュメントのサンプルコードを参考にすれば、初心者でも実装可能です。

### Q2: 複数のMCPサーバーを同時に使用できますか？

A: はい、複数のMCPサーバーを同時に使用できます。Gemini APIに複数のサーバーからのツール定義を渡すことで、すべてのツールにアクセス可能になります。ただし、ツール名の重複を避けるため、適切な命名規則を使用してください。

### Q3: MCPサーバーのデプロイメント方法は？

A: MCPサーバーは、Docker コンテナとして、またはAWS Lambda、Google Cloud Functions などのサーバーレスプラットフォーム上でデプロイできます。本番環境では、スケーラビリティと高可用性のため、Kubernetes等のオーケストレーションツールの使用を推奨します。
