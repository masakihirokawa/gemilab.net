## Google AI Studio が「フルスタック IDE」に生まれ変わった

2026年3月18日、Google は AI Studio に大規模なアップデートを実施し、テキストプロンプトから本番対応のアプリケーションを構築できる **フルスタックバイブコーディング環境** を発表しました。これまでの「プロンプトを試す場所」から、「プロンプトでアプリを作る場所」へと大きく進化したことになります。

Google 社内でも数十万のアプリがこのプラットフォーム上で構築されており、その実用性はすでに実証されています。この記事では、新しく追加された機能の詳細と、実際にどのように使うのかを解説します。

## バイブコーディングとは

バイブコーディング（Vibe Coding）とは、厳密なコードを手書きする代わりに、自然言語の指示だけで AI にアプリケーションを生成させる開発手法です。「こういうアプリが欲しい」という雰囲気（バイブ）を伝えるだけで、AI がコード全体を書き上げてくれます。

従来のバイブコーディングツールはフロントエンドの生成に留まることが多かったのですが、Google AI Studio の新バージョンではバックエンド、データベース、認証まで含めたフルスタックの構築が可能になりました。

## 新機能の全体像

### Antigravity エージェント

今回のアップデートの核となるのが **Antigravity** というコーディングエージェントです。Antigravity は2025年11月にスタンドアロンの VS Code ベース IDE としてリリースされていましたが、今回ブラウザベースの AI Studio に直接統合されました。

Antigravity が従来のコード生成と異なるのは、**プロジェクト全体の構造を理解** した上でコードを書くという点です。複数のファイルを管理し、複雑な指示を解釈して、堅牢なフルスタックアプリケーションを構築します。

```python
# Antigravity が自動で処理する範囲の例
# 1. フロントエンド（React / Next.js）
# 2. バックエンド（API ルート）
# 3. データベース（Cloud Firestore）
# 4. 認証（Firebase Authentication）
# 5. 外部ライブラリのインストール

# ユーザーがやること：プロンプトを書くだけ
prompt = """
ToDoアプリを作ってください。
- Googleアカウントでログインできるようにする
- タスクをデータベースに保存する
- リアルタイムで同期される
- Framer Motion でアニメーションを付ける
"""

# Antigravity がこのプロンプトから
# フルスタックアプリを自動生成
```

### Firebase 統合

Antigravity はアプリの要件を自動検出し、必要に応じて Firebase の統合を提案します。

データベースが必要だと判断すると **Cloud Firestore** を、ログイン機能が必要だと判断すると **Firebase Authentication** を自動でプロビジョニングします。開発者が Firebase のセットアップを手動で行う必要はありません。

```javascript
// Antigravity が自動生成するコードの例
import { initializeApp } from "firebase/app";
import { getFirestore, collection, addDoc } from "firebase/firestore";
import { getAuth, signInWithPopup, GoogleAuthProvider } from "firebase/auth";

// Firebase の設定は Antigravity が自動で注入
const app = initializeApp(firebaseConfig);
const db = getFirestore(app);
const auth = getAuth(app);

// Google ログインの実装
const signIn = async () => {
  const provider = new GoogleAuthProvider();
  const result = await signInWithPopup(auth, provider);
  console.log("Logged in as:", result.user.displayName);
  // 期待する出力: "Logged in as: ユーザー名"
};

// Firestore へのデータ保存
const addTask = async (task) => {
  await addDoc(collection(db, "tasks"), {
    text: task,
    createdAt: new Date(),
    userId: auth.currentUser.uid,
  });
};
```

### マルチプレイヤー対応

リアルタイムのマルチプレイヤー体験を構築できるようになりました。複数のユーザーが同時に接続するゲーム、共同作業ワークスペース、共有ツールなどを、プロンプトひとつで生成できます。

### Next.js サポート

これまで React と Angular のみがサポートされていましたが、新たに **Next.js** フレームワークが追加されました。サーバーサイドレンダリング（SSR）、App Router、API Routes といった Next.js の機能をフル活用したアプリケーションを生成できます。

### シークレットマネージャー

外部サービスの API キーを安全に管理するための **Secrets Manager** が追加されました。決済プロセッサ（Stripe など）、Google Maps、その他のサードパーティ API との連携時に、キーをコードにハードコードすることなく安全に使用できます。

### プロジェクトの永続化

ブラウザを閉じてもプロジェクトの状態が自動保存されるようになりました。別のデバイスからアクセスしても、作業を中断したところから再開できます。

## 実際に使ってみる

Google AI Studio でバイブコーディングを始める手順は非常にシンプルです。

```bash
# ステップ 1: Google AI Studio にアクセス
# https://aistudio.google.com にブラウザでアクセス

# ステップ 2: 「Build」モードを選択
# 画面上部のナビゲーションから Build を選択

# ステップ 3: プロンプトを入力
# 作りたいアプリを自然言語で記述

# ステップ 4: 生成されたアプリをプレビュー
# リアルタイムでアプリが生成される

# ステップ 5: 必要に応じて修正指示を追加
# 「ボタンの色を青にして」「ダークモードを追加して」など
```

Antigravity はサードパーティライブラリ（Framer Motion、Shadcn など）を自動検出してインストールしてくれるため、依存関係の管理を手動で行う必要がありません。

## 従来のバイブコーディングツールとの比較

| 機能 | Google AI Studio | 他のツール |
|------|-----------------|-----------|
| フロントエンド生成 | React / Next.js / Angular | React 中心 |
| バックエンド | Firebase 自動統合 | 手動設定が必要 |
| データベース | Cloud Firestore 自動構築 | 外部サービス連携 |
| 認証 | Firebase Auth 自動統合 | 手動実装 |
| マルチプレイヤー | ネイティブ対応 | 非対応が多い |
| 外部ライブラリ | 自動検出＆インストール | 手動追加 |
| プロジェクト永続化 | 自動保存・デバイス間対応 | ツールによる |
| 料金 | 無料（Google アカウントのみ） | 有料プランが一般的 |

## よくある質問

### Q: Google AI Studio のバイブコーディングは無料で使えますか？

A: はい。Google AI Studio は Google アカウントがあれば無料で利用できます。Firebase の利用量が無料枠を超える場合は別途料金が発生しますが、開発・テスト段階では十分な無料枠が提供されています。

### Q: 生成されたコードは商用利用できますか？

A: はい。Google AI Studio で生成されたコードの著作権は開発者に帰属し、商用利用も可能です。ただし、使用するサードパーティライブラリのライセンスは個別に確認してください。

### Q: Antigravity エージェントは Gemini のどのモデルを使用していますか？

A: Antigravity は Gemini 2.5 Pro をベースにしており、最新のコーディング能力を備えています。モデルのアップデートに伴い、生成されるコードの品質も継続的に向上しています。

### Q: 既存のプロジェクトをインポートして改修することはできますか？

A: 現時点では新規プロジェクトの生成が主な用途ですが、生成されたコードをダウンロードしてローカルで編集したり、追加のプロンプトで機能を拡張したりすることは可能です。
