## Gemini と Firebase を組み合わせる理由

Gemini API 単体でもテキスト生成や画像分析が可能ですが、実際のアプリケーションではユーザー管理、データの永続化、サーバーサイド処理が必要です。Firebase はこれらの機能をフルマネージドで提供し、Gemini と組み合わせることで本格的な AI アプリを短期間で構築できます。

この記事では、Firebase の Firestore（データベース）と Cloud Functions（サーバーレス関数）を使い、Gemini API を安全に呼び出すバックエンドを構築する方法を解説します。

## Cloud Functions から Gemini API を呼び出す

### API キーの安全な管理

Gemini API キーをコード内にハードコードしてはいけません。Firebase の環境変数機能を使って安全に管理します。

```bash
# API キーをシークレットとして設定
firebase functions:secrets:set GEMINI_API_KEY
# プロンプトが表示されたらキーを入力
```

### 基本的な Cloud Function の実装

```typescript
// functions/src/index.ts
// Gemini API を Cloud Functions から呼び出す基本実装

import { onCall, HttpsError } from "firebase-functions/v2/https";
import { defineSecret } from "firebase-functions/params";
import { GoogleGenerativeAI } from "@google/generative-ai";

// シークレットとして API キーを定義
const geminiApiKey = defineSecret("GEMINI_API_KEY");

export const generateText = onCall(
  { secrets: [geminiApiKey] },
  async (request) => {
    // 認証チェック
    if (!request.auth) {
      throw new HttpsError(
        "unauthenticated",
        "この機能を使うにはログインが必要です"
      );
    }

    const { prompt } = request.data;

    if (!prompt || typeof prompt !== "string") {
      throw new HttpsError(
        "invalid-argument",
        "プロンプトは必須です"
      );
    }

    // 入力文字数の制限（安全対策）
    if (prompt.length > 2000) {
      throw new HttpsError(
        "invalid-argument",
        "プロンプトは2000文字以内にしてください"
      );
    }

    try {
      const genAI = new GoogleGenerativeAI(geminiApiKey.value());
      const model = genAI.getGenerativeModel({
        model: "gemini-2.0-flash",
      });

      const result = await model.generateContent(prompt);
      const text = result.response.text();

      return { text, tokenCount: text.length };
    } catch (error) {
      console.error("Gemini API error:", error);
      throw new HttpsError(
        "internal",
        "テキスト生成に失敗しました"
      );
    }
  }
);
```

## Firestore に会話履歴を保存する

AI チャットアプリでは、過去の会話履歴を保存して文脈を維持することが重要です。Firestore を使えばリアルタイムでデータを同期できます。

```typescript
// functions/src/chat.ts
// 会話履歴を Firestore に保存しながら Gemini と対話する

import { onCall, HttpsError } from "firebase-functions/v2/https";
import { defineSecret } from "firebase-functions/params";
import { getFirestore, FieldValue } from "firebase-admin/firestore";
import { GoogleGenerativeAI, Content } from "@google/generative-ai";

const geminiApiKey = defineSecret("GEMINI_API_KEY");
const db = getFirestore();

interface ChatMessage {
  role: "user" | "model";
  text: string;
  timestamp: FirebaseFirestore.FieldValue;
}

export const chat = onCall(
  { secrets: [geminiApiKey] },
  async (request) => {
    if (!request.auth) {
      throw new HttpsError("unauthenticated", "ログインが必要です");
    }

    const { message, conversationId } = request.data;
    const userId = request.auth.uid;

    // 会話ドキュメントの参照
    const convRef = db
      .collection("users")
      .doc(userId)
      .collection("conversations")
      .doc(conversationId || db.collection("_").doc().id);

    // 既存の会話履歴を取得
    const messagesSnap = await convRef
      .collection("messages")
      .orderBy("timestamp", "asc")
      .limit(50) // 直近50件に制限
      .get();

    // Gemini API 用の履歴形式に変換
    const history: Content[] = messagesSnap.docs.map((doc) => {
      const data = doc.data() as ChatMessage;
      return {
        role: data.role,
        parts: [{ text: data.text }],
      };
    });

    // Gemini でレスポンス生成
    const genAI = new GoogleGenerativeAI(geminiApiKey.value());
    const model = genAI.getGenerativeModel({
      model: "gemini-2.0-flash",
    });

    const chatSession = model.startChat({ history });
    const result = await chatSession.sendMessage(message);
    const responseText = result.response.text();

    // ユーザーメッセージとAI応答を Firestore に保存
    const batch = db.batch();

    batch.set(convRef.collection("messages").doc(), {
      role: "user",
      text: message,
      timestamp: FieldValue.serverTimestamp(),
    } as ChatMessage);

    batch.set(convRef.collection("messages").doc(), {
      role: "model",
      text: responseText,
      timestamp: FieldValue.serverTimestamp(),
    } as ChatMessage);

    // 会話メタデータを更新
    batch.set(
      convRef,
      {
        updatedAt: FieldValue.serverTimestamp(),
        messageCount: FieldValue.increment(2),
      },
      { merge: true }
    );

    await batch.commit();

    return {
      response: responseText,
      conversationId: convRef.id,
    };
  }
);
```

## Firestore セキュリティルール

Firestore のセキュリティルールで、ユーザーが自分のデータのみアクセスできるよう制限します。

```
// firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // ユーザーの会話データ
    match /users/{userId}/conversations/{convId} {
      // 本人のみ読み書き可能
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;

      // メッセージのサブコレクション
      match /messages/{messageId} {
        allow read: if request.auth != null
                    && request.auth.uid == userId;
        // メッセージの書き込みは Cloud Functions からのみ
        allow write: if false;
      }
    }
  }
}
```

ポイントは、メッセージの書き込みをクライアントから禁止し、Cloud Functions 経由のみに制限していることです。これにより、ユーザーが不正なデータを直接書き込むリスクを排除できます。

## フロントエンド（Web）からの呼び出し

```typescript
// src/app.ts
// Firebase SDK からの Cloud Functions 呼び出し

import { initializeApp } from "firebase/app";
import { getFunctions, httpsCallable } from "firebase/functions";
import { getAuth, signInWithPopup, GoogleAuthProvider } from "firebase/auth";

const app = initializeApp({
  // Firebase Console からコピーした設定
  apiKey: "YOUR_FIREBASE_API_KEY",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
});

const functions = getFunctions(app, "asia-northeast1");
const auth = getAuth(app);

// Google ログイン
async function signIn(): Promise<void> {
  const provider = new GoogleAuthProvider();
  await signInWithPopup(auth, provider);
}

// Gemini とチャット
async function sendMessage(
  message: string,
  conversationId?: string
): Promise<{ response: string; conversationId: string }> {
  const chatFn = httpsCallable(functions, "chat");
  const result = await chatFn({ message, conversationId });
  return result.data as { response: string; conversationId: string };
}

// 使用例
async function main(): Promise<void> {
  await signIn();

  const result = await sendMessage("Firebaseの料金体系を教えて");
  console.log(result.response);

  // 同じ会話で続ける
  const followUp = await sendMessage(
    "無料枠の制限は？",
    result.conversationId
  );
  console.log(followUp.response);
}
```


## 本番運用のベストプラクティス

Cloud Functions で Gemini API を扱う際の注意点を整理します。

**レート制限**: Cloud Functions のインスタンス上限を設定して、API の過剰呼び出しを防ぎます。`maxInstances: 10` のようにオプションで制限できます。

**コスト管理**: Gemini API の呼び出し回数に応じて料金が発生します。ユーザーごとの日次リクエスト数を Firestore でカウントし、上限に達したらリクエストを拒否する仕組みを入れましょう。

**エラーハンドリング**: API タイムアウトや一時的なエラーに備え、リトライ処理を実装します。`timeoutSeconds: 60` の設定も忘れずに。

**キャッシュ**: 同じプロンプトに対するレスポンスを Firestore にキャッシュすれば、API 呼び出しを削減できます。

## FAQ

### Q1. Firebase の無料枠で Gemini API アプリは運用できますか？

A1. Firebase の Spark プラン（無料枠）では Cloud Functions の呼び出しに制限があり、外部 API（Gemini）への接続には Blaze プラン（従量課金）が必要です。ただし Blaze プランでも無料枠が適用されるため、小規模な開発・テストであれば費用はほとんどかかりません。

### Q2. Vertex AI と Google AI Studio の API キーはどちらを使うべきですか？

A2. プロトタイプや個人プロジェクトには Google AI Studio の API キーが手軽です。本番環境や企業利用では、IAM によるアクセス制御やリージョン指定が可能な Vertex AI を推奨します。詳しくは [Vertex AI クイックスタート](/articles/gemini-dev/vertex-ai-quickstart) を参照してください。

### Q3. 会話履歴が長くなるとトークン上限に達しませんか？

A3. はい、Gemini 2.0 Flash のコンテキストウィンドウは 100 万トークンですが、長い会話ではコストも増加します。実装例のように `limit(50)` で直近のメッセージに制限するか、要約を生成して過去の会話を圧縮する方法が有効です。

### Q4. Firebase Extensions の「Gemini API」拡張機能との違いは？

A4. Firebase Extensions はノーコードで導入できる便利な機能ですが、カスタマイズの自由度は限られます。この記事で紹介した Cloud Functions を使う方法は、プロンプトの制御、履歴管理、レート制限などを細かく実装でき、本格的なアプリに適しています。

## まとめ

Gemini API と Firebase の連携により、認証・データベース・サーバーレス関数を備えた本格的な AI アプリを構築できます。Cloud Functions でAPI キーを安全に管理し、Firestore で会話履歴を保存する構成は、チャットボットや AI アシスタントの基盤として広く活用できます。

Firebase の他の機能（Authentication、Hosting、Cloud Storage）と組み合わせれば、フルスタックの AI アプリを Firebase だけで完結させることも可能です。まずは [Gemini API クイックスタート](/articles/gemini-api/gemini-api-quickstart) で API の基本を押さえてから、この記事の構成を試してみてください。
