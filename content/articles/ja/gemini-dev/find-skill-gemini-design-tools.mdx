## はじめに

Gemini の真の力は、その多様なスキル（拡張機能）にあります。特にデザイン・UI 開発に関しては、**Find skill** という自動検出機能を使うことで、プロジェクトに最適なスキルを瞬時に導入できます。

しかし、「どのスキルを選べばいいのか分からない」「スキルの組み合わせが最適か自信がない」という方も多いのでは。本記事では、Find skill の仕組みと、デザイン領域での効果的な活用方法を、丁寧に解説していきます。

## Find skill とは何か

### スキルの役割

Gemini のスキルは、特定のドメイン（UI デザイン、フロントエンド実装、プロジェクト管理など）において、AI の能力を拡張・特化させるための「専門モジュール」です。

例えば：
- **frontend-design**：UI コンポーネント実装のベストプラクティス
- **web-design-guidelines**：Web デザインの基準・原則
- **UI Pro Max**：段階的 UI 設計ワークフロー

これらスキルをロードすることで、Gemini の出力はより実務的で、業界標準に準拠したものになります。

### Find skill の役割

Find skill は、プロジェクトの文脈や、ユーザーの質問から、**必要なスキルを自動検出** するシステムです。

「Figma で SPA のダッシュボード UI を設計してるんですが」と相談すれば、Find skill は自動的に以下のスキルを推奨します：
- web-design-guidelines（一般的な Web デザイン原則）
- UI Pro Max（段階的デザイン生成）
- 場合によっては、フロントエンド実装向けのスキル

## デザイン領域のメジャースキル

Gemini が提供するデザイン・UI 関連スキルを、主要なものから紹介します。

### 1. frontend-design

**用途**：フロントエンド UI コンポーネント設計・実装
**主な機能**：
- Button、Input、Card などの基本コンポーネントのベストプラクティス
- アクセシビリティ（ARIA）ガイドライン
- レスポンシブデザイン実装ポイント
- パフォーマンス（CSS-in-JS の最適化など）

**推奨される場面**：
- React / Vue / Svelte など、フロントエンドフレームワークでのコンポーネント開発
- デザインシステム構築
- 既存 UI の改善・リファクタリング

### 2. web-design-guidelines

**用途**：Web サイト・アプリ全体の設計原則
**主な機能**：
- レイアウト基本原則（グリッド・タイポグラフィ）
- ユーザビリティ（UX）考慮事項
- アクセシビリティ基準（WCAG）
- モバイルファースト設計

**推奨される場面**：
- ランディングページ・コーポレートサイト設計
- ユーザー体験（UX）の全体的な見直し
- 初期段階でのデザイン原則確認

### 3. UI Pro Max

**用途**：高度な段階的 UI デザイン生成
**主な機能**：
- ワイヤーフレーム自動生成
- デザイン提案・バリエーション生成
- デザイントークン自動抽出
- コンポーネント化サポート

**推奨される場面**：
- 複雑なアプリケーション UI の設計
- デザインシステムの初期構築
- 短期間での高速プロトタイプ実現

### 4. sleek-design-mobile-apps

**用途**：モバイルアプリ（iOS / Android）特化の UI 設計
**主な機能**：
- モバイルならではの操作パターン（ジェスチャー・タップターゲット）
- プラットフォーム別ガイドライン（Human Interface Guidelines / Material Design）
- モバイル特有の制約（画面サイズ・バッテリー）への対応

**推奨される場面**：
- iOS / Android ネイティブアプリ開発
- クロスプラットフォームアプリ（Flutter / React Native）
- モバイル特有のパターン（Bottom Sheet / Swipe など）

## Find skill の実際の活用フロー

### ステップ 1：プロジェクトコンテキストの提示

Gemini チャットを開き、自分のプロジェクトについて簡潔に説明します。

**入力例：**
```
「SaaS のプロジェクト管理アプリを開発してます。
フロントエンドは React + TypeScript。
Figma でデザイン、実装チームは React で UI コンポーネント化する流れです。

今後のワークフロー最適化について、
Gemini 環境でロードするべきスキルを提案してもらえますか？」
```

### ステップ 2：Find skill による自動検出

Gemini は、プロジェクト説明から、最適なスキルを自動検出します。

**返答例：**
```
「このプロジェクトでは、以下の3つのスキルの導入を推奨します：

1. **UI Pro Max** (優先度：高)
   - Figma での段階的 UI 設計を加速
   - デザイントークン生成により、実装効率化

2. **frontend-design** (優先度：高)
   - React コンポーネントのベストプラクティス確認
   - アクセシビリティ・パフォーマンス最適化

3. **web-design-guidelines** (優先度：中)
   - UX 全体の検証
   - 業界標準への準拠確認

各スキルの詳細と、有効化手順をお示しします。」
```

### ステップ 3：スキル導入の判断

推奨されたスキルを、プロジェクトのニーズに照らして判断します。

**質問例：**
```
「UI Pro Max と frontend-design は導入したいです。
しかし web-design-guidelines は、既に社内ガイドライン文書があるため、
本当に必要でしょうか？」
```

Gemini は、web-design-guidelines が役立つ具体的なシーンを説明し、導入判断を支援します。

## 各スキルの詳細と使い分け

### UI Pro Max の実践活用

**ワークフロー例：**
1. プロジェクト要件を文書化
2. Gemini（UI Pro Max 有効）にワイヤーフレーム提案を依頼
3. Figma で提案内容をベースにハイフィデリティ制作
4. デザイントークンを自動抽出
5. React チームへ受け渡し

**有効な質問例：**
```
「このタスク管理アプリの、
『タスク一覧』『タスク詳細』『タスク作成』画面を設計してください。
デバイスは iPhone 15（390×844）。
デザイントークン（色・フォント・スペーシング）も生成してください。」
```

### frontend-design の実践活用

**ワークフロー例：**
1. React コンポーネント設計の検討
2. frontend-design の指針を確認
3. コンポーネント実装
4. アクセシビリティ・パフォーマンス監査

**有効な質問例：**
```
「フォーム入力フィールド（Text Input）をコンポーネント化します。
以下の要件で、アクセシビリティ（ARIA）を最大化した実装案を提案してください：
- 必須項目は label に『※』を表示
- エラー状態を視覚的・音声的に伝える
- キーボード操作で全機能アクセス可能」
```

### web-design-guidelines の実践活用

**ワークフロー例：**
1. プロジェクト初期段階で、デザイン原則を確認
2. ユーザビリティ基準を定義
3. デザイン進行中に、基準との合致を確認
4. 最終チェックで、業界標準への準拠を検証

**有効な質問例：**
```
「このダッシュボードの、
タイポグラフィ（見出し・本文・数値表示）設計を評価してください。
WCAG AA 基準への準拠、読みやすさ、数値の視認性の観点から。」
```

## プロジェクト初期セットアップの実践ガイド

### パターン 1：Web サイト制作（ランディングページ）

**推奨スキル組み合わせ：**
- web-design-guidelines（必須）
- UI Pro Max（オプション、複雑な LP なら推奨）
- Canva / Figma の知見がもあると尚良

**初期セットアップの流れ：**
```
1. 「SaaS のランディングページを設計します。
    ターゲットは CTO・テックリード。
    参考として競合 5 社の LP を分析済み。
    Gemini で最適なデザイン方針を提案してください。」
   → web-design-guidelines が自動有効化

2. 「上記を踏まえて、
    ホワイトペーパーダウンロード CTA の
    配置・デザイン・コピーを5案提案してください。」
   → UI Pro Max が自動推奨（複数提案が効果的なため）
```

### パターン 2：モバイルアプリ開発（React Native）

**推奨スキル組み合わせ：**
- sleek-design-mobile-apps（必須）
- UI Pro Max（複雑な IA の場合推奨）
- frontend-design（React Native コンポーネント実装時）

**初期セットアップの流れ：**
```
1. 「フィットネスアプリを React Native で開発します。
    ターゲットは 20～40 代。
    iOS / Android 両対応。
    初期画面構成をデザイナーと相談中です。」
   → sleek-design-mobile-apps 推奨

2. 「このアプリのボトムタブナビゲーション
    （ホーム / ワークアウト / プロフィール）の
    アイコン・ラベル配置案を3案提案してください。」
   → UI Pro Max が効果的
```

### パターン 3：デザインシステム構築

**推奨スキル組み合わせ：**
- UI Pro Max（必須）
- frontend-design（必須）
- web-design-guidelines（補助）

**初期セットアップの流れ：**
```
1. 「エンタープライズ SaaS のデザインシステムを構築します。
    現在、複数プロダクト間のデザインが分散している状況です。
    統一されたシステム構築の方針を教えてください。」
   → 3 つのスキル全て自動推奨

2. 「共通カラーパレット・タイポグラフィスケール・
    スペーシング規則を定義してください。」
   → UI Pro Max でデザイントークン自動生成

3. 「Button、Input、Card の3つのコアコンポーネント
    について、ARIA 実装案を提案してください。」
   → frontend-design で最適な実装ガイドラインを提示
```

## スキル導入後の効率化例

### 時間削減の実績

実際のプロジェクトで、デザインスキルを導入した場合の時間削減例：

**従来のフロー（スキルなし）：**
- 要件整理：3 日
- ワイヤーフレーム：3 日
- デザイン：5 日
- デザイントークン抽出：2 日
- 実装ガイドライン作成：1 日
- **合計：14 日**

**Find skill による最適化（UI Pro Max + frontend-design）：**
- 要件整理：2 日（Gemini が構造化支援）
- ワイヤーフレーム：1 日（自動生成）
- デザイン：3 日（Gemini 提案をベース）
- デザイントークン抽出：0.5 日（自動生成）
- 実装ガイドライン作成：0.5 日（frontend-design 活用）
- **合計：7 日（50% 削減）**

### 品質向上の実績

- **アクセシビリティ準拠率**：78% → 95%（frontend-design による指導）
- **デザイン・実装の齟齬**：26 件 → 4 件（デザイントークン統一）
- **デザインレビュー時間**：3 時間 → 1 時間（基準が明確化）

## ベストプラクティス

### 1. 段階的なスキル導入

プロジェクト開始時に全スキル導入するのではなく、必要に応じて段階的に追加しましょう。

初期：web-design-guidelines
→ デザイン本格化：UI Pro Max
→ 実装段階：frontend-design

### 2. スキル間の連携を活用

複数スキルは相補的に機能します。

「UI Pro Max で提案されたデザインを、frontend-design の実装ガイドラインで検証」

といった活用により、さらに高品質な結果が得られます。

### 3. スキル推奨内容の記録

Gemini から「このプロジェクトでは X スキルを推奨」という助言が返ってきたら、プロジェクト文書に記録しておきましょう。

後々、新しいチームメンバーがプロジェクトに参画する際、「なぜこのスキル構成なのか」が一目瞭然になります。

## トラブルシューティング

### Q1：推奨されたスキルが、期待と異なる

**A**：Find skill は、入力情報をベースに推奨します。より詳しいプロジェクト情報を提供すれば、精度が向上します。

例：「React を使ってます」ではなく「React + Next.js で SSR ダッシュボード」と具体的に。

### Q2：複数スキルをロードすると、回答が混乱しないか

**A**：心配不要。各スキルは明確な役割分担を持っており、干渉しません。むしろ、複数スキルの統合知見から、より包括的な提案が得られます。

### Q3：スキルのアップデート頻度は

**A**：Gemini プラットフォーム側で随時更新されます。ユーザーは特に対応不要。最新のスキルが自動的に利用可能になります。

## まとめ

Find skill によるデザインスキル自動導入は、以下のメリットがあります。

- **効率化**：プロジェクト最適のスキルが瞬時に構成される
- **品質向上**：業界標準に準拠した、洗練されたアウトプット
- **学習**：各スキルの活用を通じて、デザイン・実装のベストプラクティスが習得できる

初めは「本当に Find skill で大丈夫？」と感じるかもしれません。しかし、実際にプロジェクトで活用してみれば、その効果は一目瞭然。

次のプロジェクトから、ぜひ Find skill を試してみてください。あなたのチームの生産性は、確実に向上するでしょう。
