## Figma と Gemini API をシームレスに繋ぐ

Figma はデザイナーとエンジニアが協業するための中心的なツールです。しかし「Figma のデザインをコードに変換する」作業は、依然として手作業が多く残っています。

**Figma MCP サーバー**を **Gemini API** に接続することで、この変換プロセスを AI が自動化します。Gemini はリアルタイムで Figma のデザインデータにアクセスし、コンポーネントの仕様を読み取り、実装コードを生成します。

---

## 必要なもの

- Figma アカウント（Personal Access Token）
- Google AI Studio API キー（Gemini API）
- Node.js 18 以上
- Python 3.10 以上（オプション）

---

## Figma MCP サーバーのセットアップ

### ステップ 1: Figma Personal Access Token の取得

1. Figma にログイン
2. 右上のプロフィール → **Settings** → **Security**
3. **Personal access tokens** セクションで **Generate new token**
4. トークンをコピー（一度しか表示されない）

### ステップ 2: Figma MCP サーバーのインストール

```bash
npm install -g @figma/mcp-server
```

または `npx` で直接実行：

```bash
npx @figma/mcp-server
```

### ステップ 3: 環境変数の設定

```bash
export FIGMA_ACCESS_TOKEN="your-figma-personal-access-token"
export GOOGLE_API_KEY="your-gemini-api-key"
```

## Gemini API との連携コード

### 基本的な接続と Figma データの取得

```python
import google.generativeai as genai
import requests
import json

# Gemini API の設定
genai.configure(api_key="YOUR_GEMINI_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

# Figma API からデザインデータを取得
FIGMA_TOKEN = "YOUR_FIGMA_TOKEN"
FILE_ID = "your-figma-file-id"  # Figma URL から取得

def get_figma_components(file_id: str, node_id: str | None = None) -> dict:
    """Figma ファイルからコンポーネント情報を取得"""
    url = f"https://api.figma.com/v1/files/{file_id}"
    if node_id:
        url += f"/nodes?ids={node_id}"

    headers = {"X-Figma-Token": FIGMA_TOKEN}
    response = requests.get(url, headers=headers)
    return response.json()

def generate_code_from_figma(figma_data: dict, framework: str = "React") -> str:
    """Figma データから Gemini API でコードを生成"""
    prompt = f"""
以下の Figma コンポーネントの仕様を分析し、
{framework} コンポーネントとして実装してください。

Figma データ:
{json.dumps(figma_data, ensure_ascii=False, indent=2)}

要件:
- TypeScript で実装
- Tailwind CSS でスタイリング
- アクセシビリティ対応（aria-label 等）
- レスポンシブデザイン
- Props インターフェースを定義
"""
    response = model.generate_content(prompt)
    return response.text

# 使用例
figma_data = get_figma_components(FILE_ID)
react_code = generate_code_from_figma(figma_data, framework="React")
print(react_code)
```

## MCP 設定ファイルでの統合

Gemini CLI や Claude Code の MCP 設定に Figma を追加することで、会話の中で直接 Figma データを参照できます。

### MCP 設定例（mcp_config.json）

```json
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "@figma/mcp-server@latest"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "YOUR_FIGMA_TOKEN"
      }
    }
  }
}
```

### 設定後の使用例

```
@figma get_file(file_id: "abc123xyz")
→ Figma のデザインデータを取得して、Gemini API でコードに変換してください。
  React + Tailwind CSS、TypeScript で実装してください。
```

## 実践例: デザインシステムの自動コード生成

### シナリオ: ボタンコンポーネントの自動生成

```python
def sync_design_system(figma_file_id: str, components: list[str]) -> dict:
    """
    Figma のデザインシステムからコンポーネントを一括生成
    """
    results = {}

    for component_name in components:
        # Figma からコンポーネントを検索
        figma_data = get_figma_components(figma_file_id)

        # コンポーネント仕様を抽出
        prompt = f"""
Figma デザインシステムのデータから "{component_name}" コンポーネントを探し、
以下の情報を抽出してください：

1. サイズ変種（sm, md, lg）
2. 状態（default, hover, disabled, loading）
3. カラースキーム（primary, secondary, destructive）
4. タイポグラフィ仕様

データ: {json.dumps(figma_data, indent=2)[:3000]}
"""

        spec_response = model.generate_content(prompt)

        # 仕様に基づいてコードを生成
        code_prompt = f"""
以下の仕様に基づいて、プロダクションレベルの React コンポーネントを生成してください：

{spec_response.text}

- TypeScript + Tailwind CSS + shadcn/ui 互換
- variants は cva (class-variance-authority) で定義
- Storybook ドキュメント付き
"""
        code_response = model.generate_content(code_prompt)
        results[component_name] = code_response.text

    return results

# デザインシステムの主要コンポーネントを一括生成
components = ["Button", "Input", "Card", "Modal", "Badge"]
generated = sync_design_system(FILE_ID, components)
```

## Figma × Gemini のマルチモーダル活用

Gemini はマルチモーダル（画像理解）に優れており、Figma のスクリーンショットを直接入力することもできます。

```python
import PIL.Image
import requests
from io import BytesIO

def analyze_figma_screenshot(image_url: str) -> str:
    """
    Figma のスクリーンショットを Gemini で分析し、
    実装の改善点や仕様を抽出
    """
    # Figma エクスポート API で画像を取得
    response = requests.get(image_url)
    image = PIL.Image.open(BytesIO(response.content))

    model_vision = genai.GenerativeModel("gemini-2.0-flash")

    prompt = """
このデザインスクリーンショットを分析して：

1. コンポーネント構造の提案（どのようにコンポーネントに分割するか）
2. アクセシビリティの懸念事項（コントラスト比、タッチターゲット等）
3. レスポンシブ対応時の注意点
4. React + Tailwind CSS での実装コード

詳細に分析してください。
"""

    response = model_vision.generate_content([prompt, image])
    return response.text

# 使用例
# Figma Export API でスクリーンショットの URL を取得
figma_export_url = f"https://api.figma.com/v1/images/{FILE_ID}?ids=node_id&format=png"
headers = {"X-Figma-Token": FIGMA_TOKEN}
export_response = requests.get(figma_export_url, headers=headers)
image_url = export_response.json()["images"]["node_id"]

analysis = analyze_figma_screenshot(image_url)
print(analysis)
```

### 2. デザイン審査・QA

```python
def design_qa_review(figma_node_id: str) -> dict:
    """デザインの品質審査を自動化"""
    figma_data = get_figma_components(FILE_ID, figma_node_id)

    qa_prompt = f"""
以下のデザインデータを審査し、問題点を報告してください：

{json.dumps(figma_data, indent=2)[:2000]}

チェック項目：
- WCAG 2.1 AA 準拠（コントラスト比 4.5:1 以上）
- タッチターゲット最小サイズ（44×44px）
- フォントサイズ（最小 12px）
- 一貫性（カラー・スペーシング・タイポグラフィ）
- レスポンシブブレークポイントの整合性
"""
    response = model.generate_content(qa_prompt)
    return {"node_id": figma_node_id, "issues": response.text}
```

### 3. デザインドキュメント自動生成

```python
def generate_design_docs(figma_file_id: str) -> str:
    """Figma ファイルから設計ドキュメントを自動生成"""
    figma_data = get_figma_components(figma_file_id)

以下の Figma デザインデータから、開発者向けの設計仕様書を作成してください：

{json.dumps(figma_data, indent=2)[:5000]}

含める内容：
- コンポーネント一覧と説明

- タイポグラフィスケール
- スペーシングシステム

- 使用例
"""
    response = model.generate_content(doc_prompt)
    return response.text
```

## まとめ

Figma MCP × Gemini API の組み合わせは、デザインと開発の距離を劇的に縮めます。

| 作業 | 従来 | Figma MCP + Gemini |
|------|------|--------------------|
| コンポーネント実装 | 30分〜2時間 | 5〜10分 |
| デザイン QA | 手動チェック | 自動レポート |
| 設計ドキュメント | 別途作成 | 自動生成 |
| デザインシステム同期 | 手作業 | バッチ自動化 |

MCP による Figma との直接連携により、Gemini API は「デザインを理解するエンジニア」として機能します。デザイナーが Figma でデザインを更新すれば、Gemini がそれを検知してコードに反映する——そんな完全自動化されたワークフローも夢ではありません。
