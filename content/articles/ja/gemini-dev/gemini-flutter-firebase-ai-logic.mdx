## Flutter アプリに Gemini を組み込む理由

スマートフォンアプリの競争が激化するなか、**AI 機能の有無**がアプリの価値を大きく左右する時代になりました。Google の Gemini API を Flutter アプリに組み込むことで、自然言語によるチャット、画像認識、文書要約など多彩な AI 機能を iOS・Android 両対応のアプリに追加できます。

2026 年現在、Flutter に Gemini を統合する最も推奨されるアプローチは **Firebase AI Logic SDK** です。API キーをクライアントに直接埋め込む必要がなく、Firebase が認証・セキュリティを一括管理してくれるため、プロダクション品質のアプリに最適です。

この記事では、Firebase AI Logic SDK を使って Flutter アプリに Gemini を統合する方法を、完全なコード例とともにステップバイステップで解説します。

## 前提条件と環境準備

### 必要なもの

- Flutter SDK 3.x 以上がインストール済み
- Dart 3.x 以上
- Firebase プロジェクトが作成済み（[Firebase Console](https://console.firebase.google.com/) から作成）
- Google AI Studio で Gemini API を有効化済み（[API キーの取得方法](/articles/gemini-api/gemini-api-quickstart)）
- FlutterFire CLI がインストール済み

### FlutterFire CLI のセットアップ

```bash
# FlutterFire CLI をグローバルインストール
dart pub global activate flutterfire_cli

# Firebase CLI もインストール（未インストールの場合）
npm install -g firebase-tools
firebase login

# Flutter プロジェクトを作成
flutter create gemini_ai_app
cd gemini_ai_app

# Firebase と連携（対話形式でプロジェクトを選択）
flutterfire configure
```

`flutterfire configure` を実行すると、`lib/firebase_options.dart` が自動生成されます。このファイルには各プラットフォームの設定が含まれており、コードにハードコードされた API キーを管理する手間が省けます。

### 依存パッケージの追加

```yaml
# pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  firebase_core: ^3.6.0
  firebase_ai: ^0.1.0          # Firebase AI Logic SDK
  firebase_app_check: ^0.3.0   # セキュリティ強化（推奨）
```

```bash
flutter pub get
```

## 基本的なチャット機能の実装

### シンプルなテキスト生成

まずは最もシンプルな例として、ユーザーの質問に Gemini が回答するチャット画面を作成します。

```dart
import 'package:firebase_ai/firebase_ai.dart';
import 'package:flutter/material.dart';

class ChatScreen extends StatefulWidget {
  const ChatScreen({super.key});

  @override
  State<ChatScreen> createState() => _ChatScreenState();
}

class _ChatScreenState extends State<ChatScreen> {
  final TextEditingController _controller = TextEditingController();
  final List<Map<String, String>> _messages = [];
  late final GenerativeModel _model;
  late final ChatSession _chat;
  bool _isLoading = false;

  @override
  void initState() {
    super.initState();
    // Firebase AI Logic SDK でモデルを初期化
    _model = FirebaseAI.googleAI().generativeModel(
      model: 'gemini-3-flash-preview',
      // システムインストラクションで動作をカスタマイズ
      systemInstruction: Content.system(
        'あなたは役に立つ AI アシスタントです。日本語で簡潔に回答してください。',
      ),
    );
    _chat = _model.startChat();
  }

  Future<void> _sendMessage(String text) async {
    if (text.trim().isEmpty) return;

    setState(() {
      _messages.add({'role': 'user', 'text': text});
      _isLoading = true;
    });
    _controller.clear();

    try {
      // Gemini にメッセージを送信
      final response = await _chat.sendMessage(
        Content.text(text),
      );
      final reply = response.text ?? '（応答なし）';

      setState(() {
        _messages.add({'role': 'assistant', 'text': reply});
      });
    } catch (e) {
      setState(() {
        _messages.add({'role': 'error', 'text': 'エラー: $e'});
      });
    } finally {
      setState(() => _isLoading = false);
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Gemini チャット')),
      body: Column(
        children: [
          Expanded(
            child: ListView.builder(
              padding: const EdgeInsets.all(16),
              itemCount: _messages.length,
              itemBuilder: (context, index) {
                final msg = _messages[index];
                final isUser = msg['role'] == 'user';
                return Align(
                  alignment:
                      isUser ? Alignment.centerRight : Alignment.centerLeft,
                  child: Container(
                    margin: const EdgeInsets.symmetric(vertical: 4),
                    padding: const EdgeInsets.all(12),
                    decoration: BoxDecoration(
                      color: isUser ? Colors.blue[100] : Colors.grey[200],
                      borderRadius: BorderRadius.circular(12),
                    ),
                    child: Text(msg['text'] ?? ''),
                  ),
                );
              },
            ),
          ),
          if (_isLoading)
            const Padding(
              padding: EdgeInsets.all(8),
              child: CircularProgressIndicator(),
            ),
          Padding(
            padding: const EdgeInsets.all(8),
            child: Row(
              children: [
                Expanded(
                  child: TextField(
                    controller: _controller,
                    decoration: const InputDecoration(
                      hintText: 'メッセージを入力...',
                      border: OutlineInputBorder(),
                    ),
                    onSubmitted: _sendMessage,
                  ),
                ),
                const SizedBox(width: 8),
                IconButton(
                  icon: const Icon(Icons.send),
                  onPressed: () => _sendMessage(_controller.text),
                ),
              ],
            ),
          ),
        ],
      ),
    );
  }
}
```

このコードを実行すると、Gemini との基本的なマルチターン会話が機能します。`ChatSession` を使うことで、会話の文脈が自動的に保持されます。

**期待する動作:**
- ユーザーが「Dart と Flutter の違いは？」と入力
- Gemini が日本語で簡潔な説明を返答
- 会話履歴がリスト形式で表示される

## ストリーミングレスポンスで UX を向上させる

API の応答を一括で待つのではなく、**ストリーミング**を使うとレスポンスが文字単位でリアルタイムに表示され、ユーザー体験が大幅に向上します。

```dart
Future<void> _sendMessageStream(String text) async {
  if (text.trim().isEmpty) return;

  setState(() {
    _messages.add({'role': 'user', 'text': text});
    // 空のアシスタントメッセージを追加して逐次更新
    _messages.add({'role': 'assistant', 'text': ''});
    _isLoading = true;
  });
  _controller.clear();

  try {
    // sendMessageStream でストリーミング応答を受け取る
    final stream = _chat.sendMessageStream(Content.text(text));

    await for (final chunk in stream) {
      final chunkText = chunk.text ?? '';
      setState(() {
        // 最後のメッセージ（アシスタント）にテキストを追記
        final lastIndex = _messages.length - 1;
        _messages[lastIndex] = {
          'role': 'assistant',
          'text': (_messages[lastIndex]['text'] ?? '') + chunkText,
        };
      });
    }
  } catch (e) {
    setState(() {
      _messages.add({'role': 'error', 'text': 'エラー: $e'});
    });
  } finally {
    setState(() => _isLoading = false);
  }
}
```

## 画像認識（マルチモーダル）の実装

Firebase AI Logic SDK はテキストだけでなく、**画像・動画・PDF** などのマルチモーダル入力にも対応しています。カメラで撮影した画像を Gemini に解析させる機能を追加してみましょう。

```dart
import 'package:image_picker/image_picker.dart';
import 'dart:io';

// pubspec.yaml に image_picker: ^1.1.0 を追加してください

Future<void> _analyzeImage() async {
  final picker = ImagePicker();
  final pickedFile = await picker.pickImage(source: ImageSource.camera);
  if (pickedFile == null) return;

  setState(() => _isLoading = true);

  try {
    final imageBytes = await File(pickedFile.path).readAsBytes();

    // テキストと画像を組み合わせたマルチモーダルリクエスト
    final response = await _model.generateContent([
      Content.multi([
        TextPart('この画像に何が写っていますか？詳しく説明してください。'),
        DataPart('image/jpeg', imageBytes),
      ]),
    ]);

    final description = response.text ?? '画像を解析できませんでした';
    setState(() {
      _messages.add({'role': 'user', 'text': '[画像を送信]'});
      _messages.add({'role': 'assistant', 'text': description});
    });
  } catch (e) {
    setState(() {
      _messages.add({'role': 'error', 'text': '画像解析エラー: $e'});
    });
  } finally {
    setState(() => _isLoading = false);
  }
}
```

**期待する出力（例）:**
```
この画像には、白い背景に赤いりんごが1つ写っています。
りんごは光沢があり、緑色の葉が1枚付いています。
画像全体は明るく、スタジオで撮影されたような印象です。
```

## よくあるエラーと対処法

### エラー 1: `FirebaseException: [firebase_ai/permission-denied]`

**原因:** Firebase プロジェクトで Gemini API が有効になっていない。

**対処:** Firebase Console → AI Logic → 「Gemini API を有効にする」ボタンをクリック。または Google Cloud Console で "Generative Language API" を有効化する。

### エラー 2: `MissingPluginException` (iOS)

**原因:** `flutterfire configure` を実行した後、`ios/Podfile` が更新されていない。

**対処:**
```bash
cd ios && pod install --repo-update && cd ..
flutter clean && flutter run
```

### エラー 3: ストリーミング中に `setState` が呼ばれない

**原因:** `await for` ループ内で `mounted` チェックをしていない。

**対処:**
```dart
await for (final chunk in stream) {
  if (!mounted) break; // Widget が破棄されていればループを抜ける
  setState(() { /* ... */ });
}
```

### エラー 4: 画像サイズが大きすぎてタイムアウト

**原因:** 高解像度の画像をそのまま送信している。

**対処:** `image_picker` の `maxWidth` / `imageQuality` パラメータで圧縮する。
```dart
final pickedFile = await picker.pickImage(
  source: ImageSource.camera,
  maxWidth: 1024,
  imageQuality: 85,
);
```

## FAQ

### Q1. firebase_ai パッケージと google_generative_ai パッケージの違いは？

`google_generative_ai` パッケージは Gemini API キーを直接使用します。ローカルでのプロトタイプ作成には便利ですが、API キーをクライアントアプリに埋め込む必要があるためセキュリティリスクがあります。一方、`firebase_ai`（Firebase AI Logic SDK）は Firebase Authentication や App Check と連携し、API キーをサーバーサイドで管理します。**プロダクション環境では firebase_ai の使用を強く推奨します。**

### Q2. 無料枠はありますか？コストはどれくらいかかりますか？

Firebase AI Logic SDK 経由では、Gemini Flash モデルは一定の無料枠が利用できます。従量課金制のため、大規模なアプリでは Gemini API の[スペンドキャップ機能](/articles/gemini-api/gemini-api-spend-caps-guide)を設定して予期しない高額請求を防ぐことをお勧めします。最新の料金は [Google AI Studio の料金ページ](https://ai.google.dev/pricing) で確認してください。

### Q3. Gemini の応答を Firestore に保存するには？

Firebase AI Logic SDK は Firebase エコシステムと完全に統合されているため、Firestore との連携は自然に行えます。詳しい実装方法は[Gemini × Firebase 連携ガイド](/articles/gemini-dev/gemini-firebase-integration-guide)をご参照ください。会話履歴の永続化、ユーザーごとのチャット管理など実践的な例を解説しています。

### Q4. iOS と Android 両方で動作しますか？

はい、Flutter は iOS と Android の両プラットフォームに対応しており、Firebase AI Logic SDK も同様にクロスプラットフォームで動作します。`flutterfire configure` を実行した際に両プラットフォームの設定ファイルが自動生成されます。

## まとめと次のステップ

この記事では、Firebase AI Logic SDK を使って Flutter アプリに Gemini を統合する基本的な流れを解説しました。

- **Firebase AI Logic SDK** は API キーをクライアントに直接埋め込まない安全な統合方法
- `GenerativeModel` と `ChatSession` でマルチターン会話を実装可能
- `sendMessageStream` でストリーミングレスポンスによる UX 向上
- `Content.multi` でテキスト＋画像のマルチモーダル入力に対応

次のステップとして、以下の機能追加を検討してみてください。

- **Function Calling**: アプリ内のデータや API と Gemini を連携させる
- **Vertex AI 経由の統合**: エンタープライズ向けのより高度なセキュリティが必要な場合は Vertex AI + Gemini ガイドを参照
- **GenUI SDK**: Gemini が動的に Flutter UI を生成する新機能（Flutter 3.x 以降）

Gemini の AI 機能をあなたのアプリに組み込んで、競合と差別化する AI 体験を作り上げましょう。
