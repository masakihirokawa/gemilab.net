## Gemini Deep Research Agent とは

2026年3月、GoogleはGemini APIに**Deep Research Agent**をプレビュー公開しました。これは、複雑な調査タスクを自律的に計画・実行・合成できるAIエージェントです。

従来のRAGや検索グラウンディングと異なり、Deep Research Agentは：
- 複数ステップの調査計画を自動で立案
- Webと提供されたドキュメントを横断して情報収集
- 出典付きの詳細なレポートを生成
- 非同期でバックグラウンド実行（長時間タスク対応）

というエンドツーエンドの自律リサーチ能力を持っています。

## 利用条件

Deep Research AgentはGemini APIの**Interactions API**経由でのみ利用可能です。

- 使用モデル: **gemini-3.1-pro**（内部動作）
- API: Interactions API（Beta）
- 実行方式: 非同期バックグラウンド実行

## セットアップ

まず `google-genai` パッケージを最新版に更新します。

```bash
pip install --upgrade google-genai
```

## 基本的な使い方

### 1. シンプルなリサーチリクエスト

```python
import google.genai as genai
import time

client = genai.Client(api_key="YOUR_API_KEY")

# Interactions APIでDeep Research Agentを起動
interaction = client.interactions.create(
    agent="deep-research",
    contents="2026年の生成AIモデルの性能比較レポートを作成してください。Gemini、GPT、Claudeを含む主要モデルを比較し、ベンチマーク結果と各モデルの特徴を整理してください。",
    config={
        "background": True  # 非同期実行（必須）
    }
)

print(f"インタラクションID: {interaction.id}")
print(f"ステータス: {interaction.status}")

# 完了を待機（ポーリング）
while interaction.status not in ["completed", "failed"]:
    time.sleep(10)
    interaction = client.interactions.get(interaction.id)
    print(f"ステータス: {interaction.status}")

# 結果を取得
if interaction.status == "completed":
    print(interaction.result.text)
```

### 2. ドキュメントと組み合わせたリサーチ

ローカルのPDF・CSVファイルをアップロードして、Webデータと組み合わせた調査も可能です。

```python
import pathlib

# ドキュメントをアップロード
uploaded_file = client.files.upload(
    path=pathlib.Path("market_report_2025.pdf"),
    config={"mime_type": "application/pdf"}
)

# ファイルとWebを組み合わせてリサーチ
interaction = client.interactions.create(
    agent="deep-research",
    contents=[
        uploaded_file,
        "添付のレポートと最新のWeb情報を組み合わせ、2026年のAI市場動向分析を作成してください。"
    ],
    config={
        "background": True,
        "tools": ["file_search", "google_search"]
    }
)
```

## 非同期実行パターン（プロダクション向け）

長時間のリサーチタスクには、ポーリングよりもWebhookやキューを使った非同期パターンを推奨します。

```python
import asyncio
import google.genai as genai

async def run_deep_research(query: str) -> str:
    client = genai.AsyncClient(api_key="YOUR_API_KEY")

    # リサーチ開始
    interaction = await client.interactions.create(
        agent="deep-research",
        contents=query,
        config={"background": True}
    )

    # 完了まで待機（最大30分）
    max_wait = 1800
    elapsed = 0
    poll_interval = 15

    while elapsed < max_wait:
        await asyncio.sleep(poll_interval)
        elapsed += poll_interval

        interaction = await client.interactions.get(interaction.id)

        if interaction.status == "completed":
            return interaction.result.text
        elif interaction.status == "failed":
            raise RuntimeError(f"リサーチ失敗: {interaction.error}")

    raise TimeoutError("リサーチがタイムアウトしました")

# 実行例
result = asyncio.run(run_deep_research(
    "Gemini 3シリーズの各モデルの違いとユースケース別の使い分けガイドを作成してください。"
))
print(result)
```

## 出力のカスタマイズ

### JSON スキーマ出力

構造化された結果を受け取るには、`response_schema` を指定します。

```python
from pydantic import BaseModel
from typing import List

class Source(BaseModel):
    title: str
    url: str
    relevance: float

class ResearchReport(BaseModel):
    summary: str
    key_findings: List[str]
    sources: List[Source]
    confidence_score: float

interaction = client.interactions.create(
    agent="deep-research",
    contents="量子コンピュータの2026年現在の商用化状況を調査してください。",
    config={
        "background": True,
        "response_schema": ResearchReport
    }
)
```

### プロンプトで出力構造を制御

JSONスキーマを使わなくても、プロンプトで出力フォーマットを指定できます。

```python
interaction = client.interactions.create(
    agent="deep-research",
    contents="""
    Gemini APIの最新価格体系を調査し、以下の形式でレポートを作成してください：

    ## エグゼクティブサマリー
    （3文以内）

    ## モデル別料金表
    | モデル | 入力 | 出力 | 特記事項 |

    ## コスト最適化のヒント
    （箇条書き5項目）

    ## 参考ソース
    （URLと取得日時付き）
    """,
    config={"background": True}
)
```

## 料金

Deep Research Agentの料金は、エージェントが自律的に実行するリサーチの深さによって変わります。

| 要素 | 課金基準 |
|------|---------|
| 入力トークン | Gemini 3.1 Pro レート（$1.25/Mトークン） |
| 出力トークン | Gemini 3.1 Pro レート（$10.00/Mトークン） |
| Web検索 | 検索回数に応じて |
| ファイル検索 | ファイルサイズ・処理量に応じて |

エージェントが複数ステップで多くの検索を行うほどコストが増加するため、**プロンプトで調査範囲を明確に指定する**ことがコスト削減の鍵です。

## Interactions API と従来の generate_content との比較

| 機能 | generate_content | Deep Research Agent |
|------|----------------|---------------------|
| 実行方式 | 同期 | 非同期バックグラウンド |
| 自律計画 | なし | あり（多段階タスク） |
| Web検索 | 手動設定 | 自動判断・実行 |
| ドキュメント解析 | 手動 | 自動横断 |
| 実行時間 | 数秒 | 数分〜数十分 |
| コスト | 低〜中 | 中〜高（タスク複雑度依存） |

## 実用ユースケース

**競合調査レポートの自動生成**
製品ラインナップの競合比較を定期実行し、Slackやメールに配信。

**技術トレンドモニタリング**
特定技術分野の最新動向を毎日調査してダッシュボードに集約。

**法的・規制情報の収集**
業界規制の変更を自動追跡し、コンプライアンス担当者へアラート。

**投資リサーチの補助**
企業分析レポートの一次調査を自動化（最終判断は人間が行う）。

## 注意事項

- 現在**プレビュー段階**のため、本番環境での利用には注意が必要です
- 調査結果は常に事実確認が推奨されます（完全な正確性は保証されません）
- バックグラウンド実行のため、結果取得までのポーリングまたはWebhookが必要です

## まとめ

Gemini Deep Research Agentは、複雑な多段階リサーチをAPI1つで自動化できる強力な機能です。Interactions APIのバックグラウンド実行と組み合わせることで、定期レポート生成や情報収集の自動化が大幅に簡単になります。プレビュー期間中に試し、本番適用に向けたユースケースを検討してみてください。
