## Deep Thinkとは何か

### 通常の推論との違い

従来のLLMは「線形推論」と呼ばれるアプローチをとります。入力を受け取り、ステップごとにトークンを生成し、単一の解答パスを辿ります。これは多くのタスクで十分ですが、高度な数学や複雑な科学的問題では不十分なことがあります。

Gemini 3 Deep Thinkは**並列思考（Parallel Thinking）**という異なるアーキテクチャを採用しています。

```
通常の推論（線形）:
入力 → 推論ステップ1 → 推論ステップ2 → 解答

Deep Think（並列）:
入力 → 仮説A → 検証A → ┐
     → 仮説B → 検証B → → 統合 → 最良の解答
     → 仮説C → 検証C → ┘
```

Googleの公式説明によれば、Deep Thinkは「多数のアイデアを同時に生成し、それらを並行して検討し、時間をかけてアイデアを修正・組み合わせながら、最良の解答に到達する」仕組みです。

### thinking_levelパラメータの設計思想

Gemini 3では推論の深さを`thinking_level`パラメータで明示的に制御できます。

| レベル | 用途 | レイテンシ | トークンコスト |
|--------|------|-----------|--------------|
| `minimal` | 事実確認・単純な質問 | 最低 | 最小 |
| `low` | スループット重視のバッチ処理 | 低 | 低 |
| `medium` | 多くの業務タスク | 中 | 中 |
| `high` | デフォルト。複雑な推論・コーディング | 高 | 高 |

Deep Thinkを最大限に活用するには`thinking_level: "high"`（デフォルト）を使用します。ただし、すべてのタスクに`high`を使うのはコスト非効率です。適切なレベルの選択が重要です。

### 基本的なDeep Think呼び出し

```python
from google import genai
from google.genai import types

# クライアントの初期化
client = genai.Client()

# Deep Think（thinking_level: high）でリクエスト
response = client.models.generate_content(
    model="gemini-3-pro-preview",  # または gemini-3.1-pro-preview
    contents="次の微分方程式を解いてください: dy/dx = x^2 * y, y(0) = 1",
    config=types.GenerateContentConfig(
        thinking_config=types.ThinkingConfig(
            thinking_level="high"  # Deep Think: 最大推論深度
        )
    )
)

# 応答の表示
print(response.text)
```

**期待する出力例:**
```
この微分方程式は変数分離法で解けます。

分離: dy/y = x^2 dx
両辺積分: ln|y| = x^3/3 + C
指数変換: y = A * e^(x^3/3)

初期条件 y(0) = 1 を適用:
1 = A * e^0 = A

よって解は: y = e^(x^3/3)
```

### 思考プロセスの可視化（Thought Summaries）

Deep Thinkの内部推論プロセスを確認することで、解答の信頼性評価や教育目的に活用できます。

```python
from google import genai
from google.genai import types

client = genai.Client()

# 思考サマリーを有効化してリクエスト
response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="""
    次の問題を解いてください:
    確率論的グラフモデルで、ノードA→BとA→Cの依存関係がある場合、
    B=1が観測されたときのC=1の条件付き確率を求めてください。
    P(A=1)=0.3, P(B=1|A=1)=0.8, P(B=1|A=0)=0.1,
    P(C=1|A=1)=0.7, P(C=1|A=0)=0.2
    """,
    config=types.GenerateContentConfig(
        thinking_config=types.ThinkingConfig(
            thinking_level="high",
            include_thoughts=True  # 思考サマリーを含める
        )
    )
)

# 思考プロセスと解答を分けて表示
for part in response.candidates[0].content.parts:
    if hasattr(part, 'thought') and part.thought:
        print("=== 思考プロセス ===")
        print(part.text)
        print()
    else:
        print("=== 最終解答 ===")
        print(part.text)
```

### マルチターン会話でのThought Signatures

複数ターンの会話でDeep Thinkを使う場合、Thought Signaturesを正しく処理することが重要です。

```python
from google import genai
from google.genai import types

client = genai.Client()

# 会話履歴を管理する関数
def deep_think_conversation(messages: list[dict]) -> str:
    """
    Deep Thinkを使ったマルチターン会話。
    Thought Signaturesを自動的に引き継ぐ。
    """
    contents = []

    for msg in messages:
        contents.append(
            types.Content(
                role=msg["role"],
                parts=msg["parts"]  # Thought Signaturesを含むpartリスト
            )
        )

    response = client.models.generate_content(
        model="gemini-3-pro-preview",
        contents=contents,
        config=types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(
                thinking_level="high"
            )
        )
    )

    return response

# 使用例: 数学証明の段階的な深掘り
conversation = [
    {
        "role": "user",
        "parts": [types.Part(text="フェルマーの最終定理を簡単に説明してください")]
    }
]

# 第1ターン
response1 = deep_think_conversation(conversation)

# レスポンスパーツを会話履歴に追加（Thought Signaturesを含む）
conversation.append({
    "role": "model",
    "parts": response1.candidates[0].content.parts
})

# 第2ターン: 深掘り質問
conversation.append({
    "role": "user",
    "parts": [types.Part(text="ワイルズの証明のアイデアをもう少し詳しく説明してください")]
})

response2 = deep_think_conversation(conversation)
print(response2.text)
```

## API実装：JavaScript（Node.js）での使い方

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GOOGLE_API_KEY });

async function deepThinkAnalysis(problem) {
  const response = await ai.models.generateContent({
    model: "gemini-3-pro-preview",
    contents: problem,
    config: {
      thinkingConfig: {
        thinkingLevel: "high", // Deep Think mode
        includeThoughts: true,  // 思考プロセスを取得
      },
    },
  });

  // 思考プロセスと最終解答を分離
  let thoughts = "";
  let answer = "";

  for (const part of response.candidates[0].content.parts) {
    if (part.thought) {
      thoughts += part.text;
    } else {
      answer += part.text;
    }
  }

  return { thoughts, answer };
}

// 使用例: 複雑なアルゴリズム設計
const problem = `
  10億件のレコードを持つデータベースで、
  範囲クエリの性能を最適化するデータ構造と
  アルゴリズムを設計してください。
  時間計算量とスペース計算量の分析も含めてください。
`;

const result = await deepThinkAnalysis(problem);
console.log("思考プロセス:", result.thoughts.substring(0, 500) + "...");
console.log("\n最終解答:", result.answer);
```

**期待する出力の概要:**
```
思考プロセス: まず問題の規模を分析します。10億件 ≈ 10^9 レコード。
範囲クエリには B-tree, R-tree, Skip List などが候補...

最終解答:
## 推奨構造: 複合インデックス戦略

1. **プライマリインデックス**: B+木（範囲クエリに最適）
   - 時間計算量: O(log n + k)（k=結果件数）
   ...
```

## よくあるエラーと対処法

### エラー1: RESOURCE_EXHAUSTED（思考トークン上限）

```
google.api_core.exceptions.ResourceExhausted: 429 Quota exceeded
```

Deep Thinkは通常モードより多くのトークンを消費します。特に`thinking_level: "high"`では思考トークンが大量に生成されます。

**対処法:**

```python
# 思考トークンの上限を設定（Gemini 2.5との互換的なアプローチ）
config = types.GenerateContentConfig(
    thinking_config=types.ThinkingConfig(
        thinking_level="medium"  # highからmediumに下げる
    ),
    # または思考を無効化してコスト削減
    # thinking_config=types.ThinkingConfig(thinking_level="minimal")
)
```

### エラー2: temperature設定による挙動の不安定さ

Gemini 3でtemperatureを低く設定すると、Deep Thinkで予期しない結果が出ることがあります。

**対処法:**

```python
# ✅ 正しい: temperatureはデフォルト（1.0）のまま使用
config = types.GenerateContentConfig(
    thinking_config=types.ThinkingConfig(thinking_level="high")
    # temperature は設定しない
)

# ❌ 避ける: 低いtemperatureはDeep Thinkと相性が悪い
config = types.GenerateContentConfig(
    temperature=0.1,  # Deep Thinkでは非推奨
    thinking_config=types.ThinkingConfig(thinking_level="high")
)
```

### エラー3: Thought Signaturesの欠落

マルチターン会話でThought Signaturesを渡さないと、Deep Thinkの文脈が途切れます。

```python
# ✅ 正しい: レスポンスのpartsをそのまま会話履歴に追加
conversation.append({
    "role": "model",
    "parts": response.candidates[0].content.parts  # Signaturesを含む
})

# ❌ 誤り: テキストだけ抽出してしまうとSignaturesが失われる
conversation.append({
    "role": "model",
    "parts": [types.Part(text=response.text)]  # Signaturesが欠落！
})
```

```python
def analyze_research_paper(paper_text: str) -> dict:

        model="gemini-3.1-pro-preview",

        )
    )

    return {"analysis": response.text}
```

## コスト最適化の戦略

Deep Thinkは強力ですが、コストが高くなりがちです。適切な使い分けが重要です。

| タスク種別 | 推奨 thinking_level | 理由 |
|-----------|-------------------|------|
| FAQへの回答 | `minimal` | 事実確認のみ |
| コード補完 | `low` | 定型パターン多い |
| バグ修正・レビュー | `medium` | 適度な分析で十分 |
| 複雑なアルゴリズム設計 | `high` | 深い推論が必要 |
| 数学的証明・科学分析 | `high` | Deep Thinkの本領 |

### 動的な thinking_level 切り替え

```python
def adaptive_thinking(query: str) -> str:
    """
    クエリの複雑さに応じてthinking_levelを自動選択する。
    """
    client = genai.Client()

    # 複雑さの簡易判定（実際のアプリではより高度な判定ロジックを使用）
    if any(kw in query for kw in ["証明", "最適化", "設計", "分析", "アルゴリズム"]):
        level = "high"
    elif any(kw in query for kw in ["実装", "コード", "修正", "レビュー"]):
        level = "medium"
    else:
        level = "low"

    response = client.models.generate_content(
        model="gemini-3-pro-preview",
        contents=query,
        config=types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(thinking_level=level)
        )
    )

    return response.text
```

## FAQ

### Q1: Deep ThinkとGemini 3.1 Pro Previewの推論、どちらを選ぶべきですか？

**A:** 用途によります。Gemini 3.1 Pro Previewは汎用的な知識・推論性能（ARC-AGI-2で77.1%）で優秀です。Deep Thinkは数学・科学の難問（GPQA Diamond 93.8%）や「新しいロジックパターンへの適応」が特に強力です。高度な科学計算や数学証明にはDeep Think、一般的な複雑タスクにはGemini 3.1 Pro Previewを選びましょう。

### Q2: Deep ThinkのAPIコストはどれくらいかかりますか？

**A:** Gemini 3 Pro系のAPIは入力$2.00/Mトークン、出力$12.00/Mトークンです。Deep Think（thinking_level: "high"）では思考トークンが追加で発生し、1リクエストあたりの総コストが通常の2〜5倍になることがあります。`thinking_level: "medium"`に下げるか、不要なタスクに`minimal`を使うことでコストを大幅に削減できます。

### Q3: Deep ThinkはいつGemini APIで一般公開されますか？

**A:** 2026年3月時点では、Deep ThinkはGoogle AI Ultra加入者向けのGeminiアプリと、早期アクセスプログラム経由でのみ利用可能です。API経由での一般公開は2026年中を予定とされていますが、正式な日程は未発表です。Googleの公式ブログやAPIチェンジログを定期的に確認してください。
