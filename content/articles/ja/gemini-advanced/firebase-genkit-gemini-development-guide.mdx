## はじめに

Firebase GenkitはGoogle提供のオープンソースAIアプリケーション開発フレームワークで、Geminiを含む複数のAIモデルをサポートしています。サーバーレス環境での構築に最適化され、複雑なAIワークフロー、エージェント、RAG（Retrieval-Augmented Generation）を簡潔に実装できます。

本ガイドでは、Genkit + Geminiの統合開発から本番デプロイまで、実践的なパターンと最適化手法を詳しく解説します。

## Firebase Genkitとは

Firebase Genkitは、AIアプリケーション開発を加速化するTypeScript/JavaScriptベースのフレームワークです：

**主な特徴:**
- **マルチモデル対応**: Gemini, OpenAI, Claude, Claudeなど複数のLLMを統一インターフェースで使用
- **Flow定義**: 複雑なAIワークフローを宣言的に記述
- **エージェント**: Tool Calling、関数実行、意思決定を自動化
- **RAG統合**: ベクトル検索、知識ベース統合が容易
- **Firebaseエコシステム**: Realtime Database, Firestore, Cloud Functionsと統合
- **開発効率**: 本来数週間かかるAIアプリを数日で構築

## Flow定義

Genkitの**Flow**は、複数のステップを組み合わせたAIワークフローです。Flowは自動的にCloudFunctionsにデプロイ可能です。

### シンプルなFlowの例

```typescript
// src/flows/greeting.ts
import { defineFlow, runFlow } from "@firebase/genkit";
import { gemini15Flash } from "@firebase/genkit-plugins";

export const greetingFlow = defineFlow(
  {
    name: "greeting",
    inputSchema: {
      type: "object",
      properties: {
        name: { type: "string" },
        language: { type: "string", enum: ["ja", "en"] },
      },
    },
  },
  async (input) => {
    const prompt = 
      input.language === "ja" 
        ? `${input.name}さんへの親切な挨拶を日本語で作成してください。`
        : `Create a friendly greeting for ${input.name} in English.`;

    const response = await gemini15Flash.generateText({ prompt });
    return response.text();
  }
);

// ローカルテスト
if (require.main === module) {
  runFlow(greetingFlow, { name: "太郎", language: "ja" });
}
```

**実行:**
```bash
npx ts-node src/flows/greeting.ts

# 出力例:
# 太郎さん、おはようございます！素敵な一日をお過ごしくださいね。
```

### 複雑なマルチステップFlow

```typescript
// src/flows/documentAnalysis.ts
import { defineFlow } from "@firebase/genkit";
import { gemini15Flash } from "@firebase/genkit-plugins";

interface DocumentInput {
  documentText: string;
  analysisType: "summary" | "sentiment" | "keywords";
}

export const documentAnalysisFlow = defineFlow(
  {
    name: "documentAnalysis",
    inputSchema: {
      type: "object",
      properties: {
        documentText: { type: "string" },
        analysisType: { 
          type: "string", 
          enum: ["summary", "sentiment", "keywords"] 
        },
      },
    },
  },
  async (input: DocumentInput) => {
    // ステップ1: テキスト前処理とクリーニング
    const cleanedText = input.documentText.trim().slice(0, 5000);

    // ステップ2: 分析タイプに応じたプロンプト作成
    const prompts = {
      summary: `以下の文書を3行で要約してください:\n${cleanedText}`,
      sentiment: `以下の文書の感情を分析し、ネガティブ/ニュートラル/ポジティブで評価:\n${cleanedText}`,
      keywords: `以下の文書から主要なキーワード5つを抽出:\n${cleanedText}`,
    };

    // ステップ3: Geminiで分析実行
    const analysisPrompt = prompts[input.analysisType];
    const response = await gemini15Flash.generateText({ 
      prompt: analysisPrompt 
    });

    // ステップ4: 結果の構造化
    return {
      analysisType: input.analysisType,
      result: response.text(),
      timestamp: new Date().toISOString(),
      tokenUsage: response.usage,
    };
  }
);
```

## エージェント実装

エージェントは複数のツールを自律的に選択・実行し、目標に向かって行動します。

### Tool Callingの実装

```typescript
// src/tools/webTools.ts
import { defineTool } from "@firebase/genkit";

export const searchWeb = defineTool(
  {
    name: "search_web",
    description: "ウェブ検索を実行（スタブ実装）",
    inputSchema: {
      type: "object",
      properties: {
        query: { type: "string", description: "検索クエリ" },
      },
      required: ["query"],
    },
  },
  async (input) => {
    // 実装例：外部API呼び出し（例：Google Custom Search API）
    return {
      query: input.query,
      results: [
        { title: "Result 1", url: "https://example.com/1", snippet: "..." },
        { title: "Result 2", url: "https://example.com/2", snippet: "..." },
      ],
    };
  }
);

export const getWeather = defineTool(
  {
    name: "get_weather",
    description: "特定の都市の天気情報を取得",
    inputSchema: {
      type: "object",
      properties: {
        city: { type: "string" },
      },
      required: ["city"],
    },
  },
  async (input) => {
    return {
      city: input.city,
      temperature: 25,
      condition: "晴れ",
      humidity: 60,
    };
  }
);
```

### エージェントFlowの実装

```typescript
// src/flows/researchAgent.ts
import { defineFlow } from "@firebase/genkit";
import { gemini15Flash } from "@firebase/genkit-plugins";
import { searchWeb, getWeather } from "../tools/webTools";

export const researchAgentFlow = defineFlow(
  {
    name: "researchAgent",
    inputSchema: {
      type: "object",
      properties: {
        question: { type: "string" },
      },
    },
  },
  async (input) => {
    const tools = [searchWeb, getWeather];

    // エージェント実行
    const agent = gemini15Flash.createAgent({
      tools: tools,
      systemPrompt: `
        あなたは有能なリサーチアシスタント。
        ユーザーの質問に対して、利用可能なツールを使用して情報を収集し、
        総合的な回答を提供してください。
      `,
    });

    const response = await agent.run({
      message: input.question,
    });

    return {
      question: input.question,
      answer: response.text(),
      toolsUsed: response.toolCalls,
    };
  }
);
```

  {
    name: "documentRAG",
    inputSchema: {
      type: "object",
      properties: {
        query: { type: "string" },

      },
    },
  },

    // ステップ1: ユーザークエリをベクトル化

## ベストプラクティスと最適化

### 1. エラーハンドリング

```typescript
export const robustFlow = defineFlow(
  { name: "robustFlow" },
  async (input) => {
    try {
      const response = await gemini15Flash.generateText({ 
        prompt: input.prompt 
      });
      return { success: true, result: response.text() };
    } catch (error) {
      if (error.code === "RATE_LIMIT_EXCEEDED") {
        // リトライロジック
        await new Promise(resolve => setTimeout(resolve, 1000));
        return retryFlow(input);
      }
      return { success: false, error: error.message };
    }
  }
);
```

### 2. コスト最適化

```typescript
// より低コストなモデルを使用
import { gemini15Pro, gemini15Flash } from "@firebase/genkit-plugins";

const cheaperFlow = defineFlow(
  { name: "cheaperFlow" },
  async (input) => {
    // 簡単なタスクはFlashを使用
    if (input.complexity === "low") {
      return await gemini15Flash.generateText({ prompt: input.prompt });
    }
    // 複雑なタスクはProを使用
    return await gemini15Pro.generateText({ prompt: input.prompt });
  }
);
```

### 3. キャッシング

```typescript
export const cachedFlow = defineFlow(
  { name: "cachedFlow" },
  async (input) => {
    const cacheKey = `flow:${JSON.stringify(input)}`;
    
    // キャッシュを確認
    const cached = await getFromCache(cacheKey);
    if (cached) return cached;

    // キャッシュミス時は実行
    const result = await gemini15Flash.generateText({ 
      prompt: input.prompt 
    });

    // 結果をキャッシュに保存
    await saveToCache(cacheKey, result, 3600); // 1時間有効
    
    return result;
  }
);
```

## FAQ

### Q1: Genkitで複数のAIモデルを同時使用できますか？

はい。複数のモデルプラグインを設定し、Flow内で使い分けられます。

### Q2: Genkitはどの言語に対応していますか？

主にTypeScript/JavaScript向けです。Go, Python版の開発も進行中です。

### Q3: 大規模トラフィック下でのスケーリングは？

Cloud Functions の自動スケーリングに対応しており、トラフィック増加に自動的に対応します。

### Q4: データの一貫性はどう保証されますか？

Firestoreのトランザクション機能を活用し、複数ステップの操作を原子的に実行できます。
