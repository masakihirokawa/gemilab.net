## NemoClawがもたらすマルチモデルAIエージェントの新時代

NVIDIAがGTC 2026で発表したNemoClawは、単一のAIモデルに依存しないマルチモデルエージェントの構築を容易にするプラットフォームです。従来、Gemini APIだけで構築していたワークフローを、タスクに応じてGeminiとNemotronを使い分けるアーキテクチャに進化させることができます。

NemoClawの核心は**OpenShell**と呼ばれるオープンソースのランタイムにあり、どのモデルがどのリソースにアクセスできるかをYAMLポリシーで宣言的に管理します。GeminiのマルチモーダルAPIで画像や動画を処理し、NVIDIAのNemotronモデルでローカル推論を高速に実行し、それぞれのセキュリティを個別に管理できるのが大きな特長です。

この記事では、NemoClawプラットフォーム上でGeminiとNemotronを組み合わせたマルチモデルエージェントを構築する方法を、実際のコード例とともに解説します。

## Gemini × Nemotron — それぞれの強みと使い分け

マルチモデルエージェントの鍵は、各モデルの長所を活かすタスク分配にあります。

### Geminiの強み

Gemini 2.5 ProはGoogleのクラウドインフラ上で動作する大規模モデルで、マルチモーダル処理（テキスト・画像・動画・音声）と長いコンテキストウィンドウに強みがあります。複雑な推論、大規模なドキュメント分析、リアルタイムのWeb情報へのアクセスが必要なタスクに最適です。

### Nemotronの強み

NVIDIAのNemotronモデルは、RTX GPUを搭載したローカルマシンで動作し、低レイテンシの推論を実現します。クラウドへの通信が不要なため、機密データの処理やオフライン環境での利用に適しています。NemoClaw経由で利用するとセキュリティポリシーが自動適用されます。

### 使い分けの指針

| タスク | 推奨モデル | 理由 |
|--------|------------|------|
| 画像・動画の分析 | Gemini 2.5 Pro | マルチモーダル処理に強い |
| 機密データの処理 | Nemotron (ローカル) | データが外部に出ない |
| リアルタイムWeb検索 | Gemini 2.5 Pro | Grounding機能で最新情報にアクセス |
| 大量の定型処理 | Nemotron (ローカル) | API費用がかからない |
| 複雑な推論・分析 | Gemini 2.5 Pro | コンテキストウィンドウが大きい |

## NemoClawでのマルチモデルエージェント構築

### Step 1: 環境セットアップ

```bash
# NemoClawのインストール
git clone https://github.com/NVIDIA/NemoClaw.git
cd NemoClaw
./setup.sh --with-openshell --with-nemotron

# Gemini API キーの設定
export GEMINI_API_KEY="your-api-key-here"

# NemoClawの起動確認
nemoclaw status
# 出力例:
# NemoClaw v1.0.0
# OpenShell: active
# Nemotron: loaded (nemotron-mini on RTX 4090)
# External APIs: Gemini API configured
```

### Step 2: マルチモデルポリシーの作成

```yaml
# multi-model-policy.yaml
# Gemini + Nemotron マルチモデル環境のセキュリティポリシー
version: "1.0"
name: "gemini-nemotron-agent"

models:
  gemini:
    # Gemini APIへのアクセス許可
    api_endpoint: "generativelanguage.googleapis.com"
    allowed_capabilities:
      - "text_generation"
      - "multimodal_analysis"
      - "grounding"          # Web検索ベースの回答
    max_tokens_per_request: 8192
    rate_limit: "60/minute"

  nemotron:
    # ローカル推論設定
    runtime: "local"
    gpu: "auto"
    allowed_capabilities:
      - "text_generation"
      - "code_completion"
    max_tokens_per_request: 4096

network:
  allowed_domains:
    - "generativelanguage.googleapis.com"
    - "www.googleapis.com"
  default: "deny"

filesystem:
  read_write:
    - "/workspace/data"
    - "/tmp/agent-cache"
  default: "deny"
```

### Step 3: エージェントのオーケストレーション

```python
# multi_model_agent.py
# NemoClaw上で動作するマルチモデルエージェント
import google.generativeai as genai
import json

# Geminiモデルの初期化
genai.configure(api_key="YOUR_API_KEY")
gemini_model = genai.GenerativeModel("gemini-2.5-pro")

class MultiModelAgent:
    """タスクに応じてGeminiとNemotronを使い分けるエージェント"""

    def __init__(self):
        self.gemini = gemini_model
        # Nemotronはnemoclaw経由で自動ルーティング
        self.task_history = []

    def route_task(self, task: dict) -> str:
        """タスクの内容に応じてモデルをルーティング"""
        if task.get("requires_multimodal"):
            return "gemini"
        if task.get("contains_sensitive_data"):
            return "nemotron"  # ローカル処理で安全に
        if task.get("requires_web_search"):
            return "gemini"  # Grounding機能を利用
        # デフォルトはコスト効率の良いNemotron
        return "nemotron"

    def process_with_gemini(self, prompt: str, image=None) -> str:
        """Gemini APIでタスクを処理"""
        if image:
            # マルチモーダル処理（画像解析など）
            response = self.gemini.generate_content([prompt, image])
        else:
            response = self.gemini.generate_content(prompt)
        return response.text

    def process_with_nemotron(self, prompt: str) -> str:
        """NemotronモデルでローカルFastタスクを処理
        NemoClaw経由でOpenShellポリシーが自動適用される"""
        # nemoclawがローカルNemotronモデルに自動ルーティング
        import subprocess
        result = subprocess.run(
            ["nemoclaw", "infer", "--model", "nemotron-mini",
             "--prompt", prompt],
            capture_output=True, text=True
        )
        return result.stdout.strip()

    def execute(self, task: dict) -> dict:
        """タスクを実行し結果を返す"""
        model = self.route_task(task)
        prompt = task["prompt"]

        if model == "gemini":
            result = self.process_with_gemini(
                prompt, task.get("image")
            )
        else:
            result = self.process_with_nemotron(prompt)

        self.task_history.append({
            "model": model,
            "task": task["prompt"][:50],
            "status": "success"
        })

        return {"model_used": model, "result": result}


# 使用例
agent = MultiModelAgent()

# タスク1: 画像を解析（→ Geminiへルーティング）
task1 = {
    "prompt": "この画像に写っている製品を分析してください",
    "requires_multimodal": True,
    "image": "product_photo.jpg"
}
result1 = agent.execute(task1)
print(f"Model: {result1['model_used']}")
# 出力: Model: gemini

# タスク2: 機密データの要約（→ Nemotronへルーティング）
task2 = {
    "prompt": "この社内文書を要約してください: ...",
    "contains_sensitive_data": True
}
result2 = agent.execute(task2)
print(f"Model: {result2['model_used']}")
# 出力: Model: nemotron
```

## 実践的な活用シナリオ

### コンテンツ制作パイプライン

記事の企画段階ではGemini 2.5 ProのGrounding機能を使ってトレンドリサーチを行い、記事本文のドラフト生成はコスト効率の良いNemotronで処理し、最終的な品質チェックと画像キャプションの生成は再びGeminiで行う——といった分業が自動的に実現します。

### データ分析ダッシュボード

社内のセンシティブなデータはNemotronでローカルに分析し、その分析結果（匿名化済み）をGeminiに送って市場データとの比較レポートを生成するワークフローも実装可能です。機密データが外部に出ないため、データガバナンスの要件を満たしやすくなります。

## よくある質問（FAQ）

### Q: NemotronモデルはどのGPUで動作しますか？

A: Nemotron-miniはRTX 3060（VRAM 12GB）以上で動作します。より大きなNemotronモデルはRTX 4090やDGXが必要です。NemoClawのセットアップスクリプトが利用可能なGPUを自動検出し、適切なモデルを選択します。

### Q: Gemini APIの料金はNemoClawを使うことで変わりますか？

A: NemoClaw自体は追加のAPI料金を発生させません。Gemini APIの料金はGoogle AI Studioの通常料金が適用されます。ただし、Nemotronでローカルに処理可能なタスクをルーティングすることで、結果的にGemini APIの呼び出し回数を削減し、コストを最適化できます。

### Q: OpenShellのポリシーでモデルごとにアクセス権限を分けられますか？

A: はい、これがマルチモデル構成での大きな利点です。例えば「Geminiはネットワークアクセスを許可するがファイル書き込みは禁止」「Nemotronはファイル読み書きを許可するがネットワークは完全遮断」といった細かなポリシーを設定できます。

### Q: 既存のGemini APIプロジェクトにNemoClawを後から追加できますか？

A: はい、可能です。既存のPythonコードはほぼそのままで、NemoClawのラッパーとして`nemoclaw run`コマンドでスクリプトを起動するだけでOpenShellのセキュリティ機能が適用されます。段階的な導入が可能です。
