# Gemini 3 の Computer Use ツール入門 — AI がブラウザを自律操作する

## はじめに

2026年3月、Google は Gemini 3 Flash Preview および Gemini 3 Pro Preview に **Computer Use ツール**のサポートを追加しました。これにより、AI がスクリーンショットを見てブラウザやデスクトップ UI を自律的に操作できる「コンピュータ操作エージェント」を構築できるようになりました。

従来の RPA（ロボティック・プロセス・オートメーション）や Selenium によるブラウザ自動化とは根本的に異なります。Gemini の Computer Use は、画面の内容を理解したうえで、**人間と同じように**クリック・タイピング・スクロールなどの操作を判断・実行します。

本記事では、Computer Use ツールの概念、対応モデル、実装方法、実践的な活用シナリオを、コード例を交えて解説します。

---

## Computer Use ツールとは

Computer Use ツールは、Gemini がスクリーンショット（画面画像）を受け取り、次に実行すべき UI 操作（マウスクリック、キーボード入力、スクロールなど）を判断して返す仕組みです。アプリケーション側がその指示を受け取り、実際の操作を実行します。

### 主なアクションタイプ

| アクション | 説明 |
|-----------|------|
| `click` | 指定座標をクリック |
| `type` | テキストを入力 |
| `key` | キーボードショートカットを送信 |
| `scroll` | 指定方向にスクロール |
| `screenshot` | 現在の画面をキャプチャ（ループ内で使用） |
| `move` | マウスカーソルを移動 |

---

## 対応モデル

Computer Use は以下の Gemini 3 モデルで利用可能です（2026年3月時点）。

| モデル | 特徴 |
|--------|------|
| `gemini-3-flash-preview` | 高速・低コスト。エージェントワークフローに最適 |
| `gemini-3-pro-preview` | 最高精度。複雑な UI 操作に対応 |

> **注意**: Gemini 2.5 シリーズでは Computer Use に専用モデルが必要でしたが、Gemini 3 では Flash・Pro どちらでも標準で利用可能になりました。

## Step 2: 基本的な Computer Use ループの実装

Computer Use の基本は「スクリーンショット → Gemini に送信 → アクション取得 → 実行 → 繰り返し」のループです。

```python
import base64
from google import genai
from google.genai import types

# クライアントの初期化
client = genai.Client(api_key="YOUR_API_KEY")

def encode_screenshot(image_path: str) -> str:
    """スクリーンショットを base64 エンコード"""
    with open(image_path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")

def run_computer_use_step(screenshot_b64: str, task: str, history: list) -> dict:
    """
    スクリーンショットを Gemini に送り、次のアクションを取得
    """
    # Computer Use ツール定義
    computer_use_tool = types.Tool(
        computer_use=types.ComputerUseTool(
            display_width=1280,
            display_height=800,
        )
    )

    # メッセージ構築
    messages = history + [
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/png",
                        "data": screenshot_b64,
                    },
                },
                {
                    "type": "text",
                    "text": f"現在の画面です。タスク: {task}\n次に実行すべき操作を指示してください。",
                },
            ],
        }
    ]

    response = client.models.generate_content(
        model="gemini-3-flash-preview",
        contents=messages,
        config=types.GenerateContentConfig(
            tools=[computer_use_tool],
            system_instruction="あなたはコンピュータを操作するエージェントです。ユーザーのタスクを達成するために、画面を見て最適な操作を実行してください。",
        ),
    )

    return response
```

## Step 3: Playwright と組み合わせたブラウザ操作エージェント

```python
import asyncio
from playwright.async_api import async_playwright
import io
from PIL import Image

async def browser_agent(task: str):
    """ブラウザを自律操作するエージェント"""
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=False)
        page = await browser.new_page(viewport={"width": 1280, "height": 800})

        # 最初のページに移動
        await page.goto("https://www.google.com")
        history = []
        max_steps = 10  # 無限ループ防止

        for step in range(max_steps):
            # 現在の画面をキャプチャ
            screenshot_bytes = await page.screenshot()
            screenshot_b64 = base64.b64encode(screenshot_bytes).decode("utf-8")

            print(f"\n--- Step {step + 1} ---")

            # Gemini にアクションを問い合わせ
            response = run_computer_use_step(screenshot_b64, task, history)

            # レスポンスを解析
            action = None
            for part in response.candidates[0].content.parts:
                if hasattr(part, "computer_use") and part.computer_use:
                    action = part.computer_use
                    break
                elif hasattr(part, "text") and "TASK_COMPLETE" in part.text:
                    print("タスク完了！")
                    await browser.close()
                    return

            if not action:
                print("アクションなし。タスク完了と判断します。")
                break

            # アクションを実行
            print(f"実行: {action.action} - {action}")
            await execute_action(page, action)

            # 履歴に追加（文脈を維持）
            history.append({
                "role": "assistant",
                "content": response.candidates[0].content.parts,
            })

            # 操作後に少し待機
            await asyncio.sleep(0.5)

        await browser.close()

async def execute_action(page, action):
    """Gemini が返したアクションを Playwright で実行"""
    if action.action == "click":
        await page.mouse.click(action.coordinate[0], action.coordinate[1])
    elif action.action == "type":
        await page.keyboard.type(action.text)
    elif action.action == "key":
        await page.keyboard.press(action.key)
    elif action.action == "scroll":
        await page.mouse.wheel(
            action.coordinate[0],
            action.coordinate[1],
            delta_x=action.delta_x if hasattr(action, "delta_x") else 0,
            delta_y=action.delta_y if hasattr(action, "delta_y") else 0,
        )

# 実行
asyncio.run(browser_agent("Google で 'Gemini 3 API' を検索して、最初の結果を開いてください"))
```

## 実践的な活用シナリオ

### シナリオ 1: フォーム自動入力エージェント

繰り返しのデータ入力作業を自動化します。例えば、スプレッドシートのデータを Web フォームに入力するタスクなどに活用できます。

```python
task = """
Google フォームに以下の情報を入力してください:
- 名前: 山田 太郎
- メールアドレス: yamada@example.com
- 希望日程: 2026年4月15日
すべて入力したら「送信」ボタンを押してください。
"""
asyncio.run(browser_agent(task))
```

### シナリオ 2: Web リサーチエージェント

複数のサイトを巡回して情報を収集します。

```python
task = """
以下の手順でリサーチしてください:
1. Google で 'Gemini 3 リリースノート' を検索
2. 公式 Google Developers ブログの記事を開く
3. 記事の主要ポイントを確認したら 'TASK_COMPLETE' と返してください
"""
asyncio.run(browser_agent(task))
```

### シナリオ 3: UI テスト自動化

```python
task = """

1. ログインページに移動（URL: http://localhost:3000/login）
2. ユーザー名に 'test@example.com'、パスワードに 'password123' を入力

確認できたら 'TASK_COMPLETE' を返してください

```

## Computer Use 使用時のベストプラクティス

### 1. タスクは具体的に記述する

曖昧な指示では、AI が誤った操作を実行する可能性があります。「ボタンをクリックして」ではなく、「ページ右上の青い『送信』ボタンをクリックして」のように具体的に記述します。

### 2. 最大ステップ数を設定する

無限ループを防ぐために、必ず `max_steps` などの終了条件を設定します。

### 3. 操作前に確認ステップを入れる

重要な操作（送信、削除など）の前に確認プロンプトを挿入することで、意図しない操作を防止できます。

### 4. エラーハンドリング

ネットワークエラーや予期しない UI 変化に対応するため、try-except を適切に実装します。

```python
try:
    await execute_action(page, action)
except Exception as e:
    print(f"操作エラー: {e}")
    # エラー状態のスクリーンショットを Gemini に送り、回復方法を問い合わせる
```

## よくある質問（FAQ）

**Q: Gemini 2.5 の Computer Use との違いは？**
A: Gemini 2.5 では Computer Use に専用の `gemini-2.5-pro-preview-computer-use` モデルが必要でしたが、Gemini 3 では Flash・Pro ともに標準機能として統合されています。

**Q: ローカル環境での実行はできる？**
A: はい。Playwright をローカルインストールすれば、ローカルのブラウザを使ってエージェントを動かすことができます。クラウド環境には Browserbase なども利用可能です。

**Q: API の料金はどうなっている？**
A: Computer Use 機能自体に追加料金はなく、通常の Gemini API トークン料金が適用されます。スクリーンショット（画像）はマルチモーダル入力として課金されます。

**Q: 動画入力でも使える？**
A: 現時点では静止画（スクリーンショット）が主な入力形式です。高頻度でスクリーンショットを送ることで「動画的な」操作体験を実現できます。
