## Thinking Modeの概要

Gemini Thinking Modeは、AIモデルが複雑な問題を解く過程における「思考」を可視化する機能です。通常、AIは最終的な回答だけを出力しますが、Thinking Modeでは、その回答に至るまでの論理的ステップ、考慮事項、検討過程をすべて出力します。

これは以下のような高難度タスクで特に価値があります：

- 数学的推論と複雑な計算
- コード設計と技術的意思決定
- データ分析と仮説検証
- 複数要因を含むトラブルシューティング

Thinking Modeを活用することで、AIが「どのように」考えているかを理解でき、回答の信頼性を検証できるようになります。

## 思考予算と制御メカニズム

### 思考予算とは

Thinking Modeでは、「思考予算（Thinking Budget）」という概念を使用します。これはAIが思考に割く計算量の制限です。

- **短い思考**: 数秒の思考で簡潔な結果を得る
- **標準的な思考**: バランスの取れた深さと処理時間
- **深い思考**: より複雑な問題を徹底的に分析

```python
import google.generativeai as genai

def analyze_with_thinking(problem: str, budget_tokens: int = 5000) -> dict:
    """Thinking Modeで問題を分析"""
    client = genai.Client(api_key="YOUR_API_KEY")

    # Thinking Modeを使用した分析
    response = client.models.generate_content(
        model="gemini-2.0-pro-exp",
        contents=problem,
        config=genai.GenerateContentConfig(
            thinking={
                "type": "enabled",
                "budget_tokens": budget_tokens
            }
        )
    )

    # 思考プロセスと最終回答を抽出
    thinking_content = ""
    answer_content = ""

    for part in response.content.parts:
        if part.thought:
            thinking_content = part.thought
        elif part.text:
            answer_content = part.text

    return {
        "thinking": thinking_content,
        "answer": answer_content,
        "input_tokens": response.usage_metadata.input_tokens,
        "output_tokens": response.usage_metadata.output_tokens
    }

# 使用例
problem = """
以下の方程式を解いてください:
3x^2 + 5x - 2 = 0

詳細な解法プロセスも説明してください。
"""

result = analyze_with_thinking(problem, budget_tokens=8000)

print("=== AIの思考プロセス ===")
print(result["thinking"][:500] + "...")
print("\n=== 最終回答 ===")
print(result["answer"])
print(f"\n使用トークン - 入力: {result['input_tokens']}, 出力: {result['output_tokens']}")

# 出力例:
# === AIの思考プロセス ===
# この二次方程式を解くには、まず判別式を計算する必要があります。
# 方程式は 3x^2 + 5x - 2 = 0 の形式です。
# a=3, b=5, c=-2 です。判別式D = b^2 - 4ac...
#
# === 最終回答 ===
# この方程式の解は x = 1/3 および x = -2 です。
```

## Chain-of-Thought推論

### Chain-of-Thoughtの実装

Chain-of-Thought（CoT）は段階的な推論を促すテクニックです。Thinking Modeと組み合わせることで、より信頼性の高い結果が得られます。

```python
def solve_reasoning_task(task: str, enable_cot: bool = True) -> dict:
    """段階的推論でタスクを解決"""
    client = genai.Client()

    # CoTを活用したプロンプト設計
    if enable_cot:
        prompt = f"""
        以下のタスクを段階的に解決してください。
        各ステップで以下を明記してください：
        1. 問題の整理
        2. 使用可能な情報
        3. 論理的なステップ
        4. 中間結果
        5. 最終的な結論

        タスク: {task}
        """
    else:
        prompt = f"タスク: {task}"

    response = client.models.generate_content(
        model="gemini-2.0-pro-exp",
        contents=prompt,
        config=genai.GenerateContentConfig(
            thinking={
                "type": "enabled",
                "budget_tokens": 10000
            }
        )
    )

    return {
        "reasoning_steps": response.content.parts[0].text,
        "thinking_process": getattr(response.content.parts[0], 'thought', '')
    }

# 使用例
task = """
商品Aの価格は商品Bの3倍です。
商品Bの価格は商品Cの2倍です。
3つの商品の合計価格は600円です。
各商品の価格を求めてください。
"""

result = solve_reasoning_task(task)

print("推論ステップ:")
print(result["reasoning_steps"])

# 出力例:
# 推論ステップ:
# 1. 問題の整理: 3つの商品の価格関係と合計が与えられている
# 2. 変数設定: C = x、B = 2x、A = 6x
# 3. 方程式: x + 2x + 6x = 600
# 4. 解: 9x = 600、x = 66.67
# 5. 結論: C = 66.67円、B = 133.33円、A = 400円
```

## パフォーマンスと最適化

### 思考予算の効率的な使用

```python
def optimize_thinking_budget(task_complexity: str) -> int:
    """タスクの複雑度に応じて最適な思考予算を決定"""
    budget_map = {
        "simple": 2000,      # 簡単な質問や計算
        "moderate": 5000,    # 標準的な問題解決
        "complex": 10000,    # 複雑な論理推論
        "expert": 20000      # 専門家レベルの分析
    }

    return budget_map.get(task_complexity, 5000)

def batch_analyze_with_thinking(tasks: list) -> list:
    """複数タスクを最適化された予算で分析"""
    client = genai.Client()
    results = []

    for task in tasks:
        complexity = determine_complexity(task)
        budget = optimize_thinking_budget(complexity)

        response = client.models.generate_content(
            model="gemini-2.0-pro-exp",
            contents=task,
            config=genai.GenerateContentConfig(
                thinking={
                    "type": "enabled",
                    "budget_tokens": budget
                }
            )
        )

        results.append({
            "task": task,
            "complexity": complexity,
            "budget_used": budget,
            "answer": response.content.parts[0].text
        })

    return results

def determine_complexity(task: str) -> str:
    """タスク内容から複雑度を判定"""
    keywords = {
        "simple": ["簡単", "単純", "基本"],
        "complex": ["複雑", "高度", "専門的"]
    }

    task_lower = task.lower()
    for level, words in keywords.items():
        if any(word in task_lower for word in words):
            return level
    return "moderate"

# 出力例:
# [{"task": "...", "complexity": "moderate", "budget_used": 5000, "answer": "..."}]
```

## よくある質問

### Q1: Thinking Modeはすべてのモデルで利用可能ですか？

**A:** いいえ。現在、Thinking Modeは`gemini-2.0-pro-exp`および特定の高度なモデルでのみ利用可能です。標準モデルではサポートされていません。利用可能なモデルは随時更新されます。

### Q2: 思考予算が大きいほど常に良い結果が得られますか？

**A:** 必ずしもそうではありません。タスクの複雑度に応じた適切な予算を設定することが重要です。過度に大きい予算はコストの無駄になり、小さすぎるとタスク実行が不完全になります。タスク内容に応じて段階的に調整してください。

### Q3: Thinking Modeはコストに影響しますか？

**A:** はい。思考プロセスの出力はトークン数として計算され、追加のコストが発生します。ただし、より正確な回答により全体的な効率が向上することが多いため、適切に使用すればコスト対効果は高いです。
