## はじめに — 推論はなぜコストが高いのか？

Gemini の推論モード（Deep Think・Adaptive Thinking）を使うと、単純なテキスト生成よりも高い品質の回答が得られます。一方で、**トークンコストが 2～5 倍に跳ね上がる** という課題があります。

本当の問題は、多くの開発者が「推論モード = 必ず使うべき」と考えていることです。実際には、**タスクの複雑度に応じて適切に選択すれば、品質を維持しながらコストを最大 50% 削減** できるのです。

この記事では、Deep Think と Adaptive Thinking の内部メカニズム、そして タスク別の最適な選択戦略 を、実装コード付きで解説します。

## Deep Think vs Adaptive Thinking：内部動作の違い

### Deep Think：深い思考を優先

**Deep Think** は、単一の入力に対して複数の推論パスを探索し、最も確信度の高い答えを返すモードです。内部的には以下のプロセスが走ります：

1. **多数のドラフト生成**：初期仮説を 5～20 個生成
2. **相互検証**：各ドラフト間の矛盾を検出
3. **スコアリング**：最も確信度の高い推論パスを選択
4. **最終出力**：選ばれたパスに基づいて回答を生成

この過程で、実際の出力より **10～30 倍多くの「思考トークン」** が消費されます。ユーザーには見えませんが、内部的には膨大な推論が行われているわけです。

**Deep Think は次のタスクに適しています：**
- 複雑な数学問題の証明
- 法律問題の判断
- 医療診断の推奨
- アルゴリズム設計
- 複数の仮説を検証する必要があるタスク

### Adaptive Thinking：状況に応じた柔軟な推論

**Adaptive Thinking** は、入力の複雑度を自動判定し、必要に応じて推論を行うモードです。単純な質問には軽い推論で対応し、複雑な問題には深い推論を適用します。

内部アルゴリズム：
1. **複雑度判定**：入力テキストのエントロピー・キーワード数・依存関係を分析
2. **推論深度の決定**：判定結果に基づいて推論ステップ数を動的に決定（5～50 ステップ）
3. **段階的推論**：必要に応じて思考を段階的に進める

**トークン消費の特性：**
- 単純な Q&A：4～6 倍の追加トークン
- 中程度の複雑さ：8～15 倍
- 複雑な問題：15～25 倍（Deep Think に近い）

**Adaptive Thinking は次のタスクに適しています：**
- 混在タスク（簡単と難しいが混在）
- リアルタイム処理（レスポンスタイムが重要）
- コスト最適化を優先する用途
- 初期段階での実験・検証

## 内部トークン消費の仕組み

### 思考トークンとコンテンツトークンの計算

```javascript
import { GoogleGenerativeAI } from "@google/generative-ai";

const client = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
const modelDeepThink = client.getGenerativeModel({
  model: "gemini-2.0-pro-deepthink",
});
const modelAdaptive = client.getGenerativeModel({
  model: "gemini-2.0-pro-adaptive-thinking",
});

async function analyzeTokenCost(prompt) {
  // Deep Think: トークン数確認
  const countDeepThink = await modelDeepThink.countTokens(prompt);
  console.log("Deep Think token estimate:", countDeepThink);

  // Adaptive Thinking: トークン数確認
  const countAdaptive = await modelAdaptive.countTokens(prompt);
  console.log("Adaptive Thinking token estimate:", countAdaptive);

  // 実際にリクエストを送信
  const response = await modelAdaptive.generateContent({
    contents: [{ role: "user", parts: [{ text: prompt }] }],
    generationConfig: {
      temperature: 1.0,
      topP: 1.0,
      topK: 40,
      maxOutputTokens: 16000,
    },
  });

  // 実際に消費されたトークン数
  const metadata = response.usageMetadata;
  console.log("=== Actual Usage ===");
  console.log("Input tokens:", metadata.promptTokenCount);
  console.log("Output tokens:", metadata.candidatesTokenCount);
  console.log("Total:", metadata.totalTokenCount);

  return metadata;
}

// 複雑な数学問題でテスト
const complexPrompt = `
以下の不定積分を求めなさい：
∫ (x^3 + 2x^2 - x + 1) * sin(x) dx

また、次の微分方程式を解きなさい：
dy/dx - 2xy = x * e^(x^2)

初期条件: y(0) = 1

計算過程を詳しく示してください。
`;

await analyzeTokenCost(complexPrompt);

// Expected output:
// Deep Think token estimate: {totalTokens: 50}
// Adaptive Thinking token estimate: {totalTokens: 50}
// === Actual Usage ===
// Input tokens: 50
// Output tokens: 8234 (推論トークン + コンテンツトークン)
// Total: 8284
```

**重要な考察：**
- 入力トークン：見積もりと実績がほぼ一致
- 出力トークン：見積もりは 100～150 トークンだが、実績は 1,000～10,000 トークン
- 推論トークンはカウントに含まれるが、**見積もり段階では分からない**

## タスク複雑度別の選択戦略

### 意思決定フローチャート

```
入力を受け取る
  ↓
「推論を必要とするか？」
  ├─ Yes → 複雑度を判定
  │        ↓
  │        「複雑度は？」
  │        ├─ 高（証明、医療判定、法律） → Deep Think
  │        ├─ 中（複合的な分析） → Adaptive Thinking
  │        └─ 低（簡単な応答） → 推論なし
  │
  └─ No → 推論モード不要（通常の Text Generation）
```

### タスク別ガイドライン

```javascript
// Utility: タスク複雑度の自動判定
function detectTaskComplexity(prompt) {
  const indicators = {
    high: [
      "証明して",
      "理由を説明",
      "法的",
      "医療",
      "診断",
      "デバッグ",
      "バグ",
      "修正",
      "最適化",
      "複雑な",
    ],
    medium: [
      "分析",
      "比較",
      "まとめ",
      "要約",
      "翻訳",
      "コード例",
      "説明",
    ],
    low: [
      "是非",
      "存在する",
      "できる",
      "何か",
      "記述",
    ],
  };

  let score = 0;
  for (const keyword of indicators.high) {
    if (prompt.includes(keyword)) score += 3;
  }
  for (const keyword of indicators.medium) {
    if (prompt.includes(keyword)) score += 1;
  }
  for (const keyword of indicators.low) {
    if (prompt.includes(keyword)) score -= 1;
  }

  // 入力の長さもフェクター
  if (prompt.length > 2000) score += 2;

  if (score >= 5) return "high";
  if (score >= 2) return "medium";
  return "low";
}

// 実装例：動的ルーティング
async function routeByComplexity(prompt) {
  const complexity = detectTaskComplexity(prompt);
  const client = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

  let model;
  switch (complexity) {
    case "high":
      model = client.getGenerativeModel({
        model: "gemini-2.0-pro-deepthink",
      });
      console.log("📊 Routing to Deep Think (high complexity)");
      break;

    case "medium":
      model = client.getGenerativeModel({
        model: "gemini-2.0-pro-adaptive-thinking",
      });
      console.log("⚙️  Routing to Adaptive Thinking (medium complexity)");
      break;

    default:
      model = client.getGenerativeModel({
        model: "gemini-2.0-pro",
      });
      console.log("⚡ Routing to standard inference (low complexity)");
  }

  const response = await model.generateContent({
    contents: [{ role: "user", parts: [{ text: prompt }] }],
    generationConfig: {
      temperature: complexity === "high" ? 1.0 : 0.7,
      maxOutputTokens: complexity === "high" ? 16000 : 2048,
    },
  });

  return response;
}

// テスト
const lowComplexityTask = "日本の首都は何ですか？";
const mediumTask = "LLM の学習と推論のプロセスを簡潔に説明してください。";
const highComplexityTask = "整数論における フェルマーの最終定理 の証明概要を説明してください。";

await routeByComplexity(lowComplexityTask);
await routeByComplexity(mediumTask);
await routeByComplexity(highComplexityTask);
```

### 2. キャッシング戦略（Context Caching）

同じ長文ドキュメント（PDFの内容、API ドキュメント、コード）を何度も分析する場合、**Context Caching** でコストを最大 90% 削減できます。

```javascript
import axios from "axios";

async function callWithCaching(
  cachedContext,
  question,
  useCaching = true
) {
  const apiKey = process.env.GEMINI_API_KEY;

  const payload = {
    contents: [
      {
        role: "user",
        parts: [
          {
            text: `以下のドキュメントを参考に、質問に答えてください。\n\n${
              useCaching
                ? "[Context will be cached on first request]"
                : cachedContext
            }\n\n質問: ${question}`,
          },
        ],
      },
    ],
    generationConfig: {
      temperature: 0.7,
      maxOutputTokens: 1024,
    },
  };

  const headers = {
    "Content-Type": "application/json",
    "x-goog-api-key": apiKey,
  };

  if (useCaching) {
    // Context Cache ヘッダーを追加
    headers["google-ai-cache-control"] = "cache-control=max-age=3600"; // 1 hour cache
  }

  const response = await axios.post(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-pro-adaptive-thinking:generateContent",
    payload,
    { headers }
  );

  const metadata = response.data.usageMetadata;
  console.log("Input tokens:", metadata.promptTokenCount);
  console.log("Cache creation tokens:", metadata.cacheCreationTokenCount || 0);
  console.log("Cache read tokens:", metadata.cacheReadTokenCount || 0);
  console.log("Output tokens:", metadata.candidatesTokenCount);

  return response.data;
}

// 500KB のドキュメントを複数の質問で分析
const largeDocument = "...（複数の API ドキュメント、5000 行以上）...";

// 1回目：キャッシュ生成
console.log("--- First request (cache creation) ---");
await callWithCaching(largeDocument, "このAPIの認証方法は？", true);

// 2～10 回目：キャッシュ読み込み（90% コスト削減）
for (let i = 2; i <= 10; i++) {
  console.log(`\n--- Request ${i} (cache hit) ---`);
  await callWithCaching(
    largeDocument,
    `質問 ${i}: このAPI${i}の使用方法は？`,
    true
  );
}
```

### 3. バッチ vs ストリーミング：コスト比較

```javascript
// バッチ処理：50% 割引、非同期
async function batchAnalysis(questions) {
  const requests = questions.map((q, i) => ({
    custom_id: `q-${i}`,
    params: {
      model: "gemini-2.0-pro-adaptive-thinking",
      contents: [
        {
          role: "user",
          parts: [{ text: q }],
        },
      ],
    },
  }));

  // JSON Lines 形式で送信
  const batchFile = requests.map(r => JSON.stringify(r)).join("\n");

  const formData = new FormData();
  formData.append("file", new Blob([batchFile], { type: "text/plain" }));

  const response = await axios.post(
    "https://generativelanguage.googleapis.com/v1beta/batches",
    formData,
    {
      headers: {
        "x-goog-api-key": process.env.GEMINI_API_KEY,
      },
    }
  );

  console.log("Batch submitted. Cost: -50% compared to normal requests");
  return response.data.name;
}

// ストリーミング：リアルタイム、通常料金
async function streamAnalysis(question) {
  const response = await fetch(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-pro-adaptive-thinking:streamGenerateContent",
    {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "x-goog-api-key": process.env.GEMINI_API_KEY,
      },
      body: JSON.stringify({
        contents: [
          {
            role: "user",
            parts: [{ text: question }],
          },
        ],
      }),
    }
  );

  console.log("Streaming response. Cost: standard rate");
  return response;
}

// 使い分け：100 個の質問がある場合
// バッチ: 処理時間 5 分、コスト ¥500
// ストリーミング: 処理時間 5 秒、コスト ¥1,000
```

## API 設定のベストプラクティス

### 推論設定の最適化

```javascript
// 💡 重要：推論モデルの temperature は常に 1.0 に設定
async function inferenceWithOptimalConfig(prompt, taskComplexity = "medium") {
  const client = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

  const modelConfigs = {
    high: {
      model: "gemini-2.0-pro-deepthink",
      temperature: 1.0,          // 深い思考には高い randomness が必須
      maxOutputTokens: 16000,
      topP: 1.0,
      topK: 40,
    },
    medium: {
      model: "gemini-2.0-pro-adaptive-thinking",
      temperature: 1.0,          // Adaptive も 1.0 推奨
      maxOutputTokens: 8000,
      topP: 0.95,
      topK: 40,
    },
    low: {
      model: "gemini-2.0-pro",
      temperature: 0.7,          // 標準推論は 0.7 で OK
      maxOutputTokens: 2048,
      topP: 0.9,
      topK: 40,
    },
  };

  const config = modelConfigs[taskComplexity];
  const model = client.getGenerativeModel({
    model: config.model,
    generationConfig: config,
  });

  const response = await model.generateContent({
    contents: [{ role: "user", parts: [{ text: prompt }] }],
  });

  return response;
}

// ❌ 間違った設定（推論が浅くなる）
// temperature: 0.3 // 低すぎる
// topK: 10        // 選択肢が制限される

// ✅ 正しい設定
// temperature: 1.0
// topP: 1.0
// topK: 40
```

## FAQ

### Q1: Deep Think と Adaptive Thinking、どちらを使うべき？

**A:** 以下の判定基準を使用してください：
- **証明・診断・検証が必要** → Deep Think
- **リアルタイム性が重要** → Adaptive Thinking（デフォルト）
- **初期段階で実験中** → Adaptive Thinking（コスト最適化）
- **最高精度が必須** → Deep Think（コストは気にしない）

### Q2: 推論トークン数の見積もりは可能ですか？

**A:** 正確な見積もりは困難ですが、推算は可能です：
- 入力トークン数は `countTokens()` で正確に取得可能
- 出力トークン数は、前知識として 3～10 倍の範囲で考えてください
- Context Caching を使えば、実質的な入力コスト は 90% 削減

### Q3: キャッシングの有効期間はどのくらい？

**A:** デフォルトは 1 時間です。Google Cloud Console で最長 24 時間まで延長可能です。キャッシュ有効期間中は同じドキュメントに対する複数の質問で利用できます。

### Q4: バッチ処理と Context Caching は併用できる？

**A:** はい。併用すれば、バッチの 50% 割引 + キャッシュの 90% 削減 = **最大 95% のコスト削減** が可能です。

### Q5: 推論モデルは日本語に対応していますか？

**A:** はい、完全に対応しています。Deep Think・Adaptive Thinking は日本語で複雑な推論を行えます。数学・論理・法律分野で特に優秀です。
