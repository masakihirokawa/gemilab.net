## Gemini App Actions とは：スマートフォンが『自動執行アシスタント』に進化

2026年3月の Pixel Drop で導入された **Gemini App Actions** は、これまでのAIアシスタントの概念を大きく拡張するものです。従来の「検索・提案型」から、実際に**複数のアプリを横断して操作を実行する『エージェント AI』** へと進化しました。

### App Actions の具体的な動作例

「土曜日の午後、渋谷で映画を見た後、近所のイタリアンレストランで夕食の予約を取って、そこまで Uber で行ってください」という自然言語の指示に対し、Gemini は以下のステップを **ユーザーの明示的な確認なしに実行** します：

1. **Google Maps**で映画館から近いイタリアンレストランを検索
2. **Tabelog** や **食べログ** のアプリで評価・空席状況を確認
3. **ホットペッパー** でテーブル予約を実行
4. **Uber** で配車リクエストを送信
5. 予定をカレンダーに自動追加

これまでのスマートフォン操作では、各アプリを手動で起動し、5～10分の入力作業が必要でした。**App Actions なら、30秒の音声指示で全て完了** します。

## App Actions が実現する背景：Android 16 QPR3 と Magic Cue

### Android 16 QPR3 の『Agentic Framework』

Google は Android 16 の Quarterly Platform Release 3（QPR3）で、**Agentic Framework** という新しい SDK を導入しました。これは開発者が自社アプリを「Gemini が直接操作できる」状態に登録するための標準規格です。

```xml
<!-- AndroidManifest.xml の例 -->
<service
    android:name=".GeminiAgentService"
    android:exported="true">
    <intent-filter>
        <action android:name="com.google.gemini.ACTION_AGENT" />
    </intent-filter>
    <meta-data
        android:name="com.google.gemini.capabilities"
        android:value="@xml/agent_capabilities" />
</service>
```

このメタデータを宣言することで、Gemini はあなたのアプリが「どのような操作に対応しているか」を自動認識し、必要に応じてアプリを実行します。

### Magic Cue：Gemini チャット内でのビジュアル操作支援

**Magic Cue** は、Gemini のチャットウィンドウ内で「次に実行する操作」をビジュアル表示する機能です。

```
ユーザー: 「来週の月曜日、オフィス近くで昼食予約を」

Gemini（Magic Cue 表示）:
┌─────────────────────────────────────────┐
│ 以下の手順を実行します：                   │
│ 1️⃣  Google Maps で飲食店を検索           │
│ 2️⃣  食べログで在庫を確認                 │
│ 3️⃣  予約を実行                           │
│                                          │
│  [確認して実行]  [キャンセル]              │
└─────────────────────────────────────────┘
```

ユーザーが「確認して実行」をタップすると、複数アプリの連鎖操作が自動実行されます。

## Circle to Search の進化：画像からのアクション生成

App Actions と並んで、**Circle to Search** も大きく進化しました。

例えば、雑誌の写真から「この靴が欲しい」と感じた場合：

1. Circle to Search で靴の画像を囲む
2. Gemini が「楽天・Amazon・ZOZOTOWN で在庫・価格を検索」と提案
3. ユーザーが「Amazon で購入手続きして」と指示
4. Gemini が Amazon アプリを起動し、購入フローを実行

**画像認識 × App Actions の組み合わせ** により、「発見から購買」までが最短 30 秒で完結します。

## 開発者向け：App Actions への統合方法

### Step 1: Agentic Capabilities の定義

アプリケーション側で、Gemini が実行可能な操作を定義します。

```xml
<!-- res/xml/agent_capabilities.xml -->
<?xml version="1.0" encoding="utf-8"?>
<agent-capabilities xmlns:android="http://schemas.android.com/apk/res/android">
    <capability
        android:name="reserve_table"
        android:label="飲食店の予約"
        android:description="テーブル予約システム" />

    <capability
        android:name="check_availability"
        android:label="空席確認"
        android:description="リアルタイム空席状況" />

    <capability
        android:name="list_menu"
        android:label="メニュー表示"
        android:description="店舗メニューの詳細表示" />
</agent-capabilities>
```

### Step 2: GeminiAgentService の実装

```kotlin
class GeminiAgentService : Service() {

    override fun onBind(intent: Intent?): IBinder? {
        return GeminiAgentBinder()
    }

    inner class GeminiAgentBinder : IGeminiAgent.Stub() {

        override fun executeAction(
            action: String,
            params: Bundle
        ): Bundle {
            return when (action) {
                "reserve_table" -> handleTableReservation(params)
                "check_availability" -> handleAvailabilityCheck(params)
                else -> Bundle().apply { putString("error", "Unknown action") }
            }
        }

        private fun handleTableReservation(params: Bundle): Bundle {
            val restaurantId = params.getString("restaurant_id") ?: ""
            val partySize = params.getInt("party_size", 2)
            val datetime = params.getString("datetime") ?: ""

            // API 呼び出し
            val result = reserveTable(restaurantId, partySize, datetime)

            return Bundle().apply {
                putBoolean("success", result.isSuccess)
                putString("confirmation_id", result.confirmationId)
                putString("message", result.message)
            }
        }
    }
}
```

### Step 3: 権限の申請と Google Play Console への登録

Google Play Console で、アプリを「Gemini 対応」として登録します。

```
Google Play Console
  → あなたのアプリ
    → 設定 → 高度な設定
      → 「Gemini Agent API 対応」をチェック
```

この登録により、ユーザーが 100 万人を超えたとき、Google が自動的にあなたのアプリを Gemini の推奨リストに追加します。

## 実務例：飲食店予約アプリの App Actions 統合

飲食店検索・予約アプリが App Actions に対応した場合の実装フローです：

```kotlin
// 1. Gemini からのアクション受信
private fun handleTableReservation(params: Bundle): Bundle {
    val restaurantId = params.getString("restaurant_id")
    val partySize = params.getInt("party_size")
    val datetime = params.getString("datetime")

    // 2. ローカルデータベースで店舗情報を取得
    val restaurant = db.getRestaurant(restaurantId)

    // 3. 在庫確認（複数のタイムゾーン・フロアを考慮）
    val availableSlots = restaurant.checkAvailability(
        datetime,
        partySize
    )

    if (availableSlots.isEmpty()) {
        return Bundle().apply {
            putBoolean("success", false)
            putString("message", "ご指定の時間は満席です")
            putStringArray("alternatives", generateAlternatives())
        }
    }

    // 4. 予約確定
    val reservation = Reservation(
        restaurantId = restaurantId,
        partySize = partySize,
        datetime = datetime,
        userId = getUserId(),
        status = "confirmed"
    )

    db.saveReservation(reservation)
    sendConfirmationEmail(reservation)

    // 5. Gemini への結果返信
    return Bundle().apply {
        putBoolean("success", true)
        putString("confirmation_id", reservation.id)
        putString("message", "予約完了：${restaurant.name} ${partySize}名様 ${datetime}")
    }
}
```

## よくある質問（FAQ）

**Q1: App Actions は全 Android デバイスで利用可能か？**

A: 現在は **Pixel 8 以降** で利用可能です。2026 年中には他社 Android デバイス（Samsung Galaxy、OPPO 等）への展開が予定されています。

**Q2: Gemini が無断で操作を実行することに対する懸念はないか？**

A: Google は「重要なアクション（支払い・予約確定）には必ず Magic Cue で確認ステップを挿入」という原則を厳守しており、完全な自動実行は起こりません。すべての操作はユーザーの確認後に実行されます。

**Q3: App Actions 統合にはコストがかかるか？**

A: **完全に無料**です。Google Play Console への登録のみで対応可能です。ただし、Agentic Framework SDK の学習コストがあります。

**Q4: Gemini が私のアプリを勝手に起動することはないか？**

A: ユーザーの明示的な指示（音声・テキスト）がある場合のみ起動します。無断での起動は設計上発生しません。ユーザーはいつでも「このアプリの使用を許可しない」に設定できます。
