## Gemini Adaptive Thinking とは？

Gemini を使っていて、「簡単な質問にも時間がかかる」あるいは「難しい問題なのに浅い回答しか返ってこない」と感じたことはないでしょうか。**Adaptive Thinking** は、まさにこの課題を解決するために設計された機能です。

Adaptive Thinking は、ユーザーが送信したクエリの複雑さをGeminiが自動的に評価し、最適な推論深度を動的に選択する仕組みです。たとえば「東京の天気は？」のようなシンプルな質問には最小限の処理で素早く応答し、「量子コンピュータのエラー訂正アルゴリズムを比較して」のような高度な問題には、Deep Thinkに近い深い推論チェーンを自動的に展開します。

従来の `thinking_level` パラメータでは、開発者が `low` / `medium` / `high` を明示的に指定する必要がありました。Adaptive Thinking はこの選択をAI自身に委ね、**レイテンシ・コスト・回答品質のバランスを自動最適化**します。

## Adaptive Thinking と既存機能の違い

Gemini の推論制御には複数の選択肢があります。それぞれの違いを理解しておくと、適切な場面で適切な機能を選べます。

| 機能 | 推論深度の決定 | 主な用途 |
|---|---|---|
| **Adaptive Thinking** | AIが自動判定 | 汎用チャットボット、不特定多数のクエリ処理 |
| [thinking_level パラメータ](/articles/gemini-api/thinking-level-guide) | 開発者が手動指定 | レイテンシ要件が厳しいAPI、コスト管理重視 |
| [Deep Think](/articles/gemini-advanced/gemini-3-deep-think-guide) | 常に最大深度 | 科学・数学・コード生成の難問 |

Adaptive Thinking は「どの推論深度が最適かわからない」場面、つまり多様なクエリが飛んでくるチャットアプリケーションやカスタマーサポートBotで特に威力を発揮します。

## API での Adaptive Thinking の使い方

Gemini API で Adaptive Thinking を有効にするには、`thinking_config` の `thinking_level` を `"adaptive"` に設定します。以下は Python SDK を使った基本的な実装例です。

```python
from google import genai
from google.genai.types import ThinkingConfig

# クライアント初期化
client = genai.Client(api_key="YOUR_API_KEY")

# Adaptive Thinking を有効化
response = client.models.generate_content(
    model="gemini-3-pro",
    contents="フーリエ変換の直感的な説明と、音声処理での応用例を教えてください",
    config={
        "thinking_config": ThinkingConfig(
            thinking_level="adaptive"  # AIが推論深度を自動選択
        )
    }
)

# レスポンスの確認
for part in response.candidates[0].content.parts:
    if part.thought:
        print(f"[思考プロセス] {part.text[:200]}...")
    else:
        print(f"[回答] {part.text[:200]}...")

# 使用トークン数の確認（推論深度が反映される）
print(f"入力トークン: {response.usage_metadata.prompt_token_count}")
print(f"出力トークン: {response.usage_metadata.candidates_token_count}")
print(f"思考トークン: {response.usage_metadata.thoughts_token_count}")
```

**期待される出力:**

```
[思考プロセス] フーリエ変換について直感的な説明を求められている。数式だけでなく...
[回答] フーリエ変換を一言で表すなら「音のレシピを分解する虫めがね」です...
入力トークン: 28
出力トークン: 1250
思考トークン: 480
```

`thoughts_token_count` の値がクエリの複雑さに応じて変動する点に注目してください。簡単なクエリでは 0 に近く、複雑なクエリでは数百〜数千トークンの思考が自動的に展開されます。

## JavaScript (Node.js) での実装例

TypeScript / JavaScript プロジェクトでも同様に Adaptive Thinking を利用できます。

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: "YOUR_API_KEY" });

async function adaptiveQuery(prompt) {
  const response = await ai.models.generateContent({
    model: "gemini-3-pro",
    contents: prompt,
    config: {
      thinkingConfig: {
        thinkingLevel: "adaptive", // 推論深度を自動調整
      },
    },
  });

  const parts = response.candidates[0].content.parts;
  const thought = parts.find((p) => p.thought)?.text || "(思考なし)";
  const answer = parts.find((p) => !p.thought)?.text || "";

  return {
    thought: thought.slice(0, 200),
    answer,
    tokens: {
      input: response.usageMetadata.promptTokenCount,
      output: response.usageMetadata.candidatesTokenCount,
      thinking: response.usageMetadata.thoughtsTokenCount,
    },
  };
}

// 簡単なクエリ
const simple = await adaptiveQuery("日本の首都はどこですか？");
console.log("簡単なクエリ:", simple.tokens);
// => { input: 12, output: 15, thinking: 0 }

// 複雑なクエリ
const complex = await adaptiveQuery(
  "マイクロサービスアーキテクチャでのサーガパターンとイベントソーシングの設計上のトレードオフを分析してください"
);
console.log("複雑なクエリ:", complex.tokens);
// => { input: 35, output: 2100, thinking: 850 }
```

## 実践的なユースケース

### 1. カスタマーサポート Bot

ユーザーの問い合わせは「営業時間は？」のような単純なものから「請求書の二重課金を調査してほしい」のような複雑な調査まで幅広いです。Adaptive Thinking を使えば、単純な質問には即座に応答しつつ、複雑な問い合わせには深い分析を自動で行えます。

```python
from google import genai
from google.genai.types import ThinkingConfig

client = genai.Client(api_key="YOUR_API_KEY")

# サポートBot用のシステムプロンプト
system_prompt = """あなたはECサイトのカスタマーサポート担当です。
- 簡単な質問には簡潔に回答してください
- 複雑な問い合わせには詳細な調査と段階的な解決策を提供してください
- 不明点がある場合は確認質問をしてください"""

def support_chat(user_message):
    response = client.models.generate_content(
        model="gemini-3-pro",
        contents=user_message,
        config={
            "system_instruction": system_prompt,
            "thinking_config": ThinkingConfig(thinking_level="adaptive"),
        }
    )
    return response.text
```

### 2. コードレビューアシスタント

提出されたコードの複雑さに応じて、レビューの深さが自動調整されます。タイポの修正には浅いレビュー、アーキテクチャの問題には深い分析が行われます。

### 3. 教育プラットフォーム

学生の質問レベルに応じて、簡潔な回答と詳細な解説を自動的に切り替えられます。基礎的な質問には端的な答えを、応用的な質問には段階的な説明を提供できます。

## Adaptive Thinking のコスト最適化効果

Adaptive Thinking の最大のメリットは、**不要な推論コストの削減**です。すべてのリクエストに `thinking_level: "high"` を設定した場合と比較すると、実際のワークロードでは大幅なコスト削減が期待できます。

| シナリオ | thinking_level: high | Adaptive Thinking | 削減率 |
|---|---|---|---|
| FAQ応答（80%が単純質問） | 全リクエストで深い推論 | 80%は推論スキップ | 約60〜70% |
| コードレビュー（混在） | 全リクエストで深い推論 | 複雑さに応じて変動 | 約30〜40% |
| 研究支援（大半が複雑） | 全リクエストで深い推論 | ほぼ同等 | 約5〜10% |

特にFAQやカスタマーサポートのように、**単純なクエリの比率が高いワークロード**ほど Adaptive Thinking のコスト削減効果は大きくなります。

## thinking_level との使い分けガイド

「Adaptive Thinking と手動の thinking_level、どちらを使うべきか？」という疑問に対する指針を示します。

**Adaptive Thinking が適している場面:**

- クエリの複雑さが予測できない（チャットBot、汎用アシスタント）
- コストとレイテンシを自動最適化したい
- 推論深度のチューニングに時間をかけたくない

**手動の thinking_level が適している場面:**

- レイテンシの上限が厳密に決まっている（リアルタイムAPI）
- 常に深い推論が必要（数学・科学の専門ツール）
- コストの予測可能性を重視する（固定予算のプロジェクト）

両方を組み合わせることも可能です。たとえば、通常は Adaptive Thinking を使い、特定のエンドポイント（例: 数学ソルバー）では `thinking_level: "high"` を固定するという戦略が有効です。

## よくある質問（FAQ）

### Q1: Adaptive Thinking はどのモデルで利用できますか？

Adaptive Thinking は Gemini 3 Pro および Gemini 3 Flash で利用できます。Gemini 2.5 系モデルでは `thinking_level` パラメータのみ対応しており、`"adaptive"` を指定するとエラーになります。利用可能なモデルの一覧は [Gemini API ドキュメント](https://ai.google.dev/)で確認できます。

### Q2: Adaptive Thinking を使うと、Thought Signatures は取得できますか？

はい、取得できます。Adaptive Thinking で深い推論が実行された場合、[Thought Signatures](/articles/gemini-advanced/gemini-thought-signatures-guide) が応答に含まれます。ただし、推論がスキップされた場合（単純なクエリ）は Thought Signatures は生成されません。`thoughts_token_count` が 0 の場合は推論がスキップされたことを意味します。

### Q3: Adaptive Thinking とストリーミングは併用できますか？

はい、問題なく併用できます。ストリーミング応答では、まず思考パート（`thought: true`）がストリームされ、次に回答パートが続きます。推論がスキップされた場合は、思考パートなしで直接回答がストリームされるため、体感速度がさらに向上します。

### Q4: 推論深度の自動判定を確認する方法はありますか？

`usage_metadata` の `thoughts_token_count` を監視することで、各リクエストでどの程度の推論が行われたかを確認できます。この値をログに記録しておくと、ワークロードの特性把握やコスト分析に役立ちます。

## まとめ

Gemini Adaptive Thinking は、推論深度の自動最適化というシンプルなコンセプトでありながら、コスト削減・レイテンシ改善・回答品質の向上を同時に実現する強力な機能です。特に多様なクエリを処理するアプリケーションでは、`thinking_level` を `"adaptive"` に設定するだけで大きな効果を得られます。

手動の [thinking_level パラメータ](/articles/gemini-api/thinking-level-guide)や [Deep Think](/articles/gemini-advanced/gemini-3-deep-think-guide) と組み合わせることで、Gemini の推論能力を最大限に引き出しましょう。
