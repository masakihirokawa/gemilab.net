## Gemini がスマホを代わりに操作する時代が到来

「コーヒーをいつもの店で注文しておいて」「今日のランチをUber Eatsで頼んで」——こんな指示を音声でするだけで、Gemini がアプリを開き、注文を完了してくれる。これが **Gemini Pixel Actions**（スクリーンオートメーション）です。

2026年2月にサムスン Galaxy S26 シリーズで先行提供された後、2026年3月18日に **Pixel 10、Pixel 10 Pro、Pixel 10 Pro XL** への本格ロールアウトが開始されました。Google が目指す「AI エージェントがスマホを代わりに操作する未来」が、いよいよ現実のものになりつつあります。

## Pixel Actions とは：仕組みを理解する

### 基本コンセプト

Gemini Pixel Actions は、ユーザーの自然言語による指示を受けて、Gemini が**スマートフォンの画面を直接操作**するオートメーション機能です。従来の音声アシスタントとは根本的に異なり、API 連携ではなく**実際にアプリの UI を操作**して目的のタスクを完了します。

### 動作の流れ

Pixel Actions は次のステップで動作します。

1. **指示の受付**: 電源ボタン長押しまたは「Hey Google」で Gemini を起動し、タスクを依頼する
2. **タスク分解**: Gemini がリクエストを複数のステップに分解する
3. **画面操作**: 仮想ウィンドウ内でアプリを起動し、タップ・スワイプ・テキスト入力を自動実行する
4. **確認ポイント**: 曖昧な選択肢（サイズ選択、店舗選択など）がある場合はユーザーに質問する
5. **最終確認**: 決済や送信の直前で必ずユーザーの承認を求める

重要なのは、**決済を伴う操作では最終確認が必ず表示される**点です。意図しない注文が勝手に完了することはありません。

## 設定方法：Pixel Actions を有効にする

### ステップバイステップの設定手順

Pixel 10 シリーズでの設定手順を解説します。

**1. システムアップデートの確認**

まず March 2026 Pixel Drop が適用されていることを確認します。「設定」→「システム」→「システムアップデート」から最新の状態にしてください。

**2. Gemini アプリの設定**

Gemini アプリを開き、右上のプロフィールアイコンをタップ → 「設定」→ 「**スクリーンオートメーション**」（Screen automation）を探します。

**3. 権限の付与**

スクリーンオートメーションを有効にすると、以下の権限が求められます。

- ユーザー補助サービスへのアクセス
- 画面表示内容の読み取り
- タップ・スワイプ操作の実行

**4. 対応アプリの確認**

設定画面内で、現在対応しているアプリの一覧を確認できます。

### 対応アプリ一覧（2026年3月時点、米国・韓国）

|---------|-------|
| フードデリバリー | DoorDash、Grubhub、Uber Eats、McDonald's |
| ライドシェア | Uber、Lyft |
| カフェ注文 | Starbucks |

日本での対応アプリは今後の発表を待つ必要がありますが、グローバル展開のロードマップが示されていることから、出前館、Uber Eats Japan などへの対応が期待されます。

## 実践的な活用シナリオ

### シナリオ 1：いつものコーヒーを再注文

```
「Hey Google、スターバックスでいつもの注文をお願い」
```

Gemini が Starbucks アプリを開き、過去の注文履歴から「いつもの」ドリンクを特定して注文画面を準備します。サイズや受取店舗の確認後、最終的な注文確認が表示されます。

### シナリオ 2：夕食のデリバリー注文

```
「今夜の夕食をDoorDashで頼んで。前回と同じお寿司がいい」
```

Gemini が DoorDash を起動し、注文履歴から前回の寿司レストランを見つけ、同じメニューをカートに追加します。

```
「空港までUberを呼んで」
```

Gemini が Uber アプリを起動し、現在地から空港までのルートを設定。車種の選択を聞かれた場合はユーザーに確認します。

### 開発者向け：操作の流れをコードで理解する

Pixel Actions は内部的に Android のアクセシビリティフレームワークを活用しています。開発者が自分のアプリを Pixel Actions に対応させるための公式 SDK はまだ発表されていませんが、アクセシビリティ対応が鍵となります。

```python
# Pixel Actions の内部動作を模擬的に示す擬似コード
# 実際の API ではなく、概念理解のためのイメージです

class PixelActionsAgent:
    """Gemini Pixel Actions の動作フローを示す擬似クラス"""

    def __init__(self, user_request: str):
        self.request = user_request
        self.steps = []
        self.requires_confirmation = False

    def plan_task(self) -> list[dict]:
        """
        ユーザーリクエストを実行ステップに分解
        例: "スタバでコーヒー注文" → アプリ起動 → メニュー選択 → 注文確定
        """
        # Gemini がリクエストを解析し、ステップを生成
        self.steps = [
            {"action": "launch_app", "target": "com.starbucks.mobilecard"},
            {"action": "navigate", "target": "order_screen"},
            {"action": "select_item", "target": "previous_order"},
            {"action": "confirm", "requires_user_approval": True},
        ]
        return self.steps

    def execute_step(self, step: dict) -> str:
        """各ステップを画面操作で実行"""
        if step.get("requires_user_approval"):
            # 決済前は必ずユーザー確認を表示
            return "AWAITING_USER_CONFIRMATION"
        # アクセシビリティ API 経由で UI 要素を操作
        return "STEP_COMPLETED"

    def run(self):
        """タスク全体を実行"""
        self.plan_task()
        for step in self.steps:
            result = self.execute_step(step)
            if result == "AWAITING_USER_CONFIRMATION":
                print(f"確認が必要: {step['action']}")
                break
            print(f"完了: {step['action']}")

# 実行例
# 期待される出力:
# 完了: launch_app
# 完了: navigate
# 完了: select_item
# 確認が必要: confirm
agent = PixelActionsAgent("スタバでいつものコーヒーを注文して")
agent.run()
```

このコードは実際のAPIではなく概念理解のための擬似コードですが、Pixel Actions が「タスク分解 → 順次実行 → 確認ポイント」という流れで動作する仕組みを示しています。

## 開発者がアプリを対応させるためのベストプラクティス

自分のアプリを Pixel Actions に対応させたい開発者は、まずアクセシビリティ対応を強化することが重要です。

### 1. コンテンツディスクリプションの充実

```kotlin
// Android の View に適切なコンテンツディスクリプションを設定
// Gemini がUIを「理解」するための重要な情報源
button.contentDescription = "カートに追加: マルゲリータピザ Lサイズ 1,800円"

// RecyclerView のアイテムにも詳細な説明を付与
itemView.contentDescription = buildString {
    append("商品: ${product.name}")
    append(", 価格: ${product.formattedPrice}")
    append(", ${if (product.inStock) "在庫あり" else "在庫切れ"}")
}
```

### 2. アクセシビリティツリーの最適化

```kotlin
// 重要なアクションは ViewCompat で明示的にマーク
ViewCompat.addAccessibilityAction(orderButton, "注文を確定する") { _, _ ->
    submitOrder()
    true
}

// ナビゲーション構造を明確にする
toolbar.importantForAccessibility = View.IMPORTANT_FOR_ACCESSIBILITY_YES
```

### 3. テスト方法

現時点で公式のテストツールはありませんが、Android の **TalkBack** を有効にしてアプリを操作し、すべての重要な操作がスクリーンリーダーで実行可能であることを確認しましょう。Pixel Actions はアクセシビリティフレームワークの上に構築されているため、TalkBack で問題なく操作できるアプリは Pixel Actions でも正しく動作する可能性が高くなります。

## 注意点とプライバシーへの配慮

### セキュリティ面のポイント

- **決済の最終確認は必ず表示される**: Gemini が勝手に支払いを完了することはありません
- **操作中はリアルタイムで画面が表示される**: 何が行われているかを常に確認できます
- **いつでも操作を中断できる**: 画面をタップするだけで Gemini の操作を停止可能

### プライバシーへの配慮

- 画面操作のデータは Gemini のモデル改善に使用される場合があります（設定でオプトアウト可能）
- 認証情報（パスワード、PIN）は Gemini がアクセスしない設計になっています
- 機密性の高いアプリ（銀行アプリなど）では Pixel Actions は動作しません

### 現時点での制限事項

- **対応地域が限定的**: 米国と韓国が先行。日本を含むその他の地域は順次展開予定
- **対応アプリが少ない**: 2026年3月時点で8アプリのみ。今後拡大予定
- **複雑なタスクには非対応**: 複数アプリを横断するタスク（「レストランを検索して予約し、Uber で行く」など）は現時点で非対応
- **日本語への対応**: 英語でのリクエストが最も安定。日本語対応は今後の発表待ち

## FAQ

### Q1: Pixel Actions は Pixel 9 以前のデバイスでも使えますか？

現時点では Pixel 10 シリーズのみが対応しています。これは Pixel 10 に搭載された Tensor G5 チップの AI 処理能力が必要なためです。Galaxy S26 シリーズでも利用可能ですが、Google の発表では Pixel 9 以前への展開は未定です。

### Q2: 日本でも Pixel Actions は使えますか？

2026年3月時点では、米国と韓国でのみ提供されています。ただし、Google はグローバル展開を計画しており、Google I/O 2026（2026年5月予定）での追加情報が期待されます。日本での対応アプリとしては、Uber Eats Japan や出前館などが候補として考えられます。

### Q3: Pixel Actions で勝手に高額な注文をされる心配はありませんか？

ありません。Pixel Actions は決済を伴うすべてのアクションの直前で**ユーザーの最終確認**を必ず求めます。確認なしに決済が実行されることはない設計です。また、操作中はリアルタイムで画面が表示されるため、意図しない操作が行われた場合は即座に中断できます。

### Q4: 自分のアプリを Pixel Actions に対応させるにはどうすればよいですか？

現時点では公式の開発者 SDK は公開されていませんが、Android のアクセシビリティガイドラインに準拠したアプリは対応しやすい構造になっています。具体的には、すべてのインタラクティブ要素に `contentDescription` を設定し、TalkBack で完全に操作可能な状態にすることが推奨されます。

### Q5: Pixel Actions と Google Assistant のルーティン機能は何が違いますか？

Google Assistant のルーティンは事前に定義されたアクション（ライトをつける、音楽を再生するなど）を順番に実行するものです。一方 Pixel Actions は、**自然言語の自由な指示**に基づいてアプリの画面を動的に操作します。決まったパターンでなくても、文脈を理解してタスクを完了できる点が大きな違いです。
