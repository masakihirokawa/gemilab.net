## Google AI Ultra の立場

Google AI Ultra は、Pro の約86倍の月額費用（$249.99）を要求します。

「そこまで払う価値があるのか」という疑問は、当然の質問です。

この記事では、実際に Ultra を3ヶ月間使い込んだ経験を踏まえて、**正直に** その価値を検証します。

結論を先に言えば、**「万人向けではないが、特定の職種には極めて高い投資対効果がある」** というのが実情です。

## Deep Think —「思考の深さ」がもたらす価値

### Deep Think とは何か

Deep Think は、「複雑な問題に対して、単なる『すぐに出される回答』ではなく、『段階的に深く考える』プロセスを経た答え」を提供する機能です。

```
比較：

【Gemini Pro での質問】
「生成AI時代に、個人が競争力を持つには何をすべきか」

→ 数秒で回答が返る
→ 「スキルアップを絶えず行う」「最新技術を学ぶ」など、
   浅い一般論が中心

【Deep Think での同じ質問】

→ 30秒〜2分程度、思考時間がかかる
→「個人の差別化は以下の3段階で達成される：
  1. 技術スキル（AIに容易に模倣される）
  2. 問題定義力（AIは既知の問題に強いが、
     新しい問題定義には弱い）
  3. 人間関係資本と信用（これが最も模倣困難）

  したがって、差別化戦略は…」

→ より深い、実行可能な思考が展開される
```

### Deep Think が活躍する場面

#### 1. 経営戦略立案

**シーン：新規事業の是非判断**

CEO が新しい事業領域での参入を検討しています。

```
質問：
「当社（SaaS 企業、年売上 ¥50億）が、AI アシスタント向けの
API 提供事業に参入すべきか。市場機会・リスク・必要リソースを
総合的に考えると、どのような判断が適切か」

Deep Think の分析プロセス：
1. 市場機会の定量化
   - API 市場の CAGR
   - 競合企業の参入状況
   - 顧客の価格感度

2. リスク要因の詳細検討
   - 既存事業へのリソース影響
   - 技術的成熟度（競合との差別化は可能か）
   - 規制リスク（AI 規制の動向）

3. 必要リソースの実装可能性
   - エンジニアリング人材
   -営業体制
   - 資本要件

4. 複数シナリオの検討
   - 楽観ケース（市場占有率 15%）
   - 基本ケース（同 5%）
   - 悲観ケース（同 1%）

5. 意思決定フレームワーク
   - IRR, 回収期間
   - 戦略的オプション価値
   - 撤退戦略

出力：
「参入は妥当。ただし以下の条件下でのみ：
- 初期投資 ¥10-15億に限定
- 2年以内の黒字化目標
- 既存事業との人材流出防止」
```

このレベルの分析を、経営コンサルティングファームに依頼すれば、**数百万円のコスト** がかかります。

Deep Think なら、**$249/月の投資** で、同等かそれ以上の質を得られます。

#### 2. 複雑な技術的判断

**シーン：マイクロサービス・アーキテクチャ導入判断**

```
質問：
「現在のモノリシック Python アプリケーション（年間開発費 ¥30M）を
マイクロサービス・アーキテクチャに移行する価値はあるか。
トレードオフを含めて、正直に分析してください」

Deep Think の分析：
1. 移行のメリット
   - スケーラビリティ向上
   - チーム間の独立性
   - テクノロジースタックの多様化

2. 移行のコスト
   - 実装期間：18ヶ月
   - 必要人員：+5名（年間費用 ¥10M）
   - 技術的負債：新たな複雑性

3. リスク要因
   - 移行期間中の不具合
   - 運用複雑性の急増
   - チーム学習曲線

4. 代替案の検討
   - フル移行ではなく、段階的移行
   - 部分的なマイクロサービス化
   - 現システムの最適化に留まる

5. 正直な判断
   「移行は『本当に必要になったとき』に実施すべき。
   現状のスケール（月間 1000万リクエスト程度）では、
   モノリシック構造で問題ない。
   スケール 10倍に達したとき、改めて評価すること」
```

この種の判断は、単なる「AI の回答」では危険です。Deep Think の「段階的思考」により、初めて信頼に値する判断が得られます。

#### 3. 論文・学術研究

Deep Think は、学術的な深掘りにも活躍します。

```
質問：
「ニューロプラスティシティ（脳の可塑性）に関する最新研究から、
個人の学習方法についてどのような含意が得られるか」

Deep Think：
- 複数の研究論文を統合
- 相互に矛盾する知見をどう調和させるか検討
- 実践的インプリケーションを導出
```

### Deep Think の弱点

正直な分析として、Deep Think の弱点も述べておきます。

1. **時間がかかる**：回答に 30秒〜2分かかるため、リアルタイムな判断には不向き
2. **英語圏に最適化**：日本語での思考の深さは、英語より若干落ちる傾向
3. **確実性の保証なし**：「深く考える」ことと「正しく考える」ことは別

### Deep Think の投資対効果

**対象者：経営者、戦略コンサルタント、研究者**

```
Deep Think による時間削減：
- 経営判断 1件：コンサルティング費用 ¥100-500万 → ¥0
  （ただし品質は同等程度と仮定）

- 複雑な技術判断 1件：内部検討会議 8時間 → Deep Think 2分

月に 5件のハイレベルな判断が必要な立場なら、
Deep Think だけで $249/月の価値は確実に超える
```

## Project Mariner — ブラウザ自動化の威力

### Project Mariner とは何か

Project Mariner は、「複雑な Web 作業を、AI が自動で実行する」機能です。

```
実装例：
1. 「100社の企業データを取得」と指示
2. AI が自動で Google Maps、企業サイト、LinkedIn を巡回
3. 企業名、住所、従業員数、売上などを自動抽出
4. Google Sheets に記入
→ 手作業で 50時間かかる仕事が 2時間で完了
```

### Project Mariner の実際の使用例

#### 1. 営業リード生成の自動化

**シーン：新規営業市場への参入**

営業チームが、新しい市場（例：製造業）に参入することになりました。

従来のプロセス：

```
1. リード獲得：営業が手作業で企業を検索（100社特定に 40時間）
2. 企業情報収集：各社の Website から情報抽出（30時間）
3. 意思決定者特定：LinkedIn で経営陣を確認（20時間）
4. 合計：90時間
```

Project Mariner を使用：

```
1. 「製造業×従業員100-500名の企業 200社を特定し、
   企業情報・経営陣情報を Google Sheets に記入」

2. AI が自動実行：
   - Google Maps で企業を検索
   - 各企業サイトで情報抽出
   - LinkedIn で経営陣を特定
   - Sheets に記入

3. 結果：2-3時間で 200社のデータベース完成
```

時間削減：**97%**

#### 2. 市場調査の自動化

**シーン：競合分析**

```
指示：
「AI ビデオ生成ツール 15社について、
公式サイトから以下を抽出：
- 価格設定
- 主要機能
- ターゲット顧客
- 最新ニュース

結果を競合比較表に整理」

Project Mariner の動作：
1. 各社サイトの pricing ページへのアクセス
2. 情報の構造化抽出
3. 最新ニュース・プレスリリースの収集
4. 比較表への自動整理

通常 40時間 → 4時間
```

#### 3. データ マイグレーション・統合

**シーン：CRM システムの統合**

複数の Web ベースの CRM（Salesforce、HubSpot、Pipedrive など）からデータを抽出して、統合データベースに移行する。

```
従来：API を用いたコード開発（1-2週間）

Project Mariner：
「複数の CRM から顧客リスト・営業パイプラインデータを
Google Sheets に統合」

→ 数時間で完了
```

#### 4. ユーザーテスト・QA 自動化

**シーン：複雑な Web アプリの QA テスト**

```
テストシナリオ：
「新規ユーザーが登録 → プロフィール入力 →
決済 → 購入完了」という一連の流れを、
10本のシナリオで自動検証

従来の手作業：4時間/日

Project Mariner：
「以下の 10シナリオを自動実行して、
スクリーンショットを記録」

→ わずか 30分で全テスト完了
```

### Project Mariner の制限と弱点

1. **複雑なインタラクションに弱い**：単純な入力・クリックは得意だが、複雑な JavaScript ベースのインタラクションでは失敗することも
2. **JavaScript 重大サイトに弱い**：SPA（Single Page Application）のような JS 駆動サイトでは、タイミング問題で失敗することがある
3. **認証が必要な場合のトラブル**：2段階認証やキャプチャを超えられない場合がある

### Project Mariner の投資対効果

**対象者：営業チーム、マーケティング、リサーチ部門、QA**

```
例：営業リード生成

従来：100社の情報収集に 60時間（時給 ¥2,000 = ¥120,000）
```
