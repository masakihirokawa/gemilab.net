「近くのおいしいラーメン屋を教えて」という質問に、AI が的確な現在情報で答えられるようになったら、あなたのアプリはどう変わるでしょうか？

## 概念と設計思想

### なぜ Grounding が必要か

通常の LLM は、トレーニング時点のデータしか持っていません。「今日営業しているカフェ」や「最新の星評価」を知ることは原理的にできません。

Maps Grounding はこの問題を解決します。Gemini が「位置情報クエリと判断した場合」、自動的に Google Maps のリアルタイムデータを参照して回答を生成します。開発者側でのデータ収集・管理は一切不要です。

### アーキテクチャの概要

```
ユーザー入力
    ↓
Gemini モデル（クエリ解析）
    ↓ （位置情報クエリと判断した場合）
Google Maps API（リアルタイムデータ取得）
    ↓
Gemini（Maps データを元に回答生成）
    ↓
レスポンス（groundingMetadata 付き）
```

Grounding のトリガーは Gemini が自動判断します。開発者が明示的に「このクエリはマップを使え」と指定する必要はありません。ただし、緯度経度を渡すことでパーソナライズされた結果を得られます。

## ステップバイステップ実装

### Step 1: 基本的なリクエスト

まずシンプルな実装から始めましょう。

```python
# basic_maps_grounding.py
import os
from google import genai
from google.genai import types

# Vertex AI クライアントの初期化
# 環境変数 GOOGLE_GENAI_USE_VERTEXAI=True が必要
client = genai.Client(
    http_options=types.HttpOptions(api_version="v1")
)

def search_nearby(query: str, latitude: float, longitude: float) -> str:
    """
    Google Maps Grounding を使って周辺検索を行う。

    Args:
        query: 検索クエリ（例: "人気のラーメン屋"）
        latitude: 緯度
        longitude: 経度

    Returns:
        AI 生成のレスポンステキスト
    """
    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=query,
        config=types.GenerateContentConfig(
            tools=[
                types.Tool(
                    google_maps=types.GoogleMaps(enable_widget=False)
                )
            ],
            tool_config=types.ToolConfig(
                retrieval_config=types.RetrievalConfig(
                    lat_lng=types.LatLng(
                        latitude=latitude,
                        longitude=longitude
                    ),
                    language_code="ja",
                )
            ),
        ),
    )
    return response.text

# 実行例: 東京駅周辺のラーメン検索
if __name__ == "__main__":
    result = search_nearby(
        query="徒歩10分以内でランチにおすすめのラーメン屋を教えてください",
        latitude=35.6812,
        longitude=139.7671,
    )
    print(result)

# 期待する出力例:
# 東京駅周辺にはいくつかの人気ラーメン店があります。
# 駅内の「東京ラーメンストリート」（八重洲地下街）は...
```

### Step 2: groundingMetadata の活用

アトリビューション表示には `groundingMetadata` が必要です。このメタデータを正しく使わないと **Google のポリシー違反**になります。

```python
# attribution_handler.py
from dataclasses import dataclass
from typing import Optional

@dataclass
class MapSource:
    """Google Maps ソース情報"""
    title: str
    uri: str
    place_id: Optional[str] = None

def extract_map_sources(response) -> list[MapSource]:
    """
    レスポンスから Maps ソース情報を抽出する。

    Returns:
        MapSource オブジェクトのリスト
    """
    sources = []
    if not hasattr(response, "candidates") or not response.candidates:
        return sources

    candidate = response.candidates[0]
    metadata = getattr(candidate, "grounding_metadata", None)
    if not metadata:
        return sources

    chunks = getattr(metadata, "grounding_chunks", [])
    for chunk in chunks:
        web = getattr(chunk, "web", None)
        if web:
            sources.append(
                MapSource(
                    title=getattr(web, "title", ""),
                    uri=getattr(web, "uri", ""),
                    place_id=getattr(web, "place_id", None),
                )
            )
    return sources

def format_attribution_html(sources: list[MapSource]) -> str:
    """
    HTML 形式でアトリビューションを生成する。
    Google のスタイルガイドライン準拠（Roboto フォント, 12-16sp）。
    """
    if not sources:
        return ""

    items = ""
    for src in sources:
        items += f'<li><a href="{src.uri}" target="_blank" rel="noopener">{src.title}</a></li>\n'

    return f"""
<div class="maps-attribution" style="font-family: 'Roboto', sans-serif; font-size: 13px; color: #5f6368; margin-top: 12px;">
  <strong>情報提供元（Google Maps）:</strong>
  <ul style="margin: 4px 0; padding-left: 16px;">
    {items}
  </ul>
</div>
"""

# 使用例
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="渋谷駅から徒歩5分以内の評価が高いコーヒーショップを3件教えてください",
    config=types.GenerateContentConfig(
        tools=[types.Tool(google_maps=types.GoogleMaps())],
        tool_config=types.ToolConfig(
            retrieval_config=types.RetrievalConfig(
                lat_lng=types.LatLng(latitude=35.6580, longitude=139.7016),
                language_code="ja",
            )
        ),
    ),
)

print(response.text)
sources = extract_map_sources(response)
print(format_attribution_html(sources))
```

### Step 3: 実用アプリ — レストラン検索 API

FastAPI を使って本番グレードのエンドポイントを構築します。

```python
# restaurant_api.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field
import os
from google import genai
from google.genai import types

app = FastAPI(title="Location-Aware Restaurant AI")

client = genai.Client(
    http_options=types.HttpOptions(api_version="v1")
)

class SearchRequest(BaseModel):
    query: str = Field(..., description="検索クエリ", example="ランチにおすすめのイタリアン")
    latitude: float = Field(..., ge=-90, le=90)
    longitude: float = Field(..., ge=-180, le=180)
    language: str = Field(default="ja", description="レスポンス言語")

class SearchResponse(BaseModel):
    text: str
    sources: list[dict]
    grounded: bool

@app.post("/search", response_model=SearchResponse)
async def search_restaurants(req: SearchRequest):
    """
    周辺の飲食店を AI で検索するエンドポイント。
    Google Maps のリアルタイムデータを使って回答を生成する。
    """
    try:
        response = client.models.generate_content(
            model="gemini-2.5-flash",
            contents=req.query,
            config=types.GenerateContentConfig(
                system_instruction=(
                    "あなたは位置情報に特化した飲食店アドバイザーです。"
                    "ユーザーの位置情報を考慮し、具体的な店名・営業時間・特徴を含めて回答してください。"
                ),
                tools=[types.Tool(google_maps=types.GoogleMaps())],
                tool_config=types.ToolConfig(
                    retrieval_config=types.RetrievalConfig(
                        lat_lng=types.LatLng(
                            latitude=req.latitude,
                            longitude=req.longitude,
                        ),
                        language_code=req.language,
                    )
                ),
                max_output_tokens=1024,
                temperature=0.3,  # 事実情報なので低めに設定
            ),
        )

        # ソース情報を抽出
        sources = []
        if (
            response.candidates
            and response.candidates[0].grounding_metadata
        ):
            metadata = response.candidates[0].grounding_metadata
            for chunk in getattr(metadata, "grounding_chunks", []):
                if hasattr(chunk, "web"):
                    sources.append({
                        "title": chunk.web.title,
                        "uri": chunk.web.uri,
                    })

        return SearchResponse(
            text=response.text,
            sources=sources,
            grounded=len(sources) > 0,
        )

    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

# 起動: uvicorn restaurant_api:app --reload
```

# real_estate_ai.py
"""
物件周辺の生活施設（スーパー・病院・学校）を AI がレポートする。
"""
from google import genai
from google.genai import types

client = genai.Client(http_options=types.HttpOptions(api_version="v1"))

SYSTEM_PROMPT = """
あなたは不動産アドバイザーです。物件の住所と周辺の生活利便性について、
Google Maps のデータに基づいた正確な情報を提供してください。

回答には必ず以下を含めてください：
- 最寄り駅と徒歩時間
- スーパー・コンビニの有無（徒歩10分圏内）
- 病院・クリニック（徒歩15分圏内）
- 子育て関連施設（保育園・小学校）
"""

def generate_neighborhood_report(
    address: str,
    latitude: float,
    longitude: float,
) -> dict:
    """
    物件周辺の生活環境レポートを生成する。

    Returns:
        {"report": str, "sources": list, "grounded": bool}
    """
    query = f"「{address}」周辺の生活利便性について詳しく教えてください。スーパー、病院、学校、駅へのアクセスも含めて。"

    response = client.models.generate_content(
        model="gemini-2.5-pro",  # 不動産は精度重視で Pro を使用
        contents=query,
        config=types.GenerateContentConfig(
            system_instruction=SYSTEM_PROMPT,
            tools=[types.Tool(google_maps=types.GoogleMaps())],
            tool_config=types.ToolConfig(
                retrieval_config=types.RetrievalConfig(
                    lat_lng=types.LatLng(
                        latitude=latitude,
                        longitude=longitude,
                    ),
                    language_code="ja",
                )
            ),
            temperature=0.1,  # 事実情報なので最低限の温度
        ),
    )

    sources = []
    grounded = False
    if response.candidates and response.candidates[0].grounding_metadata:
        chunks = response.candidates[0].grounding_metadata.grounding_chunks or []
        for chunk in chunks:
            if hasattr(chunk, "web"):
                sources.append({"title": chunk.web.title, "uri": chunk.web.uri})
        grounded = len(sources) > 0

    return {
        "report": response.text,
        "sources": sources,
        "grounded": grounded,
    }

# 使用例

### よくあるエラーと対処法

#### エラー 1: `PERMISSION_DENIED` または `API not enabled`

```
google.api_core.exceptions.PermissionDenied: 403 Permission denied on resource
```

**原因**: Vertex AI API が有効化されていない、または Maps Grounding の申請が完了していない。

```bash
# Vertex AI API を有効化
gcloud services enable aiplatform.googleapis.com

# 現在の有効化状況を確認
gcloud services list --enabled | grep aiplatform
```

#### エラー 2: `google_maps tool not recognized`

**原因**: 通常の `GOOGLE_API_KEY` 認証で呼び出している。Vertex AI 経由でないと Maps Grounding は動作しません。

```python
# ❌ 間違い: API Key 認証では Maps Grounding は使えない

# ✅ 正しい: Vertex AI 認証を使う

os.environ["GOOGLE_CLOUD_LOCATION"] = "global"
client = genai.Client(http_options=types.HttpOptions(api_version="v1"))
```

#### エラー 3: `grounded=False`（Grounding が発動しない）

**原因**: クエリが位置情報と関係ないと Gemini が判断した。

```python
# ❌ Grounding が発動しにくいクエリ

# ✅ Grounding が発動しやすいクエリ（場所・地名・施設名を含める）

"渋谷周辺で24時間営業しているドラッグストアはありますか？"
```

#### エラー 4: レート制限

```
google.api_core.exceptions.ResourceExhausted: 429 Quota exceeded
```

**対処法**: 指数バックオフで自動リトライを実装します。

```python
# retry_handler.py
import time
import random
from functools import wraps

def with_retry(max_attempts=3, base_delay=1.0):
    """指数バックオフリトライデコレータ"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if "429" in str(e) and attempt < max_attempts - 1:
                        delay = base_delay * (2 ** attempt) + random.uniform(0, 1)
                        print(f"レート制限: {delay:.1f}秒後にリトライ ({attempt+1}/{max_attempts})")
                        time.sleep(delay)
                    else:
                        raise
        return wrapper
    return decorator

@with_retry(max_attempts=3, base_delay=2.0)
def search_with_retry(query: str, lat: float, lng: float):
    return client.models.generate_content(
        model="gemini-2.5-flash",
        contents=query,
        config=types.GenerateContentConfig(
            tools=[types.Tool(google_maps=types.GoogleMaps())],
            tool_config=types.ToolConfig(
                retrieval_config=types.RetrievalConfig(
                    lat_lng=types.LatLng(latitude=lat, longitude=lng),
                )
            ),
        ),
    )
```

## コスト・パフォーマンス考慮事項

### 料金体系

Maps Grounding の料金は **1,000 リクエストあたり $25 USD** です。ただし、**1 日 500 リクエストは無料枠**があります。

重要なポイント：
- 課金が発生するのは「Maps ソースが 1 件以上含まれるレスポンス」のみ
- Grounding が発動しなかった場合（通常の Gemini 回答）は Maps 課金なし
- Gemini モデル自体の利用料（トークン課金）は別途発生

### コスト最適化のヒント

```python
# cost_optimizer.py

# ヒント 1: gemini-2.5-flash を優先使用（Pro より安価で Maps Grounding にも対応）

    tools=[types.Tool(google_maps=types.GoogleMaps())],
    temperature=0.0,
    max_output_tokens=512,  # ヒント 3: 長すぎない出力で節約
)

# ヒント 4: バッチ処理が必要な場合は Batch API と組み合わせる
# （Maps Grounding と Batch API の同時使用については Google の最新ドキュメントを確認）
```

## FAQ

**Q1. Gemini API（API Key）で Maps Grounding を試したが動かない。なぜですか？**

A1. Grounding with Google Maps は **Vertex AI 経由でのみ利用可能**です。`GOOGLE_API_KEY` を使った通常の Gemini API では動作しません。環境変数 `GOOGLE_GENAI_USE_VERTEXAI=True` を設定し、Google Cloud の認証情報を使うよう切り替えてください。

**Q2. Grounding with Google Search と Maps Grounding の違いは何ですか？**

A2. Google Search Grounding はウェブ全般の最新情報を参照するのに対し、Maps Grounding は **Google Maps のスポットデータ（営業時間・評価・住所・電話番号など）に特化**しています。「近くのお店を探す」など位置情報が絡むクエリには Maps Grounding が適しており、精度が格段に上がります。詳しくは [Grounding with Google Search の記事](/articles/gemini-advanced/grounding-and-search) も参照してください。

**Q3. アトリビューション（出典表示）は必須ですか？省略するとどうなりますか？**

A3. **アトリビューション表示は Google のポリシーで必須**です。レスポンスに Maps ソースが含まれる場合、ソースの URI とタイトルをユーザーが 1 インタラクション以内に確認できる形で表示する必要があります。省略すると利用規約違反となり、API アクセスが停止される可能性があります。

**Q4. 日本語でのクエリと英語でのクエリで精度は変わりますか？**

A4. `language_code` パラメータを適切に設定することで、ほぼ同等の精度が得られます。日本語の場合は `"ja"` 、英語の場合は `"en_US"` を指定してください。ただし、海外の情報を日本語で取得する場合は `"ja"` を指定しても英語の情報が混在する場合があります。

**Q5. Gemini 3 シリーズで Maps Grounding を使いたい。対応予定はありますか？**

A5. 2026 年 3 月時点では、**Gemini 3 は Maps Grounding に非対応**です（Gemini 2.5 シリーズは対応）。Vertex AI ドキュメントの更新を定期的に確認してください。なお、Vertex AI でのみ `Gemini 3 Pro` が Maps Grounding に対応するケースも報告されており、公式ドキュメントで随時確認することをお勧めします。

**Q6. 利用禁止地域（中国・北朝鮮など）に関してはどう対応すればよいですか？**

A6. Maps Grounding は中国・イラン・北朝鮮・シリアなど一部の地域では利用できません。グローバルサービスを開発する場合は、ユーザーの位置情報に基づいて Maps Grounding の有効/無効を切り替えるロジックを実装してください。エラー `INVALID_ARGUMENT: Grounding with Google Maps is not available in your region` が返ります。

## まとめ

Grounding with Google Maps は、Gemini の生成 AI 能力を Google Maps の膨大なリアルタイムデータと組み合わせることで、これまでの「幻覚する AI」という課題を解消する画期的な機能です。

この記事で実装した主なポイントを振り返ります：

1. **Vertex AI 必須**：通常の API Key 認証では動作しません。必ず Vertex AI 経由で使うこと
2. **緯度経度によるパーソナライズ**：`RetrievalConfig` に `lat_lng` を渡すことでユーザーの位置に応じた回答が得られます
3. **アトリビューション表示の義務化**：`groundingMetadata` からソースを抽出し、必ずユーザーに表示してください
4. **コスト管理**：1 日 500 リクエストの無料枠を活用し、`gemini-2.5-flash` + 低 `max_output_tokens` でコストを抑えましょう
5. **エラーハンドリング**：レート制限への指数バックオフと、Grounding が発動しない場合のフォールバックを必ず実装してください

次のステップとして、Gemini ADK と組み合わせたマルチエージェント位置情報システムや、Gemini のマルチモーダル機能（画像認識）と Maps Grounding を組み合わせた「写真から周辺を検索する」アプリへの発展も考えてみてください。
