## はじめに

会議のたびに議事録を作成するのは、地味に時間がかかる作業です。参加者の発言を追いかけながらメモを取り、後から整形して共有する——この一連の流れに毎回30分以上かけている方も多いのではないでしょうか。

Gemini を活用すれば、この議事録作成の負担を大幅に軽減できます。Google Meet の自動文字起こし機能と Gemini の要約能力を組み合わせることで、会議終了後すぐに構造化された議事録を手に入れることが可能です。

この記事では、Gemini による議事録自動作成の具体的な方法を、初心者の方にもわかりやすくステップバイステップで解説します。Google Workspace を日常的に使っている方であれば、すぐに実践できる内容です。

## Google Meet の自動文字起こしを有効にする

議事録の自動作成の第一歩は、会議の内容をテキストとして記録することです。Google Meet には自動文字起こし（Transcript）機能が組み込まれており、これを有効にするだけで会議の発言内容がそのまま Google Docs に保存されます。

### 文字起こしの開始手順

1. Google Meet で会議に参加する
2. 画面右下の「アクティビティ」アイコンをクリック
3. 「文字起こし」を選択し、「文字起こしを開始」をクリック
4. 会議参加者全員に文字起こしが有効であることが通知される

会議が終了すると、文字起こしの内容が自動的に Google Docs に保存され、Google Drive の「Meet Recordings」フォルダに格納されます。主催者と文字起こしを開始した参加者にはメールで通知が届きます。

### 注意点

- Google Workspace の Business Standard 以上のプランが必要です（無料の個人アカウントでは利用できない場合があります）
- 文字起こしの精度は話者の発音やマイクの品質に依存します
- 日本語の文字起こしにも対応していますが、英語と比べると精度にばらつきがある場合があります

## Gemini in Google Docs で議事録を要約する

文字起こしが完了したら、次は Gemini の出番です。Google Docs に統合された Gemini を使えば、長い文字起こしテキストからポイントを抽出し、構造化された議事録を生成できます。

### 基本的な使い方

Google Docs で文字起こしファイルを開き、サイドパネルの Gemini アイコンをクリックします。以下のようなプロンプトを入力してみましょう。

```
この会議の文字起こしを以下の形式で要約してください：

1. 会議の概要（3行以内）
2. 主な議題と決定事項（箇条書き）
3. アクションアイテム（担当者・期限付き）
4. 次回の会議で確認すべき事項
```

Gemini はドキュメントの内容を理解し、指定したフォーマットに沿って要約を生成します。生成された内容はそのままドキュメントに挿入できるので、手作業でのコピー＆ペーストは不要です。

### 効果的なプロンプトのコツ

議事録の品質を高めるために、プロンプトにはいくつかのコツがあります。

- **出力フォーマットを明示する**: 「箇条書きで」「表形式で」など、望む形式を具体的に指定する
- **重要度のフィルタリングを依頼する**: 「重要な決定事項のみ抽出して」と指示すると、雑談部分を自動的にスキップしてくれる
- **参加者の役割を伝える**: 「プロジェクトマネージャーの視点で」と指定すると、進捗管理に特化した要約が得られる

## Gemini API を使ってプログラムで議事録を生成する

より柔軟なカスタマイズや自動化を求める場合は、Gemini API を活用する方法もあります。Python スクリプトで文字起こしテキストを処理し、定型フォーマットの議事録を一括生成できます。

### 環境準備

まず、Google AI Python SDK をインストールします。

```bash
pip install google-genai
```

### 基本的な実装

以下は、テキストファイルから議事録を生成する Python スクリプトの例です。

```python
from google import genai

# クライアントの初期化
client = genai.Client(api_key="YOUR_GEMINI_API_KEY")

def generate_meeting_notes(transcript_text: str) -> str:
    """文字起こしテキストから構造化された議事録を生成する"""

    prompt = f"""以下は会議の文字起こしテキストです。
これを構造化された議事録に変換してください。

## 出力フォーマット
- 会議タイトル（内容から推測）
- 日時
- 参加者（発言者名から推測）
- 議題ごとの要約
- 決定事項
- アクションアイテム（担当者・期限）
- 次回確認事項

## 文字起こしテキスト
{transcript_text}
"""

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=prompt
    )

    return response.text

# 使用例
with open("transcript.txt", "r", encoding="utf-8") as f:
    transcript = f.read()

notes = generate_meeting_notes(transcript)
print(notes)
# 出力例:
# ## 会議タイトル: Q2 プロダクトロードマップレビュー
# ## 日時: 2026年3月29日 14:00-15:00
# ## 参加者: 田中、佐藤、山田
# ## 議題1: 新機能リリーススケジュール
#   - 4月中旬にベータ版リリース予定
#   - QAチームの体制を2名増員する方向で調整中
# ...
```

### 音声ファイルから直接議事録を生成する

Gemini のマルチモーダル機能を使えば、音声ファイルを直接入力して議事録を生成することも可能です。Google Meet の録画や外部の録音ファイルにも対応できます。

```python
import pathlib
from google import genai

client = genai.Client(api_key="YOUR_GEMINI_API_KEY")

def generate_notes_from_audio(audio_path: str) -> str:
    """音声ファイルから議事録を生成する"""

    # 音声ファイルをアップロード
    audio_file = client.files.upload(
        file=pathlib.Path(audio_path)
    )

    prompt = """この音声は会議の録音です。
以下の形式で議事録を作成してください：

1. 会議の概要
2. 各議題の要点
3. 決定事項
4. アクションアイテム（担当者名を含む）
5. 次回までの宿題
"""

    response = client.models.generate_content(
        model="gemini-2.5-pro",
        contents=[prompt, audio_file]
    )

    return response.text

# 使用例
notes = generate_notes_from_audio("meeting_recording.mp3")
print(notes)
# 出力例:
# ## 会議の概要
# マーケティングチームの週次定例。Q2キャンペーンの進捗確認と予算配分を議論。
# ...
```

Gemini の音声理解機能について詳しくは [Gemini の音声理解機能ガイド](/articles/gemini-api/gemini-audio-understanding) をご覧ください。

## Apps Script で議事録作成を完全自動化する

毎回手動でプロンプトを入力するのが面倒な場合は、Google Apps Script を使って自動化しましょう。Google Meet の文字起こしが生成されたタイミングで自動的に Gemini で要約し、チームに共有するワークフローを構築できます。

### 自動化スクリプトの例

```javascript
/**
 * Google Drive の Meet Recordings フォルダを監視し、
 * 新しい文字起こしファイルから議事録を自動生成する
 */
function processNewTranscripts() {
  const folderId = "YOUR_MEET_RECORDINGS_FOLDER_ID";
  const folder = DriveApp.getFolderById(folderId);
  const files = folder.getFilesByType(MimeType.GOOGLE_DOCS);

  // 直近24時間以内に作成されたファイルを処理
  const oneDayAgo = new Date(Date.now() - 24 * 60 * 60 * 1000);

  while (files.hasNext()) {
    const file = files.next();
    if (file.getDateCreated() > oneDayAgo) {
      const doc = DocumentApp.openById(file.getId());
      const transcript = doc.getBody().getText();

      if (transcript.length > 100) {
        const summary = callGeminiApi(transcript);
        createMeetingNotesDoc(file.getName(), summary);
      }
    }
  }
}

/**
 * Gemini API を呼び出して要約を生成
 */
function callGeminiApi(transcript) {
  const apiKey = PropertiesService
    .getScriptProperties()
    .getProperty("GEMINI_API_KEY");

  const url =
    "https://generativelanguage.googleapis.com/v1beta/models/"
    + "gemini-2.5-flash:generateContent?key=" + apiKey;

  const payload = {
    contents: [{
      parts: [{
        text: "以下の会議文字起こしを構造化された議事録に変換してください。"
          + "決定事項とアクションアイテムは必ず含めてください。\n\n"
          + transcript
      }]
    }]
  };

  const options = {
    method: "post",
    contentType: "application/json",
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  };

  const response = UrlFetchApp.fetch(url, options);
  const json = JSON.parse(response.getContentText());
  return json.candidates[0].content.parts[0].text;
}

/**
 * 要約された議事録を新しい Google Docs に保存
 */
function createMeetingNotesDoc(originalName, summary) {
  const title = "【議事録】" + originalName
    .replace("Transcript - ", "");
  const doc = DocumentApp.create(title);
  const body = doc.getBody();

  body.appendParagraph("自動生成議事録")
    .setHeading(DocumentApp.ParagraphHeading.HEADING1);
  body.appendParagraph(
    "生成日時: " + new Date().toLocaleString("ja-JP")
  );
  body.appendParagraph("---");
  body.appendParagraph(summary);

  // チームメンバーに共有
  doc.addEditor("team-lead@example.com");

  Logger.log("議事録を作成しました: " + doc.getUrl());
}
```

このスクリプトを Apps Script のトリガーに設定すれば、毎日自動的に新しい文字起こしを検出して議事録を生成してくれます。Google Apps Script と Gemini の連携についてさらに詳しく知りたい方は [Gemini × Google Apps Script 自動化ガイド](/articles/gemini-workspace/gemini-google-apps-script-automation) も参考になります。

## 議事録の品質を高めるプロンプトテンプレート集

用途に応じたプロンプトテンプレートをいくつかご紹介します。状況に合わせて使い分けることで、議事録の品質を大きく向上させることができます。

### プロジェクト進捗会議向け

```
以下の文字起こしから、プロジェクト進捗レポートを作成してください。

含めるべき項目：
- 各タスクの進捗状況（完了・進行中・未着手）
- ブロッカーとなっている課題
- マイルストーンの達成状況
- 次のスプリントで対応すべき事項
```

### ブレインストーミング向け

```
以下の文字起こしから、ブレインストーミングの成果をまとめてください。

含めるべき項目：
- 出されたアイデアの一覧（カテゴリ分け）
- 各アイデアの賛否や議論のポイント
- 最も支持されたアイデア上位3つ
- 次のステップとして検証すべき事項
```

### クライアントミーティング向け

```
以下の文字起こしから、クライアントとの打ち合わせ議事録を作成してください。

含めるべき項目：
- クライアントの要望・フィードバック
- 合意した仕様変更点
- スケジュールに関する確認事項
- 次回の打ち合わせまでに準備すべきこと
- 請求・契約に関する確認事項（もしあれば）
```

## よくある質問（FAQ）

### Q: Google Meet の文字起こし機能は無料で使えますか？

Google Workspace の Business Standard、Business Plus、Enterprise プランで利用可能です。無料の個人アカウントや Business Starter プランでは文字起こし機能が制限されている場合があります。最新のプラン別機能については Google Workspace の公式ページを確認してください。

### Q: 日本語の文字起こし精度はどの程度ですか？

Google Meet の日本語文字起こしは実用的なレベルに達していますが、専門用語や固有名詞は誤認識されることがあります。精度を上げるコツとして、発言時にゆっくり明瞭に話すこと、外付けマイクを使用すること、そして生成された議事録を Gemini で要約する際に「専門用語リスト」をプロンプトに含めることが効果的です。

### Q: Google Meet 以外の会議ツール（Zoom、Teams）でも使えますか？

Zoom や Microsoft Teams の録画・音声ファイルをダウンロードし、Gemini API に直接渡すことで同様の議事録生成が可能です。本記事の「音声ファイルから直接議事録を生成する」セクションの方法をそのまま応用できます。

### Q: 議事録の機密性は大丈夫ですか？

Google Workspace の Gemini は、エンタープライズグレードのデータ保護ポリシーに基づいて運用されています。Workspace プラン利用者のデータは Google のモデルトレーニングには使用されません。ただし、Gemini API を外部スクリプトから利用する場合は、API の利用規約とデータポリシーを確認し、機密情報の取り扱いについて社内ルールを設けることを推奨します。
