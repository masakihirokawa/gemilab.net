## はじめに

毎日のスケジュール管理に追われていませんか？ 会議の調整、空き時間の把握、予定の重複チェックなど、カレンダー管理は地味に時間を取られる作業です。

この記事では、**Gemini API** と **Google Calendar API** を組み合わせて、自然言語で予定を登録・分析できる AI スケジュールアシスタントを Python で構築する方法を解説します。「来週の火曜に1時間のミーティングを入れて」と伝えるだけで予定が作成され、「今週の空き時間を教えて」と聞けば最適な時間帯を提案してくれるアシスタントを目指します。

Gemini API の [Function Calling](/articles/gemini-advanced/function-calling-guide) 機能を活用することで、AI が自然言語を理解し、適切な Calendar API の操作に変換してくれます。

### Google Calendar API の認証設定

Google Cloud Console で OAuth 2.0 クライアント ID を作成し、`credentials.json` としてプロジェクトルートに配置します。初回実行時にブラウザで認証フローが開き、`token.json` が自動生成されます。

```python
import os
from google.auth.transport.requests import Request
from google.oauth2.credentials import Credentials
from google_auth_oauthlib.flow import InstalledAppFlow
from googleapiclient.discovery import build

SCOPES = ["https://www.googleapis.com/auth/calendar"]

def get_calendar_service():
    """Google Calendar API サービスを取得"""
    creds = None
    if os.path.exists("token.json"):
        creds = Credentials.from_authorized_user_file("token.json", SCOPES)
    if not creds or not creds.valid:
        if creds and creds.expired and creds.refresh_token:
            creds.refresh(Request())
        else:
            flow = InstalledAppFlow.from_client_secrets_file("credentials.json", SCOPES)
            creds = flow.run_local_server(port=0)
        with open("token.json", "w") as token:
            token.write(creds.to_json())
    return build("calendar", "v3", credentials=creds)
```

## Gemini API で Function Calling を設定する

AI アシスタントの核となるのが、Gemini の Function Calling 機能です。カレンダー操作に必要な関数を定義し、Gemini に「どの関数をどんな引数で呼ぶべきか」を判断させます。

```python
import google.generativeai as genai
from datetime import datetime, timedelta

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

# カレンダー操作の関数定義
calendar_tools = [

                        "start_time": {"type": "string", "description": "開始時刻 (ISO 8601形式: 2026-03-26T10:00:00+09:00)"},
                        "end_time": {"type": "string", "description": "終了時刻 (ISO 8601形式)"},
                        "description": {"type": "string", "description": "予定の説明（省略可）"},
                        "location": {"type": "string", "description": "場所（省略可）"},
                    },
                    "required": ["summary", "start_time", "end_time"],
                },
            },
            {
                "name": "list_events",
                "description": "指定期間のカレンダー予定一覧を取得する",
                "parameters": {
```

## カレンダー操作関数の実装

Function Calling で呼び出される実際の関数を実装します。

```python
def create_event(service, summary, start_time, end_time, description="", location=""):
    """予定を作成して結果を返す"""
    event = {
        "summary": summary,
        "start": {"dateTime": start_time, "timeZone": "Asia/Tokyo"},
        "end": {"dateTime": end_time, "timeZone": "Asia/Tokyo"},
    }
    if description:
        event["description"] = description
    if location:
        event["location"] = location

    created = service.events().insert(calendarId="primary", body=event).execute()
    return f"予定を作成しました: {created['summary']} ({created['start']['dateTime']})"


def list_events(service, start_date, end_date):
    """指定期間の予定一覧を取得"""
    time_min = f"{start_date}T00:00:00+09:00"
    time_max = f"{end_date}T23:59:59+09:00"

    events_result = (
        service.events()
        .list(
            calendarId="primary",
            timeMin=time_min,
            timeMax=time_max,
            singleEvents=True,
            orderBy="startTime",
        )
        .execute()
    )
    events = events_result.get("items", [])

    if not events:
        return "指定期間に予定はありません。"

    result = []
    for event in events:
        start = event["start"].get("dateTime", event["start"].get("date"))
        result.append(f"- {start}: {event['summary']}")
    return "\n".join(result)


def find_free_slots(service, start_date, end_date, duration_minutes):
    """空き時間を検索（営業時間 9:00-18:00 内）"""
    time_min = f"{start_date}T00:00:00+09:00"
    time_max = f"{end_date}T23:59:59+09:00"

    body = {
        "timeMin": time_min,
        "timeMax": time_max,
        "items": [{"id": "primary"}],
    }
    freebusy = service.freebusy().query(body=body).execute()
    busy_periods = freebusy["calendars"]["primary"]["busy"]

    # 営業時間内の空き時間を計算
    free_slots = []
    current = datetime.fromisoformat(f"{start_date}T09:00:00+09:00")
    end = datetime.fromisoformat(f"{end_date}T18:00:00+09:00")

    while current < end:
        slot_end = current + timedelta(minutes=duration_minutes)
        if slot_end.hour > 18:
            current = current.replace(hour=9, minute=0) + timedelta(days=1)
            continue

        is_free = True
        for busy in busy_periods:
            busy_start = datetime.fromisoformat(busy["start"])
            busy_end = datetime.fromisoformat(busy["end"])
            if current < busy_end and slot_end > busy_start:
                is_free = False
                current = busy_end
                break

        if is_free:
            free_slots.append(f"- {current.strftime('%m/%d %H:%M')} 〜 {slot_end.strftime('%H:%M')}")
            current = slot_end

    if not free_slots:
        return "指定期間に空き時間が見つかりませんでした。"
    return f"空き時間（{duration_minutes}分以上）:\n" + "\n".join(free_slots[:10])
```

## 対話ループの構築

ユーザーの自然言語入力を受け取り、Gemini が Function Calling で適切な操作を判断し、結果を返す対話ループを実装します。

```python
def run_assistant():
    """AI スケジュールアシスタントのメインループ"""
    service = get_calendar_service()
    chat = model.start_chat()

    # 関数名と実装のマッピング
    func_map = {
        "create_event": lambda args: create_event(service, **args),
        "list_events": lambda args: list_events(service, **args),
        "find_free_slots": lambda args: find_free_slots(service, **args),
    }

    print("📅 AI スケジュールアシスタントを起動しました。")
    print("   自然言語で予定の操作ができます（終了: quit）\n")

    while True:
        user_input = input("あなた > ").strip()
        if user_input.lower() in ("quit", "exit", "終了"):
            print("アシスタントを終了します。")
            break

        response = chat.send_message(user_input)

        # Function Call のレスポンスを処理
        while response.candidates[0].content.parts[0].function_call.name:
            fc = response.candidates[0].content.parts[0].function_call
            func_name = fc.name
            func_args = dict(fc.args)

            print(f"  🔧 {func_name} を実行中...")
            result = func_map[func_name](func_args)

            # 関数の結果を Gemini に返す
            response = chat.send_message(
                genai.protos.Content(
                    parts=[
                        genai.protos.Part(
                            function_response=genai.protos.FunctionResponse(
                                name=func_name,
                                response={"result": result},
                            )
                        )
                    ]
                )
            )

        print(f"アシスタント > {response.text}\n")


if __name__ == "__main__":
    run_assistant()
```

## よくあるエラーと対処法

### OAuth 認証エラー

`credentials.json` のスコープを変更した場合は、`token.json` を削除して再認証が必要です。

```bash
rm token.json
python calendar_assistant.py  # 再認証フロー起動
```

### タイムゾーンの不一致

Gemini が UTC で時刻を返す場合があります。System Instruction でタイムゾーンを明示するか、関数側で `Asia/Tokyo` に変換してください。

### API のレート制限

Google Calendar API のデフォルトクォータは 1,000,000 リクエスト/日です。個人利用では問題になりませんが、複数ユーザー対応時は [Exponential Backoff](https://www.amazon.co.jp/dp/4873119049/?tag=pinocchio-22) の実装を推奨します（『Web API: The Good Parts』が実装パターンの参考になります）。

## 応用・発展的な活用法

### 1. 週次レポートの自動生成

1週間の予定を取得し、Gemini に分析させることで「今週は会議が15時間、集中作業時間が8時間」のようなレポートを自動生成できます。

```python
def weekly_report(service, chat):
    today = datetime.now()
    start = (today - timedelta(days=today.weekday())).strftime("%Y-%m-%d")
    end = (today + timedelta(days=6 - today.weekday())).strftime("%Y-%m-%d")
    events = list_events(service, start, end)

    response = chat.send_message(
        f"以下の今週の予定を分析して、時間配分のレポートを作成してください:\n{events}"
    )
    return response.text
```

### 2. 定期予定の最適化提案

カレンダーの傾向を分析し、「毎週月曜の午前は会議が集中しています。一部を火曜に分散させると集中作業の時間が確保できます」のような提案ができます。

### 3. 複数カレンダーの統合管理

仕事用・個人用の複数カレンダーを横断的に管理し、プライベートの予定を考慮した上で仕事の会議をスケジューリングすることも可能です。

Google Workspace の活用をさらに深めたい方は、[Gemini × Workspace 自動化ガイド](/articles/gemini-workspace/gemini-workspace-automation-2026)もあわせてご覧ください。

## FAQ

### Q1: Gemini API の無料枠でこのアシスタントは動きますか？

はい、動きます。Gemini 2.5 Flash の無料枠は 1 分あたり 30 リクエストで、個人のスケジュール管理には十分です。ただし、大量のバッチ処理を行う場合は [Gemini API の従量課金プラン](https://aistudio.google.com/)への移行をおすすめします。

### Q2: Google Apps Script（GAS）で実装する方法はありますか？

はい、Google Apps Script を使えばサーバーレスで Calendar 連携が可能です。詳しくは [Gemini × Google Apps Script 自動化ガイド](/articles/gemini-dev/gemini-google-apps-script-automation)で解説しています。GAS の場合はトリガーによる定期実行も簡単に設定できます。

### Q3: セキュリティ面で注意すべきことは？

OAuth 2.0 の `token.json` には機密情報が含まれるため、Git リポジトリにコミットしないでください（`.gitignore` に追加）。また、Gemini API キーも環境変数で管理し、ソースコードにハードコードしないことが重要です。本番環境では [Google Secret Manager](https://cloud.google.com/secret-manager) の利用を検討してください。
