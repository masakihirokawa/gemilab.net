## はじめに — なぜ Apps Script × Gemini なのか

Google Sheets はビジネスデータの集約拠点として広く使われています。売上数値、KPI、マーケティングデータ、在庫情報——あらゆるデータがスプレッドシートに蓄積されていきます。しかし、これらのデータから定期的にレポートを作成し、チームに共有する作業は、多くの組織で依然として手動で行われています。

Apps Script は Google Workspace に組み込まれたサーバーレス実行環境です。ここに Gemini API を統合することで、スプレッドシートのデータを自動的に読み取り、AI が分析・要約し、整形されたレポートを生成して関係者に配信する——この一連のパイプラインをコードだけで構築できます。

この記事では、Apps Script から Gemini API を呼び出す基本パターンから始め、本番環境で安定稼働するレポート生成パイプラインの設計・実装まで、実際に動作するコードとともに詳しく解説します。

Google Sheets の基本的な AI 活用については[Google スプレッドシート × Gemini AI 実践ガイド](/articles/gemini-workspace/gemini-google-sheets-ai-data-analysis)で解説していますが、本記事ではさらに一歩踏み込み、Apps Script を使った完全自動化のアーキテクチャに焦点を当てます。

**対象読者:**
- Google Sheets を日常的に業務で使っている方
- Apps Script の基本は理解している方（初めての方も環境準備から説明します）
- AI を使った業務自動化に関心がある開発者・技術リーダー

## 環境準備と API キーの安全な管理

### Google AI Studio で API キーを取得

Gemini API を利用するには、まず [Google AI Studio](https://aistudio.google.com/) で API キーを発行します。Google アカウントでログイン後、左メニューの「Get API key」から新しいキーを生成してください。

### Apps Script プロジェクトの作成

Google Sheets を開き、「拡張機能」→「Apps Script」でエディタを起動します。ここが開発環境になります。

### API キーを Script Properties に保存する

**重要**: API キーをコード内にハードコードしてはいけません。Apps Script には `PropertiesService` というセキュアなキーバリューストアがあります。

```javascript
// 初回セットアップ用関数（一度だけ実行）
function setupApiKey() {
  const scriptProperties = PropertiesService.getScriptProperties();
  scriptProperties.setProperty('GEMINI_API_KEY', 'YOUR_API_KEY_HERE');
  console.log('API key saved to Script Properties');
}

// API キーの取得（以降のコードで使用）
function getApiKey() {
  return PropertiesService.getScriptProperties()
    .getProperty('GEMINI_API_KEY');
}
```

`setupApiKey()` を一度実行したら、コード内の `YOUR_API_KEY_HERE` は削除してください。Script Properties に保存された値は、プロジェクトの設定画面からも確認・編集できます。

## Gemini API 呼び出しの基本パターン

Apps Script から Gemini API を呼び出す基本クラスを実装します。このクラスは以降のすべての機能の土台になります。

```javascript
/**
 * GeminiClient — Apps Script 用 Gemini API クライアント
 * リトライ・タイムアウト・エラーハンドリング内蔵
 */
class GeminiClient {
  constructor() {
    this.apiKey = getApiKey();
    this.baseUrl = 'https://generativelanguage.googleapis.com/v1beta';
    this.model = 'models/gemini-2.5-flash';
    this.maxRetries = 3;
  }

  /**
   * テキスト生成リクエストを送信
   * @param {string} prompt - プロンプト
   * @param {Object} options - 追加オプション
   * @returns {string} 生成されたテキスト
   */
  generate(prompt, options = {}) {
    const url = `${this.baseUrl}/${this.model}:generateContent?key=${this.apiKey}`;

    const payload = {
      contents: [{
        parts: [{ text: prompt }]
      }],
      generationConfig: {
        temperature: options.temperature || 0.3,
        maxOutputTokens: options.maxTokens || 4096,
        topP: options.topP || 0.95
      }
    };

    // システムインストラクションの追加
    if (options.systemInstruction) {
      payload.systemInstruction = {
        parts: [{ text: options.systemInstruction }]
      };
    }

    return this._fetchWithRetry(url, payload);
  }

  /**
   * リトライ付き HTTP リクエスト
   */
  _fetchWithRetry(url, payload) {
    for (let attempt = 1; attempt <= this.maxRetries; attempt++) {
      try {
        const response = UrlFetchApp.fetch(url, {
          method: 'post',
          contentType: 'application/json',
          payload: JSON.stringify(payload),
          muteHttpExceptions: true
        });

        const code = response.getResponseCode();
        const body = JSON.parse(response.getContentText());

        if (code === 200) {
          return body.candidates[0].content.parts[0].text;
        }

        // レート制限 (429) またはサーバーエラー (5xx) はリトライ
        if (code === 429 || code >= 500) {
          const waitMs = Math.pow(2, attempt) * 1000 + Math.random() * 1000;
          console.log(`Retry ${attempt}/${this.maxRetries} after ${waitMs}ms (HTTP ${code})`);
          Utilities.sleep(waitMs);
          continue;
        }

        // その他のエラーは即座に失敗
        throw new Error(`Gemini API error ${code}: ${JSON.stringify(body)}`);

      } catch (e) {
        if (attempt === this.maxRetries) throw e;
        Utilities.sleep(Math.pow(2, attempt) * 1000);
      }
    }
    throw new Error('Max retries exceeded');
  }
}
```

**設計のポイント:**
- `muteHttpExceptions: true` を指定し、HTTP エラーを例外ではなくレスポンスとして受け取る
- 429（レート制限）と 5xx（サーバーエラー）は指数バックオフでリトライ
- 400 系エラー（不正なリクエスト）はリトライせずに即座に通知

## スプレッドシートデータの読み取りとAI分析

### データ抽出レイヤー

スプレッドシートから構造化データを抽出するユーティリティを実装します。

```javascript
/**
 * スプレッドシートからデータを構造化オブジェクトとして抽出
 * @param {string} sheetName - シート名
 * @returns {Object} ヘッダーとデータ行
 */
function extractSheetData(sheetName) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName(sheetName);

  if (!sheet) {
    throw new Error(`Sheet "${sheetName}" not found`);
  }

  const data = sheet.getDataRange().getValues();
  const headers = data[0];
  const rows = data.slice(1).filter(row =>
    row.some(cell => cell !== '' && cell !== null)
  );

  return {
    sheetName,
    headers,
    rows,
    rowCount: rows.length,
    summary: `${sheetName}: ${rows.length} rows × ${headers.length} columns`
  };
}

/**
 * 数値データの基本統計量を計算
 * @param {Array} data - extractSheetData の出力
 * @param {string} columnName - 対象列名
 * @returns {Object} 統計量
 */
function calculateStats(data, columnName) {
  const colIndex = data.headers.indexOf(columnName);
  if (colIndex === -1) throw new Error(`Column "${columnName}" not found`);

  const values = data.rows
    .map(row => Number(row[colIndex]))
    .filter(v => !isNaN(v));

  values.sort((a, b) => a - b);
  const sum = values.reduce((a, b) => a + b, 0);
  const avg = sum / values.length;

  return {
    count: values.length,
    sum: sum,
    average: Math.round(avg * 100) / 100,
    min: values[0],
    max: values[values.length - 1],
    median: values[Math.floor(values.length / 2)]
  };
}
```

### AI 分析プロンプトの設計

データを Gemini に渡して分析させる際、プロンプトの設計が品質を決定します。

```javascript
/**
 * 売上データ分析用のプロンプトを構築
 * @param {Object} data - 抽出データ
 * @param {Object} stats - 統計量
 * @param {string} period - 分析期間（例: "2026年3月"）
 * @returns {string} 分析プロンプト
 */
function buildAnalysisPrompt(data, stats, period) {
  // データを CSV 形式に変換（トークン節約のため）
  const csvHeader = data.headers.join(',');
  const csvRows = data.rows.slice(0, 100) // 最大100行に制限
    .map(row => row.join(','))
    .join('\n');

  return `以下は${period}の売上データです。経営レポート向けの分析を行ってください。

## データ概要
- 行数: ${data.rowCount}
- 列: ${data.headers.join(', ')}

## 基本統計
${JSON.stringify(stats, null, 2)}

## データサンプル（CSV形式）
${csvHeader}
${csvRows}

## 分析要件
1. **全体トレンド**: 売上の推移と傾向を3文以内で要約
2. **ハイライト**: 特筆すべき数値やパターンを箇条書き（3-5項目）
3. **課題**: 注意が必要なポイントを箇条書き（2-3項目）
4. **推奨アクション**: データに基づく具体的な改善提案（2-3項目）

Markdown形式で出力してください。見出しは ## を使用してください。`;
}
```

**プロンプト設計の重要なポイント:**
- データ全体ではなくサンプルと統計量を渡すことで、トークン消費を抑える
- 出力形式を明示的に指定し、後処理を容易にする
- 分析の粒度（箇条書きの項目数など）を具体的に指定する

## レポート生成パイプラインの実装

ここまでの要素を組み合わせて、完全なレポート生成パイプラインを構築します。

```javascript
/**
 * メイン: 月次レポート生成パイプライン
 * スプレッドシートのデータを読み取り、Gemini で分析し、
 * レポートシートに出力 + メール配信
 */
function generateMonthlyReport() {
  const startTime = new Date();
  const log = [];

  try {
    // ---- 1. データ抽出 ----
    log.push('Step 1: Extracting data...');
    const salesData = extractSheetData('売上データ');
    const kpiData = extractSheetData('KPI');

    const salesStats = calculateStats(salesData, '売上金額');
    const period = Utilities.formatDate(new Date(), 'Asia/Tokyo', 'yyyy年M月');

    // ---- 2. AI 分析 ----
    log.push('Step 2: Running AI analysis...');
    const client = new GeminiClient();

    const salesAnalysis = client.generate(
      buildAnalysisPrompt(salesData, salesStats, period),
      {
        systemInstruction:
          'あなたはデータアナリストです。正確で簡潔なビジネスレポートを作成してください。' +
          '数値には必ず単位を付け、前月比がある場合は増減率も記載してください。',
        temperature: 0.2
      }
    );

    // KPI サマリー生成
    const kpiSummary = client.generate(
      `以下のKPIデータから、経営ダッシュボード用の要約を生成してください。\n\n` +
      `データ: ${JSON.stringify(kpiData.rows.slice(0, 20))}\n` +
      `列: ${kpiData.headers.join(', ')}\n\n` +
      `各KPIについて現在値・目標値・達成率・トレンド（↑↓→）を1行で記述してください。`,
      { temperature: 0.1, maxTokens: 2048 }
    );

    // ---- 3. レポートシート生成 ----
    log.push('Step 3: Writing report sheet...');
    writeReportSheet(period, salesAnalysis, kpiSummary, salesStats);

    // ---- 4. メール配信 ----
    log.push('Step 4: Sending email...');
    sendReportEmail(period, salesAnalysis, kpiSummary);

    // ---- 5. 実行ログ記録 ----
    const elapsed = (new Date() - startTime) / 1000;
    log.push(`Completed in ${elapsed}s`);
    logExecution('SUCCESS', log.join('\n'));

  } catch (error) {
    log.push(`ERROR: ${error.message}`);
    logExecution('FAILED', log.join('\n'));

    // エラー通知メール
    MailApp.sendEmail({
      to: Session.getActiveUser().getEmail(),
      subject: `[ALERT] Monthly Report Generation Failed`,
      body: `Error: ${error.message}\n\nLog:\n${log.join('\n')}`
    });
  }
}

/**
 * レポートを新しいシートに書き出す
 */
function writeReportSheet(period, analysis, kpiSummary, stats) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheetName = `Report_${period.replace(/[年月]/g, '_')}`;

  // 既存シートがあれば削除して再作成
  let sheet = ss.getSheetByName(sheetName);
  if (sheet) ss.deleteSheet(sheet);
  sheet = ss.insertSheet(sheetName);

  // ヘッダー
  sheet.getRange('A1').setValue(`📊 ${period} 月次レポート`)
    .setFontSize(16).setFontWeight('bold');
  sheet.getRange('A2').setValue(`生成日時: ${new Date().toLocaleString('ja-JP')}`)
    .setFontColor('#666666');

  // 基本統計
  sheet.getRange('A4').setValue('■ 基本統計').setFontWeight('bold');
  const statsEntries = Object.entries(stats);
  statsEntries.forEach(([key, value], i) => {
    sheet.getRange(5 + i, 1).setValue(key);
    sheet.getRange(5 + i, 2).setValue(value)
      .setNumberFormat('#,##0');
  });

  // AI 分析結果
  const analysisRow = 5 + statsEntries.length + 2;
  sheet.getRange(analysisRow, 1).setValue('■ AI 分析').setFontWeight('bold');
  sheet.getRange(analysisRow + 1, 1).setValue(analysis)
    .setWrapStrategy(SpreadsheetApp.WrapStrategy.WRAP);
  sheet.setColumnWidth(1, 600);

  // KPI サマリー
  const kpiRow = analysisRow + 3;
  sheet.getRange(kpiRow, 1).setValue('■ KPI ダッシュボード').setFontWeight('bold');
  sheet.getRange(kpiRow + 1, 1).setValue(kpiSummary)
    .setWrapStrategy(SpreadsheetApp.WrapStrategy.WRAP);
}
```

## 定期実行の設定とトリガー管理

### 時間ベーストリガーの設定

Apps Script のトリガーを使って、レポート生成を自動化します。

```javascript
/**
 * トリガーのセットアップ（月次レポート: 毎月1日 9:00）
 */
function setupMonthlyTrigger() {
  // 既存のトリガーを削除
  ScriptApp.getProjectTriggers()
    .filter(t => t.getHandlerFunction() === 'generateMonthlyReport')
    .forEach(t => ScriptApp.deleteTrigger(t));

  // 新しいトリガーを作成
  ScriptApp.newTrigger('generateMonthlyReport')
    .timeBased()
    .onMonthDay(1)
    .atHour(9)
    .inTimezone('Asia/Tokyo')
    .create();

  console.log('Monthly trigger set: 1st of each month at 9:00 JST');
}

/**
 * 週次サマリートリガー（毎週月曜 8:00）
 */
function setupWeeklyTrigger() {
  ScriptApp.getProjectTriggers()
    .filter(t => t.getHandlerFunction() === 'generateWeeklySummary')
    .forEach(t => ScriptApp.deleteTrigger(t));

  ScriptApp.newTrigger('generateWeeklySummary')
    .timeBased()
    .everyWeeks(1)
    .onWeekDay(ScriptApp.WeekDay.MONDAY)
    .atHour(8)
    .inTimezone('Asia/Tokyo')
    .create();
}
```

### 実行ログの記録

自動実行の履歴を追跡するためのログシステムを実装します。

```javascript
/**
 * 実行ログを専用シートに記録
 */
function logExecution(status, details) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  let logSheet = ss.getSheetByName('_ExecutionLog');

  if (!logSheet) {
    logSheet = ss.insertSheet('_ExecutionLog');
    logSheet.getRange('A1:D1').setValues([
      ['Timestamp', 'Status', 'Duration', 'Details']
    ]).setFontWeight('bold');
  }

  logSheet.appendRow([
    new Date(),
    status,
    details.match(/Completed in (.+)/)?.[1] || 'N/A',
    details
  ]);

  // 古いログの自動削除（90日以上）
  const rows = logSheet.getDataRange().getValues();
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - 90);

  for (let i = rows.length - 1; i >= 1; i--) {
    if (rows[i][0] < cutoff) {
      logSheet.deleteRow(i + 1);
    }
  }
}
```

## メール配信とフォーマット

### HTML メールテンプレート

レポートを見やすい HTML メールとして配信します。

```javascript
/**
 * レポートを HTML メールで送信
 */
function sendReportEmail(period, analysis, kpiSummary) {
  // 配信先リストをシートから取得
  const recipientSheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('配信先');

  if (!recipientSheet) {
    console.log('No recipient sheet found, skipping email');
    return;

  const recipients = recipientSheet.getDataRange().getValues()
    .slice(1) // ヘッダーをスキップ
    .filter(row => row[0] && row[1] === true) // メールアドレスがあり、有効な行
    .map(row => row[0]);

  if (recipients.length === 0) return;

  // Markdown → HTML 変換（簡易版）
  const analysisHtml = markdownToHtml(analysis);
  const kpiHtml = markdownToHtml(kpiSummary);

  const html = `

        <p style="color: rgba(255,255,255,0.85); margin: 8px 0 0; font-size: 13px;">

      </div>

      <div style="padding: 24px; background: #fff; border: 1px solid #e0e0e0;">

      </div>

      <div style="padding: 16px; background: #f5f5f5; border-radius: 0 0 8px 8px; text-align: center; font-size: 12px; color: #999;">
        このレポートは Gemini API によって自動生成されています。
        <a href="${SpreadsheetApp.getActiveSpreadsheet().getUrl()}">スプレッドシートを開く</a>
      </div>
    </div>
  `;

  MailApp.sendEmail({
    to: recipients.join(','),
    subject: `[月次レポート] ${period} — AI 自動分析サマリー`,
    htmlBody: html
  });
}

/**
 * 簡易 Markdown → HTML 変換
 */
function markdownToHtml(md) {
  return md
    .replace(/^## (.+)$/gm, '<h3 style="color: #333; margin-top: 20px;">$1</h3>')
    .replace(/^### (.+)$/gm, '<h4 style="color: #555;">$1</h4>')
    .replace(/\*\*(.+?)\*\*/g, '<strong>$1</strong>')
    .replace(/^- (.+)$/gm, '<li style="margin: 4px 0;">$1</li>')
    .replace(/(<li.*<\/li>\n?)+/g, '<ul style="padding-left: 20px;">$&</ul>')
    .replace(/\n\n/g, '<br><br>')
    .replace(/\n/g, '<br>');
}
```

## コスト管理とレート制限対策

本番運用では、API コストの管理が不可欠です。

```javascript
/**
 * API 使用量トラッカー
 * Script Properties にトークン使用量を蓄積し、月次上限を管理
 */
class UsageTracker {
  constructor(monthlyBudgetUsd = 10) {
    this.props = PropertiesService.getScriptProperties();
    this.monthlyBudget = monthlyBudgetUsd;
    // Gemini 2.5 Flash の概算料金（入力: $0.15/1M tokens, 出力: $0.60/1M tokens）
    this.inputCostPer1k = 0.00015;
    this.outputCostPer1k = 0.0006;
  }

  /**
   * 使用量を記録
   */
  record(inputTokens, outputTokens) {
    const key = `usage_${this._currentMonth()}`;
    const current = JSON.parse(this.props.getProperty(key) || '{"input":0,"output":0,"calls":0}');

    current.input += inputTokens;
    current.output += outputTokens;
    current.calls += 1;

    this.props.setProperty(key, JSON.stringify(current));
    return current;
  }

  /**
   * 現在月の推定コストを取得
   */
  estimatedCost() {
    const key = `usage_${this._currentMonth()}`;
    const usage = JSON.parse(this.props.getProperty(key) || '{"input":0,"output":0}');

    return (usage.input / 1000 * this.inputCostPer1k) +
           (usage.output / 1000 * this.outputCostPer1k);
  }

  /**
   * 予算チェック（超過時は例外を投げる）
   */
  checkBudget() {
    const cost = this.estimatedCost();
    if (cost >= this.monthlyBudget) {
      throw new Error(
        `Monthly API budget exceeded: $${cost.toFixed(2)} / $${this.monthlyBudget}. ` +
        `Reset on the 1st of next month.`
      );
    }
    if (cost >= this.monthlyBudget * 0.8) {
      console.warn(`API budget warning: $${cost.toFixed(2)} / $${this.monthlyBudget} (80%+)`);
    }
  }

  _currentMonth() {
    return Utilities.formatDate(new Date(), 'Asia/Tokyo', 'yyyy-MM');
  }
}
```

この `UsageTracker` を `GeminiClient` に統合することで、予算を超過した場合に自動的にレポート生成を停止できます。レポート自体は翌月にリセットされるまで一時停止し、手動でオーバーライドすることも可能です。

## エラーハンドリングとフォールバック戦略

本番パイプラインでは、あらゆる障害に備える必要があります。

```javascript
/**
 * フォールバック付きレポート生成
 * Gemini API が利用できない場合は統計ベースの簡易レポートにフォールバック
 */
function generateReportWithFallback() {
  const salesData = extractSheetData('売上データ');
  const stats = calculateStats(salesData, '売上金額');
  const period = Utilities.formatDate(new Date(), 'Asia/Tokyo', 'yyyy年M月');

  let analysis;
  let source;

  try {
    const tracker = new UsageTracker(10);
    tracker.checkBudget();

    const client = new GeminiClient();
    analysis = client.generate(
      buildAnalysisPrompt(salesData, stats, period),
      { systemInstruction: 'データアナリストとして簡潔なレポートを作成してください。', temperature: 0.2 }
    );
    source = 'Gemini AI';

  } catch (apiError) {
    console.warn(`Gemini API unavailable: ${apiError.message}`);
    console.log('Falling back to stats-based report...');

    // フォールバック: 統計量ベースの定型レポート
    analysis = generateStatsFallbackReport(stats, period);
    source = 'Auto-generated (API unavailable)';
  }

  writeReportSheet(period, analysis, `Source: ${source}`, stats);
  sendReportEmail(period, analysis, `Source: ${source}`);
}

/**
 * 統計量ベースのフォールバックレポート
 */
function generateStatsFallbackReport(stats, period) {
  return `## ${period} 売上サマリー

**合計売上**: ¥${stats.sum.toLocaleString()}
**平均**: ¥${stats.average.toLocaleString()}（中央値: ¥${stats.median.toLocaleString()}）
**最大**: ¥${stats.max.toLocaleString()} / **最小**: ¥${stats.min.toLocaleString()}
**データ件数**: ${stats.count}件

> ⚠️ このレポートはAI分析なしの統計ベースレポートです。詳細な分析は次回実行時に提供されます。`;
}
```

## よくある質問（FAQ）

### Q1: Apps Script の実行時間制限（6分）に引っかかる場合はどうすればよいですか？

Apps Script には1回の実行で最大6分（Google Workspace アカウントは30分）という制限があります。大量データを分析する場合は、データを分割して複数回の実行に分ける設計が必要です。`CacheService` や `PropertiesService` に中間結果を保存し、次回の実行で続きから処理する「チェーンド実行パターン」を使うと、この制限を回避できます。

### Q2: API キーの管理で Script Properties 以外の選択肢はありますか？

Google Cloud Secret Manager と連携する方法もあります。ただし、個人利用や小規模チームであれば Script Properties で十分です。OAuth 2.0 でサービスアカウント認証を使う場合は、Vertex AI エンドポイントと組み合わせることで、API キーを完全に排除できます。

### Q3: レポートの精度を高めるにはどうすればよいですか？

3つのアプローチがあります。第一に、プロンプトに過去のレポート例を few-shot で含める方法。第二に、`temperature` を 0.1〜0.2 に下げて出力の一貫性を高める方法。第三に、生成後に数値の正確性をプログラム的に検証し、誤りがあれば再生成する方法です。本番環境ではこの3つを組み合わせることをお勧めします。

### Q4: Google Workspace 以外の環境（個人の Gmail）でも使えますか？

はい、利用可能です。ただし、個人アカウントでは Apps Script の実行時間制限が6分（Workspace は30分）、トリガー数の上限が20個（Workspace は40個）など、いくつかの制限が厳しくなります。月次レポート1本程度であれば問題ありません。

## まとめ

この記事では、Apps Script と Gemini API を組み合わせて、スプレッドシートデータの自動分析からレポート生成・配信までを一気通貫で行うパイプラインを構築しました。

重要な設計原則をまとめます。API キーは必ず Script Properties で管理すること。リトライと指数バックオフでネットワーク障害に備えること。コスト管理のために月次予算チェックを組み込むこと。そして Gemini API が利用できない場合に備えてフォールバックを用意すること——これらを押さえておけば、本番環境で安定稼働するパイプラインになります。
