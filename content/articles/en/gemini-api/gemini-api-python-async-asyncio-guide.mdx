## Why Async Processing Matters for Gemini API

When you start building production AI applications with the Gemini API, you'll quickly hit the limits of synchronous code. When a user sends multiple prompts, synchronous processing handles them one at a time — each request waits for the previous one to complete.

By switching to Python's `asyncio` for async processing, you unlock several major benefits:

- **Higher throughput**: Run multiple API calls concurrently, overlapping I/O wait times
- **Faster responses**: Users see results sooner, especially under load
- **Better resource efficiency**: Your CPU isn't blocked waiting for network responses
- **Real scalability**: Handle more concurrent connections on the same infrastructure

This guide covers the Google AI Python SDK's async features step by step — from basic async requests to concurrent batch processing, streaming, rate limiting, and robust error handling. If you're new to the Gemini API, check out our [Gemini API Quickstart Guide](/en/articles/gemini-api/gemini-api-quickstart) first.

## From Sync to Async: The Core Shift

### The Problem with Synchronous Code

```python
import google.generativeai as genai
import time

model = genai.GenerativeModel("gemini-2.5-flash")

def process_questions_sync(questions: list[str]) -> list[str]:
    """Sync processing — runs one request at a time (slow)"""
    results = []
    for q in questions:
        response = model.generate_content(q)
        results.append(response.text)
    return results

questions = ["What is Python?", "What is AI?", "What is asyncio?"]
start = time.time()
answers = process_questions_sync(questions)
print(f"Sync elapsed: {time.time() - start:.2f}s")
# Example output: Sync elapsed: 6.84s
```

Three requests run back to back, so the total time is the sum of all individual latencies.

### The Async Solution

The Google AI Python SDK provides `generate_content_async()` for non-blocking calls. Combine it with `asyncio.gather()` for concurrent execution.

```python
import google.generativeai as genai
import asyncio
import time
import os

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

async def process_single(question: str) -> str:
    """Single async request"""
    response = await model.generate_content_async(question)
    return response.text

async def process_questions_async(questions: list[str]) -> list[str]:
    """Async processing — all requests run concurrently (fast)"""
    tasks = [process_single(q) for q in questions]
    results = await asyncio.gather(*tasks)
    return list(results)

async def main():
    questions = ["What is Python?", "What is AI?", "What is asyncio?"]
    start = time.time()
    answers = await process_questions_async(questions)
    print(f"Async elapsed: {time.time() - start:.2f}s")
    for q, a in zip(questions, answers):
        print(f"Q: {q}\nA: {a[:80]}...\n")

asyncio.run(main())
# Example output: Async elapsed: 2.31s (~3x faster)
```

With `asyncio.gather()`, all coroutines run concurrently, and the total time is roughly equal to the slowest single request rather than the sum of all.

## Concurrent Batch Processing

When processing large volumes of data, it's important to limit concurrency to stay within rate limits while still maximizing throughput.

### Using a Semaphore to Limit Concurrency

```python
import asyncio
import google.generativeai as genai
from typing import Optional

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

# Limit concurrent requests (free tier: ~15 RPM)
CONCURRENCY_LIMIT = 5
semaphore = asyncio.Semaphore(CONCURRENCY_LIMIT)

async def safe_generate(
    text: str,
    index: int,
    retry_count: int = 3
) -> tuple[int, Optional[str]]:
    """Rate-limited async request with exponential backoff retry"""
    async with semaphore:
        for attempt in range(retry_count):
            try:
                response = await model.generate_content_async(text)
                return index, response.text
            except Exception as e:
                if attempt < retry_count - 1:
                    wait_time = 2 ** attempt  # exponential backoff
                    print(f"[{index}] Error ({e}): retrying in {wait_time}s")
                    await asyncio.sleep(wait_time)
                else:
                    print(f"[{index}] Max retries reached: {e}")
                    return index, None

async def batch_process(texts: list[str]) -> list[Optional[str]]:
    """Process a large batch with controlled concurrency"""
    tasks = [safe_generate(text, i) for i, text in enumerate(texts)]
    results_with_index = await asyncio.gather(*tasks)

    # Sort results back to original order
    sorted_results = sorted(results_with_index, key=lambda x: x[0])
    return [result for _, result in sorted_results]

async def main():
    sample_texts = [
        f"Summarize the following text: Document {i} content here."
        for i in range(20)
    ]

    results = await batch_process(sample_texts)
    success_count = sum(1 for r in results if r is not None)
    print(f"Done: {success_count}/{len(sample_texts)} succeeded")

asyncio.run(main())
```

The semaphore ensures you never exceed your target concurrency, while the exponential backoff handles transient errors gracefully. For a deep dive into rate limiting strategies, see our [Gemini API Rate Limiting & Quota Management Guide](/en/articles/gemini-api/gemini-api-rate-limiting-quota-management).

## Async Streaming

The Gemini API supports streaming responses — ideal for chat interfaces where you want to display output incrementally as it's generated.

### Basic Async Streaming

```python
import asyncio
import google.generativeai as genai

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

async def stream_response(prompt: str) -> str:
    """Stream response chunks and print them as they arrive"""
    full_text = ""

    async for chunk in await model.generate_content_async(
        prompt,
        stream=True
    ):
        if chunk.text:
            print(chunk.text, end="", flush=True)
            full_text += chunk.text

    print()  # newline
    return full_text

async def main():
    prompt = "Explain Python async programming to a beginner, step by step."
    result = await stream_response(prompt)
    print(f"\n--- Received {len(result)} characters total ---")

asyncio.run(main())
```

### FastAPI Integration with Server-Sent Events (SSE)

For web applications, pair async streaming with Server-Sent Events to push real-time updates to the browser.

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import google.generativeai as genai
import asyncio
import json

app = FastAPI()
genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")

async def generate_sse_stream(prompt: str):
    """Yield SSE-formatted chunks from the Gemini streaming API"""
    try:
        async for chunk in await model.generate_content_async(
            prompt,
            stream=True
        ):
            if chunk.text:
                data = json.dumps({"text": chunk.text}, ensure_ascii=False)
                yield f"data: {data}\n\n"

        # Signal completion
        yield f"data: {json.dumps({'done': True})}\n\n"

    except Exception as e:
        error_data = json.dumps({"error": str(e)})
        yield f"data: {error_data}\n\n"

@app.get("/api/stream")
async def stream_endpoint(prompt: str):
    """Streaming endpoint — returns SSE response"""
    return StreamingResponse(
        generate_sse_stream(prompt),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",  # Disable Nginx buffering
        }
    )
```

## Async Multi-Turn Chat

Chat applications need to maintain conversation history while serving async responses.

```python
import asyncio
import google.generativeai as genai
from dataclasses import dataclass, field
from typing import AsyncGenerator

genai.configure(api_key=os.environ["YOUR_GEMINI_API_KEY"])

@dataclass
class AsyncChatSession:
    """Async multi-turn chat session with history"""
    model_name: str = "gemini-2.5-flash"
    system_instruction: str = "You are a helpful AI assistant."
    history: list = field(default_factory=list)

    def __post_init__(self):
        self.model = genai.GenerativeModel(
            self.model_name,
            system_instruction=self.system_instruction
        )
        self.chat = self.model.start_chat(history=self.history)

    async def send_message(self, message: str) -> str:
        """Send a message and return the full response"""
        response = await self.chat.send_message_async(message)
        return response.text

    async def stream_message(self, message: str) -> AsyncGenerator[str, None]:
        """Send a message and stream the response"""
        async for chunk in await self.chat.send_message_async(
            message, stream=True
        ):
            if chunk.text:
                yield chunk.text

async def demo_chat():
    """Multi-turn chat demo"""
    session = AsyncChatSession(
        system_instruction="You are a Python expert. Be concise."
    )

    questions = [
        "What are Python list comprehensions?",
        "Can you combine them with asyncio?",
        "Show me a code example."
    ]

    for user_msg in questions:
        print(f"\nUser: {user_msg}")
        print("Gemini: ", end="")

        async for chunk in session.stream_message(user_msg):
            print(chunk, end="", flush=True)
        print()

asyncio.run(demo_chat())
```

## Robust Error Handling

In async code, errors can fail silently or cascade across concurrent tasks if not handled carefully.

```python
import asyncio
import google.generativeai as genai
from google.api_core import exceptions as google_exceptions
import logging

logger = logging.getLogger(__name__)

class AsyncGeminiClient:
    """Production-ready async Gemini client with full error handling"""

    def __init__(self, api_key: str, model_name: str = "gemini-2.5-flash"):
        genai.configure(api_key=api_key)
        self.model = genai.GenerativeModel(model_name)
        self.semaphore = asyncio.Semaphore(10)

    async def generate(
        self,
        prompt: str,
        timeout: float = 30.0,
        max_retries: int = 3
    ) -> str | None:
        """Safe generation with timeout, retry, and backoff"""
        async with self.semaphore:
            for attempt in range(max_retries):
                try:
                    response = await asyncio.wait_for(
                        self.model.generate_content_async(prompt),
                        timeout=timeout
                    )
                    return response.text

                except asyncio.TimeoutError:
                    logger.warning(f"Timeout after {timeout}s: attempt {attempt+1}")
                    if attempt == max_retries - 1:
                        return None

                except google_exceptions.ResourceExhausted as e:
                    # Rate limit hit — wait with exponential backoff
                    wait_time = 60 * (2 ** attempt)
                    logger.warning(f"Rate limit exceeded. Waiting {wait_time}s.")
                    await asyncio.sleep(wait_time)

                except google_exceptions.InvalidArgument as e:
                    # Bad request — no point retrying
                    logger.error(f"Invalid request: {e}")
                    return None

                except Exception as e:
                    logger.error(f"Unexpected error: {e}")
                    if attempt == max_retries - 1:
                        return None
                    await asyncio.sleep(2 ** attempt)

        return None
```

## Performance Tips

### TaskGroup for Structured Concurrency (Python 3.11+)

```python
async def process_with_taskgroup(prompts: list[str]) -> list[str]:
    """Clean structured concurrency using TaskGroup"""
    results = {}

    async with asyncio.TaskGroup() as tg:
        for i, prompt in enumerate(prompts):
            task = tg.create_task(
                model.generate_content_async(prompt),
                name=f"task_{i}"
            )
            results[i] = task

    # All tasks are guaranteed complete after the block
    return [results[i].result().text for i in range(len(prompts))]
```

### In-Memory Response Caching

```python
import hashlib

_cache: dict[str, str] = {}

async def cached_generate(prompt: str) -> str:
    """Skip the API call if we've seen this prompt before"""
    cache_key = hashlib.md5(prompt.encode()).hexdigest()

    if cache_key in _cache:
        print(f"Cache hit: {prompt[:30]}")
        return _cache[cache_key]

    response = await model.generate_content_async(prompt)
    _cache[cache_key] = response.text
    return response.text
```

## FAQ

**Q1. What's the difference between `generate_content()` and `generate_content_async()`?**

`generate_content()` is a synchronous (blocking) function — it freezes the current thread until a response arrives. `generate_content_async()` returns a coroutine that you `await`, allowing the event loop to handle other tasks while waiting. Always use the async version in FastAPI, Starlette, or any other async framework.

**Q2. Do rate limits still apply with async parallel requests?**

Yes — rate limits apply regardless of how you structure your code. In fact, parallelism makes it easier to hit them. That's why `asyncio.Semaphore` and exponential backoff are essential. The free tier allows approximately 15 requests per minute (RPM); paid tiers go up to ~1,000 RPM depending on the model.

**Q3. Can I call `asyncio.run()` more than once in a script?**

Yes, but you cannot call `asyncio.run()` from within a running event loop. In environments like Jupyter Notebook where a loop is already running, use `await coroutine` directly, or install `nest_asyncio` to allow nested loops.

**Q4. What happens if the connection drops during async streaming?**

A network exception (such as `aiohttp.ClientConnectionError`) will be raised inside the async generator. Wrap the generator in `try/except` to return a meaningful error. In FastAPI `StreamingResponse`, a client disconnect triggers `asyncio.CancelledError` — catch it to clean up resources gracefully.
