## Introduction

Working with PDFs, audio recordings, or video files in your AI workflows? The **Gemini Files API** makes it straightforward to upload media files up to **2GB** and reference them in your prompts — without having to re-send large payloads on every request.

Once uploaded, files are cached for **48 hours**, so you can run multiple analyses on the same file without incurring repeated upload costs. This is especially useful for workflows like document summarization, meeting transcription, or video analysis where you need to query the same source material multiple times.

In this guide, we'll cover everything from basic uploads to advanced use cases, with production-ready Python code examples throughout.

## What Is the Files API?

The Files API is a file management service within the Gemini API ecosystem. It provides four core operations:

- **Upload** — Send local files to Google's servers for processing
- **List** — View all currently uploaded files in your project
- **Get** — Check the status of a specific file
- **Delete** — Remove files before the 48-hour expiration

Uploaded files are scoped to your API key's project. No other users or projects can access your files, and everything is automatically purged after 48 hours.

### Supported File Types

| Category | Formats | Example MIME Types |
|----------|---------|-------------------|
| Images | PNG, JPEG, WebP, HEIC, HEIF | `image/png`, `image/jpeg` |
| Audio | MP3, WAV, FLAC, AAC, OGG | `audio/mp3`, `audio/wav` |
| Video | MP4, MPEG, MOV, AVI, WebM | `video/mp4`, `video/quicktime` |
| Documents | PDF, TXT, HTML, CSS, JS | `application/pdf`, `text/plain` |
| Data | CSV, XML, JSON | `text/csv`, `application/json` |

## Uploading and Processing Files

### Basic PDF Upload

Here's how to upload a PDF and generate a summary:

```python
from google import genai
import os

# Initialize the client
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Upload a PDF file
uploaded_file = client.files.upload(
    file="report.pdf",
    config={"display_name": "quarterly-report"}
)

print(f"Upload complete: {uploaded_file.name}")
print(f"URI: {uploaded_file.uri}")
print(f"State: {uploaded_file.state}")
# Expected output:
# Upload complete: files/xxxxxxxxxxxx
# URI: https://generativelanguage.googleapis.com/v1beta/files/xxxxxxxxxxxx
# State: ACTIVE

# Summarize the uploaded PDF
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        "Summarize this PDF in 3 key bullet points.",
        uploaded_file
    ]
)

print(response.text)
```

### Working with Video Files

Video files require processing time after upload. You need to poll the file status before using it in prompts.

```python
import time
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Upload a video
video_file = client.files.upload(file="presentation.mp4")

# Wait for processing to complete
while video_file.state == "PROCESSING":
    print("Processing video...")
    time.sleep(5)
    video_file = client.files.get(name=video_file.name)

if video_file.state == "ACTIVE":
    print("Processing complete! Starting analysis.")

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=[
            "Analyze this video and list the main topics with timestamps.",
            video_file
        ]
    )
    print(response.text)
else:
    print(f"Error: File state is {video_file.state}")
# Expected output:
# Processing video...
# Processing video...
# Processing complete! Starting analysis.
# 00:00 - Introduction: Project overview
# 02:30 - Technical architecture walkthrough
# ...
```

## Managing Uploaded Files

### Listing Files

```python
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# List all uploaded files
for f in client.files.list():
    print(f"{f.name} | {f.display_name} | {f.mime_type} | {f.state}")
# Expected output:
# files/abc123 | quarterly-report | application/pdf | ACTIVE
# files/def456 | presentation.mp4 | video/mp4 | ACTIVE
```

### Deleting Files

You can manually delete files instead of waiting for the 48-hour auto-expiration:

```python
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Delete a file
client.files.delete(name="files/abc123")
print("File deleted successfully")
```

## Practical Use Cases

### Use Case 1: Batch PDF Analysis

Upload multiple PDFs and analyze them together for cross-document insights:

```python
from google import genai
import glob

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Upload multiple PDFs
pdf_files = []
for pdf_path in glob.glob("reports/*.pdf"):
    uploaded = client.files.upload(file=pdf_path)
    pdf_files.append(uploaded)
    print(f"Uploaded: {pdf_path}")

# Cross-reference analysis
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        "Analyze all these reports together. Identify "
        "common trends and unique insights from each document.",
        *pdf_files
    ]
)

print(response.text)
```

### Use Case 2: Automated Meeting Minutes from Video

```python
from google import genai
import time

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

video = client.files.upload(file="team_meeting.mp4")

while video.state == "PROCESSING":
    time.sleep(5)
    video = client.files.get(name=video.name)

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=[
        "Create meeting minutes from this video in this format:\n"
        "1. Date and attendees\n"
        "2. Agenda items\n"
        "3. Discussion summary and decisions for each topic\n"
        "4. Action items with assignees and deadlines",
        video
    ]
)

print(response.text)
```

## Common Errors and Troubleshooting

| Error | Cause | Solution |
|-------|-------|----------|
| `FAILED_PRECONDITION` | File is still processing | Wait until `state` becomes `ACTIVE` |
| `INVALID_ARGUMENT` | Unsupported file format | Check the supported formats table above |
| `RESOURCE_EXHAUSTED` | Upload quota exceeded | Delete unused files and retry |
| `NOT_FOUND` | File expired (past 48 hours) | Re-upload the file |

## FAQ

### Q1: What is the maximum file size for the Files API?

You can upload files up to **2GB** per file. However, the total content size you can include in a single `generateContent` request depends on the model's context window. For large files, consider using `gemini-2.5-pro` which supports up to 1 million tokens.

### Q2: Are my uploaded files secure?

Yes. Uploaded files are scoped to your API key's project and cannot be accessed by other users or projects. Files are automatically deleted after 48 hours. For sensitive data, you can call `files.delete()` immediately after processing to remove them sooner.

### Q3: When should I use the Files API instead of inline Base64 data?

For small files (under a few MB), encoding as Base64 and passing them as `inline_data` works fine. The Files API becomes the better choice when:
- Your file is larger than 20MB
- You need to reference the same file across multiple requests
- You're working with video files that require server-side processing before use

### Q4: Can I combine the Files API with context caching?

Absolutely. You can use files uploaded via the Files API together with [context caching](/en/articles/gemini-api/context-caching-guide) to further reduce costs when processing large files repeatedly. This is particularly effective for long-running analysis sessions where you query the same document multiple times.
