In this guide, we'll walk through implementing image recognition and text extraction using the Google AI Python SDK (`google-genai`). Even if you've never used the Gemini API before, you'll be building practical image analysis tools by the end of this article.

## Basic Text Extraction from Images

Let's start with the simplest possible implementation — sending a single image to Gemini and extracting the text it contains.

```python
from google import genai
from google.genai import types
import pathlib

# Initialize the client
client = genai.Client()

# Load the image file
image_path = pathlib.Path("receipt.jpg")
image_data = image_path.read_bytes()

# Send the image to Gemini API for text extraction
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        types.Part.from_bytes(data=image_data, mime_type="image/jpeg"),
        "Extract all text from this image accurately."
    ],
)

print(response.text)
```

**Expected output (for a receipt image):**

```
GROCERY MART
123 Main Street, Suite 100
San Francisco, CA 94102
TEL: (415) 555-0123

03/28/2026  2:32 PM
Register #3  Cashier: Smith

Whole Milk 1 gal       $4.99
Bread Wheat Loaf        $3.49
Eggs Large 12ct         $5.29
---
Subtotal               $13.77
Tax (8.5%)              $1.17
Total                  $14.94
```

The key takeaway here is how simple the code is — you just bundle the image data and a prompt into the `contents` list and send it off. Gemini handles the recognition and returns the extracted text.

## Analyzing Images Directly from URLs

Beyond local files, you can also extract text directly from web image URLs.

```python
from google import genai
from google.genai import types

client = genai.Client()

# Fetch and analyze an image from a URL
image_url = "https://example.com/business-card.png"

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        types.Part.from_uri(file_uri=image_url, mime_type="image/png"),
        "Extract the information from this business card and organize it into: Name, Company, Title, Phone, and Email.",
    ],
)

print(response.text)
```

**Expected output:**

```
- Name: Jane Smith
- Company: Acme Technologies Inc.
- Title: Engineering Manager
- Phone: (415) 555-0456
- Email: jane.smith@example.com
```

By simply instructing Gemini to "organize it into categories," you get structured data back. This contextual understanding is something traditional OCR libraries simply can't match.

## Extracting Structured Data as JSON

In production workflows, you typically want extracted text as structured data rather than plain text. Gemini API's `response_mime_type` parameter guarantees JSON output.

```python
from google import genai
from google.genai import types
import json

client = genai.Client()

image_path = "invoice.png"
with open(image_path, "rb") as f:
    image_data = f.read()

# Extract structured data as JSON
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[
        types.Part.from_bytes(data=image_data, mime_type="image/png"),
        """Extract the following information from this invoice image in JSON format:
        - company_name: the billing company's name
        - invoice_number: the invoice number
        - date: the issue date
        - items: line items (array with name, quantity, unit_price, amount)
        - subtotal: subtotal amount
        - tax: tax amount
        - total: total amount""",
    ],
    config=types.GenerateContentConfig(
        response_mime_type="application/json",
    ),
)

# Parse and use the JSON data
data = json.loads(response.text)
print(json.dumps(data, indent=2))
```

**Expected output:**

```json
{
  "company_name": "Acme Consulting LLC",
  "invoice_number": "INV-2026-0328",
  "date": "2026-03-28",
  "items": [
    {
      "name": "Website Development",
      "quantity": 1,
      "unit_price": 5000,
      "amount": 5000
    },
    {
      "name": "Server Maintenance (monthly)",
      "quantity": 3,
      "unit_price": 300,
      "amount": 900
    }
  ],
  "subtotal": 5900,
  "tax": 531,
  "total": 6431
}
```

With this approach, you can build automated pipelines that ingest invoice and receipt data directly into your database or accounting system.

## Batch Processing Multiple Images

When you need to process a large number of images, here's a practical batch processing script.

```python
from google import genai
from google.genai import types
import pathlib
import json
import time

client = genai.Client()

def extract_text_from_image(image_path: pathlib.Path) -> dict:
    """Extract text from an image and return structured data."""
    image_data = image_path.read_bytes()

    # Determine MIME type from file extension
    mime_map = {".jpg": "image/jpeg", ".jpeg": "image/jpeg",
                ".png": "image/png", ".webp": "image/webp"}
    mime_type = mime_map.get(image_path.suffix.lower(), "image/jpeg")

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=[
            types.Part.from_bytes(data=image_data, mime_type=mime_type),
            "Extract all text from this image and return it as JSON."
            "Use keys 'extracted_text' (full text) and 'summary' (brief summary).",
        ],
        config=types.GenerateContentConfig(
            response_mime_type="application/json",
        ),
    )

    result = json.loads(response.text)
    result["source_file"] = str(image_path.name)
    return result

# Process all images in a directory
image_dir = pathlib.Path("./images")
results = []

for img_file in sorted(image_dir.glob("*.{jpg,png,jpeg,webp}")):
    print(f"Processing: {img_file.name}")
    try:
        result = extract_text_from_image(img_file)
        results.append(result)
        # Respect API rate limits
        time.sleep(1)
    except Exception as e:
        print(f"  Error: {e}")
        results.append({"source_file": img_file.name, "error": str(e)})

# Save results to a JSON file
output_path = pathlib.Path("extraction_results.json")
output_path.write_text(json.dumps(results, indent=2))
print(f"\nDone: saved {len(results)} results to {output_path}")
```

If you're dealing with hundreds of images or more, consider using asynchronous batch processing as explained in [Gemini Batch Processing API Complete Guide](/en/articles/gemini-api/gemini-batch-processing-api-guide). It processes large volumes of requests efficiently and cuts costs by 50%.

## Prompt Techniques for Better Recognition Accuracy

The quality of Gemini's image recognition depends heavily on how you craft your prompts. Here are some key techniques to keep in mind.

**Specify the language explicitly:**

```python
# For images with mixed languages
prompt = "Extract all text from this image. Distinguish between English and Japanese text."
```

**Preserve layout information:**

```python
# For spreadsheet screenshots or tabular data
prompt = """This image is a screenshot of a spreadsheet.
Extract all cell contents while preserving the table layout
(row and column relationships). Output as CSV format,
including the header row."""
```

**Handle handwritten text:**

```python
# For handwritten notes or letters
prompt = """This image contains handwritten text.
Read it as accurately as possible.
Mark any parts you're uncertain about with [?]."""
```

**Describe the image type:**

Telling Gemini what kind of image it's looking at — "This is a receipt," "This is a business card," "This is a whiteboard photo" — provides context that significantly improves recognition accuracy.

## Frequently Asked Questions

### Q: What's the maximum image size Gemini API can process?

A: Gemini API can handle images up to 20MB per request. Supported formats include JPEG, PNG, WebP, HEIC, and HEIF. For large images, resizing with `pillow` before sending will also improve API response times.

### Q: What are the advantages over traditional OCR libraries like Tesseract?

A: Gemini's biggest advantage is contextual understanding. Traditional OCR recognizes characters one by one, while Gemini understands the entire document layout — table structures, paragraph relationships, and reading order. It also performs well on handwritten text and low-quality images. Beyond that, you can instruct Gemini to summarize, convert to JSON, or translate the extracted text all in one request, which dramatically simplifies your downstream pipeline.

### Q: How much can I process with the free tier?

A: With Gemini 2.5 Flash's free tier, you can make up to 15 requests per minute. Since each image takes one request, processing several hundred images per day is perfectly feasible. For higher volumes, upgrading to a paid plan increases your rate limits. Check [Google AI Studio](https://aistudio.google.com/) for current pricing details.

### Q: Can Gemini recognize text in different scripts and writing directions?

A: Yes, Gemini handles a wide variety of scripts and writing directions, including vertical Japanese text, right-to-left Arabic and Hebrew, and complex scripts like Chinese and Korean. Adding context to your prompt — such as "This document contains vertical Japanese text" — helps improve accuracy for non-standard layouts.
