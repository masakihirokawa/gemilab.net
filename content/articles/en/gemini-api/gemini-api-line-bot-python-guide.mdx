## Introduction

LINE is the dominant messaging platform in Japan and across much of East and Southeast Asia, with over 97 million monthly active users in Japan alone. Building an AI-powered chatbot on LINE gives you direct access to this massive user base — whether for customer support automation, personal assistants, or interactive marketing tools.

In this tutorial, you'll learn how to combine Google's Gemini API with the LINE Messaging API to build a Python-based chatbot that understands context, handles multi-turn conversations, and even analyzes images users send.

**What you'll learn:**

- Creating a LINE Messaging API channel on LINE Developers
- Setting up the Gemini API with the Google AI Python SDK
- Building a Flask webhook server that routes messages through Gemini
- Implementing multi-turn conversation history
- Deploying to Google Cloud Run for production use

If you're new to the Gemini API, check out the [Gemini API Quickstart](/en/articles/gemini-api/gemini-api-quickstart) first.

## Building the LINE Bot Server

### Main Application

Here's the complete Flask webhook server that receives LINE messages and generates responses with the Gemini API:

```python
# app.py
import os
from flask import Flask, request, abort
from google import genai
from linebot.v3 import WebhookParser
from linebot.v3.messaging import (
    Configuration,
    ApiClient,
    MessagingApi,
    ReplyMessageRequest,
    TextMessage,
)
from linebot.v3.webhooks import MessageEvent, TextMessageContent

app = Flask(__name__)

# --- Configuration ---
GEMINI_API_KEY = os.environ["GEMINI_API_KEY"]
LINE_CHANNEL_SECRET = os.environ["LINE_CHANNEL_SECRET"]
LINE_CHANNEL_ACCESS_TOKEN = os.environ["LINE_CHANNEL_ACCESS_TOKEN"]

# Initialize Gemini client
gemini_client = genai.Client(api_key=GEMINI_API_KEY)

# Initialize LINE API
configuration = Configuration(access_token=LINE_CHANNEL_ACCESS_TOKEN)
parser = WebhookParser(LINE_CHANNEL_SECRET)

# Per-user conversation history (in-memory)
conversation_history: dict[str, list[dict]] = {}
MAX_HISTORY = 20  # Keep the most recent 20 messages


def get_gemini_response(user_id: str, user_message: str) -> str:
    """Generate a multi-turn response using the Gemini API."""
    # Retrieve or initialize conversation history
    if user_id not in conversation_history:
        conversation_history[user_id] = []

    history = conversation_history[user_id]

    # Append the user message
    history.append({"role": "user", "parts": [{"text": user_message}]})

    # Call the Gemini API
    response = gemini_client.models.generate_content(
        model="gemini-3-flash",
        contents=history,
        config={
            "system_instruction": (
                "You are a friendly AI assistant running on LINE. "
                "Keep your answers concise (under 200 words) and "
                "conversational. Be helpful and approachable."
            ),
            "temperature": 0.7,
            "max_output_tokens": 500,
        },
    )

    assistant_message = response.text

    # Append the assistant response to history
    history.append({"role": "model", "parts": [{"text": assistant_message}]})

    # Trim history if it gets too long
    if len(history) > MAX_HISTORY:
        conversation_history[user_id] = history[-MAX_HISTORY:]

    return assistant_message


@app.route("/callback", methods=["POST"])
def callback():
    """LINE Webhook endpoint."""
    signature = request.headers.get("X-Line-Signature", "")
    body = request.get_data(as_text=True)

    try:
        events = parser.parse(body, signature)
    except Exception:
        abort(400)

    for event in events:
        if isinstance(event, MessageEvent) and isinstance(
            event.message, TextMessageContent
        ):
            user_id = event.source.user_id
            user_message = event.message.text

            # Generate a response with Gemini
            reply_text = get_gemini_response(user_id, user_message)

            # Send the reply via LINE
            with ApiClient(configuration) as api_client:
                line_bot_api = MessagingApi(api_client)
                line_bot_api.reply_message(
                    ReplyMessageRequest(
                        reply_token=event.reply_token,
                        messages=[TextMessage(text=reply_text)],
                    )
                )

    return "OK"


@app.route("/health", methods=["GET"])
def health():
    """Health check endpoint."""
    return "OK"


if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8080, debug=True)
```

### Key Design Decisions

- **`conversation_history`**: A dictionary keyed by user ID that stores the conversation context. This enables natural follow-up messages like "tell me more about that" without losing context
- **`system_instruction`**: Tells Gemini how to behave as a LINE bot — controlling tone, response length, and personality
- **`MAX_HISTORY`**: Caps memory usage at 20 messages per user. For production, consider persisting history in Redis or Firestore instead of in-memory storage

import requests
from linebot.v3.webhooks import ImageMessageContent


def download_line_image(message_id: str) -> bytes:
    """Download an image from LINE's servers."""
    url = f"https://api-data.line.me/v2/bot/message/{message_id}/content"
    headers = {"Authorization": f"Bearer {LINE_CHANNEL_ACCESS_TOKEN}"}
    resp = requests.get(url, headers=headers)
    return resp.content


def get_gemini_image_response(user_id: str, image_bytes: bytes) -> str:
    """Send an image to Gemini and get an analysis."""
    import base64

    image_b64 = base64.b64encode(image_bytes).decode("utf-8")

    contents = [
        {
            "role": "user",
            "parts": [

        }

        model="gemini-3-flash",

            "system_instruction": (

## Deploying to Google Cloud Run

For production, Cloud Run provides serverless, auto-scaling infrastructure that's ideal for LINE Bot backends.

### Creating the Dockerfile

```dockerfile
FROM python:3.11-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY app.py .

CMD ["gunicorn", "--bind", "0.0.0.0:8080", "--workers", "2", "app:app"]
```

### requirements.txt

```text
google-genai>=1.0.0
flask>=3.0
line-bot-sdk>=3.0
gunicorn>=22.0
requests>=2.31
```

### Deploy Command

```bash
# Set your Google Cloud project
gcloud config set project YOUR_PROJECT_ID

# Deploy to Cloud Run with environment variables
gcloud run deploy gemini-line-bot \
  --source . \
  --region asia-northeast1 \
  --allow-unauthenticated \
  --set-env-vars "GEMINI_API_KEY=your_key,LINE_CHANNEL_SECRET=your_secret,LINE_CHANNEL_ACCESS_TOKEN=your_token" \
  --memory 512Mi \
  --min-instances 0 \
  --max-instances 10

# Output:
# Service URL: https://gemini-line-bot-xxxxx-an.a.run.app
```

After deployment, update the Webhook URL in LINE Developers Console to your Cloud Run URL (`https://gemini-line-bot-xxxxx-an.a.run.app/callback`).

## Frequently Asked Questions

### Q1: The bot's responses feel slow. How can I speed things up?

Gemini API response times typically range from 1–3 seconds. Here are some ways to reduce perceived latency:

- Use **`gemini-3-flash`** instead of Pro (it's optimized for speed)
- Set a lower **`max_output_tokens`** to limit response length
- Set Cloud Run's **minimum instances to 1** to avoid cold starts
- Send a "Thinking..." push message immediately, then follow up with the actual AI response asynchronously

### Q2: How do I persist conversation history?

The sample code stores history in memory, which is lost when Cloud Run instances restart. For production, consider these options:

- **Firebase Firestore**: Serverless and integrates well with Cloud Run. Store conversations as documents keyed by user ID
- **Redis (Memorystore)**: Best for low-latency reads and writes
- **Cloud SQL (PostgreSQL)**: Ideal if you also need search and analytics on conversation data

### Q3: How can I give the bot a unique personality?

Customize the `system_instruction` to shape the bot's character:

```python
config={
    "system_instruction": (
        "You are 'Mika', a cheerful cooking assistant. "
        "You love recommending recipes and sharing cooking tips. "
        "Keep your tone warm, enthusiastic, and supportive. "
        "When someone asks about non-cooking topics, be friendly "
        "and gently steer back to food when appropriate."
    ),
}
```

### Q4: What happens if I hit Gemini's rate limits?

The Gemini API free tier has per-minute request limits. When you hit them:

- Implement **exponential backoff** retry logic
- **Upgrade your API key** to a paid plan via Google AI Studio
- Use **Context Caching** to reduce token consumption across conversations

For a detailed guide on handling rate limits, see [Gemini API Rate Limiting and Quota Management](/en/articles/gemini-api/gemini-api-rate-limiting-quota-management).

### Q5: What security measures should I implement?

Key security practices for a production LINE Bot:

- **Signature verification**: Always verify the `X-Line-Signature` header on incoming webhooks (the `WebhookParser` in our code handles this automatically)
- **Environment variables**: Never hardcode API keys or tokens. Use Cloud Run environment variables or Secret Manager
- **Input validation**: Trim excessively long user inputs before sending them to the Gemini API
- **HTTPS enforcement**: Cloud Run provides HTTPS by default
