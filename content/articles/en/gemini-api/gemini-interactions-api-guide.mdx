## Introduction: What Is the Gemini Interactions API?

In March 2026, Google launched the **Gemini Interactions API** in Beta — a unified interface for interacting with Gemini models and agents that significantly expands on the traditional `generate_content`-based approach.

Where the classic Gemini API is stateless (request/response), the Interactions API **automatically manages multi-turn conversation state** and allows developers to embed advanced agentic capabilities — like Deep Research — directly into their applications.

### Key Benefits of the Interactions API

- **Automatic conversation state management** — Maintain multi-turn dialogues with a simple session ID, no manual history tracking
- **Integrated agent capabilities** — Call the Deep Research agent and other advanced AI features through a single API
- **Thought Signature support** — Preserve Gemini 3's reasoning chain across conversation turns
- **Unified tool management** — Seamlessly control Search, Code Execution, File Search, and more via one interface

## Basic Usage: Sending Your First Interaction

The core of the Interactions API is the `client.interactions` object. Use the `start()` method to open a new interaction session.

```python
import os
from google import genai

# Initialize the client
client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Start a new Interaction session
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    system_instruction="You are a Python expert. Answer coding questions clearly and concisely."
)

# Send the first message
response = interaction.send_message(
    "What is the most efficient way to generate a Fibonacci sequence in Python?"
)

print(response.text)
# Expected output:
# There are several efficient ways to generate a Fibonacci sequence in Python.
# The most memory-efficient approach uses a generator...
```

## Multi-Turn Conversations: Automatic State Management

One of the biggest advantages of the Interactions API is that you no longer need to manually track conversation history. The `interaction` object handles state internally.

```python
import os
from google import genai

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Start a session
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    system_instruction="You are an expert code reviewer."
)

# Turn 1: Submit code for review
response1 = interaction.send_message(
    """
    Please review the following code:

    ```python
    def get_user(id):
        conn = sqlite3.connect('users.db')
        cursor = conn.cursor()
        cursor.execute(f"SELECT * FROM users WHERE id = {id}")
        return cursor.fetchone()
    ```
    """
)
print("=== Code Review (Round 1) ===")
print(response1.text)

# Turn 2: Follow-up question — history is managed automatically
response2 = interaction.send_message(
    "From a security standpoint, what is the single most critical issue? Please also provide a corrected version."
)
print("\n=== Code Review (Round 2) ===")
print(response2.text)

# Inspect session info
print(f"\nSession ID: {interaction.session_id}")
print(f"Turn count: {interaction.turn_count}")
# Expected output:
# Session ID: int_a1b2c3d4e5f6
# Turn count: 2
```

## Thought Signatures: Preserving Reasoning Chains Across Turns

Gemini 3 introduced **Thought Signatures** — encrypted representations of the model's internal reasoning. Combined with the Interactions API, they allow complex multi-step reasoning chains to persist efficiently across conversation turns.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Start a session with Thought Signatures enabled
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    config=types.InteractionConfig(
        thinking_config=types.ThinkingConfig(
            include_thoughts=True,       # Include the reasoning process
            thinking_level="high",       # Depth of reasoning: low / medium / high
        )
    )
)

# Ask a complex question
response = interaction.send_message(
    "Explain how to improve an algorithm from O(n²) to O(n log n) complexity, using merge sort as an example."
)

# Check for Thought Signatures
if response.thoughts:
    print("=== Reasoning Process (Thought Signature) ===")
    print(response.thoughts[:200] + "...\n")

print("=== Final Answer ===")
print(response.text)
# Expected output:
# === Reasoning Process (Thought Signature) ===
# [encrypted thought signature data]...
#
# === Final Answer ===
# Merge sort achieves O(n log n) time complexity by applying the divide-and-conquer strategy...
```

## Calling the Deep Research Agent

The Interactions API truly shines when integrated with the **Deep Research agent**, enabling autonomous multi-step research to be embedded directly in your application.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Start a session with Deep Research enabled
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview",
    tools=[
        types.Tool(google_search=types.GoogleSearch()),   # Allow web research
        types.Tool(code_execution=types.CodeExecution()), # Allow code execution
    ],
    config=types.InteractionConfig(
        agent_config=types.AgentConfig(
            deep_research=True  # Enable Deep Research mode
        )
    )
)

print("Starting research task...")

# Submit a research task
response = interaction.send_message(
    """
    Research the topic: "AI Agent Frameworks in 2026"
    Please cover:
    1. The current state of major frameworks (LangChain, LlamaIndex, Google ADK, etc.)
    2. Key features and ideal use cases for each
    3. Trends in the developer community
    """
)

# Review the research steps taken
if response.research_steps:
    print(f"\nResearch steps executed: {len(response.research_steps)}")
    for i, step in enumerate(response.research_steps, 1):
        print(f"  Step {i}: {step.description}")

print("\n=== Research Summary ===")
print(response.text)
# Expected output:
# Research steps executed: 5
#   Step 1: Google Search for latest framework updates
#   Step 2: Reviewed LangChain official documentation
#   ...
# === Research Summary ===
# AI Agent Frameworks in 2026: A Comprehensive Overview...
```

## Saving and Resuming Sessions

With the Interactions API, you can resume conversations later using a session ID — ideal for long-running workflows or multi-session user interactions.

```python
import os
from google import genai

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# --- Start and save a session ---
interaction = client.interactions.start(
    model="gemini-3.1-pro-preview"
)

response = interaction.send_message("Give me an introduction to machine learning.")
print(response.text[:100] + "...")

# Save the session ID (e.g., to a database or file)
saved_session_id = interaction.session_id
print(f"\nSaved session ID: {saved_session_id}")

# --- Resume the session later ---
resumed_interaction = client.interactions.resume(
    session_id=saved_session_id,
    model="gemini-3.1-pro-preview"
)

# Continue from where we left off
response2 = resumed_interaction.send_message(
    "Can you go into more detail about neural networks specifically?"
)
print("\n=== Resumed Response ===")
print(response2.text)
# Expected output:
# === Resumed Response ===
# Of course! Building on the machine learning overview I just gave,
# let me explain neural networks in more depth...
```

## Streaming: Receive Responses in Real Time

For long responses or research results, use streaming mode to display output as it is generated.

```python
import os
from google import genai

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

interaction = client.interactions.start(
    model="gemini-3.1-pro-preview"
)

print("=== Streaming Response ===")
# Use send_message_stream for real-time output
for chunk in interaction.send_message_stream(
    "Explain how Python's asyncio event loop works in detail."
):
    if chunk.text:
        print(chunk.text, end="", flush=True)

print("\n\n=== Done ===")
# Expected output:
# === Streaming Response ===
# Python's asyncio event loop works by...
# (text appears in real time)
# === Done ===
```

## Interactions API vs. `generate_content`: A Comparison

| Feature | `generate_content` | Interactions API |
|---------|-------------------|-----------------|
| **Conversation state** | Manual management required | Automatic |
| **Multi-turn** | Manually append to `contents` | `send_message()` only |
| **Agent capabilities** | Limited | Deep Research and more |
| **Thought Signatures** | Not supported | Supported (Gemini 3+) |
| **Session resumption** | Not possible | Via session ID |
| **Best for** | Simple, one-shot requests | Complex dialogues & agents |

**Recommendation**: Use `generate_content` for simple single-turn requests. Use the **Interactions API** for multi-turn conversations, agentic workflows, and anything requiring persistent reasoning state.

## FAQ

**Q1: Is the Interactions API available for all Gemini models?**

A: As of March 2026, support has been confirmed for **Gemini 3.1 Pro Preview** and **Gemini 3 Flash Preview**. Support for Gemini 2.5 series and older models is expected to expand. Check the [official models documentation](https://ai.google.dev/gemini-api/docs/models) for the latest compatibility list.

**Q2: How long are Interactions API sessions retained?**

A: In the current Beta, sessions are retained server-side for up to **24 hours**. For longer-lived conversations, you can save the conversation history locally and pass it via the `history` parameter when starting a new session.

**Q3: How does Deep Research mode affect API costs?**

A: When Deep Research mode is active, multiple Google Search queries are fired internally. With the March 2026 pricing update, search is billed at **$14 per 1,000 queries** (usage-based). To control costs, use the `max_search_queries` parameter to cap the number of searches per research task.
