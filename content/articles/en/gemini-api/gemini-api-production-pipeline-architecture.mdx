## Introduction: Production Pipelines with Gemini

Building reliable AI pipelines at scale requires more than simple API calls. This guide covers architectural patterns for production Gemini implementations: request batching, streaming for low-latency responses, cost optimization strategies, and robust error handling.

We'll focus on practical patterns validated in production environments processing millions of requests monthly. The examples use TypeScript and Python and assume familiarity with asynchronous programming and HTTP client libraries.

## Architecture Overview

A production Gemini pipeline consists of:

1. **Request Router**: Determines model selection (Flash-Lite vs. Flash) based on task complexity
2. **Batch Processor**: Groups similar requests for cost efficiency
3. **Streaming Handler**: Manages token-by-token responses for real-time applications
4. **Retry Logic**: Implements exponential backoff and circuit breaker patterns
5. **Monitoring & Logging**: Tracks latency, costs, and error rates

## 1. Cost Optimization with Flash-Lite

Flash-Lite is 40% cheaper than standard Flash for most tasks. The key is routing intelligence:

```typescript
type TaskComplexity = 'simple' | 'moderate' | 'complex';

function selectModel(task: TaskComplexity): string {
  const modelMap = {
    simple: 'gemini-3.1-flash-lite',    // $0.075 per 1M tokens
    moderate: 'gemini-3.1-flash-lite',  // Use Flash-Lite for 80% of tasks
    complex: 'gemini-3.1-flash'         // $0.125 per 1M tokens
  };
  return modelMap[task];
}

interface PipelineTask {
  id: string;
  type: TaskComplexity;
  prompt: string;
  context?: string;
}

async function routeRequest(task: PipelineTask): Promise<string> {
  const model = selectModel(task.type);
  const response = await callGemini(model, task.prompt);
  return response;
}
```

- Processing 100k emails (classification): Flash-Lite = $7.50, Flash = $12.50
- Analyzing 500 documents: Flash-Lite = $3.75, Standard = $6.25
- Monthly savings at scale: 35-40%

## 2. Batch Processing for Throughput

Batch processing is essential for non-real-time workloads. Instead of making 10,000 individual API calls, group them into batches of 100-500:

```typescript
interface BatchRequest {
  requests: Array<{
    custom_id: string;
    params: {
      model: string;
      system_instruction: string;
      messages: Array<{ role: string; content: string }>;
    };
  }>;
}

class BatchProcessor {
  private queue: PipelineTask[] = [];
  private batchSize = 100;

  async addTask(task: PipelineTask): Promise<void> {
    this.queue.push(task);
    if (this.queue.length >= this.batchSize) {
      await this.processBatch();
    }
  }

  async processBatch(): Promise<void> {
    const batch = this.queue.splice(0, this.batchSize);
    const requests = batch.map(task => ({
      custom_id: task.id,
      params: {
        model: selectModel(task.type),
        system_instruction: 'Process the following with precision.',
        messages: [{ role: 'user', content: task.prompt }]
      }
    }));

    const batchInput = { requests };
    const response = await fetch('https://generativelanguage.googleapis.com/batch', {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${process.env.GEMINI_API_KEY}` },
      body: JSON.stringify(batchInput)
    });

    const result = await response.json();
    console.log(`Batch ${result.id} submitted. Processing in background.`);
    await this.pollBatchStatus(result.id);
  }

  private async pollBatchStatus(batchId: string): Promise<void> {
    let isComplete = false;
    while (!isComplete) {
      const response = await fetch(
        `https://generativelanguage.googleapis.com/batches/${batchId}`,
        { headers: { 'Authorization': `Bearer ${process.env.GEMINI_API_KEY}` } }
      );
      const batch = await response.json();
      isComplete = batch.state === 'COMPLETED';
      if (!isComplete) await new Promise(r => setTimeout(r, 30000)); // Poll every 30s
    }
  }
}

// Usage
const processor = new BatchProcessor();
for (let i = 0; i < 5000; i++) {
  await processor.addTask({
    id: `task-${i}`,
    type: 'simple',
    prompt: `Classify this text: "${generateText()}"`
  });
}
```

**Performance Impact:**
- Individual calls: 5,000 requests × 500ms = 2,500 seconds
- Batched (100 per batch): 50 batches × 2-5 minutes processing = 100-250 seconds total

## 3. Streaming for Real-Time Responses

For user-facing applications requiring immediate feedback, streaming returns tokens incrementally:

```typescript
async function* streamGeminiResponse(
  model: string,
  prompt: string
): AsyncGenerator<string> {
  const response = await fetch('https://generativelanguage.googleapis.com/stream', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.GEMINI_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model,
      contents: [{ role: 'user', parts: [{ text: prompt }] }],
      stream: true
    })
  });

  const reader = response.body?.getReader();
  if (!reader) throw new Error('No response body');

  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        try {
          const json = JSON.parse(line.slice(6));
          if (json.candidates?.[0]?.content?.parts?.[0]?.text) {
            yield json.candidates[0].content.parts[0].text;
          }
        } catch (e) {
          // Ignore parsing errors
        }
      }
    }
  }
}

// Server-Sent Events endpoint
async function handleStreamRequest(req: Request, res: Response) {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');

  try {
    for await (const chunk of streamGeminiResponse(
      'gemini-3.1-flash-lite',
      req.body.prompt
    )) {
      res.write(`data: ${JSON.stringify({ token: chunk })}\n\n`);
    }
    res.end();
  } catch (error) {
    res.write(`data: ${JSON.stringify({ error: error.message })}\n\n`);
    res.end();
  }
}
```

## 4. Robust Error Handling & Retry Logic

Production systems must handle rate limits, transient failures, and model unavailability:

```typescript
interface RetryConfig {
  maxAttempts: number;
  baseDelayMs: number;
  maxDelayMs: number;
}

class APIClient {
  private config: RetryConfig = {
    maxAttempts: 5,
    baseDelayMs: 1000,
    maxDelayMs: 32000
  };

  async callWithRetry(
    model: string,
    prompt: string
  ): Promise<string> {
    for (let attempt = 0; attempt < this.config.maxAttempts; attempt++) {
      try {
        return await this.call(model, prompt);
      } catch (error) {
        if (!this.isRetryable(error)) throw error;

        const delay = Math.min(
          this.config.baseDelayMs * Math.pow(2, attempt) +
          Math.random() * 1000,
          this.config.maxDelayMs
        );

        console.log(`Retry attempt ${attempt + 1} after ${delay}ms`);
        await new Promise(r => setTimeout(r, delay));
      }
    }
    throw new Error(`Failed after ${this.config.maxAttempts} attempts`);
  }

  private isRetryable(error: any): boolean {
    const retryableStatusCodes = [429, 500, 502, 503, 504];
    return retryableStatusCodes.includes(error.status);
  }

  private async call(model: string, prompt: string): Promise<string> {
    const response = await fetch('https://generativelanguage.googleapis.com/call', {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${process.env.GEMINI_API_KEY}` },
      body: JSON.stringify({ model, prompt })
    });

    if (!response.ok) {
      const error = new Error(`API error: ${response.status}`);
      (error as any).status = response.status;
      throw error;
    }

    const data = await response.json();
    return data.candidates[0].content.parts[0].text;
  }
}
```

**Python equivalent using exponential backoff:**

```python
import asyncio
import aiohttp
import random

class GeminiClient:
    def __init__(self, api_key: str):
        self.api_key = api_key
        self.base_url = "https://generativelanguage.googleapis.com"
        self.max_attempts = 5
        self.base_delay = 1.0

    async def call_with_retry(
        self,
        model: str,
        prompt: str
    ) -> str:
        for attempt in range(self.max_attempts):
            try:
                return await self.call(model, prompt)
            except aiohttp.ClientError as e:
                if attempt == self.max_attempts - 1:
                    raise
                delay = min(
                    self.base_delay * (2 ** attempt) + random.random(),
                    32.0
                )
                await asyncio.sleep(delay)

    async def call(self, model: str, prompt: str) -> str:
        async with aiohttp.ClientSession() as session:
            async with session.post(
                f"{self.base_url}/call",
                headers={"Authorization": f"Bearer {self.api_key}"},
                json={"model": model, "prompt": prompt}
            ) as resp:
                if resp.status != 200:
                    raise aiohttp.ClientError(f"Status {resp.status}")
                data = await resp.json()
                return data["candidates"][0]["content"]["parts"][0]["text"]

# Usage
client = GeminiClient(api_key="your-api-key")
response = await client.call_with_retry("gemini-3.1-flash-lite", "What is AI?")
```

## 5. Monitoring & Cost Tracking

Implement comprehensive logging to understand pipeline performance:

```typescript
interface PipelineMetrics {
  taskId: string;
  model: string;
  inputTokens: number;
  outputTokens: number;
  latencyMs: number;
  cost: number;
  timestamp: Date;
}

class MetricsCollector {
  private metrics: PipelineMetrics[] = [];

  recordMetric(metric: PipelineMetrics): void {
    this.metrics.push(metric);
    if (this.metrics.length % 100 === 0) {
      this.reportBatch();
    }
  }

  private reportBatch(): void {
    const batch = this.metrics.splice(0, 100);
    const totalCost = batch.reduce((sum, m) => sum + m.cost, 0);
    const avgLatency = batch.reduce((sum, m) => sum + m.latencyMs, 0) / batch.length;

    console.log(`Batch Report: ${batch.length} tasks, $${totalCost.toFixed(2)}, ${avgLatency.toFixed(0)}ms avg`);

    // Send to monitoring service
    this.sendToMonitoring({ totalCost, avgLatency, batchSize: batch.length });
  }

  private async sendToMonitoring(data: any): Promise<void> {
    // Send metrics to DataDog, CloudWatch, or similar
  }
}

// Calculate costs per model
function calculateCost(
  model: string,
  inputTokens: number,
  outputTokens: number
): number {
  const rates = {
    'gemini-3.1-flash-lite': { input: 0.075, output: 0.3 },
    'gemini-3.1-flash': { input: 0.125, output: 0.5 }
  };

  const rate = rates[model] || rates['gemini-3.1-flash'];
  return (inputTokens * rate.input + outputTokens * rate.output) / 1_000_000;
}
```

## Best Practices Summary

1. **Route requests intelligently**: 80% of tasks should use Flash-Lite.
2. **Batch non-real-time requests**: Reduce API call overhead by 90%.
3. **Stream for user-facing features**: Provide immediate feedback.
4. **Implement exponential backoff**: Handle transient failures gracefully.
5. **Monitor continuously**: Track cost, latency, and error rates.
6. **Set circuit breakers**: Fail fast when API is unavailable.

Production Gemini pipelines processing millions of requests monthly typically achieve:
- 35-40% cost reduction with Flash-Lite routing
- 90% fewer API calls with batching
- Sub-500ms latency with streaming
- 99.9% availability with proper retry logic
