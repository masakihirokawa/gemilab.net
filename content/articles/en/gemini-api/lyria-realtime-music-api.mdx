## What Is Lyria RealTime?

**Lyria RealTime** is Google DeepMind's real-time music generation model, available within the Gemini API ecosystem. Unlike conventional AI music tools that generate a fixed audio clip, Lyria RealTime produces a **continuous, theoretically infinite music stream** via WebSocket — and it lets you adjust the style, tempo, and density of that stream on the fly.

As of March 2026, the model is offered as a free experimental API, making it an attractive choice for developers building games, interactive installations, live performance tools, or any application that needs dynamic, AI-generated background music.

In this tutorial, you'll learn everything you need to go from zero to a working real-time music generation system in Python.

---

## Key Capabilities

What makes Lyria RealTime stand out from other AI music generators:

- **Continuous streaming**: Up to 10 minutes of uninterrupted music per session
- **Real-time parameter control**: Adjust BPM, density, and style while music plays; changes take effect within 2 seconds
- **High-fidelity audio**: 48 kHz stereo PCM stream
- **SynthID watermarking**: All generated audio is embedded with an imperceptible digital watermark for provenance tracking
- **Free experimental access**: No billing required while in experimental status

## Step 1: Generate Music and Save to a WAV File

Let's start with the simplest use case: generate 30 seconds of music from a text prompt and save it as a WAV file.

```python
import asyncio
import wave
import os
from google import genai
from google.genai import types

# Initialize the client with v1alpha API version
client = genai.Client(
    api_key=os.environ.get("GOOGLE_API_KEY"),
    http_options={"api_version": "v1alpha"}
)

async def generate_music_to_file(
    prompt: str,
    output_file: str = "output_music.wav",
    duration_seconds: int = 30
) -> str:
    """
    Generate music from a text prompt and save it as a WAV file.

    Args:
        prompt: Text describing the desired music style (English recommended)
        output_file: Path to save the output WAV file
        duration_seconds: How many seconds of music to generate (max 600)

    Returns:
        Path to the saved WAV file

    Expected output:
        "🎵 Generating: upbeat jazz piano with smooth bass..."
        "✅ Saved 30s of music to output_music.wav"
    """
    config = types.LiveMusicGenerationConfig(
        bpm=120,        # Tempo in beats per minute (60–200)
        density=0.7,    # Note density: 0.0 (sparse) to 1.0 (complex)
        guidance=4.0,   # Prompt adherence: 0.0 (free) to 6.0 (strict)
    )

    # Audio format: 48kHz stereo 16-bit PCM
    sample_rate = 48000
    channels = 2
    audio_data = bytearray()
    target_bytes = sample_rate * channels * 2 * duration_seconds

    print(f"🎵 Generating: {prompt[:60]}...")

    async with client.aio.live.music.connect(
        model="models/lyria-realtime-exp"
    ) as session:
        await session.set_weighted_prompts(
            prompts=[types.WeightedPrompt(text=prompt, weight=1.0)]
        )
        await session.set_music_generation_config(config=config)
        await session.play()

        async for message in session.receive():
            if message.server_content and message.server_content.audio_chunks:
                for chunk in message.server_content.audio_chunks:
                    audio_data.extend(chunk.data)
                    if len(audio_data) >= target_bytes:
                        break
            if len(audio_data) >= target_bytes:
                break

    # Save as WAV
    with wave.open(output_file, "wb") as wf:
        wf.setnchannels(channels)
        wf.setsampwidth(2)  # 16-bit
        wf.setframerate(sample_rate)
        wf.writeframes(bytes(audio_data[:target_bytes]))

    print(f"✅ Saved {duration_seconds}s of music to {output_file}")
    return output_file

# Run it
asyncio.run(generate_music_to_file(
    prompt="upbeat jazz piano with smooth bass and light drums, lounge style",
    output_file="jazz_lounge.wav",
    duration_seconds=30
))
```

## Step 2: Real-Time Parameter Control

The real power of Lyria RealTime is the ability to change music parameters during an active session. The example below walks through three distinct musical phases within a single session.

```python
import asyncio
import os
from google import genai
from google.genai import types

client = genai.Client(
    api_key=os.environ.get("GOOGLE_API_KEY"),
    http_options={"api_version": "v1alpha"}
)

async def dynamic_music_session():
    """
    Demonstrates real-time BPM and prompt changes within a single session.

    Phases: slow acoustic → mid-tempo pop → high-energy rock

    Expected output:
        "🎸 Phase 1: Slow acoustic guitar (BPM: 80)"
        "⚡ Phase 2: Tempo up! (BPM: 120)"
        "🔥 Phase 3: Energetic rock (BPM: 160)"
        "✅ Dynamic session complete"
    """
    async with client.aio.live.music.connect(
        model="models/lyria-realtime-exp"
    ) as session:

        # Phase 1: Slow acoustic fingerpicking
        print("🎸 Phase 1: Slow acoustic guitar (BPM: 80)")
        await session.set_weighted_prompts(prompts=[
            types.WeightedPrompt(
                text="slow acoustic guitar fingerpicking, gentle and calm",
                weight=1.0
            )
        ])
        await session.set_music_generation_config(
            config=types.LiveMusicGenerationConfig(bpm=80, density=0.4)
        )
        await session.play()
        await asyncio.sleep(10)

        # Phase 2: Tempo increase — requires context reset for BPM change
        print("⚡ Phase 2: Tempo up! (BPM: 120)")
        await session.set_music_generation_config(
            config=types.LiveMusicGenerationConfig(bpm=120, density=0.6)
        )
        await session.reset_context()  # Required for BPM changes
        await session.play()
        await asyncio.sleep(10)

        # Phase 3: High-energy electric rock
        print("🔥 Phase 3: Energetic rock (BPM: 160)")
        await session.set_weighted_prompts(prompts=[
            types.WeightedPrompt(
                text="energetic electric guitar rock, heavy drums, powerful riffs",
                weight=1.0
            )
        ])
        await session.set_music_generation_config(
            config=types.LiveMusicGenerationConfig(bpm=160, density=0.9)
        )
        await session.reset_context()
        await session.play()
        await asyncio.sleep(10)

        print("✅ Dynamic session complete")

asyncio.run(dynamic_music_session())
```

## Configuration Reference

### `LiveMusicGenerationConfig` Parameters

| Parameter | Type | Range | Default | Description |
|-----------|------|-------|---------|-------------|
| `bpm` | int | 60–200 | — | Beats per minute (tempo) |
| `density` | float | 0.0–1.0 | — | Note density; 0.0 = sparse, 1.0 = complex |
| `guidance` | float | 0.0–6.0 | 4.0 | How closely to follow the prompt |

### Blending Multiple Prompts

You can blend two musical styles with weighted prompts:

```python
await session.set_weighted_prompts(prompts=[
    types.WeightedPrompt(
        text="smooth jazz piano with saxophone",
        weight=0.7  # 70% jazz
    ),
    types.WeightedPrompt(
        text="cinematic orchestral strings, epic and emotional",
        weight=0.3  # 30% orchestral
    )
])
```

### When to Call `reset_context()`

A context reset is required when you change:
- **BPM** — resets the rhythmic pattern foundation
- **Musical scale or key** — resets tonal context

Prompt changes and `density` adjustments do *not* require a reset.

## Common Errors and Fixes

### Error: WebSocket Connection Refused

```
ConnectionRefusedError: WebSocket connection to generativelanguage.googleapis.com failed
```

**Fix:** Verify your API key is valid and that you're passing `http_options={"api_version": "v1alpha"}` to the client.

### Error: Session Timeout After 10 Minutes

Lyria RealTime sessions are capped at 10 minutes. For longer-running applications, implement a reconnection loop:

```python
async def long_session(prompt: str, total_minutes: int = 30):
    """Reconnects every 9.5 minutes for continuous music generation."""
    for i in range(total_minutes // 10 + 1):
        print(f"Session {i + 1} starting...")
        async with client.aio.live.music.connect(
            model="models/lyria-realtime-exp"
        ) as session:
            await session.set_weighted_prompts(
                prompts=[types.WeightedPrompt(text=prompt, weight=1.0)]
            )
            await session.play()
            await asyncio.sleep(570)  # 9.5 minutes — safely under 10-min cap
```

### Error: Parameters Not Taking Effect

Always call `play()` after `set_music_generation_config()` to apply the new settings.

## Real-World Use Case: Adaptive Game Soundtrack

Here's a practical example showing how Lyria RealTime can drive a game's adaptive soundtrack:

```python
import asyncio
import os
from enum import Enum
from google import genai
from google.genai import types

client = genai.Client(
    api_key=os.environ.get("GOOGLE_API_KEY"),
    http_options={"api_version": "v1alpha"}
)

class GameState(Enum):
    EXPLORATION = "exploration"
    COMBAT = "combat"
    BOSS = "boss"
    VICTORY = "victory"

MUSIC_CONFIGS = {
    GameState.EXPLORATION: {
        "prompt": "ambient exploration, mysterious forest, gentle flute and strings",
        "bpm": 80, "density": 0.4
    },
    GameState.COMBAT: {
        "prompt": "intense battle, fast percussion, electric guitar, urgent action",
        "bpm": 150, "density": 0.8
    },
    GameState.BOSS: {
        "prompt": "epic boss battle, massive orchestra, dramatic choir, thunderous drums",
        "bpm": 140, "density": 1.0
    },
    GameState.VICTORY: {
        "prompt": "triumphant victory fanfare, uplifting brass and strings, celebratory",
        "bpm": 120, "density": 0.6
    },
}

async def adaptive_soundtrack(state_sequence: list):
    """
    Switches background music in real time based on game state.

    Args:
        state_sequence: List of (GameState, duration_seconds) tuples

    Expected output:
        "🌲 [EXPLORATION] BPM: 80"
        "⚔️  [COMBAT] BPM: 150"
        "👹 [BOSS] BPM: 140"
        "🏆 [VICTORY] BPM: 120"
    """
    icons = {
        GameState.EXPLORATION: "🌲",
        GameState.COMBAT: "⚔️ ",
        GameState.BOSS: "👹",
        GameState.VICTORY: "🏆",
    }

    async with client.aio.live.music.connect(
        model="models/lyria-realtime-exp"
    ) as session:
        for state, duration in state_sequence:
            cfg = MUSIC_CONFIGS[state]
            print(f"{icons[state]} [{state.value.upper()}] BPM: {cfg['bpm']}")

            await session.set_weighted_prompts(prompts=[
                types.WeightedPrompt(text=cfg["prompt"], weight=1.0)
            ])
            await session.set_music_generation_config(
                config=types.LiveMusicGenerationConfig(
                    bpm=cfg["bpm"], density=cfg["density"]
                )
            )
            await session.reset_context()
            await session.play()
            await asyncio.sleep(duration)

# Demo run
asyncio.run(adaptive_soundtrack([
    (GameState.EXPLORATION, 8),
    (GameState.COMBAT, 8),
    (GameState.BOSS, 8),
    (GameState.VICTORY, 6),
]))
```

## FAQ

**Q1. Can I use Lyria RealTime for commercial projects?**

As an experimental API, usage is governed by Google's API terms of service. All generated audio carries an embedded SynthID watermark. Review the latest terms at [ai.google.dev](https://ai.google.dev) before incorporating it into commercial products.

**Q2. What's the difference between Lyria 3 and Lyria RealTime?**

Lyria 3 is a consumer-facing model available in the Gemini app that produces polished 30-second tracks (with vocals, MP3/MP4 output). Lyria RealTime is a developer API that generates a continuous PCM audio stream over WebSocket, optimized for interactive and dynamic use cases. A public Lyria 3 API is not yet available as of March 2026.

**Q3. Should I write prompts in English?**

Yes — English prompts consistently produce better results. Instead of vague single words, describe the genre, instrumentation, mood, and rhythm in detail. For example: *"smooth lo-fi hip-hop with slow piano chords, vinyl crackle, and mellow bass"* will outperform just *"lo-fi music"*.

**Q4. Can I use Lyria RealTime in a mobile app?**

Direct WebSocket connections from mobile clients are possible in theory, but a server-side architecture is recommended: run Lyria RealTime on a backend, then stream the generated audio to your mobile client. This also simplifies API key management.

**Q5. Does Lyria RealTime support vocals?**

Not currently. Lyria RealTime is designed for instrumental music generation. For vocal tracks, use Lyria 3 through the Gemini app.
