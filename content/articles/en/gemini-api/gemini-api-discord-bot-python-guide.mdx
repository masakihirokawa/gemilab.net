## Why Build a Gemini-Powered Discord Bot?

Discord has grown far beyond its gaming roots, now serving as a hub for developer communities, study groups, and business teams with over 200 million active users worldwide. Adding an AI chatbot powered by Google's Gemini API can transform how your community operates — whether that means answering technical questions around the clock, analyzing shared screenshots, or summarizing long conversations.

In this guide, you'll build a fully functional Discord bot that leverages the Gemini API to handle text conversations, analyze images using multimodal capabilities, maintain conversation context across messages, and respond to slash commands. By the end, you'll have a production-ready bot you can customize for your own server.

If you're new to the Gemini API, check out the "[Gemini API Quickstart](/en/articles/gemini-api/gemini-api-quickstart)" guide first.

## Prerequisites and Setup

### What You'll Need

- **Python 3.10+** installed on your machine
- A **Google account** to obtain a Gemini API key
- A **Discord account** with a server where you have admin permissions
- A text editor or IDE (VS Code recommended)

### Getting Your Gemini API Key

Head to [Google AI Studio](https://aistudio.google.com/), click "Get API key" in the left sidebar, and create a new key. The free tier provides generous rate limits that are more than enough for bot development.

### Creating Your Discord Bot

1. Go to the [Discord Developer Portal](https://discord.com/developers/applications)
2. Click "New Application" and give your bot a name
3. Navigate to "Bot" in the left sidebar and click "Add Bot"
4. Enable "MESSAGE CONTENT INTENT" — this is required for reading message content
5. Copy your bot token and store it securely

### Generating an Invite URL

Under "OAuth2" → "URL Generator," select the following scopes and permissions:

- **Scopes**: `bot`, `applications.commands`
- **Bot Permissions**: `Send Messages`, `Read Message History`, `Attach Files`, `Use Slash Commands`

Use the generated URL to invite the bot to your server.

## Building the Basic Bot

Let's start with a simple bot that responds to mentions using the Gemini API.

```python
# bot.py
import os
import discord
from discord.ext import commands
from google import genai
from dotenv import load_dotenv

load_dotenv()

# Initialize the Gemini client
client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))

# Configure the Discord bot
intents = discord.Intents.default()
intents.message_content = True  # Allow reading message content
bot = commands.Bot(command_prefix="!", intents=intents)

# Model to use
MODEL_NAME = "gemini-2.5-flash"

@bot.event
async def on_ready():
    print(f"✅ Logged in as {bot.user}")
    # Sync slash commands
    await bot.tree.sync()

@bot.event
async def on_message(message):
    # Ignore messages from the bot itself
    if message.author == bot.user:
        return

    # Only respond when mentioned
    if bot.user.mentioned_in(message):
        # Strip the mention to get the actual prompt
        prompt = message.content.replace(f"<@{bot.user.id}>", "").strip()

        if not prompt:
            await message.reply("Got a question? Feel free to ask!")
            return

        async with message.channel.typing():
            try:
                response = client.models.generate_content(
                    model=MODEL_NAME,
                    contents=prompt,
                )
                reply_text = response.text

                # Discord has a 2000-character message limit
                if len(reply_text) > 1900:
                    chunks = [reply_text[i:i+1900] for i in range(0, len(reply_text), 1900)]
                    for chunk in chunks:
                        await message.reply(chunk)
                else:
                    await message.reply(reply_text)

            except Exception as e:
                await message.reply(f"Something went wrong: {str(e)[:200]}")

    # Continue processing commands
    await bot.process_commands(message)

# Start the bot
bot.run(os.getenv("DISCORD_TOKEN"))
```

Run `python bot.py` in your terminal, and the bot will come online. Mention it in your Discord server with a question, and Gemini will generate a response.

## Adding Multimodal Image Analysis

One of Gemini's standout features is its multimodal capability. Let's add support for analyzing images that users attach to their messages.

```python
import io
import httpx

async def analyze_image(image_url: str, prompt: str) -> str:
    """Fetch image data from URL and analyze with Gemini."""
    async with httpx.AsyncClient() as http_client:
        img_response = await http_client.get(image_url)
        image_data = img_response.content
        content_type = img_response.headers.get("content-type", "image/png")

    # Send the image and prompt to the Gemini API
    response = client.models.generate_content(
        model=MODEL_NAME,
        contents=[
            {
                "parts": [
                    {"text": prompt if prompt else "Describe what you see in this image."},
                    {
                        "inline_data": {
                            "mime_type": content_type,
                            "data": image_data,
                        }
                    },
                ]
            }
        ],
    )
    return response.text

@bot.event
async def on_message(message):
    if message.author == bot.user:
        return

    if bot.user.mentioned_in(message):
        prompt = message.content.replace(f"<@{bot.user.id}>", "").strip()

        async with message.channel.typing():
            try:
                # Handle image attachments with multimodal processing
                if message.attachments:
                    attachment = message.attachments[0]
                    if attachment.content_type and attachment.content_type.startswith("image/"):
                        result = await analyze_image(attachment.url, prompt)
                        await message.reply(result[:1900])
                        await bot.process_commands(message)
                        return

                # Text-only messages
                if not prompt:
                    await message.reply("Got a question? Feel free to ask!")
                    await bot.process_commands(message)
                    return

                response = client.models.generate_content(
                    model=MODEL_NAME,
                    contents=prompt,
                )
                await message.reply(response.text[:1900])

            except Exception as e:
                await message.reply(f"Something went wrong: {str(e)[:200]}")

    await bot.process_commands(message)
```

Now when users attach an image and mention the bot, Gemini will analyze the image and respond. This opens up use cases like code screenshot reviews, food photo identification, diagram interpretation, and much more.

## Maintaining Conversation Context

A bot that remembers previous messages in the conversation is far more useful than one that treats each message in isolation. Here's how to implement per-channel conversation history.

```python
from collections import defaultdict, deque

# Per-channel conversation history (keeps last 10 turns)
conversation_history = defaultdict(lambda: deque(maxlen=10))

def build_chat_contents(channel_id: str, new_message: str) -> list:
    """Build a contents list that includes conversation history."""
    history = conversation_history[channel_id]
    contents = []

    for role, text in history:
        contents.append({"role": role, "parts": [{"text": text}]})

    contents.append({"role": "user", "parts": [{"text": new_message}]})
    return contents

async def chat_with_history(channel_id: str, user_message: str) -> str:
    """Query Gemini with conversation history for context."""
    contents = build_chat_contents(channel_id, user_message)

    response = client.models.generate_content(
        model=MODEL_NAME,
        contents=contents,
        config={
            "system_instruction": (
                "You are a friendly AI assistant in a Discord server. "
                "Keep your answers concise and easy to understand. "
                "Use Markdown code blocks when including code in your responses."
            ),
        },
    )

    reply = response.text

    # Add to conversation history
    conversation_history[channel_id].append(("user", user_message))
    conversation_history[channel_id].append(("model", reply))

    return reply
```

Using `deque(maxlen=10)` keeps only the last 10 turns in memory, preventing unbounded memory growth while still providing meaningful context for follow-up questions.

## Implementing Slash Commands

Slash commands offer a discoverable, user-friendly interface with Discord's built-in autocomplete support.

```python
@bot.tree.command(name="ask", description="Ask Gemini AI a question")
async def ask_command(interaction: discord.Interaction, question: str):
    """Handle questions via slash command."""
    await interaction.response.defer()  # Signal that we're processing

    try:
        reply = await chat_with_history(
            str(interaction.channel_id), question
        )
        if len(reply) > 1900:
            await interaction.followup.send(reply[:1900] + "...")
        else:
            await interaction.followup.send(reply)
    except Exception as e:
        await interaction.followup.send(f"Something went wrong: {str(e)[:200]}")

@bot.tree.command(name="clear", description="Reset conversation history")
async def clear_command(interaction: discord.Interaction):
    """Clear the channel's conversation history."""
    channel_id = str(interaction.channel_id)
    conversation_history[channel_id].clear()
    await interaction.response.send_message("Conversation history has been cleared.")

@bot.tree.command(name="summarize", description="Summarize recent messages")
async def summarize_command(interaction: discord.Interaction, count: int = 20):
    """Summarize recent messages in the channel."""
    await interaction.response.defer()

    # Fetch recent messages
    messages = []
    async for msg in interaction.channel.history(limit=count):
        if not msg.author.bot:
            messages.append(f"{msg.author.display_name}: {msg.content}")

    messages.reverse()
    conversation_text = "\n".join(messages)

    prompt = f"Please provide a concise summary of this Discord conversation:\n\n{conversation_text}"

    try:
        response = client.models.generate_content(
            model=MODEL_NAME,
            contents=prompt,
        )
        await interaction.followup.send(f"📝 **Conversation Summary**\n\n{response.text[:1800]}")
    except Exception as e:
        await interaction.followup.send(f"Something went wrong: {str(e)[:200]}")
```

With `/ask` for questions, `/clear` to reset context, and `/summarize` to recap recent discussions, your bot becomes a versatile tool for any Discord community.

## Error Handling and Rate Limiting

For production use, robust error handling and retry logic are essential.

```python
import asyncio
from google.api_core import exceptions as google_exceptions

async def safe_generate(prompt: str, max_retries: int = 3) -> str:
    """Gemini API call with exponential backoff retry."""
    for attempt in range(max_retries):
        try:
            response = client.models.generate_content(
                model=MODEL_NAME,
                contents=prompt,
            )
            return response.text

        except google_exceptions.ResourceExhausted:
            # Rate limit hit — exponential backoff
            wait_time = 2 ** attempt
            print(f"⚠️ Rate limited. Retrying in {wait_time}s...")
            await asyncio.sleep(wait_time)

        except google_exceptions.InvalidArgument as e:
            # Bad input — don't retry
            return "There seems to be an issue with the input. Please try rephrasing your question."

        except Exception as e:
            if attempt < max_retries - 1:
                await asyncio.sleep(1)
            else:
                return "Sorry, I'm unable to respond right now. Please try again later."

    return "Request failed after multiple attempts. Please try again later."
```

The `ResourceExhausted` error (HTTP 429) is the one you'll encounter most often on the free tier, so exponential backoff is critical. For more details on managing API quotas, see the "[Gemini API Rate Limiting & Quota Management Guide](/en/articles/gemini-api/gemini-api-rate-limiting-quota-management)."

## Complete Source Code

Here's the full bot with all features combined into a single file.

```python
# bot.py — Gemini API × Discord.py Complete Bot
import os
import io
import asyncio
from collections import defaultdict, deque

import discord
import httpx
from discord.ext import commands
from google import genai
from google.api_core import exceptions as google_exceptions
from dotenv import load_dotenv

load_dotenv()

# --- Initialization ---
client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))
MODEL_NAME = "gemini-2.5-flash"

intents = discord.Intents.default()
intents.message_content = True
bot = commands.Bot(command_prefix="!", intents=intents)

# Per-channel conversation history
conversation_history = defaultdict(lambda: deque(maxlen=10))

# --- Utility Functions ---
async def safe_generate(contents, system_instruction=None, retries=3):
    config = {}
    if system_instruction:
        config["system_instruction"] = system_instruction

    for attempt in range(retries):
        try:
            response = client.models.generate_content(
                model=MODEL_NAME,
                contents=contents,
                config=config if config else None,
            )
            return response.text
        except google_exceptions.ResourceExhausted:
            await asyncio.sleep(2 ** attempt)
        except Exception:
            if attempt == retries - 1:
                return "Sorry, I'm unable to generate a response right now."
            await asyncio.sleep(1)
    return "Request failed after multiple attempts."

# --- Event Handlers ---
@bot.event
async def on_ready():
    print(f"✅ Logged in as {bot.user}")
    await bot.tree.sync()

@bot.event
async def on_message(message):
    if message.author == bot.user:
        return

    if bot.user.mentioned_in(message):
        prompt = message.content.replace(f"<@{bot.user.id}>", "").strip()
        channel_id = str(message.channel.id)

        async with message.channel.typing():
            # Handle image attachments
            if message.attachments:
                att = message.attachments[0]
                if att.content_type and att.content_type.startswith("image/"):
                    async with httpx.AsyncClient() as http:
                        img = await http.get(att.url)
                    contents = [{
                        "parts": [
                            {"text": prompt or "Describe what you see in this image."},
                            {"inline_data": {
                                "mime_type": att.content_type,
                                "data": img.content,
                            }},
                        ]
                    }]
                    reply = await safe_generate(contents)
                    await message.reply(reply[:1900])
                    await bot.process_commands(message)
                    return

            if not prompt:
                await message.reply("Got a question? Feel free to ask!")
                await bot.process_commands(message)
                return

            # Response with conversation history
            history = conversation_history[channel_id]
            contents = [{"role": r, "parts": [{"text": t}]} for r, t in history]
            contents.append({"role": "user", "parts": [{"text": prompt}]})

            reply = await safe_generate(
                contents,
                system_instruction="You are a friendly AI assistant. Keep answers concise and clear.",
            )

            history.append(("user", prompt))
            history.append(("model", reply))

            if len(reply) > 1900:
                for i in range(0, len(reply), 1900):
                    await message.reply(reply[i:i+1900])
            else:
                await message.reply(reply)

    await bot.process_commands(message)

# --- Slash Commands ---
@bot.tree.command(name="ask", description="Ask Gemini AI a question")
async def ask_command(interaction: discord.Interaction, question: str):
    await interaction.response.defer()
    reply = await safe_generate(question)
    await interaction.followup.send(reply[:1900])

@bot.tree.command(name="clear", description="Reset conversation history")
async def clear_command(interaction: discord.Interaction):
    conversation_history[str(interaction.channel_id)].clear()
    await interaction.response.send_message("Conversation history has been cleared.")

@bot.tree.command(name="summarize", description="Summarize recent messages")
async def summarize_command(interaction: discord.Interaction, count: int = 20):
    await interaction.response.defer()
    messages = []
    async for msg in interaction.channel.history(limit=count):
        if not msg.author.bot:
            messages.append(f"{msg.author.display_name}: {msg.content}")
    messages.reverse()
    prompt = f"Provide a concise summary of this Discord conversation:\n\n" + "\n".join(messages)
    reply = await safe_generate(prompt)
    await interaction.followup.send(f"📝 **Conversation Summary**\n\n{reply[:1800]}")

bot.run(os.getenv("DISCORD_TOKEN"))
```

## Frequently Asked Questions

### Q1. Can I run a Discord bot on the free Gemini API tier?

Absolutely. The `gemini-2.5-flash` model on the free tier allows up to 15 requests per minute, which is plenty for small to medium-sized Discord servers. If your server grows and you start hitting rate limits regularly, you can upgrade to the pay-as-you-go plan for higher quotas.

### Q2. My bot isn't responding to messages. What should I check?

Start with these three common issues: (1) Make sure "MESSAGE CONTENT INTENT" is enabled in the Discord Developer Portal, (2) verify that your `.env` file has the correct bot token and API key, and (3) confirm the bot has the necessary permissions (Send Messages, Read Message History) in your server. If those look good, check the console for error messages.

### Q3. Does conversation history persist across bot restarts?

Not with the implementation shown above — history is stored in memory and resets when the bot restarts. For persistent history, you'd want to save conversations to a database like SQLite or Redis. A lightweight approach is to serialize the `deque` contents to a JSON file on disk.

### Q4. Can I extend the bot with Function Calling?

Yes! Combining the bot with Gemini's Function Calling feature lets you integrate external tools — weather APIs, database lookups, web searches, and more. Check out the "[Function Calling Guide](/en/articles/gemini-advanced/function-calling-guide)" for implementation details.
