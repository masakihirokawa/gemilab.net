## Why Bring Gemini to Your iOS App

The demand for AI-powered mobile apps is growing rapidly. The Gemini API offers Google's most capable generative AI models, supporting text generation, image recognition, multimodal input, and much more.

The officially recommended way to access the Gemini API from iOS apps is through the **Firebase AI Logic SDK**. The standalone `google-generative-ai-swift` package has been deprecated, and Google now recommends using the unified Firebase SDK. With Firebase AI Logic, you can call the Gemini API directly from your app without running your own server.

This guide walks you through everything from initial setup to building text generation, image analysis, and streaming features — all with practical SwiftUI code examples.

**Who this article is for:**

- Developers with basic Swift/SwiftUI experience
- Indie developers looking to add AI features to iOS apps
- Anyone who wants to leverage the Gemini API in mobile applications

## Setting Up Your Environment

### Prerequisites

To use the Firebase AI Logic SDK, you'll need the following:

- **Xcode 16.0 or later**
- A project targeting **iOS 16.0 or later**
- **Firebase SDK v12.5.0 or later**
- A **Firebase project** (create one at [Firebase Console](https://console.firebase.google.com/))
- A **Gemini API key** (enable it in the AI Logic section of Firebase Console)

### Step 1: Set Up Your Firebase Project

Create a project in the Firebase Console and register your iOS app. Download the `GoogleService-Info.plist` file and add it to your Xcode project.

### Step 2: Add the SDK via Swift Package Manager

Open your project in Xcode and navigate to **File > Add Package Dependencies**. Add the Firebase SDK repository:

```
https://github.com/firebase/firebase-ios-sdk
```

Specify version **12.5.0 or later**, and check **FirebaseAILogic** in the library selection screen.

### Step 3: Initialize Firebase in Your App

```swift
// AppDelegate.swift or App.swift
import SwiftUI
import FirebaseCore

@main
struct MyAIApp: App {
    init() {
        FirebaseApp.configure()
    }

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

### Step 4: Get a Gemini Model Instance

```swift
import FirebaseAILogic

// Use Gemini 2.5 Flash (lightweight and fast)
let ai = FirebaseAI.firebaseAI(backend: .googleAI())
let model = ai.generativeModel(modelName: "gemini-2.5-flash")
```

For `modelName`, choose the model that best fits your use case. `gemini-2.5-flash` offers a good balance of cost and speed for most applications. For tasks requiring deeper reasoning, consider `gemini-2.5-pro`.

## Basic Text Generation

Let's start with the simplest use case — generating text from a prompt.

```swift
import SwiftUI
import FirebaseAILogic

struct TextGenerationView: View {
    @State private var prompt = ""
    @State private var response = ""
    @State private var isLoading = false

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack(spacing: 16) {
            TextField("Enter a prompt", text: $prompt)
                .textFieldStyle(.roundedBorder)
                .padding(.horizontal)

            Button("Generate") {
                Task { await generateText() }
            }
            .disabled(prompt.isEmpty || isLoading)

            if isLoading {
                ProgressView()
            }

            ScrollView {
                Text(response)
                    .padding()
            }
        }
        .padding()
    }

    func generateText() async {
        isLoading = true
        defer { isLoading = false }

        do {
            let result = try await model.generateContent(prompt)
            if let text = result.text {
                response = text
            }
        } catch {
            response = "Error: \(error.localizedDescription)"
        }
    }
}

// Expected behavior:
// Enter "Explain the appeal of Japan's four seasons in 3 lines"
// Gemini returns a natural response describing the four seasons
```

Just pass a string to `model.generateContent()` to get a response from the Gemini API. Thanks to Swift's `async/await`, the asynchronous handling reads naturally and avoids callback hell.

## Implementing Streaming Responses

When generating longer text, streaming lets you display chunks as they arrive rather than waiting for the complete response. This significantly improves UX by providing immediate feedback.

```swift
import SwiftUI
import FirebaseAILogic

struct StreamingView: View {
    @State private var prompt = ""
    @State private var streamedText = ""
    @State private var isStreaming = false

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack(spacing: 16) {
            TextField("Enter a prompt", text: $prompt)
                .textFieldStyle(.roundedBorder)
                .padding(.horizontal)

            Button(isStreaming ? "Generating..." : "Stream Generate") {
                Task { await streamGenerate() }
            }
            .disabled(prompt.isEmpty || isStreaming)

            ScrollView {
                Text(streamedText)
                    .padding()
                    .frame(maxWidth: .infinity, alignment: .leading)
            }
        }
        .padding()
    }

    func streamGenerate() async {
        isStreaming = true
        streamedText = ""
        defer { isStreaming = false }

        do {
            let stream = try model.generateContentStream(prompt)
            for try await chunk in stream {
                if let text = chunk.text {
                    streamedText += text
                }
            }
        } catch {
            streamedText += "\nError: \(error.localizedDescription)"
        }
    }
}

// Expected behavior:
// After sending a prompt, text appears on screen one chunk at a time,
// creating a typing effect similar to ChatGPT or the Gemini app
```

`generateContentStream()` returns an `AsyncThrowingStream`, so you can iterate over each chunk with a `for try await` loop. UI updates happen automatically as you assign to `@State` properties.

## Image Analysis with Multimodal Input

One of Gemini's most powerful features is multimodal input. You can send both text and images together, allowing the model to analyze visual content.

```swift
import SwiftUI
import PhotosUI
import FirebaseAILogic

struct ImageAnalysisView: View {
    @State private var selectedImage: UIImage?
    @State private var photoPickerItem: PhotosPickerItem?
    @State private var analysisResult = ""
    @State private var isAnalyzing = false

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack(spacing: 16) {
            PhotosPicker(
                selection: $photoPickerItem,
                matching: .images
            ) {
                if let image = selectedImage {
                    Image(uiImage: image)
                        .resizable()
                        .scaledToFit()
                        .frame(maxHeight: 200)
                } else {
                    Label("Select an image", systemImage: "photo")
                }
            }
            .onChange(of: photoPickerItem) { _, newValue in
                Task {
                    if let data = try? await newValue?.loadTransferable(
                        type: Data.self
                    ) {
                        selectedImage = UIImage(data: data)
                    }
                }
            }

            Button("Analyze Image") {
                Task { await analyzeImage() }
            }
            .disabled(selectedImage == nil || isAnalyzing)

            if isAnalyzing { ProgressView() }

            ScrollView {
                Text(analysisResult)
                    .padding()
            }
        }
        .padding()
    }

    func analyzeImage() async {
        guard let image = selectedImage else { return }
        isAnalyzing = true
        defer { isAnalyzing = false }

        do {
            let result = try await model.generateContent(
                "Describe this image in detail.",
                image
            )
            if let text = result.text {
                analysisResult = text
            }
        } catch {
            analysisResult = "Error: \(error.localizedDescription)"
        }
    }
}

// Expected behavior:
// User selects a photo from their library and taps "Analyze Image"
// Gemini describes the image contents (subject, colors, composition, etc.)
```

You can pass both a text prompt and a `UIImage` to `generateContent()`. The Firebase AI Logic SDK handles the conversion to the appropriate format internally, so you don't need to deal with Base64 encoding manually.

## Building a Chat Feature

Implementing a chat interface with conversation context is straightforward with the Firebase AI Logic SDK.

```swift
import SwiftUI
import FirebaseAILogic

struct ChatMessage: Identifiable {
    let id = UUID()
    let role: String   // "user" or "model"
    let text: String
}

struct ChatView: View {
    @State private var messages: [ChatMessage] = []
    @State private var inputText = ""
    @State private var isResponding = false
    @State private var chat: Chat?

    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    var body: some View {
        VStack {
            ScrollView {
                LazyVStack(alignment: .leading, spacing: 12) {
                    ForEach(messages) { message in
                        HStack {
                            if message.role == "user" { Spacer() }
                            Text(message.text)
                                .padding(12)
                                .background(
                                    message.role == "user"
                                        ? Color.blue.opacity(0.2)
                                        : Color.gray.opacity(0.2)
                                )
                                .cornerRadius(12)
                            if message.role == "model" { Spacer() }
                        }
                    }
                }
                .padding()
            }

            HStack {
                TextField("Message", text: $inputText)
                    .textFieldStyle(.roundedBorder)
                Button("Send") {
                    Task { await sendMessage() }
                }
                .disabled(inputText.isEmpty || isResponding)
            }
            .padding()
        }
        .onAppear {
            chat = model.startChat()
        }
    }

    func sendMessage() async {
        let userText = inputText
        inputText = ""
        messages.append(ChatMessage(role: "user", text: userText))
        isResponding = true
        defer { isResponding = false }

        do {
            let response = try await chat?.sendMessage(userText)
            if let text = response?.text {
                messages.append(ChatMessage(role: "model", text: text))
            }
        } catch {
            messages.append(
                ChatMessage(role: "model", text: "Error: \(error.localizedDescription)")
            )
        }
    }
}

// Expected behavior:
// User and AI messages are displayed in a chat UI,
// with responses that maintain context from the conversation history
```

Call `model.startChat()` to create a `Chat` instance, then use `chat.sendMessage()` to send messages. The SDK manages conversation history internally, so you don't need to manually construct the context.

## Controlling Model Behavior with System Instructions

You'll often want to customize how Gemini responds to match your app's purpose. System Instructions let you define the model's personality and output format.

```swift
import FirebaseAILogic

// Initialize the model with System Instructions
let model = FirebaseAI.firebaseAI(backend: .googleAI())
    .generativeModel(
        modelName: "gemini-2.5-flash",
        systemInstruction: ModelContent(
            role: "system",
            parts: [
                .text("""
                You are a helpful cooking assistant.
                Follow these rules:
                - Answer questions concisely in 3 sentences or fewer
                - Always include allergy information for ingredients
                - Use language that beginners can understand
                """)
            ]
        )
    )

// Customizing generation parameters
let config = GenerationConfig(
    temperature: 0.7,      // Creativity (0.0-2.0)
    topP: 0.9,             // Nucleus sampling
    topK: 40,              // Top-K candidate selection
    maxOutputTokens: 1024  // Maximum output tokens
)

let modelWithConfig = FirebaseAI.firebaseAI(backend: .googleAI())
    .generativeModel(
        modelName: "gemini-2.5-flash",
        generationConfig: config,
        systemInstruction: ModelContent(
            role: "system",
            parts: [.text("You are a cooking assistant.")]
        )
    )
```

A lower `temperature` produces more deterministic responses, while a higher value yields more creative output. Adjust these parameters to match your app's requirements.

## Error Handling Best Practices

Robust error handling is essential for production apps. Here's how to handle the errors you might encounter with the Firebase AI Logic SDK.

```swift
import FirebaseAILogic

func generateWithErrorHandling(prompt: String) async -> String {
    let model = FirebaseAI.firebaseAI(backend: .googleAI())
        .generativeModel(modelName: "gemini-2.5-flash")

    do {
        let result = try await model.generateContent(prompt)

        // Check safety filters
        if let candidate = result.candidates.first {
            if let finishReason = candidate.finishReason,
               finishReason == .safety {
                return "Response was blocked by the safety filter."
            }
        }

        return result.text ?? "Response was empty."

    } catch let error as GenerateContentError {
        switch error {
        case .promptBlocked(let response):
            // Prompt was blocked
            return "Your prompt was flagged by safety filters."
        case .responseStoppedEarly(let reason, _):
            // Response was stopped early
            return "Response stopped early: \(reason)"
        default:
            return "Generation error: \(error.localizedDescription)"
        }
    } catch {
        // Network errors and other general errors
        return "Connection error: \(error.localizedDescription)"
    }
}
```

Mobile apps are especially susceptible to unstable network conditions, so consider implementing retry logic and offline fallbacks. ## Practical App Architecture — AI Recipe Assistant

Let's combine everything into a practical example: an AI recipe assistant that analyzes food ingredient photos and suggests recipes.

```swift
import SwiftUI
import FirebaseAILogic

class RecipeAssistantViewModel: ObservableObject {
    @Published var suggestedRecipe = ""
    @Published var isLoading = false

    private let model: GenerativeModel

    init() {
        model = FirebaseAI.firebaseAI(backend: .googleAI())
            .generativeModel(
                modelName: "gemini-2.5-flash",
                systemInstruction: ModelContent(
                    role: "system",
                    parts: [
                        .text("""
                        You are a professional cooking assistant.
                        Identify the ingredients in the image and suggest
                        one recipe using them. Format your response as:
                        - Dish name
                        - Prep time
                        - Ingredients list
                        - Step-by-step instructions (numbered)
                        - Pro tip
                        """)
                    ]
                )
            )
    }

    func analyzeAndSuggest(image: UIImage) async {
        await MainActor.run { isLoading = true }
        defer { Task { @MainActor in isLoading = false } }

        do {
            let stream = try model.generateContentStream(
                "Analyze the ingredients in this image and suggest a recipe.",
                image
            )
            var fullText = ""
            for try await chunk in stream {
                if let text = chunk.text {
                    fullText += text
                    await MainActor.run {
                        suggestedRecipe = fullText
                    }
                }
            }
        } catch {
            await MainActor.run {
                suggestedRecipe = "An error occurred. Please try again."
            }
        }
    }
}
```

This architecture uses System Instructions to control output format, and streaming to optimize UX. By combining multimodal input with streaming, users simply take a photo of their ingredients and watch recipes appear in real time.

## Frequently Asked Questions

### Q1: How do I migrate from the old `google-generative-ai-swift` package?

The old package will be fully deprecated in June 2026. To migrate, remove the `GoogleGenerativeAI` package dependency and add `FirebaseAILogic` from the Firebase SDK. Change your imports from `import GoogleGenerativeAI` to `import FirebaseAILogic`, and update model initialization to use `FirebaseAI.firebaseAI(backend: .googleAI()).generativeModel(...)`. The core API methods like `generateContent` and `generateContentStream` remain largely the same.

### Q2: Is the Firebase AI Logic SDK free to use?

The SDK itself is free. However, Gemini API usage incurs charges based on consumption. During development and testing, the free tier (15 requests per minute) is typically sufficient. For production cost management, monitor your quota and usage through [Google AI Studio](https://aistudio.google.com/).

### Q3: Can it work offline?

The Firebase AI Logic SDK calls cloud-based APIs, so a network connection is required. If offline support is needed, consider caching frequently used responses locally. While Gemini Nano's on-device inference may be integrated into the SDK in the future, it's currently limited to Pixel devices.

### Q4: Will AI features cause issues during App Store review?

Apple requires apps with AI features to clearly indicate when content is AI-generated and to implement content filtering for inappropriate material. The Firebase AI Logic SDK includes built-in safety filters that cover the basics, but you should implement additional app-specific safety measures.

### Q5: Should I use Gemini 2.5 Pro or Flash?

For general use cases (chat, text generation, basic image analysis), `gemini-2.5-flash` is the best choice — it's fast and cost-effective. For complex reasoning, long-form generation, or high-precision analysis, consider `gemini-2.5-pro`. A good architectural approach is to switch between models based on the use case within your app, balancing cost and quality.
