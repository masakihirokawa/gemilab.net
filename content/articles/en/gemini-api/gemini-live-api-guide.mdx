## Introduction

In early 2026, Google officially released the Gemini Live API as **Generally Available (GA)** on Vertex AI. This API enables developers to build real-time conversational AI that combines voice, video, and text in a single, low-latency stream. From call centers to educational apps and interactive assistants, the use cases are vast and compelling.

In this guide, we'll walk through everything you need to know — from what the Gemini Live API is, to step-by-step implementation with working code examples.

---

## What Is the Gemini Live API?

The Gemini Live API is designed for low-latency, multimodal real-time conversation. Unlike standard chat APIs, it offers:

- **Bidirectional real-time communication**: WebSocket-based streaming for audio and video data
- **Simultaneous audio and video processing**: Handle microphone and camera input at the same time
- **Low latency**: Fast response times to maintain natural conversation flow
- **Gemini 2.5 Flash Native Audio model**: Native audio understanding and synthesis built in

With GA availability on Vertex AI, you can combine the Live API with enterprise-grade SLAs and the full security ecosystem of Google Cloud Platform (GCP).

## Setup

### 1. Prepare Your Google Cloud Project

First, enable the Vertex AI API in your GCP project.

```bash
# Set your project with the gcloud CLI
gcloud config set project YOUR_PROJECT_ID

# Enable the Vertex AI API
gcloud services enable aiplatform.googleapis.com

# Set up authentication
gcloud auth application-default login
```

### 2. Install the Gen AI SDK

As of 2026, the Gemini Live API requires the **Google Gen AI SDK** (the Vertex AI SDK will drop Gemini support after June 2026, so migrating to the Gen AI SDK is strongly recommended).

```bash
pip install google-genai
```

### 3. Set Environment Variables

```bash
export GOOGLE_CLOUD_PROJECT="your-project-id"
export GOOGLE_CLOUD_LOCATION="us-central1"
```

## Basic Implementation

### Text-Based Real-Time Conversation

Let's start with the simplest case: bidirectional text conversation.

```python
import asyncio
from google import genai
from google.genai.types import LiveConnectConfig

# Initialize the client (using Vertex AI)
client = genai.Client(
    vertexai=True,
    project="your-project-id",
    location="us-central1"
)

MODEL = "gemini-2.5-flash-live-preview"

async def text_chat():
    config = LiveConnectConfig(
        response_modalities=["TEXT"],
    )

    async with client.aio.live.connect(model=MODEL, config=config) as session:
        # Send a user message
        await session.send(
            input="Tell me about the Gemini Live API.",
            end_of_turn=True
        )

        # Receive and print the response
        async for response in session.receive():
            if response.text:
                print(response.text, end="", flush=True)

asyncio.run(text_chat())
```

### Voice Input and Output

Here's how to implement microphone input and speaker output for a spoken conversation.

```python
import asyncio
import pyaudio
from google import genai
from google.genai.types import (
    LiveConnectConfig,
    SpeechConfig,
    VoiceConfig,
    PrebuiltVoiceConfig
)

# Audio configuration
FORMAT = pyaudio.paInt16
CHANNELS = 1
SEND_SAMPLE_RATE = 16000    # Microphone input sample rate
RECEIVE_SAMPLE_RATE = 24000  # Speaker output sample rate
CHUNK_SIZE = 1024

client = genai.Client(
    vertexai=True,
    project="your-project-id",
    location="us-central1"
)

MODEL = "gemini-2.5-flash-live-preview"

async def voice_chat():
    pya = pyaudio.PyAudio()

    # Open the output stream for the speaker
    output_stream = pya.open(
        format=FORMAT,
        channels=CHANNELS,
        rate=RECEIVE_SAMPLE_RATE,
        output=True
    )

    config = LiveConnectConfig(
        response_modalities=["AUDIO"],
        speech_config=SpeechConfig(
            voice_config=VoiceConfig(
                prebuilt_voice_config=PrebuiltVoiceConfig(
                    voice_name="Aoede"  # Available voices: Aoede, Charon, Fenrir, Kore, Puck, etc.
                )
            )
        )
    )

    async with client.aio.live.connect(model=MODEL, config=config) as session:
        # Open the input stream for the microphone
        input_stream = pya.open(
            format=FORMAT,
            channels=CHANNELS,
            rate=SEND_SAMPLE_RATE,
            input=True,
            frames_per_buffer=CHUNK_SIZE
        )

        async def send_audio():
            """Capture and stream microphone audio"""
            while True:
                data = await asyncio.get_event_loop().run_in_executor(
                    None, input_stream.read, CHUNK_SIZE
                )
                await session.send(input={"data": data, "mime_type": "audio/pcm"})

        async def receive_audio():
            """Receive and play back audio responses"""
            async for response in session.receive():
                if response.data:
                    output_stream.write(response.data)

        # Run send and receive concurrently
        await asyncio.gather(send_audio(), receive_audio())

asyncio.run(voice_chat())
```

## Advanced Use: Adding Video Input

You can also process live video from a camera alongside audio.

```python
import asyncio
import cv2
import base64
from google import genai
from google.genai.types import LiveConnectConfig

client = genai.Client(
    vertexai=True,
    project="your-project-id",
    location="us-central1"
)

MODEL = "gemini-2.5-flash-live-preview"

async def video_chat():
    config = LiveConnectConfig(
        response_modalities=["TEXT"],
        system_instruction="You are an AI assistant that analyzes video. Describe what you see clearly and concisely."
    )

    cap = cv2.VideoCapture(0)  # Open the default camera

    async with client.aio.live.connect(model=MODEL, config=config) as session:
        async def send_video_frames():
            """Send camera frames at regular intervals"""
            while True:
                ret, frame = cap.read()
                if not ret:
                    break

                # Encode the frame as JPEG
                _, buffer = cv2.imencode(".jpg", frame)
                image_data = base64.b64encode(buffer).decode("utf-8")

                # Send the video frame
                await session.send(
                    input={
                        "mime_type": "image/jpeg",
                        "data": image_data
                    }
                )
                await asyncio.sleep(0.5)  # Send a frame every 0.5 seconds

        async def receive_responses():
            """Receive and print text responses"""
            async for response in session.receive():
                if response.text:
                    print(f"AI: {response.text}")

        await asyncio.gather(send_video_frames(), receive_responses())

    cap.release()

asyncio.run(video_chat())
```

## Customizing with System Instructions

You can fine-tune the AI's behavior using `system_instruction` in your config.

```python
config = LiveConnectConfig(
    response_modalities=["AUDIO"],
    system_instruction="""
    You are a friendly customer support AI for a software product.
    - Always be polite and professional.
    - Provide clear, step-by-step instructions for technical questions.
    - If you don't know the answer, acknowledge it honestly and offer to transfer to a human agent.
    """,
    speech_config=SpeechConfig(
        voice_config=VoiceConfig(
            prebuilt_voice_config=PrebuiltVoiceConfig(voice_name="Kore")
        )
    )
)
```

## FAQ

**Q: How is the Gemini Live API priced?**

A: Pricing on Vertex AI is based on the number of tokens processed for audio input and output. Billing officially began on January 28, 2026. See the [Vertex AI Pricing page](https://cloud.google.com/vertex-ai/generative-ai/pricing) for current rates.

**Q: What languages are supported?**

A: The Live API supports a wide range of languages, including robust support for English and Japanese in both speech recognition and speech synthesis.

**Q: Is there a maximum session duration?**

A: Yes, the maximum session duration per connection is 15 minutes. For longer interactions, implement a reconnection strategy in your application.

**Q: Can I still use the Vertex AI SDK?**

A: Vertex AI SDK releases after June 2026 will no longer support Gemini. New Gemini features are only available in the **Google Gen AI SDK**. Migrating is straightforward — the main change is switching from the `google-cloud-aiplatform` package to `google-genai`.

**Q: What kind of latency can I expect?**

A: In general, you can expect 100ms–300ms of latency. This can vary based on your network conditions and the GCP region you're connecting to.
