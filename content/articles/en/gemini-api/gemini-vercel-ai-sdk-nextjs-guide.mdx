## Introduction — Why Vercel AI SDK + Gemini API?

Building AI-powered applications involves a surprising amount of plumbing: streaming responses, managing conversation history, parsing tool calls, handling errors gracefully. Implementing all of this from scratch is a significant time sink that takes you away from building the features that actually matter.

The **Vercel AI SDK** is an open-source TypeScript library that solves this problem with a unified, provider-agnostic API. When paired with the Gemini API through the `@ai-sdk/google` provider, you get a development experience that's both powerful and remarkably concise.

In this guide, we'll walk through building a fully functional streaming AI chat application using Next.js 15 (App Router), the Vercel AI SDK, and Gemini API. You'll go from zero to a working app with streaming, function calling, structured output, and multimodal capabilities — all with production-ready patterns.

### What You'll Learn

- Core concepts and architecture of the Vercel AI SDK
- Setting up the `@ai-sdk/google` provider
- Real-time streaming with `streamText` and `useChat`
- Integrating Function Calling (tool use)
- Type-safe Structured Output with Zod schemas
- Production best practices for error handling and model selection

## Understanding the Vercel AI SDK Architecture

The Vercel AI SDK is organized into three distinct layers, each handling a specific concern.

### AI SDK Core

The server-side foundation. It provides functions like `generateText` (single response), `streamText` (streaming response), and `generateObject` (structured output) that handle all the heavy lifting of communicating with AI providers.

### AI SDK UI

React hooks designed for building AI interfaces. `useChat` manages chat state, streaming, and message history out of the box. `useCompletion` handles simpler text completion use cases.

### AI SDK Providers

Abstraction layers that normalize the differences between AI providers. For Gemini, you use `@ai-sdk/google`, which translates the unified SDK interface into Gemini API calls behind the scenes.

```
┌─────────────────────────────────┐
│  Frontend (React)                │
│  useChat / useCompletion         │
│  ← AI SDK UI                    │
├─────────────────────────────────┤
│  API Route (Next.js)             │
│  streamText / generateText       │
│  ← AI SDK Core                  │
├─────────────────────────────────┤
│  @ai-sdk/google                  │
│  ← AI SDK Provider              │
│  → Gemini API                    │
└─────────────────────────────────┘
```

## Step 1 — Simple Text Generation

Let's start with the basics: a single request-response interaction with Gemini.

```typescript
// app/api/generate/route.ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

export async function POST(req: Request) {
  const { prompt } = await req.json();

  // Call Gemini 3 Pro with generateText
  const { text } = await generateText({
    model: google('gemini-3-pro'),
    prompt,
  });

  return Response.json({ text });
}
```

That's it — authentication, request formatting, and response parsing are all handled by the SDK. The `google()` function takes a model name and returns a provider instance that the core functions know how to work with.

## Step 2 — Building a Streaming Chat Interface

This is where the Vercel AI SDK truly shines. Building a streaming chat experience that would normally require hundreds of lines of WebSocket or SSE code becomes straightforward.

### Server Side (API Route)

```typescript
// app/api/chat/route.ts
import { google } from '@ai-sdk/google';
import { streamText } from 'ai';

export const maxDuration = 30; // Vercel timeout setting

export async function POST(req: Request) {
  const { messages } = await req.json();

  // Generate a streaming response
  const result = streamText({
    model: google('gemini-3-pro'),
    system: 'You are a helpful and knowledgeable AI assistant.',
    messages,
  });

  // Return an AI SDK-compatible data stream
  return result.toDataStreamResponse();
}
```

### Client Side (Chat UI)

```tsx
// app/chat/page.tsx
'use client';

import { useChat } from '@ai-sdk/react';

export default function ChatPage() {
  const { messages, input, handleInputChange, handleSubmit, isLoading } =
    useChat({
      api: '/api/chat',
    });

  return (
    <div className="max-w-2xl mx-auto p-4">
      <h1 className="text-2xl font-bold mb-4">Gemini AI Chat</h1>

      {/* Message list */}
      <div className="space-y-4 mb-4">
        {messages.map((m) => (
          <div
            key={m.id}
            className={`p-3 rounded-lg ${
              m.role === 'user'
                ? 'bg-blue-100 ml-12'
                : 'bg-gray-100 mr-12'
            }`}
          >
            <p className="text-sm font-semibold mb-1">
              {m.role === 'user' ? 'You' : 'Gemini'}
            </p>
            <p className="whitespace-pre-wrap">{m.content}</p>
          </div>
        ))}
      </div>

      {/* Input form */}
      <form onSubmit={handleSubmit} className="flex gap-2">
        <input
          value={input}
          onChange={handleInputChange}
          placeholder="Type a message..."
          className="flex-1 p-2 border rounded-lg"
          disabled={isLoading}
        />
        <button
          type="submit"
          disabled={isLoading}
          className="px-4 py-2 bg-blue-500 text-white rounded-lg disabled:opacity-50"
        >
          {isLoading ? 'Thinking...' : 'Send'}
        </button>
      </form>
    </div>
  );
}
```

The `useChat` hook handles everything: receiving and parsing the stream, updating message state, managing conversation history for multi-turn dialogues, and exposing loading states. You just wire it up to your UI.

## Step 3 — Function Calling (Tool Use)

One of Gemini's most powerful features — Function Calling — integrates cleanly with the Vercel AI SDK's `tool()` API.

```typescript
// app/api/chat-with-tools/route.ts
import { google } from '@ai-sdk/google';
import { streamText, tool } from 'ai';
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: google('gemini-3-pro'),
    system: 'You are a helpful assistant with access to weather and currency tools.',
    messages,
    tools: {
      // Weather lookup tool
      getWeather: tool({
        description: 'Get the current weather for a given city',
        parameters: z.object({
          city: z.string().describe('City name (e.g., Tokyo, New York)'),
          unit: z.enum(['celsius', 'fahrenheit']).default('celsius'),
        }),
        execute: async ({ city, unit }) => {
          // Replace with a real API call in production
          const weatherData = {
            city,
            temperature: unit === 'celsius' ? 22 : 72,
            unit,
            condition: 'Sunny',
            humidity: 45,
          };
          return weatherData;
        },
      }),
      // Exchange rate tool
      getExchangeRate: tool({
        description: 'Get the exchange rate between two currencies',
        parameters: z.object({
          from: z.string().describe('Source currency code (e.g., USD)'),
          to: z.string().describe('Target currency code (e.g., JPY)'),
        }),
        execute: async ({ from, to }) => {
          // Replace with a real API call in production
          return { from, to, rate: 148.5, timestamp: new Date().toISOString() };
        },
      }),
    },
    maxSteps: 5, // Allow up to 5 tool call rounds
  });

  return result.toDataStreamResponse();
}
```

Define tools with `tool()`, specify parameter schemas using Zod, and implement the `execute` function. The `maxSteps` option enables multi-step reasoning, where Gemini can call a tool, process the result, and decide whether to call another tool or return a final answer.

## Step 4 — Structured Output

When you need type-safe, schema-conformant responses from Gemini, `generateObject` is your friend.

```typescript
// app/api/analyze/route.ts
import { google } from '@ai-sdk/google';
import { generateObject } from 'ai';
import { z } from 'zod';

// Define the response schema
const SentimentSchema = z.object({
  sentiment: z.enum(['positive', 'negative', 'neutral']),
  confidence: z.number().min(0).max(1),
  keywords: z.array(z.string()).describe('Key terms from the text'),
  summary: z.string().describe('A brief summary (under 100 characters)'),
});

export async function POST(req: Request) {
  const { text } = await req.json();

  // Get a structured, typed result
  const { object } = await generateObject({
    model: google('gemini-3-pro'),
    schema: SentimentSchema,
    prompt: `Perform sentiment analysis on the following text:\n\n${text}`,
  });

  // 'object' is fully typed as z.infer<typeof SentimentSchema>
  return Response.json(object);
}
```

Under the hood, this leverages Gemini's native structured output capabilities to guarantee schema compliance. TypeScript inference flows all the way through, so your frontend code gets full type safety.

## Step 5 — Multimodal Input

Gemini's multimodal strength is easy to tap into with the SDK's content array format.

```typescript
// app/api/vision/route.ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

export async function POST(req: Request) {
  const formData = await req.formData();
  const image = formData.get('image') as File;
  const question = formData.get('question') as string;

  // Convert image to base64
  const bytes = await image.arrayBuffer();
  const base64 = Buffer.from(bytes).toString('base64');

  const { text } = await generateText({
    model: google('gemini-3-pro'),
    messages: [
      {
        role: 'user',
        content: [
          {
            type: 'image',
            image: `data:${image.type};base64,${base64}`,
          },
          {
            type: 'text',
            text: question || 'Describe what you see in this image.',
          },
        ],
      },
    ],
  });

  return Response.json({ text });
}
```

Simply include image and text content in the message's `content` array. The SDK handles the encoding and formatting for Gemini's multimodal API.

## Production Best Practices

### Error Handling

```typescript
// lib/error-handler.ts
import { APICallError } from 'ai';

export function handleAIError(error: unknown): Response {
  if (error instanceof APICallError) {
    // Rate limit errors
    if (error.statusCode === 429) {
      return Response.json(
        { error: 'Too many requests. Please wait a moment and try again.' },
        { status: 429 }
      );
    }
    // Authentication errors
    if (error.statusCode === 401) {
      return Response.json(
        { error: 'Invalid API key.' },
        { status: 401 }
      );
    }
  }
  console.error('AI API Error:', error);
  return Response.json(
    { error: 'An unexpected error occurred.' },
    { status: 500 }
  );
}
```

### Dynamic Model Selection

```typescript
// lib/model-config.ts
import { google } from '@ai-sdk/google';

export function getModel(useCase: 'chat' | 'analysis' | 'fast') {
  switch (useCase) {
    case 'chat':
      return google('gemini-3-pro');       // High-quality conversation
    case 'analysis':
      return google('gemini-3-pro', {
        structuredOutputs: true,            // Optimize for structured output
      });
    case 'fast':
      return google('gemini-3-flash');      // Low-latency responses
    default:
      return google('gemini-3-flash');
  }
}
```

## Frequently Asked Questions

### Q1: Is the Vercel AI SDK free to use?

Yes, the Vercel AI SDK is fully open source under the Apache 2.0 license. The only costs come from the Gemini API itself — Google offers a generous free tier, and paid usage is billed based on token consumption. Check the [Google AI Studio pricing page](https://ai.google.dev/pricing) for current rates.

### Q2: Can I deploy to platforms other than Vercel?

Absolutely. Despite the name, the Vercel AI SDK works anywhere Next.js runs. You can deploy to Cloudflare Workers, AWS Lambda, a VPS, or any other platform that supports Node.js. The SDK has no Vercel-specific dependencies.

### Q3: Should I use Gemini 3 Pro or Gemini 3 Flash?

It depends on your use case. Gemini 3 Pro excels at complex reasoning, nuanced conversation, and tasks that demand higher quality output. Gemini 3 Flash is optimized for speed and cost efficiency, making it ideal for high-throughput applications or latency-sensitive features. Switching between them is as simple as changing the model string — see our [Gemini 3 Pro Complete Guide](/en/articles/gemini-api/gemini-3-pro-guide) for a detailed comparison.

### Q4: What's the difference between Function Calling and Structured Output?

Function Calling enables the model to invoke external tools and APIs — think of it as giving Gemini the ability to take actions. Structured Output constrains the model's response to match a specific JSON schema — it's about getting predictable, typed data back. Use Function Calling when you need the model to *do* something; use Structured Output when you need it to *return* data in a specific format. For advanced tool use patterns, check out our [Gemini API Tool Use Advanced Guide](/en/articles/gemini-api/gemini-api-tool-use-advanced).
