One of the most practical additions in the Gemini 3 family is the `thinking_level` parameter — a developer-facing control that determines how deeply the model reasons before producing a response. By tuning this single parameter, you can strike the right balance between **response speed, API cost, and answer quality** depending on what your application needs.

This guide covers how `thinking_level` works, what each level does, when to use which, and includes working Python code you can run today.

## What Is thinking_level?

Before Gemini 3 outputs a response, it runs an internal reasoning process sometimes called "thinking." This pre-output phase is where the model works through complex logic, multi-step calculations, or nuanced analysis. The `thinking_level` parameter lets you set how much compute the model dedicates to this phase.

Prior to Gemini 3, there was no direct way for developers to control this reasoning depth. Now you can.

> **Important**: `thinking_level` is a *relative guideline*, not a hard token guarantee. The actual depth of reasoning adapts to the complexity of your prompt — simpler requests use fewer resources even at `"high"`, and vice versa.

## Available Levels

| Level | Description | Best For |
|-------|-------------|----------|
| `"none"` | Skips the thinking phase entirely. Fastest and cheapest. | Classification, simple Q&A |
| `"low"` | Light reasoning only | Summarization, translation, basic inference |
| `"medium"` | Balanced (default behavior) | General dev tasks, code generation |
| `"high"` | Deep, thorough reasoning. Highest accuracy. | Math, logic, complex agents |

### Basic Request with thinking_level

```python
import google.generativeai as genai

# Configure with your API key
genai.configure(api_key="YOUR_API_KEY")

# Use Gemini 3 Pro
model = genai.GenerativeModel("gemini-3-pro-preview")

# Ask a multi-step math problem with deep reasoning enabled
response = model.generate_content(
    "Find the next number in the sequence and explain the pattern: 1, 1, 2, 3, 5, 8, 13, ?",
    generation_config=genai.GenerationConfig(
        thinking_level="high",  # Enable deep reasoning
        temperature=0.2,
    )
)

print(response.text)
# Expected output:
# The next number is 21.
# This is the Fibonacci sequence, where each term equals the sum of the two preceding terms.
# Rule: a(n) = a(n-1) + a(n-2)
# Verification: 1+1=2, 1+2=3, 2+3=5, 3+5=8, 5+8=13, 8+13=21 ✓
```

### Skip Thinking for Fast, Simple Tasks

## Comparing Levels in Practice

Here's a benchmark showing how `thinking_level` affects latency and output quality on a multi-step reasoning task:

```python
import google.generativeai as genai
import time

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro-preview")

PROMPT = """
Stock A rose 15% yesterday compared to the day before,
and fell 8% the day before that.
If the stock was $1,000 three days ago, what is today's price?
Show your work.
"""

for level in ["none", "low", "medium", "high"]:
    start = time.time()
    response = model.generate_content(
        PROMPT,
        generation_config=genai.GenerationConfig(
            thinking_level=level,
            temperature=0.1,
        )
    )
    elapsed = time.time() - start
    print(f"[{level:6s}] {elapsed:.2f}s | {response.text[:80].strip()}...")
    print()

# Sample output (times vary by environment):
# [none  ] 0.83s | Day -3: $1,000 → Day -2: $920 (×0.92) → Today: $1,058...
# [low   ] 1.41s | Step 1: $1,000 × 0.92 = $920. Step 2: $920 × 1.15 = $1,058...
# [medium] 2.05s | Three days ago: $1,000. Two days ago: $1,000 × 0.92 = $920...
# [high  ] 3.80s | Let P₀ = $1,000. Day -2: P₁ = P₀ × (1-0.08) = $920.00...
```

|-----------|------------------|--------|
| Text classification / labeling | `none` | Pattern matching, no reasoning needed |
| Summarization / translation | `low` | Lightweight context understanding |

| Math / logic problems | `high` | Multi-step reasoning required |
| Complex agentic workflows | `high` | Correctness is paramount |

## Using Thought Signatures for Multi-Turn Reasoning

Gemini 3 introduced **Thought Signatures** alongside `thinking_level`. A Thought Signature is an encrypted representation of the model's internal reasoning chain. By passing the signature from one turn to the next, you ensure the model maintains consistent, coherent reasoning across a long conversation — critical for agentic workflows.

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro-preview")

# Turn 1: Establish reasoning context
response1 = model.generate_content(
    "Explain the attention mechanism in large language models.",
    generation_config=genai.GenerationConfig(
        thinking_level="high",
    )
)

# Retrieve the Thought Signature if available
thought_signature = getattr(
    response1.candidates[0], "thinking_signature", None
)

# Turn 2: Continue with the same reasoning thread
chat = model.start_chat(history=[])
response2 = chat.send_message(
    "Now explain how this relates to the Transformer architecture in depth.",
    generation_config=genai.GenerationConfig(
        thinking_level="high",
        thought_signature=thought_signature,  # Carry forward the reasoning chain
    )
)

print(response2.text)
```

## Streaming with thinking_level

`thinking_level` works seamlessly with streaming responses:

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro-preview")

response = model.generate_content(
    "Explain the Singleton, Factory, and Observer design patterns in Python with code examples.",
    generation_config=genai.GenerationConfig(
        thinking_level="medium",
        temperature=0.3,
    ),
    stream=True,
)

for chunk in response:
    if chunk.text:
        print(chunk.text, end="", flush=True)

# Output: Each design pattern explanation streams in progressively
```

## Dynamic Level Routing to Optimize Cost

A practical pattern for production apps is to route requests to different `thinking_level` values based on the complexity of each task:

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-flash-preview")

def pick_thinking_level(task: str) -> str:
    """Simple heuristic to select thinking_level based on task complexity."""
    complex_signals = ["prove", "optimize", "architecture", "mathematical", "multi-step", "analyze"]
    simple_signals  = ["classify", "translate", "summarize", "list", "yes or no"]

    task_lower = task.lower()
    if any(s in task_lower for s in complex_signals):
        return "high"
    elif any(s in task_lower for s in simple_signals):
        return "none"
    return "medium"

# Route dynamically
tasks = [
    "Translate this to Spanish: 'Good morning, how are you?'",
    "Prove that there are infinitely many prime numbers using Euclid's argument.",
    "Write a Python function to check if a string is a palindrome.",
]

for task in tasks:
    level = pick_thinking_level(task)
    response = model.generate_content(
        task,
        generation_config=genai.GenerationConfig(thinking_level=level)
    )
    print(f"[{level:6s}] {task[:45]}...")
    print(f"         → {response.text[:70].strip()}\n")
```

## Common Errors and Fixes

### `thinking_level` is not recognized

```
google.api_core.exceptions.InvalidArgument: 400 Unknown field: thinking_level
```

**Cause**: You're using a Gemini 2.x model, which doesn't support this parameter.

**Fix**: Switch to a Gemini 3 model:

```python
# ❌ Not supported on Gemini 2.x
model = genai.GenerativeModel("gemini-2.5-pro")

# ✅ Supported on Gemini 3 and later
model = genai.GenerativeModel("gemini-3-pro-preview")
```

### Higher-than-expected latency

**Cause**: `thinking_level="high"` significantly increases processing time.

**Fix**: For latency-sensitive use cases, use `"medium"` or lower, and consider async calls to prevent blocking:

```python
import asyncio
import google.generativeai as genai

async def generate_async(prompt: str):
    model = genai.GenerativeModel("gemini-3-pro-preview")
    response = await model.generate_content_async(
        prompt,
        generation_config=genai.GenerationConfig(thinking_level="high")
    )
    return response.text

# Run without blocking your main thread
result = asyncio.run(generate_async("Explain quantum entanglement in detail."))
print(result)
```

## FAQ

**Q: Does setting `thinking_level="none"` significantly degrade accuracy?**

A: It depends on the task. For classification, translation, and summarization, `"none"` typically delivers accuracy on par with higher levels. For multi-step math or complex logical reasoning, accuracy drops noticeably — use `"high"` for those.

**Q: Can I use `thinking_level` with Gemini 3 Flash?**

A: Yes, `gemini-3-flash-preview` supports the parameter. However, Flash is architected for speed and efficiency, so even at `"high"`, it won't match the reasoning depth of Gemini 3 Pro. Choose your model and level together based on your accuracy vs. cost requirements.

**Q: Can I combine `thinking_level` and `temperature`?**

A: Absolutely. They control different things: `thinking_level` governs *reasoning depth* (pre-output thinking), while `temperature` governs *output randomness*. For precision-critical tasks, `thinking_level="high"` + `temperature=0.1` is a strong combination.

**Q: Does `thinking_level` affect billing?**

A: There's no separate line item for `thinking_level`, but higher levels produce more internal reasoning tokens, which count toward your usage. Routing simpler tasks to `"none"` or `"low"` can meaningfully reduce costs at scale.

**Q: Does thinking_level work with async generation?**

A: Yes. Pass `generation_config` with `thinking_level` to `generate_content_async()` the same way you would for synchronous calls. Async is especially recommended at `"high"` to avoid blocking your application.

## Wrapping Up

`thinking_level` is a small parameter with a big impact. Use it strategically:

- **Speed and cost matter** → `"none"` or `"low"` for simple tasks
- **Everyday dev work** → `"medium"` as a solid default
- **Accuracy is non-negotiable** → `"high"` for complex reasoning and agentic pipelines

Pair it with dynamic routing in production and you'll get the best of both worlds: low latency where you need it and deep reasoning where it counts.
