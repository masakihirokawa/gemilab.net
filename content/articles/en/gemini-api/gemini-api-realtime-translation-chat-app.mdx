## Architecture Overview

The application consists of three layers:

**Backend (FastAPI + WebSocket)**: The core layer that receives messages from clients, runs translations through the Gemini API, and streams results back in real time.

**Gemini API Integration Layer**: Handles prompt optimization, language detection, and streaming response processing for translation tasks.

**Frontend (HTML + JavaScript)**: A chat UI that maintains a persistent WebSocket connection to the server and renders translation results as they arrive.

This design ensures that the moment a user sends a message, translated text begins appearing character by character — creating a natural, conversational experience.

## Building the Translation Engine

Let's start with the heart of the app: the translation engine. We'll use Gemini API's System Instructions to produce consistent, high-quality translations.

```python
# translator.py
import os
from google import genai

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

TRANSLATION_SYSTEM_PROMPT = """You are a professional real-time translator.
Your task is to translate the user's message into the specified target language.

Rules:
1. Translate naturally — do not produce literal, word-for-word translations.
2. Preserve the original tone and nuance (formal, casual, technical, etc.).
3. If the source language is the same as the target language, return the text unchanged.
4. Output ONLY the translated text — no explanations or annotations.
5. For technical terms, keep them in their original form if that is the common usage in the target language.
"""

LANGUAGE_DETECT_PROMPT = """Detect the language of the following text and return ONLY the ISO 639-1 language code (e.g., 'en', 'ja', 'fr', 'de', 'zh', 'ko', 'es').
Do not include any explanation. Return only the two-letter code.

Text: {text}"""


def detect_language(text: str) -> str:
    """Detect the language of the input text."""
    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=LANGUAGE_DETECT_PROMPT.format(text=text),
    )
    return response.text.strip().lower()[:2]


async def translate_stream(text: str, target_lang: str):
    """Translate text using Gemini API with streaming response.

    Yields translated text chunks as they are generated.
    """
    # Translate the text using Gemini API with streaming
    # Expected output: translated text in the target language
    prompt = f"Translate the following text into {target_lang}:\n\n{text}"

    async for chunk in client.aio.models.generate_content_stream(
        model="gemini-2.5-flash",
        contents=prompt,
        config=genai.types.GenerateContentConfig(
            system_instruction=TRANSLATION_SYSTEM_PROMPT,
            temperature=0.3,
            max_output_tokens=2048,
        ),
    ):
        if chunk.text:
            yield chunk.text
```

A key detail here is the `temperature` setting of `0.3`. For translation tasks, accuracy matters more than creativity, so we keep the output variance low.

## Building the FastAPI + WebSocket Server

Next, let's build the real-time communication server:

```python
# main.py
import json
import asyncio
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.staticfiles import StaticFiles
from fastapi.responses import HTMLResponse
from translator import detect_language, translate_stream

app = FastAPI(title="Gemini Translate Chat")

# Store connected clients
# Key: WebSocket, Value: dict with user info
connected_clients: dict[WebSocket, dict] = {}

SUPPORTED_LANGUAGES = {
    "ja": "Japanese",
    "en": "English",
    "zh": "Chinese",
    "ko": "Korean",
    "fr": "French",
    "de": "German",
    "es": "Spanish",
    "pt": "Portuguese",
    "it": "Italian",
    "ru": "Russian",
}


@app.websocket("/ws/{username}/{target_lang}")
async def websocket_endpoint(
    websocket: WebSocket, username: str, target_lang: str
):
    await websocket.accept()
    connected_clients[websocket] = {
        "username": username,
        "target_lang": target_lang,
    }

    try:
        while True:
            data = await websocket.receive_text()
            message = json.loads(data)
            original_text = message.get("text", "")

            if not original_text.strip():
                continue

            # Detect source language
            source_lang = detect_language(original_text)

            # Broadcast original message to all clients
            await broadcast(
                {
                    "type": "message",
                    "username": username,
                    "text": original_text,
                    "source_lang": source_lang,
                }
            )

            # Translate for each connected client with a different language
            for client_ws, client_info in connected_clients.items():
                client_target = client_info["target_lang"]
                if client_target == source_lang:
                    continue  # No translation needed

                # Stream translation to each client
                asyncio.create_task(
                    stream_translation_to_client(
                        client_ws,
                        original_text,
                        client_target,
                        username,
                    )
                )

    except WebSocketDisconnect:
        del connected_clients[websocket]


async def stream_translation_to_client(
    client_ws: WebSocket,
    text: str,
    target_lang: str,
    username: str,
):
    """Stream translated chunks to a specific client."""
    try:
        # Send translation start signal
        await client_ws.send_text(
            json.dumps(
                {
                    "type": "translation_start",
                    "username": username,
                    "target_lang": target_lang,
                }
            )
        )

        # Stream translated chunks
        async for chunk in translate_stream(text, target_lang):
            await client_ws.send_text(
                json.dumps(
                    {
                        "type": "translation_chunk",
                        "username": username,
                        "chunk": chunk,
                    }
                )
            )

        # Send translation end signal
        await client_ws.send_text(
            json.dumps(
                {
                    "type": "translation_end",
                    "username": username,
                }
            )
        )
    except Exception:
        pass  # Client may have disconnected


async def broadcast(message: dict):
    """Broadcast a message to all connected clients."""
    disconnected = []
    for client_ws in connected_clients:
        try:
            await client_ws.send_text(json.dumps(message))
        except Exception:
            disconnected.append(client_ws)
    for ws in disconnected:
        del connected_clients[ws]


# Serve the frontend
app.mount("/static", StaticFiles(directory="static"), name="static")


@app.get("/")
async def get_index():
    with open("static/index.html", "r") as f:
        return HTMLResponse(content=f.read())
```

The server lets each client specify their preferred display language on connection. Whenever a message comes in, the Gemini API translates it in real time and streams the result to each client who needs it.

```html
  <script>
    let ws;
    const translations = {}; // Track streaming translations by username

    function connect() {
      const username = document.getElementById('username').value.trim();
      const targetLang = document.getElementById('targetLang').value;
      if (!username) return alert('Please enter your name');

      document.getElementById('setup').style.display = 'none';
      const chatScreen = document.getElementById('chatScreen');
      chatScreen.style.display = 'flex';

      ws = new WebSocket(`ws://${location.host}/ws/${username}/${targetLang}`);

      ws.onmessage = (event) => {
        const data = JSON.parse(event.data);
        handleMessage(data);
      };
    }

    function handleMessage(data) {
      const chatArea = document.getElementById('chatArea');

      if (data.type === 'message') {
        const div = document.createElement('div');
        div.className = 'message';
        div.id = `msg-${Date.now()}`;
        div.innerHTML = `
          <div class="username">${data.username} <span style="color:#94a3b8;font-size:0.75rem;">(${data.source_lang})</span></div>
          <div class="original">${data.text}</div>
          <div class="translation streaming" id="trans-${data.username}-latest"></div>
        `;
        chatArea.appendChild(div);
        chatArea.scrollTop = chatArea.scrollHeight;
      }

      if (data.type === 'translation_start') {
        const el = document.getElementById(`trans-${data.username}-latest`);
        if (el) el.textContent = '';
      }

      if (data.type === 'translation_chunk') {
        const el = document.getElementById(`trans-${data.username}-latest`);
        if (el) {
          el.textContent += data.chunk;
          document.getElementById('chatArea').scrollTop =
            document.getElementById('chatArea').scrollHeight;
        }
      }

      if (data.type === 'translation_end') {
        const el = document.getElementById(`trans-${data.username}-latest`);
        if (el) {
          el.classList.remove('streaming');
          el.id = '';  // Remove the ID so next message gets a fresh target
        }
      }
    }

    function sendMessage() {
      const input = document.getElementById('messageInput');
      const text = input.value.trim();
      if (!text || !ws) return;
      ws.send(JSON.stringify({ text }));
      input.value = '';
    }
  </script>
</body>
</html>
```

## Performance Optimization Tips

For production deployments, consider these optimizations:

**Translation Caching**: Cache translation results for identical text-language pairs using Redis or a similar store to reduce API calls.

```python
# cache.py — simple in-memory cache example
from functools import lru_cache

@lru_cache(maxsize=1000)
def get_cached_translation(text: str, target_lang: str) -> str | None:
    """Return cached translation if available, otherwise None."""
    return None  # Replace with Redis lookup in production
```

**Context Caching**: Use Gemini API's [Context Caching](/articles/gemini-api/context-caching-guide) feature to reuse System Instructions tokens, reducing costs by up to 90%.

**Batch Translation**: Group translation requests from multiple clients within a short time window to minimize API call overhead.

## Frequently Asked Questions

### How many languages does this support?

The Gemini API supports over 100 languages. While this sample includes 7 languages in the dropdown, you can easily expand the list by adding language codes to the `SUPPORTED_LANGUAGES` dictionary. Gemini 2.5 Flash is particularly strong at multilingual tasks.

### How can I improve translation accuracy further?

Adding domain-specific glossaries or context to the System Instructions can significantly boost translation quality. For instance, if you're building a medical chat app, include a medical terminology reference. You can also apply [prompt engineering](/articles/gemini-basics/gemini-prompt-engineering-guide) techniques to iteratively refine translation output.

### How much can I use on the free tier?

Gemini 2.5 Flash's free tier allows up to 15 requests per minute and 1,500 requests per day (as of March 2026). This is more than enough for personal use and development testing. For production workloads with higher traffic, check out our guide on [API rate limiting and quota management](/articles/gemini-api/gemini-api-rate-limiting-quota-management).

### Can I use SSE (Server-Sent Events) instead of WebSocket?

Yes — if you only need one-way streaming from server to client, SSE is a viable alternative. However, for chat applications that require bidirectional communication, WebSocket is the better fit.
