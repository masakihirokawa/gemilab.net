## Introduction

NotebookLM Enterprise API represents Google's most ambitious foray into programmatic knowledge management. Unlike consumer tools that rely on manual UI interactions, the Enterprise APIs enable developers to build fully automated systems that transform diverse content—text, images, audio, video—into professional podcasts and dynamically managed knowledge bases at scale.

This guide is designed for enterprise developers who need to implement production-grade knowledge systems. We'll walk through the complete architectural patterns, from the Podcast API's audio generation capabilities to the Notebook API's resource management, culminating in seamless Gemini integration that puts the knowledge directly into your users' hands.

## Understanding NotebookLM Enterprise API Architecture

### The Three-Layer Stack

NotebookLM Enterprise APIs are organized into three complementary layers:

**Layer 1: Podcast API**
- Input formats: Text (up to 100K tokens), images, audio files, video
- Output: MP3 (192kbps standard)
- Processing: Asynchronous job-based system
- Latency: 5–15 minutes depending on content size
- Use cases: Automated podcast generation, audiobook creation, voice-first content delivery

**Layer 2: Notebook API**
- Programmatic notebook creation and CRUD operations
- Multi-source data aggregation (PDFs, web URLs, plain text, structured documents)
- Query-based knowledge retrieval
- Batch operations for enterprise-scale deployments

**Layer 3: Gemini Integration**
- 8× context window compared to standard Gemini
- 6× conversation memory retention
- Native support for EPUB, CSV, OCR-processed documents
- Direct embedding in Gemini App for end-user access

Together, these layers form a closed-loop system: ingest diverse content → organize into Notebooks → generate audio assets → serve through Gemini's natural interface.

## Podcast API: Core Implementation Patterns

### Foundational Architecture

The Podcast API's strength lies in its understanding-first approach. Rather than reading content aloud, it comprehends the material and generates conversational, engaging podcasts with multiple speakers.

```python
from google.api_core.client_options import ClientOptions
from google.auth.transport.requests import Request
from google.oauth2.service_account import Credentials
import anthropic
import asyncio
import time

class PodcastGenerator:
    """
    Enterprise-grade Podcast API client with production-level
    error handling and retry logic.
    """

    def __init__(self, credentials_path: str):
        """
        Initialize with service account credentials.

        Args:
            credentials_path: Path to Google Cloud service account JSON
        """
        self.credentials = Credentials.from_service_account_file(
            credentials_path,
            scopes=['https://www.googleapis.com/auth/notebooklm']
        )
        self.base_url = "https://notebooklm.googleapis.com/v1"
        self.max_retries = 3
        self.retry_delay = 5

    def create_notebook(self, title: str, description: str, sources: list) -> dict:
        """
        Create a new Notebook with initial sources.

        Args:
            title: Human-readable notebook name
            description: Extended description for audit trails
            sources: List of source objects
                {
                    "type": "TEXT" | "PDF_URL" | "WEB_URL" | "FILE_URI",
                    "title": "Source title",
                    "content": "..." (for TEXT),
                    "uri": "gs://bucket/file" (for FILE_URI),
                    "url": "https://..." (for URLs)
                }

        Returns:
            Notebook resource with ID and metadata
        """
        import requests

        headers = self._get_auth_headers()
        payload = {
            "displayName": title,
            "description": description,
            "sources": sources
        }

        response = requests.post(
            f"{self.base_url}/notebooks",
            headers=headers,
            json=payload,
            timeout=30
        )

        if response.status_code != 200:
            raise RuntimeError(f"Notebook creation failed: {response.text}")

        notebook = response.json()
        return {
            "id": notebook["name"],
            "displayName": notebook["displayName"],
            "createTime": notebook.get("createTime"),
            "sourceCount": len(sources)
        }

    def generate_podcast(
        self,
        notebook_id: str,
        config: dict = None
    ) -> dict:
        """
        Queue a podcast generation job for the given notebook.

        Args:
            notebook_id: Resource ID (e.g., "notebooks/abc123")
            config: Optional configuration
                {
                    "speakers": 2,           # 1-3
                    "duration_minutes": 8,   # approx
                    "tone": "conversational", # professional|casual|conversational
                    "language": "en"         # en|ja|zh|es|fr
                }

        Returns:
            Job resource with status and estimated completion time
        """
        import requests

        headers = self._get_auth_headers()
        cfg = config or {
            "speakers": 2,
            "duration_minutes": 8,
            "tone": "conversational",
            "language": "en"
        }

        payload = {
            "notebook": notebook_id,
            "config": cfg
        }

        response = requests.post(
            f"{self.base_url}/{notebook_id}:generatePodcast",
            headers=headers,
            json=payload,
            timeout=30
        )

        if response.status_code != 200:
            raise RuntimeError(f"Podcast generation failed: {response.text}")

        job = response.json()
        return {
            "jobId": job["name"],
            "status": job["state"],
            "estimatedCompletion": job.get("estimatedCompletionTime"),
            "createTime": job.get("createTime")
        }

    def poll_podcast_status(self, job_id: str) -> dict:
        """
        Check the status of a podcast generation job.

        Args:
            job_id: The job resource ID from generate_podcast()

        Returns:
            Job status with output URI when complete
        """
        import requests

        headers = self._get_auth_headers()

        response = requests.get(
            f"{self.base_url}/{job_id}",
            headers=headers,
            timeout=30
        )

        if response.status_code != 200:
            raise RuntimeError(f"Status check failed: {response.text}")

        job = response.json()
        return {
            "jobId": job["name"],
            "status": job["state"],
            "outputUri": job.get("outputUri"),  # GCS path when COMPLETED
            "duration": job.get("duration"),
            "createTime": job.get("createTime")
        }

    async def wait_for_podcast_completion(
        self,
        job_id: str,
        poll_interval: int = 30,
        max_wait_seconds: int = 3600
    ) -> dict:
        """
        Asynchronously wait for podcast generation to complete.
        Better than synchronous polling for enterprise pipelines.

        Args:
            job_id: Job resource ID
            poll_interval: Seconds between status checks
            max_wait_seconds: Maximum wait time (1 hour default)

        Returns:
            Completed job with output URI
        """
        start_time = time.time()

        while time.time() - start_time < max_wait_seconds:
            job_status = self.poll_podcast_status(job_id)

            if job_status["status"] == "COMPLETED":
                return job_status
            elif job_status["status"] == "FAILED":
                raise RuntimeError(f"Podcast generation failed: {job_id}")

            await asyncio.sleep(poll_interval)

        raise TimeoutError(f"Podcast generation timeout: {job_id}")

    def _get_auth_headers(self) -> dict:
        """Generate Authorization header from credentials."""
        auth_request = Request()
        self.credentials.refresh(auth_request)

        return {
            "Authorization": f"Bearer {self.credentials.token}",
            "Content-Type": "application/json"
        }
```

This foundation enables reliable, auditable podcast generation at enterprise scale.

## Notebook API: Building Scalable Knowledge Bases

### Multi-Source Integration and Lifecycle Management

The Notebook API's real power emerges when managing dozens or hundreds of knowledge repositories across an organization.

```python
from typing import List, Dict, Any, Optional
import requests
import json
from datetime import datetime
import logging

logger = logging.getLogger(__name__)

class EnterpriseNotebookManager:
    """
    Production-grade Notebook lifecycle manager for enterprises.
    Handles bulk operations, versioning, and governance workflows.
    """

    def __init__(self, api_key: str, project_id: str):
        self.api_key = api_key
        self.project_id = project_id
        self.base_url = "https://notebooklm.googleapis.com/v1"
        self.headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }
        self.operation_log = []

    def batch_create_notebooks(
        self,
        notebook_specs: List[Dict[str, Any]],
        department: str = None
    ) -> List[Dict]:
        """
        Create multiple notebooks in a single operation.
        Ideal for organizational onboarding scenarios.

        Args:
            notebook_specs: Array of notebook configurations
                [
                    {
                        "displayName": "Sales Q1 Proposals",
                        "description": "2026 Q1 proposal database",
                        "labels": {"department": "sales", "quarter": "q1"},
                        "sources": [
                            {
                                "displayName": "Q1 Deck",
                                "uri": "gs://corporate-docs/q1-2026.pdf"
                            }
                        ]
                    },
                    ...
                ]
            department: Optional department tag for governance

        Returns:
            List of created notebook resources
        """
        created = []
        failed = []

        for i, spec in enumerate(notebook_specs):
            try:
                logger.info(f"Creating notebook {i+1}/{len(notebook_specs)}: {spec['displayName']}")

                payload = {
                    "displayName": spec["displayName"],
                    "description": spec.get("description", ""),
                    "labels": spec.get("labels", {}),
                    "sources": spec.get("sources", [])
                }

                # Add department tag for governance
                if department:
                    payload["labels"]["department"] = department

                response = requests.post(
                    f"{self.base_url}/projects/{self.project_id}/notebooks",
                    headers=self.headers,
                    json=payload,
                    timeout=30
                )

                if response.status_code == 200:
                    notebook = response.json()
                    created_record = {
                        "id": notebook["name"],
                        "displayName": notebook["displayName"],
                        "status": "CREATED",
                        "sourceCount": len(spec.get("sources", [])),
                        "createdAt": datetime.now().isoformat()
                    }
                    created.append(created_record)
                    logger.info(f"✓ Created: {notebook['name']}")
                else:
                    failed_record = {
                        "displayName": spec["displayName"],
                        "status": "FAILED",
                        "errorCode": response.status_code,
                        "error": response.text
                    }
                    failed.append(failed_record)
                    logger.error(f"✗ Failed: {response.text}")

            except Exception as e:
                failed.append({
                    "displayName": spec.get("displayName"),
                    "status": "FAILED",
                    "error": str(e)
                })
                logger.error(f"Exception: {e}")

        self.operation_log.append({
            "operation": "batch_create_notebooks",
            "timestamp": datetime.now().isoformat(),
            "created": len(created),
            "failed": len(failed)
        })

        return {
            "created": created,
            "failed": failed,
            "summary": {
                "total": len(notebook_specs),
                "succeeded": len(created),
                "failed": len(failed)
            }
        }

    def add_sources_to_notebook(
        self,
        notebook_id: str,
        sources: List[Dict[str, str]]
    ) -> Dict:
        """
        Add new sources to an existing notebook.
        Supports continuous knowledge base updates.

        Args:
            notebook_id: Target notebook resource ID
            sources: New sources to add
                [
                    {
                        "displayName": "Latest research",
                        "sourceType": "PDF_URL" | "WEB_URL" | "TEXT" | "FILE_URI",
                        "uri": "https://..." or "gs://..."
                    }
                ]

        Returns:
            Update operation status
        """
        results = []

        for source in sources:
            try:
                payload = {
                    "displayName": source["displayName"],
                    "sourceType": source.get("sourceType", "TEXT"),
                    "uri": source.get("uri", ""),
                    "content": source.get("content", "")
                }

                response = requests.post(
                    f"{self.base_url}/{notebook_id}/sources",
                    headers=self.headers,
                    json=payload,
                    timeout=30
                )

                results.append({
                    "displayName": source["displayName"],
                    "status": "SUCCESS" if response.status_code == 200 else "FAILED",
                    "response": response.json() if response.status_code == 200 else response.text
                })

            except Exception as e:
                results.append({
                    "displayName": source.get("displayName"),
                    "status": "FAILED",
                    "error": str(e)
                })

        return {"sourceUpdates": results}

    def query_with_gemini_context(
        self,
        notebook_id: str,
        query: str,
        context_window: str = "LARGE"
    ) -> Dict:
        """
        Execute a query against notebook content with Gemini's reasoning.
        The LARGE context window processes 8x more material than standard Gemini.

        Args:
            notebook_id: Notebook to query
            query: User question or search
            context_window: "SMALL" | "LARGE" (8x)

        Returns:
            Answer with cited sources
        """
        payload = {
            "query": query,
            "model": "gemini-2.0-flash",
            "contextWindow": context_window,
            "citations": True
        }

        response = requests.post(
            f"{self.base_url}/{notebook_id}:query",
            headers=self.headers,
            json=payload,
            timeout=30
        )

        if response.status_code == 200:
            result = response.json()
            return {
                "answer": result.get("answer"),
                "citations": result.get("citations", []),
                "confidence": result.get("confidence", 0.0),
                "responseTime": result.get("processingTimeMs")
            }
        else:
            raise RuntimeError(f"Query failed: {response.text}")
```

## Gemini Integration: The End-User Layer

### Seamless Knowledge Access Through Gemini

The final piece: embedding Notebooks directly into the Gemini experience, giving end-users instant access to organizational knowledge through natural conversation.

```python
from google.generativeai import genai
from typing import Generator

class GeminiNotebookBridge:
    """
    Bridges Notebook knowledge bases with Gemini's natural language interface.
    Creates transparent knowledge access for end-users.
    """

    def __init__(self, api_key: str):
        genai.configure(api_key=api_key)
        self.model = genai.GenerativeModel('gemini-2.0-flash')

    def create_notebook_context_system_prompt(
        self,
        notebook_id: str,
        notebook_name: str,
        department: str = None
    ) -> str:
        """
        Construct a system prompt that instructs Gemini to prioritize
        knowledge from a specific Notebook.

        Args:
            notebook_id: The Notebook resource ID
            notebook_name: Human-readable name
            department: Optional department context

        Returns:
            System prompt string
        """
        system_prompt = f"""You are an expert knowledge assistant powered by the "{notebook_name}" knowledge base.

Notebook ID: {notebook_id}
Enhanced Context: 8x context window enabled
Conversation Memory: 6x extended retention

Guidelines:
1. Prioritize information from the Notebook sources
2. Always cite which source(s) informed your answer
3. If the Notebook doesn't contain relevant information, say so explicitly
4. Use professional language
5. If asked about topics outside the Notebook's scope, acknowledge the limitation

Tone: Helpful, authoritative, precise
Language: English
Format: Clear, structured responses with citations
"""
        if department:
            system_prompt += f"\nContext: This knowledge base serves the {department} department.\n"

        return system_prompt

    def chat_with_knowledge_base(
        self,
        notebook_id: str,
        notebook_name: str,
        user_message: str,
        conversation_history: list = None
    ) -> str:
        """
        Execute a single turn of conversation with Notebook context.

        Args:
            notebook_id: Source Notebook ID
            notebook_name: Display name
            user_message: User's question
            conversation_history: Prior messages for context

        Returns:
            Assistant response
        """
        system_prompt = self.create_notebook_context_system_prompt(
            notebook_id,
            notebook_name
        )

        messages = conversation_history or []
        messages.append({
            "role": "user",
            "content": user_message
        })

        response = self.model.generate_content(
            messages,
            system_instruction=system_prompt,
            safety_settings={
                "HARM_CATEGORY_HARASSMENT": "BLOCK_MEDIUM_AND_ABOVE",
                "HARM_CATEGORY_HATE_SPEECH": "BLOCK_MEDIUM_AND_ABOVE",
                "HARM_CATEGORY_SEXUALLY_EXPLICIT": "BLOCK_MEDIUM_AND_ABOVE",
                "HARM_CATEGORY_DANGEROUS_CONTENT": "BLOCK_MEDIUM_AND_ABOVE"
            },
            generation_config={
                "temperature": 0.7,
                "top_p": 0.95,
                "max_output_tokens": 2048
            }
        )

        return response.text

    def stream_notebook_conversation(
        self,
        notebook_id: str,
        notebook_name: str,
        user_message: str
    ) -> Generator[str, None, None]:
        """
        Stream responses token-by-token for responsive UI.

        Args:
            notebook_id: Notebook resource ID
            notebook_name: Display name
            user_message: User query

        Yields:
            Response tokens as they're generated
        """
        system_prompt = self.create_notebook_context_system_prompt(
            notebook_id,
            notebook_name
        )

        response = self.model.generate_content(
            user_message,
            system_instruction=system_prompt,
            stream=True,
            generation_config={
                "temperature": 0.7,
                "max_output_tokens": 2048
            }
        )

        for chunk in response:
            yield chunk.text
```

import asyncio
from concurrent.futures import ThreadPoolExecutor
from datetime import datetime

class SalesEnablementPipeline:
    """
    Complete automation pipeline:
    Ingest proposals → Create Notebooks → Generate Podcasts → Distribute via Gemini
    """

    def __init__(self, api_key: str, project_id: str):
        self.notebook_manager = EnterpriseNotebookManager(api_key, project_id)

        self.gemini_bridge = GeminiNotebookBridge(api_key)
        self.execution_log = []

    def process_quarterly_proposals(
        self,
        proposal_documents: List[Dict[str, str]]
    ) -> Dict:
        """
        Take raw proposal documents and transform them into:
        1. Organized Notebooks
        2. Podcast versions
        3. Gemini-accessible knowledge bases

        Args:
            proposal_documents:
                [
                    {
                        "filename": "Q1_2026_Enterprise_Solutions.pdf",
                        "gcs_uri": "gs://sales-proposals/q1_2026/...",
                        "category": "Enterprise"
                    },
                    ...
                ]

        Returns:
            Complete pipeline execution report
        """
        execution_report = {
            "timestamp": datetime.now().isoformat(),
            "notebooks": [],
            "podcasts": [],
            "distribution": []
        }

        # Step 1: Create Notebooks from proposals
        notebook_specs = [
            {
                "displayName": f"Proposal: {doc['filename'].replace('.pdf', '')}",
                "description": f"Generated from {doc['filename']} on {datetime.now().isoformat()}",
                "labels": {
                    "type": "proposal",
                    "category": doc["category"],
                    "quarter": "q1_2026"
                },
                "sources": [
                    {
                        "displayName": doc["filename"],
                        "sourceType": "PDF_URI",
                        "uri": doc["gcs_uri"]
                    }
                ]
            }
            for doc in proposal_documents
        ]

        notebooks_result = self.notebook_manager.batch_create_notebooks(
            notebook_specs,
            department="sales"
        )

        execution_report["notebooks"] = notebooks_result["created"]

        # Step 2: Generate podcasts for each notebook
        podcast_config = {
            "speakers": 2,  # Sales rep + technical specialist
            "duration_minutes": 10,
            "tone": "professional",
            "language": "en"
        }

        for notebook in notebooks_result["created"]:
            try:
                podcast_job = self.podcast_generator.generate_podcast(
                    notebook["id"],
                    podcast_config
                )
                execution_report["podcasts"].append({
                    "notebookId": notebook["id"],
                    "jobId": podcast_job["jobId"],
                    "status": "QUEUED",
                    "estimatedCompletion": podcast_job["estimatedCompletion"]
                })
            except Exception as e:
                execution_report["podcasts"].append({
                    "notebookId": notebook["id"],
                    "status": "FAILED",
                    "error": str(e)
                })

        # Step 3: Prepare distribution via Gemini
        for notebook in notebooks_result["created"]:
            share_url = f"https://gemini.google.com/notebook/{notebook['id']}"
            execution_report["distribution"].append({
                "notebookId": notebook["id"],
                "shareUrl": share_url,
                "accessLevel": "sales_team",
                "distributedAt": datetime.now().isoformat()
            })

        self.execution_log.append(execution_report)
        return execution_report

    async def monitor_podcast_generation(self) -> Dict:
        """
        Poll for podcast completion asynchronously.
        Prevents blocking while waiting for API processing.
        """
        pending_jobs = [
            p for p in self.execution_log[-1]["podcasts"]
            if p["status"] == "QUEUED"
        ]

        completed = []

        for job in pending_jobs:
            try:
                final_status = await self.podcast_generator.wait_for_podcast_completion(
                    job["jobId"],
                    poll_interval=30,
                    max_wait_seconds=1800
                )
                completed.append({
                    "jobId": job["jobId"],
                    "status": "COMPLETED",
                    "outputUri": final_status["outputUri"]
                })
            except asyncio.TimeoutError:
                completed.append({
                    "jobId": job["jobId"],
                    "status": "TIMEOUT"
                })

        return {
            "completed": completed,
            "timestamp": datetime.now().isoformat()
        }

## Performance Optimization

### Token Management and Content Chunking

The 100K token limit requires smart content management:

```python
def estimate_token_count(text: str) -> int:
    """
    Estimate token count for text.
    - English: ~4 chars/token
    - Japanese: ~3 chars/token
    """
    # Simple heuristic; for production, use a proper tokenizer
    english_chars = sum(1 for c in text if ord(c) < 0x3040)
    japanese_chars = len(text) - english_chars

    tokens = (english_chars / 4) + (japanese_chars / 3)
    return int(tokens)

def chunk_documents_for_notebook(
    documents: List[str],
    max_tokens: int = 100000
) -> List[str]:
    """
    Split large document collections into Notebook-compatible chunks.
    """
    chunks = []
    current_chunk = ""
    current_tokens = 0

    for doc in documents:
        doc_tokens = estimate_token_count(doc)

        if current_tokens + doc_tokens > max_tokens:
            if current_chunk:
                chunks.append(current_chunk)
            current_chunk = doc
            current_tokens = doc_tokens
        else:
            current_chunk += "\n\n---\n\n" + doc
            current_tokens += doc_tokens

    if current_chunk:
        chunks.append(current_chunk)

    return chunks
```

## FAQ

**Q: How long does podcast generation typically take?**

A: Generation typically completes in 5–15 minutes depending on input size and speaker count. For ~10K tokens, expect 5–8 minutes. For near-maximum content (80K+ tokens), budget 15+ minutes. Implement exponential backoff polling rather than frequent checks.

**Q: What's the difference between standard Gemini context and Notebook-enhanced context?**

A: Standard Gemini operates with a fixed context window (~100K tokens). When accessing a Notebook through Gemini, the context window expands 8×, enabling simultaneous analysis of significantly more material. This is crucial for complex, multi-document queries.

**Q: Should we use service accounts or OAuth 2.0?**

A: Service accounts for backend automation pipelines (no user interaction); OAuth 2.0 for client-side applications where users interact directly. Service accounts are credential-based; OAuth 2.0 is user-permission-based.

**Q: Can we generate podcasts in multiple languages simultaneously?**

A: Yes. Set the `language` parameter per podcast job (`en`, `ja`, `zh`, `es`, `fr`, etc.). This enables global enterprises to produce localized audio assets from a single source.

**Q: What are the customization options for podcast generation?**

A: Speaker count (1–3), approximate duration, tone (conversational / professional / casual), and language are currently configurable. Future releases are expected to add speaker voice selection and custom speaker personas.

**Q: How do we handle audit trails and compliance with large-scale notebook operations?**

A: Use the `labels` field during notebook creation to tag by department, compliance category, data classification, etc. Log all operations with timestamps. For regulatory requirements, archive notebooks to Cloud Storage with retention policies.
