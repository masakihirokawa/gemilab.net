## Introduction — Extending Gemini's Capabilities with Function Calling

Gemini excels at text generation, but it can't directly access real-time weather data, live database contents, or other information outside its training data. Function Calling breaks through this limitation.

With Function Calling, Gemini decides when to invoke an external tool and automatically generates the required parameters. You execute the actual API call with those parameters and return the results to Gemini, which then incorporates the live data into a natural response.

This article walks you through everything from the basics to multi-tool orchestration, with Python code at every step.

## How Function Calling Works

Function Calling operates in three steps.

1. **Define functions**: Tell Gemini about available functions — their names, descriptions, and parameters
2. **Gemini decides**: Gemini analyzes the user's question and selects the right function with appropriate parameters
3. **Integrate results**: Return function execution results to Gemini for natural language response generation

```python
# Function Calling flow (conceptual)
# User: "What's the weather in Tokyo?"
#   ↓
# Gemini: Decides to call get_weather(city="Tokyo")
#   ↓
# Your code: Actually calls the weather API
#   ↓
# Gemini: "It's currently sunny in Tokyo at 22°C"
```

## Basic Implementation: Weather Lookup

Let's start with the simplest example — fetching weather information.

```python
import google.generativeai as genai

# Configure API key
genai.configure(api_key="YOUR_API_KEY")

# Step 1: Define the function
get_weather = genai.protos.FunctionDeclaration(
    name="get_weather",
    description="Get current weather information for a specified city",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "city": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="City name (e.g., Tokyo, New York)",
            ),
            "unit": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                enum=["celsius", "fahrenheit"],
                description="Temperature unit (default: celsius)",
            ),
        },
        required=["city"],
    ),
)

# Pass tool definition to the model
tool = genai.protos.Tool(function_declarations=[get_weather])
model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    tools=[tool],
)

# Step 2: Send user's question
chat = model.start_chat()
response = chat.send_message("What's the weather like in Tokyo?")

# When Gemini requests a function call
function_call = response.candidates[0].content.parts[0].function_call
print(f"Function: {function_call.name}")
print(f"Parameters: {dict(function_call.args)}")

# Expected output:
# Function: get_weather
# Parameters: {'city': 'Tokyo', 'unit': 'celsius'}
```

### Executing the API Call and Returning Results

```python
import requests

def call_weather_api(city: str, unit: str = "celsius") -> dict:
    """Call the actual weather API."""
    api_key = "YOUR_WEATHER_API_KEY"
    url = "https://api.openweathermap.org/data/2.5/weather"
    params = {
        "q": city,
        "appid": api_key,
        "units": "metric" if unit == "celsius" else "imperial",
    }
    resp = requests.get(url, params=params)
    data = resp.json()
    return {
        "city": city,
        "temperature": data["main"]["temp"],
        "condition": data["weather"][0]["description"],
        "humidity": data["main"]["humidity"],
    }

# Step 3: Execute the function and return results to Gemini
weather_data = call_weather_api(
    city=function_call.args["city"],
    unit=function_call.args.get("unit", "celsius"),
)

# Send results back to Gemini
response = chat.send_message(
    genai.protos.Content(
        parts=[
            genai.protos.Part(
                function_response=genai.protos.FunctionResponse(
                    name="get_weather",
                    response={"result": weather_data},
                )
            )
        ]
    )
)

print(response.text)
# Expected output:
# The current weather in Tokyo is sunny with a temperature of 22°C
# and 45% humidity. It's a comfortable day!
```

## Multi-Tool Orchestration

In real-world applications, you'll define multiple tools and let Gemini choose the right one based on context.

```python
import google.generativeai as genai

# Weather tool
get_weather = genai.protos.FunctionDeclaration(
    name="get_weather",
    description="Get current weather for a city",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "city": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="City name",
            ),
        },
        required=["city"],
    ),
)

# Database search tool
search_database = genai.protos.FunctionDeclaration(
    name="search_database",
    description="Search the product database by keyword",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "query": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="Search keyword",
            ),
            "max_results": genai.protos.Schema(
                type=genai.protos.Type.INTEGER,
                description="Maximum number of results (default: 5)",
            ),
        },
        required=["query"],
    ),
)

# Email tool
send_email = genai.protos.FunctionDeclaration(
    name="send_email",
    description="Send an email to a specified address",
    parameters=genai.protos.Schema(
        type=genai.protos.Type.OBJECT,
        properties={
            "to": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="Recipient email address",
            ),
            "subject": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="Email subject",
            ),
            "body": genai.protos.Schema(
                type=genai.protos.Type.STRING,
                description="Email body",
            ),
        },
        required=["to", "subject", "body"],
    ),
)

# Register all tools
tools = genai.protos.Tool(
    function_declarations=[get_weather, search_database, send_email]
)

model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    tools=[tools],
)

# Gemini automatically selects the right tool based on the question
chat = model.start_chat()

# Example 1: Weather question → get_weather selected
response = chat.send_message("What's the weather in Osaka?")

# Example 2: Product question → search_database selected
response = chat.send_message("Find wireless earbuds")

# Example 3: Email request → send_email selected
response = chat.send_message("Email John about the meeting reschedule")
```

## Error Handling and Best Practices

### Validating Function Calls

```python
def handle_function_call(response) -> dict | None:
    """Safely process a Function Call response."""
    try:
        candidate = response.candidates[0]
        part = candidate.content.parts[0]

        # Check if a function call is present
        if not hasattr(part, "function_call") or not part.function_call.name:
            return None

        fc = part.function_call
        return {
            "name": fc.name,
            "args": dict(fc.args),
        }
    except (IndexError, AttributeError) as e:
        print(f"Failed to parse Function Call: {e}")
        return None

# Usage
result = handle_function_call(response)
if result:
    print(f"Execute {result['name']} with {result['args']}")
else:
    print(f"Text response: {response.text}")
```

### Best Practices

Here are the key principles for effective Function Calling.

**Write specific function descriptions**: Instead of "get data," write "search the product database by keyword and return product name, price, and stock count." Vague descriptions lead to Gemini making poor function selection decisions.

**Specify types and defaults for parameters**: Keep `required` parameters to a minimum, and include default values in optional parameter descriptions. This gives Gemini more flexibility in how it calls your functions.

**Return meaningful error messages**: When a function execution fails, return the error reason rather than an empty result. Gemini understands error messages and can provide users with appropriate explanations.

## Frequently Asked Questions (FAQ)

### Q1: Is Function Calling available on all Gemini models?

Function Calling is available on Gemini 2.5 Pro and Gemini 2.5 Flash. The 1.5 series also supports it, but the 2.5 series delivers more accurate function selection and parameter generation. You can also test Function Calling in Google AI Studio's playground.

### Q2: Can Gemini call multiple functions in a single request?

Yes, Gemini supports Parallel Function Calling and may return multiple function calls in a single response. For example, "Compare the weather in Tokyo and Osaka" could return both `get_weather(city="Tokyo")` and `get_weather(city="Osaka")` simultaneously.

### Q3: Are there additional charges for Function Calling?

No additional charges apply for Function Calling itself. Billing follows the standard API pricing based on input and output token counts. Keep in mind that function definition schemas count as input tokens, so defining many functions will increase your token usage.

### Q4: What security considerations should I keep in mind?

The critical point is that Function Calling only determines parameters — actual execution happens in your code. Always validate the parameters Gemini provides before execution. Implement protections against SQL injection, path traversal, and other injection attacks, just as you would with any user-supplied input.
