# Gemini Code Execution API: Complete Guide to AI-Generated Code Execution

## Introduction

The Gemini Code Execution API represents a breakthrough in AI-assisted development. Instead of asking an AI to explain how to solve a problem, you can now let the AI write and execute Python code in a secure Google-managed sandbox environment.

This capability transforms how developers approach computationally intensive tasks. Whether you need complex mathematical calculations, data analysis, statistical modeling, or algorithmic problem-solving, the Code Execution API handles all of this seamlessly. The AI not only generates the code but also executes it and interprets the results—all within a single API call.

This comprehensive guide covers everything from basic setup to advanced production patterns, with practical code examples you can use immediately.

## Understanding the Code Execution API

### How It Works

The Code Execution API follows this workflow:

1. **Request**: You submit a problem or computational task to the API
2. **Code Generation**: Gemini AI generates appropriate Python code to solve the problem
3. **Execution**: The code runs in Google's isolated, secure sandbox environment
4. **Result Retrieval**: Execution results are captured and returned to your application
5. **Interpretation**: The AI provides context and explanation based on the execution results

This entire process happens transparently through the API, eliminating the complexity of managing separate code execution infrastructure.

### What Makes It Powerful

Unlike traditional API interactions where the AI provides explanations in text, Code Execution enables:

- **Precision**: Mathematical calculations are exact, not approximate or hallucinated
- **Verification**: Results are computed, not synthesized or invented
- **Iteration**: You can refine analyses through multi-turn conversations
- **Complexity**: Handle problems that would be impractical to express as prompts
- **Transparency**: See exactly what code was executed

### Supported Environment and Limitations

Code Execution currently operates under these parameters:

- **Language**: Python (JavaScript, Java, and other languages not yet supported)
- **Execution Environment**: Google-managed, isolated sandbox
- **Time Limit**: Maximum 90 seconds per request
- **Memory**: Up to 1GB per execution
- **Network Access**: No external network calls (security by design)
- **Available Libraries**: NumPy, SciPy, Pandas, Matplotlib, and many standard Python libraries

### Your First Code Execution

Here's the simplest possible example to get you started:

```python
from google import genai
import os

# Initialize the client
api_key = os.getenv("GEMINI_API_KEY")
client = genai.Client(api_key=api_key)

# Request AI to solve a problem using code execution
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[{
        "role": "user",
        "parts": [{
            "text": "Find all prime numbers between 10 and 100. Return the count and their sum."
        }]
    }],
    tools=[{"code_execution": {}}]
)

# Process the response
for part in response.content.parts:
    if hasattr(part, 'text'):
        print("AI Response:", part.text)
    elif hasattr(part, 'executable_code'):
        print("Code Generated:", part.executable_code.code)
    elif hasattr(part, 'code_execution_result'):
        print("Execution Output:", part.code_execution_result.output)
```

### Robust Error Handling and Retries

Production-grade code requires proper error handling:

```python
from google import genai
import os
from google.api_core import exceptions
import time

api_key = os.getenv("GEMINI_API_KEY")
client = genai.Client(api_key=api_key)

def execute_with_retry(prompt, max_retries=3, backoff_factor=2):
    """Execute code with exponential backoff retry logic"""
    for attempt in range(max_retries):
        try:
            response = client.models.generate_content(
                model="gemini-2.5-flash",
                contents=[{"role": "user", "parts": [{"text": prompt}]}],
                tools=[{"code_execution": {}}]
            )
            return response

        except exceptions.DeadlineExceeded as e:
            wait_time = backoff_factor ** attempt
            if attempt < max_retries - 1:
                print(f"Timeout on attempt {attempt + 1}. Retrying in {wait_time}s...")
                time.sleep(wait_time)
            else:
                print(f"Failed after {max_retries} attempts")
                raise

        except exceptions.GoogleAPICallError as e:
            print(f"API Error: {e}")
            raise

# Usage
try:
    result = execute_with_retry(
        "Calculate the sum of digits for all numbers 1 to 10000"
    )
    print(result)
except Exception as e:
    print(f"Execution failed: {e}")
```

### Response Caching for Cost Optimization

Cache identical requests to reduce API calls and costs:

```python
import hashlib
import json
from datetime import datetime, timedelta

class CodeExecutionCache:
    def __init__(self, ttl_minutes=60):
        self.cache = {}
        self.ttl = timedelta(minutes=ttl_minutes)

    def _create_key(self, prompt):
        return hashlib.sha256(prompt.encode()).hexdigest()

    def get(self, prompt):
        key = self._create_key(prompt)
        if key in self.cache:
            result, timestamp = self.cache[key]
            if datetime.now() - timestamp < self.ttl:
                return result
            del self.cache[key]
        return None

    def set(self, prompt, result):
        key = self._create_key(prompt)
        self.cache[key] = (result, datetime.now())

cache = CodeExecutionCache(ttl_minutes=30)

def cached_execute(prompt):
    # Check cache first
    cached = cache.get(prompt)
    if cached:
        print("Returning cached result")
        return cached

    # Execute if not cached
    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=prompt,
        tools=[{"code_execution": {}}]
    )
    cache.set(prompt, response)
    return response
```

## Best Practices

### 1. Input Validation and Sanitization

Always validate and clean user-provided prompts:

```python
def validate_prompt(prompt, max_length=5000):
    """Validate user input before code execution"""
    if not prompt or not isinstance(prompt, str):
        raise ValueError("Prompt must be a non-empty string")

    if len(prompt) > max_length:
        raise ValueError(f"Prompt exceeds {max_length} characters")

    # Check for suspicious patterns
    dangerous_patterns = ["drop table", "delete from", "exec(", "eval("]
    lower_prompt = prompt.lower()
    for pattern in dangerous_patterns:
        if pattern in lower_prompt:
            raise ValueError(f"Potentially dangerous pattern detected: {pattern}")

    return prompt.strip()

# Usage

except ValueError as e:
    return {"error": str(e)}
```

### 2. Rate Limiting and Quota Management

Implement rate limiting to stay within API quotas:

```python
import time
from functools import wraps

class APIRateLimiter:
    def __init__(self, calls_per_minute=60):
        self.limit = calls_per_minute
        self.calls = []

    def __call__(self, func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            now = time.time()
            # Keep only calls from the last minute
            self.calls = [t for t in self.calls if now - t < 60]

            if len(self.calls) >= self.limit:
                sleep_time = 60 - (now - self.calls[0])
                print(f"Rate limit reached. Waiting {sleep_time:.1f}s")
                time.sleep(sleep_time)

            self.calls.append(now)
            return func(*args, **kwargs)
        return wrapper

@APIRateLimiter(calls_per_minute=30)
```

## Frequently Asked Questions

### Q1. Can Code Execution handle any Python code?

**A.** Code Execution runs in an isolated sandbox with intentional restrictions. You can use Code Execution for:

- Mathematical and statistical computations
- Data processing and manipulation
- Algorithm implementation and analysis
- Graph generation (matplotlib, plotly)
- Text processing and string operations
- Time series and financial analysis

However, external network access, arbitrary file system operations, and system commands are blocked for security. To understand the full capabilities of the Gemini API, check out [Gemini API Quickstart](/articles/gemini-api/gemini-api-quickstart).

### Q2. How does Code Execution pricing compare to standard API calls?

**A.** Using Code Execution incurs no additional charge beyond normal API pricing. However, the complete response—including the generated code, execution output, and AI explanations—is counted as tokens. This typically results in higher token consumption than simple text generation. Monitor your usage through the Google Cloud Console to understand your cost patterns.

### Q3. Can I monitor execution progress in real-time?

**A.** Code Execution runs synchronously, and you cannot track progress during execution. However, you can use [Streaming and Chat](/articles/gemini-api/streaming-and-chat) to stream AI-generated responses in real-time. For long-running computations, design your code to report progress through print statements, which will be captured in the execution output.

### Q4. What happens if the code times out?

**A.** If code execution exceeds 90 seconds, the request will return an error. For long-running analyses, break the work into smaller, independent steps. You can also use multi-turn conversations to progressively refine results.

### Q5. How can I combine Code Execution with other Gemini API features?

**A.** Code Execution works alongside other Gemini capabilities. Consider combining it with:

- **Function Calling**: Use [Function Calling Guide](/articles/gemini-api/function-calling-guide) to structure outputs from code execution as function results
- **Streaming**: Use [Streaming and Chat](/articles/gemini-api/streaming-and-chat) for real-time response delivery
- **Multi-modal Input**: Include images or documents alongside your computation requests for richer analysis
