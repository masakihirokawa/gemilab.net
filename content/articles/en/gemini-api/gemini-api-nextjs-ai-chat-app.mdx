## Build a Real AI Chat App with Gemini API and Next.js

Want to build something with the Gemini API but dread wiring up a separate frontend and backend? Next.js lets you handle both in a single project, making Gemini integration remarkably straightforward.

In this tutorial, you'll build a streaming AI chat application from scratch — the kind where responses appear token by token in real time, just like ChatGPT or Gemini's own interface. By the end, you'll have a fully functional chat app with conversation history and streaming responses.

<Callout type="info">
This tutorial uses Gemini 3 Flash — fast, cost-effective, and ideal for chat applications.
</Callout>

## Prerequisites and Setup

### What You Need

- **Node.js** 18+ (recommended: 20 LTS)
- **npm** or **pnpm**
- **A Google AI Studio API key** (free at [aistudio.google.com](https://aistudio.google.com/))
- Basic knowledge of TypeScript and React

### Getting Your API Key

Head to Google AI Studio and create a new key from the "API keys" section in the left menu. The free tier gives you 15 requests per minute and 1,500 requests per day — plenty for development and prototyping.

### Creating the Project

Run the following commands in your terminal:

```bash
npx create-next-app@latest gemini-chat-app --typescript --tailwind --app --src-dir
cd gemini-chat-app
```

Install the Google AI SDK:

```bash
npm install @google/genai
```

Create an environment variables file for your API key:

```bash
# .env.local
GEMINI_API_KEY=your_api_key_here
```

<Callout type="warning">
Never expose your API key on the client side. Always call the Gemini API from server-side code (Route Handlers or Server Actions).
</Callout>

## Architecture Overview

Here's how the pieces fit together:

```
Browser (React)
  ↓ POST /api/chat (send message)
Next.js Route Handler (server-side)
  ↓ generateContentStream()
Gemini API (streaming response)
  ↓ ReadableStream
Browser (real-time display)
```

Three key design decisions drive this architecture:

1. **API key security**: The Route Handler keeps your API key server-side only
2. **Streaming**: `ReadableStream` delivers tokens to the browser as they're generated
3. **Conversation context**: The client manages chat history and sends it with every request

## Implementing the Route Handler

First, let's build the backend that communicates with the Gemini API:

```typescript
// src/app/api/chat/route.ts
import { GoogleGenAI } from "@google/genai";
import { NextRequest } from "next/server";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

export async function POST(request: NextRequest) {
  const { messages } = await request.json();

  // Convert message format for Gemini API
  const contents = messages.map(
    (msg: { role: string; content: string }) => ({
      role: msg.role === "assistant" ? "model" : "user",
      parts: [{ text: msg.content }],
    })
  );

  // Generate a streaming response
  const response = await ai.models.generateContentStream({
    model: "gemini-3-flash",
    contents,
    config: {
      systemInstruction:
        "You are a helpful and knowledgeable AI assistant. " +
        "Provide concise, accurate answers.",
      temperature: 0.7,
      maxOutputTokens: 2048,
    },
  });

  // Convert to ReadableStream for the client
  const stream = new ReadableStream({
    async start(controller) {
      const encoder = new TextEncoder();
      try {
        for await (const chunk of response) {
          const text = chunk.text ?? "";
          if (text) {
            controller.enqueue(encoder.encode(text));
          }
        }
      } catch (error) {
        console.error("Streaming error:", error);
        controller.error(error);
      } finally {
        controller.close();
      }
    },
  });

  return new Response(stream, {
    headers: {
      "Content-Type": "text/plain; charset=utf-8",
      "Transfer-Encoding": "chunked",
    },
  });
}
```

A few things to note here. The `generateContentStream` method returns chunks as Gemini generates tokens. We pipe those chunks directly into a `ReadableStream` for the browser. Also watch the `role` mapping — Gemini uses `"model"` where most chat UIs use `"assistant"`.

## Building the Chat UI Component

Now for the frontend — a clean chat interface with real-time streaming:

```typescript
// src/app/components/ChatWindow.tsx
"use client";

import { useState, useRef, useEffect, FormEvent } from "react";

type Message = {
  role: "user" | "assistant";
  content: string;
};

export default function ChatWindow() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState("");
  const [isLoading, setIsLoading] = useState(false);
  const messagesEndRef = useRef<HTMLDivElement>(null);

  // Auto-scroll when new messages arrive
  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [messages]);

  async function handleSubmit(e: FormEvent) {
    e.preventDefault();
    if (!input.trim() || isLoading) return;

    const userMessage: Message = { role: "user", content: input.trim() };
    const updatedMessages = [...messages, userMessage];
    setMessages(updatedMessages);
    setInput("");
    setIsLoading(true);

    try {
      const response = await fetch("/api/chat", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ messages: updatedMessages }),
      });

      if (!response.ok) throw new Error("API request failed");
      if (!response.body) throw new Error("No response body");

      // Read the streaming response
      const reader = response.body.getReader();
      const decoder = new TextDecoder();
      let assistantContent = "";

      // Add empty assistant message
      setMessages((prev) => [...prev, { role: "assistant", content: "" }]);

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        const text = decoder.decode(value, { stream: true });
        assistantContent += text;

        // Update the message in real time
        setMessages((prev) => {
          const newMessages = [...prev];
          newMessages[newMessages.length - 1] = {
            role: "assistant",
            content: assistantContent,
          };
          return newMessages;
        });
      }
    } catch (error) {
      console.error("Chat error:", error);
      setMessages((prev) => [
        ...prev,
        {
          role: "assistant",
          content: "An error occurred. Please try again.",
        },
      ]);
    } finally {
      setIsLoading(false);
    }
  }

  return (
    <div className="flex flex-col h-screen max-w-3xl mx-auto">
      {/* Messages */}
      <div className="flex-1 overflow-y-auto p-4 space-y-4">
        {messages.length === 0 && (
          <div className="text-center text-gray-400 mt-20">
            <p className="text-2xl mb-2">✨</p>
            <p>Type a message to start the conversation</p>
          </div>
        )}
        {messages.map((msg, i) => (
          <div
            key={i}
            className={`flex ${
              msg.role === "user" ? "justify-end" : "justify-start"
            }`}
          >
            <div
              className={`max-w-[80%] rounded-2xl px-4 py-3 ${
                msg.role === "user"
                  ? "bg-blue-600 text-white"
                  : "bg-gray-100 text-gray-800"
              }`}
            >
              <p className="whitespace-pre-wrap">{msg.content}</p>
            </div>
          </div>
        ))}
        <div ref={messagesEndRef} />
      </div>

      {/* Input form */}
      <form
        onSubmit={handleSubmit}
        className="border-t p-4 flex gap-2"
      >
        <input
          type="text"
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Type a message..."
          className="flex-1 border rounded-xl px-4 py-3 focus:outline-none focus:ring-2 focus:ring-blue-500"
          disabled={isLoading}
        />
        <button
          type="submit"
          disabled={isLoading || !input.trim()}
          className="bg-blue-600 text-white px-6 py-3 rounded-xl hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed transition-colors"
        >
          {isLoading ? "Generating..." : "Send"}
        </button>
      </form>
    </div>
  );
}
```

## How Multi-Turn Conversations Work

Because the entire `messages` array is sent with every request, Gemini maintains full context of the conversation. For example:

```
User: What's the difference between interface and type in TypeScript?
Gemini: interface supports declaration merging, while type... (detailed explanation)

User: Can you show me a code example?
Gemini: Building on what I explained earlier, here are some examples...
```

Gemini uses previous exchanges to give contextually relevant answers, creating a natural conversation flow.

## Production-Ready Improvements

With the basics working, here are essential improvements for production use.

### Adding Rate Limiting

Protect your API key from abuse with simple rate limiting:

```typescript
// src/lib/rate-limit.ts
const rateLimitMap = new Map<string, { count: number; resetTime: number }>();

export function checkRateLimit(
  ip: string,
  maxRequests = 20,
  windowMs = 60000
): boolean {
  const now = Date.now();
  const entry = rateLimitMap.get(ip);

  if (!entry || now > entry.resetTime) {
    rateLimitMap.set(ip, { count: 1, resetTime: now + windowMs });
    return true;
  }

  if (entry.count >= maxRequests) {
    return false;
  }

  entry.count++;
  return true;
}
```

Use it in your Route Handler:

```typescript
// src/app/api/chat/route.ts (addition)
import { checkRateLimit } from "@/lib/rate-limit";

export async function POST(request: NextRequest) {
  const ip = request.headers.get("x-forwarded-for") ?? "unknown";

  if (!checkRateLimit(ip)) {
    return new Response("Too many requests", { status: 429 });
  }

  // ...existing code
}
```

### Better Error Handling

Gemini's safety filters may block certain responses. Handle this gracefully:

```typescript
// Check the finish reason in chunks
for await (const chunk of response) {
  if (chunk.candidates?.[0]?.finishReason === "SAFETY") {
    controller.enqueue(
      encoder.encode("I'm unable to respond to that query due to safety guidelines.")
    );
    break;
  }
  const text = chunk.text ?? "";
  if (text) {
    controller.enqueue(encoder.encode(text));
  }
}
```

## Frequently Asked Questions

### Q1: Should I use Gemini 3 Flash or Gemini 3 Pro for my chat app?

**Gemini 3 Flash** is the better choice for chat apps. It's faster and cheaper, which matters for real-time conversational interfaces. Use Gemini 3 Pro when your app needs complex reasoning or deep analysis. For more on choosing the right model, see the [model selection guide](/en/articles/gemini-basics/gemini-model-selection-guide).

### Q2: How much can I do with the free API tier?

The free tier allows 15 requests per minute and 1,500 per day. This is more than enough for development and small-scale use. For production, switch to a paid plan. Check the [API cost optimization guide](/en/articles/gemini-api/gemini-api-cost-optimization) for strategies to minimize costs.

### Q3: Can I add image or file upload support?

Yes. The Gemini API supports multimodal input, so you can send images and PDFs alongside text. Add an `inlineData` field to the `parts` array in your request. See the [multimodal API guide](/en/articles/gemini-api/multimodal-api-guide) for implementation details.

### Q4: How do I persist conversation history to a database?

Use Firebase Firestore, Supabase, or any database of your choice. Combine it with user authentication to let users resume past conversations. The [Firebase integration guide](/en/articles/gemini-dev/gemini-firebase-integration-guide) walks through this setup.

### Q5: Can I use Server Actions instead of Route Handlers?

Yes, Next.js Server Actions work as an alternative. However, streaming is more complex with Server Actions, so Route Handlers remain the simpler approach for chat apps.
