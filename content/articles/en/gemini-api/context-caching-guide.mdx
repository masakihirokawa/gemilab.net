## What Is Context Caching?

Gemini API's Context Caching lets you upload a large prompt or document once, cache it, and **reuse that cache in subsequent requests** — without paying to re-process it every time.

This is especially useful when asking multiple questions about the same material, or for agents that repeatedly reference a large codebase.

## Why It Cuts Costs

In a standard Gemini API request, you're billed for every input token every time. With a 100,000-token document and 10 questions:

- **Without cache**: 100,000 × 10 = 1M tokens in input costs
- **With cache**: 100,000 (first time) + 10 × (small questions only) = major savings

Cached tokens are billed at a lower rate than regular input tokens, so the effective cost savings can exceed 90% for repeated-access workloads.

## Setup

### 1. Create a cache

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

large_document = open("technical_manual.txt").read()

# Create cache with 30-minute TTL
cache = genai.caching.CachedContent.create(
    model="gemini-3-flash-preview",
    contents=[{"role": "user", "parts": [{"text": large_document}]}],
    system_instruction="You are a technical documentation expert.",
    ttl="1800s"
)

print(f"Cache created: {cache.name}")
```

### 2. Use the cache for queries

```python
model = genai.GenerativeModel.from_cached_content(cache)

# The large document tokens aren't re-billed on each request
response1 = model.generate_content("Summarize chapter 3")
response2 = model.generate_content("List the installation steps as bullet points")
response3 = model.generate_content("What's covered in the troubleshooting section?")
```

### 3. List and delete caches

```python
# List all caches
for c in genai.caching.CachedContent.list():
    print(f"{c.name}: expires {c.expire_time}")

# Delete manually
cache.delete()
```

## Best Use Cases

Context caching is most effective for:

**Document QA**: Chatbots that answer repeated questions about long PDFs or specs

**Codebase analysis**: Agents that load a large repository and run multiple analyses

**Legal/medical documents**: Analyzing long contracts or papers from multiple angles

**Data processing pipelines**: Processing many records using the same large prompt template

## Cost Example

Using Gemini 3 Flash with a 500,000-token document accessed 50 times:

| Method | Total input tokens | Estimated cost |
|--------|-------------------|----------------|
| No cache | 25,000,000 | $12.50 |
| With cache | 500,000 + small amounts | ~$1.00 |

The larger the document relative to the questions, the bigger the savings.

## TTL Configuration

Caches require a TTL (Time-To-Live). The range is 1 minute to 1 hour.

- **Short tasks**: 5–10 minutes is sufficient
- **Long-running processing**: Use the full 1-hour max
- **Regular reuse**: Schedule a script to recreate the cache as needed

Caches are auto-deleted after TTL expires. You can also extend the expiry via the API.
