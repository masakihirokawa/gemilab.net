# Building Type-Safe Applications with Gemini API Structured Output

One of the biggest challenges when integrating LLMs into applications is the unpredictability of their output. Parsing free-form text responses to feed into application logic is error-prone and fragile. Gemini API's Structured Output feature solves this by guaranteeing responses conform to a specified JSON Schema, making LLM integration reliable and type-safe.

## Structured Output Basics

Structured Output forces Gemini API responses to follow a defined JSON Schema. Simply specify `responseMimeType` and `responseSchema` in your generation config.

```typescript
import { GoogleGenerativeAI, SchemaType } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

// Define the response type
interface ProductReview {
  productName: string;
  rating: number;
  pros: string[];
  cons: string[];
  summary: string;
}

const model = genAI.getGenerativeModel({
  model: "gemini-2.5-flash",
  generationConfig: {
    responseMimeType: "application/json",
    responseSchema: {
      type: SchemaType.OBJECT,
      properties: {
        productName: { type: SchemaType.STRING, description: "Product name" },
        rating: { type: SchemaType.NUMBER, description: "Score from 1-5" },
        pros: {
          type: SchemaType.ARRAY,
          items: { type: SchemaType.STRING },
          description: "List of positive aspects"
        },
        cons: {
          type: SchemaType.ARRAY,
          items: { type: SchemaType.STRING },
          description: "List of areas for improvement"
        },
        summary: { type: SchemaType.STRING, description: "Overall evaluation summary" }
      },
      required: ["productName", "rating", "pros", "cons", "summary"]
    }
  }
});
```

With this configuration, API responses always conform to the specified JSON structure. No parsing errors to worry about, and it integrates seamlessly with TypeScript's type system.

## Classification Tasks with Enums

Structured Output pairs exceptionally well with classification tasks. The `enum` field constrains output to predefined values.

```typescript
const classifierModel = genAI.getGenerativeModel({
  model: "gemini-2.5-flash",
  generationConfig: {
    responseMimeType: "application/json",
    responseSchema: {
      type: SchemaType.OBJECT,
      properties: {
        category: {
          type: SchemaType.STRING,
          enum: ["bug", "feature", "question", "documentation"],
          description: "Issue category"
        },
        priority: {
          type: SchemaType.STRING,
          enum: ["low", "medium", "high", "critical"],
          description: "Priority level"
        },
        assignTeam: {
          type: SchemaType.STRING,
          enum: ["frontend", "backend", "infra", "design"],
          description: "Responsible team"
        }
      },
      required: ["category", "priority", "assignTeam"]
    }
  }
});

// Automatically classify issue text
const result = await classifierModel.generateContent(
  `Classify the following issue:\n\n${issueText}`
);
const classification = JSON.parse(result.response.text());
// classification.category is guaranteed to be one of the enum values
```

Using `enum` ensures the API only returns values from your predefined set, dramatically simplifying validation code on the application side.

## Working with Nested Objects

Complex data structures can be expressed through nested schema definitions. Here's an example for extracting structured article content.

```typescript
const contentSchema = {
  type: SchemaType.OBJECT,
  properties: {
    title: { type: SchemaType.STRING },
    sections: {
      type: SchemaType.ARRAY,
      items: {
        type: SchemaType.OBJECT,
        properties: {
          heading: { type: SchemaType.STRING },
          body: { type: SchemaType.STRING },
          codeExample: { type: SchemaType.STRING, nullable: true }
        },
        required: ["heading", "body"]
      }
    },
    metadata: {
      type: SchemaType.OBJECT,
      properties: {
        estimatedReadTime: { type: SchemaType.NUMBER },
        difficulty: {
          type: SchemaType.STRING,
          enum: ["beginner", "intermediate", "advanced"]
        },
        tags: {
          type: SchemaType.ARRAY,
          items: { type: SchemaType.STRING }
        }
      },
      required: ["estimatedReadTime", "difficulty", "tags"]
    }
  },
  required: ["title", "sections", "metadata"]
};
```

Use `nullable: true` to express optional fields that may or may not be present in the response.

## Validation and Error Handling

While Structured Output guarantees JSON structure, semantic correctness still needs application-level validation. Pairing with libraries like Zod is highly effective.

```typescript
import { z } from "zod";

const ReviewSchema = z.object({
  productName: z.string().min(1),
  rating: z.number().min(1).max(5),
  pros: z.array(z.string()).min(1),
  cons: z.array(z.string()),
  summary: z.string().max(500)
});

const raw = JSON.parse(result.response.text());
const validated = ReviewSchema.parse(raw);
// validated is fully type-safe
```

## Performance Best Practices

Key considerations for using Structured Output in production environments.

**Keep schemas simple**: Too many properties slows generation speed. Include only the fields you actually need.

**Leverage descriptions**: Adding `description` to each property helps Gemini understand the intent of each field, improving output quality significantly.

**Consider caching**: Structured outputs for identical inputs can be reused. Combining with Context Caching reduces both cost and latency.

**Use batch processing**: When classifying large volumes of data, the Batch API provides efficient processing at scale.

By leveraging Structured Output, you can seamlessly integrate Gemini API responses into your application's type system, building robust and maintainable code that bridges the gap between AI-generated content and type-safe application logic.
