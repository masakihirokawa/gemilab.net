## Introduction: Gemini API's Enhanced Cloud Storage Integration

In March 2026, Google significantly upgraded the Gemini API's file input capabilities. In addition to the existing **File API** (limited to 20MB), you can now specify **Google Cloud Storage (GCS) buckets** and **presigned URLs** directly as data sources. The file size limit has also been raised to **100MB**.

This update opens the door to new use cases, including:

- Analyzing large video files stored in GCS with Gemini
- Using presigned URLs from S3-compatible or Azure Blob Storage
- Building enterprise-scale pipelines with Vertex AI integration

This guide walks you through setting up Cloud Storage integration and provides practical code examples.

## How Cloud Storage Integration Works

### Comparing the Old File API vs. New Cloud Storage Integration

| Feature | Legacy File API | New Cloud Storage Integration |
|---------|----------------|-------------------------------|
| File size limit | 20MB | **100MB** |
| Storage | Google servers (temporary) | **GCS buckets / any presigned URL** |
| URL support | None | **GCS URIs & signed HTTPS URLs** |
| Persistence | Auto-deleted after 48 hours | Managed by your bucket |
| Cost | API transfer cost | GCS storage + transfer cost |

### Supported Input Formats

The new feature supports the following file reference formats:

1. **GCS URI** (e.g., `gs://my-bucket/my-video.mp4`)
2. **Public URL** (any directly accessible HTTPS file)
3. **Presigned URL** (a time-limited signed URL for private access)

## Implementation: Python with Cloud Storage

### Method 1: Directly Specify a GCS URI

The simplest approach — reference a GCS file directly in your request.

```python
import google.generativeai as genai
import os

# Configure API key
genai.configure(api_key=os.environ["GEMINI_API_KEY"])

# Use Gemini 3 Pro
model = genai.GenerativeModel("gemini-3-pro-preview")

# Provide a GCS URI (gs:// format)
gcs_uri = "gs://my-project-bucket/documents/annual_report_2025.pdf"

response = model.generate_content([
    {
        "file_data": {
            "mime_type": "application/pdf",
            "file_uri": gcs_uri  # Pass the GCS URI directly
        }
    },
    "Summarize the key financial metrics from this report in bullet points."
])

print(response.text)

# Expected output:
# - Revenue: +15% YoY ($12B)
# - Operating margin: 18.5% (+2.1pp YoY)
# - International revenue mix: 42% (+5pp YoY)
# - Capital expenditure: $800M (focused on growth areas)
```

### Method 2: Use a Presigned URL

Access files from AWS S3, Azure Blob Storage, or any other cloud storage service.

```python
import google.generativeai as genai
import os

genai.configure(api_key=os.environ["GEMINI_API_KEY"])
model = genai.GenerativeModel("gemini-3-flash-preview")

# AWS S3 presigned URL (temporary access URL)
presigned_url = "https://my-s3-bucket.s3.amazonaws.com/videos/product_demo.mp4?X-Amz-Signature=..."

response = model.generate_content([
    {
        "file_data": {
            "mime_type": "video/mp4",
            "file_uri": presigned_url  # Presigned URLs work the same way
        }
    },
    "Analyze the key features demonstrated in this product video and suggest improvements."
])

print(response.text)
```

### Method 3: Processing Large Files Up to 100MB

Take advantage of the new 100MB limit for large video or document analysis.

```python
import google.generativeai as genai
import os
from google.cloud import storage

def analyze_large_video_from_gcs(
    bucket_name: str,
    blob_name: str,
    prompt: str
) -> str:
    """
    Analyze a large video file (up to 100MB) from GCS using Gemini.

    Args:
        bucket_name: GCS bucket name
        blob_name: File path within the bucket
        prompt: Analysis instruction

    Returns:
        Gemini's analysis as a string
    """
    genai.configure(api_key=os.environ["GEMINI_API_KEY"])
    model = genai.GenerativeModel("gemini-3-pro-preview")

    # Build GCS URI
    gcs_uri = f"gs://{bucket_name}/{blob_name}"

    # Check file size (optional safeguard)
    storage_client = storage.Client()
    bucket = storage_client.bucket(bucket_name)
    blob = bucket.blob(blob_name)
    blob.reload()
    file_size_mb = blob.size / (1024 * 1024)

    if file_size_mb > 100:
        raise ValueError(f"File size {file_size_mb:.1f}MB exceeds the 100MB limit")

    print(f"📁 File size: {file_size_mb:.1f}MB")
    print(f"🚀 Analyzing with Gemini...")

    response = model.generate_content([
        {
            "file_data": {
                "mime_type": "video/mp4",
                "file_uri": gcs_uri
            }
        },
        prompt
    ])

    return response.text


# Usage
result = analyze_large_video_from_gcs(
    bucket_name="my-media-bucket",
    blob_name="recordings/webinar_2026_q1.mp4",
    prompt="Summarize the top 5 highlights from this webinar, including timestamps."
)
print(result)

# Expected output:
# [00:02:15] Introduction: 2026 AI trends overview
# [00:08:30] Gemini 3 feature demo (thinking mode, computer use tool)
# [00:22:45] Enterprise Vertex AI case studies
# [00:38:10] Q&A: Technical questions from developers
# [00:52:00] Wrap-up and next webinar announcement
```

## Implementation: JavaScript with Cloud Storage

Use GCS files with Gemini from a Node.js environment.

```javascript
import { GoogleGenerativeAI } from "@google/genai";

const genai = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

/**
 * Analyze a GCS file using the Gemini API
 * @param {string} gcsUri - gs://bucket/path or presigned HTTPS URL
 * @param {string} mimeType - MIME type of the file
 * @param {string} prompt - Analysis instruction
 * @returns {Promise<string>} Analysis result
 */
async function analyzeFileFromGCS(gcsUri, mimeType, prompt) {
  const model = genai.getGenerativeModel({ model: "gemini-3-flash-preview" });

  const result = await model.generateContent([
    {
      fileData: {
        mimeType: mimeType,
        fileUri: gcsUri, // gs:// URI or presigned HTTPS URL
      },
    },
    { text: prompt },
  ]);

  return result.response.text();
}

// Example: Analyze an image from GCS
const response = await analyzeFileFromGCS(
  "gs://product-images-bucket/catalog/item_001.jpg",
  "image/jpeg",
  "Describe this product image and write a 100-word e-commerce product description."
);

console.log(response);
// Expected output:
// This sleek aluminum laptop stand features a minimalist design with
// six adjustable angle settings. Built for durability, the heat-dissipating
// perforated surface keeps your laptop cool during extended sessions.
// Compatible with MacBooks and laptops up to 15 inches.

// Batch processing: analyze multiple files in parallel
async function batchAnalyzeFromGCS(files) {
  const promises = files.map(({ uri, mimeType, prompt }) =>
    analyzeFileFromGCS(uri, mimeType, prompt)
  );
  return Promise.all(promises);
}

const files = [
  {
    uri: "gs://reports-bucket/q1_2026.pdf",
    mimeType: "application/pdf",
    prompt: "Summarize the KPIs from this quarterly report in a table."
  },
  {
    uri: "gs://reports-bucket/q2_2026.pdf",
    mimeType: "application/pdf",
    prompt: "Summarize the KPIs from this quarterly report in a table."
  }
];

const results = await batchAnalyzeFromGCS(files);
results.forEach((result, i) => {
  console.log(`\n=== Q${i + 1} Report ===`);
  console.log(result);
});
```

## GCS Bucket Permissions Setup

For Gemini API to access a GCS bucket, you need to configure the appropriate IAM permissions.

### Setting Up a Service Account

```bash
# Create a service account
gcloud iam service-accounts create gemini-gcs-reader \
  --display-name="Gemini GCS Reader"

# Grant read access to the bucket
gcloud storage buckets add-iam-policy-binding gs://my-bucket \
  --member="serviceAccount:gemini-gcs-reader@PROJECT_ID.iam.gserviceaccount.com" \
  --role="roles/storage.objectViewer"

# Generate a key file (for local development)
gcloud iam service-accounts keys create key.json \
  --iam-account=gemini-gcs-reader@PROJECT_ID.iam.gserviceaccount.com
```

### Configure Authentication

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/key.json"
```

> **Using Vertex AI?** When accessing Gemini through Vertex AI, GCS authentication is handled automatically via Workload Identity. See the [Vertex AI Quickstart Guide](/en/articles/gemini-api/vertex-ai-quickstart) for details.

## Common Errors and Solutions

### Error 1: Permission Denied

```
PermissionDenied: 403 GET https://storage.googleapis.com/.../file.pdf:
ACCESS_DENIED
```

**Fix**: Ensure the service account used by Gemini API has the `storage.objectViewer` role on the bucket. Check the IAM policy in the Google Cloud Console.

### Error 2: File Size Exceeded

```
InvalidArgument: 400 File size exceeds the limit of 100MB
```

**Fix**: The file exceeds the 100MB limit. For videos, reduce resolution or split into segments. For PDFs, extract only the relevant pages.

### Error 3: Unsupported MIME Type

```
InvalidArgument: Unsupported MIME type
```

**Fix**: Make sure the `mime_type` parameter is correct. Common MIME types:

| File Type | MIME Type |
|-----------|-----------|
| PDF | `application/pdf` |
| MP4 Video | `video/mp4` |
| JPEG Image | `image/jpeg` |
| PNG Image | `image/png` |
| MP3 Audio | `audio/mp3` |
| WAV Audio | `audio/wav` |
| Plain Text | `text/plain` |

### Error 4: Presigned URL Expired

```
InvalidArgument: URL has expired
```

**Fix**: Check the expiration time of your presigned URL. Account for Gemini's processing time (typically 1–3 minutes) and set an expiration of at least 30 minutes.

## Advanced: Enterprise-Scale Pipeline

A scalable document processing pipeline combining GCS and Gemini via Cloud Functions.

```python
import google.generativeai as genai
from google.cloud import storage, pubsub_v1
import json
import os

def process_document_pipeline(event, context):
    """
    A document processing pipeline triggered by Cloud Functions.
    Automatically analyzes files uploaded to GCS with Gemini.

    Triggered by a GCS object finalize event.
    """
    genai.configure(api_key=os.environ["GEMINI_API_KEY"])
    model = genai.GenerativeModel("gemini-3-flash-preview")

    # Extract GCS file info from the event
    bucket_name = event["bucket"]
    file_name = event["name"]
    mime_type = event.get("contentType", "application/octet-stream")

    gcs_uri = f"gs://{bucket_name}/{file_name}"
    print(f"📄 Processing: {gcs_uri}")

    # Choose prompt based on file type
    prompts = {
        "application/pdf": "Summarize this document and extract key information in JSON format.",
        "image/jpeg": "Describe the contents of this image and return relevant tags as a JSON array.",
        "image/png": "Describe the contents of this image and return relevant tags as a JSON array.",
        "video/mp4": "List the scenes in this video with timestamps in JSON format."
    }

    prompt = prompts.get(mime_type, "Summarize the contents of this file.")

    # Analyze with Gemini
    response = model.generate_content(
        [{"file_data": {"mime_type": mime_type, "file_uri": gcs_uri}}, prompt],
        generation_config={"response_mime_type": "application/json"}
    )

    result = {
        "file": gcs_uri,
        "analysis": json.loads(response.text)
    }

    # Publish result to Pub/Sub for downstream processing
    publisher = pubsub_v1.PublisherClient()
    topic_path = publisher.topic_path(
        os.environ["GOOGLE_CLOUD_PROJECT"],
        "gemini-analysis-results"
    )
    publisher.publish(topic_path, json.dumps(result).encode())

    print(f"✅ Analysis complete: {file_name}")
    return result
```

This pattern is useful for automating tasks like:

- Contract and invoice data extraction
- Automatic product image tagging for e-commerce
- Meeting recording transcription and summarization
- Security camera footage anomaly detection

## File API vs. Cloud Storage: When to Use Each

| Use Case | Recommended Approach |
|----------|---------------------|
| Temporary files under 20MB | File API |
| Large files from 20MB to 100MB | **Cloud Storage integration (new)** |
| Systems integrated with GCP | **Direct GCS URI** |
| Files on AWS S3 or Azure Blob | **Presigned URL** |
| Files to reuse across requests | GCS + URI reference |
| Prototyping and testing | File API |

## FAQ

### Q1. Do I need Vertex AI to use Cloud Storage integration?

**A.** No. GCS URIs work with both the Gemini Developer API (API key) and Vertex AI. However, Vertex AI is recommended for enterprise use cases as it integrates seamlessly with Google Cloud's IAM and Workload Identity.

### Q2. What are the security implications of using presigned URLs?

**A.** The Gemini API server fetches the file from the presigned URL. Keep expiration times short (30 minutes to 1 hour is recommended) and revoke URLs after use. Always store your API key in environment variables or Secret Manager — never hardcode it.

### Q3. How do I handle files larger than 100MB?

**A.** Files exceeding 100MB cannot be processed directly. For videos, use FFmpeg to split or reduce resolution. For PDFs, extract only the relevant pages. For very large datasets, consider extracting keyframes from videos for multimodal processing, or use Vertex AI's Long Context capabilities.

### Q4. Can Gemini access private GCS buckets?

**A.** Yes. Grant the `storage.objectViewer` role to the service account used by the Gemini API. When using Vertex AI, Workload Identity provides a more secure, keyless authentication approach.

### Q5. Which file formats are supported?

**A.** Supported formats include PDFs, images (JPEG, PNG, WebP, HEIC), videos (MP4, MOV, AVI, etc.), audio (MP3, WAV, FLAC, etc.), and text files (TXT, HTML, Markdown, CSV, etc.). Note that video and audio processing requires a multimodal-capable model such as Gemini 3 Pro or Gemini 3 Flash.
