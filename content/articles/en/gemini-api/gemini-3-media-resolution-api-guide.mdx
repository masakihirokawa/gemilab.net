## What Is media_resolution? A New Lever for Multimodal AI

Gemini 3 introduces the `media_resolution` parameter, giving developers granular control over how images and videos are processed. By adjusting this setting, you can fine-tune the balance between token consumption, latency, and recognition accuracy to match your specific use case.

In the Gemini 2.5 series, the model automatically determined resolution when you sent an image. With Gemini 3, you can now explicitly choose from four levels: `media_resolution_low`, `media_resolution_medium`, `media_resolution_high`, and `media_resolution_ultra_high` (Flash only).

This means you can use low resolution for quick thumbnail classification while reserving high resolution for OCR or detailed chart analysis — all within the same application.

## Resolution Levels and Token Consumption

Each resolution level allocates a specific number of tokens per image:

| Resolution Level | Tokens (per image) | Supported Models |
|---|---|---|
| `media_resolution_low` | ~280 tokens | Pro / Flash |
| `media_resolution_medium` | ~560 tokens | Pro / Flash |
| `media_resolution_high` | ~1,120 tokens | Pro / Flash |
| `media_resolution_ultra_high` | ~2,240 tokens | Flash only |

Token consumption scales roughly linearly with resolution for images. **Video frames are compressed more aggressively**, making video processing even more cost-efficient.

### A Quick Cost Comparison

Consider a batch job classifying 100 product images:

- **Low resolution**: 100 × 280 = 28,000 tokens
- **High resolution**: 100 × 1,120 = 112,000 tokens

Choosing low resolution **cuts token usage by 75%**. For straightforward classification tasks where fine details aren't critical, low resolution often delivers perfectly adequate accuracy.

## Getting Started: Setting a Default with GenerationConfig

The simplest approach is to set a default resolution for all media in your request using `GenerationConfig`:

```python
import google.generativeai as genai
from google.generativeai.types import GenerationConfig

# Configure your API key
genai.configure(api_key="YOUR_API_KEY")

# Initialize the model with medium resolution as default
model = genai.GenerativeModel(
    model_name="gemini-3-pro",
    generation_config=GenerationConfig(
        media_resolution="media_resolution_medium"
    )
)

# Run inference with an image
image = genai.upload_file("product_photo.jpg")
response = model.generate_content([
    "List three key features of this product.",
    image
])

print(response.text)
# Expected output:
# 1. Compact design that's easy to carry
# 2. Matte black finish with a premium feel
# 3. USB-C port with fast charging support
```

With this configuration, every image and video in your request gets processed at `media_resolution_medium` (~560 tokens per image).

## Per-Part Resolution: Mix and Match in a Single Request

One of Gemini 3's standout features is the ability to **set different resolution levels for individual parts within a single request**. This lets you allocate high resolution to critical images while keeping supplementary ones lightweight.

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro")

# Process the detailed diagram at high resolution
diagram = genai.upload_file("architecture_diagram.png")
diagram_part = genai.types.Part(
    file_data=diagram,
    media_resolution="media_resolution_high"
)

# Process the reference screenshot at low resolution
screenshot = genai.upload_file("app_screenshot.png")
screenshot_part = genai.types.Part(
    file_data=screenshot,
    media_resolution="media_resolution_low"
)

response = model.generate_content([
    "Compare these two images. The first is a system architecture diagram, the second is an app screenshot.",
    diagram_part,
    screenshot_part
])

print(response.text)
# Expected output:
# The architecture diagram shows a frontend connecting through an API gateway
# to a microservices cluster. The app screenshot displays a user dashboard
# with metrics and navigation elements...
```

In this example, the diagram (1,120 tokens) + screenshot (280 tokens) = **1,400 total tokens**. Compared to using high resolution for both (2,240 tokens), that's roughly a 37% reduction.

## Working with Video

The `media_resolution` parameter works with video as well. Since video frames are compressed more aggressively than still images, the cost savings can be even more significant.

```python
import google.generativeai as genai
from google.generativeai.types import GenerationConfig

genai.configure(api_key="YOUR_API_KEY")

model = genai.GenerativeModel(
    model_name="gemini-3-flash",
    generation_config=GenerationConfig(
        media_resolution="media_resolution_low"
    )
)

# Upload a video file
video = genai.upload_file("meeting_recording.mp4")

# Summarize the video content
response = model.generate_content([
    "Summarize the key topics and conclusions from this meeting recording.",
    video
])

print(response.text)
# Expected output:
# - Topic 1: Q2 revenue targets — agreed on 15% YoY growth
# - Topic 2: New product launch schedule — confirmed June release milestone
# - Topic 3: Customer support improvements — chatbot pilot starting next month
```

For meeting summaries and similar tasks where visual details aren't critical, `media_resolution_low` works perfectly well.

## Recommended Settings by Use Case

Here's a practical guide to choosing the right resolution level for common scenarios.

### Low Resolution (`media_resolution_low`)

- Broad image classification (landscape / people / food)
- Thumbnail filtering and sorting
- Video scene splitting and summarization
- Social media image sentiment analysis

### Medium Resolution (`media_resolution_medium`)

- Product image description generation
- UI/UX design review
- Presentation content comprehension
- General document image reading

### High Resolution (`media_resolution_high`)

- OCR (text recognition from documents, business cards)
- Technical drawing and blueprint analysis
- Medical image preliminary screening
- Extracting data from detailed tables and charts

### Ultra High Resolution (`media_resolution_ultra_high` — Flash only)

- High-resolution map and satellite imagery analysis
- Micro-defect detection in manufacturing QA
- OCR on extremely small fonts

## Pro vs. Flash: Feature Comparison

The two model families have slightly different `media_resolution` capabilities:

| Feature | Gemini 3 Pro | Gemini 3 Flash |
|---|---|---|
| low / medium / high | ✅ | ✅ |
| ultra_high | ❌ | ✅ |
| Per-part resolution | ✅ | ✅ |
| GenerationConfig default | ✅ | ✅ |

Flash supports `ultra_high`, making it the go-to choice when maximum visual fidelity is required. Pro excels at language understanding and reasoning, so it's better suited for complex analysis based on extracted visual information.

## Pre-Check Costs with the Token Counting API

In production, you can use the `count_tokens` API to estimate costs before sending a request:

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-pro")

image = genai.upload_file("document_scan.jpg")

# Compare token counts across resolution levels
for resolution in ["media_resolution_low", "media_resolution_medium", "media_resolution_high"]:
    part = genai.types.Part(
        file_data=image,
        media_resolution=resolution
    )
    token_count = model.count_tokens([
        "Summarize the contents of this document.",
        part
    ])
    print(f"{resolution}: {token_count.total_tokens} tokens")

# Expected output:
# media_resolution_low: 295 tokens
# media_resolution_medium: 575 tokens
# media_resolution_high: 1135 tokens
```

This approach lets you estimate costs precisely before committing to a request. For more on token management, see our [Gemini Token Counting Guide](/en/articles/gemini-api/gemini-token-counting).

## Frequently Asked Questions

### Q1: What happens if I don't specify media_resolution?

When you don't explicitly set `media_resolution`, Gemini 3 uses the model's default processing level. This is generally equivalent to `media_resolution_medium`, though the exact behavior may vary based on input image size and content. If you need precise cost control, always specify the resolution explicitly.

### Q2: Can I set different resolutions for individual frames in a video?

No — a single `media_resolution` value applies to the entire video file. You can't control resolution on a per-frame basis. However, if specific scenes need higher fidelity analysis, you can extract key frames as images and process them individually at higher resolution.

### Q3: Why isn't media_resolution_ultra_high available on Pro?

As of March 2026, `media_resolution_ultra_high` is exclusive to Gemini 3 Flash. Attempting to use it with a Pro model will return an error. If you need ultra-high resolution, use Flash for the visual processing step, then pass the extracted information to Pro for deeper reasoning — a two-stage pipeline approach.

### Q4: Does media_resolution work with Gemini 2.5 models?

Setting a default resolution through `GenerationConfig` works with all multimodal models, including the Gemini 2.5 series. However, per-part resolution control is a Gemini 3-exclusive feature.
