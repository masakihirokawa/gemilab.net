## Introduction

Response speed can make or break an AI application's user experience. While the Gemini API provides powerful language models, latency can become a concern depending on the physical distance between your users and servers.

This is where **Cloudflare Workers** shines. With code execution across 300+ edge locations worldwide, you can serve responses from the server closest to each user. Combined with the Gemini API, you can build globally distributed, low-latency AI APIs that feel instant.

In this guide, we'll walk through building an edge AI API on Cloudflare Workers that leverages the Gemini API — from initial setup to production deployment.

**What you'll learn:**

- Basic patterns for calling the Gemini API from Cloudflare Workers
- Caching strategies using KV Storage to reduce costs and latency
- How to implement streaming responses
- Error handling and rate limiting strategies
- Security best practices for production deployments

**Who this is for:** Developers familiar with the Gemini API basics who want to explore (or already use) Cloudflare Workers for deployment.

### A Basic API Endpoint

Here's the simplest possible implementation for calling the Gemini API from a Worker:

```typescript
// src/index.ts
export interface Env {
  GEMINI_API_KEY: string;
  GEMINI_MODEL: string;
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    // Handle CORS preflight requests
    if (request.method === "OPTIONS") {
      return new Response(null, {
        headers: {
          "Access-Control-Allow-Origin": "*",
          "Access-Control-Allow-Methods": "POST, OPTIONS",
          "Access-Control-Allow-Headers": "Content-Type",
        },
      });
    }

    if (request.method !== "POST") {
      return new Response(
        JSON.stringify({ error: "Method not allowed" }),
        { status: 405, headers: { "Content-Type": "application/json" } }
      );
    }

    try {
      const { prompt } = await request.json() as { prompt: string };

      if (!prompt || typeof prompt !== "string") {
        return new Response(
          JSON.stringify({ error: "Invalid prompt" }),
          { status: 400, headers: { "Content-Type": "application/json" } }
        );
      }

      // Call the Gemini API
      const geminiResponse = await fetch(
        `https://generativelanguage.googleapis.com/v1beta/models/${env.GEMINI_MODEL}:generateContent?key=${env.GEMINI_API_KEY}`,
        {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({
            contents: [
              { role: "user", parts: [{ text: prompt }] }
            ],
            generationConfig: {
              temperature: 0.7,
              maxOutputTokens: 1024,
            },
          }),
        }
      );

      const data = await geminiResponse.json();

      // Expected output:
      // {
      //   "response": "Response text from Gemini...",
      //   "model": "gemini-3-flash",
      //   "edge_location": "NRT" (if served from Tokyo)
      // }
      return new Response(
        JSON.stringify({
          response: data.candidates?.[0]?.content?.parts?.[0]?.text ?? "",
          model: env.GEMINI_MODEL,
          edge_location: request.cf?.colo ?? "unknown",
        }),
        {
          headers: {
            "Content-Type": "application/json",
            "Access-Control-Allow-Origin": "*",
          },
        }
      );
    } catch (error) {
      return new Response(
        JSON.stringify({ error: "Internal server error" }),
        { status: 500, headers: { "Content-Type": "application/json" } }
      );
    }
  },
};
```

The `request.cf?.colo` property reveals which edge location processed the request, confirming that responses are being served from the closest server to each user.

## Caching AI Responses with KV Storage

Calling the Gemini API for every identical prompt is wasteful. By caching responses in Cloudflare KV, you can dramatically cut costs and improve response times.

### Creating and Configuring a KV Namespace

```bash
# Create a KV Namespace

# Add the output ID to your wrangler.toml
```

```toml
# Append to wrangler.toml

binding = "CACHE"
id = "your-namespace-id-here"
```

### Implementing Cached API Calls

```typescript
// src/index.ts (cache-enabled version)
import { createHash } from "node:crypto";

export interface Env {
  GEMINI_API_KEY: string;
  GEMINI_MODEL: string;
  CACHE: KVNamespace;
}

// Generate a cache key from the prompt
function generateCacheKey(prompt: string, model: string): string {
  const hash = createHash("sha256")
    .update(`${model}:${prompt}`)
    .digest("hex")
    .slice(0, 16);
  return `gemini:${hash}`;
}

async function callGeminiAPI(
  prompt: string,
  env: Env
): Promise<string> {
  const response = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/${env.GEMINI_MODEL}:generateContent?key=${env.GEMINI_API_KEY}`,
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        contents: [{ role: "user", parts: [{ text: prompt }] }],
        generationConfig: {
          temperature: 0.7,
          maxOutputTokens: 1024,
        },
      }),
    }
  );

  const data = await response.json();
  return data.candidates?.[0]?.content?.parts?.[0]?.text ?? "";
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    if (request.method !== "POST") {
      return new Response("Method not allowed", { status: 405 });
    }

    const { prompt, skipCache } = await request.json() as {
      prompt: string;
      skipCache?: boolean;
    };

    const cacheKey = generateCacheKey(prompt, env.GEMINI_MODEL);

    // Check the cache first
    if (!skipCache) {
      const cached = await env.CACHE.get(cacheKey);
      if (cached) {
        return new Response(
          JSON.stringify({
            response: cached,
            cached: true,
            edge_location: request.cf?.colo,
          }),
          { headers: { "Content-Type": "application/json" } }
        );
      }
    }

    // Call the Gemini API
    const result = await callGeminiAPI(prompt, env);

    // Cache the response in KV (TTL: 1 hour)
    await env.CACHE.put(cacheKey, result, { expirationTtl: 3600 });

    // Expected output:
    // { "response": "...", "cached": false, "edge_location": "NRT" }
    return new Response(
      JSON.stringify({
        response: result,
        cached: false,
        edge_location: request.cf?.colo,
      }),
      { headers: { "Content-Type": "application/json" } }
    );
  },
};
```

On cache hits, the response is returned instantly without calling the Gemini API at all, significantly reducing both latency and costs.

## Implementing Streaming Responses

For longer outputs, you don't want users staring at a loading spinner. Streaming lets them see results as they're generated. Cloudflare Workers natively supports `ReadableStream`, making it a natural fit for Gemini API streaming.

```typescript
// src/streaming.ts
async function streamGeminiResponse(
  prompt: string,
  env: Env
): Promise<ReadableStream> {
  const response = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/${env.GEMINI_MODEL}:streamGenerateContent?alt=sse&key=${env.GEMINI_API_KEY}`,
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        contents: [{ role: "user", parts: [{ text: prompt }] }],
        generationConfig: {
          temperature: 0.7,
          maxOutputTokens: 2048,
        },
      }),
    }
  );

  if (!response.ok || !response.body) {
    throw new Error(`Gemini API error: ${response.status}`);
  }

  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  return new ReadableStream({
    async pull(controller) {
      const { done, value } = await reader.read();

      if (done) {
        controller.enqueue(
          new TextEncoder().encode("data: [DONE]\n\n")
        );
        controller.close();
        return;
      }

      const chunk = decoder.decode(value, { stream: true });
      // Forward the SSE-formatted chunks as-is
      controller.enqueue(new TextEncoder().encode(chunk));
    },
    cancel() {
      reader.cancel();
    },
  });
}

// Streaming endpoint handler
// GET /stream?prompt=...
export async function handleStream(
  request: Request,
  env: Env
): Promise<Response> {
  const url = new URL(request.url);
  const prompt = url.searchParams.get("prompt");

  if (!prompt) {
    return new Response("Missing prompt", { status: 400 });
  }

  const stream = await streamGeminiResponse(prompt, env);

  return new Response(stream, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      "Connection": "keep-alive",
      "Access-Control-Allow-Origin": "*",
    },
  });
}
```

## Error Handling and Rate Limiting

Production deployments need robust error handling and rate limiting to stay reliable under load.

### Retry with Exponential Backoff

```typescript
// src/utils/retry.ts
async function fetchWithRetry(
  url: string,
  options: RequestInit,
  maxRetries: number = 3
): Promise<Response> {
  let lastError: Error | null = null;

  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      const response = await fetch(url, options);

      // Retry on 429 (Rate Limit) or 503 (Service Unavailable)
      if (response.status === 429 || response.status === 503) {
        const retryAfter = response.headers.get("Retry-After");
        const delay = retryAfter
          ? parseInt(retryAfter) * 1000
          : Math.min(1000 * Math.pow(2, attempt), 10000);

        console.log(
          `Rate limited. Retrying in ${delay}ms (attempt ${attempt + 1})`
        );
        await new Promise((r) => setTimeout(r, delay));
        continue;
      }

      return response;
    } catch (error) {
      lastError = error as Error;
      const delay = Math.min(1000 * Math.pow(2, attempt), 10000);
      await new Promise((r) => setTimeout(r, delay));
    }
  }

  throw lastError ?? new Error("Max retries exceeded");
}

// Usage:
// const response = await fetchWithRetry(geminiUrl, requestOptions);
// Expected behavior:
// - 1st failure → wait 1s → 2nd failure → wait 2s → 3rd attempt succeeds
```

### Distributed Rate Limiting

While Cloudflare Durable Objects offer true global rate limiting, a simpler KV-based approach works well for most use cases:

```typescript
// src/utils/rate-limit.ts
async function checkRateLimit(
  clientIP: string,
  kv: KVNamespace,
  limit: number = 60,
  window: number = 60
): Promise<{ allowed: boolean; remaining: number }> {
  const key = `ratelimit:${clientIP}`;
  const current = parseInt((await kv.get(key)) ?? "0");

  if (current >= limit) {
    return { allowed: false, remaining: 0 };
  }

  await kv.put(key, String(current + 1), {
    expirationTtl: window,
  });

  return { allowed: true, remaining: limit - current - 1 };
}
```

## Frequently Asked Questions (FAQ)

### Q1: Can I use the Gemini API with Cloudflare Workers' free plan?

Yes! The Cloudflare Workers free tier supports up to 100,000 requests per day with 10ms of CPU time per request. Since Gemini API calls are external fetch requests, they barely consume CPU time. KV reads and writes do use some CPU time, but the free tier is more than enough for personal projects and prototypes.

### Q2: What should I keep in mind when caching Gemini API responses?

Adjust your cache TTL (time-to-live) based on the nature of the prompts. For time-sensitive topics or current events, use shorter TTLs (minutes to tens of minutes). For evergreen knowledge queries, longer TTLs (hours to a full day) work well. For personalized responses, either skip caching entirely or include a user identifier in the cache key.

### Q3: How can I combine streaming with caching?

A practical pattern is to check the cache first — if there's a hit, return the complete cached response immediately. On a cache miss, stream the response to the client while simultaneously buffering the full output. Once streaming completes, write the buffered result to KV for future requests.

### Q4: What kind of latency can I expect when calling the Gemini API from Cloudflare Workers?

Cloudflare Workers have near-zero cold start times, but the total latency depends on the Gemini model and prompt length. With Gemini 3 Flash and short prompts, expect around 500ms to 1.5 seconds. The edge deployment typically shaves 50–200ms off the total round-trip compared to routing through a traditional origin server.
