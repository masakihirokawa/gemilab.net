The Gemini API includes two powerful features: streaming responses for real-time output delivery, and Function Calling for integrating external tools and APIs. By combining these capabilities, you can build sophisticated AI applications that are both interactive and connected to external data sources. This guide covers both features from fundamentals to practical implementation.

## Streaming Responses: The Basics

With standard API requests, the model generates the entire response before sending it back. Streaming delivers text incrementally as it's generated, dramatically improving the user experience.

### Streaming in Python

```python
import google.generativeai as genai

# Configure your API key
genai.configure(api_key="YOUR_API_KEY")

# Initialize the model
model = genai.GenerativeModel("gemini-2.5-pro")

# Receive streaming response
response = model.generate_content(
    "Explain how to build a web scraper in Python, step by step.",
    stream=True
)

# Output each chunk in real time
for chunk in response:
    if chunk.text:
        print(chunk.text, end="", flush=True)

print()  # Final newline

# Expected behavior:
# Text appears in real time (token by token)
# No waiting for the complete response to generate
```

### Streaming in TypeScript/JavaScript

```typescript
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI("YOUR_API_KEY");
const model = genAI.getGenerativeModel({ model: "gemini-2.5-pro" });

async function streamResponse(prompt: string) {
  const result = await model.generateContentStream(prompt);
  
  // Process chunks as they arrive
  for await (const chunk of result.stream) {
    const text = chunk.text();
    process.stdout.write(text);
  }
  
  // You can also get the complete final response
  const finalResponse = await result.response;
  console.log("\n\nTotal tokens:", finalResponse.usageMetadata?.totalTokenCount);
}

// Usage
streamResponse("What are the best practices for designing REST APIs in TypeScript?");

// Expected output:
// (Text streams in real time)
// Total tokens: 1234
```

## How Function Calling Works

Function Calling lets you teach Gemini about available tools (functions) so the model can request their invocation when needed. The model doesn't execute code itself — instead, it returns a request saying "call this function with these arguments," and your application handles the actual execution.

### Implementing Function Calling

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

# 1. Define functions (tools for the model)
def get_weather(city: str, unit: str = "celsius") -> dict:
    """Get weather information for a specified city"""
    weather_data = {
        "Tokyo": {"temp": 18, "condition": "Sunny", "humidity": 45},
        "London": {"temp": 12, "condition": "Cloudy", "humidity": 70},
        "New York": {"temp": 15, "condition": "Rainy", "humidity": 75},
    }
    data = weather_data.get(city, {"temp": 15, "condition": "Unknown", "humidity": 50})
    if unit == "fahrenheit":
        data["temp"] = data["temp"] * 9/5 + 32
    return {"city": city, **data}

def search_restaurants(location: str, cuisine: str = "any", budget: str = "moderate") -> dict:
    """Search for restaurants in a specified location"""
    return {
        "location": location,
        "cuisine": cuisine,
        "results": [
            {"name": f"{cuisine} Restaurant A", "rating": 4.5, "budget": budget},
            {"name": f"{cuisine} Restaurant B", "rating": 4.2, "budget": budget},
        ]
    }

# 2. Pass tool definitions to the model
tools = [get_weather, search_restaurants]

model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    tools=tools
)

# 3. Start a chat session
chat = model.start_chat()

# 4. Send a user query
response = chat.send_message(
    "What's the weather like in Tokyo right now, and can you recommend some Italian restaurants?"
)

# 5. Process Function Call requests
for part in response.parts:
    if hasattr(part, 'function_call'):
        func_name = part.function_call.name
        func_args = dict(part.function_call.args)
        
        # Execute the function
        if func_name == "get_weather":
            result = get_weather(**func_args)
        elif func_name == "search_restaurants":
            result = search_restaurants(**func_args)
        
        # Return the result to the model
        response = chat.send_message(
            genai.protos.Content(
                parts=[genai.protos.Part(
                    function_response=genai.protos.FunctionResponse(
                        name=func_name,
                        response={"result": result}
                    )
                )]
            )
        )

print(response.text)

# Expected output:
# The weather in Tokyo is currently sunny at 18°C with 45% humidity.
# 
# Here are two recommended Italian restaurants:
# 1. Italian Restaurant A (Rating: 4.5)
# 2. Italian Restaurant B (Rating: 4.2)
```

## Combining Streaming with Function Calling

By combining streaming and Function Calling, you can build advanced applications that respond in real time while also fetching external data.

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

model = genai.GenerativeModel(
    model_name="gemini-2.5-pro",
    tools=[get_weather]
)

# Streaming + Function Calling
response = model.generate_content(
    "Help me plan a picnic for tomorrow. Check Tokyo's weather and create an appropriate packing list.",
    stream=True
)

for chunk in response:
    # Handle Function Call requests in stream
    for part in chunk.parts:
        if hasattr(part, 'function_call'):
            result = get_weather(city="Tokyo")
            print(f"\n[Weather data fetched: {result}]\n")
    
    # Handle text responses
    if chunk.text:
        print(chunk.text, end="", flush=True)
```

## Frequently Asked Questions

### Does streaming cost more than regular requests?

No, streaming and standard requests consume the same number of tokens. Since billing is based on token count, streaming adds no extra cost. Streaming only changes how the response is delivered — the model's processing workload remains identical.

### Can Function Calling invoke multiple functions simultaneously?

Yes, the Gemini API supports parallel Function Calling. For a request like "Compare the weather in Tokyo and London," the model may request `get_weather("Tokyo")` and `get_weather("London")` simultaneously. Your client code handles both executions and returns all results together.

### Are there security risks with Function Calling?

Function Calling itself doesn't execute code — the model simply returns a "call request." Your client code performs the actual function execution, which means input validation, authorization checks, and rate limiting are your responsibility. Always sanitize function arguments and implement proper access controls.

### What's the difference between Gemini 2.5 Pro and Flash?

Gemini 2.5 Pro offers stronger reasoning capabilities and a longer context window, making it ideal for complex Function Calling chains. Gemini 2.5 Flash prioritizes speed and is better suited for simple streaming chats. Flash is also more cost-effective, so choose based on your use case.

## Wrapping Up

Gemini API's streaming and Function Calling capabilities are powerful building blocks for real-time AI applications. Streaming improves the user experience with instant feedback, while Function Calling seamlessly connects your AI to external data and tools.

Start with a simple streaming chat, then layer in Function Calling to integrate external APIs like weather services or news feeds. This incremental approach is the most reliable way to build sophisticated AI-powered applications.
