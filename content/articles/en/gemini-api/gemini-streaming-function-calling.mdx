The Gemini 3 release brought two significant upgrades to Function Calling that open the door to a new class of AI applications:

1. **Streaming Function Calling** — receive function arguments in real time as the model generates them
2. **Multimodal Function Responses** — pass images, PDFs, and other media back to the model as function results

This guide walks through both features with complete code examples in Python and JavaScript. If you're new to Function Calling in general, start with the [Function Calling Basics Guide](/en/articles/gemini-api/function-calling-guide) first.

---

## What Is Streaming Function Calling?

In earlier versions of the Gemini API, function arguments were delivered all at once — the model finished generating the entire JSON payload before anything arrived in your app. For complex arguments or longer reasoning steps, this meant a noticeable wait with no feedback for the user.

With **Streaming Function Calling**, setting `streamFunctionCallArguments: true` causes argument tokens to flow in incrementally, just like streaming text responses. This lets you:

- Update progress indicators or loading states as arguments arrive
- Provide immediate feedback that the model is "thinking"
- Handle large argument payloads (like nested JSON objects) without awkward delays

This feature requires **Gemini 3 Pro** or **Gemini 3 Flash** (or later models).

---

## What Are Multimodal Function Responses?

Previously, function responses could only contain text. With Gemini 3, you can include **images, PDFs, and other binary data** directly in the `functionResponse` parts you send back to the model.

This unlocks real-world scenarios like:

- A user asks "analyze this chart" → the model calls a data-fetching function → the function returns a PNG chart image → the model visually interprets it and explains the trends
- Retrieve a PDF quarterly report → pass it as a function response → the model reads and summarizes it in context
- Take a screenshot in a computer-use agent → return the image → the model decides the next UI action

Together, these features make it possible to build agents that genuinely perceive the world, not just process text.

## Streaming Function Calling — Python Example

The following example defines a weather tool and streams the function call arguments in real time.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Define the tool
get_weather_tool = types.Tool(
    function_declarations=[
        types.FunctionDeclaration(
            name="get_weather",
            description="Get the current weather for a given city",
            parameters=types.Schema(
                type=types.Type.OBJECT,
                properties={
                    "city": types.Schema(
                        type=types.Type.STRING,
                        description="City name, e.g. Tokyo, New York"
                    ),
                    "units": types.Schema(
                        type=types.Type.STRING,
                        enum=["celsius", "fahrenheit"],
                        description="Temperature unit"
                    )
                },
                required=["city"]
            )
        )
    ]
)

# Enable streaming function call arguments
config = types.GenerateContentConfig(
    tools=[get_weather_tool],
    function_calling_config=types.FunctionCallingConfig(
        mode=types.FunctionCallingMode.AUTO,
        stream_function_call_arguments=True  # ← key setting
    )
)

print("Fetching weather info...")
for chunk in client.models.generate_content_stream(
    model="gemini-3-flash-preview",
    contents="What's the weather in Tokyo right now in Celsius?",
    config=config
):
    parts = chunk.candidates[0].content.parts
    for part in parts:
        if hasattr(part, "function_call") and part.function_call:
            fc = part.function_call
            print(f"Function call detected: {fc.name}")
            print(f"  Args so far: {fc.args}")
        if hasattr(part, "text") and part.text:
            print(part.text, end="", flush=True)

# Expected output:
# Fetching weather info...
# Function call detected: get_weather
#   Args so far: {'city': 'Tokyo', 'units': 'celsius'}
```

## Streaming Function Calling — JavaScript Example

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

const getWeatherTool = {
  functionDeclarations: [
    {
      name: "get_weather",
      description: "Get the current weather for a given city",
      parameters: {
        type: "object",
        properties: {
          city: { type: "string", description: "City name" },
          units: {
            type: "string",
            enum: ["celsius", "fahrenheit"],
            description: "Temperature unit",
          },
        },
        required: ["city"],
      },
    },
  ],
};

async function streamWeather() {
  const stream = await ai.models.generateContentStream({
    model: "gemini-3-flash-preview",
    contents: [
      { role: "user", parts: [{ text: "What's the weather in Osaka in Celsius?" }] },
    ],
    config: {
      tools: [getWeatherTool],
      functionCallingConfig: {
        mode: "AUTO",
        streamFunctionCallArguments: true, // ← enable streaming
      },
    },
  });

  for await (const chunk of stream) {
    const parts = chunk.candidates?.[0]?.content?.parts ?? [];
    for (const part of parts) {
      if (part.functionCall) {
        console.log(`Function: ${part.functionCall.name}`);
        console.log(`Args:`, part.functionCall.args);
      }
      if (part.text) process.stdout.write(part.text);
    }
  }
}

streamWeather();
```

## Multimodal Function Responses — Passing an Image

The key pattern is to send the `functionResponse` part and the image `inline_data` part together in the same `Content` block.

```python
import base64
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Define a screenshot tool
screenshot_tool = types.Tool(
    function_declarations=[
        types.FunctionDeclaration(
            name="take_screenshot",
            description="Capture the current screen and return it as an image",
            parameters=types.Schema(
                type=types.Type.OBJECT,
                properties={
                    "region": types.Schema(
                        type=types.Type.STRING,
                        enum=["full", "active_window"],
                        description="Area to capture"
                    )
                },
                required=["region"]
            )
        )
    ]
)

# Step 1: Ask the model to take a screenshot
response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="Take a full screenshot and tell me what apps are open",
    config=types.GenerateContentConfig(tools=[screenshot_tool])
)

function_call = response.candidates[0].content.parts[0].function_call
print(f"Called: {function_call.name}, args: {function_call.args}")

# Step 2: Capture the real screenshot (using pyautogui or Pillow in production)
# Here we simulate with a local file
with open("screenshot.png", "rb") as f:
    image_b64 = base64.b64encode(f.read()).decode()

# Step 3: Return the screenshot image as a multimodal function response
final = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents=[
        types.Content(
            role="user",
            parts=[types.Part(text="Take a full screenshot and tell me what apps are open")]
        ),
        response.candidates[0].content,  # model's function call turn
        types.Content(
            role="user",
            parts=[
                # Function response metadata
                types.Part(
                    function_response=types.FunctionResponse(
                        name="take_screenshot",
                        response={"region": "full", "status": "ok"}
                    )
                ),
                # The actual image data alongside the response
                types.Part(
                    inline_data=types.Blob(
                        mime_type="image/png",
                        data=image_b64
                    )
                )
            ]
        )
    ]
)

print("Model analysis:")
print(final.text)
# Expected output:
# Model analysis:
# I can see the following applications are open:
# - Terminal running a Python script
# - Visual Studio Code with main.py
# - Chrome browser showing Google AI Studio
```

## Multimodal Function Responses — Passing a PDF

```python
import base64
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

report_tool = types.Tool(
    function_declarations=[
        types.FunctionDeclaration(
            name="get_report",
            description="Retrieve a quarterly business report as a PDF",
            parameters=types.Schema(
                type=types.Type.OBJECT,
                properties={
                    "quarter": types.Schema(type=types.Type.STRING, description="Quarter ID, e.g. 2026Q1")
                },
                required=["quarter"]
            )
        )
    ]
)

response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="Summarize the Q1 2026 business report",
    config=types.GenerateContentConfig(tools=[report_tool])
)

fc = response.candidates[0].content.parts[0].function_call

with open("2026Q1_report.pdf", "rb") as f:
    pdf_b64 = base64.b64encode(f.read()).decode()

final = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents=[
        types.Content(role="user", parts=[types.Part(text="Summarize the Q1 2026 business report")]),
        response.candidates[0].content,
        types.Content(
            role="user",
            parts=[
                types.Part(
                    function_response=types.FunctionResponse(
                        name="get_report",
                        response={"quarter": "2026Q1", "found": True}
                    )
                ),
                types.Part(
                    inline_data=types.Blob(mime_type="application/pdf", data=pdf_b64)
                )
            ]
        )
    ]
)

print(final.text)
```

## Common Errors and Fixes

### Error: `streamFunctionCallArguments` has no effect

**Cause**: The model you're using is Gemini 2.x, which doesn't support this parameter.

**Fix**: Switch to a Gemini 3 model.

```python
# ❌ Not supported on Gemini 2.x
model = "gemini-2.5-flash"

# ✅ Works on Gemini 3
model = "gemini-3-flash-preview"
```

### Error: `Invalid function response` when sending images

**Cause**: The image data and function response are not in the same `Content` block as separate `parts`.

**Fix**: Put both in the same `parts` list inside one `Content(role="user", ...)`.

```python
# ✅ Correct: both parts in the same Content
parts=[
    types.Part(function_response=types.FunctionResponse(...)),
    types.Part(inline_data=types.Blob(mime_type="image/png", data=...))
]
```

### Error: Function call `args` is `None` mid-stream

**Cause**: Early chunks may not yet contain any argument tokens.

**Fix**: Always check for `None` before processing.

```python
for chunk in stream:
    for part in chunk.candidates[0].content.parts:
        if hasattr(part, "function_call") and part.function_call:
            if part.function_call.args:  # guard against None
                print(part.function_call.args)
```

## FAQ

**Q: Which models support streaming function call arguments?**

A: Gemini 3 Pro and Gemini 3 Flash (preview and stable releases) support this feature. Gemini 2.x models do not. The setting works the same way through both the Google AI API and Vertex AI.

**Q: What MIME types can I use in multimodal function responses?**

A: Currently supported: `image/png`, `image/jpeg`, `image/webp`, `image/gif`, and `application/pdf`. Video and audio files are not supported directly inside `functionResponse` — use the File API to upload large files and reference them via `fileData` instead.

**Q: Do I need to wait for the full stream before calling the actual function?**

A: It depends on your use case. If the function needs the complete arguments (most cases), wait until the stream ends. However, if your function can begin processing based on partial arguments (for example, starting a network request as soon as you have a URL), you can act earlier. Always validate that required fields are present before calling downstream services.
