Voice interfaces are no longer optional for premium AI products. **Gemini Live API** delivers bidirectional audio streaming with sub-500ms latency, and Google's **Agent Development Kit (ADK)**—now GA—makes it straightforward to wire up tools like calendar lookup, weather APIs, and database search to natural voice commands.

This guide takes you from zero to a deployed, tool-capable voice assistant, with architecture decisions explained and working Python code you can run immediately.

## Component Architecture

| Component | Responsibility |
|-----------|---------------|
| **Gemini Live API** | Real-time bidirectional audio streaming, VAD, speech synthesis |
| **Agent Development Kit (ADK)** | Agent lifecycle, tool definitions, multi-turn conversation state |
| **Cloud Run** | Scalable WebSocket server hosting |

```
[Browser / Mobile Client]
       │ WebSocket (binary audio + JSON commands)
       ↓
[Cloud Run: FastAPI + WebSocket Server]
       │ google-genai Live API session
       ↓
[Gemini Live API (Gemini 3.0 Flash)]
       │ ADK tool calls
       ├── [Calendar API]
       ├── [Weather API]
       └── [Internal Search]
```

## Core Live API Session

```python
# voice_assistant/session.py
import asyncio
import logging
from typing import AsyncGenerator

from google import genai
from google.genai import types

logger = logging.getLogger(__name__)

LIVE_MODEL = "gemini-3.0-flash-preview"

SYSTEM_INSTRUCTION = """
You are a helpful voice AI assistant. Keep your answers concise and conversational.
When tools are available, use them proactively to provide accurate, real-time information.
"""

class LiveVoiceSession:
    """Manages a bidirectional audio session with Gemini Live API."""

    def __init__(self, api_key: str):
        self.client = genai.Client(api_key=api_key, http_options={"api_version": "v1alpha"})
        self.session = None

    async def connect(self, tools: list | None = None) -> None:
        config = types.LiveConnectConfig(
            response_modalities=["AUDIO", "TEXT"],
            speech_config=types.SpeechConfig(
                voice_config=types.VoiceConfig(
                    prebuilt_voice_config=types.PrebuiltVoiceConfig(voice_name="Kore")
                )
            ),
            system_instruction=types.Content(
                parts=[types.Part(text=SYSTEM_INSTRUCTION)]
            ),
            tools=tools or [],
        )

        self.session = await self.client.aio.live.connect(
            model=LIVE_MODEL, config=config
        )
        logger.info("🎤 Live API session started")

    async def stream_audio_input(self, audio_chunk: bytes) -> None:
        """Forward a PCM audio chunk from the client to Live API."""
        if not self.session:
            raise RuntimeError("Not connected. Call connect() first.")

        await self.session.send(
            input=types.LiveClientRealtimeInput(
                media_chunks=[
                    types.Blob(data=audio_chunk, mime_type="audio/pcm;rate=16000")
                ]
            )
        )

    async def receive_responses(self) -> AsyncGenerator[dict, None]:
        """
        Async generator that yields structured response events.

        Yields:
            {"type": "audio", "data": bytes}     — PCM audio output
            {"type": "text", "data": str}        — Text transcript
            {"type": "tool_call", "data": dict}  — Tool call request
            {"type": "turn_complete"}             — End of turn signal
        """
        async for response in self.session.receive():
            if response.server_content:
                content = response.server_content

                if content.model_turn:
                    for part in content.model_turn.parts:
                        if part.inline_data:
                            yield {"type": "audio", "data": part.inline_data.data}
                        elif part.text:
                            yield {"type": "text", "data": part.text}

                if content.turn_complete:
                    yield {"type": "turn_complete"}

            elif response.tool_call:
                fc = response.tool_call.function_calls[0]
                yield {"type": "tool_call", "data": {
                    "id": fc.id,
                    "name": fc.name,
                    "args": dict(fc.args),
                }}

    async def send_tool_result(self, call_id: str, result: str) -> None:
        await self.session.send(
            input=types.LiveClientToolResponse(
                function_responses=[
                    types.FunctionResponse(id=call_id, response={"result": result})
                ]
            )
        )

    async def close(self) -> None:
        if self.session:
            await self.session.close()
```

## Defining ADK Tools

```python
# voice_assistant/tools.py
import httpx, os
from google.adk.tools import FunctionTool
from datetime import datetime, timedelta

async def get_current_weather(location: str, unit: str = "celsius") -> str:
    """
    Get the current weather for a specified location.

    Args:
        location: Location name (e.g., "Tokyo", "New York, US")
        unit: Temperature unit — "celsius" or "fahrenheit"

    Returns:
        Weather summary string
    """
    async with httpx.AsyncClient(timeout=5.0) as client:
        try:
            resp = await client.get(
                "https://api.openweathermap.org/data/2.5/weather",
                params={
                    "q": location,
                    "appid": os.getenv("OPENWEATHER_API_KEY"),
                    "units": "metric" if unit == "celsius" else "imperial",
                }
            )
            resp.raise_for_status()
            data = resp.json()
            temp = data["main"]["temp"]
            desc = data["weather"][0]["description"]
            unit_symbol = "°C" if unit == "celsius" else "°F"
            return f"{location}: {desc}, {temp:.1f}{unit_symbol}"
        except Exception as e:
            return f"Could not fetch weather for {location}: {str(e)}"

async def get_calendar_events(date_str: str = "today") -> str:
    """
    Retrieve calendar events for a given date.

    Args:
        date_str: Date as "today", "tomorrow", or "YYYY-MM-DD"

    Returns:
        Formatted list of events
    """
    if date_str == "today":
        target = datetime.now()
    elif date_str == "tomorrow":
        target = datetime.now() + timedelta(days=1)
    else:
        target = datetime.strptime(date_str, "%Y-%m-%d")

    # Replace with Google Calendar API call in production
    mock_events = [
        {"time": "10:00 AM", "title": "Team standup"},
        {"time": "2:00 PM", "title": "Project review"},
        {"time": "4:30 PM", "title": "1:1 with manager"},
    ]

    date_label = target.strftime("%B %d, %Y")
    events_str = "\n".join([f"  {e['time']}: {e['title']}" for e in mock_events])
    return f"Events on {date_label}:\n{events_str}"

# Example interaction:
# User: "What's the weather like in Tokyo?"
# → get_current_weather(location="Tokyo") is called
# → Returns: "Tokyo: clear sky, 18.5°C"
# → AI responds naturally in speech: "It's currently clear and 18 degrees Celsius in Tokyo."

weather_tool = FunctionTool(func=get_current_weather)
calendar_tool = FunctionTool(func=get_calendar_events)
AVAILABLE_TOOLS = [weather_tool, calendar_tool]
```

## FastAPI WebSocket Server

```python
# server.py
import asyncio, json, os, base64, logging
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.middleware.cors import CORSMiddleware
from voice_assistant.session import LiveVoiceSession
from voice_assistant.tools import AVAILABLE_TOOLS

app = FastAPI(title="Gemini Voice Assistant")
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])
logger = logging.getLogger(__name__)

@app.websocket("/ws/voice")
async def voice_endpoint(websocket: WebSocket):
    """
    WebSocket endpoint for voice interaction.

    Client sends:
    - Binary frames: PCM audio at 16kHz, 16-bit, mono

    Server sends JSON:
    - {"type": "audio", "data": "<base64 PCM>"}
    - {"type": "text", "data": "..."}
    - {"type": "turn_complete"}
    - {"type": "error", "message": "..."}
    """
    await websocket.accept()
    session = LiveVoiceSession(api_key=os.getenv("GOOGLE_API_KEY"))

    try:
        live_tools = [{"function_declarations": [t.get_declaration() for t in AVAILABLE_TOOLS]}]
        await session.connect(tools=live_tools)

        receive_task = asyncio.create_task(_receive_audio(websocket, session))
        send_task = asyncio.create_task(_send_responses(websocket, session))
        await asyncio.gather(receive_task, send_task)

    except WebSocketDisconnect:
        logger.info("Client disconnected")
    except Exception as e:
        logger.error(f"Session error: {e}")
        await websocket.send_json({"type": "error", "message": str(e)})
    finally:
        await session.close()

async def _receive_audio(websocket: WebSocket, session: LiveVoiceSession):
    async for chunk in websocket.iter_bytes():
        await session.stream_audio_input(chunk)

async def _send_responses(websocket: WebSocket, session: LiveVoiceSession):
    async for response in session.receive_responses():
        if response["type"] == "audio":
            await websocket.send_json({
                "type": "audio",
                "data": base64.b64encode(response["data"]).decode(),
            })
        elif response["type"] == "tool_call":
            result = await _execute_tool(response["data"]["name"], response["data"]["args"])
            await session.send_tool_result(response["data"]["id"], result)
        else:
            await websocket.send_json(response)

async def _execute_tool(name: str, args: dict) -> str:
    tool_map = {t.func.__name__: t.func for t in AVAILABLE_TOOLS}
    return await tool_map[name](**args) if name in tool_map else f"Tool '{name}' not found"
```

> **Cost tip**: Set `--min-instances 1` to eliminate cold start latency for real-time voice applications. The always-on cost for a single 1 vCPU instance is roughly $30/month—much less than the UX cost of a 3-second cold start on a voice product.

## Frequently Asked Questions

**Q1. What latency should I expect in production?**

Using Gemini 3.0 Flash in `us-central1`, expect 300–500ms from end of user speech to first audio output token. Tool calls add 200–800ms depending on the external API. To minimize latency: (1) deploy in the region closest to your users, (2) set `--min-instances 1` to avoid cold starts, and (3) implement strict 5-second timeouts on all tool functions so slow APIs don't freeze the conversation.

**Q2. How do you handle concurrent users at scale?**

Cloud Run's WebSocket support handles stateful sessions naturally. Each WebSocket connection maps to one Live API session, and Cloud Run auto-scales to additional instances as load increases. A `--max-instances 20` setting with 1 vCPU per instance can handle 80+ concurrent voice sessions. For enterprise workloads, consider deploying to GKE for finer-grained resource control.

**Q3. What languages does Live API support for voice input?**

As of early 2026, Live API supports 24 languages for speech recognition including English, Japanese, Spanish, French, German, Korean, and Mandarin Chinese. Text output in all supported languages is fully available. For voice output (TTS), English prebuilt voices are GA, with Japanese and other languages in beta. The most tested pattern is accepting Japanese speech input and responding with Japanese text while using the English TTS voice.

## Wrapping Up

Key takeaways for building production voice AI with Gemini Live API and ADK:

1. **Bidirectional streaming over WebSockets** — the only architecture that delivers sub-500ms latency for voice.
2. **ADK FunctionTool** — the cleanest way to connect real-world actions (calendars, APIs, databases) to voice commands.
3. **Cloud Run for WebSockets** — simple, scalable, and cost-effective for stateful real-time sessions.
4. **Always-on minimum instances** — non-negotiable for voice UX; cold starts destroy the conversational flow.
