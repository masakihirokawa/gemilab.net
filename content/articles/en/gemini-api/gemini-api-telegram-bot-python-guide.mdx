## Introduction

Telegram boasts over 900 million monthly active users worldwide, and its powerful Bot API makes it one of the most developer-friendly messaging platforms available. By combining Telegram's Bot API with the Gemini API's advanced AI capabilities, you can build a fully featured AI assistant that handles not just text responses, but also image recognition and document analysis.

In this guide, we'll walk through building a multimodal Telegram bot from scratch using Python's `python-telegram-bot` library and Google's Gemini API. Even if you've already built bots for Discord, LINE, or Slack, Telegram offers unique features like inline mode, rich Markdown support, and generous file size limits that open up new possibilities.

**What you'll learn:**

- Setting up a Telegram bot with BotFather and configuring the Bot API
- Implementing text conversations with multi-turn context using the Gemini API
- Adding AI-powered image analysis for photos sent by users
- Handling errors and rate limits gracefully in production

## Prerequisites and Setup

### What You'll Need

- Python 3.10 or higher
- A Google AI Studio API key ([see our quickstart guide](/en/articles/gemini-api/gemini-api-quickstart))

First, message [@BotFather](https://t.me/BotFather) on Telegram to create your bot.

1. Send `/newbot` to BotFather
2. Enter a display name for your bot (e.g., `My Gemini Assistant`)
3. Enter a username for your bot (e.g., `my_gemini_assistant_bot` — must end with `bot`)
4. Save the **API token** you receive securely

## Building a Basic Text Response Bot

Let's start with a simple bot that responds to user messages using the Gemini API.

```python
import os
import logging
from dotenv import load_dotenv
from google import genai
from telegram import Update
from telegram.ext import (
    ApplicationBuilder,
    CommandHandler,
    MessageHandler,
    ContextTypes,
    filters,
)

# Load environment variables
load_dotenv()

# Configure logging
logging.basicConfig(
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s",
    level=logging.INFO,
)
logger = logging.getLogger(__name__)

# Initialize the Gemini client
client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))

async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    Handler for the /start command
    """
    await update.message.reply_text(
        "Hello! I'm a Gemini AI assistant.\n"
        "Send me text or images and I'll analyze and respond.\n\n"
        "Commands:\n"
        "/start - About this bot\n"
        "/reset - Clear conversation history"
    )

async def reset(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    Clear conversation history with /reset
    """
    context.user_data.pop("history", None)
    await update.message.reply_text("Conversation history cleared.")

async def handle_message(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    Respond to text messages using the Gemini API
    """
    user_message = update.message.text

    # Manage conversation history (max 20 turns)
    history = context.user_data.get("history", [])
    history.append({"role": "user", "parts": [{"text": user_message}]})

    # Trim history to prevent excessive token usage
    if len(history) > 40:
        history = history[-40:]

    try:
        # Call the Gemini API
        response = client.models.generate_content(
            model="gemini-3-flash",
            contents=history,
            config={
                "system_instruction": (
                    "You are a helpful and knowledgeable AI assistant. "
                    "Respond in the same language the user writes in. "
                    "Keep your answers concise and accurate."
                ),
                "max_output_tokens": 2048,
                "temperature": 0.7,
            },
        )

        reply = response.text

        # Add assistant response to history
        history.append({"role": "model", "parts": [{"text": reply}]})
        context.user_data["history"] = history

        # Handle Telegram's 4096 character message limit
        if len(reply) > 4000:
            for i in range(0, len(reply), 4000):
                await update.message.reply_text(reply[i : i + 4000])
        else:
            await update.message.reply_text(reply)

    except Exception as e:
        logger.error(f"Gemini API error: {e}")
        await update.message.reply_text(
            "Sorry, an error occurred. Please try again in a moment."
        )

def main():
    """
    Launch the bot
    """
    app = (
        ApplicationBuilder()
        .token(os.getenv("TELEGRAM_BOT_TOKEN"))
        .build()
    )

    app.add_handler(CommandHandler("start", start))
    app.add_handler(CommandHandler("reset", reset))
    app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))

    logger.info("Bot is running...")
    app.run_polling()

if __name__ == "__main__":
    main()
```

Once running, simply send a message to your bot on Telegram and it will respond using the Gemini API. Conversation history is maintained automatically, enabling contextual multi-turn conversations.

## Going Multimodal — Adding Image Analysis

One of the Gemini API's greatest strengths is its multimodal capability. Let's add image analysis so users can send photos for the AI to interpret.

```python
import io
import httpx

async def handle_photo(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    Receive and analyze images with Gemini
    """
    # Get the highest resolution version of the photo
    photo = update.message.photo[-1]
    file = await context.bot.get_file(photo.file_id)

    # Download the image data
    async with httpx.AsyncClient() as http_client:
        response = await http_client.get(file.file_path)
        image_bytes = response.content

    # Use the caption as the user's question, or fall back to a default
    caption = update.message.caption or "Please describe this image in detail."

    try:
        await update.message.reply_text("Analyzing your image...")

        # Send multimodal request to the Gemini API
        response = client.models.generate_content(
            model="gemini-3-flash",
            contents=[
                {
                    "role": "user",
                    "parts": [
                        {"text": caption},
                        {
                            "inline_data": {
                                "mime_type": "image/jpeg",
                                "data": __import__("base64").b64encode(
                                    image_bytes
                                ).decode(),
                            }
                        },
                    ],
                }
            ],
            config={
                "max_output_tokens": 2048,
                "temperature": 0.5,
            },
        )

        await update.message.reply_text(response.text)

    except Exception as e:
        logger.error(f"Image analysis error: {e}")
        await update.message.reply_text(
            "An error occurred while analyzing the image. Please try a different one."
        )
```

Add the following handler to your `main()` function:

```python
app.add_handler(MessageHandler(filters.PHOTO, handle_photo))
```

Now when users send a photo, Gemini will analyze its contents and respond. They can add a caption like "How many calories are in this dish?" or "How do I fix this error?" for more targeted analysis.

## Adding Document Analysis

You can also let users send PDFs and text files for AI-powered analysis.

```python
async def handle_document(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """
    Receive and analyze documents (PDF, text) with Gemini
    """
    document = update.message.document

    # Supported MIME types
    supported_types = {
        "application/pdf": "application/pdf",
        "text/plain": "text/plain",
        "text/csv": "text/csv",
    }

    mime_type = document.mime_type
    if mime_type not in supported_types:
        await update.message.reply_text(
            "Unsupported file format. Please send a PDF, text, or CSV file."
        )
        return

    # File size check (20MB limit)
    if document.file_size > 20 * 1024 * 1024:
        await update.message.reply_text(
            "File is too large (limit: 20MB)."
        )
        return

    file = await context.bot.get_file(document.file_id)

    async with httpx.AsyncClient() as http_client:
        response = await http_client.get(file.file_path)
        file_bytes = response.content

    caption = update.message.caption or "Please summarize this document."

    try:
        await update.message.reply_text("Analyzing your document...")

        import base64
        response = client.models.generate_content(
            model="gemini-3-flash",
            contents=[
                {
                    "role": "user",
                    "parts": [
                        {"text": caption},
                        {
                            "inline_data": {
                                "mime_type": mime_type,
                                "data": base64.b64encode(file_bytes).decode(),
                            }
                        },
                    ],
                }
            ],
            config={
                "max_output_tokens": 4096,
                "temperature": 0.3,
            },
        )

        reply = response.text
        if len(reply) > 4000:
            for i in range(0, len(reply), 4000):
                await update.message.reply_text(reply[i : i + 4000])
        else:
            await update.message.reply_text(reply)

    except Exception as e:
        logger.error(f"Document analysis error: {e}")
        await update.message.reply_text(
            "An error occurred while analyzing the document."
        )
```

Add the handler to `main()`:

```python
app.add_handler(MessageHandler(filters.Document.ALL, handle_document))
```

## Error Handling and Rate Limiting

For production deployments, robust error handling is essential. Here's a retry wrapper with exponential backoff for handling API rate limits and transient failures.

```python
import asyncio
from google.api_core.exceptions import ResourceExhausted, ServiceUnavailable

async def call_gemini_with_retry(contents, config, max_retries=3):
    """
    Call the Gemini API with automatic retry on rate limits
    """
    for attempt in range(max_retries):
        try:
            response = client.models.generate_content(
                model="gemini-3-flash",
                contents=contents,
                config=config,
            )
            return response

        except ResourceExhausted:
            # Rate limited — exponential backoff
            wait_time = 2 ** attempt
            logger.warning(
                f"Rate limited. Retrying in {wait_time}s "
                f"(attempt {attempt + 1}/{max_retries})"
            )
            await asyncio.sleep(wait_time)

        except ServiceUnavailable:
            # Service temporarily down — retry
            wait_time = 3 ** attempt
            logger.warning(
                f"Service unavailable. Retrying in {wait_time}s "
                f"(attempt {attempt + 1}/{max_retries})"
            )
            await asyncio.sleep(wait_time)

    raise Exception("Maximum retries exceeded")

#     contents=history,
#     config={"max_output_tokens": 2048, "temperature": 0.7},
# )
```

## Deployment Tips

### Running as a systemd Service (Linux)

```ini
# /etc/systemd/system/gemini-telegram-bot.service
[Unit]
Description=Gemini Telegram Bot
After=network.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/opt/gemini-telegram-bot
ExecStart=/opt/gemini-telegram-bot/venv/bin/python bot.py
Restart=always
RestartSec=10
EnvironmentFile=/opt/gemini-telegram-bot/.env

[Install]
WantedBy=multi-user.target
```

### Running with Docker

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
```

## Frequently Asked Questions

### Q: Are there usage limits for the Telegram Bot API?

The Telegram Bot API itself is completely free to use. There are no per-message costs — only rate limits of 30 messages per second globally and 1 message per second per chat. The only costs you'll incur are from Gemini API token usage.

### Q: How do I use the bot in group chats?

Run the `/setprivacy` command with BotFather and set privacy mode to `Disable`. This lets your bot receive all messages in a group. However, for group usage it's recommended to design the bot to only respond when mentioned with `@your_bot_name` to avoid noise.

### Q: How does this compare to building bots for Discord or LINE?

Telegram's Bot API stands out with its generous 4,096-character message limit, rich Markdown formatting, and inline keyboard support. It also supports both Webhook and Polling modes, giving you more deployment flexibility. For comparisons, see our guides on building a [Discord bot with Gemini](/en/articles/gemini-api/gemini-api-discord-bot-python-guide) and a [LINE bot with Gemini](/en/articles/gemini-api/gemini-api-line-bot-python-guide).

### Q: Should I switch to Webhook mode for production?

Polling mode is convenient during development, but Webhook mode is recommended for production. With webhooks, your server only processes incoming messages when they arrive, reducing resource usage and improving response times. You'll need an HTTPS endpoint, which can be easily set up using Cloudflare Tunnel or ngrok.
