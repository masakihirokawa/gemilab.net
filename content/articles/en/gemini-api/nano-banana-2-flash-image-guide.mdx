Google's **Nano Banana 2 (Gemini 3.1 Flash Image Preview)**, released in February 2026, brings Pro-level 4K image generation to the Flash architecture. It delivers high-quality images at Flash-tier speed and cost, making it ideal for developers and creators who need fast, affordable image generation at scale.

In this guide, we'll cover Nano Banana 2's key features, walk through API implementation in both Python and JavaScript, explore pricing, and share practical code examples you can use right away.

## What Is Nano Banana 2? Key Features

Nano Banana 2 (model ID: `gemini-3.1-flash-image-preview`) is the high-efficiency counterpart to Gemini 3 Pro Image, optimized for speed and high-volume use cases.

**High-Resolution Output**: Choose from 512px (0.5K) up to 4K resolution depending on your needs. The default is 1K, and you can scale up to 2K or 4K when higher quality is required.

**14 Aspect Ratios**: The model supports 1:1, 3:2, 16:9, 9:16, 21:9, and more. Whether you need square social media images or ultrawide banners, Nano Banana 2 has you covered.

**Accurate Text Rendering**: Generate legible, well-placed text within images for marketing mockups, greeting cards, and branded visuals.

**Character Consistency**: Maintain consistency for up to 5 characters and faithfully reproduce up to 14 objects within a single workflow. This is particularly useful for storytelling and visual series.

**Search Grounding**: Nano Banana 2 pulls from Google Search and image search results in real time to accurately depict specific subjects and themes.

## Generating Images with Python

Here's a basic text-to-image generation example:

```python
from google import genai
from google.genai import types
import base64

# Initialize client
client = genai.Client(api_key="YOUR_API_KEY")

# Generate image from text
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents="A serene Japanese garden with a koi pond at golden hour, photorealistic",
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        image_config=types.ImageConfig(
            aspect_ratio="16:9",
            image_size="2K"
        )
    )
)

# Save the generated image
for part in response.candidates[0].content.parts:
    if part.inline_data is not None:
        image_data = base64.b64decode(part.inline_data.data)
        with open("generated_image.png", "wb") as f:
            f.write(image_data)
        print("Image saved: generated_image.png")
    elif part.text is not None:
        print(f"Text: {part.text}")

# Expected output:
# Text: Here is a serene Japanese garden with a koi pond at golden hour.
# Image saved: generated_image.png
```

The key detail here is including `"IMAGE"` in `response_modalities`. This tells the API to return image data alongside any text response.

## Generating Images with JavaScript

Here's the Node.js implementation:

```javascript
import { GoogleGenAI } from "@google/genai";
import fs from "fs";

const ai = new GoogleGenAI({ apiKey: "YOUR_API_KEY" });

async function generateImage() {
  const response = await ai.models.generateContent({
    model: "gemini-3.1-flash-image-preview",
    contents: "A futuristic Tokyo cityscape at night with neon lights, photorealistic",
    config: {
      responseModalities: ["TEXT", "IMAGE"],
      imageConfig: {
        aspectRatio: "21:9",
        imageSize: "4K"
      }
    }
  });

  // Extract and save the image
  for (const part of response.candidates[0].content.parts) {
    if (part.inlineData) {
      const buffer = Buffer.from(part.inlineData.data, "base64");
      fs.writeFileSync("output.png", buffer);
      console.log("Image saved: output.png");
    } else if (part.text) {
      console.log("Text:", part.text);
    }
  }
}

generateImage();

// Expected output:
// Text: Here is a futuristic Tokyo cityscape at night with neon lights.
// Image saved: output.png
```

This example uses the 21:9 ultrawide aspect ratio at 4K resolution, perfect for hero images and banners.

## Image Editing with Multi-Turn Conversations

Nano Banana 2 supports conversational image editing, letting you iteratively refine generated images.

```python
from google import genai
from google.genai import types
import base64

client = genai.Client(api_key="YOUR_API_KEY")

# Load an existing image
with open("original.png", "rb") as f:
    image_bytes = f.read()

# Send an editing instruction
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[
        types.Content(
            parts=[
                types.Part(
                    inline_data=types.Blob(
                        mime_type="image/png",
                        data=base64.b64encode(image_bytes).decode()
                    )
                ),
                types.Part(text="Change the background to a snowy winter scene")
            ]
        )
    ],
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        image_config=types.ImageConfig(
            image_size="2K"
        )
    )
)

# Save the edited image
for part in response.candidates[0].content.parts:
    if part.inline_data is not None:
        edited = base64.b64decode(part.inline_data.data)
        with open("edited_image.png", "wb") as f:
            f.write(edited)
        print("Edited image saved")

# Expected output:
# Edited image saved
```

You can supply up to 10 reference images as input, making it possible to pass multiple style references and generate images with a consistent visual identity.

## Using Thinking Mode for Better Results

Nano Banana 2 supports configurable thinking levels to balance quality and speed.

```python
# Generate with high thinking level for complex compositions
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents="A detailed architectural blueprint of a modern sustainable house",
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        thinking_config=types.ThinkingConfig(
            thinking_level="HIGH",       # "MINIMAL" or "HIGH"
            include_thoughts=True        # Show thinking process
        ),
        image_config=types.ImageConfig(
            image_size="2K"
        )
    )
)

# Inspect thinking output
for part in response.candidates[0].content.parts:
    if hasattr(part, "thought") and part.thought:
        print(f"Thought: {part.text}")
    elif part.text:
        print(f"Response: {part.text}")

# Expected output:
# Thought: The user wants an architectural blueprint style...
# Response: Here is a detailed architectural blueprint of a modern sustainable house.
```

Setting `thinking_level` to `"HIGH"` improves output quality for complex prompts with detailed compositions. Note that thinking tokens are billed regardless of the `include_thoughts` setting.

## Common Errors and Troubleshooting

**`SAFETY_BLOCKED` error**: The safety filter blocked the generation. Adjust your prompt to avoid violent, sexual, or otherwise restricted content.

**`INVALID_ARGUMENT` for image_size**: You specified an unsupported resolution. Use one of `"0.5K"`, `"1K"`, `"2K"`, or `"4K"`.

**`RATE_LIMIT_EXCEEDED`**: You've hit the rate limit. Add delays between requests or switch to the Batch API for high-volume processing.

## FAQ

### What's the difference between Nano Banana 2 and Gemini 3 Pro Image?

Nano Banana 2 is built on the Flash architecture, making it roughly 2x faster than Pro Image while costing 37–63% less. Image quality approaches Pro-level output, though Pro may have a slight edge in extremely complex compositions and lighting. Nano Banana 2 is the better choice for high-volume, speed-critical workloads.

### What about licensing for generated images?

All images generated through the Gemini API include a SynthID digital watermark. Review Google's terms of service for commercial use guidelines. Personal development and prototyping are generally permitted.

### Can I use it with Function Calling or the Live API?

The current preview version does not support Function Calling, Live API, or Structured Outputs. Nano Banana 2 is focused on text-and-image input/output. Additional capabilities may be added in future updates.

### Is there a free tier?

Yes, you can try Nano Banana 2 through Google AI Studio's free tier, though rate limits apply. For production use at scale, you'll need to upgrade to a paid plan.
