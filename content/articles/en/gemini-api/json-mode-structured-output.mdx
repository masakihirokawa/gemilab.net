## Why JSON Mode Matters

Ever tried parsing LLM output and hit a broken format error? Gemini API's **JSON Mode** and **Structured Output** force the model to return JSON that always matches your specified schema — dramatically improving reliability in backend integrations and data pipelines.

## Basic JSON Mode

The simplest approach is setting `response_mime_type` to `application/json`:

```python
import google.generativeai as genai
import json

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3-flash-preview")

response = model.generate_content(
    "Extract this product info as JSON: iPhone 15 Pro, price $999, color Black, storage 256GB",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json"
    )
)

data = json.loads(response.text)
print(data)
# {"name": "iPhone 15 Pro", "price": 999, "color": "Black", "storage": "256GB"}
```

## Structured Output with Schema

For stricter control, provide a `response_schema`:

```python
import typing_extensions as typing

class Product(typing.TypedDict):
    name: str
    price: int
    color: str
    storage: str
    available: bool

response = model.generate_content(
    "Extract product info: iPhone 15 Pro, $999, Black, 256GB, in stock",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=Product
    )
)

data = json.loads(response.text)
# Always conforms to the Product schema
```

## Returning Arrays

For multiple items, use list schemas:

```python
class Review(typing.TypedDict):
    reviewer: str
    rating: int
    comment: str
    date: str

response = model.generate_content(
    """Extract all reviews from this text:
    John (5 stars): Great product, 2026/3/10
    Jane (3 stars): It's okay, 2026/3/12""",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=list[Review]
    )
)

reviews = json.loads(response.text)
for r in reviews:
    print(f"{r['reviewer']}: {r['rating']} stars - {r['comment']}")
```

## Nested Structures

Complex nested schemas work too:

```python
class Address(typing.TypedDict):
    street: str
    city: str
    postal_code: str

class Customer(typing.TypedDict):
    id: int
    name: str
    email: str
    address: Address
    is_premium: bool
```

## Using Enums

For categorical outputs, use `enum` to restrict allowed values:

```python
import enum

class Sentiment(enum.Enum):
    POSITIVE = "positive"
    NEGATIVE = "negative"
    NEUTRAL = "neutral"

class SentimentResult(typing.TypedDict):
    text: str
    sentiment: Sentiment
    confidence: float
```

## Common Use Cases

- **Information extraction**: Pulling structured data from emails or documents
- **Classification**: Categorizing text with guaranteed valid categories
- **Data transformation**: Unstructured text → database-ready JSON
- **API response generation**: Seamless backend API integration
- **Form data parsing**: Structuring free-text answers

## Summary

JSON Mode and Structured Output make Gemini API outputs predictable and reliable.

- `response_mime_type: "application/json"` enables basic JSON output
- `response_schema` enforces strict schema compliance
- Works with Python TypedDict and Enum for type-safe outputs

For any backend or data pipeline integration, structured output should be your default approach.
