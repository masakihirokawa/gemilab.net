## Introduction

Demand for AI-powered backend services is surging. Gemini API offers a versatile set of capabilities — from text generation and image recognition to Function Calling — and when paired with Node.js, it enables you to spin up lightweight, scalable AI microservices remarkably quickly.

In this guide, we'll walk through building a production-ready AI microservice using Google's Gemini API and Express.js. We'll cover streaming responses, rate limiting, robust error handling, and deployment patterns — everything you need to go from prototype to production.

This guide assumes basic familiarity with Node.js and is aimed at developers looking to integrate Gemini API into their server-side architecture.

## Setting Up Your Environment and API Key

Let's start with the initial project setup.

### Prerequisites

- Node.js 20 or later
- npm or yarn
- A Google AI Studio account (free at [Google AI Studio](https://aistudio.google.com/))

```bash
# .env
GEMINI_API_KEY=YOUR_API_KEY
PORT=3000
NODE_ENV=production
```

Never hard-code API keys in your source code — always use environment variables to keep them secure.

## Core Architecture: Express.js Meets Gemini API

Let's set up the foundation by initializing the Gemini API client and wiring it into an Express.js server.

```javascript
// src/server.js
import express from 'express';
import cors from 'cors';
import helmet from 'helmet';
import rateLimit from 'express-rate-limit';
import { GoogleGenerativeAI } from '@google/generative-ai';
import dotenv from 'dotenv';

dotenv.config();

const app = express();
const PORT = process.env.PORT || 3000;

// Initialize Gemini API client (once at startup)
const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
const model = genAI.getGenerativeModel({ model: 'gemini-2.5-flash' });

// Middleware
app.use(helmet());
app.use(cors());
app.use(express.json({ limit: '10mb' }));

// Rate limiting (30 requests per minute)
const limiter = rateLimit({
  windowMs: 60 * 1000,
  max: 30,
  message: { error: 'Too many requests. Please try again later.' }
});
app.use('/api/', limiter);

// Health check endpoint
app.get('/health', (req, res) => {
  res.json({ status: 'ok', timestamp: new Date().toISOString() });
});

app.listen(PORT, () => {
  console.log(`AI Microservice running on port ${PORT}`);
});
```

The key detail here is that we create the `GoogleGenerativeAI` instance once at startup. Recreating it per request adds unnecessary overhead.

## Implementing the Text Generation Endpoint

Here's the most fundamental endpoint — accepting a prompt and returning generated text.

```javascript
// src/routes/generate.js
import { Router } from 'express';
import { GoogleGenerativeAI } from '@google/generative-ai';

const router = Router();
const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

// POST /api/generate — Text generation
router.post('/generate', async (req, res) => {
  try {
    const { prompt, maxTokens = 1024, temperature = 0.7 } = req.body;

    if (!prompt || typeof prompt !== 'string') {
      return res.status(400).json({ error: 'prompt is required' });
    }

    const model = genAI.getGenerativeModel({
      model: 'gemini-2.5-flash',
      generationConfig: {
        maxOutputTokens: maxTokens,
        temperature: temperature,
      },
    });

    const result = await model.generateContent(prompt);
    const response = result.response;
    const text = response.text();

    // Retrieve token usage
    const usage = response.usageMetadata;

    res.json({
      text,
      usage: {
        promptTokens: usage?.promptTokenCount || 0,
        completionTokens: usage?.candidatesTokenCount || 0,
        totalTokens: usage?.totalTokenCount || 0,
      },
    });
  } catch (error) {
    console.error('Generate error:', error.message);
    res.status(500).json({ error: 'Generation failed', details: error.message });
  }
});

export default router;
```

This endpoint accepts a `prompt` via POST, generates text through Gemini API, and returns both the result and token usage metadata. The `maxTokens` and `temperature` parameters give clients fine-grained control over the output.

## Implementing Streaming Responses

For chat applications and real-time interfaces, streaming responses are essential. Here's how to implement Server-Sent Events (SSE) with Gemini's streaming API.

```javascript
// POST /api/stream — Streaming generation
router.post('/stream', async (req, res) => {
  const { prompt, systemInstruction } = req.body;

  if (!prompt) {
    return res.status(400).json({ error: 'prompt is required' });
  }

  // Set SSE headers
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  try {
    const model = genAI.getGenerativeModel({
      model: 'gemini-2.5-flash',
      systemInstruction: systemInstruction || undefined,
    });

    const result = await model.generateContentStream(prompt);

    for await (const chunk of result.stream) {
      const text = chunk.text();
      if (text) {
        // Send chunks in SSE format
        res.write(`data: ${JSON.stringify({ text })}\n\n`);
      }
    }

    // Signal stream completion
    res.write(`data: ${JSON.stringify({ done: true })}\n\n`);
    res.end();
  } catch (error) {
    console.error('Stream error:', error.message);
    res.write(`data: ${JSON.stringify({ error: error.message })}\n\n`);
    res.end();
  }
});
```

On the client side, you can consume this stream using `fetch` with a `ReadableStream`:

```javascript
// Client-side implementation
const response = await fetch('/api/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ prompt: 'Explain async programming in Node.js' }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const lines = decoder.decode(value).split('\n');
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      if (data.done) break;
      // Render data.text in the UI in real time
      process.stdout.write(data.text);
    }
  }
}
```

Streaming dramatically reduces the perceived latency for users since they start seeing output as soon as the first tokens are generated, rather than waiting for the entire response.

## Error Handling and Retry Strategies

In production, you must account for rate limit hits, transient network failures, and API outages. Here's a robust retry utility using exponential backoff.

```javascript
// src/utils/retry.js
export async function withRetry(fn, maxRetries = 3, baseDelay = 1000) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      const isRetryable =
        error.status === 429 || // Rate limit
        error.status === 503 || // Service unavailable
        error.message?.includes('RESOURCE_EXHAUSTED');

      if (!isRetryable || attempt === maxRetries) {
        throw error;
      }

      // Exponential backoff with jitter
      const delay = baseDelay * Math.pow(2, attempt - 1) + Math.random() * 500;
      console.warn(`Retry ${attempt}/${maxRetries} after ${Math.round(delay)}ms`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
```

Wrap your API calls with this utility:

```javascript
// Generation with retry
const result = await withRetry(async () => {
  return await model.generateContent(prompt);
}, 3, 1000);
```

The combination of exponential backoff and jitter prevents the "thundering herd" problem, where multiple clients simultaneously retry and overwhelm the API. ## Adding Multimodal Support

One of Gemini API's standout features is its native multimodal capability. Let's add an image analysis endpoint to our microservice.

```javascript
import multer from 'multer';

const upload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: 20 * 1024 * 1024 }, // 20MB
});

// POST /api/analyze-image — Image analysis
router.post('/analyze-image', upload.single('image'), async (req, res) => {
  try {
    if (!req.file) {
      return res.status(400).json({ error: 'Image file is required' });
    }

    const { prompt = 'Describe this image in detail' } = req.body;

    const model = genAI.getGenerativeModel({ model: 'gemini-2.5-flash' });

    const imagePart = {
      inlineData: {
        data: req.file.buffer.toString('base64'),
        mimeType: req.file.mimetype,
      },
    };

    const result = await model.generateContent([prompt, imagePart]);
    const text = result.response.text();

    res.json({ analysis: text });
  } catch (error) {
    console.error('Image analysis error:', error.message);
    res.status(500).json({ error: 'Image analysis failed' });
  }
});
```

This gives your microservice the ability to accept uploaded images and return AI-generated analysis — perfect for content moderation, product cataloging, or visual search features. ## Production Deployment Patterns

### Structured Logging

```javascript
// src/middleware/logger.js
export function requestLogger(req, res, next) {
  const start = Date.now();

  res.on('finish', () => {
    const duration = Date.now() - start;
    const log = {
      method: req.method,
      path: req.path,
      status: res.statusCode,
      duration: `${duration}ms`,
      timestamp: new Date().toISOString(),
    };
    console.log(JSON.stringify(log));
  });

  next();
}
```

### Graceful Shutdown

```javascript
// Clean shutdown on SIGTERM
const server = app.listen(PORT);

process.on('SIGTERM', () => {
  console.log('SIGTERM received. Shutting down gracefully...');
  server.close(() => {
    console.log('Server closed');
    process.exit(0);
  });

  // Force exit after 30s timeout
  setTimeout(() => {
    console.error('Forced shutdown after timeout');
    process.exit(1);
  }, 30000);
});
```

### Docker Containerization

```dockerfile
FROM node:20-alpine

WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY src/ ./src/

EXPOSE 3000
USER node

CMD ["node", "src/server.js"]
```

Using `node:20-alpine` as the base image and `npm ci --omit=dev` keeps the Docker image minimal, which is crucial for fast deployments and reduced attack surface.

## Frequently Asked Questions (FAQ)

### Q1: Can I run this microservice on Gemini API's free tier?

Absolutely — for development and prototyping, the free tier is more than sufficient. As of March 2026, Gemini 2.5 Flash offers 30 requests per minute and 1,500 requests per day at no cost. When you're ready to handle production-level traffic, you can switch to a pay-as-you-go plan.

### Q2: When should I use streaming vs. standard requests?

Use streaming for any user-facing interface where people are watching the response appear in real time (like chat UIs). For batch processing, background jobs, or cases where you need the complete response before acting on it, standard requests are simpler to implement. Streaming makes the biggest difference when responses are lengthy.

### Q3: How do I switch between different Gemini models?

Simply change the `model` parameter in `getGenerativeModel()`. A common strategy is to use `gemini-2.5-flash` for lightweight, cost-effective tasks and `gemini-2.5-pro` for high-precision analysis or complex reasoning. You can make this dynamic by accepting the model name as a request parameter and routing accordingly.

### Q4: How do I prevent memory leaks in a Node.js Gemini service?

Keep the `GoogleGenerativeAI` instance as a singleton — don't recreate it per request. For streaming responses, make sure to properly terminate the stream when clients disconnect. Implement cleanup via `req.on('close', ...)` to abort in-progress generations when the client drops the connection.
