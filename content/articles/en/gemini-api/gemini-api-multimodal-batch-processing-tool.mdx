## Introduction

If you've ever needed to caption hundreds of product images, summarize a stack of PDF reports, or transcribe a batch of meeting recordings, you know how tedious manual processing can be.

Gemini API's multimodal capabilities let you handle images, PDFs, and audio through a single unified API. In this tutorial, you'll build a **multimodal batch processing tool** in Python that automatically detects file types, processes them in parallel, and outputs structured results.

Here's what you'll learn:

- How to design an architecture that routes different file types to appropriate Gemini API prompts
- Using `asyncio` and semaphores for controlled parallel processing

## Architecture Overview

The batch processing tool follows this pipeline:

```
Input folder (mixed images/PDFs/audio)
    ↓
File classification (by extension)
    ↓
┌──────────────┬──────────────┬──────────────┐
│ Image Handler │ PDF Handler  │ Audio Handler │
│  Captioning   │  Summarize   │  Transcribe   │
└──────────────┴──────────────┴──────────────┘
    ↓
Merge results → JSON/CSV output
```

Three design principles guide the implementation:

1. **Automatic file type detection** — Extension-based routing, zero configuration needed
2. **Async parallel processing** — `asyncio.Semaphore` controls concurrent API requests
3. **Retry-aware error handling** — Transient failures don't halt the entire batch

## File-Type Processing Functions

### Image Analysis

This function reads an image file and generates a descriptive caption:

```python
import asyncio
from pathlib import Path

async def process_image(file_path: Path) -> dict:
    """Analyze an image and generate a caption."""
    image_bytes = file_path.read_bytes()
    mime_type = _get_mime_type(file_path)

    response = client.models.generate_content(
        model=MODEL_ID,
        contents=[
            {
                "role": "user",
                "parts": [
                    {"text": "Describe the contents of this image in under 200 characters. Include the main subject, composition, color palette, and mood."},
                    {"inline_data": {"mime_type": mime_type, "data": image_bytes}}
                ]
            }
        ]
    )

    return {
        "file": file_path.name,
        "type": "image",
        "caption": response.text.strip(),
        "status": "success"
    }
```

### MIME Type Detection

A helper function that maps file extensions to MIME types:

```python
def _get_mime_type(file_path: Path) -> str:
    """Determine MIME type from file extension."""
    ext = file_path.suffix.lower()
    mime_map = {
        # Images
        ".jpg": "image/jpeg", ".jpeg": "image/jpeg",
        ".png": "image/png", ".gif": "image/gif",
        ".webp": "image/webp", ".bmp": "image/bmp",
        # PDF
        ".pdf": "application/pdf",
        # Audio
        ".mp3": "audio/mpeg", ".wav": "audio/wav",
        ".m4a": "audio/mp4", ".ogg": "audio/ogg",
        ".flac": "audio/flac",
    }
    return mime_map.get(ext, "application/octet-stream")
```

## The Parallel Batch Processing Engine

This is the core of the tool. It uses `asyncio.Semaphore` to limit concurrent API requests while processing all files efficiently.

```python
import json
import csv
from datetime import datetime
from tqdm.asyncio import tqdm_asyncio

# Map file extensions to handler functions
FILE_HANDLERS = {
    ".jpg": process_image, ".jpeg": process_image,
    ".png": process_image, ".gif": process_image,
    ".webp": process_image, ".bmp": process_image,
    ".pdf": process_pdf,
    ".mp3": process_audio, ".wav": process_audio,
    ".m4a": process_audio, ".ogg": process_audio,
    ".flac": process_audio,
}

# Limit concurrent API requests (rate limit protection)
MAX_CONCURRENCY = 5


async def process_single_file(
    file_path: Path,
    semaphore: asyncio.Semaphore,
    max_retries: int = 3
) -> dict:
    """Process a single file with semaphore control and retry logic."""
    ext = file_path.suffix.lower()
    handler = FILE_HANDLERS.get(ext)

    if handler is None:
        return {
            "file": file_path.name,
            "type": "unknown",
            "status": "skipped",
            "error": f"Unsupported extension: {ext}"
        }

    for attempt in range(1, max_retries + 1):
        try:
            async with semaphore:
                result = await asyncio.to_thread(lambda: asyncio.run(handler(file_path)))
                return result
        except Exception as e:
            if attempt == max_retries:
                return {
                    "file": file_path.name,
                    "type": ext,
                    "status": "error",
                    "error": f"{type(e).__name__}: {str(e)}"
                }
            # Exponential backoff
            wait_time = 2 ** attempt
            print(f"  ⚠ {file_path.name} retry {attempt}/{max_retries} (waiting {wait_time}s)")
            await asyncio.sleep(wait_time)


async def batch_process(input_dir: str, output_path: str = "results.json"):
    """Batch-process all supported files in a directory."""
    input_path = Path(input_dir)
    if not input_path.is_dir():
        raise FileNotFoundError(f"Input directory not found: {input_dir}")

    # Collect supported files
    files = [
        f for f in input_path.iterdir()
        if f.is_file() and f.suffix.lower() in FILE_HANDLERS
    ]

    if not files:
        print("No supported files found.")
        return

    print(f"📁 Found {len(files)} files")
    print(f"  Images: {sum(1 for f in files if f.suffix.lower() in {'.jpg','.jpeg','.png','.gif','.webp','.bmp'})}")
    print(f"  PDFs:   {sum(1 for f in files if f.suffix.lower() == '.pdf')}")
    print(f"  Audio:  {sum(1 for f in files if f.suffix.lower() in {'.mp3','.wav','.m4a','.ogg','.flac'})}")

    semaphore = asyncio.Semaphore(MAX_CONCURRENCY)

    # Run all tasks in parallel with progress bar
    tasks = [process_single_file(f, semaphore) for f in files]
    results = await tqdm_asyncio.gather(*tasks, desc="Processing")

    # Calculate statistics
    success = sum(1 for r in results if r["status"] == "success")
    errors = sum(1 for r in results if r["status"] == "error")
    skipped = sum(1 for r in results if r["status"] == "skipped")

    # Write JSON output
    output = {
        "processed_at": datetime.now().isoformat(),
        "total_files": len(files),
        "success": success,
        "errors": errors,
        "skipped": skipped,
        "results": results
    }

    with open(output_path, "w", encoding="utf-8") as f:
        json.dump(output, f, ensure_ascii=False, indent=2)

    print(f"\n✅ Done: {success} succeeded / {errors} errors / {skipped} skipped")
    print(f"📄 Results saved to {output_path}")

    return output
```

## Adding CSV Export

For easy viewing in spreadsheets, add CSV export alongside JSON:

```python
def export_to_csv(results: list[dict], output_path: str = "results.csv"):
    """Export processing results to CSV format."""
    fieldnames = ["file", "type", "status", "caption", "summary",
                  "transcription_summary", "error"]

    with open(output_path, "w", newline="", encoding="utf-8-sig") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames, extrasaction="ignore")
        writer.writeheader()
        for row in results:
            writer.writerow(row)

    print(f"📊 CSV saved to {output_path}")
```

The `utf-8-sig` encoding ensures the file opens correctly in Excel without character encoding issues.

### Handling Large Files

The Gemini API has a 20MB limit for inline data. For larger files, use the File API to upload first, then reference the uploaded file:

```python
async def process_large_file(file_path: Path, prompt: str) -> dict:
    """Process files over 20MB via the File API."""
    # Upload via File API
    uploaded = client.files.upload(file=file_path)

    # Wait for processing to complete
    import time
    while uploaded.state.name == "PROCESSING":
        time.sleep(2)
        uploaded = client.files.get(name=uploaded.name)

    if uploaded.state.name == "FAILED":
        raise RuntimeError(f"File upload failed: {uploaded.name}")

    # Generate content using uploaded file reference
    response = client.models.generate_content(
        model=MODEL_ID,
        contents=[
            {"role": "user", "parts": [
                {"text": prompt},
                {"file_data": {"file_uri": uploaded.uri, "mime_type": uploaded.mime_type}}
            ]}
        ]
    )

    # Clean up uploaded file
    client.files.delete(name=uploaded.name)

    return {
        "file": file_path.name,
        "result": response.text.strip(),
        "status": "success"
    }
```

## Common Errors and Solutions

Here's a quick reference for errors you're likely to encounter during batch processing:

| Error | Cause | Solution |
|-------|-------|----------|
| `429 Resource Exhausted` | API rate limit exceeded | Lower `MAX_CONCURRENCY` (3-5 recommended) |
| `400 Invalid argument` | Unsupported file format | Check MIME type detection; corrupted files can also trigger this |
| `413 Request too large` | File exceeds 20MB inline limit | Switch to File API upload approach |
| `500 Internal error` | Temporary Gemini-side issue | Retry logic handles this automatically; skip after 3 failures |

## FAQ

### Can I use this with the free tier?

Yes, absolutely. The free tier has rate limits (15 requests per minute for Gemini 2.5 Flash), so set `MAX_CONCURRENCY` to `2` and add a small delay between requests. For heavy workloads, consider the [Gemini Batch Processing API](/en/articles/gemini-api/gemini-batch-processing-api-guide) which offers 50% cost savings on large volumes.

### Does this work with video files too?

Yes — Gemini API supports video input. Add `.mp4`, `.mov`, and `.avi` extensions to `FILE_HANDLERS` with a video-specific processing function. Since video files tend to be large, use the File API upload approach rather than inline data.

### How can I improve output quality?

A few proven techniques work well here. First, be specific in your prompts — instead of "summarize this," try "summarize in under 300 characters, highlighting 3 key takeaways." For images, higher resolution yields better recognition accuracy. For PDFs, well-structured documents with clear headings and paragraphs produce more accurate summaries.

### How should I manage API key security?

Always store your API key in environment variables or a `.env` file — never hardcode it in source files. For team environments, use a secrets management service like Google Cloud Secret Manager or AWS Secrets Manager.
