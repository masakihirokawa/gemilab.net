## What Is Multi-Turn Chat?

A standard `generateContent()` call is a one-shot interaction — each request is independent. But chat applications need the AI to maintain context across multiple exchanges. Gemini API's `startChat()` method handles this automatically, preserving conversation history so each response is informed by everything said before.

This article walks through multi-turn chat basics, history management, System Instructions integration, and building a practical chatbot step by step.

---

## Basic Multi-Turn Chat

### Simple Conversation Implementation

```python
# basic_chat.py
# Implement basic multi-turn chat with the Gemini API

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

# Start a chat session
chat = model.start_chat()

# First message
response1 = chat.send_message("How do I remove duplicates from a list in Python?")
print("AI:", response1.text)

# Expected output:
# AI: There are several ways to remove duplicates from a list:
# 1. Using set(): list(set(my_list))
# 2. Using dict.fromkeys(): list(dict.fromkeys(my_list))
# ...

# Second message (context from the first exchange is preserved)
response2 = chat.send_message("What's the time complexity of each approach?")
print("AI:", response2.text)

# Expected output:
# AI: Here's a comparison of the time complexity for each method:
# - set(): O(n) — uses hash table
# - dict.fromkeys(): O(n) — preserves order while deduplicating
# ...

# Review conversation history
for message in chat.history:
    print(f"{message.role}: {message.parts[0].text[:50]}...")
```

The chat session created by `start_chat()` automatically accumulates all sent and received messages in the `history` property. The second question about "each approach" is correctly interpreted because the first Q&A is preserved as context.

## Starting a Chat with Existing History

To resume a previous conversation, pass a `history` parameter to `start_chat()`.

```python
# resume_chat.py
# Resume chat from saved conversation history

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

# Prepare previous conversation history
history = [
    {"role": "user", "parts": ["I'm a web designer getting interested in AI tools."]},
    {"role": "model", "parts": ["There are plenty of AI tools useful for web designers — from design generation to coding assistance to image editing. Which area interests you most?"]},
    {"role": "user", "parts": ["I'm interested in coding assistance."]},
    {"role": "model", "parts": ["For coding assistance, GitHub Copilot is one of the most popular options. It provides real-time HTML/CSS completion and JavaScript logic suggestions."]},
]

# Resume chat with history
chat = model.start_chat(history=history)

# Ask a context-aware question
response = chat.send_message("What other options would you recommend?")
print(response.text)

# Expected output:
# Beyond GitHub Copilot, here are other coding assistance tools for web designers:
# - Cursor: An AI-powered editor built on VS Code...
# - Claude Code: Generate code from the command line...
# ...
```

This feature lets you load conversation history from a database and seamlessly resume sessions.

## Combining with System Instructions

Set up the AI's personality and response rules with System Instructions, then combine with multi-turn chat to create a consistently behaved chatbot.

```python
# chatbot_with_system.py
# Build a customer support bot with System Instructions

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

# Define the bot's role with System Instructions
model = genai.GenerativeModel(
    model_name="gemini-2.0-flash",
    system_instruction="""You are a customer support agent for "TechShop".
Follow these rules:

1. Always respond politely and professionally
2. Answer product questions accurately
3. Never state inaccurate pricing (say "let me check" instead)
4. Show empathy for complaints and propose solutions
5. Keep responses concise, under 150 words
6. For out-of-scope questions, say "Let me connect you with the right department"

Product catalog:
- SmartWatch SW-100 ($299)
- Wireless Earbuds WE-200 ($129)
- Portable Speaker PS-300 ($89)"""
)

chat = model.start_chat()

# Sample conversation
queries = [
    "Is the SW-100 waterproof?",
    "My WE-200 I bought last week has no sound from the left earbud",
    "How long would a repair take?",
]

for query in queries:
    response = chat.send_message(query)
    print(f"Customer: {query}")
    print(f"Support: {response.text}")
    print("---")
```

System Instructions apply to the entire conversation, so the "TechShop support agent" behavior persists regardless of how many turns have passed. For more on this, see the [Prompt Engineering Guide](/en/articles/gemini-basics/gemini-prompt-engineering-guide).

## Streaming for Real-Time Responses

Chat UIs typically display text as it's generated rather than waiting for the full response. This is called streaming.

```python
# streaming_chat.py
# Display chat responses in real-time with streaming

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

chat = model.start_chat()

def stream_chat(message: str) -> str:
    """Send a message and display the response via streaming"""
    print(f"You: {message}")
    print("AI: ", end="", flush=True)

    full_response = ""
    response = chat.send_message(message, stream=True)

    for chunk in response:
        text = chunk.text
        print(text, end="", flush=True)
        full_response += text

    print()  # newline
    return full_response

# Interactive chat
stream_chat("Explain quantum computing to a 5-year-old")
stream_chat("Now explain it at an elementary school level")
stream_chat("What can it actually be used for?")
```

Passing `stream=True` to `send_message()` returns the response in chunks. Printing each chunk immediately gives users a natural experience of the AI "thinking as it speaks."

## Saving and Loading Conversations

Production chat apps need the ability to save conversations and resume them later.

```python
# save_load_chat.py
# Save and load chat history using JSON files

import json
from pathlib import Path
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

def save_history(chat, filepath: str) -> None:
    """Save conversation history to a JSON file"""
    history_data = []
    for message in chat.history:
        history_data.append({
            "role": message.role,
            "text": message.parts[0].text,
        })

    Path(filepath).write_text(
        json.dumps(history_data, indent=2),
        encoding="utf-8"
    )
    print(f"Saved: {len(history_data)} messages -> {filepath}")

def load_and_resume(filepath: str) -> genai.ChatSession:
    """Load saved history and resume the chat"""
    data = json.loads(Path(filepath).read_text(encoding="utf-8"))

    history = [
        {"role": item["role"], "parts": [item["text"]]}
        for item in data
    ]

    model = genai.GenerativeModel("gemini-2.0-flash")
    return model.start_chat(history=history)

# Usage
model = genai.GenerativeModel("gemini-2.0-flash")
chat = model.start_chat()

chat.send_message("Compare React state management libraries")
chat.send_message("Tell me more about Zustand")

# Save
save_history(chat, "chat_history.json")

# Resume later
resumed_chat = load_and_resume("chat_history.json")
response = resumed_chat.send_message("Write a Zustand example")
print(response.text)
```

## Managing Token Usage

Token consumption grows as conversations get longer in multi-turn chat. Use `count_tokens()` to monitor usage proactively.

```python
# token_management.py
# Monitor token usage during chat

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")
chat = model.start_chat()

MAX_TOKENS = 50000  # Token budget

def send_with_token_check(message: str) -> str:
    """Check token count before sending a message"""
    # Calculate tokens for current history + new message
    token_count = model.count_tokens(
        chat.history + [{"role": "user", "parts": [message]}]
    )

    print(f"Current tokens: {token_count.total_tokens} / {MAX_TOKENS}")

    if token_count.total_tokens > MAX_TOKENS:
        print("Warning: approaching token limit. Consider summarizing.")
        return "Token limit reached. Please start a new conversation."

    response = chat.send_message(message)
    return response.text

# Usage
print(send_with_token_check("Explain how TypeScript type inference works"))
print(send_with_token_check("How does it relate to generics?"))
```

## FAQ

### Q1. What's the difference between multi-turn chat and single generateContent() calls?

A1. `generateContent()` treats each call independently with no memory of previous interactions. Multi-turn chat via `startChat()` automatically includes the full conversation history in every request, enabling context-aware responses that reference earlier messages.

### Q2. How should I handle very long conversation histories?

A2. Three approaches work well: (1) Keep only the most recent N messages. (2) Use Gemini to summarize older messages and compress token usage. (3) Monitor token count with `count_tokens()` and start a new session when a threshold is reached. Combine these strategies based on your use case.

### Q3. Can I use streaming with multi-turn chat?

A3. Yes, pass `stream=True` to `chat.send_message()` to enable streaming. It works the same way as streaming with regular `generateContent`, returning chunks that you can display incrementally. See [Streaming & Chat](/en/articles/gemini-api/streaming-and-chat) for more details.

### Q4. Can I send images or videos within a chat session?

A4. Yes, Gemini supports multimodal input in chat sessions. Include image data in the `parts` array of your message to ask questions about images while maintaining the full conversation context. See the [Multimodal API Guide](/en/articles/gemini-api/multimodal-api-guide) for details.
