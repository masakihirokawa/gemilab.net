## What Is Structured Output?

Gemini API's structured output feature lets you receive model responses in a format that strictly conforms to a JSON schema. This eliminates the need to parse free-form text output, making application integration dramatically simpler.

With traditional prompt engineering, asking the model to "return JSON" often resulted in markdown code fences leaking in or missing fields. Structured output solves this at the protocol level.

---

## Basic JSON Mode

The simplest structured output sets `responseMimeType` to `application/json`.

```typescript
// basic-json-mode.ts
// Basic JSON Mode with Gemini API

import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

const model = genAI.getGenerativeModel({
  model: "gemini-2.0-flash",
  generationConfig: {
    responseMimeType: "application/json",
  },
});

const result = await model.generateContent(
  "Return a 3-day weather forecast for Tokyo in JSON format. Include date, weather, high_temp, and low_temp for each day."
);

const forecast = JSON.parse(result.response.text());
console.log(forecast);

// Expected output:
// [
//   { "date": "2026-03-14", "weather": "sunny", "high_temp": 18, "low_temp": 8 },
//   { "date": "2026-03-15", "weather": "cloudy", "high_temp": 15, "low_temp": 7 },
//   { "date": "2026-03-16", "weather": "rain", "high_temp": 12, "low_temp": 6 }
// ]
```

However, this approach doesn't guarantee field names or types. For stricter control, use `responseSchema`.

## Type-Safe Output with Schema Definitions

When you specify `responseSchema`, Gemini returns JSON that fully conforms to your schema. No missing fields, no type mismatches.

```typescript
// schema-defined-output.ts
// Strict type control with responseSchema

import { GoogleGenerativeAI, SchemaType } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

const model = genAI.getGenerativeModel({
  model: "gemini-2.0-flash",
  generationConfig: {
    responseMimeType: "application/json",
    responseSchema: {
      type: SchemaType.OBJECT,
      properties: {
        products: {
          type: SchemaType.ARRAY,
          items: {
            type: SchemaType.OBJECT,
            properties: {
              name: { type: SchemaType.STRING, description: "Product name" },
              category: {
                type: SchemaType.STRING,
                enum: ["electronics", "clothing", "food", "other"],
                description: "Product category",
              },
              price: { type: SchemaType.NUMBER, description: "Price in USD" },
              inStock: { type: SchemaType.BOOLEAN, description: "In stock" },
            },
            required: ["name", "category", "price", "inStock"],
          },
        },
        totalCount: { type: SchemaType.INTEGER, description: "Total product count" },
      },
      required: ["products", "totalCount"],
    },
  },
});

const result = await model.generateContent(
  "Return the top 3 popular products from an online store."
);

const data = JSON.parse(result.response.text());
console.log(data);

// Expected output:
// {
//   "products": [
//     { "name": "Wireless Earbuds Pro", "category": "electronics", "price": 89.99, "inStock": true },
//     { "name": "Organic Cotton T-Shirt", "category": "clothing", "price": 29.99, "inStock": true },
//     { "name": "Premium Matcha Powder", "category": "food", "price": 24.99, "inStock": false }
//   ],
//   "totalCount": 3
// }
```

Using `enum` constrains category values to predefined options. The `required` field specification prevents missing mandatory properties.

## TypeScript Integration for End-to-End Type Safety

Combining Gemini's schema definitions with TypeScript's type system gives you complete type safety from API call to application logic.

```typescript
// typed-gemini-client.ts
// Integrating TypeScript types with Gemini structured output

import { GoogleGenerativeAI, SchemaType } from "@google/generative-ai";

// TypeScript type definition (used in application code)
interface BookReview {
  title: string;
  author: string;
  rating: number;
  summary: string;
  tags: string[];
}

interface ReviewResponse {
  reviews: BookReview[];
  generatedAt: string;
}

// Gemini schema definition (used in API request)
const reviewSchema = {
  type: SchemaType.OBJECT,
  properties: {
    reviews: {
      type: SchemaType.ARRAY,
      items: {
        type: SchemaType.OBJECT,
        properties: {
          title: { type: SchemaType.STRING },
          author: { type: SchemaType.STRING },
          rating: { type: SchemaType.NUMBER },
          summary: { type: SchemaType.STRING },
          tags: { type: SchemaType.ARRAY, items: { type: SchemaType.STRING } },
        },
        required: ["title", "author", "rating", "summary", "tags"],
      },
    },
    generatedAt: { type: SchemaType.STRING },
  },
  required: ["reviews", "generatedAt"],
} as const;

async function generateBookReviews(genre: string): Promise<ReviewResponse> {
  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);
  const model = genAI.getGenerativeModel({
    model: "gemini-2.0-flash",
    generationConfig: {
      responseMimeType: "application/json",
      responseSchema: reviewSchema,
    },
  });

  const result = await model.generateContent(
    `Return 3 recommended ${genre} books with reviews. Rating should be 1-5 with one decimal.`
  );

  const parsed: ReviewResponse = JSON.parse(result.response.text());
  return parsed;
}

const reviews = await generateBookReviews("science fiction");
console.log(`${reviews.reviews.length} reviews generated at ${reviews.generatedAt}`);
reviews.reviews.forEach((r) => {
  console.log(`  ${r.title} by ${r.author} — ${r.rating}/5`);
});

// Expected output:
// 3 reviews generated at 2026-03-14T18:00:00Z
//   The Three-Body Problem by Liu Cixin — 4.8/5
//   Project Hail Mary by Andy Weir — 4.6/5
//   Neuromancer by William Gibson — 4.3/5
```

## Adding a Validation Layer

On top of schema constraints, adding Zod validation at the application layer provides an extra safety net.

```typescript
// validation-layer.ts
// Runtime validation with Zod

import { z } from "zod";

const BookReviewSchema = z.object({
  reviews: z.array(
    z.object({
      title: z.string().min(1),
      author: z.string().min(1),
      rating: z.number().min(1).max(5),
      summary: z.string().min(10).max(500),
      tags: z.array(z.string()).min(1).max(5),
    })
  ),
  generatedAt: z.string().datetime(),
});

type ValidatedReviewResponse = z.infer<typeof BookReviewSchema>;

function validateResponse(raw: unknown): ValidatedReviewResponse {
  const result = BookReviewSchema.safeParse(raw);
  if (!result.success) {
    console.error("Validation errors:", result.error.flatten());
    throw new Error(`Invalid response: ${result.error.message}`);
  }
  return result.data;
}

// Usage
const rawData = JSON.parse(geminiResponse);
const validated = validateResponse(rawData);
console.log(`Validated ${validated.reviews.length} reviews`);

// Expected output: Validated 3 reviews
```

## Error Handling and Retry Logic

Even with structured output, rate limits and network errors can occur. Build in robust retry logic.

```typescript
// retry-handler.ts
// Exponential backoff retry

async function withRetry<T>(
  fn: () => Promise<T>,
  maxRetries = 3,
  baseDelay = 1000
): Promise<T> {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;

      const delay = baseDelay * Math.pow(2, attempt);
      const jitter = Math.random() * delay * 0.1;
      console.log(`Retry ${attempt + 1}/${maxRetries} in ${Math.round(delay + jitter)}ms`);
      await new Promise((r) => setTimeout(r, delay + jitter));
    }
  }
  throw new Error("Unreachable");
}

// Usage
const reviews = await withRetry(() => generateBookReviews("mystery"));

// Expected output (on retry):
// Retry 1/3 in 1050ms
// 3 reviews generated at ...
```

## FAQ

### Q1. What's the difference between responseSchema and responseMimeType alone?

A1. `responseMimeType: "application/json"` guarantees JSON format but not field names or types. Adding `responseSchema` ensures the response fully conforms to your specified schema — correct field names, types, and required properties.

### Q2. Can enum-defined values ever contain unexpected options?

A2. In practice, no — Gemini selects from the defined enum values. However, for safety, it's recommended to add application-side validation with Zod or similar libraries as an extra guard.

### Q3. Does it support deeply nested schemas?

A3. Yes, arrays within objects and objects within arrays are fully supported. However, overly complex schemas increase token consumption and slow response times. Keep your schema as minimal as possible.

### Q4. Can structured output be used with streaming?

A4. You can stream with `responseMimeType: "application/json"` set, but JSON completeness isn't guaranteed until the final chunk arrives. If you need partial results during streaming, you'll need to validate accumulated chunks with `JSON.parse`.

### Q5. When should I use Function Calling vs. Structured Output?

A5. Use Function Calling when you need to invoke external tools (APIs, databases). Use Structured Output when you want the model's response itself in a structured format. Searching a restaurant API calls for Function Calling; returning review analysis as JSON calls for Structured Output.
