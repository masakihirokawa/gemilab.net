# Complete Guide to Gemini API System Instructions — Control AI Behavior Like a Pro

## Introduction

If you've been building applications with the Gemini API, you've likely noticed that raw prompts alone often aren't enough to make your AI behave consistently. That's where **System Instructions** come in.

System Instructions let you define the model's persona, communication style, output format, and behavioral constraints before any user interaction takes place. Once set, these instructions persist across the entire conversation — even if the user tries to steer the model elsewhere.

In this guide, you'll learn:

- What System Instructions are and how they differ from regular prompts
- How to implement them in Python and JavaScript
- Real-world examples: customer support bots, code reviewers, multilingual assistants, and structured data extractors
- How to combine System Instructions with multi-turn chat
- Best practices for writing effective instructions
- Common errors and how to fix them
- FAQs

This guide assumes you already have a Gemini API key and the SDK installed. If you're new to the API, start with the [Gemini API Quickstart](/en/articles/gemini-api/gemini-api-quickstart) first.

## What Are System Instructions?

System Instructions are pre-conversation directives sent to the model that shape its behavior throughout the entire session. Unlike regular user messages (`contents`), they are invisible to end users and not stored in the conversation history.

Think of System Instructions as writing the job description and rules of conduct for your AI before it starts working. They answer questions like:

- What role does the AI play?
- How should it speak (formal, casual, technical)?
- What format should responses follow?
- What topics are in or out of scope?
- What languages should it use?

### What You Can Control with System Instructions

| Control Target | Example |
|---------------|---------|
| Role definition | "You are a senior Python engineer at Google" |
| Tone and style | "Always be concise and avoid jargon unless necessary" |
| Output format | "Respond only in valid Markdown" |
| Language policy | "Always respond in the user's language" |
| Scope restriction | "Only answer questions about our product" |
| Safety policy | "Never mention competitor products by name" |

## Python Implementation

### Basic Usage

```python
import os
from google import genai
from google.genai import types

# Initialize the client
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Request with System Instructions
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="How do I read a file in Python?",
    config=types.GenerateContentConfig(
        system_instruction=(
            "You are an experienced Python senior engineer. "
            "Always include type hints in your code examples. "
            "Add comments to explain key lines and include expected output."
        ),
        temperature=0.4,
    ),
)

print(response.text)
# Expected output:
# def read_file(filepath: str) -> str:
#     """Read a file and return its content as a string."""
#     with open(filepath, "r", encoding="utf-8") as f:
#         return f.read()
#
# # Expected output:
# # "Hello, World!\n"
```

### Multi-turn Chat with System Instructions

System Instructions persist across all turns in a chat session — no need to re-specify them each time.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

SUPPORT_INSTRUCTION = """
You are "GemLab Assistant," a customer support agent for Gemini Lab Pro.
Follow these rules strictly:
1. Always be professional, empathetic, and solution-focused
2. Only answer questions about Gemini Lab Pro
3. Never discuss or compare competitor products
4. If you cannot resolve an issue, say: "I'll escalate this to our support team"
5. End every response with: "Is there anything else I can help you with?"
"""

# Start a chat session with System Instructions
chat = client.chats.create(
    model="gemini-2.5-pro",
    config=types.GenerateContentConfig(
        system_instruction=SUPPORT_INSTRUCTION,
        temperature=0.3,  # Low temperature for consistent support responses
    ),
)

# Turn 1
response1 = chat.send_message("How do I upgrade my plan?")
print("Bot:", response1.text)

# Turn 2 — context maintained from previous turns
response2 = chat.send_message("Do you support PayPal?")
print("Bot:", response2.text)
```

### Code Reviewer Example

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

CODE_REVIEWER = """
You are a senior software engineer at Google specializing in code review.
For every code snippet, provide feedback in this exact structure:

🔴 Critical Issues (security bugs, crashes)
🟡 Recommendations (performance, readability)
🟢 What's Good

Always include a corrected code snippet and explain why each change matters.
End with a Quality Score: X/5.
"""

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="""
Please review this code:

```python
def get_user(id):
    query = "SELECT * FROM users WHERE id = " + id
    return db.execute(query)
```
""",
    config=types.GenerateContentConfig(
        system_instruction=CODE_REVIEWER,
    ),
)

print(response.text)
# Expected output:
# 🔴 Critical Issues
# SQL Injection vulnerability — never concatenate user input into SQL strings.
# Fix: use parameterized queries...
# Quality Score: 1/5
```

## JavaScript Implementation

```javascript
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

// Initialize model with System Instructions
const model = genAI.getGenerativeModel({
  model: "gemini-2.5-pro",
  systemInstruction: {
    role: "system",
    parts: [
      {
        text: `You are a bilingual technical writer fluent in English and Japanese.
- Respond in the same language the user writes in
- Keep explanations clear and accessible to developers at any level
- Always define technical terms on first use
- Include practical examples where relevant`,
      },
    ],
  },
  generationConfig: {
    temperature: 0.7,
    maxOutputTokens: 2048,
  },
});

const chat = model.startChat();

// English question → English response
const r1 = await chat.sendMessage("What is a context window in LLMs?");
console.log(r1.response.text());

// Follow-up question — context maintained
const r2 = await chat.sendMessage("How does Gemini 2.5 Pro compare to others?");
console.log(r2.response.text());
```

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

MULTILINGUAL_SUPPORT = """

### 2. Structured Data Extractor

Force the model to always return clean JSON — perfect for data pipelines.

```python
import os
import json
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

DATA_EXTRACTOR = """
You are a data extraction specialist. Extract information from the input text
and return ONLY valid JSON in this exact schema:

{
  "name": "Full name (null if not found)",
  "company": "Company name (null if not found)",
  "email": "Email address (null if not found)",
  "phone": "Phone number (null if not found)",
  "role": "Job title (null if not found)"
}

Return no other text — only the JSON object.
"""

text = """
Hi, I'm Sarah Chen, Product Manager at Innovatech Solutions.
You can reach me at s.chen@innovatech.com or +1-415-555-0192.
"""

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=text,
    config=types.GenerateContentConfig(
        system_instruction=DATA_EXTRACTOR,
        temperature=0.0,  # Zero temperature for deterministic output
        response_mime_type="application/json",  # Force JSON output
    ),
)

result = json.loads(response.text)
print(result)
# Expected output:
# {
#   "name": "Sarah Chen",
#   "company": "Innovatech Solutions",
#   "email": "s.chen@innovatech.com",
#   "phone": "+1-415-555-0192",
#   "role": "Product Manager"
# }
```

## Best Practices for Writing Effective System Instructions

Writing good System Instructions is part art, part engineering. Here are the principles that consistently yield the best results.

### 1. Define the Role Precisely

```
# Vague (avoid this)
"You are a helpful assistant."

# Precise (do this)
"You are a senior Python engineer with 8 years of experience building
production-grade web APIs. You favor clean, readable code over clever tricks
and always consider security implications in your suggestions."
```

### 2. Specify Output Format Explicitly

```
"Structure every response as follows:
1. Summary (2-3 sentences)
2. Step-by-step explanation with code examples
3. Common pitfalls to avoid
4. Related resources (if applicable)"
```

### 3. Frame Constraints Positively

```
# Negative framing (less effective)
"Don't talk about competitor products."

# Positive framing (more effective)
"Focus exclusively on Gemini API and Google AI Studio.
For questions about other AI services, kindly direct users
to the official documentation for those products."
```

### 4. Include Few-Shot Examples for Complex Formats

For non-standard output structures, embed examples directly in the instruction:

```python
INSTRUCTION_WITH_EXAMPLES = """
Review code and respond in this format:

Example input:
```python
x = 1
print(x)
```

Example output:
## Code Review
- ✅ Simple and readable
- ⚠️ Consider using a descriptive variable name instead of `x`
- Quality Score: 4/5
"""
```

## Combining with Context Caching

When your System Instructions include large reference documents (e.g., product manuals, codebases), use [Context Caching](/en/articles/gemini-api/context-caching-guide) to avoid re-sending the same tokens on every request.

```python
import os
import datetime
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

with open("product_manual.txt", "r") as f:
    manual_content = f.read()

# Cache the manual content for 1 hour
cache = client.caches.create(
    model="gemini-2.5-pro",
    config=types.CreateCachedContentConfig(
        system_instruction=(
            "You are a product support specialist. "
            "Reference the product manual below to answer questions accurately."
        ),
        contents=[manual_content],
        ttl=datetime.timedelta(hours=1),
    ),
)

# Subsequent requests use the cache — no need to re-send the manual
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="How do I perform a factory reset?",
    config=types.GenerateContentConfig(
        cached_content=cache.name,
    ),
)
print(response.text)
```

This approach dramatically reduces token costs and improves response latency when the same context is used across many requests.

## Common Errors and Fixes

### Error 1: Model Ignores System Instructions

**Symptom**: The model's responses don't reflect your instructions.

**Fixes**:
- Make instructions more specific — vague instructions yield inconsistent results
- Check for internal contradictions in your instruction text
- Lower `temperature` to 0.3–0.5 for more deterministic behavior

### Error 2: JSON Output Contains Extra Text

**Fix**: Use `response_mime_type` to enforce JSON output:

```python
config=types.GenerateContentConfig(
    system_instruction="Return only valid JSON.",
    response_mime_type="application/json",
)
```

See the [JSON Mode and Structured Output guide](/en/articles/gemini-api/json-mode-structured-output) for more details.

### Error 3: Instructions Stop Working Mid-Conversation

**Symptom**: After many turns, the model drifts from its instructions.

**Fixes**:
- Very long conversations can fill the context window — periodically remind the model of key rules
- Use Context Caching to manage context more efficiently

### Error 4: `400 Bad Request`

The content of your System Instructions may be triggering safety filters. Review your instructions for any content that could be interpreted as requesting harmful output.

## FAQ

**Q1: Do System Instructions consume tokens?**

Yes. System Instructions are counted as input tokens on every request. For long instructions, using [Context Caching](/en/articles/gemini-api/context-caching-guide) can significantly reduce costs at scale.

**Q2: Can users override System Instructions?**

Not through normal conversation turns. However, prompt injection attacks ("ignore all previous instructions and...") are a real threat for user-facing apps. Always validate and sanitize user input in production environments.

**Q3: Can I use multiple System Instructions?**

The API accepts a single `system_instruction` string, but you can structure multiple roles and rules within that string using headers, numbered lists, or separator lines (`---`) for clarity.

**Q4: Do System Instructions work with Function Calling?**

Absolutely. You can instruct the model to always use a specific function when certain conditions are met: "When the user asks about weather, always call the `get_weather` function." See the [Function Calling Guide](/en/articles/gemini-advanced/function-calling-guide) for details.

**Q5: Does this work the same way on Vertex AI?**

The `system_instruction` parameter works identically. The only difference is the client initialization:

```python
client = genai.Client(
    vertexai=True,
    project="your-gcp-project-id",
    location="us-central1",
)
```
