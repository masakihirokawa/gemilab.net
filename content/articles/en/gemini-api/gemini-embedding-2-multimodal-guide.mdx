## What is Gemini Embedding 2?

On March 10, 2026, Google launched **Gemini Embedding 2** (model ID: `gemini-embedding-2-preview`) as a public preview — the world's **first fully multimodal embedding model** from Google.

Traditional embedding models handle either text or images separately, requiring you to stitch together multiple models to support cross-modal retrieval. Gemini Embedding 2 solves this by mapping **text, images, video, audio, and PDFs into a single unified vector space**. For the first time, a single API call can embed any of these modalities and return vectors that are directly comparable to one another.

In this guide, we'll walk through the model's key capabilities, set up the API, and build real-world multimodal applications step by step.

---

## Key Specifications

| Property | Details |
|----------|---------|
| Model ID | `gemini-embedding-2-preview` |
| Supported modalities | Text, images, video, audio, PDF |
| Output dimensions | 128–3,072 (recommended: 768, 1536, 3072) |
| Text input limit | 8,192 tokens |
| Image input limit | Up to 6 images per request (PNG, JPEG) |
| Audio input limit | Up to 80 seconds (MP3, WAV) |
| Video input limit | Up to 128 seconds (MP4, MOV — H264/H265/AV1/VP9) |
| PDF input limit | Up to 6 pages |

**Performance gains**: Early adopters have reported up to a **70% reduction in latency** and a **20% improvement in recall** compared to traditional multi-model pipelines that use separate text and image embedding models.

> **Important**: Gemini Embedding 2 uses an embedding space that is **incompatible** with `gemini-embedding-001`. If you have an existing vector store, you'll need to re-embed all your data when migrating.

## Getting Started with Text Embeddings

Let's start with the simplest use case — embedding text strings.

```python
import os
from google import genai

# Initialize the client
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Embed a single piece of text
result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=["The Gemini API provides access to Google's AI models."]
)

embedding = result.embeddings[0].values
print(f"Embedding dimensions: {len(embedding)}")
# Output: Embedding dimensions: 3072

# Batch embedding — process multiple texts in one request
texts = [
    "How to use the Gemini API",
    "What is multimodal AI?",
    "Building a RAG pipeline"
]
batch_result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=texts
)
print(f"Batch processed: {len(batch_result.embeddings)} embeddings generated")
```

### Customizing Output Dimensions

You can control the output vector size to balance quality, storage cost, and latency:

```python
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Use 768 dimensions for a lighter, faster embedding
result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=["Test sentence"],
    config=types.EmbedContentConfig(output_dimensionality=768)
)
print(f"Dimensions: {len(result.embeddings[0].values)}")  # 768
```

Use 3,072 dimensions for maximum accuracy, 1,536 for a balanced approach, and 768 for applications where speed and storage efficiency matter most.

## Multimodal Embeddings in Practice

The real power of Gemini Embedding 2 lies in its ability to embed images, audio, and video alongside text in the same vector space.

### Embedding Images

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Read a local image file
with open("sample.jpg", "rb") as f:
    image_bytes = f.read()

# Embed the image
result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=[
        types.Part.from_bytes(data=image_bytes, mime_type="image/jpeg")
    ]
)

image_embedding = result.embeddings[0].values
print(f"Image embedding dimensions: {len(image_embedding)}")
# Output: Image embedding dimensions: 3072
```

### Cross-Modal Search: Query Images with Text

One of the most compelling applications is searching an image database using a natural language query. Because text and images share the same embedding space, cosine similarity comparisons work across modalities.

```python
import os
import numpy as np
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

def get_embedding(content):
    """Get embeddings for either a text string or image bytes."""
    if isinstance(content, str):
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[content]
        )
    else:
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[types.Part.from_bytes(data=content, mime_type="image/jpeg")]
        )
    return np.array(result.embeddings[0].values)

def cosine_similarity(a, b):
    """Calculate cosine similarity between two vectors."""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Build an image database
image_files = ["cat.jpg", "dog.jpg", "landscape.jpg"]
image_index = []

for img_path in image_files:
    with open(img_path, "rb") as f:
        emb = get_embedding(f.read())
        image_index.append((img_path, emb))

# Search using a text query
query = "a cute cat photo"
query_embedding = get_embedding(query)

# Rank by similarity score
results = [
    (path, cosine_similarity(query_embedding, emb))
    for path, emb in image_index
]
results.sort(key=lambda x: x[1], reverse=True)

print("Search results (text → images):")
for path, score in results:
    print(f"  {path}: similarity {score:.4f}")
# Output:
# Search results (text → images):
#   cat.jpg: similarity 0.8912
#   dog.jpg: similarity 0.6234
#   landscape.jpg: similarity 0.3421
```

## Building a Multimodal RAG System

Gemini Embedding 2 enables a unified RAG pipeline that spans text documents, images, and PDFs — without needing separate embedding pipelines for each modality.

```python
import os
import numpy as np
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

class MultimodalRAG:
    """A retrieval system that indexes text, images, and PDFs in one vector space."""

    def __init__(self):
        self.index = []  # List of (content_id, content_type, embedding)

    def add_text(self, doc_id, text):
        """Index a text document."""
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[text]
        )
        emb = np.array(result.embeddings[0].values)
        self.index.append((doc_id, "text", emb))
        print(f"✅ Indexed text: {doc_id}")

    def add_image(self, image_id, image_bytes):
        """Index an image."""
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[
                types.Part.from_bytes(data=image_bytes, mime_type="image/jpeg")
            ]
        )
        emb = np.array(result.embeddings[0].values)
        self.index.append((image_id, "image", emb))
        print(f"✅ Indexed image: {image_id}")

    def search(self, query, top_k=3):
        """Search the index using a text query."""
        result = client.models.embed_content(
            model="gemini-embedding-2-preview",
            contents=[query]
        )
        query_emb = np.array(result.embeddings[0].values)

        scores = []
        for content_id, content_type, emb in self.index:
            score = np.dot(query_emb, emb) / (
                np.linalg.norm(query_emb) * np.linalg.norm(emb)
            )
            scores.append((content_id, content_type, float(score)))

        scores.sort(key=lambda x: x[2], reverse=True)
        return scores[:top_k]

# Example usage
rag = MultimodalRAG()
rag.add_text("doc1", "The Gemini API enables advanced AI capabilities for developers.")
rag.add_text("doc2", "Multimodal AI processes text, images, audio, and video together.")

with open("architecture_diagram.jpg", "rb") as f:
    rag.add_image("img1", f.read())

results = rag.search("How does multimodal AI work?", top_k=3)
print("\nSearch results:")
for content_id, content_type, score in results:
    print(f"  [{content_type}] {content_id}: {score:.4f}")
```

## Migrating from gemini-embedding-001

If you're upgrading from `gemini-embedding-001`, there are a few important points to understand.

The two models use **completely different vector spaces**. You cannot compare embeddings from `gemini-embedding-001` with those from `gemini-embedding-2-preview`. This means a full re-embedding of your entire vector store is required.

Here's a recommended migration approach:

1. **Run both indexes in parallel**: Keep your existing `gemini-embedding-001` index serving production traffic while you build the new index with `gemini-embedding-2-preview`.
2. **Batch re-embed your data**: Use the batch embedding API to process your dataset efficiently.
3. **Switch over once complete**: Once the new index is fully built and validated, cut over and decommission the old index.

```python
# Migration script — re-embed existing text data in batches
existing_texts = ["text 1", "text 2", "text 3"]  # Your existing dataset
new_embeddings = []

batch_size = 10
for i in range(0, len(existing_texts), batch_size):
    batch = existing_texts[i:i+batch_size]
    result = client.models.embed_content(
        model="gemini-embedding-2-preview",
        contents=batch
    )
    for j, emb in enumerate(result.embeddings):
        new_embeddings.append({
            "text": batch[j],
            "embedding": emb.values
        })
    print(f"Progress: {min(i+batch_size, len(existing_texts))}/{len(existing_texts)}")

print(f"✅ Migration complete: {len(new_embeddings)} embeddings generated")
# Output: ✅ Migration complete: 3 embeddings generated
```

## Frequently Asked Questions

**Q1. Is Gemini Embedding 2 production-ready?**

As of March 2026, `gemini-embedding-2-preview` is in public preview. The API interface and pricing are subject to change before the stable release. For critical production systems, monitor the [official documentation](https://ai.google.dev/gemini-api/docs/models/gemini-embedding-2-preview) for GA availability.

**Q2. For text-only use cases, should I switch from gemini-embedding-001?**

For pure text workloads, both models perform well. However, if you anticipate adding image or audio search in the future, migrating to `gemini-embedding-2-preview` now avoids a costly full re-embedding later.

**Q3. Can I mix text and images in a single embedding request?**

Yes. You can pass interleaved text and image content in a single request, and the model will generate a unified embedding that reflects the combined multimodal context. This is useful for embedding product listings that include both a description and an image, for example.

**Q4. What vector database should I use with this model?**

Popular options include pgvector (PostgreSQL extension), Chroma, Pinecone, Weaviate, and Google Cloud's Vertex AI Vector Search. For large-scale deployments with 3,072-dimensional vectors, choose a solution that supports Approximate Nearest Neighbor (ANN) search to keep query latency low.

**Q5. How is Gemini Embedding 2 priced?**

Pricing details are available on the [Google AI Studio](https://aistudio.google.com) and [Vertex AI](https://cloud.google.com/vertex-ai) pricing pages. Preview pricing may differ from the final release.
