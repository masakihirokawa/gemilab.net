## Introduction — Why Gemini API with TypeScript?

As AI-powered applications become the norm, choosing the right development stack matters more than ever. **Gemini API** paired with **TypeScript** gives you the best of both worlds: access to Google's cutting-edge AI models with the safety net of static typing. TypeScript's type system catches API response mismatches at compile time, prevents runtime surprises, and makes your codebase far easier to maintain as it grows.

In this guide, you'll learn how to integrate the Gemini API into a TypeScript project from scratch. We'll walk through environment setup, basic text generation, streaming responses, multi-turn chat, structured output, and error handling — each with complete, runnable code examples.

## Setting Up Your Environment

Start by creating a new TypeScript project and installing the Google AI SDK.

```bash
# Initialize the project
mkdir gemini-ts-app && cd gemini-ts-app
npm init -y
npm install @google/genai typescript tsx
npx tsc --init
```

Here's the recommended `tsconfig.json` configuration:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "esModuleInterop": true,
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"]
}
```

Next, grab an API key from Google AI Studio at [aistudio.google.com](https://aistudio.google.com/). Store it as an environment variable rather than hardcoding it:

```bash
# Save to .env file
echo "GEMINI_API_KEY=YOUR_API_KEY" > .env
```

> **Important**: Never commit API keys to version control. Add `.env` to your `.gitignore` right away.

## Your First Request — Basic Text Generation

Let's make your first Gemini API call in TypeScript.

```typescript
// src/basic.ts
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function generateText(): Promise<void> {
  const response = await ai.models.generateContent({
    model: "gemini-2.5-flash",
    contents: "What are the three main benefits of TypeScript?",
  });

  console.log(response.text);
  // Expected output:
  // 1. Static typing catches bugs at compile time
  // 2. Better IDE support with autocompletion and refactoring
  // 3. Improved maintainability for large-scale projects
}

generateText();
```

Run it with `tsx`:

```bash
npx tsx src/basic.ts
```

That's all it takes. The `GoogleGenAI` class from the `@google/genai` package ships with built-in TypeScript definitions, so you get full IDE autocompletion out of the box — no separate `@types` package required.

## Streaming Responses — Real-Time Output

For chat-style interfaces where you want text to appear as it's generated, use the streaming API.

```typescript
// src/streaming.ts
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function streamResponse(): Promise<void> {
  const response = await ai.models.generateContentStream({
    model: "gemini-2.5-flash",
    contents: "Describe five creative use cases for the Gemini API.",
  });

  // Output each chunk as it arrives
  for await (const chunk of response) {
    if (chunk.text) {
      process.stdout.write(chunk.text);
    }
  }
  console.log("\n--- Streaming complete ---");
}

streamResponse();
```

Streaming dramatically improves perceived latency. Instead of waiting for the entire response to be generated, your users see tokens appearing in real time. This makes a huge difference in chatbots and interactive interfaces.

## Multi-Turn Conversations — Building a Chat Interface

Maintaining conversational context across multiple turns is straightforward with the chat API.

```typescript
// src/chat.ts
import { GoogleGenAI, type Content } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function chatDemo(): Promise<void> {
  // Type-safe conversation history
  const history: Content[] = [];

  const chat = ai.chats.create({
    model: "gemini-2.5-flash",
    history,
    config: {
      systemInstruction:
        "You are a TypeScript expert. Give concise, practical answers with code examples when relevant.",
    },
  });

  // First message
  const response1 = await chat.sendMessage({
    message: "What are TypeScript generics?",
  });
  console.log("AI:", response1.text);

  // Second message (context is preserved)
  const response2 = await chat.sendMessage({
    message: "Can you show me a concrete code example?",
  });
  console.log("AI:", response2.text);
}

chatDemo();
```

By importing the `Content` type, you get compile-time validation of your conversation history structure. TypeScript will flag invalid `role` values or missing fields before your code even runs.

## Structured Output — Type-Safe JSON Responses

One of Gemini API's most powerful features is **structured output**. By specifying a JSON schema, you can guarantee the response always follows a specific format — perfect for feeding AI results into downstream logic.

```typescript
// src/structured.ts
import { GoogleGenAI, Type } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

// Define the TypeScript interface
interface BookRecommendation {
  title: string;
  author: string;
  reason: string;
  difficulty: "beginner" | "intermediate" | "advanced";
}

async function getStructuredOutput(): Promise<void> {
  const response = await ai.models.generateContent({
    model: "gemini-2.5-flash",
    contents: "Recommend 3 books for learning TypeScript.",
    config: {
      responseMimeType: "application/json",
      responseSchema: {
        type: Type.ARRAY,
        items: {
          type: Type.OBJECT,
          properties: {
            title: { type: Type.STRING, description: "Book title" },
            author: { type: Type.STRING, description: "Author name" },
            reason: { type: Type.STRING, description: "Why it's recommended" },
            difficulty: {
              type: Type.STRING,
              enum: ["beginner", "intermediate", "advanced"],
              description: "Difficulty level",
            },
          },
          required: ["title", "author", "reason", "difficulty"],
        },
      },
    },
  });

  const books: BookRecommendation[] = JSON.parse(response.text ?? "[]");
  books.forEach((book) => {
    console.log(`📚 ${book.title} by ${book.author}`);
    console.log(`   Difficulty: ${book.difficulty}`);
    console.log(`   Why: ${book.reason}\n`);
  });
}

getStructuredOutput();
// Expected output:
// 📚 Programming TypeScript by Boris Cherny
//    Difficulty: intermediate
//    Why: Deep dive into the type system with practical examples
```

The combination of Gemini's structured output and TypeScript's type system means you can safely parse and process API responses without worrying about unexpected formats. For advanced schema design patterns, check out [Gemini API Structured Output Complete Guide](/en/articles/gemini-api/gemini-structured-output-advanced) which covers production-grade techniques in detail.

## Error Handling — Production-Ready Patterns

Robust error handling is essential for any API integration. Here's a pattern using exponential backoff with TypeScript's type narrowing.

```typescript
// src/error-handling.ts
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

async function safeGenerate(prompt: string): Promise<string | null> {
  const MAX_RETRIES = 3;

  for (let attempt = 1; attempt <= MAX_RETRIES; attempt++) {
    try {
      const response = await ai.models.generateContent({
        model: "gemini-2.5-flash",
        contents: prompt,
      });
      return response.text ?? null;
    } catch (error: unknown) {
      if (error instanceof Error) {
        // Rate limit — retry with backoff
        if (error.message.includes("429")) {
          const waitMs = Math.min(1000 * Math.pow(2, attempt), 30000);
          console.warn(
            `Rate limited — retrying in ${waitMs}ms (${attempt}/${MAX_RETRIES})`
          );
          await new Promise((r) => setTimeout(r, waitMs));
          continue;
        }
        // Authentication error
        if (error.message.includes("401") || error.message.includes("403")) {
          console.error("Auth error: please check your API key.");
          return null;
        }
      }
      console.error(`Unexpected error (attempt ${attempt}):`, error);
    }
  }
  console.error("Max retries reached.");
  return null;
}

// Usage
(async () => {
  const result = await safeGenerate("What is the biggest advantage of the Gemini API?");
  if (result) {
    console.log(result);
  }
})();
```

This exponential backoff pattern handles rate limits and transient network errors gracefully. In production, you'd want to layer on structured logging and alerting on top of this foundation.

## Frequently Asked Questions

### Q1. Where can I get the TypeScript SDK for Gemini API?

Install the `@google/genai` package from npm. It comes with bundled TypeScript type definitions — no separate `@types` package is needed. Just run `npm install @google/genai` and you're ready to go.

### Q2. Is the Gemini API free to use?

Yes, there's a generous free tier available through Google AI Studio. Gemini 2.5 Flash allows 15 requests per minute and 1,500 requests per day at no cost (as of March 2026). This is more than enough for learning and prototyping, though production workloads will benefit from a paid plan.

### Q3. Should I use TypeScript or JavaScript for Gemini API development?

TypeScript is strongly recommended. Gemini API responses can have deeply nested structures, and type definitions help catch property access errors and missing fields at compile time. This is especially valuable when working with structured output, where the type system ensures your parsed JSON matches the expected format.

### Q4. Does the SDK work with Deno or Bun?

The `@google/genai` package works with both Deno and Bun. For Deno, import it as `npm:@google/genai`. For Bun, install with `bun add @google/genai`. That said, Node.js remains the officially supported runtime.
