# Combining Built-in Tools and Function Calling in Gemini API: A Complete Guide

## Introduction

On March 18, 2026, Google launched a major new capability for the Gemini API: **combining built-in tools with custom Function Calling** in a single API call. This feature, powered by **Tool Context Circulation**, allows developers to use server-side built-in tools like Google Search, Google Maps, and Code Execution together with custom functions — all within a single, coherent conversation turn.

In this guide, you'll learn:

- What Tool Context Circulation is and how it works
- How to use the `include_server_side_tool_invocations` flag
- Full Python and JavaScript code examples
- How to preserve context across multi-turn conversations
- Practical agent patterns and real-world use cases

This guide assumes familiarity with Gemini's Function Calling. If you're new to it, check out the [Function Calling Complete Guide](/en/articles/gemini-advanced/function-calling-guide) first.

---

## What Is Tool Context Circulation?

Previously, the Gemini API required separate calls to use built-in tools (like Google Search) and custom functions. **Tool Context Circulation** changes this by enabling the model to:

1. Execute built-in tools server-side within a single response
2. Preserve the results (toolCall / toolResponse) in the model's context
3. Use that grounded context when invoking your custom function (functionCall)
4. Receive your function result and generate a final unified response

This dramatically simplifies agentic workflows like: "search the web for real-time data → execute business logic → synthesize and respond."

### Supported Built-in Tools

| Tool | Execution | Use Case |
|------|-----------|----------|
| Google Search | Server-side | Real-time web information |
| Google Maps | Server-side | Location and business data |
| URL Context | Server-side | Fetching web page content |
| File Search | Server-side | Document retrieval |
| Code Execution | Server-side | Running code and calculations |
| Custom Functions | Client-side | Any custom business logic |

> **Note**: This feature is currently in Preview and only supported on **Gemini 3 series models** (`gemini-3-flash-preview`, `gemini-3.1-pro-preview`, etc.).

## Python Implementation

The following example demonstrates a two-step workflow: use Google Search to find the northernmost US city, then call a custom function to get the weather there.

```python
from google import genai
from google.genai import types
import os

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

# Define your custom function
get_weather_tool = {
    "name": "getWeather",
    "description": "Gets the current weather for a given city.",
    "parameters": {
        "type": "object",
        "properties": {
            "city": {
                "type": "string",
                "description": "City name, e.g. Anchorage, Alaska",
            },
        },
        "required": ["city"],
    },
}

# Turn 1: Send query with both built-in tools and custom function
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents="What is the northernmost city in the United States? What's the weather like there today?",
    config=types.GenerateContentConfig(
        tools=[
            types.Tool(
                google_search=types.ToolGoogleSearch(),
                function_declarations=[get_weather_tool],
            ),
        ],
        # ⚠️ Required: enables tool context circulation
        include_server_side_tool_invocations=True,
    ),
)

# Parse the response parts
for part in response.candidates[0].content.parts:
    if hasattr(part, "function_call"):
        print(f"🔧 Custom function called: {part.function_call.name}")
        print(f"   Args: {part.function_call.args}")

# Turn 2: Provide function result and get final answer
# (In production, call your actual weather API here)
weather_data = {
    "city": "Utqiaġvik, Alaska",
    "temperature_celsius": -25,
    "condition": "Clear skies",
    "wind": "NW 15 km/h",
}

history = [
    types.Content(
        role="user",
        parts=[types.Part(text="What is the northernmost city in the United States? What's the weather like there today?")]
    ),
    # ⚠️ Important: include the full Turn 1 response to preserve thought_signatures
    response.candidates[0].content,
    types.Content(
        role="user",
        parts=[
            types.Part(
                function_response=types.FunctionResponse(
                    name="getWeather",
                    response=weather_data,
                )
            )
        ],
    ),
]

# Turn 2: Re-specify tools (required in every turn)
final_response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=history,
    config=types.GenerateContentConfig(
        tools=[
            types.Tool(
                google_search=types.ToolGoogleSearch(),
                function_declarations=[get_weather_tool],
            ),
        ],
        include_server_side_tool_invocations=True,
    ),
)

print("\n📝 Final answer:")
print(final_response.text)
# Expected output:
# The northernmost city in the United States is Utqiaġvik (formerly Barrow), Alaska.
# Today's weather there is clear with a temperature of -25°C and northwest winds at 15 km/h.
```

## JavaScript Implementation

```javascript
import { GoogleGenAI } from '@google/genai';

const client = new GoogleGenAI({ apiKey: process.env.GOOGLE_API_KEY });

const getWeatherTool = {
    name: "getWeather",
    description: "Gets the current weather for a given city",
    parameters: {
        type: "OBJECT",
        properties: {
            city: {
                type: "STRING",
                description: "City name, e.g. San Francisco, CA"
            }
        },
        required: ["city"]
    }
};

// Combine built-in and custom tools
const tools = [
    { googleSearch: {} },
    { functionDeclarations: [getWeatherTool] }
];

async function runCombinedTools() {
    // Turn 1: initial query
    const result = await client.models.generateContent({
        model: "gemini-3-flash-preview",
        contents: [{
            role: "user",
            parts: [{ text: "What's the northernmost US city and what's the weather there?" }]
        }],
        generationConfig: {
            includeServerSideToolInvocations: true,  // enable context circulation
        },
        tools,
    });

    const modelContent = result.candidates[0].content;

    // Find the functionCall part
    let functionCallPart = null;
    for (const part of modelContent.parts) {
        if (part.functionCall) {
            console.log(`🔧 Function called: ${part.functionCall.name}`);
            console.log(`   Args: ${JSON.stringify(part.functionCall.args)}`);
            functionCallPart = part;
        }
    }

    if (!functionCallPart) {
        console.log(result.candidates[0].content.parts[0].text);
        return;
    }

    // Simulate fetching weather data
    const weatherData = {
        city: functionCallPart.functionCall.args.city,
        temperature_celsius: -25,
        condition: "Clear skies",
        wind: "NW 15 km/h"
    };

    // Turn 2: Return function result in conversation history
    const finalResult = await client.models.generateContent({
        model: "gemini-3-flash-preview",
        contents: [
            { role: "user", parts: [{ text: "What's the northernmost US city and what's the weather there?" }] },
            modelContent, // ⚠️ Must include Turn 1 response unchanged (preserves thought_signatures)
            {
                role: "user",
                parts: [{
                    functionResponse: {
                        name: "getWeather",
                        response: weatherData
                    }
                }]
            }
        ],
        generationConfig: {
            includeServerSideToolInvocations: true,
        },
        tools,
    });

    console.log("\n📝 Final answer:");
    console.log(finalResult.candidates[0].content.parts[0].text);
}

runCombinedTools().catch(console.error);
```

## Critical: Preserving thought_signatures

The most important rule when using tool context circulation is **never modifying the Turn 1 response** before adding it to your conversation history. Each response part contains an encrypted `thought_signature` field that the model needs for the next turn.

```python
# ❌ Wrong: Manually reconstructing parts loses thought_signatures
wrong_history = [
    types.Content(role="model", parts=[
        types.Part(text=response.text)  # thought_signature is gone!
    ])
]

# ✅ Correct: Use the response object as-is
correct_history = [
    response.candidates[0].content  # includes all parts and thought_signatures
]
```

Learn more about thought signatures in the [Gemini Thought Signatures Guide](/en/articles/gemini-api/gemini-thought-signatures-guide).

## Common Errors and Fixes

### Error 1: Missing `include_server_side_tool_invocations`

**Symptom**: Built-in tool results are not included in context; custom function has no grounded data to work with.

**Fix**: Add `include_server_side_tool_invocations=True` (Python) or `includeServerSideToolInvocations: true` (JS) to every `generate_content` call.

### Error 2: Missing thought_signatures

**Symptom**: `InvalidArgument` or `FAILED_PRECONDITION` error on Turn 2.

**Fix**: Don't reconstruct history manually. Include the full Turn 1 response object unchanged.

### Error 3: Using AUTO function calling mode

**Symptom**: Error when setting `FUNCTION_CALLING_MODE.AUTO`.

**Fix**: Use the default `VALIDATED` mode. AUTO mode is not supported with tool combinations.

## FAQ

**Q1. Does this work with Gemini 2.5 models?**

A1. No. Tool Context Circulation is currently available only for **Gemini 3 series models** (e.g., `gemini-3-flash-preview`, `gemini-3.1-pro-preview`). For Gemini 2.5, you'll need to manage built-in tools and custom functions in separate API calls.

**Q2. Can I use multiple built-in tools at the same time?**

A2. Yes. You can include multiple built-in tools in a single `types.Tool` object alongside your function declarations. For example, combining Google Search and Code Execution simultaneously is supported. The model autonomously decides which tools to invoke based on the query.

**Q3. How do I handle multiple function calls in one turn?**

A3. The model may return multiple `functionCall` parts in a single response (parallel function calling). Loop through all parts in `response.candidates[0].content.parts`, collect every `function_call`, execute them all, then send back all `functionResponse` parts in a single Turn 2 request.

**Q4. What are the cost implications?**

A4. Each built-in tool invocation consumes tokens, and Google Search grounding may incur per-use charges. To control costs, configure project spend caps in Google AI Studio and review the [Gemini API Cost Optimization Guide](/en/articles/gemini-api/gemini-api-cost-optimization).

## Summary

Tool Context Circulation — the ability to combine Gemini's built-in tools with custom Function Calling — is a landmark feature for building sophisticated AI agents. Here's what to remember:

- Always set `include_server_side_tool_invocations=True` (or its JS equivalent)
- Never reconstruct Turn 1 responses manually — pass them as-is to preserve `thought_signatures`
- Use `VALIDATED` mode (default), not `AUTO`
- This feature is Gemini 3 exclusive (Preview as of March 2026)

With this capability, you can build workflows that ground themselves in real-time data and execute your business logic in a single, elegant interaction — dramatically simplifying complex agentic architectures.
