# Gemini 3 Flash Agentic Vision Guide — AI That Sees, Thinks, and Acts on Images

## What Is Agentic Vision?

Traditional AI image recognition has always been passive — you feed in an image, and the model describes what it sees. That's it. But **Agentic Vision**, introduced by Google in January 2026, fundamentally changes this paradigm.

Agentic Vision is a capability built into Gemini 3 Flash that combines visual reasoning with Python code execution. Instead of simply describing an image, the model **actively investigates** it — generating and running code to crop, annotate, measure, and manipulate images before delivering its answer. The result is responses grounded in visual evidence rather than plausibility.

In this guide, you'll learn how Agentic Vision works under the hood, how to enable it in your applications, and how to apply it to real-world use cases. This article is aimed at developers who are familiar with the Gemini API basics and want to take their image analysis capabilities to the next level.

## How It Works — The Think-Act-Observe Loop

At the core of Agentic Vision is an iterative **Think-Act-Observe** loop that drives the model's reasoning process.

### Think

The model analyzes the user's query alongside the input image and formulates a multi-step plan. For instance, when asked "Count the number of buildings in this photo," the model might plan to segment the image, detect building outlines, and count them individually.

### Act

Based on this plan, Gemini 3 Flash generates and executes Python code. It can crop, rotate, annotate, count objects, run numerical calculations, and perform a variety of other image operations.

### Observe

The modified image is fed back into the model's context window. The model re-examines the updated visual data and decides whether to loop back for further analysis or generate a final response.

This iterative process enables a level of **precision in visual analysis** that simply isn't possible with a single-pass image description.

## Basic Implementation — Enabling Agentic Vision

To activate Agentic Vision, you need to enable the `code_execution` tool and configure the `thinking_config`. Here's a minimal example:

```python
from google import genai
from google.genai import types
import os

# Initialize the client
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Upload an image file
image = client.files.upload(file="building_photo.jpg")

# Make a request with Agentic Vision enabled
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=[image, "Count the exact number of buildings in this photo"],
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)],
        thinking_config=types.ThinkingConfig(
            thinking_level="HIGH",
        ),
    ),
)

# Display the results
for part in response.candidates[0].content.parts:
    if part.text:
        print(part.text)
    elif part.executable_code:
        print(f"[Code Executed]\n{part.executable_code.code}")
    elif part.code_execution_result:
        print(f"[Execution Result]\n{part.code_execution_result.output}")
```

**Expected output:**

```
[Thinking] Analyzing the image to detect building outlines...

[Code Executed]
from PIL import Image
import numpy as np
# Performing edge detection and region segmentation
...

[Execution Result]
Buildings detected: 7

There are 7 buildings visible in this photo. Three low-rise structures
on the left, two mid-rise buildings in the center, and two high-rise
towers on the right.
```

The two key ingredients are the `code_execution` tool, which lets the model run Python code, and the `thinking_level` set to `"HIGH"` for deeper reasoning.

## Practical Example 1 — Automated Product Image Analysis

Here's how you might use Agentic Vision to automatically analyze e-commerce product images and extract structured data.

```python
from google import genai
from google.genai import types
import json
import os

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

def analyze_product_image(image_path: str) -> str:
    """Analyze a product image and return structured data."""
    image = client.files.upload(file=image_path)

    prompt = """
    Analyze this product image in detail.
    Output the following information in JSON format:
    - product_name: estimated product name
    - colors: list of colors used
    - dimensions_ratio: aspect ratio (width:height)
    - text_on_image: any text visible in the image
    - quality_score: quality score as a product photo (1-10)
    - suggestions: list of improvement suggestions

    Use code execution to manipulate the image for accurate results.
    If text is small, crop and enlarge that area before reading it.
    """

    response = client.models.generate_content(
        model="gemini-3-flash-preview",
        contents=[image, prompt],
        config=types.GenerateContentConfig(
            tools=[types.Tool(code_execution=types.ToolCodeExecution)],
            thinking_config=types.ThinkingConfig(thinking_level="HIGH"),
        ),
    )

    # Extract text parts from the response
    result_text = ""
    for part in response.candidates[0].content.parts:
        if part.text:
            result_text += part.text

    return result_text

# Usage
result = analyze_product_image("product_sneaker.jpg")
print(result)
```

With Agentic Vision enabled, the model **automatically crops and enlarges** areas with small text, achieving accuracy that traditional image recognition can't match.

## Optimizing Performance and Cost

Agentic Vision is powerful, but code execution increases token consumption compared to standard requests. Here's how to keep costs in check.

### Choosing the Right thinking_level

| thinking_level | Best For | Token Usage |
|---|---|---|
| `LOW` | Simple classification, captioning | Low |
| `MEDIUM` | General image analysis | Moderate |
| `HIGH` | Precise measurement, data extraction | High |

Use `LOW` for straightforward tasks and reserve `HIGH` for when you genuinely need precision.

### Combining with media_resolution

Pair Agentic Vision with the [media_resolution parameter](/en/articles/gemini-api/gemini-3-media-resolution-api-guide) introduced in Gemini 3 to fine-tune the accuracy-cost balance.

```python
# High-resolution detailed analysis
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=[
        types.Content(
            parts=[
                types.Part.from_uri(
                    file_uri=image.uri,
                    mime_type="image/jpeg",
                    video_metadata=types.VideoMetadata(
                        media_resolution="HIGH"
                    ),
                ),
                types.Part.from_text("Analyze the fine details of this image"),
            ]
        )
    ],
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)],
        thinking_config=types.ThinkingConfig(thinking_level="HIGH"),
    ),
)
```

### Reducing Cost with Batch Processing

For high-volume image processing, consider using the [Batch API](/en/articles/gemini-api/gemini-batch-processing-api-guide) for asynchronous processing at up to 50% cost savings.

## When Agentic Vision Shines

Agentic Vision is particularly effective in these scenarios.

**Precision-critical tasks**: Reading chart values, measuring architectural drawings, or analyzing medical images where "close enough" isn't good enough.

**Reading small text**: Extracting annotations, labels, model numbers, or fine print from images by automatically cropping and enlarging the relevant areas.

**Counting tasks**: Accurately counting objects in an image using code-driven segmentation and masking — far more reliable than visual estimation.

**Comparative analysis**: Detecting differences between two images, quantifying changes, and performing multi-image analysis.

## Frequently Asked Questions

### Q1: Can I use Agentic Vision with Gemini 3 Pro?

Agentic Vision is currently optimized for **Gemini 3 Flash**. While Gemini 3 Pro supports code execution, the Agentic Vision workflow is specifically tuned for the Flash model. Given Flash's excellent cost-performance ratio, we recommend using [Gemini 3 Flash](/en/articles/gemini-basics/gemini-3-flash-guide) for image analysis tasks.

### Q2: Are there limitations on the Python code that can be executed?

Code runs in a sandboxed environment, so there's no access to external networks or persistent file system writes. Available libraries are limited to core scientific computing and image processing packages like `numpy`, `PIL (Pillow)`, and `matplotlib`.

### Q3: Should I use standard image recognition or Agentic Vision?

For image captioning or broad classification, standard image recognition (without code execution) works just fine. For tasks requiring **precision** — exact numerical readings, fine-detail analysis, or text extraction from images — Agentic Vision is the way to go. Keep in mind that code execution adds latency and cost, so choose based on your task requirements.

### Q4: How long does an Agentic Vision response take?

It depends on how many Think-Act-Observe iterations are needed, but generally expect 2–5x the latency of a standard request. Simple tasks typically complete in 3–5 seconds, while complex analyses may take 10–20 seconds. For latency-sensitive applications, consider pairing Agentic Vision with asynchronous processing.
