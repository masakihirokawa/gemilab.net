The **Gemini API URL Context tool** lets you pass URLs — web pages, PDFs, images, and more — directly to the model, which then fetches and processes the content in real time before generating a response. It reached general availability in February 2026, with expanded support for PDFs and image formats. No manual downloads or file uploads required: just provide a URL and get context-aware answers instantly.

This guide covers everything from basic usage to advanced patterns, with working code examples in Python and JavaScript.

## What Is the URL Context Tool?

The URL Context tool enables Gemini to fetch and read the content at one or more URLs, incorporating that information into its response. Unlike the [File API](/en/articles/gemini-api/file-api-guide), which requires uploading files in advance, the URL Context tool simply accepts a URL string and handles retrieval automatically.

Common use cases include:

- Summarizing or translating news articles and blog posts
- Answering questions based on official documentation or PDF reports
- Comparing and analyzing content across multiple web pages
- Interpreting images from URLs using Gemini's multimodal capabilities

### Supported Content Types

| Format | Support |
|---|---|
| Web pages (HTML) | ✅ |
| PDF (text, tables, structure) | ✅ |
| Images (PNG, JPEG, BMP, WebP) | ✅ |
| JSON / XML / CSV | ✅ |
| Plain text / RTF | ✅ |
| JavaScript / CSS | ✅ |

## Basic Usage

### Python — Summarize a Web Page

```python
import os
from google import genai
from google.genai import types

# Initialize the client
client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Use the URL Context tool to fetch and summarize a web page
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        "Please summarize the key points from this article in 3 bullet points: "
        "https://blog.google/technology/ai/"
    ),
    config=types.GenerateContentConfig(
        tools=[types.Tool(url_context=types.UrlContext())]
    ),
)

print(response.text)
# Expected output:
# • Google announced the Gemini 3 series, featuring advanced reasoning and multimodal…
# • The new Computer Use tool is now available in Gemini 3 Pro and Flash…
# • Veo video generation now supports 4K output resolution…
```

### JavaScript — Summarize a Web Page

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

async function summarizeUrl(url) {
  const response = await ai.models.generateContent({
    model: "gemini-2.5-pro",
    contents: `Summarize the key points of this article in 3 bullet points: ${url}`,
    config: {
      tools: [{ urlContext: {} }],
    },
  });
  return response.text;
}

// Example usage
const summary = await summarizeUrl("https://blog.google/technology/ai/");
console.log(summary);
// Expected output:
// • Google's Gemini 3 series brings next-generation reasoning…
// • Computer Use tool enables agentic task automation…
```

## Analyzing PDFs

Pass a direct PDF URL and Gemini will extract text, tables, and structural information — no preprocessing needed.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

# Analyze a publicly accessible PDF
pdf_url = "https://example.com/annual-report-2025.pdf"

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        f"Extract the key financial figures from this report and present them "
        f"as a structured list: {pdf_url}"
    ),
    config=types.GenerateContentConfig(
        tools=[types.Tool(url_context=types.UrlContext())]
    ),
)

print(response.text)
# Expected output:
# Annual Report 2025 - Key Financial Figures:
# - Total Revenue: $X billion (+XX% YoY)
# - Operating Income: $X billion…
# - Net Profit Margin: XX%…
```

## Referencing Multiple URLs

A single request can reference multiple URLs simultaneously for comparison and cross-analysis.

```python

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

response = client.models.generate_content(
    model="gemini-2.5-pro",

        "Article 1: https://example.com/article-a\n"
        "Article 2: https://example.com/article-b"

    config=types.GenerateContentConfig(

    ),
)

print(response.text)
# Expected output:
# Similarities:
# - Both articles focus on LLM application patterns…

# - Article 1 emphasizes cost optimization strategies, while Article 2…
```

## Combining with Google Search Grounding

The URL Context tool works alongside [Grounding with Google Search](/en/articles/gemini-advanced/grounding-and-search), letting you blend content from specific URLs with up-to-date web search results.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        "Based on this document and the latest available information, "
        "explain the current state of Gemini 2.5 Pro features: "
        "https://example.com/gemini-overview.pdf"
    ),
    config=types.GenerateContentConfig(
        tools=[
            types.Tool(url_context=types.UrlContext()),
            types.Tool(google_search=types.GoogleSearch()),
        ]
    ),
)

print(response.text)
```

> **Note:** Combining both tools increases token consumption. Use this pattern when you need both a specific document and current web context.

## Common Errors and Solutions

### Error: URL Not Accessible

Private URLs, pages requiring authentication, or access-restricted domains cannot be fetched by Gemini.

```
Error: Failed to fetch URL content. Ensure the URL is publicly accessible.
```

**Solution:** Use only publicly accessible URLs. For authenticated content, use the [File API](/en/articles/gemini-api/file-api-guide) to upload the content directly.

### Error: Context Window Exceeded

This occurs when the combined content from all URLs exceeds the model's context limit.

**Solution:** Reduce the number of URLs per request, or use [Context Caching](/en/articles/gemini-api/context-caching-guide) to handle large documents efficiently.

### Error: Dynamic Content Not Retrieved

JavaScript-rendered content may not be captured since Gemini fetches static HTML.

**Solution:** Use the URL of a static endpoint or a direct API response URL that returns plain content.

## Real-World Example — Documentation Q&A Bot

The URL Context tool makes it straightforward to build a Q&A system grounded in official documentation.

```python
import os
from google import genai
from google.genai import types

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

DOC_URLS = [
    "https://cloud.google.com/vertex-ai/docs",
    "https://ai.google.dev/gemini-api/docs",
]

def ask_docs(question: str, urls: list[str]) -> str:
    url_list = "\n".join(f"- {u}" for u in urls)
    prompt = (
        f"Using the following official documentation pages as your source, "
        f"answer the question accurately.\n\n"
        f"Documentation:\n{url_list}\n\n"
        f"Question: {question}"
    )
    response = client.models.generate_content(
        model="gemini-2.5-pro",
        contents=prompt,
        config=types.GenerateContentConfig(
            tools=[types.Tool(url_context=types.UrlContext())]
        ),
    )
    return response.text

# Example
answer = ask_docs(
    "How do I enable streaming responses in the Gemini API?",
    DOC_URLS,
)
print(answer)

```

## FAQ

**Q. Does using the URL Context tool cost extra?**

A. The content fetched from URLs is counted as input tokens and billed at the standard Gemini API input token rate. The tool itself has no additional charge. Be mindful of token usage when referencing large documents or multiple URLs.

**Q. What is the difference between the URL Context tool and the File API?**

A. The URL Context tool fetches public content automatically from a URL at request time. The [File API](/en/articles/gemini-api/file-api-guide) requires manually uploading a file first, which is necessary for private content, local files, or large files up to 100MB. For publicly accessible resources, the URL Context tool is the simpler choice.

**Q. How many URLs can I pass in a single request?**

A. There is no hard-coded limit on the number of URLs, but each URL's content consumes tokens from the context window. Gemini 2.5 Pro supports up to 1 million tokens, but practical stability is typically best at 5–10 URLs per request.

**Q. Can I use this with Gemini 2.5 Flash?**

A. Yes. The URL Context tool is supported on `gemini-2.5-flash` and other major Gemini models. Flash is a great choice when you want lower latency and cost for straightforward URL-based tasks.
