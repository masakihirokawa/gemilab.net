## Introduction

Calendar management is one of those tasks that quietly eats into your productivity. Juggling meetings, finding open slots, and avoiding double-bookings — it all adds up.

In this guide, you'll build an **AI scheduling assistant** that combines **Gemini API** with **Google Calendar API**. The result? A conversational interface where you can say things like "Schedule a design review next Wednesday at 2pm" or "What's my availability this week?" and the assistant handles the rest.

The secret sauce is Gemini's [Function Calling](/en/articles/gemini-advanced/function-calling-guide) capability, which lets the model interpret natural language and translate it into the right Calendar API calls — no rigid command parsing required.

### Set Up Google Calendar Authentication

Create an OAuth 2.0 client ID in the Google Cloud Console and save it as `credentials.json` in your project root. The first time you run the script, a browser window will open for authorization, and a `token.json` file will be created automatically.

```python
import os
from google.auth.transport.requests import Request
from google.oauth2.credentials import Credentials
from google_auth_oauthlib.flow import InstalledAppFlow
from googleapiclient.discovery import build

SCOPES = ["https://www.googleapis.com/auth/calendar"]

def get_calendar_service():
    """Initialize and return the Google Calendar API service."""
    creds = None
    if os.path.exists("token.json"):
        creds = Credentials.from_authorized_user_file("token.json", SCOPES)
    if not creds or not creds.valid:
        if creds and creds.expired and creds.refresh_token:
            creds.refresh(Request())
        else:
            flow = InstalledAppFlow.from_client_secrets_file("credentials.json", SCOPES)
            creds = flow.run_local_server(port=0)
        with open("token.json", "w") as token:
            token.write(creds.to_json())
    return build("calendar", "v3", credentials=creds)
```

## Configuring Function Calling with Gemini

The core of the assistant is Gemini's Function Calling feature. You define the calendar operations as tool declarations, and Gemini figures out which function to call and with what arguments based on the user's natural language input.

```python
import google.generativeai as genai
from datetime import datetime, timedelta

genai.configure(api_key=os.environ["GEMINI_API_KEY"])

# Define calendar operation tools
calendar_tools = [
    {
        "function_declarations": [
            {
                "name": "create_event",
                "description": "Create a new event on Google Calendar",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "summary": {"type": "string", "description": "Event title"},
                        "start_time": {"type": "string", "description": "Start time in ISO 8601 format (e.g., 2026-03-26T10:00:00+09:00)"},
                        "end_time": {"type": "string", "description": "End time in ISO 8601 format"},
                        "description": {"type": "string", "description": "Event description (optional)"},
                        "location": {"type": "string", "description": "Event location (optional)"},
                    },
                    "required": ["summary", "start_time", "end_time"],
                },
            },
            {
                "name": "list_events",
                "description": "Retrieve calendar events within a specified date range",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "start_date": {"type": "string", "description": "Start date (YYYY-MM-DD)"},
                        "end_date": {"type": "string", "description": "End date (YYYY-MM-DD)"},
                    },
                    "required": ["start_date", "end_date"],
                },
            },
            {
                "name": "find_free_slots",
                "description": "Find available time slots within a date range",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "start_date": {"type": "string", "description": "Search start date (YYYY-MM-DD)"},
                        "end_date": {"type": "string", "description": "Search end date (YYYY-MM-DD)"},
                        "duration_minutes": {"type": "integer", "description": "Required slot duration in minutes"},
                    },
                    "required": ["start_date", "end_date", "duration_minutes"],
                },
            },
        ]
    }
]

model = genai.GenerativeModel(
    "gemini-2.5-flash",
    tools=calendar_tools,
    system_instruction=(
        "You are a scheduling assistant. "
        f"The current date and time is {datetime.now().isoformat()}. "
        "Help users manage their calendar by creating events, listing schedules, "
        "and finding available time slots. When a user specifies a day of the week, "
        "use the nearest upcoming occurrence."
    ),
)
```

## Implementing the Calendar Operations

Now let's write the actual functions that get called when Gemini triggers a function call.

```python
def create_event(service, summary, start_time, end_time, description="", location=""):
    """Create a calendar event and return confirmation."""
    event = {
        "summary": summary,
        "start": {"dateTime": start_time, "timeZone": "Asia/Tokyo"},
        "end": {"dateTime": end_time, "timeZone": "Asia/Tokyo"},
    }
    if description:
        event["description"] = description
    if location:
        event["location"] = location

    created = service.events().insert(calendarId="primary", body=event).execute()
    return f"Event created: {created['summary']} ({created['start']['dateTime']})"


def list_events(service, start_date, end_date):
    """Retrieve all events within the specified date range."""
    time_min = f"{start_date}T00:00:00+09:00"
    time_max = f"{end_date}T23:59:59+09:00"

    events_result = (
        service.events()
        .list(
            calendarId="primary",
            timeMin=time_min,
            timeMax=time_max,
            singleEvents=True,
            orderBy="startTime",
        )
        .execute()
    )
    events = events_result.get("items", [])

    if not events:
        return "No events found in the specified period."

    result = []
    for event in events:
        start = event["start"].get("dateTime", event["start"].get("date"))
        result.append(f"- {start}: {event['summary']}")
    return "\n".join(result)


def find_free_slots(service, start_date, end_date, duration_minutes):
    """Find available time slots during business hours (9 AM - 6 PM)."""
    time_min = f"{start_date}T00:00:00+09:00"
    time_max = f"{end_date}T23:59:59+09:00"

    body = {
        "timeMin": time_min,
        "timeMax": time_max,
        "items": [{"id": "primary"}],
    }
    freebusy = service.freebusy().query(body=body).execute()
    busy_periods = freebusy["calendars"]["primary"]["busy"]

    # Calculate free slots within business hours
    free_slots = []
    current = datetime.fromisoformat(f"{start_date}T09:00:00+09:00")
    end = datetime.fromisoformat(f"{end_date}T18:00:00+09:00")

    while current < end:
        slot_end = current + timedelta(minutes=duration_minutes)
        if slot_end.hour > 18:
            current = current.replace(hour=9, minute=0) + timedelta(days=1)
            continue

        is_free = True
        for busy in busy_periods:
            busy_start = datetime.fromisoformat(busy["start"])
            busy_end = datetime.fromisoformat(busy["end"])
            if current < busy_end and slot_end > busy_start:
                is_free = False
                current = busy_end
                break

        if is_free:
            free_slots.append(f"- {current.strftime('%m/%d %H:%M')} - {slot_end.strftime('%H:%M')}")
            current = slot_end

    if not free_slots:
        return "No available slots found in the specified period."
    return f"Available slots ({duration_minutes}+ minutes):\n" + "\n".join(free_slots[:10])
```

## Building the Conversation Loop

The final piece ties everything together — a chat loop that takes natural language input, lets Gemini decide which function to call, executes it, and returns the results in a conversational format.

```python
def run_assistant():
    """Main loop for the AI scheduling assistant."""
    service = get_calendar_service()
    chat = model.start_chat()

    # Map function names to their implementations
    func_map = {
        "create_event": lambda args: create_event(service, **args),
        "list_events": lambda args: list_events(service, **args),
        "find_free_slots": lambda args: find_free_slots(service, **args),
    }

    print("AI Schedule Assistant is ready.")
    print("   Manage your calendar using natural language (type 'quit' to exit)\n")

    while True:
        user_input = input("You > ").strip()
        if user_input.lower() in ("quit", "exit"):
            print("Goodbye!")
            break

        response = chat.send_message(user_input)

        # Process Function Call responses
        while response.candidates[0].content.parts[0].function_call.name:
            fc = response.candidates[0].content.parts[0].function_call
            func_name = fc.name
            func_args = dict(fc.args)

            print(f"  Running {func_name}...")
            result = func_map[func_name](func_args)

            # Send function result back to Gemini
            response = chat.send_message(
                genai.protos.Content(
                    parts=[
                        genai.protos.Part(
                            function_response=genai.protos.FunctionResponse(
                                name=func_name,
                                response={"result": result},
                            )
                        )
                    ]
                )
            )

        print(f"Assistant > {response.text}\n")


if __name__ == "__main__":
    run_assistant()
```

## Common Errors and Fixes

### OAuth Token Expired

If you change the API scopes in your `credentials.json`, you'll need to delete `token.json` and re-authenticate:

```bash
rm token.json
python calendar_assistant.py  # triggers re-auth flow
```

### Timezone Mismatches

Gemini may occasionally return times in UTC. Explicitly set the timezone in your system instruction, and always convert to your local timezone in the calendar functions.

### Rate Limits

The Google Calendar API default quota is 1,000,000 requests per day — more than enough for personal use. For multi-user deployments, implement exponential backoff. For a deeper understanding of resilient API patterns, [Designing Data-Intensive Applications](https://www.amazon.co.jp/dp/4873119049/?tag=pinocchio-22) by Martin Kleppmann is an excellent resource.

## Taking It Further

### Weekly Time Reports

Fetch a week's worth of events and have Gemini analyze the breakdown — "You spent 15 hours in meetings this week and only 8 hours on focused work." This kind of insight helps you make better scheduling decisions.

```python
def weekly_report(service, chat):
    today = datetime.now()
    start = (today - timedelta(days=today.weekday())).strftime("%Y-%m-%d")
    end = (today + timedelta(days=6 - today.weekday())).strftime("%Y-%m-%d")
    events = list_events(service, start, end)

    response = chat.send_message(
        f"Analyze this week's schedule and provide a time allocation report:\n{events}"
    )
    return response.text
```

## FAQ

### Q1: Can I use this with Gemini's free tier?

Absolutely. Gemini 2.5 Flash's free tier allows 30 requests per minute, which is plenty for personal scheduling. If you're building this for a team or running batch operations, consider upgrading to the [pay-as-you-go plan](https://aistudio.google.com/).

### Q2: Is there a Google Apps Script alternative?

Yes — if you prefer a serverless approach that runs entirely within Google's ecosystem, you can implement this using Google Apps Script. See our [Gemini × Google Apps Script Automation Guide](/en/articles/gemini-dev/gemini-google-apps-script-automation) for a detailed walkthrough. GAS also makes it easy to set up time-based triggers for automated scheduling tasks.

### Q3: What about security considerations?

Your `token.json` contains sensitive OAuth credentials — never commit it to version control (add it to `.gitignore`). Store your Gemini API key in environment variables rather than hardcoding it. For production deployments, use [Google Secret Manager](https://cloud.google.com/secret-manager) to manage all credentials securely.
