## Why Apps Script × Gemini?

Google Sheets serves as a central hub for business data across organizations of all sizes. Sales figures, KPIs, marketing metrics, inventory data — it all ends up in spreadsheets. Yet the process of analyzing this data and producing regular reports remains painfully manual in most teams.

Apps Script is a serverless execution environment built into Google Workspace. By integrating Gemini API into this environment, you can build a complete pipeline that reads spreadsheet data, runs AI-powered analysis, generates formatted reports, and delivers them to stakeholders — all through code.

This guide walks you through the entire process: from the basics of calling Gemini API in Apps Script to building a production-grade report generation pipeline with error handling, cost controls, and automated scheduling.

For an introduction to AI features in Google Sheets, see our [Google Sheets × Gemini AI Practical Guide](/en/articles/gemini-workspace/gemini-google-sheets-ai-data-analysis). This article goes a step further, focusing on full automation architecture using Apps Script.

**Who this is for:**
- Professionals who work with Google Sheets daily
- Developers familiar with Apps Script basics (we'll cover setup for newcomers too)
- Technical leaders interested in AI-powered business automation

## Setting Up Your Environment and Securing API Keys

### Getting Your API Key from Google AI Studio

To use the Gemini API, start by generating an API key at [Google AI Studio](https://aistudio.google.com/). Log in with your Google account and click "Get API key" in the left menu to create a new key.

### Creating an Apps Script Project

Open any Google Sheet, then navigate to Extensions → Apps Script. This opens the script editor where you'll write all your code.

### Storing API Keys Securely with Script Properties

**Important**: Never hardcode API keys in your source code. Apps Script provides `PropertiesService`, a secure key-value store designed for exactly this purpose.

```javascript
// One-time setup function (run once)
function setupApiKey() {
  const scriptProperties = PropertiesService.getScriptProperties();
  scriptProperties.setProperty('GEMINI_API_KEY', 'YOUR_API_KEY_HERE');
  console.log('API key saved to Script Properties');
}

// Retrieve API key (used in subsequent code)
function getApiKey() {
  return PropertiesService.getScriptProperties()
    .getProperty('GEMINI_API_KEY');
}
```

After running `setupApiKey()` once, delete the `YOUR_API_KEY_HERE` value from your code. The stored value can also be viewed and edited from the project settings panel.

## Core Pattern: Calling Gemini API from Apps Script

Let's build a reusable client class that handles API communication. This class serves as the foundation for everything else.

```javascript
/**
 * GeminiClient — Gemini API client for Apps Script
 * Built-in retry, timeout, and error handling
 */
class GeminiClient {
  constructor() {
    this.apiKey = getApiKey();
    this.baseUrl = 'https://generativelanguage.googleapis.com/v1beta';
    this.model = 'models/gemini-2.5-flash';
    this.maxRetries = 3;
  }

  /**
   * Send a text generation request
   * @param {string} prompt - The prompt
   * @param {Object} options - Additional options
   * @returns {string} Generated text
   */
  generate(prompt, options = {}) {
    const url = `${this.baseUrl}/${this.model}:generateContent?key=${this.apiKey}`;

    const payload = {
      contents: [{
        parts: [{ text: prompt }]
      }],
      generationConfig: {
        temperature: options.temperature || 0.3,
        maxOutputTokens: options.maxTokens || 4096,
        topP: options.topP || 0.95
      }
    };

    // Add system instruction if provided
    if (options.systemInstruction) {
      payload.systemInstruction = {
        parts: [{ text: options.systemInstruction }]
      };
    }

    return this._fetchWithRetry(url, payload);
  }

  /**
   * HTTP request with exponential backoff retry
   */
  _fetchWithRetry(url, payload) {
    for (let attempt = 1; attempt <= this.maxRetries; attempt++) {
      try {
        const response = UrlFetchApp.fetch(url, {
          method: 'post',
          contentType: 'application/json',
          payload: JSON.stringify(payload),
          muteHttpExceptions: true
        });

        const code = response.getResponseCode();
        const body = JSON.parse(response.getContentText());

        if (code === 200) {
          return body.candidates[0].content.parts[0].text;
        }

        // Retry on rate limit (429) or server errors (5xx)
        if (code === 429 || code >= 500) {
          const waitMs = Math.pow(2, attempt) * 1000 + Math.random() * 1000;
          console.log(`Retry ${attempt}/${this.maxRetries} after ${waitMs}ms (HTTP ${code})`);
          Utilities.sleep(waitMs);
          continue;
        }

        // Fail immediately on other errors
        throw new Error(`Gemini API error ${code}: ${JSON.stringify(body)}`);

      } catch (e) {
        if (attempt === this.maxRetries) throw e;
        Utilities.sleep(Math.pow(2, attempt) * 1000);
      }
    }
    throw new Error('Max retries exceeded');
  }
}
```

**Design decisions worth noting:**
- `muteHttpExceptions: true` lets us handle HTTP errors as responses rather than thrown exceptions
- 429 (rate limit) and 5xx (server errors) trigger exponential backoff retries
- 4xx errors (bad requests) fail immediately — no point retrying a malformed request

## Reading Spreadsheet Data and Running AI Analysis

### The Data Extraction Layer

First, we need a clean way to pull structured data from sheets.

```javascript
/**
 * Extract data from a sheet as a structured object
 * @param {string} sheetName - Name of the sheet
 * @returns {Object} Headers and data rows
 */
function extractSheetData(sheetName) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName(sheetName);

  if (!sheet) {
    throw new Error(`Sheet "${sheetName}" not found`);
  }

  const data = sheet.getDataRange().getValues();
  const headers = data[0];
  const rows = data.slice(1).filter(row =>
    row.some(cell => cell !== '' && cell !== null)
  );

  return {
    sheetName,
    headers,
    rows,
    rowCount: rows.length,
    summary: `${sheetName}: ${rows.length} rows × ${headers.length} columns`
  };
}

/**
 * Calculate basic statistics for a numeric column
 * @param {Array} data - Output from extractSheetData
 * @param {string} columnName - Target column name
 * @returns {Object} Statistical measures
 */
function calculateStats(data, columnName) {
  const colIndex = data.headers.indexOf(columnName);
  if (colIndex === -1) throw new Error(`Column "${columnName}" not found`);

  const values = data.rows
    .map(row => Number(row[colIndex]))
    .filter(v => !isNaN(v));

  values.sort((a, b) => a - b);
  const sum = values.reduce((a, b) => a + b, 0);
  const avg = sum / values.length;

  return {
    count: values.length,
    sum: sum,
    average: Math.round(avg * 100) / 100,
    min: values[0],
    max: values[values.length - 1],
    median: values[Math.floor(values.length / 2)]
  };
}
```

### Designing Effective Analysis Prompts

The quality of AI-generated reports depends heavily on prompt design.

```javascript
/**
 * Build an analysis prompt for sales data
 * @param {Object} data - Extracted data
 * @param {Object} stats - Calculated statistics
 * @param {string} period - Analysis period (e.g., "March 2026")
 * @returns {string} Analysis prompt
 */
function buildAnalysisPrompt(data, stats, period) {
  // Convert data to CSV format (saves tokens)
  const csvHeader = data.headers.join(',');
  const csvRows = data.rows.slice(0, 100) // Limit to 100 rows
    .map(row => row.join(','))
    .join('\n');

  return `Below is sales data for ${period}. Please analyze it for an executive report.

## Data Overview
- Rows: ${data.rowCount}
- Columns: ${data.headers.join(', ')}

## Basic Statistics
${JSON.stringify(stats, null, 2)}

## Data Sample (CSV)
${csvHeader}
${csvRows}

## Analysis Requirements
1. **Overall Trend**: Summarize the sales trajectory in 3 sentences or fewer
2. **Highlights**: Notable figures or patterns as bullet points (3-5 items)
3. **Concerns**: Areas requiring attention as bullet points (2-3 items)
4. **Recommended Actions**: Data-driven improvement suggestions (2-3 items)

Output in Markdown format. Use ## for headings.`;
}
```

**Key prompt design principles:**
- Send statistics and a data sample rather than the entire dataset to minimize token consumption
- Specify the output format explicitly to simplify downstream processing
- Define the expected granularity (number of bullet points, sentence counts) to get consistent results

## Building the Complete Report Pipeline

Now let's wire everything together into a production-ready pipeline.

```javascript
/**
 * Main: Monthly report generation pipeline
 * Reads spreadsheet data, analyzes with Gemini,
 * writes to a report sheet, and sends via email
 */
function generateMonthlyReport() {
  const startTime = new Date();
  const log = [];

  try {
    // ---- 1. Data Extraction ----
    log.push('Step 1: Extracting data...');
    const salesData = extractSheetData('Sales Data');
    const kpiData = extractSheetData('KPIs');

    const salesStats = calculateStats(salesData, 'Revenue');
    const period = Utilities.formatDate(new Date(), 'America/New_York', 'MMMM yyyy');

    // ---- 2. AI Analysis ----
    log.push('Step 2: Running AI analysis...');
    const client = new GeminiClient();

    const salesAnalysis = client.generate(
      buildAnalysisPrompt(salesData, salesStats, period),
      {
        systemInstruction:
          'You are a data analyst. Produce concise, accurate business reports. ' +
          'Always include units with numbers and show percentage changes where applicable.',
        temperature: 0.2
      }
    );

    // KPI summary generation
    const kpiSummary = client.generate(
      `Generate an executive dashboard summary from the following KPI data.\n\n` +
      `Data: ${JSON.stringify(kpiData.rows.slice(0, 20))}\n` +
      `Columns: ${kpiData.headers.join(', ')}\n\n` +
      `For each KPI, provide: current value, target, achievement rate, and trend (↑↓→) in one line.`,
      { temperature: 0.1, maxTokens: 2048 }
    );

    // ---- 3. Report Sheet Generation ----
    log.push('Step 3: Writing report sheet...');
    writeReportSheet(period, salesAnalysis, kpiSummary, salesStats);

    // ---- 4. Email Delivery ----
    log.push('Step 4: Sending email...');
    sendReportEmail(period, salesAnalysis, kpiSummary);

    // ---- 5. Execution Logging ----
    const elapsed = (new Date() - startTime) / 1000;
    log.push(`Completed in ${elapsed}s`);
    logExecution('SUCCESS', log.join('\n'));

  } catch (error) {
    log.push(`ERROR: ${error.message}`);
    logExecution('FAILED', log.join('\n'));

    // Send error notification
    MailApp.sendEmail({
      to: Session.getActiveUser().getEmail(),
      subject: `[ALERT] Monthly Report Generation Failed`,
      body: `Error: ${error.message}\n\nLog:\n${log.join('\n')}`
    });
  }
}

/**
 * Write the report to a new sheet
 */
function writeReportSheet(period, analysis, kpiSummary, stats) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheetName = `Report_${period.replace(/\s/g, '_')}`;

  // Delete existing sheet if present
  let sheet = ss.getSheetByName(sheetName);
  if (sheet) ss.deleteSheet(sheet);
  sheet = ss.insertSheet(sheetName);

  // Header
  sheet.getRange('A1').setValue(`📊 ${period} Monthly Report`)
    .setFontSize(16).setFontWeight('bold');
  sheet.getRange('A2').setValue(`Generated: ${new Date().toLocaleString('en-US')}`)
    .setFontColor('#666666');

  // Basic statistics
  sheet.getRange('A4').setValue('■ Key Metrics').setFontWeight('bold');
  const statsEntries = Object.entries(stats);
  statsEntries.forEach(([key, value], i) => {
    sheet.getRange(5 + i, 1).setValue(key);
    sheet.getRange(5 + i, 2).setValue(value)
      .setNumberFormat('#,##0');
  });

  // AI analysis
  const analysisRow = 5 + statsEntries.length + 2;
  sheet.getRange(analysisRow, 1).setValue('■ AI Analysis').setFontWeight('bold');
  sheet.getRange(analysisRow + 1, 1).setValue(analysis)
    .setWrapStrategy(SpreadsheetApp.WrapStrategy.WRAP);
  sheet.setColumnWidth(1, 600);

  // KPI dashboard
  const kpiRow = analysisRow + 3;
  sheet.getRange(kpiRow, 1).setValue('■ KPI Dashboard').setFontWeight('bold');
  sheet.getRange(kpiRow + 1, 1).setValue(kpiSummary)
    .setWrapStrategy(SpreadsheetApp.WrapStrategy.WRAP);
}
```

## Scheduling and Trigger Management

### Setting Up Time-Based Triggers

Use Apps Script triggers to automate report generation on a schedule.

```javascript
/**
 * Set up a monthly trigger (1st of each month at 9:00 AM)
 */
function setupMonthlyTrigger() {
  // Remove existing triggers for this function
  ScriptApp.getProjectTriggers()
    .filter(t => t.getHandlerFunction() === 'generateMonthlyReport')
    .forEach(t => ScriptApp.deleteTrigger(t));

  // Create the new trigger
  ScriptApp.newTrigger('generateMonthlyReport')
    .timeBased()
    .onMonthDay(1)
    .atHour(9)
    .inTimezone('Asia/Tokyo')
    .create();

  console.log('Monthly trigger set: 1st of each month at 9:00 JST');
}

/**
 * Weekly summary trigger (every Monday at 8:00 AM)
 */
function setupWeeklyTrigger() {
  ScriptApp.getProjectTriggers()
    .filter(t => t.getHandlerFunction() === 'generateWeeklySummary')
    .forEach(t => ScriptApp.deleteTrigger(t));

  ScriptApp.newTrigger('generateWeeklySummary')
    .timeBased()
    .everyWeeks(1)
    .onWeekDay(ScriptApp.WeekDay.MONDAY)
    .atHour(8)
    .inTimezone('Asia/Tokyo')
    .create();
}
```

### Execution Logging

Track your automated runs with a dedicated logging system.

```javascript
/**
 * Record execution details to a log sheet
 */
function logExecution(status, details) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  let logSheet = ss.getSheetByName('_ExecutionLog');

  if (!logSheet) {
    logSheet = ss.insertSheet('_ExecutionLog');
    logSheet.getRange('A1:D1').setValues([
      ['Timestamp', 'Status', 'Duration', 'Details']
    ]).setFontWeight('bold');
  }

  logSheet.appendRow([
    new Date(),
    status,
    details.match(/Completed in (.+)/)?.[1] || 'N/A',
    details
  ]);

  // Auto-cleanup logs older than 90 days
  const rows = logSheet.getDataRange().getValues();
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - 90);

  for (let i = rows.length - 1; i >= 1; i--) {
    if (rows[i][0] < cutoff) {
      logSheet.deleteRow(i + 1);
    }
  }
}
```

## Email Delivery with HTML Formatting

### Building Professional HTML Reports

Deliver reports as well-formatted HTML emails that stakeholders actually want to read.

```javascript
/**
 * Send the report via HTML email
 */
function sendReportEmail(period, analysis, kpiSummary) {
  // Get recipient list from a dedicated sheet
  const recipientSheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('Recipients');

  if (!recipientSheet) {
    console.log('No recipient sheet found, skipping email');
    return;
  }

  const recipients = recipientSheet.getDataRange().getValues()
    .slice(1) // Skip header
    .filter(row => row[0] && row[1] === true) // Has email and is active
    .map(row => row[0]);

  if (recipients.length === 0) return;

  // Simple Markdown → HTML conversion
  const analysisHtml = markdownToHtml(analysis);
  const kpiHtml = markdownToHtml(kpiSummary);

  const html = `
    <div style="font-family: 'Helvetica Neue', Arial, sans-serif; max-width: 700px; margin: 0 auto; color: #333;">
      <div style="background: linear-gradient(135deg, #4285f4, #34a853); padding: 24px; border-radius: 8px 8px 0 0;">
        <h1 style="color: white; margin: 0; font-size: 22px;">📊 ${period} Monthly Report</h1>
        <p style="color: rgba(255,255,255,0.85); margin: 8px 0 0; font-size: 13px;">
          Generated by Gemini AI — ${new Date().toLocaleString('en-US')}
        </p>
      </div>

      <div style="padding: 24px; background: #fff; border: 1px solid #e0e0e0;">
        <h2 style="color: #4285f4; border-bottom: 2px solid #4285f4; padding-bottom: 8px;">Sales Analysis</h2>
        ${analysisHtml}

        <h2 style="color: #34a853; border-bottom: 2px solid #34a853; padding-bottom: 8px; margin-top: 32px;">KPI Dashboard</h2>
        ${kpiHtml}
      </div>

      <div style="padding: 16px; background: #f5f5f5; border-radius: 0 0 8px 8px; text-align: center; font-size: 12px; color: #999;">
        This report was automatically generated using the Gemini API.
        <a href="${SpreadsheetApp.getActiveSpreadsheet().getUrl()}">Open spreadsheet</a>
      </div>
    </div>
  `;

  MailApp.sendEmail({
    to: recipients.join(','),
    subject: `[Monthly Report] ${period} — AI-Generated Analysis Summary`,
    htmlBody: html
  });
}

/**
 * Simple Markdown → HTML converter
 */
function markdownToHtml(md) {
  return md
    .replace(/^## (.+)$/gm, '<h3 style="color: #333; margin-top: 20px;">$1</h3>')
    .replace(/^### (.+)$/gm, '<h4 style="color: #555;">$1</h4>')
    .replace(/\*\*(.+?)\*\*/g, '<strong>$1</strong>')
    .replace(/^- (.+)$/gm, '<li style="margin: 4px 0;">$1</li>')
    .replace(/(<li.*<\/li>\n?)+/g, '<ul style="padding-left: 20px;">$&</ul>')
    .replace(/\n\n/g, '<br><br>')
    .replace(/\n/g, '<br>');
}
```

## Cost Management and Rate Limiting

In production, keeping API costs under control is essential.

```javascript
/**
 * API usage tracker
 * Accumulates token usage in Script Properties with monthly budget limits
 */
class UsageTracker {
  constructor(monthlyBudgetUsd = 10) {
    this.props = PropertiesService.getScriptProperties();
    this.monthlyBudget = monthlyBudgetUsd;
    // Approximate Gemini 2.5 Flash pricing (input: $0.15/1M, output: $0.60/1M)
    this.inputCostPer1k = 0.00015;
    this.outputCostPer1k = 0.0006;
  }

  /**
   * Record usage for a request
   */
  record(inputTokens, outputTokens) {
    const key = `usage_${this._currentMonth()}`;
    const current = JSON.parse(this.props.getProperty(key) || '{"input":0,"output":0,"calls":0}');

    current.input += inputTokens;
    current.output += outputTokens;
    current.calls += 1;

    this.props.setProperty(key, JSON.stringify(current));
    return current;
  }

  /**
   * Get estimated cost for the current month
   */
  estimatedCost() {
    const key = `usage_${this._currentMonth()}`;
    const usage = JSON.parse(this.props.getProperty(key) || '{"input":0,"output":0}');

    return (usage.input / 1000 * this.inputCostPer1k) +
           (usage.output / 1000 * this.outputCostPer1k);
  }

  /**
   * Check budget (throws if exceeded)
   */
  checkBudget() {
    const cost = this.estimatedCost();
    if (cost >= this.monthlyBudget) {
      throw new Error(
        `Monthly API budget exceeded: $${cost.toFixed(2)} / $${this.monthlyBudget}. ` +
        `Resets on the 1st of next month.`
      );
    }
    if (cost >= this.monthlyBudget * 0.8) {
      console.warn(`API budget warning: $${cost.toFixed(2)} / $${this.monthlyBudget} (80%+)`);
    }
  }

  _currentMonth() {
    return Utilities.formatDate(new Date(), 'America/New_York', 'yyyy-MM');
  }
}
```

By integrating this `UsageTracker` into the `GeminiClient`, report generation automatically pauses when the budget is exceeded. The budget resets at the start of each month, and you can always override it manually if needed.

## Error Handling and Fallback Strategies

A production pipeline needs to handle every possible failure mode gracefully.

```javascript
/**
 * Report generation with fallback
 * Falls back to stats-based report if Gemini API is unavailable
 */
function generateReportWithFallback() {
  const salesData = extractSheetData('Sales Data');
  const stats = calculateStats(salesData, 'Revenue');
  const period = Utilities.formatDate(new Date(), 'America/New_York', 'MMMM yyyy');

  let analysis;
  let source;

  try {
    const tracker = new UsageTracker(10);
    tracker.checkBudget();

    const client = new GeminiClient();
    analysis = client.generate(
      buildAnalysisPrompt(salesData, stats, period),
      { systemInstruction: 'Produce concise, data-driven business reports.', temperature: 0.2 }
    );
    source = 'Gemini AI';

  } catch (apiError) {
    console.warn(`Gemini API unavailable: ${apiError.message}`);
    console.log('Falling back to stats-based report...');

    // Fallback: template-based report using statistics only
    analysis = generateStatsFallbackReport(stats, period);
    source = 'Auto-generated (API unavailable)';
  }

  writeReportSheet(period, analysis, `Source: ${source}`, stats);
  sendReportEmail(period, analysis, `Source: ${source}`);
}

/**
 * Statistics-based fallback report
 */
function generateStatsFallbackReport(stats, period) {
  return `## ${period} Sales Summary

**Total Revenue**: $${stats.sum.toLocaleString()}
**Average**: $${stats.average.toLocaleString()} (Median: $${stats.median.toLocaleString()})
**Highest**: $${stats.max.toLocaleString()} / **Lowest**: $${stats.min.toLocaleString()}
**Data Points**: ${stats.count}

> ⚠️ This is a statistics-only report generated without AI analysis. Full AI-powered analysis will be available in the next run.`;
}
```

## Frequently Asked Questions

### Q1: What if my script hits the Apps Script execution time limit (6 minutes)?

Apps Script has a maximum execution time of 6 minutes for consumer accounts (30 minutes for Google Workspace). For large datasets, you'll need to split processing across multiple executions. Use `CacheService` or `PropertiesService` to store intermediate results, then pick up where you left off in the next run. This "chained execution" pattern effectively removes the time limit.

### Q2: Are there alternatives to Script Properties for managing API keys?

You can integrate with Google Cloud Secret Manager for enterprise-grade key management. However, for personal use or small teams, Script Properties is perfectly adequate. If you prefer to eliminate API keys entirely, use OAuth 2.0 with a service account and the Vertex AI endpoint instead.

### Q3: How can I improve the accuracy of generated reports?

Three approaches work well together. First, include examples of previous reports as few-shot examples in your prompt. Second, lower the `temperature` to 0.1–0.2 for more consistent output. Third, programmatically verify numerical accuracy after generation and re-generate if errors are found. For production environments, we recommend combining all three.

### Q4: Does this work with personal Gmail accounts (not Workspace)?

Yes, it works. However, personal accounts have stricter limits: 6-minute execution time (vs. 30 minutes for Workspace), a maximum of 20 triggers (vs. 40), and lower daily email quotas. For a single monthly report, these limits won't be an issue.
