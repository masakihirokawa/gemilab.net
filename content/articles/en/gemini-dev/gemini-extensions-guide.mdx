## What Are Gemini Extensions?

Gemini Extensions enable Gemini to access external services and data, providing more accurate and up-to-date information. In addition to built-in extensions, developers can build custom ones.

## Available Standard Extensions

### Google Search

Searches for the latest information and reflects it in responses. Automatically activated for questions requiring news, current events, or real-time data.

**Examples:**
- "What's the weather in Tokyo today?"
- "Summarize recent AI news"
- "What are the latest earnings for company X?"

### Google Maps

Searches for places, provides directions, and retrieves restaurant and hotel information.

**Examples:**
- "How to get from Shibuya Station to Tokyo Tower"
- "Best Italian restaurants nearby"
- "Show hours and review ratings"

### YouTube

Searches YouTube videos and suggests relevant content.

**Examples:**
- "Find Python machine learning tutorial videos"
- "Find the music video for this song"

### Google Flights & Hotels

Search and compare flights and hotel accommodations.

**Examples:**
- "Cheapest flights from Tokyo to Paris next month"
- "Compare 4-star hotels in Barcelona"

## Enabling Extensions

1. Open Gemini settings
2. Navigate to the "Extensions" section
3. Toggle on the extensions you want to use

Each extension can be enabled or disabled individually.

## Using Tools via API

Developers can use the `tools` parameter in the Gemini API to access similar functionality programmatically.

```python
import google.genai as genai

client = genai.Client(api_key="YOUR_API_KEY")

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="What's the current weather in Tokyo?",
    config={
        "tools": [{"google_search": {}}]
    }
)

# Check grounding metadata
for candidate in response.candidates:
    if candidate.grounding_metadata:
        for chunk in candidate.grounding_metadata.grounding_chunks:
            print(f"Source: {chunk.web.uri}")
```

## Custom Function Calling

Beyond standard extensions, you can define custom functions for Gemini to invoke.

```python
# Define a function
get_weather = {
    "name": "get_weather",
    "description": "Get current weather for a specified city",
    "parameters": {
        "type": "object",
        "properties": {
            "city": {"type": "string", "description": "City name"},
            "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
        },
        "required": ["city"]
    }
}

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="What's the weather in Tokyo?",
    config={
        "tools": [{"function_declarations": [get_weather]}]
    }
)
```

## Best Practices

- **Enable only needed extensions**: Unnecessary extensions may trigger unintended external calls
- **Use grounding information**: Verify source URLs from search results to validate reliability
- **Consider latency**: Extension calls involve network access, increasing response time
- **Manage costs**: API grounding searches may incur additional charges

## Summary

Gemini Extensions dramatically expand what Gemini can do. Start with the built-in Google extensions, and as you get comfortable, build custom Function Calling integrations for your own tools and services.
