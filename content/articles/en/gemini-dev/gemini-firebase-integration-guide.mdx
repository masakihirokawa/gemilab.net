## Why Combine Gemini with Firebase?

While the Gemini API alone can generate text and analyze images, real-world applications need user management, data persistence, and server-side processing. Firebase provides all of these as fully managed services, and pairing it with Gemini lets you build production-grade AI apps quickly.

This article walks through building a backend that safely calls the Gemini API using Firebase's Firestore (database) and Cloud Functions (serverless functions).

## Calling the Gemini API from Cloud Functions

### Secure API Key Management

Never hardcode your Gemini API key in source code. Use Firebase's secrets management to store it securely.

```bash
# Store the API key as a secret
firebase functions:secrets:set GEMINI_API_KEY
# Enter your key when prompted
```

### Basic Cloud Function Implementation

```typescript
// functions/src/index.ts
// Basic implementation calling Gemini API from Cloud Functions

import { onCall, HttpsError } from "firebase-functions/v2/https";
import { defineSecret } from "firebase-functions/params";
import { GoogleGenerativeAI } from "@google/generative-ai";

// Define API key as a secret
const geminiApiKey = defineSecret("GEMINI_API_KEY");

export const generateText = onCall(
  { secrets: [geminiApiKey] },
  async (request) => {
    // Authentication check
    if (!request.auth) {
      throw new HttpsError(
        "unauthenticated",
        "You must be signed in to use this feature"
      );
    }

    const { prompt } = request.data;

    if (!prompt || typeof prompt !== "string") {
      throw new HttpsError(
        "invalid-argument",
        "A prompt is required"
      );
    }

    // Input length limit (safety measure)
    if (prompt.length > 2000) {
      throw new HttpsError(
        "invalid-argument",
        "Prompt must be under 2000 characters"
      );
    }

    try {
      const genAI = new GoogleGenerativeAI(geminiApiKey.value());
      const model = genAI.getGenerativeModel({
        model: "gemini-2.0-flash",
      });

      const result = await model.generateContent(prompt);
      const text = result.response.text();

      return { text, tokenCount: text.length };
    } catch (error) {
      console.error("Gemini API error:", error);
      throw new HttpsError(
        "internal",
        "Failed to generate text"
      );
    }
  }
);
```

## Storing Conversation History in Firestore

For AI chat applications, preserving conversation history is essential for maintaining context across interactions. Firestore enables real-time data synchronization out of the box.

```typescript
// functions/src/chat.ts
// Chat with Gemini while persisting history to Firestore

import { onCall, HttpsError } from "firebase-functions/v2/https";
import { defineSecret } from "firebase-functions/params";
import { getFirestore, FieldValue } from "firebase-admin/firestore";
import { GoogleGenerativeAI, Content } from "@google/generative-ai";

const geminiApiKey = defineSecret("GEMINI_API_KEY");
const db = getFirestore();

interface ChatMessage {
  role: "user" | "model";
  text: string;
  timestamp: FirebaseFirestore.FieldValue;
}

export const chat = onCall(
  { secrets: [geminiApiKey] },
  async (request) => {
    if (!request.auth) {
      throw new HttpsError("unauthenticated", "Sign-in required");
    }

    const { message, conversationId } = request.data;
    const userId = request.auth.uid;

    // Reference to conversation document
    const convRef = db
      .collection("users")
      .doc(userId)
      .collection("conversations")
      .doc(conversationId || db.collection("_").doc().id);

    // Fetch existing conversation history
    const messagesSnap = await convRef
      .collection("messages")
      .orderBy("timestamp", "asc")
      .limit(50) // Limit to last 50 messages
      .get();

    // Convert to Gemini API history format
    const history: Content[] = messagesSnap.docs.map((doc) => {
      const data = doc.data() as ChatMessage;
      return {
        role: data.role,
        parts: [{ text: data.text }],
      };
    });

    // Generate response with Gemini
    const genAI = new GoogleGenerativeAI(geminiApiKey.value());
    const model = genAI.getGenerativeModel({
      model: "gemini-2.0-flash",
    });

    const chatSession = model.startChat({ history });
    const result = await chatSession.sendMessage(message);
    const responseText = result.response.text();

    // Save both user message and AI response to Firestore
    const batch = db.batch();

    batch.set(convRef.collection("messages").doc(), {
      role: "user",
      text: message,
      timestamp: FieldValue.serverTimestamp(),
    } as ChatMessage);

    batch.set(convRef.collection("messages").doc(), {
      role: "model",
      text: responseText,
      timestamp: FieldValue.serverTimestamp(),
    } as ChatMessage);

    // Update conversation metadata
    batch.set(
      convRef,
      {
        updatedAt: FieldValue.serverTimestamp(),
        messageCount: FieldValue.increment(2),
      },
      { merge: true }
    );

    await batch.commit();

    return {
      response: responseText,
      conversationId: convRef.id,
    };
  }
);
```

## Firestore Security Rules

Use Firestore security rules to ensure users can only access their own data.

```
// firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // User conversation data
    match /users/{userId}/conversations/{convId} {
      // Only the owner can read and write
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;

      // Messages subcollection
      match /messages/{messageId} {
        allow read: if request.auth != null
                    && request.auth.uid == userId;
        // Write access restricted to Cloud Functions only
        allow write: if false;
      }
    }
  }
}
```

The key design decision here is blocking client-side writes to the messages subcollection and routing all writes through Cloud Functions. This prevents users from injecting arbitrary data directly into their conversation history.

## Calling from the Frontend (Web)

```typescript
// src/app.ts
// Calling Cloud Functions from the Firebase client SDK

import { initializeApp } from "firebase/app";
import { getFunctions, httpsCallable } from "firebase/functions";
import { getAuth, signInWithPopup, GoogleAuthProvider } from "firebase/auth";

const app = initializeApp({
  // Copy your config from Firebase Console
  apiKey: "YOUR_FIREBASE_API_KEY",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
});

const functions = getFunctions(app, "asia-northeast1");
const auth = getAuth(app);

// Google sign-in
async function signIn(): Promise<void> {
  const provider = new GoogleAuthProvider();
  await signInWithPopup(auth, provider);
}

// Chat with Gemini
async function sendMessage(
  message: string,
  conversationId?: string
): Promise<{ response: string; conversationId: string }> {
  const chatFn = httpsCallable(functions, "chat");
  const result = await chatFn({ message, conversationId });
  return result.data as { response: string; conversationId: string };
}

// Usage
async function main(): Promise<void> {
  await signIn();

  const result = await sendMessage("Explain Firebase pricing");
  console.log(result.response);

  // Continue the same conversation
  const followUp = await sendMessage(
    "What are the free tier limits?",
    result.conversationId
  );
  console.log(followUp.response);
}
```

## Production Best Practices

Here are key considerations when running Gemini API calls in Cloud Functions.

**Rate limiting**: Set instance caps on your Cloud Functions to prevent excessive API calls. Use the `maxInstances: 10` option to control concurrency.

**Cost management**: Gemini API charges accrue with each request. Track per-user daily request counts in Firestore and reject requests once the limit is reached.

**Error handling**: Implement retry logic for API timeouts and transient errors. Don't forget to set `timeoutSeconds: 60` for functions that call external APIs.

**Caching**: Cache Gemini responses in Firestore for identical prompts to reduce API call volume and latency.

## FAQ

### Q1. Can I run a Gemini API app on Firebase's free tier?

A1. Firebase's Spark plan (free tier) doesn't allow outbound network calls from Cloud Functions. You need the Blaze plan (pay-as-you-go) to call external APIs like Gemini. However, the Blaze plan still includes free-tier allowances, so small-scale development and testing costs very little.

### Q2. Should I use Vertex AI or Google AI Studio API keys?

A2. For prototypes and personal projects, Google AI Studio API keys are the quickest path. For production environments or enterprise use, Vertex AI offers IAM-based access control and region selection. See [Vertex AI Quickstart](/en/articles/gemini-dev/vertex-ai-quickstart) for details.

### Q3. Won't conversation history hit the token limit as it grows?

A3. Gemini 2.0 Flash supports a 1 million token context window, but longer histories increase costs. Use `limit(50)` as shown in the example to cap recent messages, or generate summaries to compress older conversation history.

### Q4. How does this differ from the Firebase Extensions "Gemini API" extension?

A4. Firebase Extensions offer a no-code setup, but with limited customization. The Cloud Functions approach in this article gives you full control over prompt engineering, history management, rate limiting, and error handling — making it better suited for production applications.
