## Introduction

CI/CD (Continuous Integration / Continuous Delivery) pipelines are the backbone of modern software development. GitHub Actions is one of the most widely adopted CI/CD platforms, and by integrating the Gemini API into your workflows, you can automate code reviews, generate PR summaries, suggest test cases, and supercharge your entire development process.

In this guide, we'll walk through practical examples of connecting Gemini API with GitHub Actions, complete with working workflow definitions and Python scripts you can start using right away. This article is aimed at developers who are familiar with GitHub Actions basics and want to bring AI into their development workflow.

## Setting Up Gemini API in GitHub Actions

### Prerequisites

Before getting started, you'll need:

1. A **Google AI Studio account** and **API key** (get one at [Google AI Studio](https://aistudio.google.com/))
2. **Admin access** to your GitHub repository (to configure Secrets)
3. Basic knowledge of **Python 3.11+**

### Storing Your API Key in GitHub Secrets

To securely use your Gemini API key in workflows, store it as a repository secret:

1. Navigate to your repository's **Settings → Secrets and variables → Actions**
2. Click **New repository secret**
3. Set Name to `GEMINI_API_KEY` and paste your API key as the Value

This lets you reference the key as `${{ secrets.GEMINI_API_KEY }}` in your workflows. Never hardcode API keys directly in your source code.

## Automated Pull Request Reviews

One of the most impactful use cases is sending PR diffs to Gemini API for automated review comments.

### Workflow Definition

```yaml
# .github/workflows/gemini-review.yml
name: Gemini Code Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai requests

      - name: Get PR diff
        id: diff
        run: |
          git diff origin/${{ github.base_ref }}...HEAD > pr_diff.txt
          echo "diff_size=$(wc -c < pr_diff.txt)" >> $GITHUB_OUTPUT

      - name: Run Gemini Review
        if: steps.diff.outputs.diff_size > 0
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
        run: python scripts/gemini_review.py
```

### Review Script

```python
# scripts/gemini_review.py
import os
from google import genai

def main():
    # Initialize the Gemini API client
    client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])

    # Read the PR diff
    with open("pr_diff.txt", "r") as f:
        diff = f.read()

    # Trim if the diff is too large
    max_chars = 30000
    if len(diff) > max_chars:
        diff = diff[:max_chars] + "\n... (diff truncated for length)"

    # Request a review from Gemini API
    prompt = f"""You are an experienced software engineer.
Review the following code diff and identify areas for improvement.

## Review Criteria
- Bugs or logic issues
- Security concerns
- Performance improvements
- Code readability

## Output Format
For each issue, use this format:
- **[Severity: High/Medium/Low]** filename:~line - description

---
{diff}
"""

    response = client.models.generate_content(
        model="gemini-3-flash",
        contents=prompt,
    )

    review_comment = response.text
    print("Review generated successfully")
    print(review_comment)

    # Post the review as a PR comment via the GitHub API
    import requests
    repo = os.environ.get("GITHUB_REPOSITORY", "")
    pr_number = os.environ.get("PR_NUMBER", "")
    token = os.environ.get("GITHUB_TOKEN", "")

    if repo and pr_number and token:
        url = f"https://api.github.com/repos/{repo}/issues/{pr_number}/comments"
        headers = {
            "Authorization": f"token {token}",
            "Accept": "application/vnd.github.v3+json",
        }
        body = f"## Gemini AI Code Review\n\n{review_comment}"
        requests.post(url, json={"body": body}, headers=headers)
        # Expected output: An AI review comment is posted to the PR

if __name__ == "__main__":
    main()
```

## Commit Message Quality Checks

Maintaining consistent commit message quality across a team is easier with an AI-powered linter built into your CI pipeline.

```yaml
# .github/workflows/gemini-commit-check.yml
name: Commit Message Quality Check

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  check-commits:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai

      - name: Check commit messages
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
        run: |
          # Extract commit messages
          git log origin/${{ github.base_ref }}..HEAD \
            --pretty=format:"%s" > commits.txt

          python -c "
          import os
          from google import genai

          client = genai.Client(api_key=os.environ['GEMINI_API_KEY'])

          with open('commits.txt') as f:
              commits = f.read()

          prompt = f'''Evaluate the following commit messages.
          Check whether they follow Conventional Commits format
          (feat:, fix:, docs:, etc.) and whether the content is clear.

          Commit messages:
          {commits}

          For each message, rate it as OK or Needs Improvement.
          If improvement is needed, suggest a revised message.'''

          response = client.models.generate_content(
              model='gemini-3-flash',
              contents=prompt,
          )
          print(response.text)
          # Expected output:
          # feat: add user auth → OK
          # fix stuff → Needs Improvement: 'fix: resolve login validation error'
          "
```

## Practical Tips: Token Limits and Cost Management

When using the Gemini API in CI/CD pipelines, it's important to stay mindful of costs and rate limits.

### Controlling Diff Size

```python
def trim_diff(diff: str, max_tokens: int = 8000) -> str:
    """Limit the diff by estimated token count."""
    if len(diff) > max_tokens * 4:  # Rough estimate: 1 token ≈ 4 chars
        # Split by file and prioritize source code
        files = diff.split("diff --git")
        priority_files = []
        other_files = []

        for f in files:
            if any(ext in f for ext in [".py", ".ts", ".js", ".go"]):
                priority_files.append(f)
            else:
                other_files.append(f)

        # Include source code files first
        result = "diff --git".join(priority_files)
        if len(result) < max_tokens * 4:
            remaining = max_tokens * 4 - len(result)
            result += "diff --git".join(other_files)[:remaining]

        return result[:max_tokens * 4]
    return diff
```

### Optimizing API Calls

Here are some strategies to minimize unnecessary workflow runs:

- Use **path filters** to skip reviews for documentation-only changes
- **Skip small diffs** (under 10 lines) where AI review adds little value
- Use **caching** to prevent re-running on the same diff

```yaml
on:
  pull_request:
    paths:
      - "src/**"
      - "lib/**"
      - "!**/*.md"
      - "!**/*.txt"
```

## Security Considerations

Integrating AI into your CI/CD pipeline requires careful attention to security.

1. **API key management**: Always use GitHub Secrets — never hardcode keys in your source
2. **Diff content**: Ensure that secrets and personal data are excluded from diffs with a properly configured `.gitignore`
3. **External PRs**: Secrets aren't available for PRs from forks — be careful with `pull_request_target`
4. **Rate limiting**: Use the `concurrency` setting to prevent overwhelming the API with parallel runs

```yaml
concurrency:
  group: gemini-review-${{ github.event.pull_request.number }}
  cancel-in-progress: true
```

## Frequently Asked Questions (FAQ)

### Q: Can I use Gemini API's free tier for CI/CD?

A: Absolutely. Gemini 3 Flash includes a generous free tier that works well for personal projects and small teams. However, there are rate limits (requests per minute), so if your repository generates frequent PRs, you may want to consider a paid plan for higher throughput.

### Q: Does this work with other CI/CD tools like GitLab CI or CircleCI?

A: Yes. The Gemini API is a standard HTTP-based REST API, so it works with any CI/CD platform. The Python scripts from this guide can be reused as-is — you only need to adapt the workflow definition syntax for your specific platform.

### Q: How should I handle very large diffs (thousands of lines)?

A: For large diffs, splitting the content by file and sending separate API requests is effective. You can also leverage Gemini 3 Pro's 1-million-token context window to process large diffs in a single call. To manage costs, prioritize source code files and exclude configuration files, lock files, and auto-generated content from the diff sent to the API.
