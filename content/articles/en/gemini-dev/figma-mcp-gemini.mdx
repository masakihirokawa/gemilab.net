## Connecting Figma and Gemini API Seamlessly

Figma is the central hub where designers and engineers collaborate. But "converting Figma designs into code" still requires a significant amount of manual work.

By connecting the **Figma MCP Server** to the **Gemini API**, AI can automate this conversion process. Gemini accesses Figma design data in real time, reads component specifications, and generates implementation code.

---

## What You Need

- Figma account (Personal Access Token)
- Google AI Studio API key (Gemini API)
- Node.js 18 or higher
- Python 3.10 or higher (optional)

---

## Setting Up the Figma MCP Server

### Step 1: Get Your Figma Personal Access Token

1. Log in to Figma
2. Click your profile (top right) → **Settings** → **Security**
3. Under **Personal access tokens**, click **Generate new token**
4. Copy the token (it's only shown once)

### Step 2: Install the Figma MCP Server

```bash
npm install -g @figma/mcp-server
```

Or run directly with `npx`:

```bash
npx @figma/mcp-server
```

### Step 3: Set Environment Variables

```bash
export FIGMA_ACCESS_TOKEN="your-figma-personal-access-token"
export GOOGLE_API_KEY="your-gemini-api-key"
```

## Gemini API Integration Code

### Basic Connection and Fetching Figma Data

```python
import google.generativeai as genai
import requests
import json

# Configure Gemini API
genai.configure(api_key="YOUR_GEMINI_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")

# Fetch design data from Figma API
FIGMA_TOKEN = "YOUR_FIGMA_TOKEN"
FILE_ID = "your-figma-file-id"  # Extract from the Figma URL

def get_figma_components(file_id: str, node_id: str | None = None) -> dict:
    """Retrieve component information from a Figma file"""
    url = f"https://api.figma.com/v1/files/{file_id}"
    if node_id:
        url += f"/nodes?ids={node_id}"

    headers = {"X-Figma-Token": FIGMA_TOKEN}
    response = requests.get(url, headers=headers)
    return response.json()

def generate_code_from_figma(figma_data: dict, framework: str = "React") -> str:
    """Generate code from Figma data using the Gemini API"""
    prompt = f"""
Analyze the following Figma component specification and
implement it as a {framework} component.

Figma Data:
{json.dumps(figma_data, indent=2)}

Requirements:
- Implement in TypeScript
- Style with Tailwind CSS
- Accessibility compliant (aria-label etc.)
- Responsive design
- Define a Props interface
"""
    response = model.generate_content(prompt)
    return response.text

# Usage
figma_data = get_figma_components(FILE_ID)
react_code = generate_code_from_figma(figma_data, framework="React")
print(react_code)
```

## Practical Example: Auto-Generating a Design System

### Scenario: Automatically Generating Button Components

```python
def sync_design_system(figma_file_id: str, components: list[str]) -> dict:
    """
    Batch-generate components from a Figma design system
    """
    results = {}

    for component_name in components:
        figma_data = get_figma_components(figma_file_id)

        prompt = f"""
Find the "{component_name}" component in this Figma design system data and extract:

1. Size variants (sm, md, lg)
2. States (default, hover, disabled, loading)
3. Color schemes (primary, secondary, destructive)
4. Typography specs

Data: {json.dumps(figma_data, indent=2)[:3000]}
"""

        spec_response = model.generate_content(prompt)

        code_prompt = f"""
Generate a production-grade React component based on the following spec:

{spec_response.text}

- TypeScript + Tailwind CSS + shadcn/ui compatible
- Define variants with cva (class-variance-authority)
- Include Storybook documentation
"""
        code_response = model.generate_content(code_prompt)
        results[component_name] = code_response.text

    return results

# Batch-generate key design system components
components = ["Button", "Input", "Card", "Modal", "Badge"]
generated = sync_design_system(FILE_ID, components)
```

## Multimodal Use of Figma × Gemini

Gemini excels at multimodal understanding (images), so you can feed Figma screenshots directly as input.

```python
import PIL.Image
import requests
from io import BytesIO

def analyze_figma_screenshot(image_url: str) -> str:
    """
    Analyze a Figma screenshot with Gemini to extract
    implementation suggestions and specs
    """
    response = requests.get(image_url)
    image = PIL.Image.open(BytesIO(response.content))

    model_vision = genai.GenerativeModel("gemini-2.0-flash")

    prompt = """
Analyze this design screenshot and provide:

1. Component structure suggestions (how to break it into components)
2. Accessibility concerns (contrast ratio, touch targets, etc.)
3. Notes for responsive design
4. Implementation code in React + Tailwind CSS

Please be thorough.
"""

    response = model_vision.generate_content([prompt, image])
    return response.text
```

### 2. Design Review & QA

```python
def design_qa_review(figma_node_id: str) -> dict:
    """Automate design quality review"""
    figma_data = get_figma_components(FILE_ID, figma_node_id)

    qa_prompt = f"""
Review the following design data and report issues:

{json.dumps(figma_data, indent=2)[:2000]}

Checklist:
- WCAG 2.1 AA compliance (contrast ratio 4.5:1 minimum)
- Touch target minimum size (44×44px)
- Font size (minimum 12px)
- Consistency (colors, spacing, typography)
- Responsive breakpoint alignment
"""
    response = model.generate_content(qa_prompt)
    return {"node_id": figma_node_id, "issues": response.text}
```

### 3. Auto-Generate Design Documentation

```python
def generate_design_docs(figma_file_id: str) -> str:

Include:
- Component list with descriptions
- Color palette (with HEX codes)
- Typography scale
- Spacing system
- Component Props interfaces
- Usage examples
"""
    response = model.generate_content(doc_prompt)
    return response.text
```
