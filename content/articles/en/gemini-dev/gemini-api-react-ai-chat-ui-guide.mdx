## Introduction — Why React × Gemini API?

React dominates the frontend ecosystem, and Google's Gemini API offers one of the most capable large language models available today. Combining the two lets you build polished AI chat applications in remarkably little time.

By choosing a lean Vite + React + Express.js stack instead of a full-stack framework like Next.js, you gain the flexibility to embed AI features into existing projects without restructuring your architecture. In this tutorial, we'll walk through every step — from project setup to streaming responses and multi-turn conversation management — to build a production-ready AI chat UI.

What you'll learn:

- Setting up a React + TypeScript project with Vite
- Building an Express.js proxy to keep your API key secure on the server
- Displaying Gemini API streaming responses in real time
- Managing multi-turn conversation state with custom hooks

## Prerequisites and Setup

### What You'll Need

- **Node.js 20 or later**
- **A Google AI Studio API key** (free at [Google AI Studio](https://aistudio.google.com/))
- **A code editor** (VS Code recommended)

Start by scaffolding a React + TypeScript project with Vite.

```bash

```

Your directory structure will look like this:

- `src/` — React frontend
- `server/` — Express.js backend (API proxy)
- `.env` — Environment variables (API key)

```bash

```

**Important**: Always add `.env` to your `.gitignore` to prevent accidentally committing your API key.

## Building the Backend API Proxy

Calling the Gemini API directly from the browser would expose your API key to anyone inspecting network traffic. An Express.js proxy server keeps the key safely on the server side.

```typescript
// server/index.ts
import express from "express";
import cors from "cors";
import { GoogleGenAI } from "@google/genai";

const app = express();
app.use(cors({ origin: "http://localhost:5173" }));
app.use(express.json());

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! });

// ── Standard response ──
app.post("/api/chat", async (req, res) => {
  try {
    const { messages } = req.body;

    // Convert to Gemini API contents format
    const contents = messages.map(
      (msg: { role: string; text: string }) => ({
        role: msg.role === "user" ? "user" : "model",
        parts: [{ text: msg.text }],
      })
    );

    const response = await ai.models.generateContent({
      model: "gemini-2.5-flash",
      contents,
    });

    res.json({
      text: response.text ?? "",
    });
  } catch (error: any) {
    console.error("Gemini API error:", error.message);
    res.status(500).json({ error: error.message });
  }
});

// ── Streaming response ──
app.post("/api/chat/stream", async (req, res) => {
  try {
    const { messages } = req.body;

    const contents = messages.map(
      (msg: { role: string; text: string }) => ({
        role: msg.role === "user" ? "user" : "model",
        parts: [{ text: msg.text }],
      })
    );

    // Set SSE headers
    res.setHeader("Content-Type", "text/event-stream");
    res.setHeader("Cache-Control", "no-cache");
    res.setHeader("Connection", "keep-alive");

    const response = await ai.models.generateContentStream({
      model: "gemini-2.5-flash",
      contents,
    });

    for await (const chunk of response) {
      const text = chunk.text ?? "";
      if (text) {
        res.write(`data: ${JSON.stringify({ text })}\n\n`);
      }
    }

    res.write("data: [DONE]\n\n");
    res.end();
  } catch (error: any) {
    console.error("Stream error:", error.message);
    res.status(500).json({ error: error.message });
  }
});

const PORT = 3001;
app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
```

Add server scripts to your `package.json`:

```json
{
  "scripts": {
    "dev": "vite",
    "server": "tsx --env-file=.env server/index.ts"
  }
}
```

Run `npm run server` and `npm run dev` in separate terminals, and your frontend and backend will be connected.

### Custom Hook — useChat

Extracting chat logic into a custom hook keeps your UI components clean and testable.

```typescript
// src/hooks/useChat.ts
import { useState, useCallback, useRef } from "react";
import type { Message } from "../types";

const API_BASE = "http://localhost:3001";

export function useChat() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const abortRef = useRef<AbortController | null>(null);

  const sendMessage = useCallback(async (text: string) => {
    const userMessage: Message = {
      id: crypto.randomUUID(),
      role: "user",
      text,
      timestamp: new Date(),
    };

    const assistantMessage: Message = {
      id: crypto.randomUUID(),
      role: "model",
      text: "",
      timestamp: new Date(),
      isStreaming: true,
    };

    setMessages((prev) => [...prev, userMessage, assistantMessage]);
    setIsLoading(true);

    // Build conversation history for the API
    const history = [...messages, userMessage].map((m) => ({
      role: m.role,
      text: m.text,
    }));

    try {
      abortRef.current = new AbortController();

      const response = await fetch(`${API_BASE}/api/chat/stream`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ messages: history }),
        signal: abortRef.current.signal,
      });

      if (!response.ok) throw new Error("API request failed");

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();
      let accumulated = "";

      while (reader) {
        const { done, value } = await reader.read();
        if (done) break;

        const chunk = decoder.decode(value, { stream: true });
        const lines = chunk.split("\n");

        for (const line of lines) {
          if (line.startsWith("data: ")) {
            const data = line.slice(6);
            if (data === "[DONE]") break;

            try {
              const parsed = JSON.parse(data);
              accumulated += parsed.text;

              // Update the streaming message in real time
              setMessages((prev) =>
                prev.map((m) =>
                  m.id === assistantMessage.id
                    ? { ...m, text: accumulated }
                    : m
                )
              );
            } catch {
              // Skip lines that fail to parse
            }
          }
        }
      }

      // Mark streaming as complete
      setMessages((prev) =>
        prev.map((m) =>
          m.id === assistantMessage.id
            ? { ...m, isStreaming: false }
            : m
        )
      );
    } catch (error: any) {
      if (error.name !== "AbortError") {
        setMessages((prev) =>
          prev.map((m) =>
            m.id === assistantMessage.id
              ? {
                  ...m,
                  text: "An error occurred. Please try again.",
                  isStreaming: false,
                }
              : m
          )
        );
      }
    } finally {
      setIsLoading(false);
      abortRef.current = null;
    }
  }, [messages]);

  const stopGeneration = useCallback(() => {
    abortRef.current?.abort();
  }, []);

  const clearMessages = useCallback(() => {
    setMessages([]);
  }, []);

  return { messages, isLoading, sendMessage, stopGeneration, clearMessages };
}
```

### Chat Window Component

```tsx
// src/components/ChatWindow.tsx
import { useRef, useEffect } from "react";
import type { Message } from "../types";

interface Props {
  messages: Message[];
}

export function ChatWindow({ messages }: Props) {
  const bottomRef = useRef<HTMLDivElement>(null);

  // Auto-scroll when new messages arrive
  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [messages]);

  return (
    <div className="chat-window">
      {messages.length === 0 && (
        <div className="empty-state">
          <h2>Gemini AI Chat</h2>
          <p>Type a message to start the conversation</p>
        </div>
      )}

      {messages.map((msg) => (
        <div key={msg.id} className={`message ${msg.role}`}>
          <div className="message-avatar">
            {msg.role === "user" ? "👤" : "✨"}
          </div>
          <div className="message-content">
            <p>{msg.text}</p>
            {msg.isStreaming && <span className="cursor-blink" />}
          </div>
        </div>
      ))}

      <div ref={bottomRef} />
    </div>
  );
}
```

### Input Form

```tsx
// src/components/ChatInput.tsx
import { useState, type FormEvent } from "react";

interface Props {
  onSend: (text: string) => void;
  isLoading: boolean;
  onStop: () => void;
}

export function ChatInput({ onSend, isLoading, onStop }: Props) {
  const [input, setInput] = useState("");

  const handleSubmit = (e: FormEvent) => {
    e.preventDefault();
    if (!input.trim() || isLoading) return;
    onSend(input.trim());
    setInput("");
  };

  return (
    <form className="chat-input" onSubmit={handleSubmit}>
      <textarea
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Type your message..."
        rows={1}
        onKeyDown={(e) => {
          if (e.key === "Enter" && !e.shiftKey) {
            e.preventDefault();
            handleSubmit(e);
          }
        }}
      />
      {isLoading ? (
        <button type="button" onClick={onStop} className="stop-btn">
          Stop
        </button>
      ) : (
        <button type="submit" disabled={!input.trim()}>
          Send
        </button>
      )}
    </form>
  );
}
```

### Bringing It Together in App

```tsx
// src/App.tsx
import { useChat } from "./hooks/useChat";
import { ChatWindow } from "./components/ChatWindow";
import { ChatInput } from "./components/ChatInput";
import "./App.css";

export default function App() {
  const { messages, isLoading, sendMessage, stopGeneration, clearMessages } =
    useChat();

  return (
    <div className="app">
      <header>
        <h1>Gemini Chat</h1>
        <button onClick={clearMessages} className="clear-btn">
          New Chat
        </button>
      </header>
      <ChatWindow messages={messages} />
      <ChatInput
        onSend={sendMessage}
        isLoading={isLoading}
        onStop={stopGeneration}
      />
    </div>
  );
}
```

## Persisting Conversation History

If you want conversations to survive page reloads, `sessionStorage` works well for browser-session-scoped persistence.

```typescript
// Modify useChat.ts
import { useEffect } from "react";

// Change useState initializer
const [messages, setMessages] = useState<Message[]>(() => {
  try {
    const saved = sessionStorage.getItem("chat-messages");
    return saved ? JSON.parse(saved) : [];
  } catch {
    return [];
  }
});

// Save whenever messages change
useEffect(() => {
  sessionStorage.setItem("chat-messages", JSON.stringify(messages));
}, [messages]);
```

Keep in mind that `sessionStorage` has a typical limit of 5 MB. For production applications with long conversations, consider using IndexedDB or leveraging [Gemini API's context caching feature](/articles/gemini-api/gemini-api-context-caching-cost-optimization) to manage history efficiently on the server side.

## Error Handling Best Practices

Robust error handling is essential for production deployments. Here's how to handle common failure modes.

```typescript
// Add to server/index.ts
app.use(
  (
    err: Error,
    req: express.Request,
    res: express.Response,
    next: express.NextFunction
  ) => {
    // Rate limit (429) — suggest retry
    if (err.message.includes("429")) {
      return res.status(429).json({
        error: "Rate limit exceeded. Please wait a moment and try again.",
        retryAfter: 60,
      });
    }

    // Model overloaded (503)
    if (err.message.includes("503")) {
      return res.status(503).json({
        error: "The server is currently busy. Please try again later.",
      });
    }

    res.status(500).json({ error: "An unexpected error occurred." });
  }
);
```

On the frontend, distinguish between network errors and API errors to show appropriate messages. For a comprehensive treatment of security practices, see [Gemini API Production Security Guide](/articles/gemini-api/gemini-api-production-security-guide).

## Frequently Asked Questions

### Q1: How much can I use the Gemini API for free?

With an API key from Google AI Studio, Gemini 2.5 Flash allows up to 30 requests per minute and 1,500 requests per day at no cost (as of March 2026). That's plenty for personal projects and prototyping. Check [Google AI Studio](https://aistudio.google.com/) for the latest quotas.

### Q2: Can I use this same approach with Vue or Svelte instead of React?

Absolutely. The Express.js proxy backend is framework-agnostic. You'd only need to replace the React custom hook with the equivalent state management pattern in your framework of choice — Vue's Composition API, Svelte stores, and so on.

### Q3: How does this differ from using Next.js?

With Next.js, you can use API Routes or Server Actions to handle the backend within the same project, eliminating the need for a separate Express.js server. The pure React + Vite approach is better suited when you have an existing backend, or when you want to deploy frontend and backend independently. If you're interested in the Next.js approach, check out our [Gemini API × Next.js AI Chat App Guide](/articles/gemini-api/gemini-api-nextjs-ai-chat-app).

### Q4: Does streaming make a noticeable performance difference?

The difference in perceived speed is significant. Without streaming, users see nothing until Gemini finishes generating the entire response. With streaming, text appears as soon as the first token is ready, which can shave several seconds off the perceived wait time.
