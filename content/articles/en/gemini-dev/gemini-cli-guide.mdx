## Harness AI Power from Your Terminal with Gemini CLI

Gemini CLI brings the capabilities of the Gemini API directly to your terminal, enabling code generation, analysis, documentation, and workflow automation without leaving the command line. Whether you're reviewing code, generating documentation, or integrating AI into shell scripts, Gemini CLI streamlines your development process.

## Installation and Setup

### Installing Gemini CLI

Install Gemini CLI globally using npm:

```bash
# Global installation (recommended)
npm install -g @anthropic-ai/gemini-cli

# Verify installation
gemini --version
```

### Configuring Your API Key

Before using Gemini CLI, you need a Gemini API key:

```bash
# Set via environment variable (recommended)
export GEMINI_API_KEY="your-api-key-here"

# Or configure globally
gemini config set api-key "your-api-key-here"

# Verify configuration
gemini config show
```

For local development, use a `.env` file:

```bash
# Create .env file
echo "GEMINI_API_KEY=your-api-key-here" > .env

# Add to .gitignore (CRITICAL!)
echo ".env" >> .gitignore
```

## Core Commands

### Text Generation

Generate text from command-line prompts:

```bash
# Simple text generation
gemini prompt "Explain asynchronous programming in Python"

# Multi-line prompt
gemini prompt "
Optimize this function:
def calculate_sum(numbers):
    total = 0
    for num in numbers:
        total = total + num
    return total
"

# Generate with specific model
gemini prompt "Write a REST API endpoint" --model gemini-2.5-pro
```

### File Analysis

```bash
# Analyze code for issues
gemini analyze app.py

# Diagnose errors with suggestions
gemini analyze error.log --analyze-errors

# Batch analyze multiple files
gemini analyze src/*.py --summary
```

### Code Generation

```bash
# Generate code in specific language
gemini generate "REST API with FastAPI" --lang python

# Generate multiple related files
gemini generate "User authentication system" --lang python --output-dir generated/

# Use predefined templates
gemini generate-from-template --template express-api
```

## Practical Use Cases

### Use Case 1: Code Review and Refactoring

Automatically review code for improvements:

```bash
# Review code quality
gemini review my-script.py

# Focus on specific aspects
gemini review my-script.py --focus security

# Generate detailed report
gemini review src/ --detailed --output code-review.md
```

Example output:

```bash
$ gemini review data_processor.py --focus performance

2. Line 78: Multiple iterations - consolidate into single pass

- Use NumPy for vectorized operations (10x speedup potential)
- Implement caching for expensive calculations
- Consider generator expressions for memory efficiency

Estimated impact: 0.8 tokens/execution
```

### Use Case 3: Shell Workflow Integration

Integrate Gemini CLI into bash scripts for automated pipelines:

```bash
#!/bin/bash
# build-and-optimize.sh

echo "🔨 Building project..."
npm run build || exit 1

echo "🧪 Running tests..."
npm test || exit 1

echo "📊 Analyzing code quality..."
gemini review src/ --focus quality --output reports/code-quality.md

echo "📝 Generating documentation..."
gemini doc generate src/ --output docs/api.md

echo "✅ Pipeline complete!"
```

Run it:

```bash
chmod +x build-and-optimize.sh
./build-and-optimize.sh
```

### Use Case 4: Batch File Processing

```bash
# Analyze all Python files in a directory
for file in src/**/*.py; do
  echo "Processing $file..."
  gemini analyze "$file" --output "reports/$(basename $file .py).txt"
done

# Process with pipe input
cat problematic-logs.txt | gemini prompt "Extract all critical errors"

# Chain multiple operations
cat requirements.txt | gemini prompt "Generate installation and usage docs"
```

### CI/CD Pipeline Integration

```bash
#!/bin/bash
# ci-checks.sh

# Run tests
npm test || exit 1

# Code quality check (fail on critical issues)
gemini review src/ --fail-on-critical --output quality.md

# Security scan
gemini security-scan src/ --strict || exit 1

# Generate test coverage report
gemini analyze coverage/index.json --format html --output coverage.html

echo "✅ All checks passed"
```

### Custom Prompts and Templates

```bash
# Create custom prompt templates
mkdir -p ~/.gemini/prompts

# Save reusable prompt
cat > ~/.gemini/prompts/security-review.txt << 'EOF'
Review the following code for security vulnerabilities:

1. Authentication and authorization issues
2. Input validation problems
3. Data exposure risks
4. Injection attack vectors
5. Dependency vulnerabilities
EOF

# Use template
gemini prompt "$(cat ~/.gemini/prompts/security-review.txt)" < mycode.py
```

## Complete Development Workflow

### End-to-End Feature Development

```bash
#!/bin/bash
# feature-workflow.sh

FEATURE="user-authentication"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

echo "🚀 Starting feature: $FEATURE"

# 1. Create implementation plan
echo "📋 Generating implementation plan..."
gemini prompt "Design a $FEATURE system with architecture, database schema, and API endpoints" \
  > docs/plan-$FEATURE-$TIMESTAMP.md

# 2. Generate boilerplate
echo "🤖 Generating code..."
gemini generate "Express.js with JWT $FEATURE" --lang javascript --output src/$FEATURE/

# 3. Generate tests
echo "✅ Generating tests..."
gemini generate "Unit tests for $FEATURE system" --lang javascript --output test/$FEATURE/

# 4. Security review
echo "🔒 Security check..."
gemini review src/$FEATURE/ --focus security --output reports/security-$TIMESTAMP.md

# 5. Generate docs
echo "📖 Documentation..."
gemini doc generate src/$FEATURE/ --output docs/api-$FEATURE.md

# 6. Performance analysis
echo "⚡ Performance analysis..."
gemini analyze src/$FEATURE/ --stats --output reports/stats-$TIMESTAMP.json

echo "✨ Feature development complete!"
echo "📁 Review reports in: reports/"
```

Run the workflow:

```bash
chmod +x feature-workflow.sh
./feature-workflow.sh
```

## Performance and Optimization

### Caching for Efficiency

```bash
# Enable prompt caching
gemini config cache-prompts on

# Check cache status
gemini cache status

# Clear cache if needed
gemini cache clear
```

### Parallel Processing

```bash
# Analyze multiple files in parallel
gemini batch-analyze src/ \
  --pattern "*.py" \
  --output-dir reports/ \
  --parallel 4  # Process 4 files simultaneously
```

## Troubleshooting

### Common Issues and Solutions

```bash
# Issue: API key not found
# Solution: Verify key configuration
gemini config show | grep api-key

# Issue: Network timeout
# Solution: Retry with exponential backoff
gemini prompt "..." --retry 3 --retry-delay 2000

# Issue: Rate limiting
# Solution: Reduce parallelism
gemini batch-analyze src/ --parallel 1 --delay 1000

# Issue: Token limit exceeded
# Solution: Process smaller chunks
gemini analyze large-file.py --max-tokens 100000 --chunk-size 50000
```

## Best Practices

1. **Environment Management**: Store API keys in environment variables or `.env` files, never in code
2. **Output Preservation**: Redirect CLI output to files for audit trails and reference
3. **Error Handling**: Use `||` in scripts to handle failures gracefully
4. **Incremental Integration**: Start with simple tasks, gradually add complexity
5. **Token Optimization**: Monitor token usage, avoid redundant API calls
6. **Version Control**: Include `.gemini` config files in repos when appropriate (without secrets)
