## Why Gemini and Cursor Make a Powerful Combination

**Cursor** has rapidly become one of the most popular AI-powered code editors, and while many developers use it with OpenAI's GPT models by default, you can also connect it to Google's **Gemini** models for a distinctly different experience. Gemini 3.1 Pro's massive context window and the blazing-fast inference of Gemini 3.1 Flash bring unique strengths to your daily coding workflow.

In this guide, you'll learn how to configure Gemini models in Cursor from scratch, including obtaining your API key, setting up the connection, and making the most of Cursor's Chat, Composer, and code completion features with Gemini under the hood.

## Getting Your Gemini API Key

To use Gemini models in Cursor, you'll first need an API key from **Google AI Studio**.

### Step-by-Step API Key Setup

1. Visit [Google AI Studio](https://aistudio.google.com/) and sign in with your Google account
2. Click "Get API Key" in the left sidebar
3. Select "Create API key" and choose a project for your key
4. Copy the generated key and store it securely

```python
# Verify your API key works (Python)
import google.generativeai as genai

genai.configure(api_key="YOUR_GEMINI_API_KEY")

model = genai.GenerativeModel("gemini-3.1-pro")
response = model.generate_content("Hello, Gemini!")
print(response.text)
# Expected output: Hello! How can I help you today?
```

**Important**: Always manage your API key through environment variables or `.env` files — never hardcode it. The free tier provides generous quotas that work well for individual development.

## Configuring Gemini Models in Cursor

Cursor supports **OpenAI-compatible APIs**, which means you can connect to Gemini through Google's OpenAI-compatible endpoint. Here's how to set it up.

### Configuration Steps (OpenAI-Compatible Mode)

1. Open Cursor and navigate to Settings with `Cmd + ,` (Mac) or `Ctrl + ,` (Windows/Linux)
2. Go to the "Models" section in the left menu
3. Click "Add model" and enter the following details

Here's what you'll need to fill in:

- **Model name**: `gemini-3.1-pro` (or `gemini-3.1-flash`)
- **API Base URL**: `https://generativelanguage.googleapis.com/v1beta/openai/`
- **API Key**: The key you obtained from Google AI Studio

```javascript
// Example settings.json configuration for reference
// (Using the Cursor UI is recommended for security)
{
  "cursor.ai.models": [
    {
      "name": "gemini-3.1-pro",
      "apiBase": "https://generativelanguage.googleapis.com/v1beta/openai/",
      "apiKey": "YOUR_GEMINI_API_KEY",
      "provider": "openai-compatible"
    },
    {
      "name": "gemini-3.1-flash",
      "apiBase": "https://generativelanguage.googleapis.com/v1beta/openai/",
      "apiKey": "YOUR_GEMINI_API_KEY",
      "provider": "openai-compatible"
    }
  ]
}
```

### Choosing the Right Model

Each Gemini model has distinct strengths, and switching between them based on your task can optimize both cost and quality.

- **Gemini 3.1 Pro**: Best for complex refactoring, architecture design, and understanding large codebases. Its context window exceeding 1 million tokens is a standout feature
- **Gemini 3.1 Flash**: Ideal for everyday code completion, quick questions, and snippet generation. Fast response times and lower cost
- **Gemini 3.1 Flash-Lite**: The most cost-efficient option, suitable for simple completions and formatting tasks

Cursor's chat interface lets you switch models on the fly before each message, so you can match the model to the complexity of your question in real time.

## Using Gemini in Cursor's Chat Feature

The **Chat** panel on the right side of Cursor is perfect for having conversations with Gemini about your code.

### Effective Prompt Patterns

Gemini excels at understanding context, so it can leverage the contents of your open files when answering questions. Here are some prompt patterns that work particularly well:

```
# Pattern 1: Code Explanation
Walk me through this function step by step.
Point out any improvements for the error handling.

# Pattern 2: Refactoring Suggestions
Refactor this component following React best practices.
Include performance optimizations and accessibility improvements.

# Pattern 3: Test Generation
Write unit tests for this function using Jest.
Include happy path, error cases, and edge cases.
```

### Leveraging `@` Mentions for Context

Cursor's chat supports `@` mentions to reference specific files and symbols. Combined with Gemini's massive context window, this enables project-wide understanding in a single query.

- `@filename.ts` — Include a specific file in the context
- `@folder` — Reference an entire folder
- `@codebase` — Search the entire project for relevant code

Gemini 3.1 Pro's context window of over 1 million tokens is particularly powerful for cross-file refactoring, dependency analysis, and understanding complex codebases that would exceed the limits of other models.

## Code Generation with Cursor's Composer and Gemini

Cursor's **Composer** (`Cmd + I` / `Ctrl + I`) generates and edits code from natural language instructions. Selecting a Gemini model here lets you tap into its unique strengths for code generation.

### Practical Example: Creating an API Endpoint

Here's an example of using Composer with Gemini to generate a REST API endpoint:

```typescript
// Composer instruction:
// "Create a blog article CRUD API with Express.
//  Include authentication, validation, error handling, and type definitions."

// Example code generated by Gemini:
import express, { Request, Response, NextFunction } from 'express';
import { z } from 'zod';

// Validation schema definition
const createArticleSchema = z.object({
  title: z.string().min(1).max(200),
  content: z.string().min(10),
  tags: z.array(z.string()).optional(),
});

type CreateArticleInput = z.infer<typeof createArticleSchema>;

const router = express.Router();

// Create an article
router.post('/articles', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const validated: CreateArticleInput = createArticleSchema.parse(req.body);

    // Save to database
    const article = await createArticle({
      ...validated,
      authorId: req.user.id,
      createdAt: new Date(),
    });

    res.status(201).json({ success: true, data: article });
  } catch (error) {
    if (error instanceof z.ZodError) {
      res.status(400).json({
        success: false,
        errors: error.errors,
      });
      return;
    }
    next(error);
  }
});

// Expected output: Express router registered successfully,
// POST /articles endpoint with validation-backed article creation
```

Gemini particularly shines in **type definition accuracy** and **comprehensive error handling**. When working in TypeScript projects, many developers find that Gemini generates fewer unnecessary `any` types and produces more precise type inference.

## Troubleshooting and Best Practices

Here are common issues when using Gemini in Cursor and how to resolve them.

### Common Issues and Solutions

**Connection errors**: Double-check your API Base URL. The trailing slash matters — `https://generativelanguage.googleapis.com/v1beta/openai/` is the correct format. A missing or extra slash can cause connection failures.

**Slow responses**: Gemini 3.1 Pro generates high-quality responses but can take a bit longer. If speed is your priority, switch to Gemini 3.1 Flash for faster results.

**Token limit errors**: The free tier has per-minute request limits and daily token quotas. If you're hitting errors frequently, check your quota in [Google AI Studio](https://aistudio.google.com/) or consider upgrading to a pay-as-you-go plan.

### Security Considerations

- Enter your API key through Cursor's settings UI rather than storing it in plain text in `settings.json`
- Review Gemini API's data handling policies before working with sensitive or proprietary code
- In team environments, issue individual API keys for each developer instead of sharing a single key

## FAQ

### Q: Does it cost money to use Gemini in Cursor?

Gemini API has a free tier that's generous enough for individual development. However, Cursor itself requires a Pro subscription for full features. The free tier includes approximately 15 requests per minute (for 3.1 Flash) and 1,500 requests per day.

### Q: Should I use Gemini or Cursor's built-in GPT-4o?

It depends on your use case. When you need to process large amounts of code at once, Gemini 3.1 Pro's context window gives it a clear advantage. For general coding assistance, GPT-4o is also excellent. The best approach is to try both and see which fits your workflow better.

### Q: Can I use Gemini in AI editors other than Cursor?

Absolutely. You can use Gemini models through the VS Code + Roo Code extension or Google's official Gemini Code Assist extension for VS Code. For more details, see [VS Code × Roo Code × Gemini Development Environment Guide](/en/articles/gemini-dev/vscode-roo-code-gemini-dev-environment).

### Q: Any tips for using Gemini in Cursor's Composer?

Be specific and structured with your instructions. Instead of just saying "build X," try something like "build X in TypeScript with Zod validation, proper error handling, and comprehensive type definitions." Providing clear technical requirements helps Gemini generate more accurate and production-ready code.
