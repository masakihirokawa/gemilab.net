## Introduction

Running AI agents safely in production is a critical challenge for enterprises. **NemoClaw**, developed by NVIDIA, is an open-source framework that provides a comprehensive stack for securely managing and executing AI agents, including those powered by Google Gemini.

In this beginner's guide, we'll explore NemoClaw's core concepts, architecture, and how to implement your first secure agent.

## What Is NemoClaw? Core Concepts

NemoClaw comprises three interconnected components:

- **Nemo Framework**: LLM lifecycle management, model optimization, and safe deployment
- **Claw (Controllable Language Agent Wrapper)**: Agent behavior control, real-time monitoring, and security enforcement
- **OpenShell Interface**: Secure, controlled connections between agents and external systems

### Why Integrate with Gemini?

- 🔒 **Multi-layer security**: Prevent unauthorized actions and enforce safety policies
- 🎯 **Real-time monitoring**: Track agent behavior and audit all decisions
- 🔄 **Model flexibility**: Switch between Gemini, Claude, GPT-4o seamlessly
- 📊 **Compliance-ready**: Automatic audit trails and comprehensive logging

## Build Your First NemoClaw Agent

### Basic Implementation

```python
from nemoclaw import Agent, GeminiConfig, OpenShellRuntime
import os

# Configure Gemini integration
config = GeminiConfig(
    api_key=os.getenv("GEMINI_API_KEY"),
    model="gemini-2.5-pro",
    temperature=0.7
)

# Initialize OpenShell runtime with security
runtime = OpenShellRuntime(
    security_level="strict",  # Production-grade security
    audit_enabled=True  # Enable comprehensive logging
)

# Create the agent
agent = Agent(
    name="data-analyst",
    config=config,
    runtime=runtime,
    description="Agent for data analysis tasks"
)

# Execute task
response = agent.run(
    task="Analyze sales data from the past 3 months",
    context={"data_source": "sales_db"}
)

print(response.output)
print(f"Execution time: {response.execution_time}s")
```

**Expected Output:**
```
Analysis Complete:
- Month-over-month growth: +15%
- Top 3 categories: SaaS, FinTech, E-Commerce
- Execution time: 2.341s
```

## Security Configuration Best Practices

### OpenShell Security Levels

NemoClaw provides security levels tailored to different environments:

| Level | Description | Best For |
|---|---|---|
| `lenient` | No restrictions | Local development |
| `standard` | Basic sandboxing | Staging environments |
| `strict` | Strict isolation | Production deployments |
| `ultra-strict` | Maximum restrictions | Regulated industries |

### Define Custom Security Policies

```python
from nemoclaw import SecurityPolicy

policy = SecurityPolicy(
    allowed_actions=["read_file", "api_call", "compute"],
    blocked_actions=["delete", "write_system", "exec_shell"],
    rate_limit=100,  # Max requests per minute
    timeout=30  # Execution timeout in seconds
)

agent.set_security_policy(policy)
```

## Frequently Asked Questions (FAQs)

**Q1: How does NemoClaw differ from building agents manually?**

A: Manual implementations require you to build security layers yourself. NemoClaw provides a battle-tested, production-ready stack with built-in sandboxing, audit logging, and multi-layer verification—all automated.

**Q2: Can I use models other than Gemini?**

A: Absolutely. NemoClaw supports Claude, GPT-4o, and other models. You simply swap the configuration and the runtime handles the rest.

**Q3: Can I develop locally and deploy to production?**

A: Yes. We recommend a progressive approach: `lenient` mode for local development, `standard` for staging, and `strict` mode for production deployments.
