## Why Add Gemini to Your Flutter App?

AI-powered features have shifted from a nice-to-have to a competitive necessity for mobile apps. By integrating Google's Gemini API into your Flutter project, you can add natural-language chat, image understanding, document summarization, and much more — all with a single codebase that runs on both iOS and Android.

In 2026, the recommended way to integrate Gemini into Flutter is through the **Firebase AI Logic SDK**. Unlike embedding an API key directly in the client, Firebase AI Logic handles authentication and security through Firebase App Check, making it the right choice for production-grade apps.

This guide walks you through everything from initial setup to implementing streaming chat and multimodal image analysis in your Flutter app.

## Prerequisites and Setup

### What You'll Need

- Flutter SDK 3.x or later
- Dart 3.x or later
- A Firebase project (create one at the [Firebase Console](https://console.firebase.google.com/))
- Gemini API enabled for your project (see [Gemini API Quickstart](/en/articles/gemini-api/gemini-api-quickstart))
- FlutterFire CLI installed

### Setting Up FlutterFire

```bash
# Install FlutterFire CLI globally
dart pub global activate flutterfire_cli

# Install Firebase CLI if not already installed
npm install -g firebase-tools
firebase login

# Create a new Flutter project
flutter create gemini_ai_app
cd gemini_ai_app

# Connect your app to Firebase (interactive project selection)
flutterfire configure
```

Running `flutterfire configure` auto-generates `lib/firebase_options.dart` with platform-specific configuration. This is the secure approach — no hard-coded API keys in your codebase.

### Add Dependencies

```yaml
# pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  firebase_core: ^3.6.0
  firebase_ai: ^0.1.0          # Firebase AI Logic SDK
  firebase_app_check: ^0.3.0   # Recommended for production security
```

```bash
flutter pub get
```

import 'package:flutter/material.dart';
import 'package:firebase_core/firebase_core.dart';
import 'package:firebase_app_check/firebase_app_check.dart';
import 'firebase_options.dart';

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // Initialize Firebase
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );

  // Enable App Check (recommended for production)
  await FirebaseAppCheck.instance.activate(
    androidProvider: AndroidProvider.playIntegrity,

## Building a Multi-Turn Chat Interface

### Core Chat Implementation

Here's a complete, functional chat screen using Firebase AI Logic SDK:

```dart
import 'package:firebase_ai/firebase_ai.dart';
import 'package:flutter/material.dart';

class ChatScreen extends StatefulWidget {
  const ChatScreen({super.key});

  @override
  State<ChatScreen> createState() => _ChatScreenState();
}

class _ChatScreenState extends State<ChatScreen> {
  final TextEditingController _controller = TextEditingController();
  final List<Map<String, String>> _messages = [];
  late final GenerativeModel _model;
  late final ChatSession _chat;
  bool _isLoading = false;

  @override
  void initState() {
    super.initState();
    // Initialize the model via Firebase AI Logic SDK
    _model = FirebaseAI.googleAI().generativeModel(
      model: 'gemini-3-flash-preview',
      systemInstruction: Content.system(
        'You are a helpful AI assistant. Keep your responses concise and clear.',
      ),
    );
    _chat = _model.startChat();
  }

  Future<void> _sendMessage(String text) async {
    if (text.trim().isEmpty) return;

    setState(() {
      _messages.add({'role': 'user', 'text': text});
      _isLoading = true;
    });
    _controller.clear();

    try {
      // Send message to Gemini and await response
      final response = await _chat.sendMessage(Content.text(text));
      final reply = response.text ?? '(No response)';

      setState(() {
        _messages.add({'role': 'assistant', 'text': reply});
      });
    } catch (e) {
      setState(() {
        _messages.add({'role': 'error', 'text': 'Error: $e'});
      });
    } finally {
      setState(() => _isLoading = false);
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Gemini Chat')),
      body: Column(
        children: [
          Expanded(
            child: ListView.builder(
              padding: const EdgeInsets.all(16),
              itemCount: _messages.length,
              itemBuilder: (context, index) {
                final msg = _messages[index];
                final isUser = msg['role'] == 'user';
                return Align(
                  alignment:
                      isUser ? Alignment.centerRight : Alignment.centerLeft,
                  child: Container(
                    margin: const EdgeInsets.symmetric(vertical: 4),
                    padding: const EdgeInsets.all(12),
                    decoration: BoxDecoration(
                      color: isUser ? Colors.blue[100] : Colors.grey[200],
                      borderRadius: BorderRadius.circular(12),
                    ),
                    child: Text(msg['text'] ?? ''),
                  ),
                );
              },
            ),
          ),
          if (_isLoading)
            const Padding(
              padding: EdgeInsets.all(8),
              child: CircularProgressIndicator(),
            ),
          Padding(
            padding: const EdgeInsets.all(8),
            child: Row(
              children: [
                Expanded(
                  child: TextField(
                    controller: _controller,
                    decoration: const InputDecoration(
                      hintText: 'Type a message...',
                      border: OutlineInputBorder(),
                    ),
                    onSubmitted: _sendMessage,
                  ),
                ),
                const SizedBox(width: 8),
                IconButton(
                  icon: const Icon(Icons.send),
                  onPressed: () => _sendMessage(_controller.text),
                ),
              ],
            ),
          ),
        ],
      ),
    );
  }
}
```

`ChatSession` automatically maintains conversation history, so Gemini has full context across multiple turns without any extra work on your end.

## Streaming Responses for a Better UX

Rather than waiting for the full response, streaming displays text as it's generated — a much more natural feel for users.

```dart
Future<void> _sendMessageStream(String text) async {
  if (text.trim().isEmpty) return;

  setState(() {
    _messages.add({'role': 'user', 'text': text});
    // Add empty assistant message to fill in progressively
    _messages.add({'role': 'assistant', 'text': ''});
    _isLoading = true;
  });
  _controller.clear();

  try {
    final stream = _chat.sendMessageStream(Content.text(text));

    await for (final chunk in stream) {
      if (!mounted) break; // Safety check: exit if widget is disposed
      final chunkText = chunk.text ?? '';
      setState(() {
        final lastIndex = _messages.length - 1;
        _messages[lastIndex] = {
          'role': 'assistant',
          'text': (_messages[lastIndex]['text'] ?? '') + chunkText,
        };
      });
    }
  } catch (e) {
    setState(() {
      _messages.add({'role': 'error', 'text': 'Error: $e'});
    });
  } finally {
    setState(() => _isLoading = false);
  }
}
```

## Image Understanding with Multimodal Input

Firebase AI Logic SDK supports images, video, and PDFs alongside text. Here's how to let users snap a photo and have Gemini describe it:

```dart
import 'package:image_picker/image_picker.dart';
import 'dart:io';

// Add to pubspec.yaml: image_picker: ^1.1.0

Future<void> _analyzeImage() async {
  final picker = ImagePicker();
  final pickedFile = await picker.pickImage(
    source: ImageSource.camera,
    maxWidth: 1024,      // Compress to avoid timeouts
    imageQuality: 85,
  );
  if (pickedFile == null) return;

  setState(() => _isLoading = true);

  try {
    final imageBytes = await File(pickedFile.path).readAsBytes();

    // Combine text prompt and image data in a single request
    final response = await _model.generateContent([
      Content.multi([
        TextPart('What is in this image? Describe it in detail.'),
        DataPart('image/jpeg', imageBytes),
      ]),
    ]);

    final description = response.text ?? 'Could not analyze image.';
    setState(() {
      _messages.add({'role': 'user', 'text': '[Image sent]'});
      _messages.add({'role': 'assistant', 'text': description});
    });
  } catch (e) {
    setState(() {
      _messages.add({'role': 'error', 'text': 'Image analysis error: $e'});
    });
  } finally {
    setState(() => _isLoading = false);
  }
}
```

The apple has a natural glossy sheen with a green leaf attached at the stem.
The lighting appears even, suggesting a studio or lightbox setup.

## Common Errors and Fixes

### Error 1: `FirebaseException: [firebase_ai/permission-denied]`

**Cause:** The Gemini API hasn't been enabled in your Firebase project.

**Fix:** Go to Firebase Console → AI Logic → click "Enable Gemini API". Or enable "Generative Language API" in Google Cloud Console for your project.

### Error 2: `MissingPluginException` on iOS

**Cause:** CocoaPods hasn't been updated after running `flutterfire configure`.

**Fix:**
```bash
cd ios && pod install --repo-update && cd ..
flutter clean && flutter run
```

### Error 3: App crashes when closing chat during streaming

**Cause:** `setState` is called after the widget is disposed.

**Fix:** Always check `mounted` inside the `await for` loop:
```dart
await for (final chunk in stream) {
  if (!mounted) break;
  setState(() { /* ... */ });
}
```

### Error 4: Large images cause timeout errors

**Cause:** High-resolution camera images exceed the upload size limit.

**Fix:** Use the `maxWidth` and `imageQuality` parameters in `image_picker` to reduce file size before sending.

## FAQ

### Q1. What's the difference between `firebase_ai` and `google_generative_ai`?

The `google_generative_ai` package uses Gemini API keys directly. It's fine for local prototyping, but embedding API keys in client apps is a security risk. The `firebase_ai` package (Firebase AI Logic SDK) routes requests through Firebase, leveraging Firebase Authentication and App Check to secure API access without exposing keys to the client. **For any app you ship to production, always use `firebase_ai`.**

### Q2. How much does it cost to use Gemini in a Flutter app?

Firebase AI Logic SDK pricing follows the Gemini API model. Gemini Flash models include a free tier, and beyond that you pay per token. For apps with significant traffic, set up [Gemini API Spend Caps](/en/articles/gemini-api/gemini-api-spend-caps-guide) to prevent unexpected bills. Check [Google AI Studio pricing](https://ai.google.dev/pricing) for the latest rates.

### Q3. Can I persist chat history to Firestore?

Yes — because Firebase AI Logic integrates natively with the Firebase ecosystem, wiring up Firestore is straightforward. The [Gemini + Firebase Integration Guide](/en/articles/gemini-dev/gemini-firebase-integration-guide) covers storing conversation history, user-specific chat sessions, and server-side Gemini calls via Cloud Functions.

### Q4. Does this work on both iOS and Android?

Absolutely. Flutter's cross-platform nature means the same Dart code runs on both platforms. The `flutterfire configure` command generates platform-specific configuration automatically, so you don't need to write any native code.
