## Introduction

Google AI Studio (`aistudio.google.com`) is the easiest way to test the Gemini API in a browser, but many newcomers hit setup roadblocks. This guide covers the six most common errors and their definitive solutions.

## 1. API Key Not Recognized

### The Problem: "API key not valid" Error

You've pasted your API key, but it keeps rejecting. Usually it's one of these:

1. **Regional restrictions** — API key generation is blocked in some countries (including parts of Japan)
2. **Billing not linked** — the project has no payment method registered
3. **Extra whitespace** — trailing spaces copied when pasting the key
4. **Wrong project** — API key from a different Google Cloud project

### Solution: Enable Billing and Link to Project

**Step 1: Verify billing is active**

```
1. Open https://console.cloud.google.com
2. Click the project selector dropdown (top left)
3. Select the correct project
4. Left menu → "Billing"
5. Confirm "Billing account linked"
6. If no billing account, create one
```

**Step 2: Enable the Generative Language API**

```
1. Left menu → "APIs & Services" → "Enabled APIs & services"
2. Click "+ Enable APIs and Services"
3. Search for "Generative Language API"
4. Click "Enable"
```

**Step 3: Create API Key**

```bash
# Test the API key with curl
curl -H "x-goog-api-key: YOUR_API_KEY" \
  "https://generativelanguage.googleapis.com/v1beta/models?key=YOUR_API_KEY"

# Success: returns JSON with available models
# Error: "Invalid API key" or "Permission denied"
```

**Best Practice: Use Environment Variables**

```python
import os
from dotenv import load_dotenv

# Create .env file:
# GOOGLE_API_KEY=your_actual_key_here

load_dotenv()
api_key = os.getenv("GOOGLE_API_KEY")

if not api_key:
    raise ValueError(
        "GOOGLE_API_KEY not found. "
        "Create a .env file with GOOGLE_API_KEY=your_key"
    )

import google.generativeai as genai
genai.configure(api_key=api_key)
```

⚠️ **Never commit `.env` files to Git. Add to `.gitignore`:**

```
# .gitignore
.env
.env.local
*.key
```

## 2. Model Not Available Error

### The Problem: Old Model Names Don't Work

Google frequently deprecates and updates models. Using an old name causes errors.

**Deprecated Models (Don't Use):**
- ❌ `gemini-1.5-pro-latest` (deprecated January 2026)
- ❌ `gemini-pro` (sunset)
- ❌ `gemini-pro-vision` (legacy)

**Current Recommended Models:**
- ✅ `gemini-2.5-pro` (latest, most capable)
- ✅ `gemini-2.5-flash` (fast, low-cost)
- ✅ `gemini-2.0-flash-exp` (image generation)

### Solution: Verify Available Models Programmatically

```python
import google.generativeai as genai

def list_available_models():
    """Display all currently available models"""
    print("Available models:\n")
    for model in genai.list_models():
        print(f"Model: {model.name}")
        print(f"  Display: {model.display_name}")
        print(f"  Input limit: {model.input_token_limit:,}")
        print(f"  Output limit: {model.output_token_limit:,}")
        print(f"  Supported: {model.supported_generation_methods}")
        print()

list_available_models()
```

**Expected output:**
```
Available models:

Model: models/gemini-2.5-pro
  Display: Gemini 2.5 Pro
  Input limit: 1,000,000
  Output limit: 4,096
  Supported: ['generateContent', 'countTokens']

Model: models/gemini-2.5-flash
  Display: Gemini 2.5 Flash
  ...
```

### Choosing Models in Google AI Studio

```
In Google AI Studio (https://aistudio.google.com):

1. Click "+" button → "New chat"
2. Click model dropdown at top
3. Recommended models:
   • General use: gemini-2.5-pro
   • Speed priority: gemini-2.5-flash
```

## 3. Different Results: Google AI Studio vs. API

### The Problem: Studio Works, API Gives Different Answers

Three key settings differ between environments:

1. **Temperature** — controls randomness (creativity)
2. **System instruction** — defines model behavior
3. **Safety settings** — content filter aggressiveness

### Solution: Match Parameters Between Environments

```python
import google.generativeai as genai

def configure_consistent_params():
    """Match Google AI Studio defaults exactly"""
    model = genai.GenerativeModel(
        model_name="gemini-2.5-pro",
        # These match Google AI Studio defaults
        generation_config=genai.GenerationConfig(
            temperature=1.0,        # Default creativity level
            top_p=0.95,            # Diversity
            top_k=40,              # Candidate pool size
            max_output_tokens=8192,  # Max response length
        ),
        system_instruction="Be helpful and accurate.",
        safety_settings=[
            {
                "category": genai.types.HarmCategory.HARM_CATEGORY_HARASSMENT,
                "threshold": genai.types.HarmBlockThreshold.BLOCK_ONLY_HIGH,
            },
            {
                "category": genai.types.HarmCategory.HARM_CATEGORY_HATE_SPEECH,
                "threshold": genai.types.HarmBlockThreshold.BLOCK_ONLY_HIGH,
            },
            {
                "category": genai.types.HarmCategory.HARM_CATEGORY_SEXUALLY_EXPLICIT,
                "threshold": genai.types.HarmBlockThreshold.BLOCK_ONLY_HIGH,
            },
            {
                "category": genai.types.HarmCategory.HARM_CATEGORY_DANGEROUS_CONTENT,
                "threshold": genai.types.HarmBlockThreshold.BLOCK_ONLY_HIGH,
            },
        ]
    )
    return model

model = configure_consistent_params()
response = model.generate_content("Explain machine learning briefly")
print(response.text)
```

**Expected output:**
```
Machine learning is a subset of artificial intelligence...
```

### Parameter Reference

| Parameter | AI Studio Default | Effect |
|-----------|------------------|--------|
| Temperature | 1.0 | Higher = more creative, 0 = deterministic |
| Top P | 0.95 | Higher = more diverse |
| Top K | 40 | Limits candidate pool |
| Max tokens | 8192 | Maximum response length |

**Debugging Steps:**

1. Copy exact prompt from AI Studio
2. Set temperature = 0 (deterministic)
3. Add same system instruction
4. Run same safety settings
5. Compare outputs (should be identical)

## 4. File Upload Failures

### The Problem: "File too large" or Format Error

Size and format limits in Google AI Studio:

- **Free plan**: Maximum 20 MB
- **Paid plan**: Maximum 2 GB
- **Supported formats**: PDF, PNG, JPG, WEBP, MP4, plain text

### Solution: Check Size and Format Before Upload

```python
import google.generativeai as genai
import os

def upload_file_with_validation(file_path, max_size_mb=20):
    """Validate and upload file"""

    # Check file size
    file_size_mb = os.path.getsize(file_path) / (1024 * 1024)
    if file_size_mb > max_size_mb:
        print(f"❌ File is {file_size_mb:.1f} MB, limit is {max_size_mb} MB")
        print("Solutions:")
        print("  1. Compress the file")
        print("  2. Split into smaller files")
        print("  3. Upgrade to paid plan (2 GB limit)")
        return None

    # Check file format
    supported = {
        '.pdf': 'application/pdf',
        '.txt': 'text/plain',
        '.png': 'image/png',
        '.jpg': 'image/jpeg',
        '.jpeg': 'image/jpeg',
        '.webp': 'image/webp',
        '.mp4': 'video/mp4',
    }

    _, ext = os.path.splitext(file_path)
    if ext.lower() not in supported:
        print(f"❌ Format {ext} not supported")
        print(f"Supported: {', '.join(supported.keys())}")
        return None

    # Upload
    try:
        file = genai.upload_file(file_path)
        print(f"✓ Uploaded: {file.name}")
        print(f"  MIME: {file.mime_type}")
        print(f"  Size: {os.path.getsize(file_path) / 1024:.1f} KB")
        return file
    except Exception as e:
        print(f"❌ Upload failed: {e}")
        return None

# Usage
file = upload_file_with_validation("document.pdf")
if file:
    model = genai.GenerativeModel("gemini-2.5-pro")
    response = model.generate_content([
        "Summarize this document",
        file
    ])
    print(response.text)
```

**Expected output:**
```
✓ Uploaded: files/xyz123abc456
  MIME: application/pdf
  Size: 2048.5 KB
```

### Splitting Large Files

```python
def split_text_file(file_path, chunk_size_mb=15):
    """Split large text file into smaller chunks"""
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()

    file_size_mb = len(content.encode('utf-8')) / (1024 * 1024)
    if file_size_mb <= chunk_size_mb:
        return [file_path]

    chunk_size_bytes = chunk_size_mb * 1024 * 1024
    parts = []

    for i in range(0, len(content), chunk_size_bytes):
        chunk = content[i:i+chunk_size_bytes]
        part_file = f"{file_path}.part{len(parts)+1}.txt"
        with open(part_file, 'w', encoding='utf-8') as f:
            f.write(chunk)
        parts.append(part_file)

    print(f"✓ Split into {len(parts)} parts")
    return parts

# Usage
parts = split_text_file("large_book.txt")
for part in parts:
    upload_file_with_validation(part)
```

## 5. Fine-Tuning Model Errors

### The Problem: Model Not Available for Tuning

Gemini fine-tuning is **API-only** — Google AI Studio doesn't support it. The `models/` names used in fine-tuning are different from regular models.

### Solution: Fine-Tune via API

```python
import google.generativeai as genai
import time

def tune_model(training_data_file, model_display_name):
    """Create a fine-tuned custom model"""

    # training_data_file is JSONL format:
    # {"text_input": "...", "output": "..."}

    operation = genai.create_tuned_model(
        source_model="models/gemini-1.5-flash-001",
        training_data=training_data_file,
        id=f"custom-{int(time.time())}",
        display_name=model_display_name,
        epoch_count=3,
        batch_size=4,
        learning_rate=0.001,
    )

    print(f"Tuning started: {operation.name}")

    # Wait for completion
    result = operation.result()
    print(f"✓ Complete: {result.name}")
    return result.name

def use_tuned_model(tuned_model_name, prompt):
    """Use the fine-tuned model"""
    model = genai.GenerativeModel(model_name=tuned_model_name)
    response = model.generate_content(prompt)
    return response.text

# Training data format (save as JSONL)
training_data = [
    {"text_input": "What is AI?", "output": "Artificial Intelligence is..."},
    {"text_input": "Explain machine learning", "output": "Machine learning is..."},
    {"text_input": "What is NLP?", "output": "Natural Language Processing..."},
]

# Save JSONL file
import json
with open("training_data.jsonl", "w") as f:
    for item in training_data:
        f.write(json.dumps(item) + "\n")

# Tune model
tuned_name = tune_model("training_data.jsonl", "Custom FAQ Bot")

# Use it
answer = use_tuned_model(tuned_name, "What is AI?")
print(answer)
```

**Training Data Format (JSONL):**
```json
{"text_input": "How do I use Python?", "output": "Python is a programming language..."}
{"text_input": "What is a function?", "output": "A function is a reusable block..."}
{"text_input": "Explain loops", "output": "Loops repeat code multiple times..."}
```

Google AI Studio's "Export to Code" feature sometimes generates code with:

- ❌ `ModuleNotFoundError: No module named 'google'`
- ❌ Outdated API syntax

## FAQ

**Q. Where should I store my API key?**

A. Use `.env` files locally (with `dotenv`), and **never commit to Git**. In production, use Google Cloud Secret Manager or your platform's native secrets (AWS Secrets Manager, GitHub Secrets, etc.).

**Q. Can I use Google AI Studio offline?**

A. No. Google AI Studio is cloud-only and requires internet. For offline capabilities, consider [Claude](/articles/claude-ai) or self-hosted models like Llama.

**Q. How long are uploaded files retained?**

A. Files are kept for **2 days**, then auto-deleted. For long-term storage, use Google Cloud Storage and reference files by URL.

**Q. Can I save and share chat sessions?**

A. Your chat history is saved locally in your browser. You can resume sessions from the left sidebar, but if you clear browser data, the history is lost. To share, export the conversation manually.

**Q. What's the difference between Gemini 2.5 Pro and Flash?**

A. **Pro** = better quality, slower, more expensive. **Flash** = faster, cheaper, sufficient for most tasks. Use Pro for complex reasoning, Flash for conversational AI.

## Best Practices Summary

1. **API keys in environment variables** — never hardcode them
2. **Match temperature between Studio and API** — for reproducible results
3. **Check available models monthly** — Google updates them frequently
4. **File retention is 2 days** — don't rely on uploads being permanent
5. **Test with `temperature=0` for consistency** — deterministic for debugging
