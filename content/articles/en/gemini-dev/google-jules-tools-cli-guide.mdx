## What Is Jules Tools CLI? A New Way to Control AI Coding from the Terminal

**Jules Tools** is the official command-line interface for Google's AI coding agent, Jules. Released in March 2026, this CLI lets developers delegate coding tasks to Jules without ever leaving their terminal workflow.

Jules itself leverages the advanced reasoning capabilities of Gemini 2.5 Pro to autonomously handle bug fixes, test generation, refactoring, and dependency updates in an asynchronous fashion. Jules Tools serves as both a dashboard and a command surface, enabling you to manage remote coding sessions, track progress, pull results, and script complex multi-task workflows — all from the command line.

## Installation

Jules Tools is distributed as an official npm package by Google. If you have Node.js installed, setup takes a single command.

### Global Installation

```bash
# Install globally via npm
npm install -g @google/jules

# Verify the installation
jules --version
```

### Quick Run with npx

If you want to try it without a permanent install, use `npx`:

```bash
# Run without installing
npx @google/jules remote list --repo
```

### Initial Authentication

On first run, Jules Tools will prompt you to authenticate with your Google account. Once authenticated, you can execute commands against any GitHub repository connected to Jules.

```bash
# Start the authentication flow
jules auth login

# Check your authentication status
jules auth status
```

## Core Commands

The Jules Tools command structure is designed to be intuitive. Here are the essential commands you need to know.

### `jules remote list` — View Repositories and Sessions

Check your connected repositories or monitor active sessions:

```bash
# List all repositories connected to Jules
jules remote list --repo

# Expected output:
# ┌─────────────────────────────────┐
# │ Connected Repositories          │
# ├─────────────────────────────────┤
# │ your-org/frontend-app           │
# │ your-org/api-server             │
# └─────────────────────────────────┘

# List all active sessions
jules remote list --session

# Expected output:
# ┌──────────┬──────────────────────┬──────────┐
# │ ID       │ Task                 │ Status   │
# ├──────────┼──────────────────────┼──────────┤
# │ 123456   │ Fix login bug        │ Running  │
# │ 123457   │ Add unit tests       │ Complete │
# └──────────┴──────────────────────┴──────────┘
```

### `jules new` — Create a New Task

Delegate a coding task to Jules. The tool automatically detects the repository from your current working directory, so the `--repo` flag is often optional.

```bash
# Create a task using the current directory's repo
jules new "Add unit tests for the user authentication module"

# Specify a repository explicitly
jules new --repo your-org/api-server "Translate the README to Japanese"

# Expected output:
# ✓ Session created: #128901
# ℹ Jules is working on: "Add unit tests for the user authentication module"
# ℹ Repository: your-org/api-server
# ℹ Track progress: jules remote list --session
```

### `jules remote pull` — Retrieve Completed Results

Pull the code changes from a completed Jules session into your local environment:

```bash
# Pull results by session ID
jules remote pull --session 128901

# Expected output:
# ✓ Changes pulled from session #128901
# ℹ Modified files:
#   - src/auth/__tests__/login.test.ts (+142 lines)
#   - src/auth/__tests__/signup.test.ts (+98 lines)
```

## Parallel Execution for Maximum Productivity

One of Jules Tools' most powerful features is **parallel task execution**. The `--parallel` flag lets you run multiple sessions simultaneously, either exploring different approaches to the same problem or batch-processing a list of tasks.

### Parallel Sessions for the Same Task

```bash
# Spin up 3 parallel sessions for the same task
jules new --parallel 3 "Identify performance bottlenecks and suggest optimizations"

# Expected output:
# ✓ Created 3 parallel sessions: #128902, #128903, #128904
# ℹ Each session will independently work on the task
```

### Batch Task Submission

You can script bulk task submission from a TODO list or issue tracker:

```bash
# Submit each line of TODO.md as a separate Jules task
cat TODO.md | while IFS= read -r line; do
  jules new "$line"
  sleep 2  # Respect API rate limits
done

# Expected output:
# ✓ Session created: #128905 — "Fix CORS issue in API"
# ✓ Session created: #128906 — "Add pagination to user list"
# ✓ Session created: #128907 — "Update deprecated dependencies"
```

## Practical Use Case: CI/CD Pipeline Integration

Jules Tools is designed to be scriptable, making it straightforward to integrate into GitHub Actions and other CI/CD systems for automated development workflows.

### Auto-Generate Tests on Pull Requests

```yaml
# .github/workflows/jules-auto-test.yml
name: Jules Auto Test Generation
on:
  pull_request:
    types: [opened]

jobs:
  generate-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Jules Tools
        run: npm install -g @google/jules

      - name: Authenticate with Jules
        run: jules auth login --token ${{ secrets.JULES_API_TOKEN }}

      - name: Generate tests for changed files
        run: |
          CHANGED=$(git diff --name-only origin/main...HEAD | grep -E '\.(ts|js)$')
          if [ -n "$CHANGED" ]; then
            jules new "Write unit tests for the following files: $CHANGED"
          fi
```

### TUI (Text User Interface) Mode

For developers who prefer interactive workflows, Jules Tools also offers a built-in TUI mode:

```bash
# Launch the dashboard view
jules remote

# Interactively create a new task
jules /new
```

The TUI provides guided workflows for repository selection and task description, making it accessible even if you are not deeply familiar with CLI tooling.

## Jules Tools vs. Gemini CLI: Key Differences

Both tools operate from the terminal, but they serve fundamentally different purposes:

| Feature | Jules Tools | Gemini CLI |
|---------|------------|-----------|
| Purpose | Asynchronous task delegation | Real-time interactive coding |
| Execution | Google Cloud VM | Local machine |
| Processing | Async (background) | Sync (real-time) |
| Output | Pull Requests / patches | Direct file changes |
| Best For | Bug fixes, test generation, refactoring | Code exploration, debugging, interactive dev |

## FAQ

### Q: Do I need to connect a GitHub repository to use Jules Tools?

Yes, Jules operates by connecting to GitHub repositories. You need to first link your repository through the Jules web interface at jules.google.com. After that, all operations can be performed entirely from the CLI.

### Q: How do I know when a session is complete?

Run `jules remote list --session` to check the status of all sessions. Statuses include "Running," "Complete," and "Failed." You can also configure email notifications or webhooks through the Jules settings.

### Q: Are my uncommitted local changes reflected in Jules sessions?

No, Jules works from the remote GitHub repository. If you want your latest changes included, commit and push them before creating a Jules task.

### Q: Does Jules review its own generated code?

Yes. Jules includes an **Adversarial Review** system that automatically critiques every proposed change before presenting it to the developer. This built-in peer review helps maintain high code quality and reduces the chance of introducing bugs.

### Q: Can I use Jules Tools with private repositories?

Absolutely. Jules Tools works with both public and private repositories. Simply grant the appropriate permissions when connecting your repository through GitHub, and Jules will be able to access and work on your private codebase.
