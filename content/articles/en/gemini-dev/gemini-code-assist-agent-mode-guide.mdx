## What Is Gemini Code Assist Agent Mode?

Google's **Gemini Code Assist** is an AI coding assistant that runs inside VS Code and IntelliJ IDEs. While its standard features handle line-by-line completions and inline suggestions, **Agent Mode** operates as a full-fledged AI pair programmer capable of tackling complex, multi-file tasks autonomously.

Unlike standard autocomplete, Agent Mode employs **autonomous multi-step reasoning** to plan, execute, and verify changes across multiple files, frameworks, and contexts. It can generate code from design documents, implement features from TODO comments, and connect to external tools through MCP servers.

This guide covers everything from setup to advanced workflows to help you get the most out of Agent Mode.

## Setting Up Agent Mode

### VS Code Configuration

Agent Mode is available in both VS Code and IntelliJ IDEs. Here's how to get started in VS Code.

```bash
# Prerequisites:
# 1. VS Code installed
# 2. Gemini Code Assist extension installed
# 3. Signed in with your Google account

# Setup steps:
# 1. Click the Gemini Code Assist icon in the activity bar
# 2. The chat panel opens
# 3. Click the "Agent" toggle to switch to Agent Mode
# 4. Type your prompt and press Enter

# When Agent Mode is active:
# [Agent Mode: ON] indicator appears with a different background color
```

### IntelliJ Setup

Agent Mode also works in IntelliJ-based IDEs (IntelliJ IDEA, WebStorm, PyCharm, etc.). Install the Gemini Code Assist plugin and switch to the Agent tab in the chat panel.

## Core Capabilities

### 1. Multi-Step Reasoning

Agent Mode's greatest strength is breaking complex tasks into multiple steps and executing them autonomously.

```python
# Example Agent Mode prompt:
# "Create a user authentication API with FastAPI.
#  Include JWT tokens, password hashing,
#  and refresh token support."

# → Agent Mode execution flow:
# Step 1: Analyze project structure
# Step 2: Check dependencies (fastapi, python-jose, passlib)
# Step 3: Define models (User, Token, TokenData)
# Step 4: Create security utilities (hashing, JWT generation/verification)
# Step 5: Implement API endpoints (register, login, refresh, me)
# Step 6: Configure middleware

# Generated file structure:
# app/
# ├── main.py           ← FastAPI application
# ├── models.py         ← Pydantic models
# ├── auth.py           ← JWT / hashing utilities
# ├── routes/
# │   └── auth.py       ← Auth endpoints
# └── requirements.txt  ← Dependencies

# Example generated auth.py:
from datetime import datetime, timedelta
from jose import JWTError, jwt
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

SECRET_KEY = "your-secret-key"  # Use environment variable in production
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

# Expected output:
# → 5 files auto-generated
# → Imports correctly wired between files
# → requirements.txt populated with needed packages
```

### 3. MCP Server Integration

Agent Mode connects to external tools through **MCP (Model Context Protocol)** servers. This enables database operations, API calls, and file system access beyond the IDE.

```json
// MCP server configuration in .vscode/settings.json
{
  "gemini.mcp.servers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${env:GITHUB_TOKEN}"
      }
    },
    "database": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "${env:DATABASE_URL}"
      }
    }
  }
}

// With MCP connected, you can prompt:
// "Read GitHub Issue #42 and implement the fix"
// → Agent fetches the issue via GitHub API
// → Understands the problem description
// → Generates the fix automatically
```

## Standard Mode vs Agent Mode

| Feature | Standard Mode | Agent Mode |
|---------|--------------|------------|
| Completions | 1–few lines | Multiple files |
| Reasoning | Single step | Multi-step |
| File operations | Read only | Create and edit |
| MCP integration | No | Yes |
| Plan review | No | Yes |
| Terminal access | No | Yes |
| Best for | Quick completions | Complex implementations |

**When to use which**: For small, single-file changes, standard mode is faster. Agent Mode shines when adding features across multiple files or performing large-scale refactoring.

## Frequently Asked Questions

### Is Agent Mode free to use?

Gemini Code Assist's Individual plan (free) includes Agent Mode access, though with daily request limits and some feature restrictions. The Enterprise plan unlocks full capabilities.

### Can Agent Mode change files without my permission?

Agent Mode presents a plan and seeks approval before making changes. However, changes to resources outside your IDE (such as database operations) cannot be undone, so review plans carefully when external actions are involved.

### Which programming languages are supported?

Agent Mode supports Python, JavaScript/TypeScript, Java, Go, Kotlin, C++, Rust, and other major languages. Python and TypeScript have the most comprehensive support.

### How does Agent Mode compare to GitHub Copilot?

The key differences are MCP integration and multi-step reasoning. GitHub Copilot excels at inline code completions, while Gemini Code Assist Agent Mode is stronger at autonomous execution of complex, multi-file tasks. It also has deeper integration with Google Cloud services.
