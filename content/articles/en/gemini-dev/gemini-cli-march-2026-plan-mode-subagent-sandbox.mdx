## Plan Mode—Detailed Pre-Execution Planning

### Plan Mode Fundamentals

Plan Mode generates detailed execution plans before code runs, requiring user confirmation before proceeding. Default-enabled in v0.34.0.

### Basic Workflow

```bash
# v0.34.0+ includes Plan Mode by default
gemini run --script=deploy.rb --target=production

# Output shows detailed plan
# Step 1: DB connection check
# Step 2: Backup creation
# Step 3: Migration
# Step 4: Validation
# Total time: 167 seconds, Risk: LOW

# Continue? (Y/n)
```

### Research Subagents

Plan Mode automatically investigates multiple execution paths:

```bash
gemini run --script=optimize-api.rb --research

# Primary Plan (Strategy A):
#   - Caching (Redis)
#   - Query optimization
#   - Index addition
#   Improvement: +35%

# Alternative Plan 1 (Strategy B):
#   - CDN deployment
#   - Client-side caching
#   - Image optimization
#   Improvement: +28%

# Recommended: Strategy A + C combination
# Total improvement: +60%
```

### Annotation Support

```bash
gemini run --script=deploy.rb \
  --annotation="Important: Release notes v2.1.0 documented" \
  --annotation="Monitor: CPU, memory leaks" \
  --annotation="On-call: @devops-team"
```

## gVisor/LXC Sandbox—Secure Execution

### v0.34.0 Sandbox Introduction

gVisor (Google-developed secure container runtime) + LXC provide isolated script execution.

### Sandbox Architecture

```
Host Machine
  │
  ├─ gVisor User-space Kernel
  │  └─ LXC Containers
  │     ├─ Container A (filesystem isolated)
  │     ├─ Container B (network isolated)
  │     └─ Container C (process isolated)
  │
  └─ Protected Host Resources
```

### Activation & Configuration

```bash
# v0.34.0 enables by default
gemini run --script=untrusted-code.rb

# Check status
gemini sandbox --status
# Output: gVisor runtime (LXC backend) - ACTIVE

# Detailed config
gemini config sandbox.yml

# sandbox.yml
---
runtime: gvisor
backend: lxc
security_level: strict
resource_limits:
  memory: 2G
  cpu: 2
  disk: 10G
isolation:
  filesystem: enabled
  network: enabled
```

### External Resource Access Control

```bash
# Allow specific network access
gemini run --script=api-call.rb \
  --allow-network=https://api.example.com \
  --allow-network=https://api.third-party.com

# Allow filesystem access
gemini run --script=file-processor.rb \
  --allow-read=/data/input \
  --allow-write=/data/output \
  --deny-all-others
```

### Security Policy Definition

```ruby
# policy.rb
Gemini.define_sandbox_policy :api_integration do
  # Network access
  allow_network :https, hosts: [
    'api.example.com',
    'auth.service.com'
  ]

  # File access
  allow_read '/etc/config/api_keys.enc'
  allow_write '/var/log/api_calls.log'

  # Environment variables
  allow_env 'DATABASE_URL'
  allow_env 'API_KEY'

  # Prohibit dangerous operations
  deny :system_call, 'rm -rf'
  deny :file_delete, '/etc'
  deny :network, 'http://localhost:6379'
end

# Apply during execution
gemini run --script=api.rb --policy=api_integration
```

## A2A Remote Agent—HTTP Auth Agent Communication

### A2A (Agent-to-Agent) Authentication Setup

```bash
# Primary agent
gemini agent create \
  --name="primary-agent" \
  --port=9000 \
  --auth=http_bearer

# Satellite agent
gemini agent create \
  --name="satellite-agent" \
  --port=9001 \
  --upstream="https://primary-agent.local:9000" \
  --upstream-token="eyJhbGc..."
```

### Multi-Agent Distributed Tasks

```bash
# Coordinate multiple remote agents
gemini run --script=distributed-build.rb \
  --agents=[
    "primary-agent@localhost:9000",
    "build-agent@ci.example.com:9001",
    "test-agent@ci.example.com:9002",
    "deploy-agent@prod.example.com:9003"
  ]

# distributed-build.rb
Gemini.distributed_task do |task|
  task.coordinate_with :primary_agent do
    # Task distribution
  end

  task.send_to :build_agent do
    compile_source
  end

  task.send_to :test_agent do
    run_tests
  end

  task.send_to :deploy_agent do
    deploy_to_production
  end
end
```

### HTTP Bearer Token Authentication

```bash
# Token generation
gemini auth generate-token \
  --agent="primary-agent" \
  --ttl="30d" \
  --scopes="read,write,execute"

# HTTP header auth
curl -H "Authorization: Bearer eyJhbGc..." \
  https://primary-agent.local:9000/execute \
  -X POST \
  -d @task.json
```

## Loop Detection—Prevent Infinite Loops

### Loop Detection Mechanism

v0.34.0 automatically detects and safely halts infinite loops.

### Configuration

```bash
# Default loop detection settings
gemini config loop-detection.yml

# loop-detection.yml
---
enabled: true
detection_methods:
  - stack_trace_pattern_matching
  - execution_time_threshold
  - cpu_usage_anomaly

stack_trace:
  max_same_function_depth: 50
  timeout: 300s

cpu_anomaly:
  threshold: 95%
  duration: 30s

auto_stop:
  action: graceful_shutdown
  notify: true
  save_state: true
```

```bash

  --routing-strategy=failover \
```
