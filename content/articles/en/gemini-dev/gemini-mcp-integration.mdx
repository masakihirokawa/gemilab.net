## Extending Gemini with Model Context Protocol

The Model Context Protocol (MCP) offers a powerful way to expand Gemini's capabilities beyond its core functions. By connecting MCP servers, you can grant Gemini direct access to external tools, databases, and APIs—enabling more sophisticated automation and richer interactions with your application ecosystem. This guide covers everything you need to know about MCP integration with Gemini.

## What is Model Context Protocol?

MCP is a standardized protocol that establishes a bridge between AI assistants and external tools, services, or data sources. Think of it as a universal adapter that allows Gemini to interact with virtually any external system using a consistent, well-defined interface.

Key advantages of MCP:

- **Unified Interface**: JSON-based communication ensures compatibility across different tools and platforms
- **Dynamic Tool Discovery**: Clients automatically detect available tools without hardcoding
- **Enhanced Security**: Local server operation keeps sensitive credentials and data protected
- **Extensibility**: New tools can be added to your MCP server without modifying Gemini integration code

## Building an MCP Server

Creating an MCP server requires implementing three core components: tool definitions, execution logic, and result formatting.

### Core Architecture

An MCP server typically exposes tools through these fundamental operations:

1. **Tool Declaration**: Define available tools with their parameters and descriptions
2. **Tool Invocation**: Execute the requested operation
3. **Result Serialization**: Format and return results to the client

### Example: Weather Data Integration

Here's a practical implementation of an MCP server that integrates with weather data APIs:

```python
# weather_mcp_server.py
import json
import requests
from typing import Any, Dict, Optional

class WeatherMCPServer:
    def __init__(self, api_key: str):
        self.api_key = api_key
        self.base_url = "https://api.openweathermap.org/data/2.5"

        # Define available tools for this server
        self.tools = {
            "get_weather": {
                "description": "Retrieves current weather information for a specified city",
                "inputSchema": {
                    "type": "object",
                    "properties": {
                        "city": {
                            "type": "string",
                            "description": "City name"
                        },
                        "units": {
                            "type": "string",
                            "enum": ["metric", "imperial"],
                            "description": "Temperature units (metric or imperial)"
                        }
                    },
                    "required": ["city"]
                }
            }
        }

    def get_weather(self, city: str, units: str = "metric") -> Dict[str, Any]:
        """Fetch current weather from OpenWeatherMap API"""
        try:
            response = requests.get(
                f"{self.base_url}/weather",
                params={
                    "q": city,
                    "appid": self.api_key,
                    "units": units
                }
            )
            data = response.json()

            return {
                "success": True,
                "city": data["name"],
                "country": data["sys"]["country"],
                "temperature": data["main"]["temp"],
                "feels_like": data["main"]["feels_like"],
                "description": data["weather"][0]["description"],
                "humidity": data["main"]["humidity"],
                "wind_speed": data["wind"]["speed"],
                "pressure": data["main"]["pressure"]
            }
        except requests.exceptions.RequestException as e:
            return {
                "success": False,
                "error": f"API request failed: {str(e)}"
            }

    def execute_tool(self, tool_name: str, parameters: Dict[str, Any]) -> Dict[str, Any]:
        """Route tool requests to appropriate handler"""
        if tool_name == "get_weather":
            return self.get_weather(
                parameters.get("city"),
                parameters.get("units", "metric")
            )
        return {"success": False, "error": f"Unknown tool: {tool_name}"}

# Expected output example:
# {
#     "success": true,
#     "city": "Tokyo",
#     "country": "JP",
#     "temperature": 18.5,
#     "feels_like": 17.2,
#     "description": "scattered clouds",
#     "humidity": 65,
#     "wind_speed": 3.2,
#     "pressure": 1013
# }
```

### Step 2: Gemini API Integration

Use the [Gemini API](/articles/gemini-api/gemini-api-quickstart) to fetch tool definitions from your MCP server and enable tool use:

```python
import anthropic
import httpx
import json

client = anthropic.Anthropic(api_key="YOUR_API_KEY")
mcp_server_url = "http://localhost:8000"

def get_mcp_tools() -> list:
    """Retrieve tool definitions from MCP server"""
    response = httpx.get(f"{mcp_server_url}/tools")
    return response.json()

def execute_mcp_tool(tool_name: str, parameters: dict) -> str:
    """Execute a tool on the MCP server"""
    response = httpx.post(
        f"{mcp_server_url}/execute",
        json={"tool": tool_name, "parameters": parameters}
    )
    return json.dumps(response.json())

# Initialize with MCP tools
mcp_tools = get_mcp_tools()

# Conversation with tool use
messages = [
    {"role": "user", "content": "What's the weather like in Tokyo and Paris?"}
]

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    tools=mcp_tools,
    messages=messages
)

# Process tool calls in an agentic loop
while response.stop_reason == "tool_use":
    tool_use_block = next(
        (block for block in response.content if block.type == "tool_use"),
        None
    )

    if not tool_use_block:
        break

    # Execute the tool on MCP server
    tool_result = execute_mcp_tool(tool_use_block.name, tool_use_block.input)

    # Add to conversation history
    messages.append({"role": "assistant", "content": response.content})
    messages.append({
        "role": "user",
        "content": [
            {
                "type": "tool_result",
                "tool_use_id": tool_use_block.id,
                "content": tool_result
            }
        ]
    })

    # Continue conversation
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=1024,
        tools=mcp_tools,
        messages=messages
    )

# Extract final response
final_response = next(
    (block.text for block in response.content if hasattr(block, "text")),
    "No response generated"
)
print(final_response)
```

## Common MCP Server Patterns

### Database Connectivity

Expose database queries through an MCP server for controlled access:

```python
from sqlalchemy import create_engine, text

class DatabaseMCPServer:
    def __init__(self, connection_string: str):
        self.engine = create_engine(connection_string)

    def query_safe(self, query: str, parameters: dict = None) -> Dict[str, Any]:
        """Execute sanitized database queries"""
        try:
            with self.engine.connect() as conn:
                # Use parameterized queries to prevent SQL injection
                result = conn.execute(
                    text(query),
                    parameters or {}
                )
                return {
                    "success": True,
                    "rows": [dict(row) for row in result]
                }
        except Exception as e:
            return {"success": False, "error": str(e)}
```

### File System Operations

Safely provide file access with permission boundaries:

```python
from pathlib import Path

class FileSystemMCPServer:
    def __init__(self, base_path: str):
        self.base_path = Path(base_path).resolve()

    def is_allowed(self, file_path: str) -> bool:
        """Verify file is within allowed directory"""
        requested_path = (self.base_path / file_path).resolve()
        return str(requested_path).startswith(str(self.base_path))

    def read_file(self, file_path: str) -> Dict[str, Any]:
        """Read file content with safety checks"""
        if not self.is_allowed(file_path):
            return {"success": False, "error": "Access denied"}

        try:
            content = (self.base_path / file_path).read_text()
            return {"success": True, "content": content}
        except Exception as e:
            return {"success": False, "error": str(e)}
```

## Security Best Practices

### Authentication & Authorization

Protect your MCP server with proper authentication:

- **Token-Based Auth**: Use JWT or OAuth 2.0 for API access
- **Role-Based Access Control**: Implement permission levels for different operations
- **Request Validation**: Validate all inputs on the server side before execution

### Data Protection

- **Encryption in Transit**: Use HTTPS/TLS for all communications
- **Audit Logging**: Record all tool invocations for compliance and debugging
- **Rate Limiting**: Implement throttling to prevent abuse and resource exhaustion

## Frequently Asked Questions

### Q1: Do I need special Gemini API permissions to use MCP servers?

A: No, standard Gemini API access supports tool use. The [function calling](/articles/gemini-api/function-calling-guide) capability in Gemini handles MCP server integration seamlessly. You simply need to provide properly formatted tool definitions.

### Q2: Can I use multiple MCP servers simultaneously?

A: Absolutely. You can merge tool definitions from multiple servers into a single array and pass them to Gemini. Just ensure tool names don't conflict—use prefixes like "weather_" or "database_" to disambiguate.

### Q3: What's the best way to deploy MCP servers in production?

A: Consider containerizing your server with Docker and deploying to Kubernetes, Google Cloud Run, or AWS ECS. This ensures scalability and high availability. For sensitive operations, use private networks and implement comprehensive monitoring and alerting.

### Q4: How do I handle timeouts and failures from MCP tools?

A: Implement retry logic in your integration code with exponential backoff. For timeouts, set reasonable thresholds in your HTTP client. Always include error handling in the tool response and communicate failures clearly back to the user through Gemini's response.
