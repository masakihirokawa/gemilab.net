## What Is Gemini AppFunctions? — A New Way to Let AI Control Your App

In February 2026, Google introduced **AppFunctions**, a new framework that lets Gemini execute actions inside Android apps — not by simulating screen taps or reading the UI, but through a **structured API that apps declare explicitly**.

Traditional AI assistants relied on UI automation to interact with apps on behalf of users. AppFunctions changes the game by letting developers define **self-describing functions** that AI agents can discover and call directly. Think of it as the on-device equivalent of [MCP (Model Context Protocol)](/en/articles/gemini-dev/gemini-mcp-integration) for cloud services.

### Why AppFunctions Matters

Here's what sets AppFunctions apart from previous approaches:

- **Structured API access**: Instead of parsing screens, Gemini calls explicitly defined functions to interact with your app
- **On-device execution**: Functions run locally on the device without routing through cloud servers
- **Automatic discovery**: Gemini detects available app functions and calls them at the right time
- **Massive scale**: Expected to be available on over 200 million devices by the end of 2026

Major services like **Uber**, **DoorDash**, and **OpenTable** have already implemented AppFunctions integrations with Gemini. Users can say things like "book me a restaurant" or "get me a ride" and Gemini handles the cross-app orchestration seamlessly.

## Understanding the Architecture

### How AppFunctions Works

The AppFunctions architecture consists of three layers:

1. **Function Definition Layer**: Developers use the `AppFunctions` Jetpack library to define what capabilities to expose
2. **Discovery Layer**: Gemini searches for available functions across installed apps on the device
3. **Execution Layer**: After getting user permission, Gemini calls the function and processes the result

```kotlin
// Conceptual flow of how AppFunctions works
// 1. Your app registers functions
// 2. Gemini receives: "Find a nearby restaurant"
// 3. Gemini discovers restaurant search functions on the device
// 4. After user approval, it calls the function and returns results

// User: "Book me an Italian restaurant nearby"
// → Gemini: discovers OpenTable's searchRestaurants()
// → Gemini: asks user for permission to proceed
// → Gemini: calls searchRestaurants(cuisine="italian", location="nearby")
// → Gemini: shows results and offers to make a reservation
```

### How It Differs from Function Calling

Gemini API's [Function Calling](/en/articles/gemini-advanced/function-calling-guide) is a server-side mechanism that runs in the cloud. AppFunctions, on the other hand, is a **client-side mechanism that runs on the device**. Here's how they compare:

| Aspect | Function Calling | AppFunctions |
|--------|-----------------|--------------|
| Execution | Cloud (server-side) | On-device (local) |
| Target | Web APIs / backends | Android app features |
| Definition | JSON Schema | Jetpack library |
| Auth | API key-based | Android permissions |
| Latency | Network-dependent | Fast local execution |

## Implementing AppFunctions — A Step-by-Step Kotlin Guide

### Prerequisites

To implement AppFunctions, you'll need:

- Android Studio Narwhal (2025.3) or later
- Kotlin 2.0 or higher
- `minSdk` 26 or above
- Jetpack AppFunctions library (`androidx.appfunctions`)

### Step 1: Add Dependencies

Start by adding the AppFunctions library to your `build.gradle.kts`:

```kotlin
// build.gradle.kts (Module: app)
dependencies {
    implementation("androidx.appfunctions:appfunctions-runtime:1.0.0-alpha01")
    annotationProcessor("androidx.appfunctions:appfunctions-compiler:1.0.0-alpha01")
    // If using KSP instead:
    // ksp("androidx.appfunctions:appfunctions-compiler:1.0.0-alpha01")
}
```

### Step 2: Define Your AppFunctions

Use the `@AppFunction` annotation to define functions you want to expose to Gemini:

```kotlin
import androidx.appfunctions.AppFunction
import androidx.appfunctions.AppFunctionContext

class RestaurantFunctions {

    /**
     * Search for restaurants by cuisine and location
     * @param cuisine Type of cuisine (e.g., "italian", "japanese")
     * @param location Search area (e.g., "tokyo", "nearby")
     * @param partySize Number of guests (default: 2)
     * @return List of matching restaurants
     */
    @AppFunction(
        displayName = "Search Restaurants",
        description = "Search for restaurants by cuisine type and location"
    )
    suspend fun searchRestaurants(
        context: AppFunctionContext,
        cuisine: String,
        location: String,
        partySize: Int = 2
    ): List<Restaurant> {
        // Actual restaurant search logic
        val repository = RestaurantRepository.getInstance()
        return repository.search(
            cuisine = cuisine,
            location = location,
            partySize = partySize
        )
    }

    /**
     * Make a restaurant reservation
     * @param restaurantId Unique restaurant identifier
     * @param date Reservation date (ISO 8601 format)
     * @param time Reservation time (HH:mm format)
     * @param partySize Number of guests
     * @return Reservation confirmation details
     */
    @AppFunction(
        displayName = "Make Reservation",
        description = "Make a restaurant reservation"
    )
    suspend fun makeReservation(
        context: AppFunctionContext,
        restaurantId: String,
        date: String,
        time: String,
        partySize: Int
    ): ReservationConfirmation {
        val repository = RestaurantRepository.getInstance()
        return repository.reserve(
            restaurantId = restaurantId,
            date = date,
            time = time,
            partySize = partySize
        )
    }
}

// Data class definitions
data class Restaurant(
    val id: String,
    val name: String,
    val cuisine: String,
    val rating: Double,
    val address: String,
    val priceRange: String // "$", "$$", "$$$", "$$$$"
)

data class ReservationConfirmation(
    val confirmationId: String,
    val restaurantName: String,
    val date: String,
    val time: String,
    val partySize: Int,
    val status: String // "confirmed", "pending", "waitlist"
)
```

### Step 3: Register in the Manifest

Add the AppFunctions service to your `AndroidManifest.xml`:

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <application>
        <!-- Register AppFunctions service -->
        <service
            android:name="androidx.appfunctions.AppFunctionService"
            android:exported="true"
            android:permission="android.permission.BIND_APP_FUNCTION_SERVICE">
            <intent-filter>
                <action android:name="androidx.appfunctions.ACTION_APP_FUNCTION" />
            </intent-filter>
            <meta-data
                android:name="androidx.appfunctions.APP_FUNCTION_CLASS"
                android:value="com.example.myapp.RestaurantFunctions" />
        </service>
    </application>
</manifest>
```

### Step 4: Implement User Consent

AppFunctions requires **explicit user consent** before Gemini can call any function in your app.

```kotlin
import androidx.appfunctions.AppFunctionConsentManager

class ConsentHandler(private val activity: Activity) {

    private val consentManager = AppFunctionConsentManager.getInstance(activity)

    /**
     * Request user permission for function execution
     */
    suspend fun requestConsent(
        functionName: String,
        callerPackage: String
    ): Boolean {
        // Show system dialog asking for user approval
        val result = consentManager.requestConsent(
            functionName = functionName,
            callerPackage = callerPackage,
            // Whether to remember the user's choice for future calls
            rememberChoice = true
        )
        return result.isGranted
    }
}
```

## Practical Use Case — Integrating a Task Manager

Let's walk through a more practical example: adding AppFunctions to a task management app.

```kotlin
class TaskManagerFunctions {

    @AppFunction(
        displayName = "Create Task",
        description = "Create a new task with title, due date, and priority"
    )
    suspend fun createTask(
        context: AppFunctionContext,
        title: String,
        dueDate: String? = null,
        priority: String = "medium", // "low", "medium", "high"
        category: String? = null
    ): TaskResult {
        val taskDao = AppDatabase.getInstance().taskDao()
        val task = Task(
            title = title,
            dueDate = dueDate?.let { LocalDate.parse(it) },
            priority = Priority.valueOf(priority.uppercase()),
            category = category,
            createdAt = Instant.now()
        )
        val id = taskDao.insert(task)
        return TaskResult(
            success = true,
            taskId = id.toString(),
            message = "Task '$title' created successfully"
        )
    }

    @AppFunction(
        displayName = "List Tasks",
        description = "List tasks filtered by status, priority, or due date"
    )
    suspend fun listTasks(
        context: AppFunctionContext,
        status: String = "pending", // "pending", "completed", "all"
        priority: String? = null,
        dueBefore: String? = null
    ): List<TaskSummary> {
        val taskDao = AppDatabase.getInstance().taskDao()
        return taskDao.queryTasks(
            status = status,
            priority = priority,
            dueBefore = dueBefore
        ).map { task ->
            TaskSummary(
                id = task.id.toString(),
                title = task.title,
                dueDate = task.dueDate?.toString(),
                priority = task.priority.name.lowercase(),
                status = task.status
            )
        }
    }
}

// Natural language usage examples:
// "Add a high-priority task to submit my report by tomorrow"
// → createTask(title="Submit report", dueDate="2026-03-24", priority="high")
//
// "Show me my incomplete tasks for this week"
// → listTasks(status="pending", dueBefore="2026-03-29")
```

With this implementation, users can simply tell Gemini "add a task" or "check my schedule" and your app's features work seamlessly through voice commands.

## Best Practices for AppFunctions

### Security and Privacy

Follow these guidelines to keep your AppFunctions implementation secure:

1. **Principle of least privilege**: Only expose the functions users actually need — avoid exposing sensitive data
2. **Enforce user consent**: Always require explicit consent for financial transactions or personal data access
3. **Validate all inputs**: Treat inputs from AI agents the same way you'd treat untrusted user input
4. **Maintain audit logs**: Record which functions were called, when, and by which agent

```kotlin
// Example: input validation for a financial function
@AppFunction(
    displayName = "Transfer Funds",
    description = "Transfer money to another account"
)
suspend fun transferFunds(
    context: AppFunctionContext,
    recipientId: String,
    amount: Double,
    currency: String = "JPY"
): TransferResult {
    // Validate: check amount bounds
    require(amount > 0) { "Transfer amount must be greater than 0" }
    require(amount <= 1_000_000) { "Maximum single transfer is 1,000,000 JPY" }

    // Validate: check currency code
    require(currency in listOf("JPY", "USD", "EUR")) {
        "Supported currencies: JPY, USD, EUR"
    }

    // Require biometric authentication for sensitive operations
    val authResult = BiometricAuthManager.authenticate(context)
    if (!authResult.isSuccess) {
        return TransferResult(success = false, message = "Authentication failed")
    }

    // Execute the transfer
    return performTransfer(recipientId, amount, currency)
}
```

### Performance Optimization

Since AppFunctions run on-device, response speed matters:

- **Use coroutines**: Leverage `suspend` functions to avoid blocking the main thread
- **Cache frequently accessed data**: Keep commonly requested data in local storage
- **Keep responses lightweight**: Return only what's needed — let follow-up calls fetch details

## Frequently Asked Questions

### Q1: Can AppFunctions work on all Android devices?

Not yet. As of March 2026, AppFunctions is available primarily on the Samsung Galaxy S26 series and Google Pixel devices. It requires `minSdk` 26, and support will expand significantly with Android 17. Google projects availability on 200+ million devices by the end of 2026.

### Q2: Can I use AppFunctions alongside Function Calling?

Absolutely. The two serve complementary roles: Function Calling handles cloud-based API integrations, while AppFunctions manages on-device app interactions. For example, you might use Function Calling to fetch weather data from a remote API and AppFunctions to add an event to the user's local calendar.

### Q3: How is security handled for exposed functions?

AppFunctions integrates with Android's permission system. No function can execute without explicit user consent. Developers can add additional security layers including input validation, rate limiting, and biometric authentication for sensitive operations.

### Q4: Is it worth adopting AppFunctions as an indie developer?

Definitely. By integrating AppFunctions, your app becomes accessible through Gemini's natural language interface, letting users interact with your features by voice. With 200+ million potential devices, making your app "Gemini-ready" is a meaningful competitive advantage — especially for utility and productivity apps.
