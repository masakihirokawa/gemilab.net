# NVIDIA NeMo Framework × Gemini API — Enterprise LLM Fine-Tuning and Hybrid Inference Pipeline

Generic foundation models fall short for enterprise needs. Domain-specific knowledge, corporate protocols, and industry compliance require custom models tailored to your organization. NVIDIA NeMo Framework is the industry standard for end-to-end LLM training pipelines, scaling from single GPUs to cloud deployments with thousands of nodes.

This guide walks through production-ready patterns for GPU-accelerated fine-tuning with NeMo Framework and building a **hybrid inference architecture** that combines custom-trained models with Gemini API. The result: cost-efficient, high-accuracy enterprise LLM systems.

## The Three-Layer Architecture of NeMo Framework

NVIDIA NeMo unifies pretraining, post-training, and reinforcement learning (RLHF) under a single PyTorch API. NeMo 2.0 introduces native PyTorch custom loops for unprecedented flexibility.

```python
# NeMo 2.0: PyTorch Native Custom Loop for Full Control
import nemo.lightning as nl
from nemo.collections import llm

# Layer 1: Define Model Architecture
config = llm.GPTModel(
    num_layers=32,
    hidden_size=4096,
    num_attention_heads=32,
    ffn_hidden_size=16384,
    max_position_embeddings=8192,
    use_flash_attention=True,  # Optimized for NVIDIA H100
)

# Layer 2: Prepare Tokenizer (pre-trained or custom)
tokenizer = nl.TokenizerFactory.get(
    name="nemotron",
    vocab_size=256000,
)

# Layer 3: Distributed Training Configuration
trainer = nl.Trainer(
    devices=8,  # 8 GPUs on single machine
    strategy="fsdp",  # Fully Sharded Data Parallel
    max_epochs=3,
    precision="bf16",  # Recommended for A100/H100
)

# Layer 4: Data and Optimization
model = llm.GPTModel(config)
optim = nl.OptimizerFactory.get(
    name="megatron_adam",
    learning_rate=2e-4,
    weight_decay=0.1,
)

# Layer 5: Custom Training Loop (NeMo 2.0 strength)
trainer.fit(model, train_dataloaders=train_loader)
```

This architecture enables **domain adaptation at scale**, turning proprietary datasets into competitive advantages.

## Fine-Tuning Best Practices

### Step 1: Data Preparation and Preprocessing

```python
from nemo.collections.common.tokenizers.sentencepiece_tokenizer import SentencePieceTokenizer
from torch.utils.data import DataLoader, IterableDataset
import torch

class EnterpriseCorpusDataset(IterableDataset):
    """Load domain-specific documents: customer contracts, technical specs, internal wikis"""

    def __init__(self, data_path: str, tokenizer, max_seq_len: int = 2048):
        self.tokenizer = tokenizer
        self.max_seq_len = max_seq_len

        with open(data_path, 'r', encoding='utf-8') as f:
            self.texts = f.readlines()

    def __iter__(self):
        for text in self.texts:
            tokens = self.tokenizer.text_to_ids(text.strip())

            # Pad or truncate to fixed length
            if len(tokens) > self.max_seq_len:
                tokens = tokens[:self.max_seq_len]
            else:
                tokens = tokens + [self.tokenizer.pad_id] * (self.max_seq_len - len(tokens))

            yield {
                "input_ids": torch.tensor(tokens),
                "labels": torch.tensor(tokens),  # Self-supervised learning
            }

# Initialize data loader
train_dataset = EnterpriseCorpusDataset(
    data_path="./data/enterprise_docs.txt",
    tokenizer=tokenizer,
)

train_loader = DataLoader(
    train_dataset,
    batch_size=32,
    num_workers=4,
)
```

### Step 2: LoRA (Low-Rank Adaptation) for Efficient Fine-Tuning

LoRA reduces trainable parameters from 100% to **0.1%-1%** of the original model while maintaining comparable performance. Memory usage drops by 3-4x compared to full fine-tuning.

```python
from nemo.collections.nlp.parts.peft import LoRA

# Load pre-trained Nemotron-4 70B
base_model = llm.GPTModel.from_pretrained("nvidia/Nemotron-4-70B-base")

# LoRA Configuration: Adapt only query and value projections in attention
lora_config = {
    "adapter_name": "enterprise_lora",
    "lora_r": 16,  # Rank: higher = more expressive, more memory
    "lora_alpha": 32,  # Scaling factor
    "target_modules": ["q_proj", "v_proj"],  # Attention layers only
    "lora_dropout": 0.05,
}

model_with_lora = base_model.add_adapter("enterprise", lora_config)

# Freeze all base parameters, enable only LoRA weights
for param in model_with_lora.parameters():
    param.requires_grad = False

for param in model_with_lora.get_adapter_state_dict("enterprise").values():
    param.requires_grad = True

# Lightweight training loop
optimizer = torch.optim.AdamW(
    [p for p in model_with_lora.parameters() if p.requires_grad],
    lr=1e-4,
)

for epoch in range(3):
    for batch in train_loader:
        input_ids = batch["input_ids"].cuda()
        labels = batch["labels"].cuda()

        logits = model_with_lora(input_ids=input_ids)
        loss = torch.nn.functional.cross_entropy(
            logits.view(-1, tokenizer.vocab_size),
            labels.view(-1)
        )

        loss.backward()
        torch.nn.utils.clip_grad_norm_(model_with_lora.parameters(), 1.0)
        optimizer.step()
        optimizer.zero_grad()

        print(f"Loss: {loss.item():.4f}")
```

Memory savings with LoRA:
- Full fine-tuning (70B): ~300GB VRAM needed
- LoRA (70B, r=16): ~48GB VRAM (6x reduction)

## Hybrid Inference Architecture

Don't force a single model for all tasks. A **hybrid architecture routes queries intelligently** to either your fine-tuned NeMo model or Gemini API based on task characteristics.

```python
from google.generativeai import genai
import asyncio
import json

class HybridInferencePipeline:
    """
    Route queries to NeMo fine-tuned model (on-premise GPU) or Gemini API.
    Optimizes for cost, latency, and accuracy simultaneously.
    """

    def __init__(self, nemo_checkpoint_path: str, gemini_api_key: str):
        # Load fine-tuned NeMo model (private GPU infrastructure)
        self.nemo_model = self._load_nemo_checkpoint(nemo_checkpoint_path)

        # Initialize Gemini API client
        genai.configure(api_key=gemini_api_key)
        self.gemini_client = genai.GenerativeModel("gemini-2.0-flash")

    def _load_nemo_checkpoint(self, path: str):
        """Restore NeMo fine-tuned model from checkpoint"""
        from nemo.collections import llm
        model = llm.GPTModel.restore_from_checkpoint(path)
        model.eval()
        model.cuda()
        return model

    async def infer(
        self,
        query: str,
        context: str = None,
        force_engine: str = None,
    ) -> dict:
        """
        Intelligent routing to optimal inference engine

        Args:
            query: User question or task
            context: Enterprise knowledge base, documents, or previous conversation
            force_engine: "nemo" or "gemini" to override auto-routing

        Returns:
            {
                "response": str,
                "engine": "NeMo" | "Gemini",
                "confidence": 0.0-1.0,
                "latency_ms": float,
                "reasoning": str
            }
        """

        import time
        start_time = time.time()

        # Classify the query to determine optimal engine
        classification = await self._classify_query(query)

        # Select routing strategy
        if force_engine:
            use_nemo = (force_engine.lower() == "nemo")
        elif classification["requires_domain_knowledge"]:
            use_nemo = True  # Enterprise-specific → NeMo
        elif classification["estimated_complexity"] == "high":
            use_nemo = True  # Complex reasoning → NeMo (after fine-tuning)
        else:
            use_nemo = False  # General knowledge → Gemini (faster, cheaper)

        # Execute inference
        if use_nemo:
            response, confidence = await self._nemo_inference(query, context)
            engine = "NeMo"
        else:
            response, confidence = await self._gemini_inference(query, context)
            engine = "Gemini"

        latency_ms = (time.time() - start_time) * 1000

        return {
            "response": response,
            "engine": engine,
            "confidence": confidence,
            "latency_ms": latency_ms,
            "classification": classification,
        }

    async def _classify_query(self, query: str) -> dict:
        """Use Gemini API for fast query classification"""
        classification_prompt = f"""Analyze this query:
"{query}"

Output JSON:
{{
  "requires_domain_knowledge": boolean,
  "task_type": "summary|analysis|generation|retrieval|coding",
  "estimated_complexity": "low|medium|high",
  "estimated_tokens": integer
}}"""

        response = await self.gemini_client.generate_content_async(classification_prompt)
        try:
            return json.loads(response.text)
        except json.JSONDecodeError:
            return {
                "requires_domain_knowledge": False,
                "task_type": "general",
                "estimated_complexity": "medium",
                "estimated_tokens": 256,
            }

    async def _nemo_inference(self, query: str, context: str = None) -> tuple:
        """Generate response using fine-tuned NeMo model"""
        import torch

        # Construct prompt with context
        system = "You are a helpful enterprise AI assistant trained on company-specific knowledge."
        full_prompt = f"{system}\n\nContext:\n{context or 'N/A'}\n\nQuery:\n{query}\n\nResponse:"

        # Tokenize
        tokens = self.nemo_model.tokenizer.text_to_ids(full_prompt)
        input_ids = torch.tensor([tokens], device="cuda")

        # Generate with beam search
        with torch.no_grad():
            output_ids = self.nemo_model.generate(
                input_ids=input_ids,
                max_new_tokens=512,
                temperature=0.7,
                top_p=0.95,
                num_beams=4,
                length_penalty=0.6,
                repetition_penalty=1.2,
            )

        # Decode response
        response_text = self.nemo_model.tokenizer.ids_to_text(
            output_ids[0][input_ids.shape[1]:].tolist()  # Skip prompt tokens
        )

        # Confidence: NeMo fine-tuned models are generally high-confidence
        confidence = 0.85

        return response_text, confidence

    async def _gemini_inference(self, query: str, context: str = None) -> tuple:
        """Generate response using Gemini API"""
        full_prompt = f"""Context (if provided):
{context or 'N/A'}

Question:
{query}

Please provide a clear, accurate answer."""

        response = await self.gemini_client.generate_content_async(full_prompt)

        if not response.text:
            # Safety filter or content policy triggered
            return "[Unable to generate response - content policy applied]", 0.0

        # Gemini API provides good general-purpose responses
        confidence = 0.78

        return response.text, confidence


# Usage Example
async def main():
    pipeline = HybridInferencePipeline(
        nemo_checkpoint_path="./checkpoints/enterprise_ft_v2.nemo",
        gemini_api_key="YOUR_API_KEY",
    )

    # Enterprise-specific query
    result = await pipeline.infer(
        query="Summarize Q2 sales performance and highlight regional underperformance",
        context="""Q2 Sales Report:
        - North America: $5.2M (up 12%)
        - EMEA: $3.1M (down 3%)
        - APAC: $2.8M (up 8%)
        Target: $12M quarterly revenue""",
    )

    print(f"Response: {result['response']}")
    print(f"Engine: {result['engine']}")
    print(f"Confidence: {result['confidence']:.0%}")
    print(f"Latency: {result['latency_ms']:.0f}ms")

asyncio.run(main())
```

```python
# NIM Client: Standard OpenAI-compatible API
import httpx
import json

class NIMClient:
    def __init__(self, endpoint: str = "http://localhost:8000"):
        self.endpoint = endpoint
        self.client = httpx.AsyncClient(timeout=30.0)

    async def chat_completion(
        self,
        messages: list,
        max_tokens: int = 512,
        temperature: float = 0.7,
    ) -> str:
        """OpenAI-compatible /v1/chat/completions endpoint"""

        payload = {
            "model": "nemotron-4-340b-instruct",
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens,
        }

        response = await self.client.post(
            f"{self.endpoint}/v1/chat/completions",
            json=payload,
        )

        data = response.json()
        return data["choices"][0]["message"]["content"]

    async def close(self):
        await self.client.aclose()


# Simple wrapper for hybrid inference with fallback
async def unified_inference(
    prompt: str,
    use_nemo: bool = True,
) -> str:
    """Try NeMo first, fallback to Gemini on error"""

    if use_nemo:
        try:
            nim = NIMClient("http://nemo-nim:8000")
            response = await nim.chat_completion(
                messages=[{"role": "user", "content": prompt}]
            )
            await nim.close()
            return response
        except Exception as e:
            print(f"NeMo inference failed: {e}, falling back to Gemini...")

    # Fallback to Gemini API
    response = genai.GenerativeModel("gemini-2.0-flash").generate_content(prompt)
    return response.text
```

## RLHF: Aligning Models with Human Values

Fine-tuning alone optimizes prediction accuracy. RLHF (Reinforcement Learning from Human Feedback) aligns your model with **human preferences, safety, and compliance**.

```python
from nemo.collections.nlm.peft import RLHF
import numpy as np

class RLHFTrainer:
    """
    Three-stage RLHF pipeline:
    1. SFT (Supervised Fine-Tuning): Base model fine-tuned on curated data
    2. Reward Model: Learns human preferences from pairwise comparisons
    3. PPO: Optimizes model to maximize predicted rewards
    """

    def __init__(self, sft_model_path: str):
        self.sft_model = llm.GPTModel.restore_from_checkpoint(sft_model_path)
        self.reward_model = self._build_reward_model()

    def _build_reward_model(self):
        """Build a smaller model that predicts human preference scores"""
        model = llm.GPTModel(
            num_layers=12,
            hidden_size=2048,
            num_attention_heads=16,
        )
        return model

    def train_reward_model(self, preference_data: list, epochs: int = 5):
        """
        Train reward model on human preference annotations

        Args:
            preference_data: List of {
                "prompt": str,
                "response_a": str,
                "response_b": str,
                "human_preference": "a" | "b" | "tie"
            }
        """

        optimizer = torch.optim.Adam(self.reward_model.parameters(), lr=1e-5)

        for epoch in range(epochs):
            for sample in preference_data:
                prompt = sample["prompt"]
                resp_a = sample["response_a"]
                resp_b = sample["response_b"]
                preference = sample["human_preference"]

                # Tokenize
                tokens_a = self.reward_model.tokenizer.text_to_ids(f"{prompt}\n{resp_a}")
                tokens_b = self.reward_model.tokenizer.text_to_ids(f"{prompt}\n{resp_b}")

                # Forward pass
                reward_a = self.reward_model(torch.tensor([tokens_a]).cuda())
                reward_b = self.reward_model(torch.tensor([tokens_b]).cuda())

                # Preference loss: maximize difference favoring preferred response
                if preference == "a":
                    loss = torch.nn.functional.softplus(reward_b - reward_a)
                elif preference == "b":
                    loss = torch.nn.functional.softplus(reward_a - reward_b)
                else:
                    loss = torch.abs(reward_a - reward_b)

                loss.backward()
                optimizer.step()
                optimizer.zero_grad()

    def train_ppo(
        self,
        prompts: list,
        num_episodes: int = 1000,
        lr_policy: float = 1e-5,
    ):
        """
        PPO training: Optimize policy to maximize rewards from reward model
        while staying close to SFT baseline (KL divergence penalty)
        """

        policy = self.sft_model
        optimizer = torch.optim.AdamW(policy.parameters(), lr=lr_policy)

        for episode in range(num_episodes):
            # Sample from policy
            prompt = prompts[episode % len(prompts)]
            generated_ids = policy.generate(
                prompt,
                max_new_tokens=256,
                temperature=0.7,
            )

            # Get reward from reward model
            reward = self.reward_model(generated_ids)

            # PPO loss: maximize (reward - baseline) while minimizing KL divergence
            # Simplified version:
            loss = -reward.mean()  # Gradient ascent on reward
            loss.backward()
            optimizer.step()
            optimizer.zero_grad()
```

## Frequently Asked Questions

**Q: NeMo vs. Gemini API—which should I choose?**

NeMo for proprietary knowledge and specialized domains; Gemini for broad general knowledge and fast prototyping. The hybrid approach in this guide gets you both. NeMo requires GPU upfront investment; Gemini uses pay-as-you-go pricing.

**Q: What's the difference between LoRA rank 16 vs. 64?**

Rank 16: Sufficient for most tasks, memory-efficient, trains in days. Rank 64: More expressive, captures complex patterns, but 4x longer training and higher memory. Start at r=16; increase if model accuracy plateaus.

**Q: How much human annotation data do I need for RLHF?**

1,000-5,000 pairwise preference comparisons typically suffice to train a useful reward model. You can crowdsource this via platforms like Scale AI or Mechanical Turk within 2-3 weeks.

**Q: Can I fine-tune with multimodal data (text + images)?**

Yes. NeMo 2.0 supports vision-language models. Gemini API also accepts images natively. You can build end-to-end multimodal pipelines combining both.

**Q: What's the licensing situation for fine-tuned models?**

Nemotron models are commercially licensed (Apache 2.0 compatible). Your fine-tuned versions retain the same license. However, outputs generated by your model fall under your own copyright—be careful with training data attribution.

**Q: What latency should I target?**

Real-time chat: 0.5-1.5s per response. Batch analysis: 5-30s per document. With NIM on H100s, you can hit 0.3-0.8s. Network round-trips often dominate; co-locate NIM with application servers.
