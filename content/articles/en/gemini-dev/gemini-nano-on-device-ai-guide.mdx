## What Is Gemini Nano

Gemini Nano is the smallest and most efficient model in Google's Gemini family. Unlike Gemini Pro or Flash, which run on Google's cloud infrastructure, Nano is designed to execute AI inference directly on a user's device — no network connection required.

This architecture delivers three key advantages: zero network latency for near-instant responses, no API costs since inference runs locally, and strong privacy guarantees because user data never leaves the device.

## Why On-Device AI Matters

Cloud-based AI services have served developers well, but they come with inherent tradeoffs. Every request requires a network round-trip, adding latency that feels sluggish for real-time interactions. Each API call incurs costs that scale with usage. And sending user data to external servers raises legitimate privacy concerns, especially for sensitive information like health data or personal messages.

On-device AI eliminates these constraints at the source. The model runs entirely on the local processor, so it works in airplane mode, inside tunnels, and in areas with poor connectivity. There are no per-request charges. And sensitive data can be processed without ever leaving the user's phone or laptop.

## Where Gemini Nano Is Available

### Android Devices

Gemini Nano ships natively on Pixel 8 Pro and later Pixel devices, as well as the Samsung Galaxy S24 series. Through Android's AICore service, third-party apps can access Gemini Nano's capabilities without bundling the model themselves. Google handles model updates and optimization transparently through Play Services.

### Google Chrome

Starting with Chrome 126, Gemini Nano is embedded directly in the browser. Web developers can access it through a set of Built-in AI APIs — including the Prompt API, Summarization API, Writer API, and Rewriter API. This means you can add AI features to any web application without standing up server-side infrastructure.

## Using Chrome's Built-in AI APIs

Chrome's Gemini Nano integration exposes several JavaScript APIs that run entirely in the browser. Here are the primary ones available today.

### Prompt API

The Prompt API provides general-purpose text generation. You can create a session with a system prompt and then generate responses locally.

```javascript
// Create a session with a system prompt
const session = await ai.languageModel.create({
  systemPrompt: "You are a helpful assistant. Keep responses concise."
});

// Generate text
const response = await session.prompt("Explain TypeScript type inference");
console.log(response);

// Streaming response
const stream = await session.promptStreaming("List 3 React best practices");
for await (const chunk of stream) {
  console.log(chunk);
}
```

### Summarization API

The Summarization API condenses long text into brief summaries without any network calls. This is particularly useful for article previews, document digests, and content triage.

```javascript
const summarizer = await ai.summarizer.create({
  type: "tl;dr",
  length: "short",
  format: "plain-text"
});

const summary = await summarizer.summarize(longArticleText);
console.log(summary);
```

### Writer API and Rewriter API

The Writer API generates text based on a given context, while the Rewriter API transforms existing text into a different tone or style.

```javascript
// Writer: generate text from context
const writer = await ai.writer.create({
  tone: "formal",
  length: "medium"
});
const draft = await writer.write("Write a project status update email");

// Rewriter: transform existing text
const rewriter = await ai.rewriter.create({
  tone: "more-casual"
});
const rewritten = await rewriter.rewrite(formalEmail);
```

## Using Gemini Nano on Android

Android developers can integrate Gemini Nano into their apps through the Google AI Edge SDK and the AICore system service.

### Setup

```kotlin
// build.gradle.kts
dependencies {
    implementation("com.google.ai.edge.litert:litert:1.0.0")
    implementation("com.google.android.gms:play-services-aicore:16.0.0")
}
```

### Text Generation

```kotlin
import com.google.android.gms.aicore.GenerativeAIClient

val client = GenerativeAIClient.create(context)

val request = GenerateTextRequest.newBuilder()
    .setPrompt("Summarize this text: $inputText")
    .build()

val response = client.generateText(request)
println(response.text)
```

### Smart Reply

One of Gemini Nano's signature use cases is generating contextual reply suggestions for messaging apps. The model understands conversation flow and proposes relevant responses in real time.

```kotlin
val smartReplyClient = SmartReplyClient.create(context)

val conversation = listOf(
    Message("Hey, are you free tomorrow?", isLocalUser = false),
    Message("Yeah, I don't have any plans!", isLocalUser = true),
    Message("Want to grab lunch?", isLocalUser = false)
)

val suggestions = smartReplyClient.suggestReplies(conversation)
// → ["Sounds great!", "What time works?", "Where should we go?"]
```

## Limitations and When to Use Cloud Instead

Gemini Nano is not a drop-in replacement for larger models. Its compact size means it handles straightforward tasks well — input completion, short summaries, text classification, and quick suggestions — but struggles with complex reasoning, long-form generation, and nuanced analysis.

Tasks well suited for on-device inference include real-time autocomplete and smart replies, short text summarization, content classification and filtering of sensitive data, and basic AI features in offline environments.

Tasks that still call for cloud-based Gemini models include complex code generation and debugging, lengthy analytical reports, advanced multimodal processing of images, video, and audio, and question answering that requires access to current information.

The recommended pattern for production apps is a hybrid approach: attempt local inference with Gemini Nano first, then fall back to the Gemini API for tasks that exceed the on-device model's capabilities. This gives users the best of both worlds — instant responses for simple tasks and full model power when needed.
