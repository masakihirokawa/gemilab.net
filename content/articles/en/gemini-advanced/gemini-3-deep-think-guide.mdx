## What Is Deep Think?

### Linear vs. Parallel Reasoning

Traditional LLMs use linear inference: given an input, the model generates tokens sequentially along a single solution path. This works well for most tasks, but fails on problems requiring exploration of multiple hypotheses.

Gemini 3 Deep Think uses **Parallel Thinking** — it generates and evaluates multiple hypotheses simultaneously before converging on the best answer.

```
Standard Reasoning (Linear):
Input → Step 1 → Step 2 → Step 3 → Answer

Deep Think (Parallel):
Input → Hypothesis A → Evaluate A → ┐
      → Hypothesis B → Evaluate B → → Synthesize → Best Answer
      → Hypothesis C → Evaluate C → ┘
```

According to Google, Deep Think "generates many ideas at once and considers them simultaneously, even revising or combining different ideas over time, before arriving at the best answer."

### The thinking_level Parameter

Gemini 3 exposes reasoning depth through the `thinking_level` parameter:

| Level | Best For | Latency | Token Cost |
|-------|----------|---------|------------|
| `minimal` | Fact lookups, simple Q&A | Lowest | Minimal |
| `low` | High-throughput batch tasks | Low | Low |
| `medium` | Most business tasks | Moderate | Moderate |
| `high` | Complex reasoning, coding, science | High | High |

Deep Think is fully engaged at `thinking_level: "high"` (the default). Applying `high` to every request is cost-inefficient — matching the level to task complexity is essential in production.

### Basic Deep Think Call

```python
from google import genai
from google.genai import types

# Initialize the client
client = genai.Client()

# Request with Deep Think (thinking_level: high)
response = client.models.generate_content(
    model="gemini-3-pro-preview",  # or gemini-3.1-pro-preview
    contents="Solve this differential equation: dy/dx = x^2 * y, y(0) = 1",
    config=types.GenerateContentConfig(
        thinking_config=types.ThinkingConfig(
            thinking_level="high"  # Enable Deep Think
        )
    )
)

print(response.text)
```

**Expected output:**
```
Using separation of variables:

Separate: dy/y = x^2 dx
Integrate both sides: ln|y| = x^3/3 + C
Exponentiate: y = A * e^(x^3/3)

Apply initial condition y(0) = 1:
1 = A * e^0 → A = 1

Solution: y = e^(x^3/3)
```

### Visualizing the Thought Process

Enabling thought summaries lets you inspect Deep Think's internal reasoning — useful for validation, debugging, or educational applications.

```python
from google import genai
from google.genai import types

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="""
    In a Bayesian network where A→B and A→C:
    P(A=1)=0.3, P(B=1|A=1)=0.8, P(B=1|A=0)=0.1,
    P(C=1|A=1)=0.7, P(C=1|A=0)=0.2

    What is P(C=1 | B=1)?
    """,
    config=types.GenerateContentConfig(
        thinking_config=types.ThinkingConfig(
            thinking_level="high",
            include_thoughts=True  # Return thought summaries
        )
    )
)

# Separate thought process from final answer
for part in response.candidates[0].content.parts:
    if hasattr(part, 'thought') and part.thought:
        print("=== REASONING PROCESS ===")
        print(part.text)
        print()
    else:
        print("=== FINAL ANSWER ===")
        print(part.text)
```

### Multi-Turn Conversations with Thought Signatures

For multi-turn conversations, you must preserve Thought Signatures — encrypted representations of the model's internal reasoning state — to maintain coherent context.

```python
from google import genai
from google.genai import types

client = genai.Client()

def deep_think_conversation(messages: list[dict]) -> object:
    """
    Multi-turn Deep Think conversation.
    Automatically carries Thought Signatures across turns.
    """
    contents = [
        types.Content(
            role=msg["role"],
            parts=msg["parts"]  # Includes Thought Signatures
        )
        for msg in messages
    ]

    return client.models.generate_content(
        model="gemini-3-pro-preview",
        contents=contents,
        config=types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(thinking_level="high")
        )
    )

# Example: Step-by-step exploration of a mathematical proof
conversation = [
    {
        "role": "user",
        "parts": [types.Part(text="Explain Fermat's Last Theorem briefly")]
    }
]

# Turn 1
response1 = deep_think_conversation(conversation)

# Add response parts including Thought Signatures
conversation.append({
    "role": "model",
    "parts": response1.candidates[0].content.parts  # ← Must include full parts
})

# Turn 2: Follow-up question
conversation.append({
    "role": "user",
    "parts": [types.Part(text="Walk me through Wiles' key proof strategy")]
})

response2 = deep_think_conversation(conversation)
print(response2.text)
```

## JavaScript (Node.js) Implementation

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GOOGLE_API_KEY });

async function deepThinkAnalysis(problem) {
  const response = await ai.models.generateContent({
    model: "gemini-3-pro-preview",
    contents: problem,
    config: {
      thinkingConfig: {
        thinkingLevel: "high",     // Deep Think mode
        includeThoughts: true,      // Retrieve thought process
      },
    },
  });

  // Separate thought process from final answer
  let thoughts = "";
  let answer = "";

  for (const part of response.candidates[0].content.parts) {
    if (part.thought) {
      thoughts += part.text;
    } else {
      answer += part.text;
    }
  }

  return { thoughts, answer };
}

// Example: Complex algorithm design
const problem = `
  Design a data structure and algorithm to optimize range query performance
  for a database with one billion records.
  Include time complexity and space complexity analysis.
`;

const result = await deepThinkAnalysis(problem);
console.log("Reasoning process:", result.thoughts.substring(0, 400) + "...");
console.log("\nFinal answer:", result.answer);
```

## Common Errors and How to Fix Them

### Error 1: RESOURCE_EXHAUSTED (Thinking Token Quota)

```
google.api_core.exceptions.ResourceExhausted: 429 Quota exceeded
```

Deep Think at `thinking_level: "high"` can generate large volumes of thinking tokens, which count toward your quota.

**Fix:**

```python
# Lower thinking level to reduce token consumption
config = types.GenerateContentConfig(
    thinking_config=types.ThinkingConfig(
        thinking_level="medium"  # Reduce from "high"
    )
)
```

### Error 2: Unstable Outputs with Custom Temperature

Setting temperature below the default of 1.0 can cause unexpected behavior with Deep Think in Gemini 3 models.

**Fix:**

```python
# ✅ Correct: Leave temperature at default
config = types.GenerateContentConfig(
    thinking_config=types.ThinkingConfig(thinking_level="high")
    # Do NOT set temperature explicitly
)

# ❌ Avoid: Low temperature is not recommended with Deep Think
config = types.GenerateContentConfig(
    temperature=0.1,  # Can cause instability with Deep Think
    thinking_config=types.ThinkingConfig(thinking_level="high")
)
```

### Error 3: Lost Context Due to Missing Thought Signatures

In multi-turn conversations, extracting only the text response discards Thought Signatures, breaking reasoning continuity.

```python
# ✅ Correct: Preserve full parts including Thought Signatures
conversation.append({
    "role": "model",
    "parts": response.candidates[0].content.parts
})

# ❌ Wrong: Text-only extraction drops Thought Signatures
conversation.append({
    "role": "model",
    "parts": [types.Part(text=response.text)]  # Signatures lost!
})
```

## When to Use Deep Think

Not every task benefits from Deep Think. Here's a practical guide:

| Task Type | Recommended Level | Rationale |
|-----------|-------------------|-----------|
| FAQ and factual lookup | `minimal` | No reasoning needed |
| Code completion | `low` | Pattern-based, fast |
| Bug fixing and code review | `medium` | Moderate analysis sufficient |
| Complex algorithm design | `high` | Deep reasoning required |
| Mathematical proofs | `high` | Deep Think's core strength |
| Scientific hypothesis evaluation | `high` | Deep Think's core strength |

### Adaptive thinking_level Selection

```python
def adaptive_thinking(query: str) -> str:
    """
    Automatically select thinking_level based on query complexity.
    """
    client = genai.Client()

    # Simple complexity heuristic
    complex_keywords = ["prove", "optimize", "design", "analyze", "algorithm", "derive"]
    medium_keywords = ["implement", "code", "fix", "review", "debug"]

    if any(kw in query.lower() for kw in complex_keywords):
        level = "high"
    elif any(kw in query.lower() for kw in medium_keywords):
        level = "medium"
    else:
        level = "low"

    response = client.models.generate_content(
        model="gemini-3-pro-preview",
        contents=query,
        config=types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(thinking_level=level)
        )
    )

    return response.text
```

## FAQ

### Q1: Should I use Deep Think or Gemini 3.1 Pro Preview for complex reasoning tasks?

**A:** It depends on the task. Gemini 3.1 Pro Preview scores 77.1% on ARC-AGI-2 and excels at broad reasoning and general knowledge tasks. Deep Think scores 93.8% on GPQA Diamond and 45.1% on ARC-AGI-2, with particular strength in mathematical and scientific problem-solving. Use Deep Think when your task involves formal reasoning, mathematical proofs, or graduate-level science. Use 3.1 Pro Preview for general-purpose complex tasks.

### Q2: How much does Deep Think cost compared to regular Gemini 3 API calls?

**A:** Gemini 3 Pro models are priced at $2.00 per million input tokens and $12.00 per million output tokens. With `thinking_level: "high"`, thinking tokens are generated in addition to response tokens, and the total cost per request can be 2–5× higher than a standard call. Use `thinking_level: "medium"` for tasks that don't require maximum reasoning depth, and `minimal` for simple lookups to keep costs manageable.

### Q3: When will Deep Think be generally available via the Gemini API?

**A:** As of March 2026, Deep Think is available to Google AI Ultra subscribers through the Gemini app and via an early access program for researchers and enterprises. General API availability is expected in 2026, but no official release date has been announced. Monitor the [Gemini API changelog](https://ai.google.dev/gemini-api/docs/changelog) for updates.
