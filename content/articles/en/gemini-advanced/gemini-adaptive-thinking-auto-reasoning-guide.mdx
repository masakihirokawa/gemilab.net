## What Is Gemini Adaptive Thinking?

If you've ever noticed Gemini taking too long on a simple question — or giving a shallow answer to something genuinely complex — you've encountered the exact problem **Adaptive Thinking** was designed to solve.

Adaptive Thinking lets Gemini automatically evaluate the complexity of each incoming query and dynamically select the optimal reasoning depth. A straightforward question like "What's the weather in Tokyo?" gets a near-instant response with minimal processing, while a nuanced request like "Compare error-correction algorithms in quantum computing" triggers a deep chain-of-thought similar to Deep Think.

Previously, developers had to manually set the `thinking_level` parameter to `low`, `medium`, or `high`. Adaptive Thinking hands that decision over to the model itself, **automatically balancing latency, cost, and answer quality** on a per-request basis.

## How Adaptive Thinking Differs from Other Reasoning Modes

Gemini offers several reasoning control options. Understanding the differences helps you pick the right tool for each situation.

| Feature | Who Decides Depth | Best For |
|---|---|---|
| **Adaptive Thinking** | The model, automatically | General-purpose chatbots, mixed-complexity workloads |
| [thinking_level parameter](/en/articles/gemini-api/thinking-level-guide) | Developer, manually | Latency-sensitive APIs, strict cost control |
| [Deep Think](/en/articles/gemini-advanced/gemini-3-deep-think-guide) | Always maximum depth | Hard problems in math, science, and code generation |

Adaptive Thinking shines when you can't predict query complexity in advance — which is exactly the case for chat applications, customer support bots, and general-purpose assistants.

## Using Adaptive Thinking with the Gemini API

To enable Adaptive Thinking, set `thinking_level` to `"adaptive"` in your `thinking_config`. Here's a basic example using the Python SDK.

```python
from google import genai
from google.genai.types import ThinkingConfig

# Initialize client
client = genai.Client(api_key="YOUR_API_KEY")

# Enable Adaptive Thinking
response = client.models.generate_content(
    model="gemini-3-pro",
    contents="Give me an intuitive explanation of Fourier transforms and how they're used in audio processing",
    config={
        "thinking_config": ThinkingConfig(
            thinking_level="adaptive"  # Model auto-selects reasoning depth
        )
    }
)

# Inspect the response
for part in response.candidates[0].content.parts:
    if part.thought:
        print(f"[Thinking] {part.text[:200]}...")
    else:
        print(f"[Answer] {part.text[:200]}...")

# Check token usage (reflects actual reasoning depth)
print(f"Input tokens: {response.usage_metadata.prompt_token_count}")
print(f"Output tokens: {response.usage_metadata.candidates_token_count}")
print(f"Thinking tokens: {response.usage_metadata.thoughts_token_count}")
```

**Expected output:**

```
[Thinking] The user is asking for an intuitive explanation of Fourier transforms...
[Answer] Think of a Fourier transform as a prism for sound...
Input tokens: 25
Output tokens: 1250
Thinking tokens: 480
```

Notice how `thoughts_token_count` varies with query complexity. Simple queries produce values near zero, while complex ones generate hundreds or thousands of thinking tokens automatically.

## JavaScript (Node.js) Implementation

The same feature works seamlessly in TypeScript and JavaScript projects.

```javascript
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: "YOUR_API_KEY" });

async function adaptiveQuery(prompt) {
  const response = await ai.models.generateContent({
    model: "gemini-3-pro",
    contents: prompt,
    config: {
      thinkingConfig: {
        thinkingLevel: "adaptive", // Auto-adjust reasoning depth
      },
    },
  });

  const parts = response.candidates[0].content.parts;
  const thought = parts.find((p) => p.thought)?.text || "(no thinking)";
  const answer = parts.find((p) => !p.thought)?.text || "";

  return {
    thought: thought.slice(0, 200),
    answer,
    tokens: {
      input: response.usageMetadata.promptTokenCount,
      output: response.usageMetadata.candidatesTokenCount,
      thinking: response.usageMetadata.thoughtsTokenCount,
    },
  };
}

// Simple query
const simple = await adaptiveQuery("What is the capital of Japan?");
console.log("Simple query:", simple.tokens);
// => { input: 10, output: 12, thinking: 0 }

// Complex query
const complex = await adaptiveQuery(
  "Analyze the design trade-offs between the Saga pattern and Event Sourcing in microservices architecture"
);
console.log("Complex query:", complex.tokens);
// => { input: 22, output: 2100, thinking: 850 }
```

## Practical Use Cases

### 1. Customer Support Bots

Customer inquiries range from "What are your business hours?" to "I need you to investigate a duplicate charge on my invoice." With Adaptive Thinking, simple questions get instant answers while complex issues receive thorough analysis — all without any manual configuration.

```python
from google import genai
from google.genai.types import ThinkingConfig

client = genai.Client(api_key="YOUR_API_KEY")

# System prompt for the support bot
system_prompt = """You are a customer support agent for an e-commerce platform.
- Answer simple questions concisely
- For complex inquiries, provide detailed investigation and step-by-step solutions
- Ask clarifying questions when information is missing"""

def support_chat(user_message):
    response = client.models.generate_content(
        model="gemini-3-pro",
        contents=user_message,
        config={
            "system_instruction": system_prompt,
            "thinking_config": ThinkingConfig(thinking_level="adaptive"),
        }
    )
    return response.text
```

### 2. Code Review Assistant

The depth of review automatically scales with code complexity. A typo fix gets a quick review, while an architectural change triggers deep analysis of design patterns, edge cases, and performance implications.

### 3. Education Platforms

Student questions naturally vary in difficulty. Adaptive Thinking delivers concise answers for foundational questions and detailed, step-by-step explanations for advanced topics — matching the pedagogical depth to the student's actual need.

## Cost Optimization with Adaptive Thinking

The headline benefit of Adaptive Thinking is **eliminating unnecessary reasoning overhead**. Compared to setting `thinking_level: "high"` for every request, real-world workloads see significant cost reductions.

| Scenario | thinking_level: high | Adaptive Thinking | Savings |
|---|---|---|---|
| FAQ responses (80% simple queries) | Deep reasoning on all | Reasoning skipped for 80% | ~60–70% |
| Code review (mixed complexity) | Deep reasoning on all | Varies by complexity | ~30–40% |
| Research assistance (mostly complex) | Deep reasoning on all | Nearly equivalent | ~5–10% |

The takeaway: **the higher the proportion of simple queries in your workload, the greater the cost savings** from Adaptive Thinking.

## When to Use Adaptive Thinking vs. Manual thinking_level

Here's a practical decision framework for choosing between the two approaches.

**Choose Adaptive Thinking when:**

- Query complexity is unpredictable (chatbots, general assistants)
- You want automatic cost and latency optimization
- You don't want to spend time tuning reasoning depth per endpoint

**Choose manual thinking_level when:**

- You have strict latency ceilings (real-time APIs)
- Deep reasoning is always required (math/science specialist tools)
- Cost predictability matters more than cost efficiency (fixed-budget projects)

You can also combine both approaches. For example, use Adaptive Thinking as the default and override with `thinking_level: "high"` on specific endpoints like a math solver — giving you the best of both worlds.

## FAQ

### Q1: Which models support Adaptive Thinking?

Adaptive Thinking is available on Gemini 3 Pro and Gemini 3 Flash. Gemini 2.5 models only support the standard `thinking_level` values (`low`, `medium`, `high`) — passing `"adaptive"` will return an error. Check the [Gemini API documentation](https://ai.google.dev/) for the latest list of supported models.

### Q2: Can I still get Thought Signatures with Adaptive Thinking?

Yes. When Adaptive Thinking triggers deep reasoning, [Thought Signatures](/en/articles/gemini-advanced/gemini-thought-signatures-guide) are included in the response just like with manual thinking levels. However, when reasoning is skipped (simple queries), no Thought Signatures are generated. A `thoughts_token_count` of 0 means reasoning was skipped.

### Q3: Does Adaptive Thinking work with streaming?

Absolutely. In streaming mode, thinking parts (`thought: true`) are streamed first, followed by the answer. When reasoning is skipped, the answer streams immediately with no thinking parts — making the perceived response time even faster.

### Q4: How can I verify the automatic reasoning depth selection?

Monitor `usage_metadata.thoughts_token_count` in each response. Logging this value over time gives you visibility into how the model is distributing reasoning effort across your workload, which is invaluable for cost analysis and performance tuning.

## Wrapping Up

Gemini Adaptive Thinking is a deceptively simple feature — set `thinking_level` to `"adaptive"` and you're done — but the impact on cost, latency, and answer quality is substantial. It's especially powerful for applications that handle diverse query types, where predicting the right reasoning depth in advance would be impractical.

Pair it with the manual [thinking_level parameter](/en/articles/gemini-api/thinking-level-guide) for specialized endpoints and [Deep Think](/en/articles/gemini-advanced/gemini-3-deep-think-guide) for maximum-depth reasoning to get the full spectrum of Gemini's reasoning capabilities working for you.
