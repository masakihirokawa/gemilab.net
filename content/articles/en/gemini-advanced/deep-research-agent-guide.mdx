## What Is the Gemini Deep Research Agent?

In March 2026, Google launched the **Deep Research Agent** as a preview feature in the Gemini API. It is an AI agent capable of autonomously planning, executing, and synthesizing multi-step research tasks.

Unlike traditional RAG or search grounding, the Deep Research Agent can:
- Automatically create multi-step research plans
- Gather information across both the web and provided documents
- Generate detailed reports with full citations
- Run asynchronously in the background for long-duration tasks

This gives you an end-to-end autonomous research capability in a single API call.

## Requirements

The Deep Research Agent is **only accessible via the Interactions API** (Beta).

- Underlying model: **gemini-3.1-pro**
- API: Interactions API (Beta)
- Execution: Asynchronous background execution

## Basic Usage

### 1. Simple Research Request

```python
import google.genai as genai
import time

client = genai.Client(api_key="YOUR_API_KEY")

# Launch the Deep Research Agent via Interactions API
interaction = client.interactions.create(
    agent="deep-research",
    contents="Create a performance comparison report of major generative AI models in 2026. Include Gemini, GPT, and Claude with benchmark results and key differentiators.",
    config={
        "background": True  # Required for async execution
    }
)

print(f"Interaction ID: {interaction.id}")
print(f"Status: {interaction.status}")

# Poll for completion
while interaction.status not in ["completed", "failed"]:
    time.sleep(10)
    interaction = client.interactions.get(interaction.id)
    print(f"Status: {interaction.status}")

# Retrieve results
if interaction.status == "completed":
    print(interaction.result.text)
```

### 2. Research with Document Uploads

You can upload local PDF or CSV files and combine them with live web data.

```python
import pathlib

# Upload a document
uploaded_file = client.files.upload(
    path=pathlib.Path("market_report_2025.pdf"),
    config={"mime_type": "application/pdf"}
)

# Run research combining the file and web search
interaction = client.interactions.create(
    agent="deep-research",
    contents=[
        uploaded_file,
        "Using the attached report and the latest web information, create a 2026 AI market trend analysis."
    ],
    config={
        "background": True,
        "tools": ["file_search", "google_search"]
    }
)
```

## Production-Grade Async Pattern

For long-running research tasks, an async polling pattern is recommended over blocking calls.

```python
import asyncio
import google.genai as genai

async def run_deep_research(query: str) -> str:
    client = genai.AsyncClient(api_key="YOUR_API_KEY")

    # Start research
    interaction = await client.interactions.create(
        agent="deep-research",
        contents=query,
        config={"background": True}
    )

    # Wait for completion (max 30 minutes)
    max_wait = 1800
    elapsed = 0
    poll_interval = 15

    while elapsed < max_wait:
        await asyncio.sleep(poll_interval)
        elapsed += poll_interval

        interaction = await client.interactions.get(interaction.id)

        if interaction.status == "completed":
            return interaction.result.text
        elif interaction.status == "failed":
            raise RuntimeError(f"Research failed: {interaction.error}")

    raise TimeoutError("Research task timed out")

# Usage
result = asyncio.run(run_deep_research(
    "Create a usage guide comparing Gemini 3 series models by use case."
))
print(result)
```

## Customizing Output

### JSON Schema Output

Specify a `response_schema` to receive structured results:

```python
from pydantic import BaseModel
from typing import List

class Source(BaseModel):
    title: str
    url: str
    relevance: float

class ResearchReport(BaseModel):
    summary: str
    key_findings: List[str]
    sources: List[Source]
    confidence_score: float

interaction = client.interactions.create(
    agent="deep-research",
    contents="Research the current state of quantum computing commercialization in 2026.",
    config={
        "background": True,
        "response_schema": ResearchReport
    }
)
```

### Control Output Structure via Prompt

You can also define the output format directly in your prompt without a schema:

```python
interaction = client.interactions.create(
    agent="deep-research",
    contents="""
    Research the latest Gemini API pricing and produce a report in the following format:

    ## Executive Summary
    (3 sentences max)

    ## Pricing Table by Model
    | Model | Input | Output | Notes |

    ## Cost Optimization Tips
    (5 bullet points)

    ## References
    (URLs with retrieval dates)
    """,
    config={"background": True}
)
```

## Pricing

Deep Research Agent costs depend on the depth of research the agent autonomously performs.

| Component | Billing Basis |
|-----------|--------------|
| Input tokens | Gemini 3.1 Pro rate ($1.25/M tokens) |
| Output tokens | Gemini 3.1 Pro rate ($10.00/M tokens) |
| Web searches | Per search call |
| File searches | Based on file size and processing |

Since the agent runs multiple searches across multiple steps, **clearly scoping your research request in the prompt** is the most effective way to control costs.

## Deep Research Agent vs. generate_content

| Feature | generate_content | Deep Research Agent |
|---------|----------------|---------------------|
| Execution | Synchronous | Async background |
| Autonomous planning | No | Yes (multi-step) |
| Web search | Manual configuration | Auto-determined |
| Document analysis | Manual | Automatic cross-referencing |
| Execution time | Seconds | Minutes to tens of minutes |
| Cost | Low–Medium | Medium–High (complexity-dependent) |

## Important Caveats

- The Deep Research Agent is currently in **preview** — use with caution in production
- Always fact-check research results; accuracy is not guaranteed
- Background execution requires polling or webhooks to retrieve results
