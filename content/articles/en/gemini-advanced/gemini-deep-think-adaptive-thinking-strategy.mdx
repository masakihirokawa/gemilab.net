## Introduction: Why Reasoning is Expensive

Using Gemini's reasoning modes (Deep Think, Adaptive Thinking) delivers significantly higher-quality answers than simple text generation. The trade-off: **token costs can jump 2–5x higher**.

The real issue is that many developers treat reasoning modes as "always use this." In reality, **strategic task-complexity matching lets you maintain quality while cutting costs by up to 50%**.

This guide walks you through the internal mechanics of both modes, helping you make data-driven decisions about when to invest in reasoning and when to skip it.

## Deep Think vs Adaptive Thinking: Internal Operations

### Deep Think: Exploring Multiple Reasoning Paths

**Deep Think** explores multiple reasoning paths simultaneously, returning the answer with the highest confidence score. Internally, this process works like:

1. **Generate drafts**: Create 5–20 initial hypotheses
2. **Cross-validate**: Identify contradictions between drafts
3. **Score**: Rate each reasoning path by confidence
4. **Select and output**: Generate final answer from highest-confidence path

During this exploration, **10–30x more "thinking tokens" are consumed** than the visible output. Users see only the final answer, but internally, massive computational effort is happening.

**Deep Think excels at:**
- Mathematical proofs requiring rigor
- Legal/medical judgments with high stakes
- Complex debugging of algorithms
- Multi-hypothesis validation problems
- Scientific reasoning with competing theories

### Adaptive Thinking: Smart Inference Budget Allocation

**Adaptive Thinking** automatically measures input complexity and allocates reasoning effort accordingly. Simple questions get light reasoning; hard problems trigger deeper exploration.

Internal algorithm:
1. **Complexity detection**: Analyze input entropy, keyword density, logical dependencies
2. **Depth decision**: Dynamically choose reasoning steps (5–50 total)
3. **Stepped reasoning**: Apply increasing cognitive effort as needed

**Token consumption characteristics:**
- Simple Q&A: 4–6x additional tokens
- Medium complexity: 8–15x
- Complex problems: 15–25x (approaching Deep Think)

**Adaptive Thinking is ideal for:**
- Mixed-difficulty workloads
- Real-time systems where latency matters
- Cost-conscious applications
- Exploratory work and prototyping

## Task Complexity Classification Guide

### Decision Flowchart

```
Receive input
  ↓
Does this task require reasoning?
  ├─ Yes → Measure complexity
  │        ↓
  │        What's the complexity level?
  │        ├─ High (proof, medical, legal) → Deep Think
  │        ├─ Medium (analysis, synthesis) → Adaptive Thinking
  │        └─ Low (factual, routine) → No reasoning
  │
  └─ No → Standard generation (no thinking tokens)
```

### Automated Complexity Detection

```javascript
// Utility function to auto-classify task complexity
function detectComplexity(prompt) {
  const indicators = {
    high: [
      "prove",
      "explain why",
      "legal",
      "medical",
      "diagnose",
      "debug",
      "bug",
      "fix",
      "optimize",
      "complex",
      "algorithm",
    ],
    medium: [
      "analyze",
      "compare",
      "summarize",
      "explain",
      "example",
      "translate",
      "code",
    ],
    low: [
      "is",
      "does",
      "can",
      "what",
      "where",
      "when",
    ],
  };

  let score = 0;
  for (const keyword of indicators.high) {
    if (prompt.toLowerCase().includes(keyword)) score += 3;
  }
  for (const keyword of indicators.medium) {
    if (prompt.toLowerCase().includes(keyword)) score += 1;
  }
  for (const keyword of indicators.low) {
    if (prompt.toLowerCase().includes(keyword)) score -= 1;
  }

  // Factor in length
  if (prompt.length > 2000) score += 2;

  if (score >= 5) return "high";
  if (score >= 2) return "medium";
  return "low";
}

// Dynamic routing based on detected complexity
async function smartInference(prompt) {
  const complexity = detectComplexity(prompt);
  const client = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

  let model;
  switch (complexity) {
    case "high":
      model = client.getGenerativeModel({
        model: "gemini-2.0-pro-deepthink",
      });
      console.log("📊 Deep Think (high complexity)");
      break;

    case "medium":
      model = client.getGenerativeModel({
        model: "gemini-2.0-pro-adaptive-thinking",
      });
      console.log("⚙️  Adaptive Thinking (medium complexity)");
      break;

    default:
      model = client.getGenerativeModel({
        model: "gemini-2.0-pro",
      });
      console.log("⚡ Standard inference (low complexity)");
  }

  const response = await model.generateContent({
    contents: [{ role: "user", parts: [{ text: prompt }] }],
    generationConfig: {
      temperature: complexity === "high" ? 1.0 : 0.7,
      maxOutputTokens: complexity === "high" ? 16000 : 2048,
    },
  });

  return response;
}

// Examples
const easyTask = "What is the capital of France?";
const mediumTask =
  "Explain how transformers work in machine learning models.";
const hardTask =
  "Prove that the square root of 2 is irrational using proof by contradiction.";

await smartInference(easyTask);      // Standard
await smartInference(mediumTask);    // Adaptive
await smartInference(hardTask);      // Deep Think
```

### 2. Context Caching: 90% Cost Reduction for Repeated Analysis

When analyzing the same long document multiple times, **Context Caching** can reduce costs by up to 90%.

```javascript
async function callWithCaching(context, question, useCache = true) {
  const apiKey = process.env.GEMINI_API_KEY;

  const payload = {
    contents: [
      {
        role: "user",
        parts: [
          {
            text: `Reference this document:\n\n${
              useCache ? "[Cached]" : context
            }\n\nQuestion: ${question}`,
          },
        ],
      },
    ],
    generationConfig: {
      temperature: 0.7,
      maxOutputTokens: 1024,
    },
  };

  const headers = {
    "Content-Type": "application/json",
    "x-goog-api-key": apiKey,
  };

  if (useCache) {
    headers["google-ai-cache-control"] = "cache-control=max-age=3600";
  }

  const response = await fetch(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-pro-adaptive-thinking:generateContent",
    {
      method: "POST",
      headers,
      body: JSON.stringify(payload),
    }
  );

  const data = await response.json();
  const metadata = data.usageMetadata;

  console.log("Input tokens:", metadata.promptTokenCount);
  console.log("Cache creation:", metadata.cacheCreationInputTokens || 0);
  console.log("Cache read:", metadata.cacheReadInputTokens || 0);
  console.log("Output:", metadata.candidatesTokenCount);

  return data;
}

// Analyze 500KB document with 10 questions
const largeDoc = "...( Comprehensive API documentation, 5000+ lines )...";

// Request 1: Cache creation
console.log("--- Request 1 (cache creation) ---");
await callWithCaching(largeDoc, "How do I authenticate?", true);

// Requests 2–10: Cache hits (90% savings)
for (let i = 2; i <= 10; i++) {
  console.log(`\n--- Request ${i} (cache hit) ---`);
  await callWithCaching(largeDoc, `Question ${i}: How to use feature X?`, true);
}
```

### 3. Batching vs Streaming: Cost Trade-offs

```javascript
// Batch processing: 50% discount, asynchronous
async function batchInference(questions) {
  const requests = questions.map((q, i) => ({
    custom_id: `q-${i}`,
    params: {
      model: "gemini-2.0-pro-adaptive-thinking",
      contents: [{ role: "user", parts: [{ text: q }] }],
    },
  }));

  const batchFile = requests.map(r => JSON.stringify(r)).join("\n");

  const formData = new FormData();
  formData.append("file", new Blob([batchFile], { type: "text/plain" }));

  const response = await fetch(
    "https://generativelanguage.googleapis.com/v1beta/batches",
    {
      method: "POST",
      headers: { "x-goog-api-key": process.env.GEMINI_API_KEY },
      body: formData,
    }
  );

  console.log("Batch submitted. 50% cost reduction vs normal.");
  return response.json();
}

// Streaming: Real-time, standard pricing
async function streamInference(question) {
  return fetch(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-pro-adaptive-thinking:streamGenerateContent",
    {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "x-goog-api-key": process.env.GEMINI_API_KEY,
      },
      body: JSON.stringify({
        contents: [{ role: "user", parts: [{ text: question }] }],
      }),
    }
  );
}

// Example: Process 100 questions
// Batch: 5min processing, ¥500 cost
// Stream: 5sec latency, ¥1,000 cost
```

## API Configuration Best Practices

### Optimal Temperature Settings for Reasoning

```javascript
// ⚠️ Critical: Always set temperature=1.0 for reasoning models
async function optimalReasoningConfig(prompt, complexity) {
  const client = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

  const configs = {
    high: {
      model: "gemini-2.0-pro-deepthink",
      temperature: 1.0,      // High randomness enables better exploration
      maxOutputTokens: 16000,
      topP: 1.0,
      topK: 40,
    },
    medium: {
      model: "gemini-2.0-pro-adaptive-thinking",
      temperature: 1.0,      // Adaptive also prefers high temperature
      maxOutputTokens: 8000,
      topP: 0.95,
      topK: 40,
    },
    low: {
      model: "gemini-2.0-pro",
      temperature: 0.7,      // Standard inference: 0.7 is fine
      maxOutputTokens: 2048,
      topP: 0.9,
      topK: 40,
    },
  };

  const config = configs[complexity];
  const model = client.getGenerativeModel({
    model: config.model,
    generationConfig: config,
  });

  return model.generateContent({
    contents: [{ role: "user", parts: [{ text: prompt }] }],
  });
}

// ❌ Mistakes (shallow reasoning)

// topK: 10           // Restricts search space

// ✅ Correct
// temperature: 1.0
// topP: 1.0
```

## FAQ

### Q1: Should I use Deep Think or Adaptive Thinking?

**A:** Use this decision tree:
- **Proof/diagnosis/verification required** → Deep Think
- **Real-time response needed** → Adaptive Thinking
- **Cost optimization priority** → Adaptive Thinking
- **Maximum accuracy required** → Deep Think (cost is secondary)

### Q2: Can I predict thinking token usage?

**A:** Precise prediction is difficult, but estimation is possible:
- Input tokens: `countTokens()` gives exact count
- Output tokens: Assume 3–10x input as rough multiplier
- Context Caching: Reduces effective input cost by 90%

### Q3: How long does cache persist?

**A:** Default is 1 hour. Extendable to 24 hours via Google Cloud Console. Cache works across multiple questions on the same document.

### Q4: Can I combine batching with context caching?

**A:** Yes. Batch discount (50%) + caching (90%) = **up to 95% total savings**.

### Q5: Do reasoning models support languages other than English?

**A:** Yes. Deep Think and Adaptive Thinking work excellently with Japanese, Spanish, French, and 90+ other languages. Mathematical and logical reasoning quality is consistent across languages.
