## What Is Gemini Structured Output — Making AI Responses Type-Safe

Gemini API's Structured Output forces AI responses to conform to a predefined JSON schema. Unlike free-form text output, you get guaranteed-parseable JSON every time, which is a game-changer for production pipeline reliability.

"I want to process AI output programmatically, but the format is slightly different every time" — Structured Output solves this problem at its root.

This guide takes you from basic schema definitions through nested complex structures, validation strategies, and production-ready data extraction pipelines.

## Basic Schema Definition and Usage

Let's start with the simplest structured output setup.

```typescript
// basic-structured-output.ts
import { GoogleGenerativeAI, SchemaType } from '@google/generative-ai';

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

// Schema definition — strictly specify the AI output format
const productReviewSchema = {
  type: SchemaType.OBJECT,
  properties: {
    product_name: {
      type: SchemaType.STRING,
      description: "Name of the reviewed product",
    },
    rating: {
      type: SchemaType.NUMBER,
      description: "Rating score from 1 to 5",
    },
    pros: {
      type: SchemaType.ARRAY,
      items: { type: SchemaType.STRING },
      description: "List of positive aspects",
    },
    cons: {
      type: SchemaType.ARRAY,
      items: { type: SchemaType.STRING },
      description: "List of areas for improvement",
    },
    summary: {
      type: SchemaType.STRING,
      description: "Brief summary under 100 characters",
    },
    recommended: {
      type: SchemaType.BOOLEAN,
      description: "Whether the product is recommended",
    },
  },
  required: [
    "product_name", "rating", "pros", "cons", "summary", "recommended"
  ],
};

async function analyzeReview(reviewText: string) {
  const model = genAI.getGenerativeModel({
    model: "gemini-2.5-pro",
    generationConfig: {
      responseMimeType: "application/json",
      responseSchema: productReviewSchema,
    },
  });

  const result = await model.generateContent(
    `Analyze the following review text:\n\n${reviewText}`
  );

  // Type-safe JSON — no parsing errors to worry about
  const review = JSON.parse(result.response.text());
  console.log(`Product: ${review.product_name}`);
  console.log(`Rating: ${"★".repeat(review.rating)}${"☆".repeat(5 - review.rating)}`);
  console.log(`Recommended: ${review.recommended ? "Yes" : "No"}`);

  return review;
}
```

Combining `responseMimeType: "application/json"` with `responseSchema` guarantees Gemini outputs JSON conforming to your schema.

## Building a Validation Layer

Structured output guarantees schema-conforming JSON, but business logic validation is a separate concern.

```typescript
// validation.ts — Business logic validation
import { z } from 'zod';

// Zod schema for runtime validation
const InvoiceSchema = z.object({
  invoice_number: z.string().min(1),
  date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  vendor: z.object({
    name: z.string().min(1),
    address: z.string().optional(),
    tax_id: z.string().optional(),
  }),
  line_items: z.array(z.object({
    description: z.string(),
    quantity: z.number().positive(),
    unit_price: z.number().min(0),
    tax_rate: z.number().min(0).max(1).optional(),
    total: z.number().min(0),
  })).min(1),
  subtotal: z.number().min(0).optional(),
  tax_total: z.number().min(0).optional(),
  grand_total: z.number().min(0),
  payment_status: z.enum(["paid", "pending", "overdue"]),
  currency: z.enum(["JPY", "USD", "EUR"]),
});

type Invoice = z.infer<typeof InvoiceSchema>;

// Consistency check — verify line items sum matches grand total
function validateInvoiceConsistency(invoice: Invoice): {
  valid: boolean;
  errors: string[];
} {
  const errors: string[] = [];

  // Calculate line items total
  const calculatedTotal = invoice.line_items.reduce(
    (sum, item) => sum + item.total, 0
  );

  // Check against grand total (accounting for floating point)
  if (Math.abs(calculatedTotal - invoice.grand_total) > 1) {
    errors.push(
      `Line items total (${calculatedTotal}) doesn't match grand total (${invoice.grand_total})`
    );
  }

  // Validate quantities
  for (const item of invoice.line_items) {
    if (item.quantity !== Math.floor(item.quantity) && item.quantity < 0) {
      errors.push(`Invalid quantity: ${item.description} (${item.quantity})`);
    }
  }

  return { valid: errors.length === 0, errors };
}

// Usage
async function processInvoice(rawText: string) {
  // Step 1: Structured extraction with Gemini
  const extracted = await extractInvoiceWithGemini(rawText);

  // Step 2: Schema validation with Zod
  const parsed = InvoiceSchema.safeParse(extracted);
  if (!parsed.success) {
    console.error("Schema error:", parsed.error.format());
    throw new Error("Invalid invoice structure");
  }

  // Step 3: Business logic validation
  const consistency = validateInvoiceConsistency(parsed.data);
  if (!consistency.valid) {
    console.warn("Consistency warnings:", consistency.errors);
    // Log as warning but continue processing
  }

  return parsed.data;
}
```

## Building a Data Extraction Pipeline

Here's a pipeline for batch-processing multiple documents:

```typescript
// pipeline.ts — Data extraction pipeline
interface PipelineConfig {
  concurrency: number;     // Max parallel requests
  retryLimit: number;      // Retry attempts
  batchSize: number;       // Batch size
}

class ExtractionPipeline<T> {
  private config: PipelineConfig;
  private results: Map<string, {
    status: 'success' | 'error';
    data?: T;
    error?: string;
  }> = new Map();

  constructor(
    private schema: any,
    private validator: (data: unknown) => T,
    config?: Partial<PipelineConfig>
  ) {
    this.config = {
      concurrency: 3,
      retryLimit: 2,
      batchSize: 10,
      ...config,
    };
  }

  async process(
    documents: { id: string; content: string }[]
  ): Promise<Map<string, typeof this.results extends Map<string, infer V> ? V : never>> {
    // Split into batches
    const batches = this.chunk(documents, this.config.batchSize);

    for (const batch of batches) {
      // Process with concurrency control
      const semaphore = new Semaphore(this.config.concurrency);

      await Promise.all(
        batch.map(async (doc) => {
          await semaphore.acquire();
          try {
            const result = await this.processDocument(doc);
            this.results.set(doc.id, {
              status: 'success',
              data: result,
            });
          } catch (error) {
            this.results.set(doc.id, {
              status: 'error',
              error: error instanceof Error ? error.message : String(error),
            });
          } finally {
            semaphore.release();
          }
        })
      );
    }

    return this.results;
  }

  private async processDocument(
    doc: { id: string; content: string }
  ): Promise<T> {
    let lastError: Error | null = null;

    for (let attempt = 0; attempt <= this.config.retryLimit; attempt++) {
      try {
        const model = genAI.getGenerativeModel({
          model: "gemini-2.5-pro",
          generationConfig: {
            responseMimeType: "application/json",
            responseSchema: this.schema,
          },
        });

        const result = await model.generateContent(
          `Extract data from the following document:\n\n${doc.content}`
        );

        const parsed = JSON.parse(result.response.text());
        return this.validator(parsed);
      } catch (error) {
        lastError = error instanceof Error ? error : new Error(String(error));
        if (attempt < this.config.retryLimit) {
          // Exponential backoff
          await new Promise(r =>
            setTimeout(r, 1000 * Math.pow(2, attempt))
          );
        }
      }
    }

    throw lastError;
  }

  private chunk<U>(array: U[], size: number): U[][] {
    const chunks: U[][] = [];
    for (let i = 0; i < array.length; i += size) {
      chunks.push(array.slice(i, i + size));
    }
    return chunks;
  }
}

// Simple semaphore implementation
class Semaphore {
  private current = 0;
  private queue: (() => void)[] = [];

  constructor(private max: number) {}

  async acquire(): Promise<void> {
    if (this.current < this.max) {
      this.current++;
      return;
    }
    return new Promise<void>(resolve => {
      this.queue.push(resolve);
    });
  }

  release(): void {
    this.current--;
    const next = this.queue.shift();
    if (next) {
      this.current++;
      next();
    }
  }
}
```

## Error Handling Best Practices

Here are common errors with structured output and how to handle them:

```typescript
// error-handling.ts — Error handling patterns
async function safeStructuredGeneration<T>(
  model: any,
  prompt: string,
  validator: (data: unknown) => T,
  options?: { maxRetries?: number; fallbackModel?: string }
): Promise<{ success: true; data: T } | { success: false; error: string }> {
  const maxRetries = options?.maxRetries ?? 2;

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      const result = await model.generateContent(prompt);
      const text = result.response.text();

      // Parse JSON
      let parsed: unknown;
      try {
        parsed = JSON.parse(text);
      } catch {
        // Fallback for rare cases where JSON is wrapped in markdown
        const match = text.match(/```json\n?([\s\S]*?)\n?```/);
        if (match) {
          parsed = JSON.parse(match[1]);
        } else {
          throw new Error("Failed to parse JSON response");
        }
      }

      // Validate
      const validated = validator(parsed);
      return { success: true, data: validated };
    } catch (error) {
      const errorMsg = error instanceof Error ? error.message : String(error);

      if (attempt === maxRetries) {
        return { success: false, error: errorMsg };
      }

      // Adjust prompt before retry
      prompt = `${prompt}\n\nNote: The previous response caused an error (${errorMsg}). Please respond with valid JSON.`;
    }
  }

  return { success: false, error: "Max retries exceeded" };
}
```

## Performance Optimization Tips

### Keep Schemas Lean

Overly complex schemas increase processing time. Stick to the minimum required fields.

```typescript
// ❌ Over-engineered schema — too many fields
const overEngineeredSchema = {
  type: SchemaType.OBJECT,
  properties: {
    // 20+ fields...
  }
};

// ✅ Right-sized schema
const optimizedSchema = {
  type: SchemaType.OBJECT,
  properties: {
    // Core fields only, extensible later
    name: { type: SchemaType.STRING },
    value: { type: SchemaType.NUMBER },
    category: { type: SchemaType.STRING, enum: ["a", "b", "c"] },
  },
  required: ["name", "value"],
};
```

### Leverage Response Caching

Cache results for identical inputs to reduce API calls:

```typescript
// Cached structured output
const cache = new Map<string, { data: any; expiry: number }>();

async function cachedGeneration(
  prompt: string,
  schema: any,
  ttlMs: number = 3600000 // 1 hour
) {
  const cacheKey = JSON.stringify({ prompt, schema });
  const cached = cache.get(cacheKey);

  if (cached && cached.expiry > Date.now()) {
    return cached.data;
  }

  const result = await generateStructuredOutput(prompt, schema);
  cache.set(cacheKey, { data: result, expiry: Date.now() + ttlMs });
  return result;
}
```

## FAQ

### Q: Which Gemini models support structured output?

Gemini 2.5 Pro, Gemini 2.5 Flash, Gemini 2.0 Flash, and other major models support it. Enable it by setting both `responseMimeType: "application/json"` and `responseSchema` in your generation config.

### Q: Are there limitations on structured output?

Yes — there are limits on schema depth and property count. As a general rule, stay within 5 levels of nesting and under 100 properties. For very complex schemas, consider splitting across multiple API calls.

### Q: How does structured output differ from asking for JSON in the prompt?

When you instruct the model via prompt to "return JSON," there's no guarantee it will comply. Structured output enforces the JSON schema at the API level, guaranteeing 100% parseable JSON. For production environments, always use structured output.

## Wrapping Up — Building Reliable AI Pipelines with Structured Output

Gemini's structured output is the most reliable way to turn AI responses into usable data. Define your output format with a JSON schema, validate consistency with libraries like Zod, and handle errors with retry strategies. This three-layer approach gives you a stable AI data pipeline that's ready for production.

Start with a simple schema, get comfortable with the pattern, and then tackle more complex data extraction challenges from there.
