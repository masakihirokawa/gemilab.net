# Getting Started with Gemini 3 Computer Use — AI That Operates Your Browser

## Introduction

In March 2026, Google introduced **Computer Use tool** support for Gemini 3 Flash Preview and Gemini 3 Pro Preview. This breakthrough enables developers to build computer-operating agents — AI systems that can look at a screenshot and autonomously navigate a browser or desktop UI by clicking, typing, and scrolling just like a human would.

This is fundamentally different from traditional browser automation tools like Selenium or Playwright scripting. Rather than following a rigid set of pre-programmed instructions, Gemini's Computer Use understands the content of the screen and decides on the most appropriate action in real time, adapting to unexpected changes and dynamic interfaces.

In this guide, we'll cover everything you need to know: what Computer Use is, which models support it, how to implement it, and real-world scenarios where it shines.

---

## What Is the Computer Use Tool?

The Computer Use tool allows Gemini to receive a screenshot of a screen, reason about what it sees, and return a structured action — such as a mouse click at specific coordinates, keyboard input, or a scroll gesture. Your application then executes the action and feeds the next screenshot back to Gemini, creating an autonomous feedback loop.

### Supported Action Types

| Action | Description |
|--------|-------------|
| `click` | Click at specified (x, y) coordinates |
| `type` | Type a string of text |
| `key` | Send a keyboard shortcut or key press |
| `scroll` | Scroll in a given direction at given coordinates |
| `screenshot` | Request a new screenshot (for loop continuation) |
| `move` | Move the mouse cursor without clicking |

---

## Supported Models

Computer Use is available on the following Gemini 3 models as of March 2026:

| Model | Characteristics |
|-------|----------------|
| `gemini-3-flash-preview` | Fast and cost-efficient. Ideal for agentic workflows |
| `gemini-3-pro-preview` | Highest accuracy. Best for complex UI interactions |

> **Note:** In the Gemini 2.5 series, Computer Use required a dedicated `gemini-2.5-pro-preview-computer-use` model. With Gemini 3, the capability is built directly into both Flash and Pro, simplifying your setup considerably.

## Step 2: The Core Computer Use Loop

The heart of Computer Use is a simple loop: **capture screenshot → send to Gemini → receive action → execute → repeat**.

```python
import base64
from google import genai
from google.genai import types

# Initialize the client
client = genai.Client(api_key="YOUR_API_KEY")

def encode_screenshot(image_bytes: bytes) -> str:
    """Encode screenshot bytes to base64 string"""
    return base64.b64encode(image_bytes).decode("utf-8")

def get_next_action(screenshot_b64: str, task: str, history: list):
    """
    Send the current screenshot to Gemini and get the next action.
    """
    # Define the Computer Use tool
    computer_use_tool = types.Tool(
        computer_use=types.ComputerUseTool(
            display_width=1280,
            display_height=800,
        )
    )

    # Build the message with screenshot + task description
    messages = history + [
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/png",
                        "data": screenshot_b64,
                    },
                },
                {
                    "type": "text",
                    "text": f"This is the current screen. Task: {task}\nWhat should I do next?",
                },
            ],
        }
    ]

    response = client.models.generate_content(
        model="gemini-3-flash-preview",
        contents=messages,
        config=types.GenerateContentConfig(
            tools=[computer_use_tool],
            system_instruction=(
                "You are an AI agent that controls a computer. "
                "Look at the screen and decide the best action to accomplish the user's task. "
                "When the task is complete, respond with 'TASK_COMPLETE'."
            ),
        ),
    )
    return response
```

## Step 3: Building a Full Browser Agent with Playwright

```python
import asyncio
from playwright.async_api import async_playwright

async def browser_agent(task: str):
    """An AI agent that autonomously controls a browser."""
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=False)
        page = await browser.new_page(viewport={"width": 1280, "height": 800})

        # Start from Google
        await page.goto("https://www.google.com")

        history = []
        max_steps = 15  # Safety limit to prevent infinite loops

        for step in range(max_steps):
            # Capture the current state of the browser
            screenshot_bytes = await page.screenshot()
            screenshot_b64 = encode_screenshot(screenshot_bytes)

            print(f"\n--- Step {step + 1} ---")

            # Ask Gemini what to do next
            response = get_next_action(screenshot_b64, task, history)

            # Parse the response
            action = None
            for part in response.candidates[0].content.parts:
                if hasattr(part, "text") and "TASK_COMPLETE" in part.text:
                    print("Task complete!")
                    await browser.close()
                    return
                if hasattr(part, "computer_use") and part.computer_use:
                    action = part.computer_use
                    break

            if not action:
                print("No action returned. Assuming task is complete.")
                break

            # Execute the action
            print(f"Executing: {action.action}")
            await execute_action(page, action)

            # Append to history so Gemini remembers context
            history.append({
                "role": "assistant",
                "content": response.candidates[0].content.parts,
            })

            # Brief pause between actions
            await asyncio.sleep(0.5)

        await browser.close()

async def execute_action(page, action):
    """Translate a Gemini Computer Use action into a Playwright command."""
    if action.action == "click":
        await page.mouse.click(action.coordinate[0], action.coordinate[1])
    elif action.action == "type":
        await page.keyboard.type(action.text)
    elif action.action == "key":
        await page.keyboard.press(action.key)
    elif action.action == "scroll":
        await page.mouse.wheel(
            action.coordinate[0],
            action.coordinate[1],
            delta_x=getattr(action, "delta_x", 0),
            delta_y=getattr(action, "delta_y", 0),
        )
    elif action.action == "move":
        await page.mouse.move(action.coordinate[0], action.coordinate[1])

# Run the agent
asyncio.run(
    browser_agent("Search for 'Gemini 3 API' on Google and open the first result.")
)
```

## Best Practices

### Write Specific Task Descriptions

Vague instructions lead to unpredictable behavior. Instead of "click the button," say "click the blue Submit button in the top-right corner of the form." The more specific you are, the more reliably Gemini will understand the goal.

### Always Set a Maximum Step Limit

Never run the agent loop without a `max_steps` guard. Edge cases like captchas, unexpected pop-ups, or ambiguous UI states can otherwise lead to indefinite loops.

### Add Confirmation Steps for Critical Actions

For irreversible actions — form submissions, purchases, deletions — insert a confirmation prompt before execution. This prevents unintended consequences when the AI misinterprets context.

### Implement Robust Error Handling

Unexpected UI changes happen. Wrap action execution in try-except blocks and optionally feed the error state screenshot back to Gemini to let it self-recover:

```python
try:
    await execute_action(page, action)
except Exception as e:
    print(f"Action failed: {e}")
    # Take an error screenshot and ask Gemini how to recover
    error_screenshot = await page.screenshot()
    recovery_response = get_next_action(
        encode_screenshot(error_screenshot),
        f"The previous action failed with: {e}. How should we recover?",
        history
    )
```

## Frequently Asked Questions

**Q: How is this different from Gemini 2.5's Computer Use?**
A: Gemini 2.5 required a separate dedicated model (`gemini-2.5-pro-preview-computer-use`). With Gemini 3, Computer Use is natively integrated into both Flash and Pro, so you don't need a separate model endpoint.

**Q: Can I run this locally?**
A: Absolutely. With Playwright installed locally, the agent controls a real browser on your machine. For cloud-based execution, Browserbase is a popular option.

**Q: Is there an additional charge for Computer Use?**
A: No extra fee. Standard Gemini API token pricing applies. Screenshots are billed as multimodal (image) inputs.

**Q: What about security? Can the agent access sensitive data?**
A: The agent only sees what's on screen. Use sandboxed browser environments for sensitive workflows, and never run an untrusted task prompt with access to authenticated sessions containing sensitive information.

**Q: Does it support video input instead of screenshots?**
A: Currently, the primary input is still screenshots. For fluid interaction, you can capture screenshots at high frequency. Native video support may come in future releases.
