## Understanding KV Cache and Its Memory Problem

In Transformer-based models (Gemini, LLaMA, Mistral, and others), the Attention mechanism computes relationships between input tokens by storing **Key** and **Value** tensors in memory—collectively known as the "KV cache."

For long-context processing, this cache grows rapidly:

- Every token from the beginning of the sequence requires its Key and Value stored in memory
- For Gemini 2.5's 1-million-token context window (~1.5 million words), the KV cache alone consumes most GPU memory
- As we generate each new token, we compute attention against all prior tokens, requiring all previous Keys and Values

This means inference throughput is bottlenecked by **memory bandwidth**, not compute capacity. Even the fastest GPUs become starved for data.

## What TurboQuant Achieves

TurboQuant solves this by compressing the KV cache as follows:

| Metric | Conventional | TurboQuant (3-bit) |
|---|---|---|
| KV Cache Bit Depth | 16 bits (float16) | 3 bits |
| Memory Usage | 100% | ~17% (5.9x reduction) |
| Inference Speed (NVIDIA H100) | Baseline | Up to 8x faster |
| Accuracy Loss | Baseline | Zero (verified on LongBench) |

The standout feature: **training-free**. TurboQuant applies to already-trained models without re-training. No fine-tuning, no data preparation—just plug-and-play inference acceleration.

## PolarQuant and QJL: Two Breakthrough Algorithms

TurboQuant's power comes from two innovations:

### 1. PolarQuant: Eliminating Redundancy via Polar Coordinates

Conventional Key and Value tensors use **Cartesian (x, y) coordinates**. PolarQuant transforms them into **polar coordinates (angle θ, radius r)**.

The insight: **direction (angle) matters far more than magnitude (radius) for accurate attention computation**. You can quantize the radius aggressively while preserving angular precision. This separation eliminates quantization overhead entirely—a clever mathematical trick that prior approaches missed.

### 2. QJL: 1-Bit Error Correction via Johnson-Lindenstrauss

Even with smart quantization, some error is inevitable. QJL corrects this error using **random projections** based on the Johnson-Lindenstrauss lemma. It embeds the error vector into a low-dimensional space and reconstructs it with just **1 additional bit** of information. The result: accuracy loss approaches zero.
