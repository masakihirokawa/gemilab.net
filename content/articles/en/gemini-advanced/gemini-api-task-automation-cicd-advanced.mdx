## Integrating Gemini API into GitHub Actions

Let's build a workflow that automatically posts Gemini review comments when a PR is opened.

### Workflow Definition

```yaml
# .github/workflows/gemini-review.yml
name: Gemini Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install google-genai

      - name: Run Gemini Review
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: python .github/scripts/gemini_review.py
```

### Review Script

```python
# .github/scripts/gemini_review.py
import os
import subprocess
import json
from google import genai

client = genai.Client()

# Get the PR diff
diff = subprocess.run(
    ["git", "diff", "origin/main...HEAD", "--", ".", ":!package-lock.json"],
    capture_output=True, text=True
).stdout

if not diff.strip():
    print("No meaningful changes to review")
    exit(0)

# Generate structured review
prompt = f"""Review the following code diff and return results in JSON format.

## Output format
{{
  "summary": "Overview of all changes (1-2 sentences)",
  "issues": [
    {{
      "severity": "critical|warning|suggestion",
      "file": "file path",
      "description": "issue description"
    }}
  ],
  "positive": ["1-2 positive observations"]
}}

## Review criteria
- Potential bugs (severity: critical)
- Performance implications (severity: warning)
- Readability and maintainability improvements (severity: suggestion)

## Diff
{diff[:15000]}
"""

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=prompt,
    config={"response_mime_type": "application/json"}
)

review = json.loads(response.text)

# Post as PR comment
comment_body = f"## Gemini Code Review\\n\\n"
comment_body += f"**Summary**: {review['summary']}\\n\\n"

if review.get("issues"):
    comment_body += "### Issues Found\\n"
    for issue in review["issues"]:
        icon = {"critical": "🔴", "warning": "🟡", "suggestion": "💡"}
        comment_body += f"- {icon.get(issue['severity'], '📝')} **{issue['file']}**: {issue['description']}\\n"

if review.get("positive"):
    comment_body += "\\n### What's Good\\n"
    for p in review["positive"]:
        comment_body += f"- ✅ {p}\\n"

pr_number = os.environ.get("GITHUB_REF", "").split("/")[2] if "pull" in os.environ.get("GITHUB_REF", "") else None
if pr_number:
    subprocess.run(
        ["gh", "pr", "comment", pr_number, "--body", comment_body],
        check=True
    )
    print(f"✅ Review posted to PR #{pr_number}")
# Output: ✅ Review posted to PR #42
```

## Structured Output (JSON Mode) for Reliable Parsing

When processing AI responses programmatically, free-form text is fragile. Gemini API's **JSON Mode** guarantees structured data that conforms to your schema.

### Specifying response_mime_type

```python
from google import genai

client = genai.Client()

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="List the 3 most important new features in Node.js 22",
    config={
        "response_mime_type": "application/json",
        "response_schema": {
            "type": "object",
            "properties": {
                "features": {
                    "type": "array",
                    "items": {
                        "type": "object",
                        "properties": {
                            "name": {"type": "string"},
                            "description": {"type": "string"},
                            "impact": {"type": "string", "enum": ["high", "medium", "low"]}
                        },
                        "required": ["name", "description", "impact"]
                    }
                }
            },
            "required": ["features"]
        }
    }
)

data = json.loads(response.text)
for f in data["features"]:
    print(f"[{f['impact'].upper()}] {f['name']}: {f['description']}")
# Example output:
# [HIGH] require(esm): Directly require ES modules from CommonJS
# [HIGH] V8 Engine 13.0: Performance improvements and reduced memory usage
# [MEDIUM] Stable test runner: node:test is now production-ready
```

### Optimizing Costs with Batch Processing

For processing multiple tasks, async batching improves throughput significantly:

```python
import asyncio
from google import genai

client = genai.Client()

async def analyze_file(filepath: str, content: str) -> dict:
    """Analyze a single file and return results"""
    prompt = f"Rate this code's quality (1-10) and suggest improvements in JSON format:\n\n{content[:5000]}"

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=prompt,
        config={"response_mime_type": "application/json"}
    )
    return {"file": filepath, "analysis": json.loads(response.text)}

async def batch_review(files: dict[str, str]) -> list:
    """Review multiple files concurrently"""
    tasks = [analyze_file(path, content) for path, content in files.items()]
    return await asyncio.gather(*tasks)

# Usage
# results = asyncio.run(batch_review({"app.py": "...", "utils.py": "..."}))
```

## Systematic Prompt Tuning

Prompts aren't write-once artifacts — they need continuous refinement. Use this framework to systematically improve quality.

### Stabilize Output with Few-Shot Prompts

Including 2-3 examples of expected output dramatically improves consistency:

```python
prompt = """Generate a user-facing release note line from the following git commit message.

## Examples
Input: "fix(auth): handle expired refresh tokens gracefully"
Output: "Authentication: Seamless re-authentication when refresh tokens expire"

Input: "feat(search): add fuzzy matching support"
Output: "Search: Find what you're looking for even with typos"

Input: "perf(api): cache user preferences for 5min"
Output: "Performance: User settings now load faster"

## Target
Input: "fix(upload): retry failed chunk uploads with exponential backoff"
Output: """

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=prompt
)
print(response.text)
# Example output: "Uploads: Failed file uploads now automatically retry with smart backoff"
```

### Set Consistent Behavior with System Instructions

Apply common context across all requests using System Instructions:

```python
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="Review this PR: ...",
    config={
        "system_instruction": """You are a senior software engineer.
Follow these code review guidelines:
- Be specific in feedback (include file names and line numbers)
- Always suggest fixes (don't just point out problems)
- Mention at least one positive aspect
- Use a direct but respectful tone"""
    }
)
```

## Frequently Asked Questions

### How do I securely manage API keys in GitHub Actions?

Add `GEMINI_API_KEY` to your repository's Settings → Secrets and variables → Actions. Reference it in workflow files as `${{ secrets.GEMINI_API_KEY }}`. Secrets are automatically masked in logs.

### What if the diff exceeds the API's token limit?

Trim the diff to around 15,000 characters or split by file for batch processing. Gemini 2.5 Flash has a 1 million token context window, so typical PRs rarely hit the limit. For large monorepo refactors, consider file-level splitting.

### Should I trust AI review results directly?

Treat AI reviews as "drafts" — final judgment should always be human. This is especially important for security and business logic concerns. AI excels at detecting routine patterns (unused variables, missing error handling), while humans focus on architecture decisions and business requirement alignment.
