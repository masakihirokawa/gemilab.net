## Introduction — Why ADK and TypeScript?

AI agent development has evolved beyond simple single-model calls into **systems where multiple specialized agents collaborate** to solve complex tasks. Google's **Agent Development Kit (ADK)** is an open-source, code-first framework designed to build exactly these kinds of multi-agent systems.

While ADK was initially Python-only, the **TypeScript version (`@google/adk`)** was officially released in 2026. For teams already building web applications and Node.js backends in TypeScript, being able to construct AI agents in the same language eliminates context switching and simplifies the entire development workflow.

In this guide, you'll learn how to build a complete multi-agent system from scratch using ADK TypeScript, leveraging all three workflow agent types: **SequentialAgent, ParallelAgent, and LoopAgent**.

**What you'll learn:**

- Setting up ADK TypeScript and understanding core concepts
- Adding custom tools (Function Calling) to `LlmAgent`
- When and how to use `SequentialAgent`, `ParallelAgent`, and `LoopAgent`
- Passing state between agents in a pipeline
- MCP tool integration and deployment patterns
- Cost and performance optimization strategies

**Target audience:** Intermediate to advanced developers with TypeScript experience and basic knowledge of the Gemini API.

## Concepts and Architecture

### Three Agent Types

ADK provides two categories of agents: **LLM agents** and **workflow agents**.

**LlmAgent (LLM Agent):**
The core agent type that calls Gemini models for reasoning, tool execution, and conversation. You define its `instruction` (system prompt), `tools` (available capabilities), and `model` (which Gemini model to use).

**Workflow Agents (three types):**

| Agent | Use Case | Execution Pattern |
|---|---|---|
| `SequentialAgent` | Execute tasks in a fixed order | A → B → C |
| `ParallelAgent` | Run independent tasks simultaneously | A, B, C in parallel |
| `LoopAgent` | Iterate until a condition is met | (A → B) × N times |

### State Management

Data flows between agents through **state** (`context.state`). Setting the `outputKey` property on an `LlmAgent` automatically saves its final response to state under that key.

```typescript
// Save the writer agent's output to the "draft" state key
const writerAgent = new LlmAgent({
  name: "writer",
  model: "gemini-2.5-flash",
  instruction: "Write a report based on the provided data.",
  outputKey: "draft"
});

// Subsequent agents can read context.state["draft"]
```

## Step-by-Step Implementation

### Step 1: Define Custom Tools

Start by defining the tools your agents will use. ADK TypeScript uses `zod` schemas for type-safe parameter definitions.

```typescript
// tools/search.ts
import { FunctionTool } from "@google/adk";
import { z } from "zod";

export const webSearchTool = new FunctionTool({
  name: "web_search",
  description: "Search the web for a given query and return results",
  parameters: z.object({
    query: z.string().describe("The search query"),
    maxResults: z.number().default(5).describe("Maximum number of results")
  }),
  handler: async ({ query, maxResults }) => {
    console.log(`Searching: ${query} (max ${maxResults} results)`);

    // Call an actual search API here (e.g., Google Custom Search)
    const results = [
      { title: `Latest on ${query}`, url: "https://example.com/1", snippet: "Recent developments..." },
      { title: `Complete Guide to ${query}`, url: "https://example.com/2", snippet: "Comprehensive overview..." }
    ];

    return JSON.stringify(results.slice(0, maxResults));
  }
});
```

```typescript
// tools/analyzer.ts
import { FunctionTool } from "@google/adk";
import { z } from "zod";

export const sentimentAnalyzer = new FunctionTool({
  name: "analyze_sentiment",
  description: "Analyze the sentiment of a given text as positive, negative, or neutral",
  parameters: z.object({
    text: z.string().describe("The text to analyze")
  }),
  handler: async ({ text }) => {
    const wordCount = text.split(/\s+/).length;
    const positiveWords = ["excellent", "great", "good", "amazing", "outstanding"];
    const negativeWords = ["bad", "terrible", "poor", "failing", "problematic"];

    let score = 0;
    positiveWords.forEach(w => { if (text.toLowerCase().includes(w)) score++; });
    negativeWords.forEach(w => { if (text.toLowerCase().includes(w)) score--; });

    return JSON.stringify({
      sentiment: score > 0 ? "positive" : score < 0 ? "negative" : "neutral",
      confidence: Math.min(Math.abs(score) * 0.2 + 0.5, 1.0),
      wordCount
    });
  }
});
```

### Step 3: Build the Multi-Agent Pipeline

Combine the three workflow agent types to create a sophisticated pipeline.

```typescript
// agent.ts (root agent)
import { SequentialAgent, ParallelAgent, LoopAgent, LlmAgent } from "@google/adk";
import { researcherAgent } from "./agents/researcher.js";
import { analystAgent } from "./agents/analyst.js";
import { writerAgent } from "./agents/writer.js";

// --- Pattern 1: ParallelAgent (concurrent research) ---
const marketResearcher = new LlmAgent({
  name: "market_researcher",
  model: "gemini-2.5-flash",
  instruction: "Research market trends and competitive landscape for the topic.",
  outputKey: "market_data"
});

const techResearcher = new LlmAgent({
  name: "tech_researcher",
  model: "gemini-2.5-flash",
  instruction: "Research technical trends and innovations for the topic.",
  outputKey: "tech_data"
});

const parallelResearch = new ParallelAgent({
  name: "parallel_research",
  description: "Research market and technical aspects simultaneously",
  subAgents: [marketResearcher, techResearcher]
});

// --- Pattern 2: LoopAgent (iterative refinement) ---
const reviewerAgent = new LlmAgent({
  name: "reviewer",
  model: "gemini-2.5-pro",
  instruction: `Review the "draft" in state and evaluate for:
- Accuracy, logical consistency, readability
- If satisfactory, output "APPROVED"
- Otherwise, list specific improvements needed`,
  outputKey: "review_feedback"
});

const editorAgent = new LlmAgent({
  name: "editor",
  model: "gemini-2.5-flash",
  instruction: `Improve the "draft" based on "review_feedback".
If feedback contains "APPROVED", output the draft as-is.`,
  outputKey: "draft"
});

const refinementLoop = new LoopAgent({
  name: "refinement_loop",
  description: "Iteratively review and refine the report",
  subAgents: [reviewerAgent, editorAgent],
  maxIterations: 3
});

// --- Root Agent: Full pipeline ---
export const rootAgent = new SequentialAgent({
  name: "intelligent_report_generator",
  description: "Runs parallel research, analysis, writing, and iterative refinement",
  subAgents: [
    parallelResearch,  // Parallel research
    analystAgent,      // Analysis
    writerAgent,       // Writing
    refinementLoop     // Iterative refinement
  ]
});
```

### Step 4: Run and Test

Use ADK DevTools to test your agents interactively in the browser:

```bash
# Run interactively in the terminal
npx @google/adk-devtools run agent.ts

# Launch the web UI (http://localhost:8000)
npx @google/adk-devtools web agent.ts
```

To run programmatically, use `Runner` and `InMemorySessionService`:

```typescript
// run.ts
import { Runner, InMemorySessionService } from "@google/adk";
import { rootAgent } from "./agent.js";

async function main() {
  const sessionService = new InMemorySessionService();
  const runner = new Runner({
    agent: rootAgent,
    appName: "report-generator",
    sessionService
  });

  const session = await sessionService.createSession({
    appName: "report-generator"
  });

  const response = runner.runAsync({
    userId: "user-1",
    sessionId: session.id,
    newMessage: {
      role: "user",
      parts: [{ text: "Create a report on the latest Gemini API developments" }]
    }
  });

  for await (const event of response) {
    if (event.content?.parts) {
      for (const part of event.content.parts) {
        if (part.text) {
          console.log(`[${event.author}] ${part.text}`);
        }
      }
    }
  }
}

main().catch(console.error);
```

## Advanced Patterns

### MCP Tool Integration

ADK TypeScript supports the **Model Context Protocol (MCP)**, allowing you to connect existing MCP servers as agent tools.

```typescript
import { LlmAgent, MCPTool } from "@google/adk";

const mcpTool = new MCPTool({
  serverUrl: "http://localhost:3001",
  toolName: "database_query",
  description: "Execute SQL queries against the database"
});

const dataAgent = new LlmAgent({
  name: "data_agent",
  model: "gemini-2.5-flash",
  instruction: "Query the database based on user questions and return results.",
  tools: [mcpTool]
});
```

### Using Agents as Tools

In ADK, you can wrap one agent as a **tool** for another. This enables modular, composable system design.

```typescript
import { LlmAgent, AgentTool } from "@google/adk";

const translatorAgent = new LlmAgent({
  name: "translator",
  model: "gemini-2.5-flash",
  instruction: "Translate the given text from English to Japanese."
});

const mainAgent = new LlmAgent({
  name: "main_assistant",
  model: "gemini-2.5-pro",
  instruction: "Use the translator tool when the user needs translation.",
  tools: [new AgentTool({ agent: translatorAgent })]
});
```

## Troubleshooting

### Common Errors and Solutions

**`GOOGLE_API_KEY is not set` error:**
The environment variable is missing. Either set it directly with `export GOOGLE_API_KEY="..."` or use a `.env` file with the `dotenv` package.

**`Module not found` error:**
Ensure `"type": "module"` is set in `package.json` and `module` is set to `nodenext` in `tsconfig.json`. Import paths must include the `.js` extension explicitly.

**LoopAgent runs indefinitely:**
Always set `maxIterations`. Additionally, include clear exit conditions in the reviewer agent's instructions, such as outputting "APPROVED" when quality criteria are met.

**State not passed to subsequent agents:**
Verify that `outputKey` is set on each agent. Without it, an agent's output won't be saved to state for downstream agents to read.

## Cost and Performance Considerations

### Optimize Costs with Model Selection

In multi-agent systems, assigning the right model to each agent is the key to cost optimization.

| Use Case | Recommended Model | Reasoning |
|---|---|---|
| Information gathering | `gemini-2.5-flash` | Fast and cost-effective |
| Analysis and reasoning | `gemini-2.5-pro` | High-precision reasoning required |
| Text generation | `gemini-2.5-flash` | Suited for large text output |
| Quality review | `gemini-2.5-pro` | Critical thinking needed |

### Token Consumption Estimates

For a SequentialAgent with 3 sub-agents in series, assuming each consumes roughly 1,000–3,000 tokens, a single pipeline run requires approximately 3,000–9,000 tokens. ParallelAgent reduces latency but doesn't change total token consumption.

### Latency Optimization Tips

- Use `ParallelAgent` to run independent tasks concurrently
- Keep `maxIterations` in `LoopAgent` to the minimum necessary
- Default to Flash models; reserve Pro for tasks requiring deeper reasoning
- Use `outputKey` to save only essential data to state, preventing context bloat

## FAQ

### Q1: Are there feature differences between the TypeScript and Python versions of ADK?

The core agent types (`LlmAgent`, `SequentialAgent`, `ParallelAgent`, `LoopAgent`) are fully supported in TypeScript. Some experimental features and third-party integrations may land in the Python version first, but as of March 2026, both versions offer near-equivalent functionality.

### Q2: Can I use non-Gemini models (Claude, GPT-4o, etc.) with ADK?

ADK is optimized for Gemini models, but its architecture allows for extension to other LLM providers. However, features like Function Calling and streaming depend on the Gemini API, so full compatibility with other models is not guaranteed.

### Q3: How should I deploy to production?

ADK agents can be deployed via: (1) **Cloud Run** as containerized services, (2) **Vertex AI Agent Engine** as a managed service, or (3) on your own Node.js server integrated with Express or Fastify. Vertex AI Agent Engine is recommended for production due to its built-in scalability and monitoring.

### Q4: How should I handle errors between agents?

When a sub-agent in a `SequentialAgent` encounters an error, the pipeline stops by default. For robust systems, explicitly instruct each agent on error behavior in its `instruction` — such as generating fallback values or logging error messages to state.

### Q5: How should I design LoopAgent exit conditions?

Setting `maxIterations` as an upper bound is essential. Beyond that, build exit logic into the reviewer agent's instructions, such as outputting a keyword like "APPROVED" when quality criteria are met. ADK also provides a built-in `exit_loop` tool that agents can invoke to autonomously break out of a loop.
