## Introduction to Thinking Mode

Gemini Thinking Mode reveals the internal reasoning process that an AI model uses to solve complex problems. Normally, the model returns only the final answer. With Thinking Mode enabled, you get transparency into the logic, intermediate steps, and considerations the model explored before arriving at its conclusion.

This capability proves invaluable for challenging tasks like:

- Mathematical problem-solving and abstract reasoning
- System architecture decisions and code design validation
- Multi-step data analysis and hypothesis testing
- Debugging and troubleshooting complex issues
- Research and synthesis of information

By witnessing how the model thinks, you can better evaluate answer quality and identify gaps in reasoning.

## Understanding Thought Budgets

### What Are Thought Budgets?

Thinking Mode operates on a token budget system. You allocate a maximum number of tokens (computational resources) the model can spend "thinking" before outputting its response.

- **Lightweight Budget** (2,000-3,000 tokens): Quick reasoning for straightforward problems
- **Standard Budget** (5,000-8,000 tokens): Balanced depth for typical complex tasks
- **Deep Budget** (10,000-20,000+ tokens): Thorough exploration for intricate problems

```python
import google.generativeai as genai

def solve_with_thinking(problem: str, budget_tokens: int = 5000) -> dict:
    """Use Thinking Mode to solve a problem"""
    client = genai.Client(api_key="YOUR_API_KEY")

    # Enable Thinking Mode with budget
    response = client.models.generate_content(
        model="gemini-2.0-pro-exp",
        contents=problem,
        config=genai.GenerateContentConfig(
            thinking={
                "type": "enabled",
                "budget_tokens": budget_tokens
            }
        )
    )

    # Extract thinking and response
    thinking_text = ""
    response_text = ""

    for part in response.content.parts:
        if hasattr(part, 'thought') and part.thought:
            thinking_text = part.thought
        elif hasattr(part, 'text') and part.text:
            response_text = part.text

    return {
        "thinking_process": thinking_text,
        "final_answer": response_text,
        "input_tokens": response.usage_metadata.input_tokens,
        "output_tokens": response.usage_metadata.output_tokens,
        "thinking_tokens": getattr(response.usage_metadata, 'cache_creation_input_tokens', 0)
    }

# Example: Solve a challenging math problem
problem = """
Solve this system of equations:
2x + 3y - z = 7
x - 2y + z = 4
3x + y + 2z = 13

Show your complete working.
"""

result = solve_with_thinking(problem, budget_tokens=8000)

print("THINKING PROCESS:")
print(result["thinking_process"][:400] + "...\n")
print("FINAL ANSWER:")
print(result["final_answer"])
print(f"\nTokens - Input: {result['input_tokens']}, Output: {result['output_tokens']}")

# Expected output:
# THINKING PROCESS:
# I need to solve this system of three equations with three unknowns.
# Let me use substitution or elimination method...
# From equation 1: 2x + 3y - z = 7
# From equation 2: x - 2y + z = 4
# Adding equations 1 and 2: 3x + y = 11...
#
# FINAL ANSWER:
# x = 2, y = 1, z = -1
```

## Chain-of-Thought Reasoning

### Implementing Structured Reasoning

Chain-of-Thought (CoT) prompting guides the model through step-by-step logic. Combined with Thinking Mode, this produces more reliable and transparent results.

```python
def step_by_step_analysis(task: str, enable_cot: bool = True) -> dict:
    """Solve task with explicit step-by-step guidance"""
    client = genai.Client()

    # Craft CoT-optimized prompt
    if enable_cot:
        prompt = f"""
        Solve this problem methodically. Break it down:

        STEP 1: Problem Analysis
        - What is being asked?
        - What constraints exist?

        STEP 2: Information Assessment
        - What facts do we have?
        - What's missing or assumed?

        STEP 3: Strategy Planning
        - What approach will work?
        - Are there alternative paths?

        STEP 4: Execution
        - Work through the solution
        - Show intermediate results

        STEP 5: Verification
        - Does the answer make sense?
        - Check against constraints

        PROBLEM: {task}
        """
    else:
        prompt = f"PROBLEM: {task}"

    response = client.models.generate_content(
        model="gemini-2.0-pro-exp",
        contents=prompt,
        config=genai.GenerateContentConfig(
            thinking={
                "type": "enabled",
                "budget_tokens": 10000
            }
        )
    )

    # Extract structured reasoning
    reasoning = ""
    for part in response.content.parts:
        if hasattr(part, 'text'):
            reasoning = part.text
            break

    return {
        "structured_answer": reasoning,
        "model": "gemini-2.0-pro-exp"
    }

# Example: Logic puzzle
task = """
Alice, Bob, and Carol each have different favorite colors (red, blue, green).
- Alice doesn't like red
- Bob doesn't like blue
- The person who likes blue also likes green (they're complementary)
- Actually, scratch that - each person has ONE favorite

Who likes which color?
"""

result = step_by_step_analysis(task)
print(result["structured_answer"])

# Expected output:
# STEP 1: Problem Analysis
# We have 3 people and 3 colors to match...
# STEP 2: Constraints
# Alice ≠ red, Bob ≠ blue
# ...
```

## Real-World Applications

### Code Architecture Review

Leverage Thinking Mode to validate complex design decisions:

```python
def review_architecture(architecture_description: str) -> dict:
    """Use Thinking Mode to evaluate system design"""
    client = genai.Client()

    review_prompt = f"""
    Review this system architecture from multiple angles:

    ARCHITECTURE:
    {architecture_description}

    EVALUATION CRITERIA:
    1. Scalability: Can it handle 10x traffic growth?
    2. Security: Where are potential vulnerabilities?
    3. Performance: What are bottlenecks?
    4. Maintainability: Is it clear and well-structured?
    5. Cost: Is this efficient resource-wise?

    Provide detailed analysis with specific recommendations.
    """

    response = client.models.generate_content(
        model="gemini-2.0-pro-exp",
        contents=review_prompt,
        config=genai.GenerateContentConfig(
            thinking={
                "type": "enabled",
                "budget_tokens": 15000  # Complex analysis needs more budget
            }
        )
    )

    analysis = ""
    for part in response.content.parts:
        if hasattr(part, 'text'):
            analysis = part.text
            break

    return {
        "detailed_review": analysis,
        "budget_allocated": 15000
    }

# Example
architecture = """
Microservices architecture with Kubernetes orchestration.
PostgreSQL for data, Redis for caching.
API Gateway routing to 8 services.
Event-driven communication via Kafka.

print(review["detailed_review"])
```

## Budget Management and Optimization

### Selecting Appropriate Budget Levels

```python
def estimate_thinking_budget(task_type: str) -> int:
    """Calculate reasonable thinking budget by problem type"""
    budget_guidelines = {
        "simple_qa": 2000,           # Basic questions
        "calculation": 3000,         # Math/arithmetic
        "logic_puzzle": 6000,        # Reasoning chains
        "code_design": 8000,         # Architecture decisions
        "research": 12000,           # Synthesis and analysis
        "optimization": 15000        # Multi-criteria decisions
    }

    return budget_guidelines.get(task_type, 5000)

def process_tasks_efficiently(tasks: list) -> list:
    """Process multiple tasks with optimized budgets"""
    client = genai.Client()
    results = []

    for task in tasks:
        # Classify task difficulty
        task_type = classify_task(task)
        budget = estimate_thinking_budget(task_type)

        response = client.models.generate_content(
            model="gemini-2.0-pro-exp",
            contents=task,
            config=genai.GenerateContentConfig(
                thinking={
                    "type": "enabled",
                    "budget_tokens": budget
                }
            )
        )

        answer = ""
        for part in response.content.parts:
            if hasattr(part, 'text'):
                answer = part.text
                break

        results.append({
            "task": task[:100],
            "task_type": task_type,
            "budget": budget,
            "answer": answer
        })

    return results

def classify_task(task_text: str) -> str:
    """Simple heuristic-based task classification"""
    if any(word in task_text.lower() for word in ["how", "why"]):
        return "logic_puzzle"
    elif any(word in task_text.lower() for word in ["design", "architect"]):
        return "code_design"
    elif any(word in task_text.lower() for word in ["calculate", "solve"]):
        return "calculation"
    return "simple_qa"

# Process batch
tasks = [
    "What is the square root of 144?",
    "Design a caching layer for a social media feed",
    "Explain why quantum computers are powerful"
]

batch_results = process_tasks_efficiently(tasks)
for result in batch_results:
    print(f"{result['task_type']}: {result['budget']} tokens")

# Expected output:
# calculation: 3000 tokens
# code_design: 8000 tokens
# logic_puzzle: 6000 tokens
```

## Frequently Asked Questions

### Q1: Which Gemini models support Thinking Mode?

**A:** Currently, Thinking Mode is available on `gemini-2.0-pro-exp` and select advanced models. It's not available on standard models like `gemini-2.0-flash`. Check the latest documentation for updated model availability, as this continues to expand.

### Q2: Does a larger budget always produce better results?

**A:** Not necessarily. The optimal budget depends on problem complexity. A small problem wastes tokens with a 20,000-token budget, while a complex problem may feel rushed with 2,000 tokens. Start with 5,000-8,000 for typical problems and adjust based on performance. Monitor where diminishing returns appear.

### Q3: How does Thinking Mode affect API costs?

**A:** Thinking tokens count toward your usage, increasing costs proportionally. However, better reasoning often means fewer follow-up requests needed, improving overall efficiency. Use Thinking Mode selectively for problems that truly benefit from deeper reasoning.
