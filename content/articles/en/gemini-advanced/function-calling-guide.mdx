# Gemini Function Calling Complete Guide — External API Integration

Function calling elevates Gemini from a language model into an intelligent agent capable of interacting with external systems. By declaring functions (tools) that Gemini can call, you enable AI-driven automation, real-time data retrieval, and seamless integration with your existing APIs.

## Understanding Function Calling

Function calling is a mechanism where:
1. You define a set of functions/tools your application supports
2. You provide these definitions to Gemini
3. When appropriate, Gemini decides to call a function instead of generating text
4. You execute the function and return the result to Gemini
5. Gemini uses the result to formulate its next response

This creates a powerful loop where Gemini reasons about which tools to use and when to use them.

## Why Function Calling Matters

**Without function calling**, a model can only provide information from its training data. Ask "What's the weather?" and you get a generic response.

**With function calling**, Gemini can:
- Call a weather API to get current conditions
- Query a database for real-time information
- Process transactions through payment systems
- Retrieve documents and analyze them
- Perform complex calculations
- Trigger automation workflows

This transforms Gemini into an intelligent intermediary between users and your systems.

## Declaring Functions for Gemini

### Function Declaration Structure

A function declaration tells Gemini what tools are available:

```python
function_declaration = {
    "name": "get_weather",
    "description": "Get current weather for a location",
    "parameters": {
        "type": "object",
        "properties": {
            "location": {
                "type": "string",
                "description": "City name or location"
            },
            "unit": {
                "type": "string",
                "enum": ["celsius", "fahrenheit"],
                "description": "Temperature unit"
            }
        },
        "required": ["location"]
    }
}
```

### Key Components

**name**: The function name Gemini uses to reference the tool. Use snake_case, make it descriptive and unique.

**description**: Clear explanation of what the function does. Gemini reads this to decide whether to call it. Be concise but informative.

**parameters**: A JSON Schema object describing input parameters. This is critical:
- Define each parameter's type and description
- Mark required parameters
- Include enums for restricted values
- Provide examples in descriptions when helpful

### Good vs. Poor Function Declarations

**Poor declaration:**
```python
{
    "name": "fn1",
    "description": "Does stuff",
    "parameters": {}
}
```

**Good declaration:**
```python
{
    "name": "search_products",
    "description": "Search the product catalog by name, category, or price range. Returns matching products with prices and availability.",
    "parameters": {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "Product name or keyword to search for"
            },
            "category": {
                "type": "string",
                "enum": ["electronics", "clothing", "books", "home"],
                "description": "Filter results by category"
            },
            "max_price": {
                "type": "number",
                "description": "Maximum price in USD. Omit for no limit."
            }
        },
        "required": ["query"]
    }
}
```

### Implementing Tool Functions

Create actual Python functions that do the work:

```python
def get_product_info(product_id: str) -> dict:
    """Simulate fetching product information from a database"""
    products = {
        "PROD001": {
            "name": "Wireless Headphones",
            "price": 79.99,
            "description": "Premium sound quality with noise cancellation",
            "rating": 4.5
        },
        "PROD002": {
            "name": "USB-C Cable",
            "price": 12.99,
            "description": "Fast charging and data transfer cable",
            "rating": 4.8
        }
    }
    return products.get(product_id, {"error": "Product not found"})

def check_stock(product_id: str, location: str) -> dict:
    """Simulate checking inventory"""
    inventory = {
        ("PROD001", "New York"): {"available": 15, "status": "in stock"},
        ("PROD001", "Los Angeles"): {"available": 0, "status": "out of stock"},
        ("PROD002", "New York"): {"available": 50, "status": "in stock"}
    }
    return inventory.get(
        (product_id, location),
        {"error": "Location not found"}
    )

# Create a mapping for easy lookup
TOOLS = {
    "get_product_info": get_product_info,
    "check_stock": check_stock
}
```

### Full Function Calling Loop

```python
def process_function_call(tool_name: str, tool_input: dict) -> str:
    """Execute a tool and return results"""
    if tool_name not in TOOLS:
        return json.dumps({"error": f"Unknown tool: {tool_name}"})

    try:
        result = TOOLS[tool_name](**tool_input)
        return json.dumps(result)
    except Exception as e:
        return json.dumps({"error": str(e)})

def chat_with_tools(user_message: str):
    """Multi-turn conversation with function calling"""

    # Start conversation
    messages = [
        {"role": "user", "parts": [{"text": user_message}]}
    ]

    while True:
        # Call Gemini with tools available
        response = client.models.generate_content(
            model="gemini-2.5-pro",
            contents=messages,
            tools=[{"function_declarations": tools}]
        )

        # Check if model wants to call a function
        if response.candidates[0].content.parts:
            for part in response.candidates[0].content.parts:
                if part.function_call:
                    # Extract function name and arguments
                    function_name = part.function_call.name
                    function_args = dict(part.function_call.args)

                    print(f"Calling function: {function_name}")
                    print(f"Arguments: {function_args}")

                    # Execute the function
                    result = process_function_call(function_name, function_args)

                    # Add model's function call request to messages
                    messages.append({
                        "role": "model",
                        "parts": [part]
                    })

                    # Add function result to messages
                    messages.append({
                        "role": "user",
                        "parts": [{
                            "function_response": {
                                "name": function_name,
                                "response": json.loads(result)
                            }
                        }]
                    })

                    # Continue the loop to get Gemini's response
                    break
        else:
            # No function call, model provided text response
            text_response = response.candidates[0].content.parts[0].text
            print(f"Assistant: {text_response}")
            return

# Usage
chat_with_tools("What's the price of PROD001 and is it in stock in New York?")
```

### Function Implementation and Execution Loop

```typescript
async function executeFunction(
  functionName: string,
  args: Record<string, string>
): Promise<string> {
  if (functionName === "get_current_temperature") {
    // Simulate API call
    const temperature = Math.floor(Math.random() * 30) + 5;
    return JSON.stringify({
      location: args.location,
      temperature: temperature,
      unit: args.unit || "celsius"
    });
  }
  return JSON.stringify({ error: "Unknown function" });
}

async function chatWithFunctions(userMessage: string) {
  const model = client.getGenerativeModel({
    model: "gemini-2.5-pro"
  });

  const chat = model.startChat({
    tools
  });

  let response = await chat.sendMessage(userMessage);

  while (true) {
    // Check for function calls
    const functionCalls = response.response.functionCalls();

    if (functionCalls && functionCalls.length > 0) {
      const functionResponses = [];

      for (const call of functionCalls) {
        console.log(`Calling function: ${call.name}`);
        console.log(`Arguments:`, call.args);

        const result = await executeFunction(call.name, call.args);

        functionResponses.push({
          name: call.name,
          response: JSON.parse(result)
        });
      }

      // Send function results back to Gemini
      response = await chat.sendMessage(
        functionResponses.map(fr => ({
          functionResponse: fr
        }))
      );
    } else {
      // No more function calls, print final response
      console.log("Assistant:", response.response.text());
      break;
    }
  }
}

chatWithFunctions("What's the temperature in Paris?");
```

## Advanced Patterns

### Automatic Function Calling

For simpler use cases, enable automatic function calling where Gemini directly receives function results:

```python
# Python
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=user_message,
    tools=[{"function_declarations": tools}],
    tool_config={
        "function_calling_config": {
            "mode": "AUTO",
            "allowed_function_names": ["get_product_info", "check_stock"]
        }
    }
)
```

### Combining Multiple Functions

Declare multiple related functions and let Gemini compose them:

```python
tools = [
    {"name": "search_inventory", ...},
    {"name": "calculate_shipping", ...},
    {"name": "process_payment", ...}
]
# Gemini can call these in sequence to complete complex tasks
```

### Error Recovery

Handle function execution failures gracefully:

```python
def process_with_error_handling(tool_name: str, args: dict) -> str:
    try:
        result = TOOLS[tool_name](**args)
        return json.dumps(result)
    except ValueError as e:
        return json.dumps({
            "error": f"Invalid input: {str(e)}",
            "suggestion": "Please provide valid parameters"
        })
    except ConnectionError as e:
        return json.dumps({
            "error": "Service temporarily unavailable",
            "retry": True
        })
```

## Best Practices

1. **Clear Declarations**: Write precise function descriptions so Gemini understands when to use them

2. **Validate Inputs**: Always validate function arguments before execution

3. **Meaningful Responses**: Return structured, detailed results—Gemini uses these to reason

4. **Error Handling**: Provide helpful error messages when functions fail

5. **Limited Tool Sets**: Declare only the functions your application needs. Too many options confuses the model

6. **Test Thoroughly**: Test function calling with various user inputs and edge cases

7. **Monitor Usage**: Track which functions are called most frequently to optimize your toolkit

8. **Security**: Validate that function calls are appropriate before execution. Never allow arbitrary code execution
