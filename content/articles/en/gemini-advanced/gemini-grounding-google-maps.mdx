## Introduction: How Location-Aware AI Transforms App Experiences

What if your app could answer "find me the best ramen spot nearby" with accurate, real-time data rather than outdated training knowledge?

**Grounding with Google Maps** integrates Gemini's advanced generative AI with Google Maps' database of over 250 million places in real time. Since its general availability in early 2026, developers can now build location-aware AI that doesn't hallucinate — because every answer is grounded in live map data.

In this guide you'll learn:

- How to set up Maps Grounding on Vertex AI step by step
- Working Python code for restaurant search, real estate, and travel itinerary use cases
- How to implement source attribution correctly (it's required by Google policy)
- Cost calculations and rate limit handling strategies
- The most common errors and how to fix them

This article targets intermediate to advanced engineers who have basic familiarity with Python and Google Cloud and want to ship production-quality location AI.


## Concepts and Design Philosophy

### Why Grounding Matters

Standard LLMs are frozen at training time. They can't tell you which coffee shop is open right now, or what the current star rating of a restaurant is. Maps Grounding solves this by letting Gemini reach out to live Google Maps data whenever it determines the query is location-relevant.

### How It Works

```
User Input
    ↓
Gemini (analyzes query)
    ↓ (if location-relevant)
Google Maps API (real-time data fetch)
    ↓
Gemini (generates answer grounded in Maps data)
    ↓
Response (with groundingMetadata)
```

Gemini decides automatically whether to invoke Maps Grounding. You don't need to tag each query — but providing `lat_lng` coordinates enables personalized, location-specific results.

## Step-by-Step Implementation

### Step 1: Basic Request

Start with a minimal working implementation.

```python
# basic_maps_grounding.py
import os
from google import genai
from google.genai import types

# Initialize Vertex AI client
# Requires GOOGLE_GENAI_USE_VERTEXAI=True in environment
client = genai.Client(
    http_options=types.HttpOptions(api_version="v1")
)

def search_nearby(query: str, latitude: float, longitude: float) -> str:
    """
    Run a location-aware search using Google Maps Grounding.

    Args:
        query: Natural language search query
        latitude: User's latitude
        longitude: User's longitude

    Returns:
        AI-generated response grounded in Google Maps data
    """
    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=query,
        config=types.GenerateContentConfig(
            tools=[
                types.Tool(
                    google_maps=types.GoogleMaps(enable_widget=False)
                )
            ],
            tool_config=types.ToolConfig(
                retrieval_config=types.RetrievalConfig(
                    lat_lng=types.LatLng(
                        latitude=latitude,
                        longitude=longitude
                    ),
                    language_code="en_US",
                )
            ),
        ),
    )
    return response.text

# Example: search near Times Square, NYC
if __name__ == "__main__":
    result = search_nearby(
        query="What are the top-rated ramen restaurants within a 10-minute walk?",
        latitude=40.7580,
        longitude=-73.9855,
    )
    print(result)

# Expected output:
# Near Times Square, several highly-rated ramen spots stand out.
# Ivan Ramen (West 40th St) is praised for its rich tonkotsu broth...
```

### Step 2: Handling groundingMetadata for Attribution

Displaying source attribution is **mandatory under Google's terms of service**. Here's a clean implementation.

```python
# attribution_handler.py
from dataclasses import dataclass
from typing import Optional

@dataclass
class MapSource:
    """Represents a single Google Maps source citation."""
    title: str
    uri: str
    place_id: Optional[str] = None

def extract_map_sources(response) -> list[MapSource]:
    """
    Extract Maps source citations from a grounded response.

    Returns:
        List of MapSource objects (empty list if not grounded)
    """
    sources = []
    if not hasattr(response, "candidates") or not response.candidates:
        return sources

    candidate = response.candidates[0]
    metadata = getattr(candidate, "grounding_metadata", None)
    if not metadata:
        return sources

    for chunk in getattr(metadata, "grounding_chunks", []):
        web = getattr(chunk, "web", None)
        if web:
            sources.append(
                MapSource(
                    title=getattr(web, "title", ""),
                    uri=getattr(web, "uri", ""),
                    place_id=getattr(web, "place_id", None),
                )
            )
    return sources

def format_attribution_html(sources: list[MapSource]) -> str:
    """
    Generate attribution HTML compliant with Google's style guidelines.
    (Roboto font, 12-16sp, displayed within one user interaction)
    """
    if not sources:
        return ""

    items = "".join(
        f'<li><a href="{s.uri}" target="_blank" rel="noopener">{s.title}</a></li>'
        for s in sources
    )
    return f"""
<div class="maps-attribution"
     style="font-family: 'Roboto', sans-serif; font-size: 13px;
            color: #5f6368; margin-top: 12px;">
  <strong>Sources (Google Maps):</strong>
  <ul style="margin: 4px 0; padding-left: 16px;">{items}</ul>
</div>
"""

# Usage
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="Top 3 highly-rated coffee shops near Shibuya station, walkable distance",
    config=types.GenerateContentConfig(
        tools=[types.Tool(google_maps=types.GoogleMaps())],
        tool_config=types.ToolConfig(
            retrieval_config=types.RetrievalConfig(
                lat_lng=types.LatLng(latitude=35.6580, longitude=139.7016),
                language_code="en_US",
            )
        ),
    ),
)

print(response.text)
sources = extract_map_sources(response)
print(format_attribution_html(sources))
```

### Step 3: Production Restaurant Search API

A FastAPI endpoint ready for production deployment.

```python
# restaurant_api.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field
from google import genai
from google.genai import types

app = FastAPI(title="Location-Aware Restaurant AI")
client = genai.Client(http_options=types.HttpOptions(api_version="v1"))

class SearchRequest(BaseModel):
    query: str = Field(..., example="Best lunch spots under $15")
    latitude: float = Field(..., ge=-90, le=90)
    longitude: float = Field(..., ge=-180, le=180)
    language: str = Field(default="en_US")

class SearchResponse(BaseModel):
    text: str
    sources: list[dict]
    grounded: bool

@app.post("/search", response_model=SearchResponse)
async def search_restaurants(req: SearchRequest):
    """
    AI-powered restaurant search endpoint.
    Uses real-time Google Maps data — no hallucinations.
    """
    try:
        response = client.models.generate_content(
            model="gemini-2.5-flash",
            contents=req.query,
            config=types.GenerateContentConfig(
                system_instruction=(
                    "You are a local dining expert. "
                    "Use the user's location to recommend specific restaurants "
                    "with names, hours, price range, and what they're known for."
                ),
                tools=[types.Tool(google_maps=types.GoogleMaps())],
                tool_config=types.ToolConfig(
                    retrieval_config=types.RetrievalConfig(
                        lat_lng=types.LatLng(
                            latitude=req.latitude,
                            longitude=req.longitude,
                        ),
                        language_code=req.language,
                    )
                ),
                max_output_tokens=1024,
                temperature=0.3,
            ),
        )

        sources = []
        if (
            response.candidates
            and response.candidates[0].grounding_metadata
        ):
            for chunk in (
                response.candidates[0].grounding_metadata.grounding_chunks or []
            ):
                if hasattr(chunk, "web"):
                    sources.append({
                        "title": chunk.web.title,
                        "uri": chunk.web.uri,
                    })

        return SearchResponse(
            text=response.text,
            sources=sources,
            grounded=len(sources) > 0,
        )

    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

# Run: uvicorn restaurant_api:app --reload
```

## Advanced Patterns

### Pattern 1: Real Estate Neighborhood Report

```python
# real_estate_ai.py
"""
AI-generated neighborhood report for a property listing.
Covers transit, groceries, medical facilities, and schools.
"""

SYSTEM_PROMPT = """
You are a real estate advisor. Based on Google Maps data,
provide an accurate neighborhood analysis covering:
- Nearest transit (station name + walking time)
- Grocery stores and convenience stores (within 10 min walk)
- Medical facilities and clinics (within 15 min walk)
- Schools and childcare facilities
Be specific: use real place names and distances.
"""

def generate_neighborhood_report(
    address: str,
    latitude: float,
    longitude: float,
) -> dict:
    """Generate a living-convenience report for a property."""
    query = (
        f"Analyze the neighborhood around '{address}'. "
        "Include transit access, grocery stores, medical facilities, "
        "and schools within walking distance."
    )

    response = client.models.generate_content(
        model="gemini-2.5-pro",   # Use Pro for higher accuracy in real estate
        contents=query,
        config=types.GenerateContentConfig(
            system_instruction=SYSTEM_PROMPT,
            tools=[types.Tool(google_maps=types.GoogleMaps())],
            tool_config=types.ToolConfig(
                retrieval_config=types.RetrievalConfig(
                    lat_lng=types.LatLng(latitude=latitude, longitude=longitude),
                    language_code="en_US",
                )
            ),
            temperature=0.1,
        ),
    )

    sources = []
    grounded = False
    if response.candidates and response.candidates[0].grounding_metadata:
        chunks = response.candidates[0].grounding_metadata.grounding_chunks or []
        sources = [
            {"title": c.web.title, "uri": c.web.uri}
            for c in chunks if hasattr(c, "web")
        ]
        grounded = len(sources) > 0

    return {"report": response.text, "sources": sources, "grounded": grounded}

# Example
result = generate_neighborhood_report(
    address="350 5th Ave, New York, NY 10118",  # Empire State Building
    latitude=40.7484,
    longitude=-73.9967,
)
print(result["report"])
# Expected output:
# The area around 350 5th Ave is extremely well-served for daily needs.
# Transit: Multiple subway lines (B/D/F/M/N/Q/R/W) at 34th St–Herald Sq, 2 min walk...
```

## Troubleshooting

### Common Errors and Fixes

#### Error 1: `PERMISSION_DENIED` / API not enabled

```
google.api_core.exceptions.PermissionDenied: 403 Permission denied
```

**Cause**: Vertex AI API is not enabled, or Maps Grounding enrollment is pending.

```bash
gcloud services enable aiplatform.googleapis.com
gcloud services list --enabled | grep aiplatform
```

#### Error 2: Maps tool not recognized / no grounding

**Cause**: Using API key authentication instead of Vertex AI.

```python
# ❌ Wrong — API key auth does NOT support Maps Grounding
client = genai.Client(api_key="your-api-key")

# ✅ Correct — Vertex AI auth required
import os
os.environ["GOOGLE_GENAI_USE_VERTEXAI"] = "True"
os.environ["GOOGLE_CLOUD_PROJECT"] = "your-project-id"
os.environ["GOOGLE_CLOUD_LOCATION"] = "global"
client = genai.Client(http_options=types.HttpOptions(api_version="v1"))
```

#### Error 3: `grounded=False` — Grounding not triggered

**Cause**: The query doesn't signal location intent clearly enough.

```python
# ❌ Unlikely to trigger grounding
"What is AI?"

# ✅ Clearly location-relevant queries
"Coffee shops open right now near me with good WiFi"
"24-hour pharmacies within walking distance"
"Best sushi restaurants with parking nearby"
```

#### Error 4: Rate limit / 429 errors — Exponential backoff

```python
# retry_handler.py
import time, random
from functools import wraps

def with_retry(max_attempts=3, base_delay=1.0):
    """Exponential backoff retry decorator."""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if "429" in str(e) and attempt < max_attempts - 1:
                        delay = base_delay * (2 ** attempt) + random.uniform(0, 1)
                        print(f"Rate limited. Retrying in {delay:.1f}s ({attempt+1}/{max_attempts})")
                        time.sleep(delay)
                    else:
                        raise
        return wrapper
    return decorator

@with_retry(max_attempts=3, base_delay=2.0)
def search_with_retry(query: str, lat: float, lng: float):
    return client.models.generate_content(
        model="gemini-2.5-flash",
        contents=query,
        config=types.GenerateContentConfig(
            tools=[types.Tool(google_maps=types.GoogleMaps())],
            tool_config=types.ToolConfig(
                retrieval_config=types.RetrievalConfig(
                    lat_lng=types.LatLng(latitude=lat, longitude=lng),
                )
            ),
        ),
    )
```

Key points:
- You're only charged when the response includes at least one Maps source
- If Gemini answers without triggering grounding, there's no Maps charge

## FAQ

**Q1. I'm getting "google_maps tool not recognized" — what's wrong?**

A1. This almost always means you're authenticating with a standard API key (`GOOGLE_API_KEY`) rather than Vertex AI credentials. Set `GOOGLE_GENAI_USE_VERTEXAI=True` and use Application Default Credentials. See the setup section above.

**Q2. How is Maps Grounding different from Google Search Grounding?**

A2. Google Search Grounding retrieves general web content, while Maps Grounding specifically taps into Google Maps' place database — business hours, ratings, addresses, phone numbers. For location-intent queries ("find me X near Y"), Maps Grounding gives dramatically more accurate and useful results. See also our [Grounding with Google Search guide](/en/articles/gemini-advanced/grounding-and-search).

**Q3. Is attribution display really required? What happens if I skip it?**

A3. Yes, **attribution is mandatory** per Google's terms of service. Whenever the response contains Maps sources, you must display the source URI and title in a way the user can see within one interaction. Skipping this can result in API access suspension. The `format_attribution_html()` helper in Step 2 handles this for you.

**Q4. Does the language of the query affect grounding quality?**

A4. Set the `language_code` parameter to match your users' language (e.g., `"en_US"` for English, `"ja"` for Japanese). Gemini will attempt to return Maps data in the specified language. When querying non-English sources in English, some mixing may occur — this is normal.

**Q5. Gemini 3 doesn't support Maps Grounding — when will it?**

A5. As of March 2026, Gemini 3 does not support Maps Grounding. Gemini 2.5 Flash and Pro are the recommended models. Monitor the [Vertex AI release notes](https://cloud.google.com/vertex-ai/generative-ai/docs/release-notes) for updates.

**Q6. How should I handle restricted territories (China, North Korea, etc.)?**

A6. Maps Grounding returns `INVALID_ARGUMENT: Grounding with Google Maps is not available in your region` for restricted areas. In global apps, detect the user's country from their coordinates and fall back to standard Gemini (without Maps grounding) when they're in a restricted territory.

## Summary

Grounding with Google Maps eliminates one of the biggest pain points of production AI apps: location hallucinations. By combining Gemini's language understanding with Google Maps' real-time place data, you can build restaurant finders, real estate tools, and travel assistants that actually know what's open, where it is, and what users think of it.

Key takeaways:

1. **Vertex AI is required** — standard API key auth won't work
2. **Pass `lat_lng`** to `RetrievalConfig` for personalized, location-specific answers
3. **Attribution is not optional** — always render `groundingMetadata` sources for your users
4. **Cost management** — use Flash over Pro, keep `max_output_tokens` reasonable, and stay within the 500-request free tier for development
5. **Error handling** — implement exponential backoff for rate limits and a non-grounding fallback for restricted regions
