## A New Era for Multi-Model AI Agents

NVIDIA's NemoClaw, announced at GTC 2026, makes it straightforward to build AI agents that aren't locked into a single model. Instead of relying solely on Gemini API, you can architect workflows that dynamically route tasks between Gemini and Nemotron based on what each does best.

At the heart of NemoClaw is **OpenShell**, an open-source runtime that lets you declaratively manage which models can access which resources through YAML policies. Process images and video with Gemini's multimodal API, run fast local inference with NVIDIA's Nemotron models, and govern each model's permissions independently — all from a single platform.

This article walks through building a multi-model agent on NemoClaw with practical code examples.

## Gemini vs. Nemotron — Playing to Each Model's Strengths

The key to effective multi-model agents is smart task routing.

### Where Gemini Excels

Gemini 2.5 Pro runs on Google's cloud infrastructure and shines at multimodal processing (text, images, video, audio), long context windows, and complex reasoning. It's the go-to choice when you need web-grounded answers or large document analysis.

### Where Nemotron Excels

NVIDIA's Nemotron models run locally on RTX GPUs with low latency and zero network dependency. They're ideal for processing sensitive data, offline environments, and high-volume repetitive tasks where API costs add up. NemoClaw automatically applies security policies to Nemotron workloads.

### Routing Guidelines

| Task | Recommended Model | Why |
|------|-------------------|-----|
| Image/video analysis | Gemini 2.5 Pro | Superior multimodal processing |
| Sensitive data processing | Nemotron (local) | Data never leaves your machine |
| Real-time web search | Gemini 2.5 Pro | Grounding feature for up-to-date info |
| High-volume batch work | Nemotron (local) | No API costs |
| Complex reasoning | Gemini 2.5 Pro | Larger context window |

## Building a Multi-Model Agent on NemoClaw

### Step 1: Environment Setup

```bash
# Install NemoClaw
git clone https://github.com/NVIDIA/NemoClaw.git
cd NemoClaw
./setup.sh --with-openshell --with-nemotron

# Configure your Gemini API key
export GEMINI_API_KEY="your-api-key-here"

# Verify the setup
nemoclaw status
# Expected output:
# NemoClaw v1.0.0
# OpenShell: active
# Nemotron: loaded (nemotron-mini on RTX 4090)
# External APIs: Gemini API configured
```

### Step 2: Create a Multi-Model Policy

```yaml
# multi-model-policy.yaml
# Security policy for Gemini + Nemotron multi-model environment
version: "1.0"
name: "gemini-nemotron-agent"

models:
  gemini:
    # Gemini API access configuration
    api_endpoint: "generativelanguage.googleapis.com"
    allowed_capabilities:
      - "text_generation"
      - "multimodal_analysis"
      - "grounding"          # Web-grounded responses
    max_tokens_per_request: 8192
    rate_limit: "60/minute"

  nemotron:
    # Local inference configuration
    runtime: "local"
    gpu: "auto"
    allowed_capabilities:
      - "text_generation"
      - "code_completion"
    max_tokens_per_request: 4096

network:
  allowed_domains:
    - "generativelanguage.googleapis.com"
    - "www.googleapis.com"
  default: "deny"

filesystem:
  read_write:
    - "/workspace/data"
    - "/tmp/agent-cache"
  default: "deny"
```

### Step 3: Agent Orchestration Code

```python
# multi_model_agent.py
# Multi-model agent running on NemoClaw
import google.generativeai as genai
import json

# Initialize Gemini model
genai.configure(api_key="YOUR_API_KEY")
gemini_model = genai.GenerativeModel("gemini-2.5-pro")

class MultiModelAgent:
    """Agent that routes tasks between Gemini and Nemotron"""

    def __init__(self):
        self.gemini = gemini_model
        # Nemotron is automatically routed via nemoclaw
        self.task_history = []

    def route_task(self, task: dict) -> str:
        """Route to the best model based on task requirements"""
        if task.get("requires_multimodal"):
            return "gemini"
        if task.get("contains_sensitive_data"):
            return "nemotron"  # Keep data local
        if task.get("requires_web_search"):
            return "gemini"  # Use Grounding feature
        # Default to cost-efficient Nemotron
        return "nemotron"

    def process_with_gemini(self, prompt: str, image=None) -> str:
        """Process a task using Gemini API"""
        if image:
            # Multimodal processing (e.g., image analysis)
            response = self.gemini.generate_content([prompt, image])
        else:
            response = self.gemini.generate_content(prompt)
        return response.text

    def process_with_nemotron(self, prompt: str) -> str:
        """Process a task locally with Nemotron
        OpenShell policies are automatically enforced via NemoClaw"""
        import subprocess
        result = subprocess.run(
            ["nemoclaw", "infer", "--model", "nemotron-mini",
             "--prompt", prompt],
            capture_output=True, text=True
        )
        return result.stdout.strip()

    def execute(self, task: dict) -> dict:
        """Execute a task and return results"""
        model = self.route_task(task)
        prompt = task["prompt"]

        if model == "gemini":
            result = self.process_with_gemini(
                prompt, task.get("image")
            )
        else:
            result = self.process_with_nemotron(prompt)

        self.task_history.append({
            "model": model,
            "task": task["prompt"][:50],
            "status": "success"
        })

        return {"model_used": model, "result": result}


# Usage examples
agent = MultiModelAgent()

# Task 1: Analyze an image (→ routed to Gemini)
task1 = {
    "prompt": "Analyze the product shown in this image",
    "requires_multimodal": True,
    "image": "product_photo.jpg"
}
result1 = agent.execute(task1)
print(f"Model: {result1['model_used']}")
# Output: Model: gemini

# Task 2: Summarize confidential docs (→ routed to Nemotron)
task2 = {
    "prompt": "Summarize this internal document: ...",
    "contains_sensitive_data": True
}
result2 = agent.execute(task2)
print(f"Model: {result2['model_used']}")
# Output: Model: nemotron
```

## Practical Use Cases

### Content Production Pipeline

Use Gemini 2.5 Pro's Grounding feature for trend research during the planning phase, generate article drafts with cost-efficient Nemotron, and loop back to Gemini for final quality checks and image captioning. This division of labor happens automatically through the routing logic.

### Data Analytics Dashboard

Analyze sensitive internal data locally with Nemotron, then send anonymized results to Gemini for comparison with market data and report generation. Since raw data never leaves your machine, this architecture simplifies data governance compliance.

## Frequently Asked Questions

### Q: What GPU do I need for Nemotron?

A: Nemotron-mini runs on RTX 3060 (12GB VRAM) or higher. Larger Nemotron models require RTX 4090 or DGX systems. NemoClaw's setup script auto-detects available GPUs and selects the appropriate model.

### Q: Does NemoClaw change my Gemini API costs?

A: NemoClaw itself adds no API fees. Standard Google AI Studio pricing applies for Gemini API calls. However, by routing suitable tasks to local Nemotron inference, you reduce your Gemini API call volume and optimize overall costs.

### Q: Can I set different permissions per model in OpenShell?

A: Yes — this is one of the biggest advantages of multi-model setups. For example, you can allow Gemini network access but block file writes, while giving Nemotron full file read/write but zero network access.

### Q: Can I add NemoClaw to an existing Gemini API project?

A: Absolutely. Your existing Python code stays largely unchanged. Just wrap your script execution with `nemoclaw run` to apply OpenShell security policies. Incremental adoption is fully supported.
